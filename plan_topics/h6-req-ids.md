**Spec.** [`../spec.md` Appendix — REQ-ID prefix table](../spec.md), [`lexical.md`](../spec_topics/lexical.md), [`type-system.md`](../spec_topics/type-system.md), [`schemas.md`](../spec_topics/schemas.md), [`descriptions.md`](../spec_topics/descriptions.md), [`schema-subset.md`](../spec_topics/schema-subset.md), [`frontmatter.md`](../spec_topics/frontmatter.md), [`query.md`](../spec_topics/query.md), [`expressions.md`](../spec_topics/expressions.md), [`bindings.md`](../spec_topics/bindings.md), [`control-flow.md`](../spec_topics/control-flow.md), [`errors-and-results.md`](../spec_topics/errors-and-results.md), [`return.md`](../spec_topics/return.md), [`functions.md`](../spec_topics/functions.md), [`tool-calls.md`](../spec_topics/tool-calls.md), [`invocation.md`](../spec_topics/invocation.md), [`imports.md`](../spec_topics/imports.md), [`discovery.md`](../spec_topics/discovery.md), [`slash-invocation.md`](../spec_topics/slash-invocation.md), [`binder.md`](../spec_topics/binder.md), [`cancellation.md`](../spec_topics/cancellation.md), [`diagnostics.md`](../spec_topics/diagnostics.md), [`runtime-value-model.md`](../spec_topics/runtime-value-model.md), [`pi-integration-contract.md`](../spec_topics/pi-integration-contract.md), [`implementation-notes.md`](../spec_topics/implementation-notes.md), [`pi-integration.md`](../spec_topics/pi-integration.md), [`grammar.md`](../spec_topics/grammar.md).

**Adds.** A one-time anchor pass over every non-narrative spec page listed in the [REQ-ID prefix table](../spec.md). For each normative obligation in each page, insert a `**PREFIX-N.**` marker (or, where in-line bolding is unsuitable, an `<a id="prefix-n"></a>` HTML anchor) using the page's stable prefix from the table; numbering starts at `1` per page and is dense at first numbering (immutability per the cross-cutting REQ-ID discipline takes effect from this leaf onward). Re-pivot [`coverage-matrix.md`](coverage-matrix.md) so its rows are keyed per REQ-ID rather than per spec section: each existing section-keyed row is expanded into one row per REQ-ID introduced by that section, carrying forward the same closing-leaf set (or a tighter subset where a section's REQ-IDs map to distinct leaves). Pure-narrative pages (`overview.md`, `glossary.md`, `influences.md`, `comparison.md`, `related-work.md`, `future-considerations.md` — marked `(no IDs — narrative)` in the prefix table) are not visited and contribute no rows. The leaf does not add or change any normative spec text; it only annotates existing rules with stable identifiers. The `BIND` / `BNDG` collision call-out in the prefix table is honoured: `bindings.md` numbers `BNDG-N`, `binder.md` numbers `BIND-N`.

**Tests.**
- Every non-narrative spec page (the union of pages in the prefix table whose Prefix column is not `(no IDs — narrative)`) contains at least one REQ-ID marker matching its assigned prefix; the test computes the prefix set from the appendix table itself (not a hard-coded literal) and asserts the per-page `grep -ohE '\b(<PREFIX-UNION>)-[0-9]+\b'` count is `≥ 1`.
- Pure-narrative pages contain zero REQ-ID markers (negative assertion against the same prefix union).
- `coverage-matrix.md` contains one row per REQ-ID emitted by any non-narrative spec page; every REQ-ID grepped from `spec_topics/*.md` appears as a row key in the matrix (the V18s `comm -23` diff is empty against current spec text once H6 closes).
- REQ-ID density at first numbering is dense per page (each page numbers `PREFIX-1, PREFIX-2, …, PREFIX-K` with no holes); the test grepped per-page for `\bPREFIX-[0-9]+\b`, sorts numerically, and asserts the sequence equals `1..K`.
- `bindings.md` numbers `BNDG-N` only (no `BIND-N` markers); `binder.md` numbers `BIND-N` only (no `BNDG-N` markers).

**Deps.** H4. (H6 does not depend on the Pi e2e harness, but it must land before V18s can enforce a non-vacuous gate, and it must land before M so that every leaf from M onward cites real REQ-IDs at first write per the leaf-format convention.)

**Ships when.** Every non-narrative spec page carries `PREFIX-N` markers, `coverage-matrix.md` is keyed per REQ-ID, and the V18s coverage-gate `comm -23` diff over current spec content returns empty.
