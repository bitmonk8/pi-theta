## Finding

The *Citation-token grammar* paragraph and the *Generated-output-in-src limitation* paragraph point "below"/"there" to the *Stale-citation limitation* and *Substitution limitation* sub-bullets, which live on `audit-failures.md` under the *Stale-marker discriminator* — only the discriminator's heading remains on `audit-resolution.md`.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** e5cd514 — pi-loom spec: resolve "T35 SDK capability inventory closed-set negative-direction audit" (2026-05-09, Thomas Andersen); f5e89f4 — docs: enforce 100KB hard / 30KB recommended size cap on spec set (2026-06-04, Thomas Andersen)
**History:** e5cd514 authored the inventory-closure-audit prose inside the single monolithic `docs/spec_topics/pi-integration-contract.md`, where *Citation-token grammar*, the *Stale-marker discriminator* sub-bullets, the seven-capability enumeration, *Wire serialisation*, and *Failure-surface contract* were all contiguous, so the "below"/"above"/"there" directional words and the "clause of the *Failure-surface contract*" nesting all resolved correctly within one page. f5e89f4 enforced a size cap by splitting that 967-line page into sibling shard files (`audit-resolution.md`, `audit-failures.md`, `audit-wire-and-canary.md`, `inventory-audit-intro.md`, `capability-inventory-items.md`, …) but carried the intra-page directional language over verbatim, turning each reference into a broken cross-file pointer. A later cross-ref sweep (23d28a1, F-0958) realigned link basenames on `inventory-audit-intro.md` but did not rewrite these directional words.

## Solution Space

**Shape:** single

### Recommendation

Replace the "below"/"there" qualifiers with cross-file links to the *Stale-citation limitation* and *Substitution limitation* sub-bullets on `audit-failures.md`; since those two sub-bullets currently carry no `<a id>` anchors, add anchors to them on `audit-failures.md` so the links target the precise sub-bullets rather than the page.

## Consequence

**Severity:** advisory

A reader landing on any one of the three shard files and following its intra-page directional words ("below"/"above"/"there") or the "clause of the *Failure-surface contract*" nesting cannot locate the referenced material in the named file, because it now lives in a sibling shard with no link. The contradictory below/above directions on `inventory-audit-intro.md` and the false parent-clause nesting on `audit-wire-and-canary.md` actively misdirect. Implementers can still build a correct audit (the V18a/V18b leaves open all shard files together), so this is a navigation/guidance gap rather than a correctness divergence.
