## Finding

The always-log set on `runtime-event-channel.md` partitions by the `details`
payload shape that carries each failure: members routed through
`details: { event: RuntimeEvent }` versus members routed through
`details: { diagnostics: Diagnostic[] }`. The spec names these two halves
"group A" and "group B" — ordinal labels that encode nothing about the
distinguishing criterion (which `details` variant routes the member). A reader
who encounters a back-reference such as "applies to group A only", "the group-A
[Binder — Failure modes] rows", or "the cascade-twin dedup key does not apply to
group B" must navigate back to the partition definition to recover which
routing shape is meant, every time.

The labels are also load-bearing across multiple pages: the glossary
`always-log set` and `occurrence`/`origin` entries, `diagnostics/diagnostic-shape.md`
("for group-A members of the always-log set"), `errors-and-results/error-model.md`
("PIC group B"), and `session-shutdown-semantics.md` ("a `details: { event: RuntimeEvent }`
group-A emission") all carry the ordinal labels forward, so the navigation cost
is paid corpus-wide, not just on the owning page.

Replacing the ordinal labels with descriptive ones that name the routing channel
("event-routed" / "diagnostic-routed") makes every back-reference
self-explanatory at the point of use and removes the lookup. No behaviour
changes; the partition membership and routing rules are unaffected.

## Solution Space

**Shape:** single

### Recommendation

Rename the two partition halves to descriptive labels that name the routing
channel: **event-routed** (members emitting `details: { event: RuntimeEvent }`)
and **diagnostic-routed** (members emitting `details: { diagnostics: Diagnostic[] }`).

Apply the rename at every site in lockstep so no page mixes conventions:

- `runtime-event-channel.md` — the partition sentence, the two enumeration list
  headers ("Group A — …" / "Group B — …"), and every back-reference ("applies to
  group A only", "no analogue for group B", "all `QueryError` kinds (group A) or
  runtime panics (group B)", the group-A binder-failure sourcing paragraph, the
  group-A `Clock.wallNow()`/`RuntimeEvent`-construction fallback references).
- `glossary.md` — the `always-log set` entry ("group A (routed via …) / group B
  (routed via …)") and the `occurrence`/`origin` entry ("for group-A events").
- `diagnostic-shape.md` ("for group-A members"), `error-model.md` ("PIC group B"),
  and `session-shutdown-semantics.md` ("group-A emission").

Keep the existing `details`-shape phrasing alongside the new labels on first use
(e.g. "event-routed members emit `details: { event: RuntimeEvent }`") so the
label and its meaning are co-located; downstream back-references then stand
alone without a lookup.

## Consequence

**Severity:** cosmetic

Pure navigation/readability cost: every cross-reference to "group A"/"group B"
forces a round-trip to the partition definition to recover which `details`
shape is meant. No implementer divergence and no behavioural ambiguity — the
routing rules are fully pinned regardless of the label chosen.
