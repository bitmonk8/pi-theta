## Finding

`V9i` bundles `PIC-22` — the N≥2 parallel-spawn concurrency path that dispatches `createAgentSession` for all N callables before any returns via `Promise.all` — into the single-session lifecycle. `PIC-22` has its own blocking-fake `AgentSession` harness and ships/tests independently of the single-session spawn/dispose path.

## Consequence

**Severity:** advisory

Implementing all eight-plus obligations in one leaf produces an oversized single commit that no reviewer can hold in scope at once, degrading reviewability and inflating the diff surface. The leaf still ships a working subagent path, so pickability and spec-conformance are not blocked; the cost is structural — atomicity and the per-finding fix loop's convergence.

Strike `PIC-22` from the `V9i.Adds` `PIC` anchor enumeration. Retarget the `PIC-22` mapping in `coverage-matrix.md` from the `V9i` row to a new row owned by the new leaf. Link the new implementation leaf into the V9 leaf list in `docs/plan.md`. `subagent.md`'s `PIC-22` anchor is read-only for this fix.
