## Finding

H4b's first Tests bullet asserts that the response-programming surface "drives the double **deterministically** — replaying the same scripted assistant turns, `tool_use` results, binder responses/failures, `tool_loop.max_rounds` round-exhaustion, invoke-child completions, subagent-mode callee dispatch, and cancellation-injection points yields the same observable transcript **on every run**." The phrase "on every run" states a universal property over an unbounded run space.

The self-check that backs the claim is a finite replay: it runs the scripted sequence some bounded number of times and compares transcripts. Such a check can only witness "identical across the replays it actually performed." It cannot observe low-probability ordering variance — e.g. a race in the double's emission scheduling that surfaces only under interleavings the scripted sequence never exercises. The acceptance language therefore promises a guarantee strictly stronger than the gate delivers, and the same overclaim is mirrored in the Ships-when phrase "the determinism gate."

This is a wording-precision defect, not a behavioural one: the self-check itself is well-formed. Narrowing the claim to the replays the self-check performs keeps the assertion honest about what passing the gate establishes.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H4b-response-programming-surface.md`, edit the first Tests bullet so the reproducibility claim is scoped to the replays the self-check actually performs. Replace the trailing clause "yields the same observable transcript **on every run**" with "yields the same observable transcript across the self-check's replays" (equivalently "on repeated replay"). The leading "drives the double **deterministically**" may stand, since determinism scoped to the replayed sequence is the intended property; the corrected trailing clause supplies that scope.

For consistency, the Ships-when reference to "the determinism gate" needs no rename — it names the gate, not the over-broad property — but if the implementer prefers, it can read "the repeated-replay determinism gate" to match the narrowed Tests wording. The substantive edit is the Tests-bullet clause.

## Consequence

**Severity:** advisory

An implementer reading "deterministically … on every run" may believe the self-check proves universal reproducibility and either over-engineer the gate to chase an unbounded property or trust a guarantee the finite replay never establishes. No downstream leaf is blocked and no spec rule ships unimplemented; the cost is a misleading statement of what passing H4b's gate certifies.
