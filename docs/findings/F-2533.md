## Finding

The non-goal bullet buries its single normative sentence (loom 1.0 presupposes one active user session and defines no new disposition for an unanticipated second session) under a ~600-word detection-gap analysis that constrains no implementer.

**Spec edits:** In `model-changes-and-non-goals.md`, rewrite the "No concurrent user sessions in the same host process" bullet down to: (1) the non-goal statement; (2) the single load-bearing invariant — every site loom 1.0 scopes to one active user session remains so scoped, and the runtime defines no new disposition (no refuse-to-load, no bind-to-first-session, no per-session registry keying, no host-incompatibility diagnostic) for a second session; (3) the existing `*Recorded at:*` line. Move the detection-gap walkthrough (typecheck vs surface-inventory vs editorial-review, `session_shutdown` closed-reason validator behaviour, the `pi.openParallelSession` hypothetical) to an ADR or non-normative provenance note outside the normative bullet, or delete it as recoverable from the cited bump-procedure pages.

**Relationship:** same-cluster — resolves independently in `model-changes-and-non-goals.md`.
