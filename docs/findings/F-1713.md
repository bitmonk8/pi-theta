## Problem

The `RuntimeEvent` payload's `query_site` and `discard_site` source-location fields (`{ file, line, column }`) state *what site* each denotes but never *where* the `file`/`line`/`column` values come from, nor what indexing convention applies. Every sibling field has a stated provenance — `occurred_at` via `Clock.wallNow()`, `loom`/`invocation_id` via the `ActiveInvocationRegistry` per **Binder-failure sourcing** — but these two are left to implementer invention. Two reasonable implementers can diverge: one reads a 1-indexed parse-time span from the originating IR node (the diagnostic span model), another reconstructs positions from a host stack trace, yielding a different index base, a transpiled-host file, or unstable coordinates. Because `query_site` participates in the dedup tuple `(kind, query_site, message, occurred_at)`, inconsistent or unstable positions break occurrence identity and cross-tool correlation.

## Issue introduction

**Verdict:** present-since-inception (of the field)
**Introducing commits:** `b3bc4ce` ("spec: pin failure-observability surface (Cluster A, Option C)", 2026-05-04)
**History:** `git log -S 'query_site'` traces the field to `b3bc4ce`, which first pinned the `RuntimeEvent` payload shape. From that commit `query_site` carried only a semantic comment ("source location of the @-template, tool call, invoke, or discarding `let _ =`") with no producing contract for the `file`/`line`/`column` values and no indexing convention. The `discard_site` field was later split out as a distinct field (`6afca02`, `13e58fe`) but inherited the same omission. The gap survived the 100KB file-split (`f5e89f4`) into `runtime-event-channel.md` and the subsequent sibling-sourcing fixes that pinned every *other* `RuntimeEvent` field's provenance — `loom` / `invocation_id` via the `ActiveInvocationRegistry` (`20f9645`, "Binder-failure RuntimeEvents have no pinned source for invocation_id / loom") and `occurred_at` via `Clock.wallNow()` — none of which extended to `query_site` / `discard_site`. The defect is an omission introduced with the feature and never closed.

## Solution approach

Add a producing contract for `query_site` and `discard_site` beside the `RuntimeEvent` payload shape in `runtime-event-channel.md`, mirroring the adjacent **Binder-failure sourcing** paragraph. Source both fields from the parse-time source span the parse pipeline attaches to the originating IR node — the `@`-template / tool call / `invoke` for `query_site`, the discarding `let _ =` / void-tail expression for `discard_site` — read from the captured span rather than reconstructed from runtime coordinates. Pin `line` / `column` as 1-indexed logical positions on the BOM-stripped, newline-normalised stream, cross-referencing `lexical.md` **Diagnostic spans** and `diagnostic-shape.md` `range`.
