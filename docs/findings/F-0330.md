## Problem

The `H2a` blocking-runtime check (the `no-restricted-syntax` rule flagging `fs.readFileSync`, `execSync`, and the `*Sync` family over `src/**`) excludes "the `V8*` FileSystem seam-adapter modules that legitimately own file/process I/O" in prose, with no path, glob, or in-source marker the lint rule can mechanically recognise. The leaf is sequenced at `Deps: H1a` and lands before any `V8*` module exists, so the exclusion cannot enumerate concrete file names either, forcing the `H2a` implementer to invent a module-identification discriminator that may not match where `V8b` ultimately places its `FileSystem` adapter. The check already carries a per-call `// allow-sync: <reason>` comment allow-list, yet additionally asserts this wholesale module-level exclusion that has no discriminator, leaving the gate's module-level arm undefined.

**History:** `git log -S` / `git show` on `docs/plan_topics/H2a-cross-cutting-gates.md` show the entire blocking-runtime `*Sync` arm — including the `// allow-sync:` comment allow-list and the wholesale "excluding … the `V8*` FileSystem seam-adapter modules that legitimately own file/process I/O" clause — was added fresh in `c33d309`; the prior revision of `H2a` carried no blocking-runtime / sync / seam content. The defect entered the corpus with the arm itself: the wholesale module-level exclusion shipped without a path/glob/marker discriminator from inception. The matching prose in `conventions.md` (*Sequential by default* rule, self-review step, residue-inspection item 3) was added in the same commit.

## Solution approach

Delete the wholesale `V8*` FileSystem seam-adapter module exclusion from the `H2a` blocking-runtime `*Sync` check — both the `Adds` clause and the matching `Tests` bullet — leaving the per-call `// allow-sync: <reason>` comment allow-list as the sole seam-adapter exemption mechanism, mirroring how `H3a` resolves the analogous "which modules are seam adapters" problem with its per-site `// allow-ambient:` comment allow-list. Keep the `**/*.test.ts` glob exclusion in place. Reconcile the matching prose in `conventions.md` (*Sequential by default* rule, self-review step, residue-inspection item 3) so it no longer asserts the dropped module-level exclusion.
