## Finding

`V9i`'s ERR-8 witness — that a mid-stream cancellation inside the real subagent `AgentSession` leaves that session's committed turns unmutated — is deliberately routed off `npm test`. Both the V9i `Tests.` `ERR-8` bullet and the V9i `Ships when.` line state the outcome is a "real-host-only behaviour … not asserted under `npm test`" and witnessed only at the manual real-host smoke gate criterion (c). The deferral itself is justified: there is no offline driver that feeds a real `createAgentSession` a scripted cancellable stream, so an offline assertion is not currently constructible.

The residual defect is that criterion (c) — now ERR-8's sole verification surface — does not state a reproducible procedure. Its text ("cancellation propagation is observed at the injected cancel point, and a mid-stream cancellation inside the subagent-mode `AgentSession` the fixture spawns leaves that session's committed turns unmutated") names the observable but not how a human operator reproduces it: which fixture construct triggers the subagent spawn, how and at what point the mid-stream cancel is injected, and how the operator captures the committed-turn set before and after the cancel to assert it is unchanged. With the procedure underspecified, ERR-8 degrades to a "check it looks right" judgement on the one surface that is supposed to verify it.

The reproducibility of criterion (c) is further conditioned on H7a's committed fixture actually exercising a subagent spawn with an injected cancel point and committed turns — a fixture-content requirement that is itself not pinned at the H7a leaf (tracked by a separate finding).

## Consequence

**Severity:** advisory

ERR-8 (no committed-turn mutation under mid-stream subagent cancellation) is a correctness invariant whose only verification surface is manual smoke criterion (c). Because criterion (c) states the observable but not a reproducible procedure, two smoke operators could score it differently and a real ERR-8 violation could pass the gate, shipping the invariant effectively unverified. The leaf can still ship — the gap is in the verification guidance, not in a build break.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/real-host-smoke-gate.md`, expand criterion (c) so the committed-turn-no-mutation check is a reproducible procedure rather than a bare observable. State inline, alongside the existing "leaves that session's committed turns unmutated" wording: (1) which fixture construct triggers the subagent spawn — the subagent-mode `.loom` callable in H7a's committed multi-feature fixture; (2) the cancel-injection mechanism and the point at which it fires — a mid-stream cancel delivered before the subagent session's in-flight turn commits; and (3) how the operator captures the subagent session's committed-turn set immediately before and after the cancel and asserts the two sets are equal.

Edge cases the implementer must watch: this manual criterion is ERR-8's only verification surface, so the procedure must be executable by a human operator against a live Pi host (the documented reason `F-1363` relocated the witness here is that no offline driver feeds a real `createAgentSession` a scripted cancellable stream). If the `H4a`/`H4c` harness is ever extended to script a cancellable real-session-shaped stream, an offline `npm test` assertion can then be added to `V9i`/`V9i-T` and criterion (c)'s sole-surface status relaxed.

Constraints: the ERR-8 rule in `cancellation.md` / `return.md` is spec-owned and read-only for this fix — do not restate or alter it. Pinning H7a's fixture to contain the subagent spawn, injected cancel point, and committed turns is owned by the separate "Smoke criteria presuppose specific H7a fixture content" finding; this fix references that fixture but does not author the H7a fixture-content requirement.
