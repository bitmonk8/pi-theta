## Problem

`conversation-drive.md` carries normative loom 1.0 boundaries across four heterogeneous in-text devices with no single place that collects them: the inline `deliverAs: "followUp"` non-emission clause, two "loom 1.0 seam" blockquotes (mid-loom user-session replacement, typed-query supported-provider set), and the `loom 1.0 diagnostic limitation` block. A reader asking what conversation-drive deliberately excludes in loom 1.0 must read the page end-to-end and recognise each scattered device individually. The central non-goals page (`#v1-non-goals`) records only the cross-cutting single-active-session disposition, not the page-local deferred seams. This is an organisation/discoverability gap, not under-specification — every boundary is stated normatively at its site.

## Issue introduction
**Verdict:** present-since-inception
**Introducing commits:** none isolable — no commit ever added or removed a consolidated boundary/non-goals statement on this page.
**History:** The spec corpus is git-tracked. `git log --follow` for `conversation-drive.md` bottoms out at `f5e89f4` (2026-06-04, "enforce 100KB hard / 30KB recommended size cap on spec set"), the split that carved this page out of the larger Pi-integration-contract file; the conversation-drive content predates that split in the predecessor file. `git log -S 'non-goal' -i --follow` over the file returns no commits, confirming a collected non-goals/boundary statement has never existed on this page (nor migrated in from its predecessor). The scattered-boundary arrangement is therefore the page's original shape, not a regression introduced by a later edit.

## Solution approach

Add a short orientation note near the top of `conversation-drive.md` that enumerates the page's loom 1.0 exclusions in one place and forward-links to each canonical site rather than restating the normative text. Forward-link to the inline `deliverAs: "followUp"` clause, the `#loom-1-0-seam-mid-loom-user-session-replacement` and `#loom-1-0-seam-typed-query-supported-provider-set` seam blockquotes, the `#pic-typed-query-noncompliance` diagnostic-limitation block, and the central `#v1-non-goals` entry. Add an anchor target for any cited site that lacks one (the inline `deliverAs` clause currently carries none).

## Solution constraints

- Forward-link to, do not copy, the canonical normative text at each cited site — the seam blockquotes, the inline `deliverAs` clause, and the `#pic-typed-query-noncompliance` block stay single-sourced.
- Out of scope: the existing seam blockquotes, the `#pic-typed-query-noncompliance` block, and the central non-goals page — beyond adding a missing anchor target, leave their content unchanged.
