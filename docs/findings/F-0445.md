## Finding

Release-time residue-inspection item 9 (*Ambient timing-primitive list reconciliation*) in `conventions.md` reconciles the `H3a` ambient-primitive scan's hand-maintained timing-primitive list against the spec's `Clock`/`WallClock` ban (PIC-12). Its operative constraint is the superset invariant: "The list is required to be a superset of the spec ban (scan list ŌŖć spec ban)", and the **Clean when** condition restates exactly this ŌĆö "every primitive the current spec ban enumerates is covered by the scan's list (the scan list ŌŖć the spec ban) and any scan-list entry beyond the spec ban is a deliberate superset addition recorded here."

Wedged between the invariant and the **Clean when** condition is the clause "and currently equals the PIC-12 enumeration `Date.now`/`performance.now`/`Date.prototype.getTime`/`setTimeout`/`clearTimeout`, so this item does not require strict equality with the spec ban." This records a point-in-time snapshot of the spec ban's membership. The moment the spec extends the `Clock`/`WallClock` surface, the snapshot is wrong, and nothing flags the divergence ŌĆö the very failure mode this item exists to catch. The clause carries no constraint the `ŌŖć` invariant and **Clean when** condition do not already own; the "does not require strict equality" point is already implied by an invariant that is a superset relation, not an equality.

The identical snapshot pattern appears one section earlier in the No-globals cross-cutting rule ("currently equal to the PIC-12 enumerationŌĆ”"); both were introduced together and are independently stale-prone.

## Solution Space

**Shape:** single

### Recommendation

In `conventions.md` item 9, strike the snapshot clause "and currently equals the PIC-12 enumeration `Date.now`/`performance.now`/`Date.prototype.getTime`/`setTimeout`/`clearTimeout`, so this item does not require strict equality with the spec ban" from the sentence that begins "The list is required to be a superset of the spec ban (scan list ŌŖć spec ban)". Leave that sentence ending at "(scan list ŌŖć spec ban)." Retain the superset invariant and the unchanged **Clean when** condition, which already carry the operative constraint and the "no strict equality" semantics (a superset relation is by definition not an equality). The edit is confined to item 9; do not touch the **Clean when** clause or the PIC-12 anchor.

## Consequence

**Severity:** cosmetic

The snapshot clause goes stale the next time the spec's `Clock`/`WallClock` ban (PIC-12) grows, presenting a wrong membership list inside the very checklist item meant to detect ban drift. No leaf's acceptance criteria depend on it; the operative `ŌŖć` invariant and **Clean when** condition are unaffected, so the impact is reviewer-facing wording rot rather than a coverage or pickability defect.
