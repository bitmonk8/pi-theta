## Finding

The §Leaf format **Deps** *Pickability predicate* in `conventions.md` declares that a `Deps.` entry is satisfied "exactly when the §Per-phase TDD ritual completion tag for that entry is present" — `<id>-complete` for an implementation/horizontal leaf, `<id>-T-complete` for a tests task — and states "those git tags are the authoritative pickability signal." It never states *where* the tag must exist for the evaluation to be sound: that completion tags are pushed to the shared remote, and that pickability is evaluated in a clone that has fetched tags.

Git tags are local until pushed and absent from a shallow / no-tags clone until explicitly fetched. Two evaluation environments therefore disagree on the same predicate: an author who tags `<id>-complete` locally but does not push it leaves the dep unsatisfied for everyone else; a CI or fresh clone fetched without tags (or shallow) reads every completion tag as absent and judges every leaf un-pickable. Because the predicate presents the tag's presence as authoritative, neither failure surfaces as an error — it silently misreads satisfaction.

The gap is the visibility precondition, not the mechanism. The tag-based predicate itself is sound; it needs the precondition under which "the tag exists" is a well-defined, environment-independent question.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/conventions.md`, §Leaf format, **Deps** *Pickability predicate*, add the tag-visibility precondition adjacent to the "those git tags are the authoritative pickability signal" sentence: state that completion tags (`<id>-complete` / `<id>-T-complete`) are pushed to the shared remote when the leaf's exit gate is met, and that pickability is evaluated against a clone that has fetched tags (a full clone fetched with tags, not a shallow or no-tags checkout). The precondition is what makes "the tag exists" an environment-independent test; the existing tagging step in §Per-phase TDD ritual (phase exit gate, "tag commit `<id>-complete`") is where the push obligation attaches, so the predicate may cross-reference that step rather than restate it.

## Consequence

**Severity:** advisory

If shipped unfixed, two contributors in different clone states (one with an unpushed local tag, one in a shallow/no-tags CI checkout) reach opposite conclusions about whether a leaf is pickable, with no error raised — the dep-driven build order silently diverges across environments. Implementers can still produce working leaves; the cost is wasted/blocked pickups and disagreement, not an incorrect artefact.
