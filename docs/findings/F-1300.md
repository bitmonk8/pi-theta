## Problem

The `*details* nesting convention*` clause in `diagnostic-emission-isolation.md` names the upstream pin via the fragment "the closed-at-`{reason}`-in-loom-1.0.0 pin governing additions". The `{reason}` token there is the literal `details.event.reason` field name, but read in isolation it looks like an un-substituted Mustache-style template placeholder — especially because the surrounding prose uses the `{reason}` form precisely where a placeholder slot would be expected. The cross-reference to the upstream contract is the only thing communicating what the pin does; the catch-phrase token itself conveys nothing without that round-trip.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commits:** `52b9101` (2026-05-11) — "resolve 'T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state'"
**History:** `52b9101` introduced the nested `details.event.{reason}` wire-format convention with its closed-shape pin and the "closed-at-`{reason}`-in-loom-1.0.0 pin governing additions" catch-phrase in the `*details* nesting convention` clause, in one large multi-concern edit. Subsequent commits (`5e5f2f1`, `c326bfa`, `f5e89f4`, `035d82e`) edited the surrounding paragraph but preserved the phrasing verbatim.

## Solution approach

Rewrite the catch-phrase fragment "the closed-at-`{reason}`-in-loom-1.0.0 pin governing additions" in the `*details* nesting convention*` clause into plain prose that names the contract directly — the loom 1.0.0 closure pinning `details.event.*` to the single SDK-derived `reason` sibling and forbidding further SDK-derived siblings between Pi-SDK-pin bumps. Retain the existing markdown link to `diagnostics/diagnostic-shape.md#session-shutdown-details-conventions`.

## Solution constraints

- Editorial only: the rewrite MUST NOT change the wire-format contract semantics imported by reference from `diagnostics/diagnostic-shape.md#session-shutdown-details-conventions`.
