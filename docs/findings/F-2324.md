## Finding

The `DIAG-3` Tests bullet in both paired leaf files — `V7b` (`docs/plan_topics/V7b-code-registry.md:9`) and `V7b-T` (`docs/plan_topics/V7b-T-code-registry.md:9`) — reads:

> `DIAG-3`: codes are stable identifiers — a renamed code fails the gate (renames deferred to loom 2.0).

The trailing parenthetical `(renames deferred to loom 2.0)` is roadmap rationale, not part of the testable acceptance criterion. The normative obligation the bullet states — "a renamed code fails the gate" — is complete and self-contained without it. Per `conventions.md` §Leaf format, a `Tests.` bullet states what the leaf asserts; a forward-version deferral note is not something the `V7b` gate asserts.

The deferral fact is already owned normatively by the spec: `DIAG-3` in `docs/spec_topics/diagnostics/diagnostic-shape.md` (`:73`) states "A rename is therefore deferred to loom 2.0 migration, outside the GOV-15 diagnostic-registry carve-out." Restating it inside the leaf's acceptance bullet duplicates spec rationale into the plan and risks the subtle misreading that the DIAG-3 *gate itself* is deferred rather than the rename *operation*. The parenthetical appears verbatim in both paired files, so any fix must touch both to keep them in lockstep.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e ("pi-loom plan: build/update plan for spec.md + review", 2026-06-10)

The parenthetical was present in the DIAG-3 bullet from the leaves' first authoring; no later edit introduced or altered it.

## Solution Space

**Shape:** single

### Recommendation

Strike the trailing parenthetical from the `DIAG-3` Tests bullet in both paired files so each bullet reads:

> `DIAG-3`: codes are stable identifiers — a renamed code fails the gate.

Apply the identical edit to `docs/plan_topics/V7b-code-registry.md:9` and `docs/plan_topics/V7b-T-code-registry.md:9` to keep the paired files in lockstep. The deferral fact need not be re-homed elsewhere in the plan — it is already carried normatively by `DIAG-3` in `docs/spec_topics/diagnostics/diagnostic-shape.md`, which the leaf cites under its **Spec.** field. The spec is read-only for this fix.

## Consequence

**Severity:** cosmetic

The DIAG-3 acceptance criterion remains correct and implementable; the parenthetical is excess content that duplicates a spec-owned deferral rationale inside a normative Tests bullet. At worst a reader could misparse "renames deferred" as deferring the gate itself rather than the rename operation, but the leaf still ships the right behaviour.
