## Problem

`conventions.md` §Loom 1.0 release-time residue inspection enumerates nine residue classes the cross-cutting mechanical scans concede they cannot witness, and asserts that a residue defect found at any item "blocks the loom 1.0 release until fixed." But the inspection names no owning leaf, no executor, and no wiring into any leaf's exit gate — it is described only as "a reviewer reading `src/**`" at the release gate. The terminal release-gate leaf `H6a`, which is the loom 1.0 release gate the inspection says it runs at, carries no reference to the residue inspection in its **Adds**, **Tests**, or **Ships when**; no other leaf references it either. The release can therefore ship green with the entire conceded-residue surface never swept, because nothing observable obligates the inspection.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commits:** `e94ca10` (2026-06-16) — "pi-loom plan: resolve \"GOV-15 release-time reviewer inspection has no enumerated residue checklist\""
**History:** `git log -S "Loom 1.0 release-time residue inspection"` and `-S "blocks the loom 1.0 release until fixed"` on `conventions.md` both return `e94ca10` as the sole introducing commit. That commit added the entire residue-inspection block (heading, intro prose, and the checklist) and touched `conventions.md` only (22 insertions; `--stat` shows no `H6a` / `plan.md` / `H7a` edit). The added prose describes the executor as "a reviewer reading `src/**`" and asserts the inspection "blocks the loom 1.0 release until fixed" but assigns it no owning leaf and wires it into no leaf's exit gate. `git log -G "residue inspection"` over `H6a-live-corpus-activation.md` and `plan.md` returns no commits, confirming the inspection has never been referenced from a leaf. The ownerless/unwired state is therefore present since the block's inception in `e94ca10`.

## Solution approach

Wire the residue inspection into `H6a` as a release-gate exit-gate obligation, parallel to the manual real-host smoke run `H6a`'s **Ships when** already carries: add it to `H6a`'s **Ships when** so the loom 1.0 release does not pass until the inspection has been performed against the live corpus with every item clean or each finding dispositioned, recorded in the same gate-activation commit message that already records the smoke run. In `conventions.md` §Loom 1.0 release-time residue inspection, name `H6a` as the owning leaf and executor, replacing the ownerless "a reviewer reading `src/**`" framing. Keep `conventions.md` authoritative for the checklist content and `H6a` authoritative for the gating obligation.
