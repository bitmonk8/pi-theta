## Finding

The bullet immediately under the field-contract table reads:

> `bind_model`, `bind_context`, and `bind_echo` configure slash-command argument binding. All three are optional with sensible defaults; see [Slash-Command Argument Binding](../binder.md).

The "optional with sensible defaults" tail is uninformative once the field-contract table immediately above is normative and already pins, per field, both the Required? column (`no` for all three) and the exact "Default when absent" column (`looms.binderModel` setting (no further fallback); `none`; `true`). The bullet therefore restates — vaguely — what the table has just stated precisely, and "sensible" is a non-falsifiable modifier (defaults are sensible by stipulation).

The bullet's useful payload is the cross-link to `../binder.md`. Stripping the redundant clause keeps that payload while removing the only weasel word on the page.

**History:** `83802db` introduced the `bind_model`/`bind_context`/`bind_echo` fields together with the "All three are optional with sensible defaults" bullet at a point when the spec had no field-contract table, so the clause was the only place those defaults were stated. Later the same day, `fb54586` added the normative field-contract table that pins each field's Required? and Default-when-absent columns explicitly, making the bullet's tail redundant. The redundancy then survived the spec split into `frontmatter-fields-a.md` and subsequent edits.

## Solution Space

**Shape:** single

### Recommendation

Replace the redundant clause with a pointer to the contract table. Rewrite the bullet as:

> `bind_model`, `bind_context`, and `bind_echo` configure slash-command argument binding; their required/default contract is the field-contract table above. See [Slash-Command Argument Binding](../binder.md) for the binding flow.

Keep the `../binder.md` cross-link (that is the bullet's actual payload). Do not add new defaults prose here — the field-contract table is the single source of truth and any restatement would re-open the same drift risk.

## Consequence

**Severity:** cosmetic

A single bullet on the frontmatter page carries a redundant, slightly weasel-worded clause. No implementer is misled and no behaviour is at stake; the only cost is one sentence of authorial noise next to a normative table.
