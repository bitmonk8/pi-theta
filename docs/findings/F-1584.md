## Finding

`LoomRegistry.readDrainState` exposes a single closed three-arm dispatch state machine (PIC-29: (a) steady-state dispatch, (b) `"shutting down"` note, (c) `"degraded; /reload to recover"` note) that the slash-command handler consults on every invocation. Its acceptance fixtures — the conformance set that pins the tuple-to-arm map and the branch coverage a test author must satisfy — are split across two files, organised around each file's local concern rather than around the arm enumeration:

- `drain-state-contract.md` § "Acceptance criteria — `(drained: true, tag: undefined)` → arm (b)" owns the arm-(b) routing fixtures (reachability of the `(true, undefined)` tuple under `reason ∈ {"quit","reload"}`, plus the arm-(b) routing assertion).
- `session-only-degraded-state.md` § "Acceptance criteria" owns fixtures (a)–(m), covering the `runtime-degraded` emission predicate, arm-(c) routing, the `/reload` recovery path, and both `readDrainState` read-failure fail-safe sites.

The two sections reference each other only at the level of individual fixtures (drain-state (b) → session-only branch (j); session-only (l)/(m) → the drain-state read-failure clause); there is no top-level map enumerating which fixtures cover which arm. A test author opening either file sees only that file's slice and cannot determine from either alone whether PIC-29's three arms are fully covered. Arm (a) (steady-state dispatch) has no dedicated routing fixture in either acceptance section — it is asserted only incidentally by the session-only recovery fixture (g) and by the all-three-throw corner case (whose missing fixture is itself a separate finding).

A constraint shapes the fix: the split is partly load-bearing. The session-only acceptance fixtures (a)–(m) are **gated** (blocked pending the host-prerequisites clause (a) open contradiction) per that page's implementation-gate banner, whereas the `drain-state-contract.md` arm-(b) fixtures are non-gated and buildable now. The page gate "scopes by clause, not by file", so a consolidating index must preserve that gating boundary rather than collapse gated and non-gated fixtures into one undifferentiated set.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** f5e89f4 (`docs: enforce 100KB hard / 30KB recommended size cap on spec set`, 2026-06-04), 8008b91 (`pi-loom spec-plan: fix F-1217 — add acceptance-criteria/fixture blocks for four Shard-08 MUSTs`, 2026-06-25)
**History:** Both `drain-state-contract.md` and `session-only-degraded-state.md` were created by the spec-split commit f5e89f4. The session-only acceptance set was established on the new page shortly after (its throwing-`readDrainState` fixtures (l)/(m) were added by 2775382, 2026-06-05, resolving "Two throwing-readDrainState MUSTs have no acceptance criteria"). The cross-file partition was created much later by 8008b91 (`git show 8008b91 -- drain-state-contract.md` confirms the `## Acceptance criteria — \`(drained: true, tag: undefined)\` → arm (b)` heading was net-new in that commit), which added a second acceptance-criteria section in `drain-state-contract.md` for the arm-(b) MUST without consolidating it with, or cross-indexing it against, the pre-existing session-only set. The defect is the interaction: neither commit is wrong in isolation, but together they leave the single PIC-29 arm enumeration with two unconnected fixture homes and no index.

## Solution Space

**Shape:** single

### Recommendation

Add a consolidating acceptance index for the `readDrainState` arm enumeration and keep the fixtures where they live (so the gated/non-gated boundary is preserved). Concretely:

- Place a short mapping table adjacent to PIC-29 in `drain-state-contract.md` (the non-gated owner of the arm enumeration). Rows: one per PIC-29 arm ((a) dispatch, (b) shutting-down note, (c) degraded note), each listing the covering fixture IDs, the owning file, and the gating status (non-gated for the arm-(b) set; gated-pending-clause-(a) for the session-only (a)–(m) set).
- For arm (a), the table states that steady-state-dispatch routing is currently asserted only indirectly (session-only fixture (g) after `/reload`, and the all-three-throw corner case) and points at the dedicated arm-(a) routing fixture obligation; do not author that fixture here — it is owned by the PIC-30 corner-case finding.
- Add a reciprocal one-line pointer from the `session-only-degraded-state.md` § "Acceptance criteria" preamble back to this index, so the gated page declares it is one half of a two-file set whose map lives on the non-gated page.

This gives a test author a single location from which to verify completeness of the three-arm coverage without merging gated and non-gated fixtures into one block. Edge cases the implementer must respect: the index must restate, not relax, the per-page gate (it lives on the non-gated page but its rows for fixtures (a)–(m) carry the gated marker); and if the "arm"→"branch" rename (see Related Findings) lands, the index's arm labels and the arm-(b) heading must move with it in the same edit.

## Consequence

**Severity:** advisory

The fixtures all exist and do not conflict, so a working system is still producible. The risk is under-coverage: a test author cannot confirm the PIC-29 arm enumeration is fully exercised from either file alone, and arm (a) lacks a dedicated routing fixture, so the steady-state dispatch path is the easiest to leave untested.
