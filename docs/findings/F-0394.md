## Problem

The respond-repair-loop Tests bullet in both `docs/plan_topics/V13d-query-failure-repair.md` and `docs/plan_topics/V13d-T-query-failure-repair.md` ends with the clause `` `none`/`0` means no follow-up ``. The slash-separated pair gives no indication that `none` and `0` are values on two different frontmatter fields — `none` belongs to `respond_repair.methodology` and `0` to `respond_repair.attempts` — rather than two spellings of one value. An implementer can satisfy the bullet by asserting a single field, leaving the other field and the silent-ignore interaction untested while the test still appears to cover the "no follow-up" contract.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (`pi-loom plan: build/update plan for spec.md + review`, 2026-06-10)
**History:** The `V13d` and `V13d-T` leaf files were both created in `c6a664e`, and the `` `none`/`0` means no follow-up `` clause appears verbatim in that genesis revision of each file (`git log --diff-filter=A --follow` reports `c6a664e` as the first commit for the file; `git log -S` / `-G` over the clause returns only `c6a664e`). The three later commits that touched the file (`8f9e160`, `c9f4db5`, `fd6169b`) left the clause unchanged. The ambiguity entered with the leaf's first authoring.

## Solution approach

Rewrite the trailing clause of the respond-repair-loop Tests bullet so it distinguishes the two fields and names what each suppresses: `respond_repair.methodology: none` and `respond_repair.attempts: 0` each independently suppress all follow-up (the two are equivalent per `query-failure-and-repair.md`), and a declared non-zero `respond_repair.attempts` is silently ignored under `methodology: none`. Apply the same corrected clause to both `V13d-query-failure-repair.md` and `V13d-T-query-failure-repair.md`, which carry the bullet verbatim.

## Solution constraints

- The spec (`docs/spec_topics/query/query-failure-and-repair.md` and the frontmatter topic pages) is read-only for this fix; it already states the contract.
