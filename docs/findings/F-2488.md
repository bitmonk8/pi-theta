## Finding

DISC-4's "Detection runs on the final derived name (after `pi.looms` mapping, with the filename stem taken verbatim for sources that have no `pi.looms` mapping)" uses "mapping", connoting a rename feature `pi.looms` does not provide; slash names are always derived from the filename stem verbatim.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** f5e89f4 — docs: enforce 100KB hard / 30KB recommended size cap on spec set (2026-06-04, Thomas Andersen); 1cb54c7 — pi-loom spec: resolve "Glob !/+/- precedence and matcher engine unspecified for pi.looms / loomPaths" (2026-06-06, Thomas Andersen); 901d49c — pi-loom spec: resolve "ContextOverflowError raw_response schema reconciliation" (2026-06-08, Thomas Andersen)
**History:** The five bundled items entered independently across three commits rather than through a single causal change. The DISC-1 bare-`~` wording, the DISC-4 "after `pi.looms` mapping" phrasing, and the ERR-7 placement under "before evaluation begins" are all present at the inception of their topic files in f5e89f4, the spec-sharding commit. The DISC-5 "insertion order is preserved" sentence was added by 1cb54c7 when the `!`/`+`/`-` precedence rules were authored. The undefined "output-boundary overflow" term was introduced by 901d49c when the `ContextOverflowError.raw_response` schema was reconciled.

## Solution Space

**Shape:** single

### Recommendation

In `discovery-sources.md` DISC-4, reword "after `pi.looms` glob resolution" (and the parallel "sources that have no `pi.looms` entry") so the sentence describes which file each candidate resolves to, not a name transform.

## Consequence

**Severity:** advisory

Each defect is a localised guidance gap rather than a behavioural fork: an implementer can still build a working system, but may waste effort (bare-`~` handling, the meaning of "output-boundary overflow"), implement a no-op (same-class insertion order), or mis-classify ERR-7's lifecycle. The undefined "output-boundary overflow" term is the most consequential since it gates when `raw_response` carries partial text, but the field is nullable and best-effort, so divergence is observable only as variation in a diagnostic payload.
