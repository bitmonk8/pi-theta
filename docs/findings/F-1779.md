## Finding

`docs/spec_topics/pi-integration-contract/host-interfaces-core.md` is the host-interface inventory page — it enumerates the loom-load-bearing surface of `ExtensionContext`, `ExtensionCommandContext`, `ModelRegistry`, `Model`, `AgentSession`, and the cancellation entry points, each as a Pi-owned snapshot to be re-validated on each Pi minor bump. The §*Tool execution from loom code* surface inside that page goes further: it pins the full loom-runtime outcome lowering for a code-side `<name>(args)` call, including the `type === "text"` content filter and single-`"\n"` join, the `Ok("")` empty-text contract, the 4096-byte UTF-8 truncation rule (with the code-point-boundary cut and worked U+2026 example), the synthesised `"tool reported an error with no text content"` literal, the `Err(CodeToolError { cause: "execution", … })` lowering for an `execute()` throw including the underlying-error-string coercion, the *Outcome routing summary* enumerating the cancelled / non-conforming-shape / non-settling-Promise / late-settlement dispositions, the [CNCL-1](../cancellation.md#cncl-1) / [CNCL-2](../cancellation.md#cncl-2) / [CNCL-3](../cancellation.md#cncl-3) discard rules, and the swallowing-handler installation MUST (with the *before the first microtask boundary* obligation). These are loom-runtime behaviour, not Pi-owned surface — none of them are a snapshot of an `@earendil-works/pi-coding-agent` type and none participate in the SDK surface-inventory literal-read assertion the surrounding paragraphs anchor.

The same page already owns the genuinely Pi-side pieces of code-side tool execution: the touched-member subset of `ExtensionContext`, the per-mode override table for `signal` / `sessionManager` / `abort()`, and the cited `AgentToolResult` / `{ content, isError }` envelope declaration. What does **not** belong here is the runtime contract about *what loom does with* the returned envelope. The downstream owners are already named and already cross-referenced: [`tool-calls.md`](../../docs/spec_topics/tool-calls.md) is the loom-author-facing surface for tool calls and the *Outcome enumeration (normative)* block there is the explicit canonical home for the routing of `execute()`'s further outcomes (non-conforming shape, non-settling Promise, post-cancel late settlement), with `cancellation.md` (CNCL-1/2/3, late-settlement discard) owning the swallowing-handler / discard machinery and `errors-and-results/queryerror-variants.md` (`### Code-side tool-call variant`) owning the `CodeToolError` `cause` enum and field shape. The PIC page currently restates rules whose normative home is each of those three pages, producing parallel normative text that any future edit to lowering, truncation, or discard must keep in sync across files.

The scope drift is the same shape the spec corpus has flagged elsewhere (see *Misplaced sections within the pi-integration-contract pages*, *F-1260 — relocate no-extra-mediation NFR to trust-boundary.md*, and *#cwd-project-root-presupposition homed in a DI-seam bullet, not the presupposition list*): a section grows runtime semantics out of an interface-pinning paragraph, the runtime semantics then become the de-facto home, and the genuine host-interface anchor on the same page (which `cka-46` and `V14a` already cite by `#tool-execution-from-loom-code`) ends up dual-purpose. The fix is to relocate the runtime-side content to its owning pages and leave PIC with the host-interface portion (the `execute(...)` parameter pinning and the `AgentToolResult` envelope declaration), preserving the `#tool-execution-from-loom-code` anchor for the inbound `cka-46` / `V14a` / `query/query-tool-loop.md` / `tool-calls.md` / `cancellation.md` citations.

## Issue introduction

**Verdict:** present-since-inception

**Introducing commits:** `fae85d3` *Tighten spec for implementation-plan readiness* (2026-05-04) introduced the §*Tool execution from loom code* surface verbatim into the monolithic `spec.md`, already mixing the host-interface bullets (`toolCallId`, `params`, `signal`, `onUpdate`, `ctx`) with the runtime-side lowering rule (`{ content, isError }` → string return value; `Err(QueryError { kind: "tool_call", cause: "execution", … })`) under a single section.

**History:** `git log --reverse -G "Tool execution from loom code"` lands on `fae85d3` as the introducing commit; subsequent edits accreted the runtime-side content rather than relocating it. `8f8acbf` (2026-05-05) added the 4096-byte UTF-8 truncation rule (resolving "tool-error message truncation rule sloppy") at the same site; `11ef3b9` (resolving "Tool failure modes beyond throw / isError: true unspecified") introduced the non-settling / non-conforming-shape language; `2ebe288` (2026-05-07) added the post-cancel late-settlement discard and the swallowing-handler MUST; `5f04646` added the non-text-discard `Ok("")` paragraph; `482098b` (2026-06-04) split the late-settlement discard into the CNCL-1/2/3 sub-anchors but kept the rule in PIC. `fecb504` split `spec.md` into per-topic files (the section moved into `spec_topics/pi-integration-contract.md` unchanged), `31ff060` moved that file under `docs/`, and `f5e89f4` (2026-06-04) split the 576KB `pi-integration-contract.md` into the index+children layout, creating the present `host-interfaces-core.md` file with the section's content carried forward verbatim. The scope mixing is therefore traceable to the section's first appearance: every subsequent commit treated PIC as the canonical home for the runtime rule and reinforced the placement rather than challenging it.

## Solution Space

**Shape:** single

### Recommendation

Relocate the runtime-semantics content from `host-interfaces-core.md` §*Tool execution from loom code* to its owning pages, leaving only the host-interface anchor on this page. Specifically:

1. **Keep on `host-interfaces-core.md` under the `#tool-execution-from-loom-code` anchor** the host-interface paragraph: the call `execute(toolCallId, params, signal, onUpdate, ctx)` and its five-bullet parameter sheet (`toolCallId` / `params` / `signal` / `onUpdate` / `ctx`), and the immediately-following paragraph that cites the `AgentToolResult` / `{ content, isError }` envelope declaration in `@earendil-works/pi-agent-core`'s `dist/types.d.ts` (this is genuinely a Pi-owned snapshot subject to the surrounding "MUST be re-validated against the cited declarations on each Pi minor bump" discipline; the *Pi-owned snapshot* marker already pinned to the page's other surfaces should be appended to this paragraph). Add a one-sentence forward-link explaining that the lowering of the returned envelope to the loom-level `Result<string, QueryError>` value is specified at [Tool Calls — Failures](../tool-calls.md#failures) and [Tool Calls — *Outcome enumeration (normative)*](../tool-calls.md), with cancellation/discard behaviour at [Cancellation — Race semantics](../cancellation.md#race-semantics) and [CNCL-1](../cancellation.md#cncl-1)/[CNCL-2](../cancellation.md#cncl-2)/[CNCL-3](../cancellation.md#cncl-3).

2. **Move to `tool-calls.md`** (preserving every MUST/MUST-NOT verbatim and the [F-1396](../../findings/F-1396.md) `toolCallId` UUID-body/PIC-20 pin already landed at the PIC site):
   - The envelope-to-string lowering rule (`type === "text"` filter, single-`"\n"` join, no separator before first / after last block).
   - The non-text-content discard paragraph (`Ok("")` legal, no `RuntimeEvent` / `loom-system-note` / diagnostic on the discard path, MAY emit non-normative debug logs).
   - The 4096-byte UTF-8 truncation rule, including the code-point-boundary cut, the up-to-three-byte shortfall, and the U+2026 worked example.
   - The `"tool reported an error with no text content"` sentinel.
   - The `Err(QueryError { kind: "code_tool", cause: "execution", message: <m>, tool_name, … })` lowering for `execute()`-throw, including the [underlying-error string](../diagnostics/placeholder-rendering-b.md#underlying-error-coercion) coercion and the 4096-byte truncation rule applying uniformly to both `<m>` forms.
   - The *Outcome routing summary* paragraph in its entirety. The "Three further tool-side outcomes" enumeration already on `tool-calls.md` is the natural host for this paragraph and reads as its expansion rather than a duplicate.

3. **Move to `cancellation.md`** (consolidating with the existing *Race semantics — late-settlement discard* paragraph that already owns CNCL-1/2/3):
   - The swallowing-handler installation MUST: "the runtime MUST keep a swallowing handler attached to the underlying `execute()` Promise (and, symmetrically, to the underlying `@`-query provider Promise and the `invoke` callee's top-level execution Promise) for the lifetime of that Promise, installed at construction time before the first microtask boundary." The symmetry to `@`-query and `invoke` Promises already belongs with the cancellation page's cross-primitive discard rule.
   - The "discard is total" language (no `Err` rebinds the call site, no `RuntimeEvent` reaches the always-log channel, no diagnostic of any severity, discarded value is not traversed).

4. **Verify and repoint citations.** The `#tool-execution-from-loom-code` anchor stays on `host-interfaces-core.md` so inbound citations from `coverage-matrix.md` `cka-46`, plan leaves `V14a` / `V14a-T` / `V14c` / `V14c-T` / `V14f` / `V14f-T`, `query/query-tool-loop.md`, `tool-calls.md`, `cancellation.md`, and `diagnostics/code-registry-runtime.md` (the `tool-return-shape` row) continue to resolve. Update each inbound citation whose pointed prose is now on `tool-calls.md` (envelope-lowering MUSTs, truncation, `Ok("")` case, outcome-routing summary) or `cancellation.md` (swallowing-handler MUST, CNCL-1/2/3 discard total-ness) to point at the new home; the `coverage-matrix.md` `cka-46` row's pointed-prose enumeration is the largest single rewrite. Confirm via `grep -rn '#tool-execution-from-loom-code' docs/` that every remaining citation is satisfied by the host-interface paragraph that stays.

Edge cases for the implementer:

- The §*per-mode override semantics* table (the `signal` / `sessionManager` / `abort()` rows and the "every other member forwards" enumeration) is host-interface material and stays on `host-interfaces-core.md`; it already lives in its own block above §*Tool execution from loom code* and is not part of this relocation.
- The `loom-direct:` `toolCallId` UUID-body pin and the PIC-20 `IdSource` minting cross-reference (F-1396) are host-interface content (the `toolCallId` parameter sheet bullet) and stay with the parameter sheet.
- The `CodeToolError` alias parenthetical inside the lowering paragraph (the `("…this value is also referred to as `CodeToolError`…")` clause) belongs with the lowering rule and moves with it to `tool-calls.md`.
- Keep the relocated text's section title — e.g. "Code-side `execute()` outcome lowering" — referenced inline from the kept host-interface paragraph so reviewers re-validating PIC can navigate to the runtime contract in one click.

## Consequence

**Severity:** advisory

The §*Tool execution from loom code* surface today produces two coupled costs. First, the runtime-semantics paragraphs (envelope lowering, 4096-byte truncation, late-settlement discard, swallowing-handler MUST) are duplicated in spirit across `host-interfaces-core.md`, `tool-calls.md`, `cancellation.md`, and `errors-and-results/queryerror-variants.md`; any minor edit (e.g. the existing F-1396 `toolCallId` pin or a hypothetical truncation tweak) requires four-file synchronisation, and the absence of a single owning anchor invites drift. Second, the host-interface re-validation discipline that the page's "Pi-owned snapshot" markers attach to every other surface does not apply to most of this section — re-validation reviewers reading the page must mentally partition each paragraph into "actually a Pi-owned snapshot" vs "runtime contract that happens to live here," weakening the page's contract.
