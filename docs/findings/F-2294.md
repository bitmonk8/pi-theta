## Finding

`conventions.md` §*Specific exception types only* defines a five-category broad-catch exemption framework: a `catch (e: unknown)` is permitted only at one of five enumerated exempt sites — (1) the Pi SDK boundary (which "traces to no spec obligation"), (2) *Self-failure* probe traps, (3) the *Unknown-reason rule* `event.reason` read, (4) *Per-step isolation* `session_shutdown` sub-step wraps, and (5) *Diagnostic-emission isolation* `console.error` wraps. Several of these are deliberately log-and-continue or no-spec-obligation catches — exactly the shapes the project-wide rule in `CLAUDE.md` ("Never `catch(...)`. Catch specific types or let crash", elaborated in `code_style_guidelines.md`) forbids in general code.

The rule never states that this framework *is* the pi-loom (TypeScript / Pi-SDK) realization of that project-wide rule — the sanctioned set of carve-outs, directly analogous to the C# AcceleratorClient NFR-2.1 background-loop carve-out that `CLAUDE.md` itself records. The sibling cross-cutting rule one bullet below it, *Sequential by default*, explicitly bridges to the parent policy ("This rule subsumes the CLAUDE.md *Never block the async runtime* directive"). The exception rule carries no such bridge, leaving an asymmetry: an implementer or reviewer consulting both the project-wide exception rule and `conventions.md` sees the five exempt sites as in apparent tension with the parent policy, with nothing identifying them as the authorized realization rather than a drift.

This is a documentation-alignment gap, not an enforcement defect — the five sites, their citable tokens, and their mechanical/manual gates are precisely specified. The missing note is the acknowledgment of provenance. (`CLAUDE.md` and `code_style_guidelines.md` are repository-root project docs outside the plan corpus and are read-only context for this fix.)

## Solution Space

**Shape:** single

### Recommendation

Restore a provenance note to `conventions.md` §*Specific exception types only* identifying the five-category broad-catch exemption framework as the pi-loom (TypeScript / Pi-SDK) realization of the project-wide exception-handling rule, mirroring the bridge the sibling *Sequential by default* rule already carries.

- **Where:** the *Specific exception types only* bullet in `conventions.md` (line 59), the same bullet that enumerates the five exempt sites.
- **What it must accomplish:** name the parent rule (`CLAUDE.md` "Catch specific types or let crash", elaborated in `code_style_guidelines.md`); state that the five enumerated exempt sites are its sanctioned TS/Pi-SDK carve-outs (directly analogous to the C# AcceleratorClient NFR-2.1 background-loop carve-out `CLAUDE.md` records); and state explicitly that the note introduces **no enforcement-posture change** — the existing mechanical (`no-broad-catch` lint + closing-gate token resolution) and manual (release-time residue inspection item 6) gates are unchanged.
- **Content example (phrasing, not mandated shape):** "This framework is the TypeScript/Pi-SDK realization of the project-wide *catch specific types or let crash* rule (`CLAUDE.md`; full rules in `code_style_guidelines.md`): the five exempt-broad-catch-site categories are its sanctioned carve-outs — analogous to the C# AcceleratorClient NFR-2.1 background-loop carve-out — not an exception to it. No enforcement-posture change."
- **Constraints:** the edit is confined to `docs/plan_topics/conventions.md`. Do **not** edit `CLAUDE.md`, `code_style_guidelines.md`, or any `spec_topics/**` page — those are read-only for this fix. Use the project's leaf-ID and section conventions already present in the file.

## Consequence

**Severity:** advisory

A contributor or reviewer reading both the project-wide exception rule ("catch specific types or let crash") and `conventions.md` finds the five broad-catch carve-outs — especially the no-spec-obligation Pi-SDK-boundary site and the log-and-continue isolation wraps — apparently contradicting the parent policy, with no note marking them as its authorized TS/Pi-SDK realization. The asymmetry with the adjacent *Sequential by default* rule (which already bridges to `CLAUDE.md`) invites a contributor to question the framework's authority or to "correct" a legitimately-exempt site. No leaf output diverges; the framework's sites and tokens are fully pinned.
