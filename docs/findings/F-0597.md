## Finding

Three `glossary.md` entries each restate a closed enumeration or live range that their canonical owner page has since outgrown, so the glossary summary is narrower or staler than the rule it summarises.

1. **`always-log set` entry omits the void-tail-expression discard form.** The entry enumerates the dispositions across which exactly-once emission is guaranteed as three: "matched the `Err`, propagated it via `?`, or discarded it via `let _ =`." The spec recognises a fourth disposition: the void-tail-expression discard form (a `@`…`` query in tail position of a void-returning function, equivalent to `let _ = @`…``), named on the canonical owner page (`pi-integration-contract/runtime-event-channel.md` — "Discard-site disposition") and inside the neighbouring `query-terminating` glossary entry ("discarded via `let _ = …` or the equivalent `void`-tail-expression form"). An implementer reading only the always-log set entry would conclude the void-tail-discard case needs no `loom-system-note` emit.

2. **`GOV-N` entry embeds a stale live-range upper bound.** The `GOV-N (governance rule)` entry parenthetically states the live range is "currently running through `GOV-23`, with some IDs retired." The authoritative set on `governance.md` (and its `governance/` sub-pages) now runs `GOV-1 … GOV-31` with `GOV-2`, `GOV-4`, `GOV-10`–`GOV-14`, and `GOV-21` retired — so "through `GOV-23`" is eight numbers low. The entry is self-undermining: it warns "cite `governance.md` for the authoritative range rather than a fixed upper bound" and then embeds exactly such a literal.

3. **`.warp` file entry under-enumerates permitted top-level forms.** The `` `.warp` file (library module) `` entry opens with "A library file containing only top-level `schema`, `enum`, and `fn` declarations…", three forms. The normative pages enumerate five: `imports.md` ("Top-level may contain only `import`, `export`, `schema`, `enum`, and `fn` declarations", pinning `loom/parse/warp-top-level-statement`) and `overview-and-orientation.md` ("currently five: `import`, `export`, `fn`, `schema`, `enum`"). The "containing only …" clause omits both `import` and `export`.

All three are the same pattern: a glossary entry restating an enumeration/range owned authoritatively elsewhere, with the restatement narrower than (or contradicting) the owner. None reaches a runtime — the canonical pages and the leaf gates (`V15c-imports.md`, the `V13`-side tests) are written against the owner pages — but each costs a reader-friction round of "which page wins", and the `always-log set` case can mislead an implementer working from the glossary alone into a silent operator-observability gap.

## Solution Space

**Shape:** single
**State:** shaped

### Recommendation

Re-sync all three entries to their canonical owners in one glossary sweep:

- **`always-log set`.** Replace "discarded it via `let _ =`" with "discarded it via `let _ =` or the equivalent `void`-tail-expression form", matching the `query-terminating` entry's phrasing verbatim so a future grep on either phrasing finds both sites. No change to the canonical PIC page or the `query-terminating` entry is needed.
- **`GOV-N`.** Drop the concrete `GOV-23` literal and let the existing "cite `governance.md` for the authoritative range" directive stand alone. Replace the inner clause "currently running through `GOV-23`, with some IDs retired — see [Retired REQ-IDs](./governance/anchor-scheme-and-retired.md#retired-req-ids)" with "with some IDs retired — see [Retired REQ-IDs](./governance/anchor-scheme-and-retired.md#retired-req-ids)". This is preferable to bumping the literal to `GOV-31`: any future GOV coinage re-stales the same number, and removing it closes the self-contradiction. Do not delete the "with some IDs retired" half-sentence or its anchor link — only the "currently running through `GOV-23`," fragment is dropped.
- **`.warp` file.** Reword the opening clause to defer to the owner page rather than re-enumerate: replace "A library file containing only top-level `schema`, `enum`, and `fn` declarations, importable from `.loom` and `.warp` files via `import`, …" with "A library file whose top level is restricted to the declaration forms enumerated in [Imports — Permitted top-level forms](./imports.md#permitted-top-level-forms), importable from `.loom` and `.warp` files via `import`, …". `overview-and-orientation.md` already uses this defer-by-anchor pattern. The rest of the entry (pairing with `.loom`, byte-exact lowercase extension matching, See-links) stays unchanged.

Implementer edge case: the unifying fix shape is "defer to the owner page or mirror its phrasing verbatim", not "restate the current value" — restating re-introduces the same drift on the next owner-page edit.

## Consequence

**Severity:** advisory

Each entry contradicts its own canonical page on a closed enumeration or a count: the `always-log set` reader could omit `loom-system-note` emission for `Err` values dropped via void-tail discard, producing a silent operator-observability gap on the exact discard form the runtime-event-channel page took pains to cover; the `GOV-N` reader under-counts the live GOV set by eight rules and may miss obligations from `GOV-24` onward (load-bearing for plan gates `H5e`/`H5d`); the `.warp` reader proposes a smaller top-level allowlist than the normative pages prescribe. Readers who consult the canonical pages are unaffected; readers who treat the glossary as a faithful summary diverge, and each entry weakens its own "defer to the owner" intent.
