## Problem

The `loom/load/missing-source` row in `docs/spec_topics/diagnostics/code-registry-load.md` declares severity `E/W`, which the `#column-legend` in `diagnostic-shape.md` defines as "both arms reachable, resolved per diagnostic by that row's own Trigger prose" — and cites this row among its motivating "discovery-source codes" examples. But the row's own Trigger says "warning never", and DISC-2's failure-modes table (`#disc-2` in `discovery-sources.md`) reads `silent` for all three conventional sources and `error` for all three explicit ones, so no source produces a warning. The `W` arm therefore has no triggering condition, breaking the legend's contract at one of its own cited examples. The row is the lone outlier among the three discovery-source codes; its siblings `loom/load/unreadable-source` and `loom/load/wrong-type-source` are genuinely `E/W` against DISC-2.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commit:** `5b772b6` — *pi-loom spec: resolve "Discovery source failure-mode policy is unspecified"* (Mon May 4 2026)
**History:** The row was added by `5b772b6` in `spec_topics/diagnostics.md` already carrying `Sev: E/W` together with the `warning never` Trigger clause, in the same edit that introduced the DISC-2 failure-modes table whose *Missing path* column has no warning entries. The mismatch has therefore existed since the row first appeared. Commit `f5e89f4` (Thu Jun 4 2026) later relocated the row verbatim into `docs/spec_topics/diagnostics/code-registry-load.md` when the diagnostics page was split for the 30/100 KB size caps; no edit since has touched the severity column or the `warning never` clause (`git log -S "loom/load/missing-source" -- docs/spec_topics/diagnostics/code-registry-load.md` reports only the file-split commit). The two sibling discovery-source codes added in the same `5b772b6` edit (`unreadable-source`, `wrong-type-source`) are *genuinely* `E/W` against DISC-2, consistent with the row having been typed `E/W` by sibling-analogy rather than by checking the table.

## Solution approach

In the `loom/load/missing-source` row of `code-registry-load.md`, narrow the *Sev* value from `E/W` to `E`, and rewrite the Trigger prose to drop the dead `warning never` disjunct so the row reads as a single-severity error code whose Trigger documents the adjacent silence rule rather than a two-arm severity rule with an empty arm. The Trigger should keep its grounding cross-reference to DISC-2 (`#disc-2`) for the conventional-vs-explicit source split.

## Solution constraints

- Out of scope: the sibling rows `loom/load/unreadable-source` and `loom/load/wrong-type-source`, which are genuinely `E/W` against DISC-2 and must keep their severity.
