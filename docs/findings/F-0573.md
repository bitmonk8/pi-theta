## Finding

`V16a` builds a stateless cross-ceiling arbitration seam `arbitrate(candidate) → { surfaced, masked? }` that runs no breach check and witnesses only what is decidable from a synthesised ceiling-candidate. The leaf's section framing is explicit about this boundary: it lists the seam-decidable properties under "Verified at the seam in isolation" (the surfacing precedence, the within-site sub-check ordering *as encoded by the arbitration*, the at-most-one rule, and the `masked` co-fire) and routes the genuinely temporal CIO-1 and CIO-5 properties to a "Not witnessable at the stateless seam — owned by downstream sites" list.

Two per-bullet `Tests` sentences are not consistent with that framing. The `CIO-2` bullet ends "resolves ceiling #1 (`invoke` depth) **before the callee body**", and the `CIO-3` bullet ends "resolves ceiling #4 (JSON depth) as the first sub-check; **the depth walk is ordered before AJV**". "Before the callee body" and "the depth walk is ordered before AJV" are runtime-site temporal relations between a ceiling firing and other execution at that site — properties the arbitration function never sees, because its only input is a `{check-site tag, satisfied-ceiling set}` candidate. The seam can encode that an `invoke`-entry candidate maps to ceiling #1 and that an AJV-boundary candidate maps to ceiling #4 as the first sub-check; it cannot witness the runtime ordering those trailing clauses assert. Those runtime orderings are owned by the downstream first-enforcement leaves — ceiling #1 by `V15b`, ceiling #4 (JSON/AJV depth) by `V5e`.

The same two strong clauses appear in the paired tests leaf `V16a-T` (CIO-2 "evaluated at `invoke` entry before the callee body"; CIO-3 "the depth walk is ordered before AJV"), so the wording must be aligned in both files to keep the implementation leaf and its tests leaf consistent.

## Solution Space

**Shape:** single

### Recommendation

Reword the `CIO-2` and `CIO-3` `Tests` bullets in both `docs/plan_topics/V16a-ceiling-order-masked.md` and `docs/plan_topics/V16a-T-ceiling-order-masked.md` so each asserts only the seam-decidable mapping and routes the runtime temporal relation to its owning downstream leaf, mirroring the CIO-1/CIO-5 routing pattern the leaf already uses.

- `CIO-2`: drop the trailing "before the callee body" as a seam assertion. Assert that an `invoke`-entry candidate is arbitrated to ceiling #1 (`invoke` depth); attribute the runtime "before the callee body" ordering to `V15b`'s first-enforcement site, witnessed downstream rather than at this stateless seam.
- `CIO-3`: drop "the depth walk is ordered before AJV" as a seam assertion. Assert that an AJV-boundary candidate is arbitrated to ceiling #4 (JSON depth) as the first sub-check *as encoded by the arbitration*; attribute the runtime depth-walk-before-AJV ordering to `V5e`'s first-enforcement site, witnessed downstream.

Keep the `Ships when` and section-framing language as-is — they already describe the seam as witnessing only "the CIO-2/CIO-3/CIO-4 within-site sub-check ordering"; this fix only brings the two per-bullet sentences into line with that scope. No `Deps`, `Spec`, or coverage-matrix change is required.

## Consequence

**Severity:** cosmetic

The section framing directly above the bullets already routes the seam-vs-runtime split correctly, so a reader who absorbs the framing is not misled. The residual cost is local wording inconsistency: the per-bullet sentences read as if the stateless seam asserts a runtime temporal ordering it cannot reach, inviting a seam test that tries to witness "before the callee body" / "depth walk before AJV" and then either widens the seam input (contradicting the stateless framing) or quietly drops the clause.
