## Finding

`match`'s grammar is currently bisected. The left-hand side — the normative six-form Pattern table (wildcard / identifier / literal / constructor / object-schema / array, plus the disambiguation note and the deferred-features paragraph) — lives in `errors-and-results/error-model.md` interleaved with the `Result` / `?` / `MatchError` material. The right-hand side — the `MatchArm` / `ArmBody` productions and the statement-in-arm-body diagnostic — lives in `grammar.md` under `## match arm body`. The `MatchArm ::= Pattern "=>" ArmBody` production cites a non-terminal `Pattern` that the same file does not define, and the grammar appendix's only mention of pattern grammar appears as an aside under `LetStmt` ("full destructuring patterns appear only inside `match` arms (see Errors and Results — Pattern grammar)"), not at the `MatchArm` production where a reader would look. Going the other direction, the Pattern grammar table in `error-model.md` carries no pointer to the grammar appendix that owns the rest of the surface syntax.

A reader of `grammar.md` alone — for example an implementer building the type-grammar / arm-body parser in `V2a` — cannot resolve `Pattern` without already knowing that the table lives in `error-model.md`. A reader of `error-model.md` alone has the table but no signal that the grammar appendix is the home of related surface productions. The grammar appendix's own stated scope ("the few surface-syntax forms that no single topic page owns end-to-end") leaves the split defensible — `match`'s pattern grammar arguably belongs with `match`'s semantics — but the current pages do not honour either resolution: the table is in `error-model.md` *and* the appendix declines to cross-reference it from the `MatchArm` production.

**Introducing commits:** `53f5831` (2026-05-03, "spec: rewrite query primitive, system prompts, and schema syntax") landed the Pattern grammar table in the then-monolithic `spec.md`; `f841b12` (2026-05-04, "spec: resolve Cluster 2 (literal sublanguage + grammar appendix)") created the grammar appendix and decided to leave the pattern table in the errors-and-results material while introducing `MatchArm ::= Pattern "=>" ArmBody` in the new appendix without an inline back-pointer at the `MatchArm` production.

It crystallised in `f841b12`, which authored the appendix with the express scope of owning "surface-syntax forms no single topic page owns end-to-end," authored `MatchArm`/`ArmBody` there, attached a `Pattern` cross-reference only to the unrelated `LetStmt` production, and chose not to relocate or back-reference the pattern table from the appendix. The split-into-topics commit `fecb504` (2026-05-04) is mechanical and did not contribute to the defect. The 100 KB-cap rename pass `f5e89f4` is a verbatim search-and-replace and is also non-contributory.

## Solution Space

**Shape:** single
**State:** shaped

### Recommendation

Leave the table in place and add reciprocal cross-references. The grammar appendix's stated charter explicitly defers forms a topic page already owns end-to-end, and `errors-and-results/error-model.md` owns `match` semantics — pattern grammar is part of those semantics, not a stray surface form. The defect is navigability, not ownership; reciprocal cross-references at the `MatchArm` production and under the pattern table close the navigability gap at one-tenth the surface area of a move, leave the diagnostics-registry "Spec home" cells unchanged, and avoid disrupting `error-model.md`'s opening orientation.

Spec edits:
- `grammar.md`: directly under the `MatchArm ::= Pattern "=>" ArmBody` code block, add a sentence: "`Pattern` is the left-hand side of a `match` arm; its six forms are defined at [Errors and Results — Pattern grammar](./errors-and-results/error-model.md#pattern-grammar-loom-10)."
- `error-model.md`: immediately under the Pattern grammar table, add a sentence pointing at `## match arm body` in the appendix as the home of the arm-body grammar.

Edge cases the implementer must watch:
- Add an explicit `<a id="pattern-grammar-loom-10"></a>` (or equivalent kebab-cased anchor) in `error-model.md` immediately above the `**Pattern grammar (loom 1.0).**` heading so the new `MatchArm`-adjacent link references the table by stable anchor, not by section title.
- Keep the existing `LetStmt` aside in `grammar.md` that already cross-references "Errors and Results — Pattern grammar"; do not collapse it into the new `MatchArm`-adjacent pointer — the two pointers serve different productions.
- Do not edit `queryerror-variants.md`'s "Pattern grammar (loom 1.0) table at the top of this file" wording — the table stays at the top of `error-model.md`, so that pointer remains accurate.

## Consequence

**Severity:** advisory

An implementer of `V2a` working from `grammar.md` alone hits an undefined non-terminal at the `MatchArm` production and must discover, by transitive grep or by reading the `LetStmt` aside, that pattern grammar lives in `error-model.md`. An implementer of `V4a` working from `error-model.md` alone has the pattern table but no signal that the rest of `match`'s grammar is in the appendix. Neither implementer is led to a wrong answer — both files do exist and do cover their halves — but the navigation cost is real and the appendix's claim to own surface syntax is partial.
