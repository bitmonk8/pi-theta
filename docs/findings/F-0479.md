## Problem

`H3a` declares the full member signatures of seven host seam interfaces (`Checkpoint`, `SchemaValidator`, `Clock`, `FileSystem`, `FileWatcher`, `TokenEstimator`, `IdSource`) and its `Adds.` field makes those declarations the single source of each interface's shape — the `V8*` leaves implement against them rather than redeclaring members. `H3a`'s `Tests.` bullets assert only seam-graph isolation and the ambient-primitive access ban; neither checks that each declared interface carries the member set its contract page defines (`host-interfaces-core.md` / `host-interfaces-services.md`, with `Checkpoint` anchored at `host-interfaces-services.md#checkpoint-seam`). A signature that is self-consistent but wrong — a missing method, a wrong parameter or return type, a stale member — passes `H3a` green and surfaces only later at a downstream `V8*` compile, against the very artefact the plan names as authoritative.

## Issue introduction

**Verdict:** present-since-inception (materiality sharpened by a later commit)
**Introducing commits:** c6a664e (2026-06-10, "pi-loom plan: build/update plan for spec.md + review") — created `H3a` with exactly the two `Tests.` bullets (isolation + ambient ban) and no member-set assertion; 592219d (2026-06-11, "pi-loom plan: resolve \"H3a seam interfaces cite no source for member shapes\"") — added the `Adds.` commitment that `H3a` "declares each seam interface's full member signatures … the single source of each interface's shape; the `V8*` leaves implement against them rather than redeclaring members."
**History:** `git log --follow` over `docs/plan_topics/H3a-di-seam-skeleton.md` and `git show c6a664e:` confirm the `Tests.` block has never carried a member-set / signature-completeness assertion since the file first appeared. The gap was latent at inception: until 592219d the `Adds.` field only said "Seam interfaces only; their normative behaviour is implemented by the `V8*` leaves," so nothing yet depended on `H3a` being the authoritative shape. 592219d introduced the single-source-of-shape contract and the "`V8*` implement against them rather than redeclaring" relationship, which is what elevates the always-absent assertion into a real validation defect. The durable fix is the missing assertion, not a revert of 592219d (the single-source design is intentional).

## Solution approach

Add a `Tests.` assertion to `H3a` (`docs/plan_topics/H3a-di-seam-skeleton.md`) that checks each of the seven declared seam interfaces (`Checkpoint`, `SchemaValidator`, `Clock`, `FileSystem`, `FileWatcher`, `TokenEstimator`, `IdSource`) carries the member set its contract page defines — `host-interfaces-core.md` / `host-interfaces-services.md`, with `Checkpoint` resolved at `host-interfaces-services.md#checkpoint-seam` — so a missing, added, or mis-typed member reds out at `H3a` rather than at a downstream `V8*` compile. Update the `Ships when` line so its observable condition includes this assertion. Where a member contract is not expressible as a mechanical structural check, name the reviewer-checked residue rather than leaving the bullet's coverage implicit.

## Solution constraints

- Out of scope: the `V8*` adapters' normative behaviour (owned by `V8a`–`V8e`); `H3a` verifies interface shape only.
- The spec contract pages (`host-interfaces-core.md` / `host-interfaces-services.md`) are read-only for this fix.
