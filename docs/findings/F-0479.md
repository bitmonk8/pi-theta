## Finding

`H3a` declares the full member signatures of seven host seam interfaces — `Checkpoint`, `SchemaValidator`, `Clock`, `FileSystem`, `FileWatcher`, `TokenEstimator`, `IdSource` — and its `Adds.` field makes these declarations the **single source of each interface's shape**: the `V8*` leaves "implement against them rather than redeclaring members." The shape contract therefore lives entirely in `H3a`.

`H3a`'s two `Tests.` bullets assert only (a) that constructing two runtime roots yields isolated seam graphs and (b) the ambient-primitive access ban. Neither bullet checks that each declared interface carries the member set its contract page defines (`host-interfaces-core.md` / `host-interfaces-services.md`, with `Checkpoint` anchored at `host-interfaces-services.md#checkpoint-seam`). A signature that is internally self-consistent but wrong — a missing method, a wrong parameter or return type, a stale member — passes `H3a` green.

Because the `V8*` consumers implement against `H3a`'s declarations rather than the contract pages, a wrong signature is not caught at `H3a` time and surfaces only later, at a downstream `V8*` compile, against the very artefact the plan names as authoritative. The leaf that owns the shape contract verifies everything about the seam graph except the shapes themselves.

## Issue introduction

**Verdict:** present-since-inception (materiality sharpened by a later commit)
**Introducing commits:** c6a664e (2026-06-10, "pi-loom plan: build/update plan for spec.md + review") — created `H3a` with exactly the two `Tests.` bullets (isolation + ambient ban) and no member-set assertion; 592219d (2026-06-11, "pi-loom plan: resolve \"H3a seam interfaces cite no source for member shapes\"") — added the `Adds.` commitment that `H3a` "declares each seam interface's full member signatures … the single source of each interface's shape; the `V8*` leaves implement against them rather than redeclaring members."
**History:** `git log --follow` over `docs/plan_topics/H3a-di-seam-skeleton.md` and `git show c6a664e:` confirm the `Tests.` block has never carried a member-set / signature-completeness assertion since the file first appeared. The gap was latent at inception: until 592219d the `Adds.` field only said "Seam interfaces only; their normative behaviour is implemented by the `V8*` leaves," so nothing yet depended on `H3a` being the authoritative shape. 592219d introduced the single-source-of-shape contract and the "`V8*` implement against them rather than redeclaring" relationship, which is what elevates the always-absent assertion into a real validation defect. The durable fix is the missing assertion, not a revert of 592219d (the single-source design is intentional).

## Solution Space

**Shape:** single

### Recommendation

Add a third `Tests.` bullet to `docs/plan_topics/H3a-di-seam-skeleton.md` that introduces an architectural / type-level test over the seven seam interfaces (`Checkpoint`, `SchemaValidator`, `Clock`, `FileSystem`, `FileWatcher`, `TokenEstimator`, `IdSource`) asserting each declared interface's member set matches the contract its source page defines — `host-interfaces-core.md` / `host-interfaces-services.md`, with `Checkpoint` resolved at `host-interfaces-services.md#checkpoint-seam`. The test must red out when a declared interface omits, adds, or mis-types a member relative to its contract page, so a self-consistent-but-wrong signature fails at `H3a` rather than at a downstream `V8*` compile.

Update the `Ships when` line so its observable condition includes this assertion (it currently names only seam isolation and the ambient-access ban).

Edge cases the implementer must watch: the test target is the seven interfaces' shapes, not the `V8*` adapters' behaviour (behaviour stays owned by the `V8*` leaves per the existing `Adds.`); the `Checkpoint` contract anchor is on `host-interfaces-services.md` even though `V8a` lists it under `host-interfaces-core.md`. Where the spec member contracts are not expressible as a mechanical structural check, the irreducible residue is reviewer-checked — name that disposition rather than leaving the bullet's coverage implicit.

## Consequence

**Severity:** correctness

A seam interface declared with the wrong member set ships through `H3a`'s green gate and is detected only at a downstream `V8*` compile. Because `H3a` is the declared single source of shape, the late failure invites a fix in the wrong direction (editing the consumer to match the wrong declaration), and the leaf that exists to own the shape contract provides no evidence the shape is correct.
