## Finding

The *No globals, statics, singletons* cross-cutting rule in `conventions.md` describes the H3a ambient-primitive scan's timing-primitive list as a manually-maintained derivation of the `host-interfaces-services.md` `Clock`/`WallClock` ban (PIC-12). It states the operative constraint correctly — the list must be a superset of the spec ban (`scan list ⊇ spec ban`) — and then appends a redundant temporal snapshot: "and currently equal to the PIC-12 enumeration `Date.now`/`performance.now`/`Date.prototype.getTime`/`setTimeout`/`clearTimeout`".

The snapshot records a point-in-time equality that the rule's own surrounding prose immediately undermines: the same parenthetical notes the list is re-reconciled against the current spec ban by release-time residue-inspection item 9, precisely because the spec ban can be extended. If PIC-12 grows, the snapshot becomes stale with no gate to flag it, while the `⊇` invariant and the item-9 reconciliation continue to express the actual constraint. The snapshot adds no normative content beyond the invariant and is a stale-prone restatement.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 1c9ede1 — pi-loom plan: resolve "ambient-primitive ban list lockstep over-promise" (2026-06-18, Thomas Andersen)
**History:** The "currently equal to the PIC-12 enumeration …" snapshot clause was added to the *No globals, statics, singletons* parenthetical (and, in the same commit, to release-time residue-inspection item 9) by commit 1c9ede1, a prior plan-review fix that relaxed an over-promised strict-equality "lockstep" claim to the `scan list ⊇ spec ban` superset invariant. That fix introduced the redundant snapshot equivalence alongside the durable invariant; the snapshot was not present before this commit.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/conventions.md`, within the *No globals, statics, singletons* cross-cutting rule's parenthetical, strike the clause `and currently equal to the PIC-12 enumeration `Date.now`/`performance.now`/`Date.prototype.getTime`/`setTimeout`/`clearTimeout``. The surrounding text then reads "…required to cover at least every primitive the spec ban enumerates (scan list ⊇ spec ban), and is re-reconciled against the current spec ban by the [loom 1.0 release-time residue inspection]…", retaining the `⊇` invariant and the item-9 reconciliation pointer. Do not touch the spec (`host-interfaces-services.md` PIC-12 is read-only for this fix). The identical sibling snapshot in residue-inspection item 9 is the subject of its own finding (see Related Findings) and is fixed there.

## Consequence

**Severity:** cosmetic

If left, the snapshot enumeration goes stale the moment the PIC-12 `Clock`/`WallClock` ban is extended, since nothing mechanically updates it; a reader could then mistake the stale list for an authoritative equality. No gate or leaf depends on the snapshot — the `⊇` invariant plus item-9 reconciliation carry the real constraint — so the defect is pure wording cruft, not a correctness risk.
