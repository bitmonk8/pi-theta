## Finding

`V9j` is the closing leaf for `cka-35` (coverage-matrix.md:139), which bundles the provider-errorâ†’`QueryError` mapping table, `TransportError.retryable` population, context-overflow token extraction, **stop-reason classification**, and the **per-provider seed-field mapping** MUSTs. Its **Adds** correctly enumerates all of these: the classifier does "status / `errorMessage` classification, `retryable`, context-overflow extraction, stop-reason, the per-provider seed-field table," and `complete()` carries "the seed-field mapping."

The **Tests**, however, assert these surfaces only generically. The classifier bullet says "the provider classifier maps **each** provider response to its `QueryError` variant and `retryable` flag" as one undifferentiated assertion rather than a per-row check, so a wrong `QueryError`-variant or `retryable` value for any provider row that the single exercised path does not happen to drive passes green. The envelope bullet asserts "the mapped seed field" (singular) â€” one provider's seed field, not the per-provider table the Adds promises â€” so a wrong seed-field row for a non-exercised provider also passes. Stop-reason classification appears in the Adds and in the `cka-35` mapping but has **no Tests bullet at all**, so it can ship entirely unexercised.

The **Ships when** ("`npm test` asserts the providerâ†’`QueryError` table and the deterministic `complete()` envelope") claims a table-level gate the Tests do not deliver. The drift gate in `V18c` (the per-provider seed-field fixture and `Api`-coverage assertion) is a version-bump trigger that catches a renamed/retyped/moved seed field on a Pi bump; it does not establish steady-state per-row correctness of `V9j`'s own mapping at the leaf's ship, and it does not cover the `QueryError`-variant or stop-reason mappings at all.

## Consequence

**Severity:** correctness

A wrong per-provider seed-field row, a wrong providerâ†’`QueryError`-variant/`retryable` mapping, or a missing stop-reason classification for any provider not on the single exercised path ships green; the leaf's own gate cannot witness it, and `cka-35` is treated as closed. Two faithful implementers could populate divergent provider tables and both pass `npm test`, so the binder inference call would misclassify real provider responses in production while the plan reports the obligation covered.

- Update `V9j`'s **Ships when** so the claimed "providerâ†’`QueryError` table" gate matches what the Tests now assert (per-provider seed field, per-row variant/`retryable`, and stop-reason), keeping the existing deterministic-envelope clause.
