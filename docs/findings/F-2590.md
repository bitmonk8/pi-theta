## Finding

The intro link `[Scope of a Loom File](#scope-of-a-loom-file)` in `overview.md`, and the `[Overview — Scope of a Loom File]` link in `pi-integration-contract/subagent.md`, capitalise "Loom"/"File" against the documented lowercase convention for the file-unit *loom* (glossary.md `loom (file unit)`: "Lowercase") and against the target heading `### Scope of a loom file`.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `81bf52a` (duplicate `## Overview`); `53b8970` (link-text casing mismatch)
**History:** The two halves of this finding entered the corpus independently, each as the by-product of an edit that was locally correct.

- Duplicate `## Overview` H2: `overview.md` has carried a `## Overview` H2 since its creation in `fecb504` ("Split spec.md and plan.md into per-topic / per-phase files", 2026-05-04). `overview-and-orientation.md` acquired its own `## Overview` H2 in `81bf52a` ("docs: collapse spec/ section pages into spec_topics/", 2026-06-11), confirmed by `git log --follow -S "## Overview"` on that file. Neither commit is wrong in isolation; the duplication emerges from the two pages coexisting in `spec_topics/` with the same heading after `81bf52a`.
- Link-text casing mismatch: at creation (`fecb504`) `overview.md` had heading `### Scope of a Loom File` and matching capitalised link text — no mismatch. `53b8970` ("pi-loom spec: resolve 'loom / Loom / pi-loom — three referents, no glossary disambiguation'", 2026-05-05) lowercased the heading to `### Scope of a loom file` (confirmed by `git log --follow -S` on both the old and new heading strings) but left the link text capitalised, creating the heading↔link disagreement. The capitalised link in `subagent.md` rides the same pre-`53b8970` casing.

The corpus is git-tracked; both attributions are first-hand from `git log --follow` history.

## Solution Space

**Shape:** single

### Recommendation

Change the link text in both sites to `Scope of a loom file` (the URL fragment `#scope-of-a-loom-file` is already correct and unchanged). Grep the corpus for any further `Scope of a Loom File` link occurrences and normalise them in the same edit.

## Consequence

**Severity:** cosmetic

A reader navigating the two `## Overview` sections cannot tell which page is authoritative for a given claim, and a cross-reference that lands on "Overview" is ambiguous. The capitalised link text contradicts the documented lowercase convention for the file-unit *loom*. Neither defect changes implementer behaviour; both are organisation/wording.
