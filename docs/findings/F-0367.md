## Problem

The `FRNT-1` Tests bullet, carried identically in both `V6e` and `V6e-T`, references "the CIO-4 `max_rounds`-final branch" as bare inline prose with no link to its owning spec definition. Both leaves cite only `frontmatter-fields-b-and-templates.md` in their `Spec.` field, so a reader checking the typed-path unreachability assertion (`tool_loop_exhausted` is unreachable on that path) cannot reach the CIO-4 definition at `ceilings-3-and-4.md#cio-4` without an out-of-band search. Sibling leaves that depend on CIO-4 already follow the citation convention: `V13c` cites the page in `Spec.` and `V16a` links precise CIO anchors.

**Verdict:** single-commit
**Introducing commits:** 5117cb5 — pi-loom plan: resolve "V6e FRNT-1 tool_loop_exhausted not scoped to untyped-query path" (2026-06-16, Thomas Andersen)
**History:** Before 5117cb5 the `FRNT-1` bullet read simply "exhaustion produces `QueryError{tool_loop_exhausted}`", with no mention of CIO-4. That commit — a prior plan-review fix scoping exhaustion to the untyped-query path — rewrote the bullet in both `V6e` and `V6e-T` to add the typed-query "CIO-4 `max_rounds`-final branch" sentence, but did not add a cross-reference to the owning `ceilings-3-and-4.md` page. The unlinked CIO-4 reference is therefore exactly as old as that fix.

## Solution approach

Add a cross-reference from the bare "CIO-4 `max_rounds`-final branch" phrase in the `FRNT-1` Tests bullet to its owning anchor `ceilings-3-and-4.md#cio-4`, mirroring how `V13c` and `V16a` cite the same page. Apply the same edit to both `V6e-respond-repair-tool-loop.md` and `V6e-T-respond-repair-tool-loop.md`, whose `FRNT-1` text is identical.
