## Problem

The §4 numeric-placeholder **Rule** paragraph in `docs/spec_topics/diagnostics/placeholder-rendering-a.md` declares that `Infinity` and `NaN` "are unreachable for these placeholders by construction" and then, in the same sentence, says a renderer that encounters one MUST surface it through `loom/runtime/internal-error`. The formal "by construction" unreachability claim and the defensive MUST pull in opposite directions: a strict reader treats the branch as dead code and may drop it (and its conformance vector), while a defensive reader implements it. The two readings produce observably different end states for a future arithmetic-bound regression — clean `internal-error` routing versus a raw `Infinity`/`NaN` token leaked into a panic message — silently weakening the runtime-defect safety net §4 plugs into.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `61ea1a5` (2026-05-06, "pi-loom spec: resolve \"Diagnostic message placeholders have no defined rendering\"")
**History:** The §4 paragraph was introduced verbatim by `61ea1a5`, which created the eight placeholder-rendering categories in the then-monolithic `spec_topics/diagnostics.md`. Subsequent commits — `f5e89f4` (size-cap split of the diagnostics shard into `diagnostics/placeholder-rendering-a.md` and `…-b.md`), `7995d3c` (GOV-15 relocation), and `72e2c31` (closure-clause widening) — relocated and surrounded the paragraph but left the §4 **Rule** sentence untouched (`git log -G "unreachable for these placeholders"` returns only `61ea1a5` and `f5e89f4`; `git log --follow` on the post-split file returns only the three relocations). The "unreachable by construction" framing and the MUST therefore co-exist in the §4 rule from the moment the rule was authored; no later commit altered their relationship.

## Solution approach

Rewrite the closing sentence of §4's **Rule** paragraph to drop the formal "unreachable by construction" claim that licenses the dead-code reading, reframing the bounded emitting sites as descriptive justification that any non-finite value reaching these placeholders is a runtime defect outside the closed loom 1.0.0 panic-source list. Preserve the defensive MUST and name its destination explicitly by forward-linking the runtime-defect surface at [Errors and Results — Runtime panics](../errors-and-results/error-model.md#runtime-panics).
