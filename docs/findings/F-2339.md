## Problem

`V8a`'s Adds enumerates the `Checkpoint` seam deliverables with a bare token `per-invocation` that modifies no noun, so it admits two materially different DI-wiring readings: the cancellation signal is scoped per invocation, or one `Checkpoint` instance is constructed per invocation. The cited spec PIC-10 (`pi-integration-contract/host-interfaces-services.md`, anchor `#checkpoint-seam` / `#pic-10`) is unambiguous and means the latter — one `Checkpoint` per `loomAbort`, with `invoke` parent and child each owning their own instance from the same factory. The Adds is the authoritative deliverable enumeration for the leaf and does not surface this; an implementer working from it could wire a single shared `Checkpoint`, silently defeating the per-invocation isolation that makes the no-retroactive-rewrite and no-tail-synthesis race rules deterministic.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e — pi-loom plan: build/update plan for spec.md + review (2026-06-10, Thomas Andersen)
**History:** The `V8a` leaf file was first authored in c6a664e (the initial plan build against spec.md), and the bare unqualified `per-invocation` token was present in its Adds from that first version. `git log -S 'per-invocation'` on the file reports only c6a664e; the three later commits that touched the file (d36eb03 loop-iter substrate seam, cb8737a over-bundling split, 86ab9af Spec-field fix) never refined the term. The ambiguity has therefore existed unchanged since the leaf's inception.

## Solution approach

Clarify the `per-invocation` token in the `V8a` Adds Checkpoint-seam clause to name the PIC-10 property: one `Checkpoint` instance is constructed per loom invocation (per `loomAbort`), and for `invoke` the parent and child each own their own instance from the same factory, neither observing the other's `before(...)` calls. Optionally pin the `#checkpoint-seam` (or `#pic-10`) anchor in the leaf's Spec field so the production-wiring rule is one click away.

## Solution constraints

- The spec is read-only for this fix; `host-interfaces-services.md` (PIC-10) is the authoritative source for the property and must not be altered.
