## Problem

The Per-loom registration `label` field-derivation bullet defines `label` as "the loom file's basename with hyphens preserved and the leading character capitalised", but its two worked examples (`summarise.loom → "Summarise"`, `code-review.loom → "Code-review"`) strip the `.loom` extension before capitalising. Under the natural Node `path.basename` reading, the prose rule and the examples are inconsistent: a prose-following implementer materialises `"Summarise.loom"` while an examples-following implementer materialises `"Summarise"`. The divergence surfaces in operator-visible slash-command tool labels and model-visible tool metadata with no diagnostic to flag the mismatch.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `cb43deb` (2026-05-04, "pi-loom spec: resolve \"ToolDefinition is mis-shaped: missing label, and parameters must be a TypeBox TSchema\"")
**History:** Commit `cb43deb` first added the `label` field-derivation bullet to the per-loom registration paragraph (then in `spec_topics/pi-integration-contract.md`, later relocated to `extension-bootstrap-and-per-loom.md` by `fecb504`'s split), introducing both halves of the contradiction in the same edit: the prose word "basename" and the worked examples `summarise.loom → "Summarise"` / `code-review.loom → "Code-Review"`, the latter case-corrected to `"Code-review"` by `6d8b472` (2026-06-05, "resolve 'label derivation rule and example capitalisation disagreement'") without touching the basename / extension-strip disagreement. No earlier revision carries the rule; subsequent edits to the surrounding paragraph (`841a085`, `bf85092`, `79ea4e8`, `9568906`, `6a42c1a`, `4b4382f`) did not touch this bullet's wording. The defect has been present since the rule first entered the corpus.

## Solution approach

Rewrite the `label` rule's input phrase in the Per-loom registration bullet so it names the loom filename stem (the filename with the trailing `.loom` extension removed) rather than the "basename", aligning the prose with the worked examples it already carries. Leave the capitalisation rule, both worked examples (`summarise.loom → "Summarise"`, `code-review.loom → "Code-review"`), and the synthesised typed-query one-shot literal `"Loom typed-query response"` unchanged.
