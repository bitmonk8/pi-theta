## Finding

`H4a` requires the in-process Pi session double to reproduce a Pi-facing surface: the factory's registration calls (`pi.registerMessageRenderer`, `pi.registerCommand`, `pi.registerFlag`, `pi.registerTool`, `pi.on`) and four fidelity behaviours (streamed-token order relative to `ctx.waitForIdle()`, single-turn prompt-mode append, the `pi.on` cancel-forward subscription, cancellation propagation). The leaf names these by prose but cites no spec page that enumerates the `ExtensionAPI` / `ExtensionContext` / `ExtensionCommandContext` member set the double must implement to be a faithful stand-in.

That enumeration exists and is authoritative. `host-interfaces-core.md` carries the closed loom-load-bearing member surface in named blocks — `ExtensionContext` (`#extensioncontext-interface`: `cwd`, `signal`, `sessionManager`, `modelRegistry`, `model`, `ui`, `isIdle`, `abort`, `getContextUsage`, `hasPendingMessages`, `compact`, `shutdown`, `getSystemPrompt`), `ExtensionCommandContext` (`#extensioncommandcontext-interface`: adds `waitForIdle()`), and `AgentSession` (`#agentsession-interface`) — and `conversation-drive.md` PIC-18 (`#pic-18`) enumerates the exact five `pi.on` turn-lifecycle events the cancel-forward subscription consumes (`tool_call`, `tool_result`, `message_update`, `turn_end`, `agent_end`). `H4a` reaches none of these from its own text.

With no pointer to the closed surface set, an implementer assembles the double member-by-member from whatever the harness tests happen to exercise. The four fidelity self-checks only assert four behaviours, so a double that omits members the double does not itself exercise but downstream harness-driven leaves will (e.g. `ctx.getContextUsage`, `ctx.modelRegistry`, the read-only `ReadonlySessionManager` `Pick<>`) passes `H4a`'s gate while being under-built for its consumers.

## Issue introduction

**Verdict:** present-since-inception

The fidelity-contract axes were elaborated across later commits (`eeb0014`, `e7f14dd`, `27e12be`, all 2026-06-10) but none added a member-surface citation: `git log -G 'host-interfaces-core' -- docs/plan_topics/H4a-factory-shell-and-harness.md` returns no commits, and the current file contains zero `host-interfaces-core` references. The gap has therefore existed continuously since the session-double concept entered the leaf; no specific later edit introduced it.

## Solution Space

**Shape:** single

### Recommendation

In `H4a`, at the point the Adds prose names the session-double fidelity contract (and the factory's `pi.register*` / `pi.on` registration surface), add a cross-reference to the canonical SDK touched-surface enumeration the double must reproduce:

- `host-interfaces-core.md#extensioncontext-interface` — the `ExtensionContext` member surface loom touches.
- `host-interfaces-core.md#extensioncommandcontext-interface` — the `ExtensionCommandContext` per-handler subtype (adds `waitForIdle()`), the context the harness's slash dispatch drives against.
- `conversation-drive.md#pic-18` — the closed five-member `pi.on` turn-lifecycle event set (`tool_call`, `tool_result`, `message_update`, `turn_end`, `agent_end`) the cancel-forward subscription consumes.

The citation anchors the double to a closed, owned surface set so the implementer builds against the enumerated members rather than inferring them from the harness's own assertions. `H4a` is a horizontal leaf that cites a `Convention.` field rather than a `Spec.` field (per `conventions.md` §Leaf format), so add the reference as a cross-link in the Adds prose where the double is described — do not introduce a `Spec.` field that the horizontal-leaf format does not use. The cited spec pages are read-only for this fix; no spec edit is required (the enumerations already exist).

## Consequence

**Severity:** advisory

A double-implementer with no pointer to the canonical member-surface enumeration builds the double member-by-member against the harness's own assertions. `H4a`'s four fidelity self-checks do not cover the full touched-surface set, so an under-implemented double can ship green and only surface as a gap when a later harness-driven leaf (`M`, the `V*` slices) reads a member the double never modelled.
