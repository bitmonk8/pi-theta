## Problem

In `docs/plan_topics/coverage-matrix.md`, the `cancellation.md` (CNCL) §*Race semantics — swallowing-handler attachment on every abandonable Promise* row closes its obligation with a flat closing-leaf cell of five backtick IDs (`V17a`, `V14f`, `V13f`, `V15h`, `V9o`) and no inline annotation saying which abandonable-Promise site each leaf witnesses. The spec rule enumerates exactly four abandonable-Promise sites, each closed by a distinct per-site leaf, with `V17a` covering only the `Checkpoint`-seam substrate; a reviewer reading the row in isolation cannot recover that one-leaf-per-site partition. The row breaks the inline per-leaf annotation convention the `DISC-4` and `PIC-17, PIC-18` rows in the same file already follow. `H5d` tokenises closing-leaf cells to backtick IDs and applies at-least-one transitive-completeness checking by design, so a silently dropped per-site leaf keeps the cell green — making this an auditability/traceability gap in the matrix row, not a coverage hole in the leaves.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `1db2787` (pi-loom plan: resolve "cancellation.md swallowing-handler attachment MUST has no coverage-matrix row")
**History:** The corpus is git-tracked. The swallowing-handler row was born in `1db2787` (2026-06-17) already carrying a flat, un-annotated five-ID closing-leaf cell (`V17a, V14a, V13f, V15a, V9i` at the time). Subsequent commits retargeted three of those IDs as their owning leaves were split — `V14a`→`V14f` (`5a1915d`), `V15a`→`V15h` (`5f4d271`), `V9i`→`V9o` (`6d7661d`) — but none added per-site annotations. The missing-annotation defect has therefore been continuously present since the row's introduction; the leaf-ID retargets are incidental and never touched the annotation gap.

## Solution approach

Annotate each backtick leaf-ID in the closing-leaf cell of the swallowing-handler row with the site it closes, following the inline parenthetical pattern the `DISC-4` and `PIC-17, PIC-18` rows already use. Record the mapping: `V17a` — the `Checkpoint`-seam swallowing-handler substrate; `V14f` — the code-side `execute()` Promise site; `V13f` — the `@`-query underlying provider Promise site; `V15h` — the `invoke` child top-level execution Promise site; `V9o` — the subagent-mode `AgentSession.abort()` Promise site. Keep the four per-site labels aligned with the four sites `cancellation.md` §*Race semantics — swallowing-handler attachment* enumerates.

## Solution constraints

- Out of scope: the spec is read-only (do not edit `cancellation.md` or any other spec file), and the five leaf files (`V17a`, `V14f`, `V13f`, `V15h`, `V9o`) already name their sites and must not be modified — the change is confined to the one matrix row.
- Do not change `H5d`'s at-least-one closing-leaf tokenisation semantics; the annotations are for human auditability of the row only and are not expected to make `H5d` hard-gate per-site presence.
