## Finding

`V3a` (Expression evaluator) carries a Tests bullet — "a short-circuited right operand's observable effect does not run" — and an Adds clause naming "left-to-right short-circuit with skipped-operand observability". The spec ([`expressions.md` §Evaluation order and short-circuiting](../spec_topics/expressions.md#evaluation-order-and-short-circuiting)) defines the *observable* of a skipped operand strictly in terms of side-effecting constructs: a short-circuited right operand's `@`-queries, tool calls, and `invoke` children "do not run, produce no transcript entries, and spend no tokens".

None of those constructs exists at `V3a`'s declared dependency scope. `V3a`'s `Deps` are `V3a-T`, `V2b` (type-compatibility engine), and `V2c` (runtime value model) — `@`-queries land in V13, tool calls in V14, and `invoke` children in V15. Loom expressions are otherwise pure, so a short-circuited pure operand produces the same final `boolean` value whether or not it ran; there is no spec-sanctioned observable at this scope that distinguishes "evaluated" from "skipped".

An implementer therefore cannot make `V3a-T`'s short-circuit test red for the intended reason. They are pushed to invent something the leaf does not own: a white-box evaluator probe that records which operands were visited, or a panic/throw path used as a side-effect proxy (panics are V4b, also outside `V3a`'s `Deps`). Two reasonable implementers diverge on the witness mechanism, and a vacuously-passing short-circuit test — the *No silent test skipping* failure mode the plan forbids — is a live outcome.

## Solution Space

**Shape:** multiple

### Option A — Sanction an evaluator-level non-evaluation probe at V3a

**Approach.** Define, in `V3a`/`V3a-T`, a sanctioned evaluator-level observable that witnesses non-evaluation of a skipped operand (e.g. an evaluator-visit record the test inspects), and rewrite the Tests bullet to assert against that named contract instead of an unspecified "observable effect".

**Plan edits.** In `docs/plan_topics/V3a-expression-evaluator.md` and `docs/plan_topics/V3a-T-expression-evaluator.md`, replace the bullet "a short-circuited right operand's observable effect does not run" with text naming the concrete evaluator-level witness the test reads (the visited-operand record / evaluation trace the evaluator exposes for test). Align the `Ships when` "proves short-circuit observability" phrase to the same witness.

**Spec edits.** None — the witness is a test-facing evaluator contract, not a product behaviour the spec governs.

**Pros.** Keeps the EXPR short-circuit closure inside its `cka-3` closing leaf `V3a`; no cross-leaf coverage move; testable at `V3a`'s existing `Deps`.

**Cons.** Introduces a test-only evaluator surface that has no spec basis; risks the probe diverging from the real side-effect observables the spec actually cares about.

**Risks.** A probe that only witnesses "visited / not visited" under-tests the spec's stronger claim (no transcript entries, no tokens) for side-effecting operands; that stronger claim still needs a downstream witness.

### Option B — Relocate the side-effecting short-circuit witness to a leaf that owns a spec-named side-effecting construct

**Approach.** Narrow `V3a`'s short-circuit assertion to the pure operator semantics it can actually witness (correct `boolean` result, left-to-right order over pure operands), and move the "skipped operand's side effect does not commit" assertion to a leaf whose `Deps` include `@`-queries / tool calls / `invoke` children.

**Plan edits.** In `docs/plan_topics/V3a-expression-evaluator.md` / `V3a-T`, restrict the short-circuit bullet to the value-and-order semantics provable at scope. Add the side-effect-non-evaluation assertion (short-circuited operand runs no `@`-query / tool call / `invoke`, produces no transcript entry, spends no tokens) to a leaf that already depends on one such construct — `V13a` (query), `V14a` (tool calls), or `V15a` (invocation). If no existing leaf is a clean home, add a `<new>` leaf for it. Update `coverage-matrix.md` `cka-3` (and any consuming row) so the relocated facet's closing leaf is cited.

**Spec edits.** None.

**Pros.** Each assertion is witnessed by a spec-defined observable at a scope where it exists; matches the spec's own observability framing.

**Cons.** Splits the EXPR short-circuit obligation across leaves; requires a coverage-matrix update and care that the relocated facet remains traceable (split-row joint-completeness, residue-inspection item 7).

**Risks.** If the relocated facet is added to a multi-leaf row without a per-facet citing test, it can ship untested while the row's gate stays green.

### Recommendation

Adopt **Option B**: it grounds the side-effect-non-evaluation assertion in the spec's own observables rather than a test-only probe. Keep `V3a`'s short-circuit bullet scoped to the pure operator-result and left-to-right-order semantics it can witness at its `Deps`, and place the side-effect-non-evaluation assertion on the earliest leaf whose `Deps` include a spec-named side-effecting construct (`@`-query / tool call / `invoke`), updating `coverage-matrix.md` so the relocated facet has a cited closing leaf. When relocating, ensure the moved facet carries its own facet-naming citing test so it is not absorbed vacuously into a multi-leaf row. Use the literal `<new>` placeholder if a fresh leaf is required; do not invent a final ID. Honour the plan's leaf-ID scheme and the `no-invented-ids` rule per `conventions.md`.

## Consequence

**Severity:** correctness

The short-circuit-observability assertion cannot be witnessed by any spec-defined observable reachable from `V3a`'s `Deps`, so the `V3a-T` test cannot fail-red for the intended reason. Implementers either invent a divergent white-box probe / panic proxy the leaf does not own, or ship a vacuous green test that certifies nothing about skipped-operand behaviour.
