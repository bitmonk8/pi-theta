## Finding

The `H4b` heading introduces two names for the same artefact: `Response-programming surface` (the primary name) and `(scripted-injection contract)` (a parenthetical). The heading appears in both `docs/plan.md` (the Horizontal-phases link list) and as the H1 of `docs/plan_topics/H4b-response-programming-surface.md`.

The leaf body uses `response-programming surface` throughout — in `Adds.`, both `Tests.` bullets, and `Ships when.`. The parenthetical token `scripted-injection contract` never recurs anywhere in the leaf body; the nearest body phrasing is "this harness's own scripting contract" / "single shared scripting contract", which is not the same token. The heading therefore presents a synonym that is never tied back to the primary name by an equivalence statement, leaving a reader to decide whether `scripted-injection contract` denotes the same surface or a distinct one.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** fd6169b ("pi-loom plan: resolve \"H4a bundles three independent units into one leaf\"", 2026-06-12)
**History:** The H4b leaf was created in fd6169b when H4a was split into H4a + H4b. The same commit added both the leaf file (with the H1 `# \`H4b\` — Response-programming surface (scripted-injection contract)`) and the `docs/plan.md` link line carrying the identical parenthetical. The dual-name heading has been present from the leaf's first revision; no later commit added an equivalence statement to the body. `git log -S 'scripted-injection contract'` over both files reports fd6169b as the sole introducing commit.

## Solution Space

**Shape:** single

### Recommendation

Drop the parenthetical from both heading occurrences so each reads `H4b — Response-programming surface`:

- `docs/plan.md` line 31: change the link text `` `H4b` — Response-programming surface (scripted-injection contract) `` to `` `H4b` — Response-programming surface ``.
- `docs/plan_topics/H4b-response-programming-surface.md` line 1: change the H1 `# `H4b` — Response-programming surface (scripted-injection contract)` to `# `H4b` — Response-programming surface`.

Both occurrences must change together so the link text and the target H1 stay identical.

Edge case: if the "contract" framing carries meaning the author wants to keep, an acceptable alternative is to retain the primary heading name and add an inline equivalence in the leaf body the first time the surface is named (stating that the response-programming surface is the leaf's scripted-injection contract), rather than leaving the synonym only in the heading. Pick one of the two — do not leave the parenthetical standing alone.

## Consequence

**Severity:** cosmetic

A reader encountering `scripted-injection contract` in the heading has no anchor mapping it to the `response-programming surface` the body defines, and may search for a separate contract artefact that does not exist. No implementer behaviour diverges — the body uses one term consistently — so the impact is confined to heading wording.
