## Problem

The `PIC-1` Tests bullet in both `V9d-runtime-event-channel.md` and its paired
`V9d-T-runtime-event-channel.md` scopes the `masked` field's per-site reachable
domain to "in V1", meaning the initial release. `V1` is a reserved plan-phase
token and a live slice group in this plan (the lexer slice, leaves
`V1a`/`V1b`), so the clause is ambiguous: a literal reader can parse it as
"co-occurring with the `V1*` lexer slice" rather than "in loom 1.0". The wording
also violates the `conventions.md` (Leaf format) rule reserving `V1`–`Vn` for
plan phases and requiring prose about the initial release to write "loom 1.0" or
"the initial release".

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e ("pi-loom plan: build/update plan for spec.md + review")
**History:** `git log --follow` on both `docs/plan_topics/V9d-runtime-event-channel.md` and `docs/plan_topics/V9d-T-runtime-event-channel.md` shows a single commit each, c6a664e, which created the leaf pair. `git log -S'in V1'` on the leaf file returns the same commit, confirming the "in V1" wording was present from the leaf's first revision rather than introduced by a later edit. The defect has existed since the V9d leaf was authored.

## Solution approach

In the `PIC-1` Tests bullet of both `V9d-runtime-event-channel.md` and
`V9d-T-runtime-event-channel.md`, replace the substring `in V1` with
`in loom 1.0`.
