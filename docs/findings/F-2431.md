## Finding

Rule 5 ("`ambiguous.candidates` is not rendered in loom 1.0") trails the speculative parenthetical "(and the array-truncation rules a future revision would need)"; the forward pointer to Future Considerations is legitimate, the speculative design detail is not.

## Issue introduction

**Verdict:** multi-commit-interaction (accretion across distinct commits)
**Introducing commits:**
- `83802db` (2026-05-04) "spec: collapse to single typed argument style; add LLM-driven slash-command binder" — the binder section's inception; the Claude Haiku / GPT-4o-mini / Gemini Flash example names are present from this commit.
- `0fa81fa` (2026-05-05) "spec: pin binder model resolution; document envelope two-arm rationale (Decisions #5, #6)" — introduced the rule-5 "deferred along with the binder refinement loop" deferral text.
- `6efbd8a` (2026-05-05) "pi-loom spec: resolve 'System-note 120-codepoint cap truncation unit'" — added the `Array.from(str).length` / `string.length` JS-primitive measurement note.
- `52219c4` (2026-05-06) "Address all 41 spec-review findings" — introduced the duck-typed `(model as { strictCapable?: boolean }).strictCapable` cast.
- `3cd40c5` (2026-06-03) "pi-loom spec: resolve 'Compact-transcript byte-exact rendering — state the rationale'" — added the `Why the transcript bytes are pinned` rationale block.

**History:** `git log -S` over each defect token (across the pre-`docs/` and pre-split monolith histories, following the `f5e89f4` size-cap split and the `31ff060` move) shows no single introducing change. The cluster accreted across five distinct commits over a month. Two of the items — the transcript-rationale block (`3cd40c5`) and the code-point-cap measurement note (`6efbd8a`) — entered as well-intentioned prior review-fix additions, so the durable fix is to relocate/demote the accreted prose, not to revert any one commit.

## Solution Space

**Shape:** single

### Recommendation

Delete the "(and the array-truncation rules a future revision would need)" parenthetical; keep the normative "not rendered in loom 1.0" statement and the forward reference to *Future Considerations — Binder refinement loop*.

## Consequence

**Severity:** advisory

Stale product names will date and invite spurious "fix the model list" churn; the unlabelled rationale buries the normative rules behind motivation; and the pinned host primitives (the TS cast, the `string.length`/`Array.from` note) read as binding implementation mandates rather than guidance, so an implementer in a non-TS or differently-shaped host may believe the spec forbids their otherwise-conforming approach. Implementation can still proceed correctly in every case.
