## Finding

The `V15a` implementation leaf (`docs/plan_topics/V15a-invocation-core.md`) carries an `ERR-13` Tests bullet — the *delegated live-carrier witness for `V4f`'s completed-callee-finality deferral* ("an `invoke` child driven to completion on the live `V15a` surface, then a downstream `?`/panic/cancel fired, leaves the completed callee's side effect in place with no compensating turn injected") — and lists `V4f` in its **Deps**. The paired tests-task `V15a-T` (`docs/plan_topics/V15a-T-invocation-core.md`) contains neither: its Tests block has no `ERR-13` bullet, and its **Deps** are `V10a`, `V2b`, `V3d`, `V8b` (no `V4f`).

The per-phase TDD ritual (`conventions.md` §"MVP and vertical features — tests-task → implementation-task pairing") requires the `-T` tests-task to "Write the failing tests for *every* spec REQ-ID the feature introduces," landing them red-first before the implementation task. Because the impl's Tests/Deps strictly exceed the tests-task's for the `ERR-13` co-witness, that assertion would be authored during implementation rather than test-first, never passing through the red-first gate the ritual mandates.

This is one instance of a corpus-wide pattern: the same asymmetry exists on the sibling ERR-13 delegated-witness leaves `V13c` and `V14a`, whose impl Tests/Deps carry the `ERR-13` bullet + `V4f` while `V13c-T`/`V14a-T` omit both.

## Issue introduction

**Verdict:** single-commit-introduced
**Introducing commits:** 7bd65a7
**History:** `V15a-T` was created at `c6a664e` (initial plan build) with no `ERR-13` bullet — correct at the time, since the feature did not yet carry the delegated live-carrier obligation. Commit `7bd65a7` ("resolve \"V4c/V4f defer ERR-8 to ERR-13 live-surface confirmation to downstream owners that never re-assert it\"") added the `ERR-13` co-witness bullet to `V15a`'s Tests and `V4f` to its Deps — making the identical edit to the `V13c` and `V14a` impl leaves — but did not mirror the bullet or the dep into the paired `-T` tests-tasks (`V15a-T`, `V13c-T`, `V14a-T`), introducing the test-first asymmetry across all three co-witness leaves at once.

## Solution Space

**Shape:** single

### Recommendation

Bring `V15a-T` into line with the `ERR-13` obligation `V15a` already declares:

- Add an `ERR-13` bullet to `V15a-T`'s **Tests** mirroring `V15a`'s, retaining the live-`V15a`-surface framing — e.g. `` `ERR-13` (delegated live-carrier witness for `V4f`'s completed-callee-finality deferral): an `invoke` child driven to completion on the live `V15a` surface, then a downstream `?`/panic/cancel fired, leaves the completed callee's side effect in place with no compensating turn injected. ``
- Add `V4f` to `V15a-T`'s **Deps** (currently `V10a`, `V2b`, `V3d`, `V8b`).

Edge case: if the red-first `ERR-13` test references no not-yet-existing `V4f` production symbol, `V4f` is not strictly required to gate `V15a-T`'s pickability; adding it nonetheless matches the dependency `V15a` already declares for the same bullet and is harmless. The related `V13c`/`V14a` finding additionally asks that the `ERR-13` assertion be named in the impl leaves' **Ships when** — apply that to `V15a` too if resolving the cluster jointly.

## Consequence

**Severity:** advisory

The `ERR-13` completed-callee-finality co-witness for the live `V15a` invoke surface is authored during implementation instead of test-first, so it never passes through the red-first gate the TDD ritual requires and `V15a-T`'s Deps under-declare `V4f`. The leaf still ends up with the assertion in `V15a`'s Tests, so coverage is not lost, but the red-first evidence for this REQ-ID is skipped.
