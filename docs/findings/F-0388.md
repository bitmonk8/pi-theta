## Finding

`V11e` (binder system-note rendering) is sequenced *before* `V11f` (binder per-class retry budget and failure taxonomy): `V11f`'s **Deps** list `V11e`, so `V11e` and `V11e-T` are built first. Yet two of `V11e`'s rendering-rule tests reach into artifacts the `V11f` failure taxonomy owns. Rule 3 asserts that "a failure-arm note matches the grammar `loom /<name>: <fixed-phrase> — <sanitised-suffix>`", and rule 4 asserts that empty model content is "classified as a malformed envelope and routed to the malformed-envelope failure row." The malformed-envelope failure row, the four-class taxonomy, and the verbatim templates are all enumerated in `V11f`'s **Adds** ("the four-class failure taxonomy with its six verbatim failure-mode templates").

Engine-before-content layering is sound only if the structural pieces those `V11e` tests reference — the classification predicate that maps empty content to "malformed envelope", and the routing-slot enum naming the set of failure rows — are owned at `V11e` (or lower), with `V11f` contributing only the per-row template text. The plan partially signals this split: `V11e`'s rule-4 bullet notes "the malformed-envelope row template itself is asserted by `V11f`." But neither `V11e`'s **Adds** nor `V11f`'s **Adds** states where the classification predicate and the routing-slot enum live. `V11e`'s **Adds** claims only "the five note rules"; `V11f`'s **Adds** claims "the four-class failure taxonomy", which reads as owning the routing slots too.

The ownership boundary is therefore implicit. The spec separation supports putting it at `V11e`: rule-4's classification rule lives in `V11e`'s cited topic (`defaulting-system-note-echo.md`), while the failure-mode templates live in `V11f`'s cited topic (`determinism-cancellation-failure.md`). The plan should state that boundary rather than leave it for the implementer to infer.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** `475d018` — "pi-loom plan: resolve \"V11e names five system-note rules but asserts only two\"" (2026-06-11)
**History:** Before `475d018`, `V11e`'s Tests asserted only two rendering behaviours (single-line collapse and the 120-code-point cap) and referenced no `V11f`-owned artifact. `475d018` expanded `V11e` to assert all five rendering rules, introducing rule 3 (failure-arm note grammar) and rule 4 (empty content → malformed-envelope routing) — both of which reference the failure taxonomy and failure-row set owned by the downstream `V11f` leaf. The expansion added the "template itself is asserted by `V11f`" note but no matching ownership statement to `V11e`'s Adds, leaving the engine/content layering boundary implicit. `git log -S "routed to the malformed-envelope failure row"` confirms `475d018` is the sole commit that introduced the routing phrasing.

## Solution Space

**Shape:** multiple

### Option A — V11e owns the classification predicate and routing-slot enum

**Approach:** State in `V11e`'s **Adds** that `V11e` owns (a) the empty-content→malformed-envelope classification predicate and (b) the failure-row routing-slot enum (the named set of failure rows the taxonomy routes into), and that `V11f` fills only each row's verbatim template text. Narrow `V11f`'s **Adds** wording ("the four-class failure taxonomy with its six verbatim failure-mode templates") so it fills the slots `V11e` defines rather than appearing to own the enum.

**Plan edits:** `V11e` Adds (add ownership clause); `V11f` Adds (narrow to template ownership).

**Spec edits:** None.

**Pros:** Aligns with the spec layering — rule-4 classification is in `V11e`'s cited `defaulting-system-note-echo.md`; templates are in `V11f`'s `determinism-cancellation-failure.md`. Keeps `V11e`/`V11e-T` self-contained: their rule-3/rule-4 tests reference only `V11e`-owned slots and are writeable before `V11f` exists. Honours the build order (`V11f` Deps `V11e`).

**Cons:** Reassigns the routing-slot enum's ownership from `V11f`'s current Adds claim to `V11e`, requiring the `V11f` Adds to be narrowed in the same edit.

**Risks:** If `V11f`'s Adds is not narrowed in lockstep, both leaves claim the enum.

### Option B — Move rule-4's routing assertion to V11f

**Approach:** Strike "and routed to the malformed-envelope failure row" from `V11e` rule 4 (and `V11e-T`), leaving `V11e` to assert only that empty content is classified as malformed (not surfaced as an empty note); add the routing-to-failure-row assertion to `V11f` (and `V11f-T`).

**Plan edits:** `V11e` Tests + Ships when; `V11e-T` Tests; `V11f` Tests; `V11f-T` Tests.

**Spec edits:** None.

**Pros:** `V11e` no longer asserts a `V11f`-owned routing target.

**Cons:** Splits one observable behaviour (empty content → malformed-envelope row, not empty note) across two leaves; `V11e`'s rule-4 keeps a destination-less "not surfaced as an empty note" half until `V11f` lands. Fights the spec, which places the classification rule in `V11e`'s cited topic.

**Risks:** `V11e`'s rule 4 becomes a partial assertion with no observable routing target at its own ship point.

### Recommendation

Take Option A. The classification rule (rule 4) lives in `V11e`'s own cited spec topic, and the engine-before-content layering wants the structural routing slot defined at the lower leaf. Add to `V11e`'s **Adds** that it owns the malformed-envelope classification predicate and the failure-row routing-slot enum, and narrow `V11f`'s **Adds** so it contributes the verbatim templates into those slots. Watch that the `V11f` Adds narrowing lands in the same edit so the enum has exactly one owner.

## Consequence

**Severity:** advisory

An implementer building `V11e`/`V11e-T` first meets rule-3 and rule-4 tests that name the failure-arm grammar and route to the malformed-envelope failure row — both `V11f`-owned concepts built later. With no explicit ownership statement, two reasonable implementers diverge on where the classification predicate and the routing-slot enum live (defined ad-hoc in `V11e`, or deferred to `V11f`, which `V11e` cannot reference because `V11f` is downstream), forcing a later reconciliation pass between the two leaves.
