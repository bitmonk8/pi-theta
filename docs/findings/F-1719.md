## Problem

PIC-13's `realpath` bullet binds two normative MUSTs on the production `PiFileSystem` adapter — a **Case-canonicalisation guarantee** (case-variant inputs resolving to one directory entry yield byte-identical output) and a **Byte-stable canonical-output guarantee** (two paths naming the same on-disk target yield byte-identical output) — and states that bare delegation to `fs.promises.realpath` is insufficient because `fs.promises.realpath` and `fs.promises.realpath.native` disagree on leaf-case output, concluding "the adapter must canonicalise leaf case to satisfy it." It then never names the canonicalisation target (on-disk `readdir` casing? `realpath.native` casing? input-byte-preserving?) or any constructive call sequence, and Node provides no single call that satisfies the MUST. Two conforming adapters can therefore pick different canonical leaf casings, each individually satisfying "byte-identical for case-variant inputs" while disagreeing on which bytes the byte-for-byte containment comparison in [Invocation — Resolution](../../docs/spec_topics/invocation.md) (load-time and INV-1) and the DISC-3 dedup ever see.

**History:** `a765f6ca` added the **Case-canonicalisation guarantee** sentence together with the "bare delegation to `fs.promises.realpath` is insufficient … the adapter must canonicalise leaf case to satisfy it" caveat, but stopped at naming the obligation. Three days later `6ab9f7b` widened the surface with the **Transitive-symlink-resolution** and **Byte-stable canonical-output** guarantees on the same bullet; the second commit inherited the first's mechanism-free posture and added a second normative byte-stability MUST without filling the gap. The defect is the interaction of the two: each commit introduced one normative guarantee on an adapter call whose Node primitives admittedly cannot satisfy it without an unspecified extra step, and neither commit pinned that step.

## Solution approach

Clarify the PIC-13 `realpath` bullet so the **Case-canonicalisation** and **Byte-stable canonical-output** guarantees name a single canonical leaf-casing target the production adapter must produce, rather than asserting only that the adapter "must canonicalise leaf case." Ground that target in the entry bytes `readdir` returns on the parent directory, reusing the **Entry-name encoding guarantee** already bound on the `readdir`/`lstat` line (raw on-disk UTF-8, no Unicode normalisation), so both guarantees fall out of one rule.

## Solution constraints

- Out of scope: DISC-3's same-directory case-collision dedup in [`discovery-sources.md`](../../docs/spec_topics/discovery/discovery-sources.md), which owns the lexicographically-first-wins rule one layer up — do not re-specify it on the PIC-13 bullet.
