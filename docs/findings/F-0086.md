## Problem

The "Binder-call cancellation forwarding" Tests bullet in `V11f` and `V11f-T` marks its scope boundary as "distinct from the pre-binder pre-call abort `V17a` owns" — terminology with no referent in the spec. The spec defines exactly one binder-related checkpoint: `CheckpointKind` `"binder-call"` (host-interfaces-services.md PIC-10), firing immediately before the binder's LLM call is issued (cancellation.md *Granularity*); there is no separate "pre-binder" checkpoint, so "pre-binder pre-call" conflates two notions onto the single `binder-call` site. The phrasing leaves the `V17a`/`V11f` ownership split stated in vocabulary an implementer cannot map back onto the spec's checkpoint model.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commits:** d5dd4de
**History:** The phrase entered with commit d5dd4de (2026-06-10, "pi-loom plan: resolve 'binder-call cancellation forwarding never asserted'"), which added the "Binder-call cancellation forwarding" Tests bullet — including the "distinct from the pre-binder pre-call abort `V17a` owns" clause — to both `V11f-binder-retry-taxonomy.md` and `V11f-T-binder-retry-taxonomy.md` in the same commit. The defect was introduced by that resolving edit; the binder-call cancellation bullet did not exist at the leaf's inception. No later commit touched the clause.

## Solution approach

Rewrite the boundary clause "distinct from the pre-binder pre-call abort `V17a` owns" in both the `V11f` and `V11f-T` "Binder-call cancellation forwarding" Tests bullets so it names the `binder-call` `CheckpointKind` (host-interfaces-services.md PIC-10) and the pre-call abort timing whose presence `V17a` asserts (cancellation.md *Granularity* binder-call clause). State the boundary identically in both paired leaf files.

## Solution constraints

- Leave the bullet's positive obligation unchanged: `V11f`/`V11f-T` still test that an abort landed *during* the in-flight binder provider call (initial attempt or budgeted retry) surfaces the cancelled-binder note while the loom does not run.
- The spec is read-only for this fix.
