## Finding

`V5f`'s SUBS-1 Tests bullet states the union-lowering contract as "arms are emitted in source order with `\"null\"` last whenever the union admits it" — i.e. when `null` is *not* already last in loom source order, the lowering pass must move it to the end of the `{ "type": [...] }` array. This matches SUBS-1 in `schema-subset.md` (the *Array element order* clause: source order, with `"null"` last whenever the union admits it).

All three illustrative examples in the bullet, however, sidestep that reorder: `string | number` contains no `null`; `string | null` already has `null` last in source order; `string | Author` is the `anyOf` (non-primitive arm) case. None presents a union where `null` precedes another arm in source order, so none demonstrates the move-`null`-last behaviour.

A test author deriving cases from the example set — rather than from the prose rule — can author tests that all pass against an implementation that emits arms in raw source order and never reorders `null`. The reorder branch then ships unverified. The identical bullet appears in the paired tests leaf `V5f-T`, so the example gap is duplicated on the surface that actually authors the assertions.

## Consequence

**Severity:** advisory

The reorder rule is stated in prose, so a careful implementer can still produce a correct leaf; but the example set is the surface a test author is most likely to mechanise, and it admits a passing test suite over an implementation that never moves `null` last. SUBS-1's only closing leaf is `V5f`, so an unexercised reorder branch is the difference between a conformant canonical hash and a silently divergent one for any `null | string`-shaped union.

## Solution Space

**Shape:** single

### Recommendation

In the SUBS-1 Tests bullet, add a fourth illustrative example that forces the reorder, where `null` is not last in source order:

```
`null | string` → `{ "type": ["string", "null"] }`
```

Insert it alongside the existing `string | number` / `string | null` / `string | Author` examples in the parenthetical example list. Apply the identical edit to the same bullet in `docs/plan_topics/V5f-T-subset-lowering-hash.md` so the implementation leaf and its paired tests leaf stay in sync. No spec edit is needed — `schema-subset.md` SUBS-1 already mandates the move-`null`-last behaviour; the example added here illustrates the rule the spec already states.
