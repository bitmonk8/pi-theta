## Finding

`V4d`'s **Adds** claims ownership of "the nine-variant `QueryError` union (its `kind`/`cause` wire forms)", and its **Ships when** states `npm test` "asserts the nine-variant shape". The spec page `queryerror-variants.md` is the canonical declaration of all nine loom 1.0.0 variants (`ValidationError`, `TransportError`, `ModelToolError`, `ContextOverflowError`, `CancelledError`, `ToolLoopExhaustedError`, `CodeToolError`, `InvokeInfraError`, `InvokeCalleeError`), so V4d is the single source of each variant's wire schema.

V4d's **Tests** (mirrored in `V4d-T`) name only four obligations: `ERR-14` (`ValidationIssue` ordering), `ERR-15` (`kind: string` openness seam), `ERR-17` (forced-respond synthesised `ValidationIssue`), and `ERR-19` (`ToolLoopExhaustedError` shape). Only `ValidationError`-family fields and `ToolLoopExhaustedError` get a direct wire-form assertion. The remaining variants' wire forms — `TransportError`, `ModelToolError`, `ContextOverflowError`, `CancelledError`, `CodeToolError`, `InvokeInfraError`, `InvokeCalleeError` — have no asserting test in V4d, so the gate backing "asserts the nine-variant shape" is vacuous for them.

`ContextOverflowError` is the sharpest instance: V4d's Adds explicitly singles it out as the sole NOCEIL-2 token-domain failure surface and states "`V4d` covers only the variant's wire shape", yet no Tests bullet asserts its `kind: "context_overflow"` discriminator or its `tokens_used` / `tokens_limit` / `raw_response` field set. Firing-path leaves (`V9j` extracts context-overflow to `ContextOverflowError`; `V13d` short-circuits on it; `V9n` maps `TransportError`; `V14a` `CodeToolError`; etc.) assert mapping/behaviour, but the canonical wire schema is delegated to V4d by V4d's own Adds. The mismatch lets a malformed or wrong-shaped un-named variant (wrong `kind` string, missing/renamed field) ship green.

## Solution Space

**Shape:** multiple

### Option A — Broaden V4d's Tests to cover every variant's wire form

**Approach:** In `V4d-T` and `V4d`, add Tests coverage that asserts each of the nine variants' wire form — the `kind` discriminator string and the field set declared in `queryerror-variants.md` — explicitly including `ContextOverflowError` (`kind: "context_overflow"`, `tokens_used`, `tokens_limit`, `raw_response`). Keep Ships-when's "nine-variant shape" wording, which then matches what the gate proves.

**Plan edits:** Add the wire-form assertions to `V4d-T`'s **Tests** and the mirroring entry in `V4d`'s **Tests**; no Ships-when change.

**Spec edits:** None.

**Pros:** Keeps V4d as the single canonical wire-shape owner consistent with `queryerror-variants.md`; the gate matches the claim; `ContextOverflowError`'s shape is verified despite having no loom 1.0 firing path.

**Cons:** Larger V4d test surface.

**Risks:** Wire-form assertions must track the spec schemas; a future variant addition needs a matching test (already the intent of the open-seam `ERR-15`).

### Option B — Narrow V4d's claim and delegate the remaining wire forms to firing leaves

**Approach:** Reword V4d's Adds and Ships-when so they claim only the four tested obligations, and state explicitly that the wire forms of the other variants are asserted by their firing-path leaves (`TransportError`→`V9n`, `ModelToolError`/`ToolLoopExhaustedError` firing→`V13c`, `ContextOverflowError`→`V9j`/`V13d`, `CodeToolError`→`V14a`, invoke variants→`V15a`/`V15f`).

**Plan edits:** Narrow `V4d`/`V4d-T` Adds + Ships-when; add a per-variant wire-form assertion to each named firing leaf's **Tests** where one is not already present.

**Spec edits:** None.

**Pros:** Co-locates each wire-form assertion with the behaviour that fires it.

**Cons:** Disperses the wire schema away from V4d, contradicting `queryerror-variants.md`'s "canonical declaration here" framing; risks a variant whose wire form falls between leaves (no firing path covers `ContextOverflowError` end-to-end in loom 1.0).

**Risks:** Requires auditing every firing leaf to confirm it actually asserts the full field set, not just the `kind`.

### Recommendation

Take Option A: assert all nine variants' wire forms in `V4d-T`/`V4d`, including `ContextOverflowError`'s `context_overflow` discriminator and field set, so the existing "nine-variant shape" Ships-when is backed by the gate. This keeps V4d aligned with the spec's canonical-declaration framing. Watch the `ERR-15` open-seam boundary: the test asserts the closed loom 1.0.0 variant set is the runtime's emission set while the `kind` *type* stays `string` — do not retighten `kind` to a closed enum.

## Consequence

**Severity:** correctness

The closing gate behind "asserts the nine-variant shape" certifies only four of the nine variants' wire forms, so a variant V4d solely owns — including `ContextOverflowError`, which has no firing path in loom 1.0 but is still V4d-owned wire shape — can ship with a wrong `kind` discriminator or a missing/renamed field while `npm test` stays green. Two implementers would diverge on what the gate actually proves, and a wire-shape regression in the un-named variants surfaces only downstream at a consumer mismatch.
