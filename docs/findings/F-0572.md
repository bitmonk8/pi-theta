## Finding

`V16a` defines a deliberately stateless cross-ceiling arbitration seam: `arbitrate(candidate) → { surfaced, masked? }`, whose only input is a *ceiling-candidate* (a check-site / ceiling-class tag plus the satisfied-ceiling set), and whose Adds prose states it "does not receive or intercept events, run any breach check, or own any per-ceiling surface." The leaf even draws an explicit scope line — a "Verified at the seam in isolation" list versus a "Not witnessable at the stateless seam — owned by downstream sites" list — but that downstream list routes only CIO-1 and CIO-5 to the live enforcement leaves, classifying CIO-2/CIO-3/CIO-4 as "within-site sub-check ordering as encoded by the arbitration."

The per-bullet Tests text contradicts that classification. The CIO-4 bullet asks the seam to resolve ceiling #2 "post-slot-increment, pre-next-turn; `max_rounds:0` typed takes the final branch at start" — these are live `tool_loop` runtime properties owned by `V13c` (round advancement, slot counting, forced-respond branch), not decidable from a stateless `{tag, satisfied-ceiling set}` candidate. The CIO-2 bullet ("…before the callee body") and the CIO-3 bullet ("the depth walk is ordered before AJV") have the same shape: they assert temporal runtime-site relations that the seam, by its own contract, cannot witness — those are owned by `V15b` (invoke-entry, ceiling #1) and `V5e` (AJV boundary, ceiling #4) respectively.

An implementer is left with a contradiction at exactly the leaf that is picked up before any of the downstream enforcement sites exist. Either widen the seam input to carry tool-loop / call-site timing state — breaking the stateless contract that `V4e`, `V11f`, `V5e`, `V13c`, and `V15b` all consult — or implement only the trivial tag→surfaced mapping and treat the temporal clauses as unverifiable prose. The two readings produce materially different seam shapes and test suites.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V16a-ceiling-order-masked.md` and `docs/plan_topics/V16a-T-ceiling-order-masked.md`, restate the CIO-2, CIO-3, and CIO-4 Tests bullets so each asserts only what the stateless `{check-site tag, satisfied-ceiling set}` candidate makes decidable — the check-site tag → surfaced-ceiling mapping and the arbitration-encoded sub-check ordering — and route the temporal/runtime substance to its owning downstream leaf, exactly as the CIO-1 and CIO-5 bullets already point to `V4e`/`V11f` vs `V5e`/`V13c`/`V15b` and to `H7a`:

- CIO-4: drop "post-slot-increment, pre-next-turn; `max_rounds:0` typed takes the final branch at start" from the seam assertion; route that round-boundary timing and the `max_rounds:0` final-branch behaviour to `V13c` (which already carries them in its free-phase, `max_rounds:0`, and depth-6 co-fire tests).
- CIO-2: drop "before the callee body"; route the invoke-entry temporal relation to `V15b`.
- CIO-3: drop "the depth walk is ordered before AJV"; route the depth-walk-before-AJV temporal relation to `V5e`.

Also align `V16a`'s **Ships when** clause, which currently claims the seam proves "the CIO-2/CIO-3/CIO-4 within-site sub-check ordering," so it does not overstate the temporal portion now routed downstream. Confirm `V13c`, `V5e`, and `V15b` each carry the corresponding temporal assertion (verified: `V13c` covers round/`max_rounds:0`; `V5e` covers depth-6 `maxDepth`; `V15b` owns the invoke-entry ceiling-#1 enforcement); add the assertion to any that does not. No spec edit is required — `ceilings-3-and-4.md`'s CIO definitions are unchanged and already correct.

## Consequence

**Severity:** correctness

Two reasonable implementers diverge at the seam that every ceiling enforcement site (`V4e`/`V11f`/`V5e`/`V13c`/`V15b`) consults: one widens the candidate input to carry live tool-loop / call-site timing, breaking the stateless contract those sites depend on; the other implements only the tag→surfaced mapping and leaves the temporal CIO-2/CIO-3/CIO-4 clauses asserted-but-unverifiable. Either way the seam's test footing at `V16a` (where no downstream site exists yet) does not match what the leaf text claims it proves.
