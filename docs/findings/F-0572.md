## Problem

`V16a` defines a deliberately stateless cross-ceiling arbitration seam, `arbitrate(candidate) → { surfaced, masked? }`, whose only input is a `{check-site tag, satisfied-ceiling set}` candidate and whose Adds prose states it "does not receive or intercept events, run any breach check, or own any per-ceiling surface." The leaf classifies CIO-2/CIO-3/CIO-4 as seam-encoded "within-site sub-check ordering," but the per-bullet Tests text contradicts that: CIO-4 asks the seam to resolve ceiling #2 "post-slot-increment, pre-next-turn" with `max_rounds:0` final-branch behaviour (live `tool_loop` runtime state owned by `V13c`), CIO-2 asserts "before the callee body" (invoke-entry timing owned by `V15b`), and CIO-3 asserts "the depth walk is ordered before AJV" (AJV-boundary timing owned by `V5e`). These temporal runtime relations are not decidable from the stateless candidate. An implementer at `V16a` — picked up before any downstream enforcement site exists — must either widen the seam input (breaking the stateless contract `V4e`/`V11f`/`V5e`/`V13c`/`V15b` all consult) or treat the temporal clauses as unverifiable prose, producing materially different seam shapes and test suites.

## Solution approach

Rewrite the CIO-2, CIO-3, and CIO-4 Tests bullets in both `V16a` and `V16a-T` so each asserts only what the stateless `{check-site tag, satisfied-ceiling set}` candidate makes decidable — the check-site-tag → surfaced-ceiling mapping and the arbitration-encoded sub-check ordering — and route the temporal/runtime substance to its owning downstream leaf, mirroring how the CIO-1 and CIO-5 bullets already point downstream: CIO-4's round-boundary timing and `max_rounds:0` final-branch behaviour to `V13c`, CIO-2's invoke-entry temporal relation to `V15b`, and CIO-3's depth-walk-before-AJV relation to `V5e`. Align `V16a`'s **Ships when** clause so it no longer claims the seam proves the CIO-2/CIO-3/CIO-4 temporal portion now routed downstream. Confirm each downstream owner (`V13c`, `V5e`, `V15b`) carries the corresponding temporal assertion and add it where absent.
