## Problem

`H2a`'s first *Tests* bullet (`Specific exception types only`) and `H5c` validate the same `// allow-broad-catch:` comment token against the same four-arm predicate from `conventions.md` *Specific exception types only*, but describe the Code-keyed-obligation-areas arm at different precision. `H2a` admits "an enumerated *Code-keyed obligation areas* entry" without stating what an entry is or how it resolves, while `H5c` pins it to a row's canonical `cka-<n>` token matching exactly one row's *Token* cell in `coverage-matrix.md`. An implementer building H2a's lint fixture from the vague phrasing can encode the arm by a different rule than the `cka-<n>` exactly-one-Token-cell match, so the lint gate and the H5c CI reconciliation gate validate the same allow-list inconsistently.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `96563f9` (2026-06-10, "resolve broad-catch allow-comment predicate unsatisfiable for exempt sites"); `7259777` (2026-06-18, "resolve code-keyed obligation areas lack a canonical citable token")
**History:** `96563f9` authored H2a's widened-predicate *Tests* bullet with the arm "an enumerated *Code-keyed obligation areas* entry" — correct at the time, because no canonical citable token for those rows yet existed. `7259777` then introduced the canonical `cka-<n>` token and propagated the precise one-`Token`-cell resolution into `H5c`, `conventions.md`, `coverage-matrix.md`, `V14b`, and `V9g`, but did not touch `H2a` (confirmed by its file list). The divergence opened at `7259777`: the formalisation landed in the sibling gate while H2a's parallel arm was left at its pre-canonical-token phrasing. The durable fix back-propagates that formalisation into H2a, not a revert of either commit.

## Solution approach

In `docs/plan_topics/H2a-cross-cutting-gates.md`, rewrite the Code-keyed-obligation-areas arm of the first *Tests* bullet to name the canonical `cka-<n>` token and its exactly-one-*Token*-cell resolution rule against `coverage-matrix.md`, aligning it with the form `H5c` already pins. Leave the other three arms (coverage-matrix REQ-ID, concrete `loom/...` registry code, structural `pi-sdk-boundary` token) unchanged.

## Solution constraints

- Out of scope: `H5c`, `coverage-matrix.md`, `conventions.md`, and the spec already carry the authoritative `cka-<n>` form and are read-only for this fix.
