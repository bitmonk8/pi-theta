## Problem

The `V14a` / `V14a-T` Tests bullet citing `host-interfaces-core.md` §*Tool execution from loom code* (sub-clauses (3)/(4)), and the `coverage-matrix.md` `cka-46` row that repeats them verbatim, mandate two contract details that the cited page no longer states: a fixed no-text `CodeToolError` message string `"tool reported an error with no text content"`, and an `isError: true` joined-text truncation-to-4096-bytes lowering rule. The current page scopes 4096-byte truncation only to the `execute()`-throw coerced message and states that `AgentToolResult` carries no `isError` field, so it defines no isError-based lowering at all. The plan therefore gates a `npm test` assertion on an invented contract attributed to a spec anchor that no longer contains it.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** authored correct-for-its-time in `77f2e88` (`pi-loom plan: resolve "code-side tool-call envelope-lowering MUSTs unanchored"`, 2026-06-18); invalidated by `31debf1` (`pi-loom spec-plan: fix F-1578 — re-anchor execute() outcome routing on AgentToolResult (no isError); collapse dead Err branch`, 2026-06-27) with no corresponding plan update.
**History:** `git log -S 'tool reported an error with no text content'` shows the string lived in both the spec (`host-interfaces-core.md`) and the plan (`V14a`/`V14a-T`/`coverage-matrix.md`). The plan leaf was authored at `77f2e88`, transcribing the then-current spec mechanics — at that point `host-interfaces-core.md` defined the `isError: true` lowering, the fixed no-text string, and the 4096-byte truncation applied to the joined isError text, so the leaf was faithful. `31debf1` (F-1578) re-anchored `host-interfaces-core.md` onto `AgentToolResult` (which has no `isError` field), collapsing the isError Err branch and deleting the no-text string and the isError-path truncation from the spec page. The plan's `V14a`/`V14a-T` Tests bullets and the `coverage-matrix.md` `cka-46` row were not updated in that commit and still cite the removed mechanics. The defect is the un-propagated spec re-anchor, not an original authoring error.

## Solution approach

Rewrite the `host-interfaces-core.md` envelope-lowering Tests bullet in both `V14a` and `V14a-T` to stop asserting sub-clause (4)'s fixed no-text string and sub-clause (3)'s isError-path joined-text truncation, re-aligning the asserted accepted-path mechanics with the current §*Tool execution from loom code* page (no `isError` field on `AgentToolResult`; 4096-byte code-point-boundary truncation scoped to the `execute()`-throw coerced message only). Update the `V14a` *Ships when* summary to match. Rewrite the `coverage-matrix.md` `cka-46` row in step so it no longer enumerates the removed mechanics.

## Solution constraints

- Out of scope: the residual `tool-calls.md` vs `host-interfaces-core.md` cross-page disagreement over code-side `isError` exposure is a spec-internal defect for the spec-lens reviewers; the spec is read-only for this fix.
