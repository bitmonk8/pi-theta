## Finding

Step 0(a) routes a syntactically invalid `process.versions.node` to `kind: "probe-failed"` via `semver.valid(...) → null` (a value test, not a throw), but *Self-failure* defines `probe-failed` solely as a trapped throw with `details.cause` = the coerced underlying-error string, and `code-registry-load.md` mirrors that throw-only framing. The non-throw route has no defined `details.cause`, so implementers diverge.

**Verdict:** multi-commit-interaction
**Introducing commits:** 126ff8a — pi-loom spec: Step 0 capability probe rewrite + V1.0 deferrals + Choice B (2026-05-06, Thomas Andersen); eb33bf9 — pi-loom spec: resolve "Post-probe SDK-shape drift has no stated routing or operator attribution" (2026-05-07, Thomas Andersen)
**History:** The bundled defects have distinct (non-causal) origins, so the union spans two commits. The Step 0 capability-probe rewrite 126ff8a introduced obligations 1, 2, and 3 simultaneously: it added the `PiExtensionAPI` adapter-layer wording in the *On failure* bullet, the invalid-SemVer-`semver.valid`-routes-to-`probe-failed` clause alongside a throw-only *Self-failure* `details.cause` contract, and the "MUST stay disjoint" sentence in Step 0(c). Obligation 4's inline deferral entered one day later in eb33bf9, which added the *Post-probe SDK-shape drift* section (including the `details.kind`-discriminator-deferred-past-loom-1.0 statement). The 2026-06-04 commit f5e89f4 only relocated the content from `docs/spec/` into `docs/spec_topics/` (the size-cap collapse) and did not introduce any of the four defects.

## Solution Space

**Shape:** single

### Recommendation

In Step 0(a), state the invalid-SemVer path as an explicit pre-check (no try/catch) that emits `loom/load/host-incompatible` with `details.kind = "probe-failed"`, `details.step = "node-floor"`, and `details.cause` **absent** (or a fixed non-throw sentinel). Amend *Self-failure* and the `code-registry-load.md` `probe-failed` description so `details.cause` is documented as present-on-throw / absent-on-the-invalid-SemVer-precheck rather than universally "the caught throw's underlying string".

## Consequence

**Severity:** correctness

Obligation 2 makes the invalid-SemVer failure path divergent: two conforming implementations produce different `loom/load/host-incompatible` payloads (one omits `details.cause`, another synthesises one, a third re-routes to `node-floor`), which is operator-observable. Obligation 1 leaves an implementer guessing what `PiExtensionAPI` denotes. Obligations 3 and 4 degrade discoverability — a cross-code MUST and a deferral are invisible at the pages that own those concepts — but do not change behaviour.
