## Problem

The four cross-cutting exemption rules in `docs/plan_topics/conventions.md` `## Cross-cutting rules (every leaf)` — the `allow-ambient` ambient-access exemption, the `allow-broad-catch` exemption, the `allow:` Promise-combinator exemption, and the `allow-sync` blocking-runtime exemption — each anchor their exemption on a comment carried by "its line" or a "same-line comment", but none defines which source line is "the line" when the exempted construct spans multiple lines. The exemption is discovered independently by the H2a/H3a enforcement scans and by the H5a/H5c closing-gate reconciliation. If the two resolve "the line" differently for a multi-line site (e.g. a `catch` clause whose keyword and binding sit on different lines, or a wrapped `Promise.allSettled([` call), they classify the same site inconsistently, producing either a spurious CI failure on a legitimately-exempt site or a silently-unaudited exemption.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `358b5df` ("ambient-access allow-list representation", 2026-06-12), `506d71c` ("no-broad-catch allow-list has no defined representation or location", 2026-06-12), `07d8022` ("Sequential-by-default allow-list representation undefined at its own rule site", 2026-06-17), `c33d309` ("Blocking-runtime ban concedes enforcement to manual review", 2026-06-17) / `cf47e6f` (allow-sync refinement, 2026-06-18)
**History:** `git log -S 'its line'` / `-S 'same-line comment'` / `-S 'allow-ambient'` / `-S 'allow-sync'` on `docs/plan_topics/conventions.md` shows the four allow-* comment families were each given their single-line comment representation in a separate allow-list-representation fix, on different dates. Each fix independently adopted the "its line" / "same-line comment" phrasing to anchor its own family's exemption, and none defined "the line" for a multi-line construct. The cross-family ambiguity is therefore an emergent property of these convergent representation fixes — no single commit introduced it; it surfaced once all four families standardised on the same under-specified phrase.

## Solution approach

Add a single definition of "the line" at the top of the `## Cross-cutting rules (every leaf)` section, before the four bullet rules, governing all four `allow-*` comment families. Identify the qualifying source line as the one carrying each family's primary exempted token — the `catch` keyword for `allow-broad-catch`, the banned ambient-primitive identifier for `allow-ambient`, the Promise-combinator token for `allow:`, and the `*Sync` callee identifier for `allow-sync` — and state that a comment on a continuation line of the same expression does not satisfy the exemption. Make clear this one definition is the shared contract that both the H2a/H3a enforcement scans and the H5a/H5c closing-gate reconciliation read, so the two sides classify a multi-line site identically.
