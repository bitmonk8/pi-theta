## Problem

PIC-54 (runtime-event-channel.md, anchor `id="pic-54"`) is a named-hazard MUST: the terminal `console.error` step of the `sendSystemNote` best-effort fallback chain must be wrapped in `try`/`catch` and a throw from it silently swallowed, never propagating out of the chain, across all three reach-paths (per-invocation `finally` clean-cancel, extension-bootstrap SDK-failure, capability-probe `host-incompatible`). No fixture in the corpus injects a throwing terminal `console.error` and asserts the throw is absorbed, so the obligation is non-observable and untested. The sibling construction-site swallow obligations on the same surface (PIC-26, building on PIC-25) carry explicit fixtures (a)/(b)/(c) in `diagnostic-emission-isolation.md`, but PIC-54 — imported by reference into the same paragraph — has none, leaving an implementation that omits the terminal `try`/`catch` (or covers only some reach-paths) passing every existing test.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** cfb3943 — pi-loom spec-plan: fix F-1395 — coin PIC-54 wrapping terminal console.error at fallback-chain definition site (2026-06-25, Thomas Andersen)
**History:** A narrower swallow sentence for the terminal `sendSystemNote` `console.error` previously lived inside `diagnostic-emission-isolation.md`'s *Pi-side stdio visibility* clause (scoped only to the per-invocation `finally` reach-path). Commit cfb3943 promoted it to a named MUST, PIC-54, at the fallback-chain definition site in `runtime-event-channel.md`, broadening it to all three reach-paths and rewriting the `diagnostic-emission-isolation.md` clause to import it by reference. That commit coined the obligation but added no acceptance fixture for it, even though the same neighbouring `## Acceptance criteria` section already carried fixtures for the sibling PIC-25/PIC-26 swallow obligations — so the gap entered with the codification of PIC-54.

## Solution approach

Add an acceptance fixture for PIC-54 in the `## Acceptance criteria` section of `diagnostic-emission-isolation.md`, adjacent to the existing PIC-25/PIC-26 fixtures. Key it on PIC-54 and have it inject a throwing `console.error` on the fallback chain's terminal step (after both the `pi.sendMessage` emission and the step-2 `loom/runtime/system-note-delivery-failed` diagnostic have failed), asserting that the throw does not propagate out of the chain and that no further diagnostic is emitted after the swallow. Exercise each of the three reach-paths — per-invocation `finally` clean-cancel, extension-bootstrap SDK-failure, and capability-probe `host-incompatible` — since covering only the `finally` path leaves the broadened PIC-54 scope unverified.

## Solution constraints

- Introduce no new diagnostic code or wire field for the fixture.
