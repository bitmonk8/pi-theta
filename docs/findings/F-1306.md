## Problem

The **Transient toasts (auxiliary)** paragraph in `diagnostic-shape.md` specifies `ctx.ui.notify(message, "error")` as the only user-visible string emitted along the transient-toast surface, but carries no contract for `message` — no template, no enumeration of allowed substrings, and no explicit non-normative carve-out. Two conformant implementations can ship `message`s that share no tokens, and a conformance test has nothing to assert against on the only author-visible string this surface emits.

The only later edit, `32d2b76`, appended the `notify`-throw swallow rule but did not touch this gap.

## Solution approach

Amend the **Transient toasts (auxiliary)** paragraph to fix a normative `message` content contract covering the three trigger sites it enumerates (the chokidar watcher throwing, an uncovered settings I/O exception, an internal invariant violation), so a conformance harness has a stable surface to assert against. Apply the §8 placeholder-rendering first-line-truncation rule (the one §8 pins for host-derived freeform tail) to any host-supplied substring so a non-finite host string cannot overflow the toast line. Prefer a per-trigger-site template form over declaring `message` non-normative freeform — the freeform carve-out declares the gap rather than closing it.

## Solution constraints

- Preserve the existing surfacing rule (loom-author-facing `loom/parse/*` / `loom/load/*` / `loom/runtime/*` diagnostics MUST go through the persistent channel) and the `ctx.ui.notify`-throw swallow rule unchanged.
- Out of scope: the in-progress-scan effect of the settings-IO exception (F-1307) and the watcher post-error recovery posture (F-1308), which edit the same paragraph.
