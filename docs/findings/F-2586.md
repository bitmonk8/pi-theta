## Finding

GOV-15 (and the Source-language stability orientation bullet that restates it) defines observable (c) — equivalence of `loom-system-note` content strings across loom 1.x releases — as: two content strings are equivalent when they are "byte-identical … *after normalising* the placeholder sub-fields whose rendered value is permitted to vary per invocation or per run." The clause then names *which* sub-fields are normalised (the per-invocation identifiers `<invocation-id>` / `<uuid>`, the wall-clock-derived values `<ms>`, and the category-8 host-derived freeform tails) and cross-references the placeholder-rendering categories that classify them. What it never states is the *normalisation operation itself*: given a variable sub-field's rendered bytes, what does "normalise" produce?

The candidate operations are not equivalent. Replacing the rendered bytes with the empty string, replacing them with a fixed canonical sentinel (e.g. the placeholder token `<uuid>`), masking them with a wildcard match, or stripping them along with adjacent delimiters all yield different byte sequences to compare, and on adversarial inputs they yield different equivalence verdicts — e.g. when a variable sub-field abuts literal template bytes, "strip" and "replace-with-sentinel" disagree on whether the surrounding bytes still line up. The cross-reference to the placeholder-rendering categories fixes how each placeholder *renders* and therefore the byte *extent* of each variable sub-field, but it does not define a canonicalisation/masking operation for comparison; rendering and normalisation-for-equivalence are distinct.

Because observable (c) is stated as a definition of an equivalence relation, an undefined normalisation step leaves the relation underdetermined. The verb "normalising" is asserted but its operational meaning is missing, so two independent parties applying the rule (loom 1.0 reviewers inspecting an inter-release diff, or authors of the deferred conformance-fixture suite) can implement materially different masking operations and disagree on whether two releases satisfy the equivalence promise.

## Solution Space

**Shape:** single

### Recommendation

Define the normalisation operation at the owner (GOV-15 in `source-language-stability.md`) and restate it in the orientation aggregator (`overview-and-orientation.md`) in the same commit, per the existing aggregator-lock-step discipline these two sites already follow.

State normalisation as a deterministic masking operation applied identically to both content strings under comparison: for each variable sub-field (the categories already enumerated — per-invocation identifiers, wall-clock-derived values, and category-8 host-derived freeform tails), replace its rendered bytes with a single fixed canonical sentinel determined solely by the sub-field's placeholder category and independent of the rendered value. Two content strings are then equivalent under observable (c) iff they are byte-identical after both have undergone this replacement. The byte extent of each variable sub-field is the rendered span fixed by the placeholder-rendering categories the clause already cross-references (`placeholder-rendering-a.md` / `placeholder-rendering-b.md`); make explicit that the extent for the category-8 host-derived freeform tail is the post-first-line-truncation rendered span, since that is the only category whose extent is not a fixed-shape token.

Edge cases the fixer must pin: (a) the sentinel must be a constant per category so that two notes differing only in a variable sub-field collapse to byte-identical masked strings; (b) the surrounding literal template bytes and the fixed (non-variable) placeholder renderings are *not* masked — they are compared byte-for-byte, consistent with the clause's existing "the surrounding literal template bytes are themselves byte-identical" sentence; (c) the replacement, not a wildcard match, is the operation — this avoids a variable sub-field's masked form spuriously matching literal bytes in the other string.

## Consequence

**Severity:** advisory

Observable (c)'s equivalence relation is underdetermined: two reviewers — or two implementers of the deferred conformance-fixture suite — can mask variable sub-fields differently (strip vs canonical sentinel vs wildcard) and reach divergent equivalence verdicts on inputs where a variable sub-field abuts literal template bytes. Immediate impact is bounded because loom 1.0 verifies the goal by reviewer inspection and the mechanical gate is explicitly deferred post-1.0, but the gap is inherited by whoever later builds that gate.
