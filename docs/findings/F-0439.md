## Finding

The *Sequential by default* cross-cutting rule in `conventions.md` permits the four banned `Promise` combinators (`Promise.all`/`race`/`allSettled`/`any`) in `src/**` only when "the calling leaf's `Spec.` field cites an obligation whose normative text **mandates concurrency at this site**" — repeated for the code-keyed-area arm as "whose normative text mandates concurrency **at the site**". The anchoring phrase "at this site" / "at the site" admits two materially different scopes:

1. **Call-site scope** — the cited obligation's normative text must mandate concurrency *at the exact operation the combinator call implements* (e.g. "the batch awaits every sibling to settle before the next turn").
2. **Feature-area scope** — it suffices that the cited obligation lives anywhere in the feature/spec area the leaf works in, and somewhere mandates concurrency, even if not at the specific call site.

The two readings produce different sets of admissible `// allow: <REQ-ID-or-cka-token> — <spec-page>` exemptions: under reading (2) a combinator could be exempted at a call site whose own operation has no concurrency mandate merely because a sibling obligation in the same area does, which defeats the rule's intent. The loom 1.0 closing gate only checks that the cited token *resolves* to a REQ-ID or `cka-<n>` row; it never checks the "mandates concurrency" semantic, so the scope question is resolved entirely at authoring and self-review time, where the ambiguity bites.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/conventions.md`, *Sequential by default* bullet, replace both occurrences of the ambiguous anchor with a call-site-scoped form. Change "cites an obligation whose normative text mandates concurrency at this site" and the code-keyed-area arm's "whose normative text mandates concurrency at the site" to text that pins the scope to the operation the combinator call implements, e.g.:

> "cites an obligation whose normative text mandates concurrent execution **at the operation this specific call site implements** — the items joined by the combinator execute without each waiting for the previous"

Apply the same phrasing to the code-keyed-area arm so both arms carry the identical call-site scope. The three existing carve-out sites already satisfy this reading, so no leaf edit follows.

## Consequence

**Severity:** correctness

Two reasonable implementers (and reviewers performing the self-review broad-`Promise` check) can disagree on whether a given combinator use is exempt. The feature-area reading admits exemptions the call-site reading rejects, so a combinator can ship at a site that has no concurrency mandate while a sibling obligation launders the exemption past review — exactly the unguarded-concurrency case the rule exists to prevent. The closing gate cannot catch this because it only resolves the cited token, not the concurrency semantic.
