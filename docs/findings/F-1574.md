## Problem

The HTML anchor `id="ceiling-interaction-order"` is defined twice in the spec corpus. Its canonical home is `docs/spec_topics/hard-ceilings/ceilings-3-and-4.md`, under the `## Interaction between ceilings` heading and directly above the normative CIO-1 … CIO-6 fixed-evaluation-order list. A duplicate ghost copy sits on the "Cross-ceiling content" orientation paragraph in `docs/spec_topics/overview-and-orientation.md`, co-located with a second ghost anchor `#no-additional-ceilings`. The ghost captures no inbound traffic — every `#ceiling-interaction-order` reference in the corpus resolves to the canonical target — but it makes the id non-unique across the corpus, which an anchor-uniqueness audit flags and which could silently capture a future bare or mis-pathed `#ceiling-interaction-order` link into the orientation summary instead of the normative CIO table.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** f5e89f4 ("docs: enforce 100KB hard / 30KB recommended size cap on spec set")
**History:** The anchor was first added as a single definition inside the monolithic `spec.md` by 96b80ec ("resolve 'Hard-ceiling interaction paragraph: missing anchor, undefined jargon, and bundled ordering rules'"). The size-cap split commit f5e89f4 — which ran the `split_spec.py` tooling to break `spec.md` into topic pages — relocated the canonical CIO-1 … CIO-6 content into `hard-ceilings/ceilings-3-and-4.md` (canonical anchor) while simultaneously creating the `overview-and-orientation.md` aggregator whose "Cross-ceiling content" summary paragraph retained a copy of the same `<a id="ceiling-interaction-order">`; that copy is the duplicate. The later 81bf52a ("collapse spec/ section pages into spec_topics/") only moved the file path `docs/spec/` → `docs/spec_topics/`, carrying the ghost verbatim — it did not introduce it.

## Solution approach

Delete the `<a id="ceiling-interaction-order"></a>` tag from the "Cross-ceiling content" paragraph in `docs/spec_topics/overview-and-orientation.md`, leaving the canonical anchor in `ceilings-3-and-4.md` as the sole definition. The paragraph already forwards to the canonical anchor via its outbound link and is the target of no inbound `#ceiling-interaction-order` reference, so it needs no replacement anchor.
