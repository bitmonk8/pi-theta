## Finding

The HTML anchor `<a id="ceiling-interaction-order"></a>` is defined in two pages of the spec corpus. Its canonical home is `docs/spec_topics/hard-ceilings/ceilings-3-and-4.md` (line 33), sitting directly under the `## Interaction between ceilings` heading and immediately above the normative CIO-1 … CIO-6 fixed-evaluation-order list — the content every consumer of the anchor expects to land on. A second, identical definition is a ghost copy on the compound "Cross-ceiling content" orientation paragraph in `docs/spec_topics/overview-and-orientation.md` (line 54), where it is co-located with a second ghost anchor (`#no-additional-ceilings`) on the very same line.

The overview paragraph that hosts the ghost is purely a forwarding summary: its own outbound link for the interaction order points to `./hard-ceilings/ceilings-3-and-4.md#ceiling-interaction-order` (the canonical anchor), not to itself. Every inbound reference in the corpus — from `diagnostics/diagnostic-shape.md`, `query/query-tool-loop.md`, `ceilings-3-and-4.md` itself, and the overview page's own prose — resolves to the canonical `ceilings-3-and-4.md` target. The ghost copy is therefore an unreferenced duplicate: it captures no current traffic, but it makes the same id non-unique across the corpus, which a duplicate-id / anchor-uniqueness audit flags and which can silently capture a future bare or mis-pathed `#ceiling-interaction-order` link into the orientation summary instead of the normative CIO table.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** f5e89f4 ("docs: enforce 100KB hard / 30KB recommended size cap on spec set")
**History:** The anchor was first added as a single definition inside the monolithic `spec.md` by 96b80ec ("resolve 'Hard-ceiling interaction paragraph: missing anchor, undefined jargon, and bundled ordering rules'"). The size-cap split commit f5e89f4 — which ran the `split_spec.py` tooling to break `spec.md` into topic pages — relocated the canonical CIO-1 … CIO-6 content into `hard-ceilings/ceilings-3-and-4.md` (canonical anchor) while simultaneously creating the `overview-and-orientation.md` aggregator whose "Cross-ceiling content" summary paragraph retained a copy of the same `<a id="ceiling-interaction-order">`; that copy is the duplicate. The later 81bf52a ("collapse spec/ section pages into spec_topics/") only moved the file path `docs/spec/` → `docs/spec_topics/`, carrying the ghost verbatim — it did not introduce it.

## Solution Space

**Shape:** single

### Recommendation

Remove the `<a id="ceiling-interaction-order"></a>` tag from the compound "Cross-ceiling content" paragraph in `docs/spec_topics/overview-and-orientation.md` (line 54), leaving the canonical anchor in `ceilings-3-and-4.md` as the sole definition. The overview paragraph already forwards to the canonical anchor via its outbound link and is the target of no inbound `#ceiling-interaction-order` reference, so it needs no replacement anchor. If the orientation paragraph ever needs to be linkable in its own right, give it a distinct id (e.g. `#cross-ceiling-content-overview`) rather than re-using the canonical one. Note the same line also carries the `#no-additional-ceilings` ghost anchor handled by the sibling finding; removing both tags is a single edit to this line.

## Consequence

**Severity:** advisory

No reader is presently misdirected, because no inbound link targets the ghost. The latent hazard is that the same id is non-unique across the corpus: a corpus-wide anchor-uniqueness check flags it, and any future bare or relative `#ceiling-interaction-order` reference could resolve to the orientation summary rather than the normative CIO-1 … CIO-6 table.
