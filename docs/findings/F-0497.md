## Finding

The `V2e` and `V2e-T` Tests bullets assert "inbound translation rebuilds loom-side names so loom code never sees wire names; **defaults bypass inbound translation**." The word "defaults" has no antecedent in the bullet, the leaf, or the cited spec heading title.

The governing spec is narrow: `runtime-value-model.md` states only that **frontmatter `params:` defaults** bypass the inbound translation pass, and only because they are written in the Loom literal sublanguage, parsed at frontmatter-parse time, and therefore arrive already branded and loom-side-named. Read literally against the bare bullet, an implementer could broaden "defaults" to any default value (e.g. JSON Schema `default` keywords on a lowered schema, AJV-applied defaults during validation, or all frontmatter defaults regardless of how they are sourced). Those broader readings describe an over-broad bypass that the spec does not sanction — AJV-applied schema defaults arrive on the wire side and must still be translated inbound.

The bullet does link the RVM area that carries the precise statement, so a careful implementer who re-reads the spec sentence recovers the correct scope; the gap is that the bullet's own wording invites a wider test than the spec licenses.

## Solution Space

**Shape:** single

### Recommendation

In both `docs/plan_topics/V2e-wire-name-translation.md` and `docs/plan_topics/V2e-T-wire-name-translation.md`, edit the Tests bullet so the bypass clause names its scope, matching the spec. Replace the trailing clause `defaults bypass inbound translation` with text that pins it to frontmatter `params:` defaults and states why they arrive pre-branded, e.g.:

> frontmatter `params:` defaults bypass inbound translation (written in the Loom literal sublanguage and parsed at frontmatter-parse time, so they arrive already branded and loom-side-named).

The spec is read-only for this fix — `runtime-value-model.md` already states the rule correctly; only the plan bullets need to be brought into line with it. Keep the wording of the two leaves identical to each other so the `-T` and implementation partners continue to assert the same vector.

## Consequence

**Severity:** advisory

Two implementers building the test from the bullet alone could diverge on which defaults bypass translation; the broad reading asserts that wire-sourced defaults skip the inbound rebuild, which contradicts the spec and would let a real translation bug ship green. The spec link mitigates this for an implementer who re-reads the precise sentence, so a working leaf is still reachable.
