## Finding

The manual real-host smoke gate (`docs/plan_topics/real-host-smoke-gate.md`) defines an **Activation window**: both smoke triggers — (1) a Pi version bump and (2) a merge touching the four fidelity-contract axes — are dormant until `H7a` lands, because the smoke run is defined entirely in terms of two `H7a`-owned inputs (the committed multi-feature fixture `.loom` and the committed permitted-code list). A trigger event during the `H4a`→`H7a` window is therefore **deferred**: its owner records the deferral in the triggering commit message and is obligated to re-run the smoke once `H7a` has landed.

The protocol records the deferral but charges no surface with closing it. Nothing tracks the outstanding re-run obligation, and no gate reconciles a recorded deferral against a later recorded smoke result. The smoke gate is already "a contributor obligation, not a CI-enforced gate," detectable "only after the fact" by reading commit messages — but the after-the-fact reader (the `H6a` release-gate auditor) is given no checklist item to chase deferred runs. `H6a`'s **Ships when** smoke confirmation verifies that a smoke run "was executed and passed against the shipping Pi-SDK pin"; it does not enumerate the `H4a`→`H7a`-window deferral records and confirm each was later satisfied. The nine-item release-time residue inspection in `conventions.md` likewise has no item for it.

Consequently a window-deferred smoke run can be silently dropped: the deferral is recorded, `H7a` lands, and the owner simply never re-runs. The release gate flips green with no record that the deferred fidelity check ever executed.

## Consequence

**Severity:** advisory

A real-host smoke trigger that fires during the `H4a`→`H7a` window is deferred with no tracking that the obligated re-run ever happens; the deferral record has no closing surface, so a dropped re-run leaves a divergence the trigger was meant to catch unverified at release with nothing flagging the omission.

**History:** `docs/plan_topics/real-host-smoke-gate.md` is git-tracked. `git log --follow --diff-filter=A` shows the file was created in `fd22a3d`, and `git log -G "Activation window"` / `git log -G "deferred"` return only `fd22a3d` (creation) and `c768005` (a later unrelated edit relocating the `V9i` ERR-8 witness). The **Activation window** paragraph that defines the deferral was authored in the creating commit, and it shipped with the tracking gap already present: it instructs the owner to record the deferral and re-run once `H7a` lands, but never charges any surface with reconciling the recorded deferral against a later run. The defect is co-extensive with the deferral mechanism itself — it has been present since the paragraph's inception, not introduced by a later edit.

## Solution Space

**Shape:** single

### Recommendation

Charge the `H6a` release-gate auditor with reconciling every `H4a`→`H7a`-window deferral against a later recorded smoke result.

In `docs/plan_topics/H6a-live-corpus-activation.md`, extend the **Ships when** manual-real-host-smoke release-gate confirmation so the release does not pass until the auditor confirms that, for every deferral recorded in a triggering commit message during the `H4a`→`H7a` window, a corresponding smoke run was subsequently executed and recorded (passed, or its divergence dispositioned via the merge-block / revert path). An unreconciled deferral — a recorded deferral with no later recorded smoke result — blocks the loom 1.0 release until resolved, the same disposition the existing smoke and residue confirmations carry.

In `docs/plan_topics/real-host-smoke-gate.md`, add to the **Activation window** paragraph a forward-reference stating that the deferral record is reconciled at the `H6a` release gate, so the recorded deferral has a named closing surface rather than resting on the owner's memory.

Place the reconciliation at `H6a` rather than `V18d`: `V18d` is the version-bump (trigger-1) revert path and only sees bump-triggered deferrals, whereas trigger-(2) fidelity-axis-merge deferrals have no version-bump occasion. The `H6a` release-gate auditor is the trigger-agnostic surface that already reads the commit-message evidence record for both triggers, so it is the only home that closes every deferral class.
