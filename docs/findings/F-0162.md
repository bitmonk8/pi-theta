## Problem

`V9k`'s `pi.getCommands()` read-failure bullet (in both `docs/plan_topics/V9k-extension-bootstrap-failures.md` Tests and `docs/plan_topics/V9k-T-extension-bootstrap-failures.md` Tests) asserts the handler "swallows the throw and MUST NOT set drain state (drain state is owned by `V9m`'s `LoomRegistry` contract)". Witnessing that negative requires reading the drain-state surface after the handler runs, but that surface is owned by `V9m` and does not exist until `V9m` lands. Neither leaf lists `V9m` in Deps (`V9k`: `V9k-T, H4a, V9b, V7a`; `V9k-T`: `H4a, V9b, V7a`), so each is pickable before `V9m` exists — yielding either a compile-time failure or a vacuous test that never witnesses the "MUST NOT" guarantee. The sibling `V9h`/`V9h-T`, which observes the same surface, already declares `V9m`, so the omission is an inconsistency.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commit:** `cb8737a` ("pi-loom plan: resolve \"Systemic leaf over-bundling across the plan corpus\"", 2026-06-12)
**History:** `V9k` and `V9k-T` were created at `6df646a` ("…extension-bootstrap SDK-failure rule has no closing leaf", 2026-06-12). At that point the read-failure bullet read "drain state is owned by `V9b`'s `LoomRegistry` contract" and `V9k` correctly listed `V9b` in Deps — the asserted surface and the declared dependency agreed. `cb8737a` split the drain-state contract out into a new `V9m` leaf and re-pointed `V9k`/`V9k-T`'s prose to "`V9m`'s `LoomRegistry` contract", but left both Deps lines unchanged at `V9b` only. The same commit *did* migrate the sibling `V9h`/`V9h-T` Deps from `V9b` to `V9m` for the identical surface, so the defect is an incomplete propagation of that split: the Deps edge was updated for `V9h` and missed for `V9k`/`V9k-T`.

## Solution approach

Add `V9m` to the Deps field of `docs/plan_topics/V9k-extension-bootstrap-failures.md` and of `docs/plan_topics/V9k-T-extension-bootstrap-failures.md`, so each leaf's dependency edge matches the drain-state surface its read-failure bullet observes and mirrors the sibling `V9h`/`V9h-T` declarations.
