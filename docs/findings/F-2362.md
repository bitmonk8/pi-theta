## Problem

The `explicit-schema-mismatch` Tests bullet in both `V13b` and `V13b-T` ends with "the four vectors", which names the test scope but neither enumerates the four inputs nor links to where they are defined. The four normative test vectors do live in the cited spec, in the `*Test vectors (normative).*` block of the `### Explicit form` section (anchor `qry-4` in `query-forms.md`), but nothing in the bullet connects "the four vectors" to that block. A fresh implementer has no signal that "four" means precisely those four normative cases, and the bullet's "…fires … — the four vectors" phrasing mis-frames the set, since only one of the four fires the warning while the other three are no-warning compatibility cases. The same "the four vectors" phrasing also appears in the `V13b` Ships-when sentence.

The defect is an original-authoring under-specification, not a regression introduced by a later edit.

## Solution approach

In both `V13b-query-schema-inference.md` and `V13b-T-query-schema-inference.md`, rewrite the bare "the four vectors" in the `explicit-schema-mismatch` Tests bullet to cross-reference the spec's normative test-vectors block at `../spec_topics/query/query-forms.md#qry-4`, and make clear that of those four vectors only the `integer`-annotated `@<number>` case fires while the other three are no-warning compatibility cases. Apply the same cross-reference to the `Ships when` sentence in `V13b-query-schema-inference.md`. The existing `qry-4` anchor already reaches the block, so no new spec anchor is needed.

## Solution constraints

- The spec is read-only for this fix; the four vectors already exist under `qry-4`, so add only the cross-reference and gloss.
