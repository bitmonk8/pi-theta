## Problem

`V13f` and `V13f-T` instrument the `@`-query provider's abandonable Promise ÔÇö asserting that the swallowing handler attaches at the Promise-construction site and that a late settlement is suppressed along all three side channels. That construction site does not exist until the `@`-query construct (`V13a`) and the provider-call tool loop (`V13c`) land, yet neither leaf's `Deps` names any V13 leaf (`V13f` Ôćĺ `V13f-T`, `V9n`, `V8a`, `H4b`; `V13f-T` Ôćĺ `V9n`, `V8a`, `H4b`). Because pickability is decided purely on direct-Deps completion, both leaves become pickable before `V13a`/`V13c`, leaving an implementer with no `@`-query to dispatch and no constructible late-settlement test. Every sibling per-site routing leaf (`V14f Ôćĺ V14a`, `V15h Ôćĺ V15a`, `V9o Ôćĺ V9i`, `V13g Ôćĺ V13a`) declares its site-owner; `V13f` is the lone exception.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `eaa5000` (2026-06-16, "pi-loom plan: resolve \"V13c bundles four independent spec obligations into one leaf\"")
**History:** `V13f-query-cancellation-routing.md` and `V13f-T-query-cancellation-routing.md` were both created in `eaa5000` (verified via `git log --follow --diff-filter=A`). At creation the Deps were `V13f-T, V9c, V8a, H4b` ÔÇö already carrying no V13 site-owner (`V9c` is the prompt-mode conversation-drive leaf, not the `@`-query construct). A later commit `0c3a2a2` (2026-06-17) retargeted that single non-V13 edge `V9c Ôćĺ V9n` during the V9c split but did not add a V13 dependency. The omission has therefore been present from the leaf's first commit; no later edit introduced or removed it.

## Solution approach

Add the V13 leaf that owns the `@`-query Promise-construction site to the `Deps` field of both `V13f` and `V13f-T`. `V13c` owns the provider-call tool loop that constructs the abandonable provider Promise and transitively reaches the `@`-query construct via `V13c Ôćĺ V13b Ôćĺ V13a`; if the instrumented Promise is the bare `@`-query render/dispatch rather than the tool-loop provider call, add `V13a` instead. Match whichever V13 leaf actually owns the construction site `V13f`'s tests attach to, so both leaves become pickable only after that site exists.
