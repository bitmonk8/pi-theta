## Finding

`H5b`'s **Deps** parenthetical asserts coverage completeness in the present indicative: "The dependency set **is** the complete coverage-producing set … It **comprises every leaf** that can introduce an executable REQ-ID, a numbered-REQ-ID citing test, or an un-anchored normative MUST." That phrasing reads as a standing fact, but the set is maintained by hand — the same parenthetical names "the standing plan-maintenance obligation stated in *REQ-ID discipline*" as what keeps it complete.

The only mechanical backstop is `H5d` (the transitive-completeness gate arm). By its own description `H5d` "reddens only for a closing-leaf cell none of whose listed leaves reaches `H5b`'s `Deps.`, and stays green for a genuine multi-facet split row where one listed facet-closer is present while a sibling facet-closer for the same cell is omitted." It is a per-cell at-least-one test; it cannot catch a secondary facet-closer dropped from `H5b.Deps` while a sibling remains. So facet-level completeness of the set is an obligation, not a guarantee, and the present-tense "comprises every leaf" overstates it.

The adjacent parenthetical clause already concedes the manual obligation, so a reader is not actively misled — this is a wording overclaim, not a substantive defect. Softening the factual present-tense to intended-language brings the claim into line with the guarantee that actually holds.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H5b-warn-only-canary.md`, soften the **Deps** note parenthetical so the completeness of the coverage-producing set reads as the intended target of the standing plan-maintenance obligation rather than a present-tense fact. Concretely, rephrase the two present-indicative completeness assertions:

- "The dependency set **is** the complete coverage-producing set" → intended-language, e.g. "The dependency set **is intended to be** the complete coverage-producing set".
- "It **comprises every leaf** that can introduce an executable REQ-ID, a numbered-REQ-ID citing test, or an un-anchored normative MUST" → intended-language, e.g. "It **is intended to comprise every leaf** that can introduce an executable REQ-ID, a numbered-REQ-ID citing test, or an un-anchored normative MUST (kept complete by the standing plan-maintenance obligation)".

Leave the existing clauses naming the `H5d`/`H5f`/canary backstops and the *REQ-ID discipline* plan-maintenance obligation intact — those already state correctly how the set is kept complete. No enforcement-posture change and no edit to `H5d`, `conventions.md`, or the gate machinery is required; this is a wording reconciliation internal to `H5b`'s Deps note.

## Consequence

**Severity:** cosmetic

A plan maintainer reading "comprises every leaf" / "is the complete coverage-producing set" as a mechanical guarantee may assume `H5d` enforces facet-level completeness, when `H5d`'s per-cell at-least-one test stays green if a secondary facet-closer is dropped while a sibling remains. The same parenthetical already concedes the manual maintenance obligation, so no leaf becomes unpickable and no gate misfires; the impact is a mismatch between asserted certainty and the guarantee that actually holds.
