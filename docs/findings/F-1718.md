## Problem

PIC-13's `realpath` bullet binds the **Case-canonicalisation guarantee** — and the **Byte-stable canonical-output guarantee** of which it is the case-insensitive-filesystem instance — onto both the production `PiFileSystem` adapter and `FakeFileSystem`, explicitly stating that bare delegation to `fs.promises.realpath` is insufficient for the production half because Node's `fs.promises.realpath` and `fs.promises.realpath.native` differ on leaf-case output. The witnessing fixture — two case-variant paths that resolve to one directory entry — cannot be constructed on a case-sensitive filesystem, which is the only CI substrate `npm test` commits to. PIC-13 carries no platform-host requirement, no skip protocol, and no delegation to the manual real-host smoke gate, and the "no silent test skipping" rule (`CLAUDE.md` §Testing) forbids an early-return/skip-macro workaround on Linux. The production-adapter half of the guarantee is therefore exercised by no witness, leaving leaf-case canonicalisation in `PiFileSystem` unverified, which DISC-3 case-collision dedup and the INV-1 invocation-time containment re-check both depend on.

  - `6ab9f7b` (2026-06-25) "pi-loom spec-plan: fix F-1293 — pin PIC-13 realpath transitive resolution + byte-stability" — extended the same bullet with the **Byte-stable canonical-output guarantee** (of which case-canonicalisation is named the case-insensitive-FS instance) and the **Transitive-symlink-resolution guarantee**, again binding both halves to the production adapter without addressing the production-adapter testability gap.
**History:** the defect entered with the guarantee itself: `a765f6ca` is the only commit that has ever touched the **Case-canonicalisation guarantee** clause on this file (verified via `git log -S "byte-identical" -- docs/spec_topics/pi-integration-contract/host-interfaces-services.md`, which returns exactly the two commits above; the more general byte-stable phrasing arrived at `6ab9f7b`). Neither fix paired its normative bind-to-both-adapters with a testability protocol — no platform-host requirement, no skip-on-case-sensitive-host clause, no delegation to the real-host smoke gate, and no companion edit to `V8b` / `V8b-T` adding a production-adapter test bullet. The guarantee was therefore untestable against the production adapter under `npm test` from its first commit, and remains so on HEAD.

## Solution approach

Scope the `npm test` witness of PIC-13's case-canonicalisation guarantee to `FakeFileSystem`, and delegate the production `PiFileSystem` half to the manual real-host smoke gate (`real-host-smoke-gate.md`). Clarify PIC-13's `realpath` bullet that the production-adapter half of both the case-canonicalisation guarantee and the byte-stable canonical-output guarantee is witnessed at the smoke gate rather than under `npm test`. Add a Pass/fail criterion to `real-host-smoke-gate.md` witnessing the production-adapter guarantee on a case-insensitive live host, and extend its fidelity-axis activation trigger to fire on edits to the adapter's case-canonicalisation step. Add a `FakeFileSystem` test bullet to `V8b` / `V8b-T` and note the production half's delegation to the smoke gate.

## Solution constraints

- The production `PiFileSystem` adapter MUST remain bound by the case-canonicalisation and byte-stable canonical-output guarantees as a conformance obligation; the fix re-routes where the production half is witnessed, not whether it binds.
