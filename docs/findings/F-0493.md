## Finding

The `H7a` leaf's H1 heading — `# `H7a` — Terminal integration-acceptance run (cross-slice end-to-end gate)` (and the identical link text in `docs/plan.md`'s Horizontal-phases list) — makes "run" the primary noun classifying the leaf. Everywhere else the leaf is a gate: its own **Adds** opens "A terminal integration-acceptance **gate** that runs …" and later "it is a cross-slice integration-regression **gate**"; `docs/plan_topics/real-host-smoke-gate.md` names it "the `H7a` integration-acceptance fixture" / "the integration-acceptance **gate**"; `H6a` calls it "the terminal integration-acceptance leaf"; and `H6a`'s **Ships when** treats it as a release-gate input. The parenthetical in the heading itself even ends in "(cross-slice end-to-end **gate**)", so the same heading carries both classifiers.

The sibling closing-gate family all advertise "gate" in the heading's primary classifier — `H5a` "closing-gate automation", `H5c` "closing-gate reconciliation", `H5d` "closing-gate arm", `H5e` "closing-gate arm", `H6a` "closing-gate activation (loom 1.0 release gate)". `H7a` is the one CI-gate leaf whose heading leads with "run", so a reader scanning headings for the gate family does not recognise it as a member.

This is a pure heading-vs-body naming mismatch on one leaf. The leaf's behaviour, **Deps**, **Tests**, and **Ships when** are unaffected; only the heading's lead classifier diverges from how the rest of the corpus refers to the leaf.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H7a-integration-acceptance.md`, change the H1 heading to lead with "gate" so the primary classifier matches the body and the sibling gate family, e.g. `# `H7a` — Terminal integration-acceptance gate (cross-slice end-to-end)`. Apply the identical text to the H7a link in `docs/plan.md`'s Horizontal-phases list so the two stay in sync. Dropping "gate" from the trailing parenthetical (now "(cross-slice end-to-end)") avoids the heading carrying "gate" twice.

The filename `H7a-integration-acceptance.md` contains no "run" token and needs no change; cross-references resolve by file path and leaf-ID (`H7a`), not by heading text, so no link or anchor breaks. This edit is confined to the two heading occurrences — no **Adds** / **Tests** / **Deps** / **Ships when** change is required.

## Consequence

**Severity:** cosmetic

A reader scanning leaf headings for the CI closing-gate family (H5a/H5c/H5d/H5e/H6a, each headed "…gate…") does not find H7a, because its heading leads with "run". The leaf's content makes its gate role unambiguous, so no implementer produces a wrong leaf; the cost is discoverability and a momentary heading-vs-body reconciliation.
