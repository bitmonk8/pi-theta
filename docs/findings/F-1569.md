## Finding

The corpus marks a pre-evaluation-failure definition with the convention `<a id="err-N"></a> **ERR-N.**` — an anchor immediately followed by the bold REQ-ID token. ERR-7's canonical definition follows this convention in `errors-and-results/error-model.md` (line 66: `7. <a id="err-7"></a> **ERR-7.** watcher-time reload failures …`).

In `discovery/package-and-settings.md` the "Watcher-time reload failures" paragraph (line 98) cites ERR-7 with a standalone bold token — "surfaces its rebuild failures as the **ERR-7** pre-evaluation failure defined in [Errors and Results — ERR-7](../errors-and-results/error-model.md#err-7)". This is a citation, not a definition (the link in the same clause points to the canonical owner), but the bold `**ERR-7**` token is shaped identically to a definition marker. Any traceability index or audit that extracts REQ-ID definition sites by scanning for bold `**REQ-ID**` tokens registers a false second ERR-7 definition here, inverting the definition→citation relationship and breaking the single-definition assumption such audits rely on. The two later mentions in the same file (lines 103, 105) are already plain text and are not affected.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** d488eaa — pi-loom spec: resolve "ERR-7 defining anchor + collision payload field" (2026-06-06, Thomas Andersen)
**History:** The "Watcher-time reload failures" paragraph — including the standalone bold `**ERR-7**` token on line 98 — was added to `discovery/package-and-settings.md` whole-cloth by commit d488eaa, the same commit that established ERR-7's defining anchor in `error-model.md`. `git log -S '**ERR-7**'` on the file returns only this commit, and the paragraph (anchor `watcher-time-reload-failures`) has no prior history. The bold styling was applied to the citation at the moment the cross-reference was authored.

## Solution Space

**Shape:** single

### Recommendation

Convert the bold `**ERR-7**` token on line 98 to a plain hyperlinked citation so it cannot be mistaken for a definition marker. Replace `**ERR-7**` with `[ERR-7](../errors-and-results/error-model.md#err-7)`; the existing trailing "defined in [Errors and Results — ERR-7](…)" link then becomes redundant and may be folded into the same reference or dropped. The defining convention (`<a id>` + bold) stays exclusive to the canonical site in `error-model.md`. Leave the plain-text mentions on lines 103 and 105 as-is — they already carry no bold styling.

## Consequence

**Severity:** cosmetic

A definition-extraction index or audit that treats bold `**REQ-ID**` tokens as definition sites records a false second ERR-7 definition in `package-and-settings.md` alongside the canonical one in `error-model.md`, corrupting the definition→citation map and any single-definition-uniqueness check that consumes it. No implementer behaviour is affected; the defect is confined to traceability tooling.
