## Finding

`V6c`'s Tests carry a single combined bullet — "The resolved callable set is frozen (no ambient inheritance); both YAML spellings parse." — to close the "frozen resolution snapshot" deliverable named in its Adds. The word *frozen* bundles two distinct spec properties that need distinct observables. The first, *no ambient inheritance* (`frontmatter-fields-a.md` §`tools`: "The Pi session's ambient tools are deliberately *not* inherited"), is testable as written. The second, *snapshot immutability* — the spec's **Resolution snapshot** rule (`frontmatter-fields-b-and-templates.md` §Resolution snapshot: "Loom-load time resolution produces a frozen per-loom table … the runtime never re-queries Pi's tool registry by name during execution") — has no stated assertion: the bullet names the property but supplies no observable a test author can check.

Because the bullet conflates the two, a test author can satisfy "frozen" by asserting only that ambient tools are not inherited, while the snapshot-stability behaviour the spec's **Resolution snapshot** rule mandates — that a change to Pi's tool registry after load does not alter the loom's already-resolved callable set — ships with no exercising test. The `V6c-T` tests leaf carries the identical bullet, so the gap is mirrored on the failing-test side and the leaf can pass its `npm test` gate with the snapshot-immutability branch never exercised.

## Consequence

**Severity:** advisory

The snapshot-immutability half of "frozen" has no test observable, so `V6c`/`V6c-T` can pass their gates with the spec's **Resolution snapshot** stability behaviour (post-load registry change does not alter the resolved set) never asserted. Two reasonable test authors diverge on what closes "frozen": one asserts only no-ambient-inheritance, the other also asserts registry-change isolation.

## Solution Space

**Shape:** single

### Recommendation

In `V6c` and `V6c-T`, replace the single "frozen" Tests bullet with two bullets, each naming an observable:

- A no-ambient-inheritance bullet: a Pi session ambient tool not listed in `tools:` is absent from the resolved callable set — the model cannot call it and loom code cannot resolve it via `<name>(...)`.
- A snapshot-immutability bullet: after load resolves the callable set, a change to Pi's tool registry (e.g. unregistering an already-resolved Pi tool) does not alter the loom's resolved set; an in-flight invocation continues to dispatch through the captured reference. Cite the **Resolution snapshot** rule in `frontmatter-fields-b-and-templates.md` (`#resolution-snapshot`).

Keep "both YAML spellings parse" as its own bullet (it is an unrelated observable already concrete). Apply the same bullet set to `V6c-T` so the failing-test leaf and the implementation leaf stay in lockstep.
