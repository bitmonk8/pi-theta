## Finding

`V11a`'s **Adds** binds the shared model-reference-matcher contract by explicitly referencing a seam that another leaf owns: "the shared model-reference-matcher contract that `V6a`'s load-time `model:` resolution also binds through **the injection seam `V6a` defines**". `V6a`'s own Adds confirms this split — `V6a` "defines the **model-reference-matcher injection seam**" (the interface the parser's `model:` resolution hook calls), while `V11a` owns the shared `findExactModelReferenceMatch` resolution semantics, and `V9b` constructs/injects the concrete matcher.

`V11a`'s **Deps** (`V11a-T`, `V9b`, `V10c`) names the injector (`V9b`) and the settings source (`V10c`) but not `V6a`, even though the Adds names `V6a` as the definer of the seam interface `V11a` binds against. The paired `V11a-T` Deps (`V9b`, `V10c`) likewise omit it. The implementer picking up `V11a` is pointed at the injector and the prose contract but is not directed to read `V6a`'s seam-interface definition. Because `V6a` and `V11a` must decide the "reference matches no available model" condition identically, omitting the read-pointer to the seam owner invites a divergent private matcher-wrapper on that condition.

The build-order prerequisite is satisfied transitively (`V6a` → `V9b` → `V11a`, since `V9b` lists `V6a` in Deps), so this is a read-pointer / reconciliation gap rather than a missing ordering edge.

**History:** `V11a-binder-model-resolution.md` was created at `c6a664e` (2026-06-10) with Deps `V11a-T, V9b, V10c` and an Adds that did not reference `V6a` or the matcher seam (`git log -S'injection seam'` and `git show 5349d5f:` confirm no `V6a`/matcher/seam mention before `5953b18`). Commit `5953b18` added the shared model-reference-matcher cross-reference ("the injection seam `V6a` defines") to `V11a`'s Adds while resolving a separate `V6a`-producer finding, but left the Deps line unchanged — introducing the Adds-references-`V6a`/Deps-omits-`V6a` mismatch. The Deps line `V11a-T, V9b, V10c` has carried unchanged across all four revisions of the file.

## Solution Space

**Shape:** single

### Recommendation

Add `V6a` to the **Deps** field of `docs/plan_topics/V11a-binder-model-resolution.md` (from `V11a-T`, `V9b`, `V10c`) and to `docs/plan_topics/V11a-T-binder-model-resolution.md` (from `V9b`, `V10c`). This directs the `V11a` implementer to `V6a`'s model-reference-matcher injection-seam definition — the interface `V11a`'s `findExactModelReferenceMatch` resolution binds against — so the implementation reconciles against the established seam instead of a private wrapper.

The edge is consistent with the existing transitive order (`V6a` → `V9b` → `V11a`) and introduces no cycle: `V6a`'s Deps (`V6a-T`, `V1a`, `V5a`) do not reference `V11a`, and `V6a` deliberately carries no forward edge onto the binder-model machinery. Edge case for the implementer: the new edge records `V11a`'s dependency on `V6a`'s seam *interface*; it does not transfer contract ownership — `V11a` remains the owner of the shared resolution semantics and `V6a` remains the owner of the injection-seam interface, exactly as both leaves' Adds already state.

## Consequence

**Severity:** advisory

If `V11a` ships unfixed, an implementer reading only the cited Deps reaches the injector (`V9b`) but not the seam-interface definition (`V6a`), and may build a private matcher-wrapper that diverges from `V6a`'s `model:`-resolution behaviour on the "no available model" condition. The inline Adds prose ("the two paths cannot diverge") mitigates the risk, so a working leaf is still producible; the gap is an incomplete read-pointer rather than a blocker.
