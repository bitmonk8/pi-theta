## Problem

The leaf-file pair `docs/plan_topics/V8b-clock-fs-id-watch-token-seams.md` and its `-T` counterpart carry a slug that enumerates five seam categories (`Clock`, `FileSystem`, `IdSource`, `FileWatcher`, `TokenEstimator`), but V8b now owns only the `FileSystem` seam — its `plan.md` display title and `Adds` field scope it to `FileSystem` and hand the other four seams to V8d and V8e. The stale slug propagates to every markdown link spelling out the path: the §V8 list item in `docs/plan.md` and the reciprocal cross-links in V8d's and V8e's `Adds` fields. A reader navigating by filename or following those links is told V8b owns four seams it does not implement. The leaf ID `V8b` itself is correct; only the file slug and the link targets are stale.

## Issue introduction

**Verdict:** introduced-by-later-edit (single commit)
**Introducing commit:** `cb8737a` — "pi-loom plan: resolve 'Systemic leaf over-bundling across the plan corpus'" (2026-06-12)
**History:** At the initial plan build (`c6a664e`, 2026-06-10) V8b was a single leaf that owned all five seams; its `plan.md` entry read `V8b — `Clock`, `FileSystem`, `IdSource`, `FileWatcher`, `TokenEstimator` seams`, and the slug `V8b-clock-fs-id-watch-token-seams` accurately described that scope. Commit `cb8737a` split the leaf — V8b retained `FileSystem` only, the new V8d took `Clock`/`IdSource`, and the new V8e took `FileWatcher`/`TokenEstimator` — and updated `plan.md`'s display title and V8b's `Adds` to the narrowed scope, but did not rename the file. The slug has named seams the leaf no longer owns since that split.

## Solution approach

Rename the `V8b` leaf-file pair to a slug that reflects its `FileSystem`-only scope (e.g. `V8b-filesystem-seam.md` and `V8b-T-filesystem-seam.md`). Repoint the three markdown links that target the old path: the §V8 list-item link in `docs/plan.md`, the `V8b` cross-link in `docs/plan_topics/V8d-clock-id-seams.md`'s `Adds` field, and the `V8b` cross-link in `docs/plan_topics/V8e-watch-token-seams.md`'s `Adds` field.

## Solution constraints

- Out of scope: the leaf ID `V8b` is correct and unchanged — do not alter it or the bare `V8b` Deps tokens that reference the ID rather than the path (V1a/V1a-T, V10a–c, M, H4a, `coverage-matrix.md`).
