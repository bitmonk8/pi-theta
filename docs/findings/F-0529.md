## Finding

`V9f` (Tool-registration lifetime and visibility) is built on the active-set gating window that `V9c` (Prompt-mode conversation drive and active-set gating) creates. V9f's **Adds.** explicitly names "the active-set gating window's snapshot/restore on the prompt path (defined in [`V9c`](./V9c-conversation-drive.md))", and both of V9f's failure-path tests perturb that window: `PIC-8` exercises a step-4 restore throw, `PIC-19` a step-1/step-2 snapshot/swap-install throw. The window's snapshot → `setActiveTools` → query → `finally` restore bracket is owned by V9c (`PIC-17`/`PIC-2`), not V9f.

V9f's **Deps.** are `V9f-T`, `V9a`, `V5d`. Neither V9a (Deps `V9a-T`, `H4a`, `V7d`) nor V5d (Deps `V5d-T`, `V5a`, `V5b`, `V2d`) lists or transitively reaches V9c — in fact V9c depends on V9a, so V9a is upstream of V9c, never downstream. The dependency on V9c's window is therefore undeclared. Under the pickability predicate (How-to-use step 3), V9f becomes pickable as soon as `V9f-T`, `V9a`, `V5d` are complete, at which point the active-set gating window does not yet exist and V9f cannot be implemented or red-tested as written.

The paired tests leaf `V9f-T` (Deps `V9a`, `V5d`) carries the same gap: its `PIC-8`/`PIC-19` tests perturb the window's failure modes, so they can only "fail red for the intended reason" against an existing window seam — which V9c provides.

## Solution Space

**Shape:** single

### Recommendation

Add `V9c` to the **Deps.** of `docs/plan_topics/V9f-tool-registration-lifetime.md` so the field reads `` `V9f-T`, `V9a`, `V9c`, `V5d` ``, and add `V9c` to the **Deps.** of `docs/plan_topics/V9f-T-tool-registration-lifetime.md` so its field reads `` `V9a`, `V9c`, `V5d` ``. This makes V9f and V9f-T unpickable until V9c's active-set gating window exists, matching the dependency the Adds prose and the PIC-8/PIC-19 tests already assume.

Edge cases for the implementer: V9c's own Deps (`V9c-T`, `V9a`, `V8a`, `V17a`) already include V9a, so the explicit `V9a` entry on V9f becomes redundant once V9c is added; keeping it explicit is harmless and need not be stripped. Adding V9c pulls in V8a/V17a transitively, which is the intended ordering — the window seam must be present before V9f perturbs its failure modes.

## Consequence

**Severity:** blocking

An implementer following the pickability predicate can pick `V9f` (and `V9f-T`) before `V9c` is complete, because the declared Deps are satisfiable without V9c. At that point the active-set gating window the PIC-8/PIC-19 tests perturb does not exist, so the leaf cannot be implemented or red-tested as written — the contributor stalls or invents a divergent window shape.
