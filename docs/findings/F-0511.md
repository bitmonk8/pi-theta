## Finding

`V4g`'s `ERR-7` Tests bullet names the same injection point twice in one sentence: the failure is "injected via `ReloadFailureInjector.injectReloadFailure` … at the **watcher-time reload failure-injection seam**". The interface symbol (`ReloadFailureInjector.injectReloadFailure`) and the bolded free-standing descriptive term ("watcher-time reload failure-injection seam") denote one boundary, but their side-by-side use lets a reader read them as two distinct seams. `V4g`'s Adds compounds this by calling the same thing "`V9b`'s `ReloadFailureInjector.injectReloadFailure` seam" — a third phrasing for the one concept.

The descriptor is not local to `V4g`: it is a deliberate, corpus-wide defined term. `V9b` establishes it ("the single declaration site of the test-only `ReloadFailureInjector` interface for the whole **watcher-time reload failure-injection seam**"), and it recurs in `V4g-T`, `V10c`, and `V10d`. So the defect is a presentational naming inconsistency spanning the reload-failure seam's leaves, currently surfaced in `V4g`'s Tests bullet — not an ad-hoc one-off coinage.

This breaks the interface-name seam pattern the plan uses everywhere else: each `V8` host seam is named by its interface (`Checkpoint` seam, `FileSystem` seam, `Clock` seam, `IdSource` seam, `FileWatcher` seam, etc.). The reload-failure seam is the lone DI seam that carries a free-standing descriptive name alongside a differently-named interface. The binding code-level symbol is used consistently, so no implementer would build the wrong interface; the cost is the dual defined-term presentation.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `b2d5a1f` (2026-06-11, "ERR-7 test bullet states different normative content") first introduced the descriptive "watcher-time reload failure-injection seam" term into the plan leaves; `d64dce5` (2026-06-11, "Watcher-time reload failure-injection seam under-specified and ungated") added the `ReloadFailureInjector` interface name alongside the already-present descriptive term, making the two-name coexistence live. The dual naming propagated into `V4g` at `324813d` (2026-06-18, "split ERR-7 reload scenario out of V4e routing checks"), which created `V4g`/`V4g-T` by carving the `ERR-7` reload scenario (and its dual-named seam prose) out of `V4e`.
**History:** The plan corpus is git-tracked. `git log -G "watcher-time reload failure-injection seam" --reverse -- docs/plan_topics/` first surfaces the descriptive seam term at `b2d5a1f`; `git log -S "ReloadFailureInjector" --reverse -- docs/plan_topics/` first surfaces the interface name at `d64dce5` (same day). The coexistence was then preserved through `fdce2a2` (2026-06-12, "ReloadFailureInjector seam has two owners") and `f14e929` (2026-06-16, the `V10c`→`V10d` split), neither of which collapsed the descriptive term into the interface name, and finally inherited by `V4g` at the `324813d` split (`git log --follow` shows `V4g`'s file originating there). No single commit introduced "two names for one concept" in isolation — it emerged from the interface name landing next to a pre-existing descriptive term and surviving subsequent reworks, including the `V4e`→`V4g` split.

## Solution Space

**Shape:** single

### Recommendation

Unify the reload-failure DI seam onto its interface name so one concept reads as one seam, matching the `V8` interface-name seam pattern.

In `V4g` (`ERR-7` Tests bullet) and its mirror in `V4g-T`, the clause "at the **watcher-time reload failure-injection seam**" is redundant with "via `ReloadFailureInjector.injectReloadFailure`"; strike the bolded descriptor so the seam is referenced only by its interface name. In `V4g`'s Adds, refer to the injection point as the `ReloadFailureInjector` seam declared by `V9b` rather than "`V9b`'s `ReloadFailureInjector.injectReloadFailure` seam". The word "watcher-time" may remain as ordinary prose where it usefully says *when* the failure occurs; it should no longer stand as the seam's name.

For full consistency across the seam's leaves, apply the same unification to the descriptor occurrences in `V9b` Adds ("for the whole watcher-time reload failure-injection seam"), `V10c` Adds, and `V10d` Adds/Deps. The interface name `ReloadFailureInjector` and method `injectReloadFailure` already match the target, so there are no code-level symbol changes. The spec is read-only for this fix (the seam is a plan-internal, test-only interface).

## Consequence

**Severity:** cosmetic

A reader encountering the bolded "watcher-time reload failure-injection seam" next to the `ReloadFailureInjector` interface name can read them as two distinct seams rather than one injection point. The interface symbol is unambiguous and consistently used, so the leaf stays implementable; the defect is presentational consistency only.
