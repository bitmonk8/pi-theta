## Problem

`V17b` ("Forwarding-listener throw-trap") binds the `cancellation.md` §*Forwarding-listener throw* obligation — the steady-state throw-trap MUST that routes a throw from any forwarding listener's `loomAbort.abort(source.reason)` through `loom/runtime/internal-error` without swallowing the cancellation — and `V17a` explicitly disclaims that clause. `docs/plan_topics/coverage-matrix.md` carries no row whose closing leaf is `V17b`: the `CNCL-1 … CNCL-6` rows all close at `V17a`, and the code-keyed obligation-area table has no `cancellation.md` entry. The clause is an un-anchored MUST under GOV-22 mapped to no closing leaf, so the H5a closing gate has no cell to witness V17b's closure.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commits:** `cb8737a` ("pi-loom plan: resolve \"Systemic leaf over-bundling across the plan corpus\"")
**History:** The plan corpus is git-tracked. `git log --follow` shows `docs/plan_topics/V17b-forwarding-listener-throw-trap.md` and `V17b-T-forwarding-listener-throw-trap.md` were first created in `cb8737a`, which peeled the throw-trap sub-mechanism out of `V17a` during the over-bundling split. The same commit edited `coverage-matrix.md`, but its edits did not touch the cancellation rows — the `CNCL-1 … CNCL-6 → V17a` rows were left intact and no row was added for the new `V17b` leaf or for the un-anchored *Forwarding-listener throw* MUST it closes. `git log -S "V17b" -- docs/plan_topics/coverage-matrix.md` returns no commit: `V17b` has never appeared in the coverage matrix. The gap therefore entered the corpus with the leaf's inception, not as a later regression.

## Solution approach

Add one row to the code-keyed obligation-area table in `docs/plan_topics/coverage-matrix.md` (under the *Code-keyed obligation areas* header, alongside the other `cancellation`-adjacent residue rows) citing `cancellation.md` §*Forwarding-listener throw*, marked `(un-anchored; GOV-22 residue)` consistent with the sibling rows, with `V17b` as the closing leaf.

## Solution constraints

- Out of scope: do not edit `cancellation.md` — the spec is read-only; the MUST already exists there and the matrix reads from it.
- Leave the `CNCL-1 … CNCL-6 → V17a` rows unchanged and scope the new row to the steady-state forwarding listeners only; the `session_shutdown` teardown-iteration swallow is governed separately and closes at `V9g`.
- Add only the implementation leaf `V17b`; the matrix lists implementation leaves, not `-T` tests tasks.
