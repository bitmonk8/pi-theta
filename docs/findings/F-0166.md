## Problem

`V9l`'s **Adds** bullet ends with a labelled `**Fallback (clause-(a) recovery).**` sentence describing a conditional implementation path gated on an unresolved spec open question (host-prerequisites clause (a)): if the branch resolves as *unreachable as written*, the leaf collapses to full teardown + `/reload` with no degraded tag write and the obligations are reworked spec-side before the leaf ships. The Adds field is the home for a leaf's unconditional deliverables; conditional blocked-leaf metadata belongs in `plan.md` §Blocked obligations, whose `V9l` entry already summarises this fallback. The canonical/summary relationship is therefore inverted — the full narrative sits in Adds while §Blocked obligations merely back-references it — and `V9l`'s Ships-when points the reader at "the Adds fallback" as authoritative, so the blocked-obligation policy is split across two surfaces that can drift.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 55ed974db389eaa12e81411172e20b753af5a940 ("pi-loom plan: resolve \"degraded-state obligations green over open spec contradiction\"")
**History:** `docs/plan_topics/V9l-session-only-degraded-branch.md` has a single commit in its `git log --follow` history (55ed974), the commit that split the session-only degraded-state branch out of `V9h` into `V9l`. `git log -S'Fallback (clause-(a) recovery)'` and `git log -G'Fallback \(clause'` across the repo both return only 55ed974, confirming the conditional fallback prose entered Adds at the file's creation. The defect has been present since `V9l` first existed; no later edit introduced it.

## Solution approach

Strike the `**Fallback (clause-(a) recovery).**` sentence from `V9l`'s Adds in `docs/plan_topics/V9l-session-only-degraded-branch.md`, leaving Adds to enumerate only the unconditional degraded-branch deliverables. Relocate the full conditional path into the existing `V9l` entry under `docs/plan.md` §Blocked obligations by extending its "Pre-designed fallback (in `V9l`)" clause to carry the complete fallback text, making §Blocked obligations the canonical home. Re-point `V9l`'s Ships-when so the *branch unreachable* case reads against `plan.md` §Blocked obligations rather than "the Adds fallback".

## Solution constraints

- Out of scope: `V9l`'s Deps blocking note and the `V18c` resolution-owner edge — they already point at §Blocked obligations and correctly gate pickability; leave them untouched.
- The spec is read-only for this fix: relocate the spec-side-rework description (`session-only-degraded-state.md`, `SM-4`/`SM-5`/`SM-6`/`SM-3b`) as text only; do not edit those spec sites.
