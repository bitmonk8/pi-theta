## Finding

Two distinct, substantive spec-topic files sit side by side in `docs/spec_topics/` with names that differ only by a trailing `-and-orientation`:

- `docs/spec_topics/overview.md` — titled **"Overview and Conceptual Model"**. It is the conceptual-model document (Code and Model, Scope of a Loom File). It is the file registered in the REQ-ID prefix table (`overview.md | (no IDs — narrative)`) and the file the Reading-order list in `language-and-architecture.md` links to ("Overview and Conceptual Model").
- `docs/spec_topics/overview-and-orientation.md` — titled **"Overview and orientation"**. It is the larger, implementer-facing orientation document (Prerequisites, Scope, Non-functional requirements, Hard-ceilings overview). It is **absent** from the REQ-ID prefix table and is not in the Reading-order list, yet it is the most heavily cross-referenced of the two (≈24 inbound references across the corpus and plan, versus ≈6 for `overview.md`).

Both files open with an identical `## Overview` heading and near-identical first sentences, so the only signal distinguishing them is the filename suffix. A maintainer following a cross-reference, or grepping for "the overview page", cannot tell from the filename alone which document carries which role, and risks editing the wrong file (e.g. appending orientation content to the conceptual-model page). The confusion is compounded by sibling findings in which GOV-30/GOV-31 prose labels `overview-and-orientation.md` targets as "`spec.md`".

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `fecb504` (2026-05-04, original spec-topic split) and `f5e89f4` (2026-06-04, 100KB/30KB size-cap split), compounded by `81bf52a` (2026-06-11, `spec/` → `spec_topics/` collapse)
**History:** `fecb504` created `spec_topics/overview.md` with the title `# Overview and Conceptual Model` as part of the original monolithic-`spec.md` split, at which point that page was the only "Overview"-titled page in `spec_topics/`. `f5e89f4` then enforced the 100KB hard / 30KB recommended size cap by splitting the root `spec.md` into three section pages, one of which — `spec/overview-and-orientation.md`, titled `# Overview and orientation` — duplicated the leading "Overview and …" stem; at that point the collision existed across two directories (`spec/` vs `spec_topics/`). `81bf52a` then collapsed `spec/` into `spec_topics/` to restore the single-delegation-target architecture, content-preservingly, which co-located the two "Overview and …" pages in the same directory. No single commit authored the collision; it is the cumulative effect of the size-cap split inheriting an "Overview and orientation" label without checking it against the pre-existing `overview.md` title, followed by the directory collapse that moved them next to each other.

## Solution Space

### Recommendation

Rename `docs/spec_topics/overview.md` → `docs/spec_topics/conceptual-model.md` and retitle its H1 to "Conceptual Model". Update the REQ-ID prefix-table row in `req-id-prefix-table-active-b.md` (`overview.md` → `conceptual-model.md`) and the Reading-order entry in `language-and-architecture.md` (both link text and target). Repoint the ≈6 inbound references, including the `M-minimal-slash-command` / `M-T-minimal-slash-command` plan Spec-field citations.

Before considering the sweep complete, grep the corpus and the plan for `overview.md` anchored so the pattern does not also match `overview-and-orientation.md`, and watch for inbound links that refer to "overview" generically. A stray un-swept link to `overview.md` would 404.

Leave `overview-and-orientation.md` and its GOV-30/GOV-31 lock-step links untouched. The separate concern that `overview-and-orientation.md` is labelled `spec.md` is tracked by the related GOV-31 finding.

## Consequence

**Severity:** advisory

The two files are functionally distinct but their names do not signal which is which, so a maintainer can navigate to or edit the wrong document, and cross-references are easy to misdirect. No runtime behaviour diverges and implementers can still produce a working system; the cost is navigational and editorial-correctness risk concentrated on the heavily-cited orientation page.
