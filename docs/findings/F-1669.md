## Finding

Two topic pages co-located under `docs/spec_topics/` share the same opening-word stem in their `# H1`: `overview.md` is titled **Overview and Conceptual Model** and `overview-and-orientation.md` is titled **Overview and orientation**. Both are reachable from the spec corpus root — `overview-and-orientation.md` directly from `spec.md`'s Contents list, `overview.md` indirectly via `language-and-architecture.md`'s *Reading order* bullet — so a reader scanning either link list, the rendered file tree, or a search-result snippet sees two near-identical "Overview and …" entries with nothing on the page itself indicating which is the orientation entry point and which is the conceptual deep-dive.

The collision propagates into inbound link text. `glossary.md` line 29 cites the page as `[Overview — Scope of a loom file](./overview.md)` (truncating "and Conceptual Model" to nothing), while line 51 cites it as `[Overview and Conceptual Model](./overview.md)`; `pi-integration-contract/subagent.md` line 61 uses a third capitalisation, `[Overview — Scope of a Loom File](../overview.md#scope-of-a-loom-file)`. The three citations refer to the same file but read like three different destinations, compounding the navigation ambiguity introduced by the title clash itself.

The defect is purely a naming/navigation one — the two pages carry distinct content and roles, and `overview.md`'s `# H1` already includes a non-overlapping discriminator ("Conceptual Model"). The repair is to drop the leading "Overview and" from the conceptual-model page so the corpus has exactly one "Overview …" page (the orientation entry point), and to align the two glossary citations and the subagent.md citation on the new title.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `fecb504` (2026-05-04, original spec-topic split) and `f5e89f4` (2026-06-04, 100KB/30KB size-cap split), compounded by `81bf52a` (2026-06-11, `spec/` → `spec_topics/` collapse)
**History:** `fecb504` created `spec_topics/overview.md` with the title `# Overview and Conceptual Model` as part of the original monolithic-`spec.md` split, at which point that page was the only "Overview"-titled page in `spec_topics/`. `f5e89f4` then enforced the 100KB hard / 30KB recommended size cap by splitting the root `spec.md` into three section pages, one of which — `spec/overview-and-orientation.md`, titled `# Overview and orientation` — duplicated the leading "Overview and …" stem; at that point the collision existed across two directories (`spec/` vs `spec_topics/`). `81bf52a` then collapsed `spec/` into `spec_topics/` to restore the single-delegation-target architecture, content-preservingly, which co-located the two "Overview and …" pages in the same directory. No single commit authored the collision; it is the cumulative effect of the size-cap split inheriting an "Overview and orientation" label without checking it against the pre-existing `overview.md` title, followed by the directory collapse that moved them next to each other.

## Solution Space

**Shape:** single

### Recommendation

Rename `docs/spec_topics/overview.md`'s `# H1` from `Overview and Conceptual Model` to `Conceptual Model`. The file path stays `overview.md` so no inbound link target needs repointing — only the link *text* changes at the inbound sites. Update, in the same commit:

- `language-and-architecture.md`'s `### Reading order` bullet: `[Overview and Conceptual Model](./overview.md)` → `[Conceptual Model](./overview.md)`.
- `glossary.md` line 51 (`prompt mode` / `subagent mode` entry): `[Overview and Conceptual Model](./overview.md)` → `[Conceptual Model](./overview.md)`.
- `glossary.md` line 29 (`loom (file unit)` entry): `[Overview — Scope of a loom file](./overview.md)` → `[Conceptual Model — Scope of a loom file](./overview.md)`, eliminating the bare-"Overview" abbreviation that collides with `overview-and-orientation.md`'s `## Overview` section.
- `pi-integration-contract/subagent.md` line 61: `[Overview — Scope of a Loom File](../overview.md#scope-of-a-loom-file)` → `[Conceptual Model — Scope of a loom file](../overview.md#scope-of-a-loom-file)` (also lower-casing "Loom File" to match the heading anchor and the glossary entry's casing).
- `plan_topics/M-minimal-slash-command.md` and `plan_topics/M-T-minimal-slash-command.md` **Spec.** fields: the bracketed link text `[`../spec_topics/overview.md`]` is a path render, not a title render, so the value need not change — but verify the leaves carry no prose referring to the page by its old title.

Watch for two edge cases: (a) the heading-derived anchor `#overview-and-conceptual-model` on the renamed file becomes `#conceptual-model`; grep the spec corpus for any inbound link with that fragment before committing (none exist at the time of writing, but the rename is mechanical only if that holds). (b) The sibling finding *"`## Overview` carries dense terminal-outcomes / file-extension-grammar paragraphs"* (recorded in the source review under `### Overview section — architecture-level rules placed in conceptual intro`) targets `overview-and-orientation.md`'s `## Overview`; this rename does not pre-empt it, but the title's new uniqueness makes the residual collision (a `## Overview` H2 inside `overview-and-orientation.md` vs. nothing similar in `Conceptual Model`) less load-bearing.

## Consequence

**Severity:** advisory

Two sibling topic pages with the same leading "Overview and …" stem confuse readers navigating the corpus and confuse contributors editing the inbound links (which already disagree across three sites — two in `glossary.md` and one in `subagent.md`). Nothing in the runtime contract depends on the page title, so the defect is purely documentational; an implementer can still build a correct system, but the corpus's navigation surface degrades and the inconsistent inbound link text invites further drift.
