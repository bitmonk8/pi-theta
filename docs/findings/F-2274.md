## Problem

The `caller` glossary entry (`#caller` in `docs/spec_topics/glossary.md`) carries a SHOULD-level usage mandate — authors and implementers SHOULD use the disambiguated terms `slash caller` and `invoke parent` whenever bare `caller` is ambiguous — but the glossary preamble declares every entry descriptive and non-normative, with the canonical `See:` page winning on any disagreement. A normative authoring/implementation mandate placed inside a self-declared non-normative page is structurally self-contradictory: the corpus's own precedence rule renders it overridable where it sits. The two `See:` targets that motivate the distinction (`slash-invocation.md`, which uses both terms at SLSH-3, and `invocation.md`) carry no such directive, so authors editing those pages in isolation never encounter the convention they are meant to follow.

## Issue introduction
**Verdict:** single-commit (introduced at coinage)
**Introducing commits:** d54b426 ("pi-loom spec: resolve \"Glossary cluster: six independent defects in spec_topics/glossary.md\"", 2026-05-07)
**History:** `git log -S "disambiguated terms" -- '*glossary.md'` resolves to exactly one commit, d54b426, which authored the entire `caller` glossary entry as a new bullet. The diff shows the SHOULD disambiguation sentence present in the entry's first version; no earlier revision of the entry exists. The placement defect therefore co-arrived with the coinage of `slash caller` / `invoke parent`: the disambiguated terms and their usage mandate were introduced directly into the descriptive glossary, and the canonical `See:` pages (`slash-invocation.md`, `invocation.md`) were never given the directive. It was not a later regression — the mandate has only ever lived in the glossary.

## Solution approach

Move the normative SHOULD usage directive out of the glossary into `slash-invocation.md` as its canonical home, near the SLSH-3 prose that already uses both `slash caller` and `invoke parent`. Add a forward-link from `invocation.md` to that canonical statement rather than restating it. Rewrite the `#caller` glossary entry's SHOULD sentence as a descriptive pointer to the canonical directive, leaving the entry purely descriptive as its preamble requires.
