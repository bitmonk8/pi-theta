## Finding

The cross-format collision-candidate set is pinned by one blockquote seam in `registration-steps.md`, headed **"loom 1.0 seam — Pi-owned subagents collision source set"** and dual-anchored `#loom-1-0-seam-pi-owned-subagents-collision-source-set` (canonical arm) plus `#v1-seam-pi-owned-subagents-collision-source-set` (back-compat alias). Three in-corpus cross-references point at this seam, and they are inconsistent in two ways:

- **They cite the alias arm, not the canonical arm.** All three references — `registration-steps.md` step 3, `host-prerequisites.md` prerequisite (c-ii), and `future-considerations/surface-extensions.md`'s *Carrier* line — link to `#v1-seam-pi-owned-subagents-collision-source-set`. GOV-25 (*canonical-arm citation*, `governance/release-version-naming.md`) requires cross-references to cite the canonical `loom-1-0`-segment arm and keep the `v1-` arm only as a resolution target for legacy inbound links.
- **Their link text diverges from the heading and from each other.** `registration-steps.md` and `host-prerequisites.md` render the link text "loom 1.0 cross-format collision source set"; `surface-extensions.md` renders "Pi Integration Contract — loom 1.0 seam: Pi-owned subagents collision source set". Neither matches the seam heading ("…Pi-owned subagents collision source set") consistently, so a reader cannot tell from the link text alone that all three resolve to the same set.

The fix is link-text normalisation plus repointing to the canonical anchor. The dual `v1-`/`loom-1-0-` anchor pair on the seam itself is **not** redundant and must be preserved: GOV-26 (*alias permanence*) makes the `v1-*` arm a permanent back-compat alias, retirable only under the narrow GOV-28 discharge. The original framing's "retire the redundant alias" direction conflicts with that convention.

The original heading also raised a "binder model" vs "binder LLM model" naming concern on the capability-probe drift note; that half is not a defect. The glossary defines **binder model** as "the LLM the binder calls", and the *Binder-model root-word convention* (`frontmatter/frontmatter-fields-a.md`) pins running prose to "binder model" — so `capability-probe.md`'s "binder model handle" already conforms, there is no colliding non-LLM "binder model" sense, and "binder LLM model" appears nowhere in the corpus. That sub-claim is dropped here.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `7a6cf96` (2026-05-07), `2e0dad2` (2026-05-29)
**History:** The corpus is git-tracked. The collision-source-set seam and its descriptive cross-reference link text ("…cross-format collision source set") were authored together in `7a6cf96` ("V1 seam-preservation MUSTs hidden inside the deferred-features narrative page"); the link text was a paraphrase rather than the heading verbatim from inception, so the link-text divergence is present-since-inception for that vector. The same-day prose rename `a6d82db` ("Prose V1 / V1.0 / V1.x rename to canonical spelling at non-closure callsites") relabelled heading prose `V1` → `loom 1.0` without aligning the cross-reference link text. The canonical-arm-citation gap was introduced by the dual-anchor migration `2e0dad2` ("HTML anchor dual-anchor authoring for v1- sites"), which added the canonical `loom-1-0-seam-*` arm alongside the legacy `v1-seam-*` arm per GOV-25–GOV-29 but did not repoint the pre-existing `#v1-seam-*` cross-references onto the new canonical arm. The defect is therefore the interaction of inception authoring with the later dual-anchor pass, not a single regressing commit.

## Solution Space

**Shape:** single

### Recommendation

Repoint all three cross-references (`registration-steps.md` step 3, `host-prerequisites.md` prerequisite (c-ii), `surface-extensions.md` *Carrier* line) from `#v1-seam-pi-owned-subagents-collision-source-set` to the canonical `#loom-1-0-seam-pi-owned-subagents-collision-source-set` arm, satisfying GOV-25, and normalise their link text to a single form aligned with the seam heading (e.g. "loom 1.0 cross-format collision source set" applied uniformly, or the heading's own "Pi-owned subagents collision source set"). Preserve both `<a id>` anchors on the seam itself — the `v1-*` arm is a permanent back-compat alias under GOV-26 and must not be removed. Edge cases: the canonical-arm slug is already unique across the corpus (GOV-29 satisfied), so no slug collision arises; and the seam heading text need not change — only the inbound link text and anchor target.

## Consequence

**Severity:** cosmetic

Both anchor arms resolve to the same target and the divergent link text is descriptive, so the normative collision-candidate set stays unambiguous and no implementer behaviour is affected. The concrete residue is a GOV-25 canonical-arm-citation non-conformance across three sites plus reader friction in correlating three differently-named links to one seam.
