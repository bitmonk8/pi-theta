## Finding

The *Code-keyed obligation areas (no numbered REQ-IDs)* table in `docs/plan_topics/coverage-matrix.md` interleaves ten retired rows — `cka-7`, `cka-12`, `cka-18`, `cka-21`, `cka-22`, `cka-23`, `cka-24`, `cka-25`, `cka-28`, `cka-29` — among the live obligation rows. That is ten of the forty-eight `cka-<n>` rows (~21% of the table). Each retired row carries a multi-sentence retirement rationale ("retired as a code-keyed-area row: the … obligations are now coined as numbered `…` REQ-IDs … and mapped to … no residual un-anchored / code-keyed obligation remains"), a "(GOV-22 drain)" process label, and a `*(numbered above)*` placeholder in its *Closing leaf(s)* cell.

These rows record no live obligation — every one of them has migrated its obligations to numbered `PREFIX-N` REQ-IDs that already appear in the *Numbered REQ-IDs* table above. Their only durable function is to keep the append-only `cka-<n>` token sequence intact so retired tokens are never reused (per the *Token* column's append-only rule in the preamble). That function needs only the token plus a one-line "migrated / no residual obligation" marker; the multi-sentence rationale and the "(GOV-22 drain)" process labels are authoring residue from the draining campaign that produced them.

An implementer or reviewer scanning the table for the set of active code-keyed obligations must read past, and filter out, ten dead rows that look identical in shape to live rows (same *Token* / *Spec area* / *Closing leaf(s)* columns) and are only distinguishable by parsing each row's prose for the word "retired". The live obligation set is obscured by a fifth of the table being inert.

## Consequence

**Severity:** cosmetic

A reader enumerating the live code-keyed obligation set must filter ten inert rows (~21% of the table) that are shape-identical to live rows and distinguishable only by parsing prose for "retired", plus the "(GOV-22 drain)" process labels add noise that carries no implementer-actionable content. The obligations themselves are all correctly stated, so no implementer diverges; the cost is a standing readability tax on every pass over the table.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/coverage-matrix.md`, lift the ten retired rows — `cka-7`, `cka-12`, `cka-18`, `cka-21`, `cka-22`, `cka-23`, `cka-24`, `cka-25`, `cka-28`, `cka-29` — out of the live *Code-keyed obligation areas (no numbered REQ-IDs)* table and record them in a dedicated retired-token subsection (e.g. a `## Retired code-keyed-area tokens` heading after the active table). For each retired token, keep the `cka-<n>` token, its former spec area, and a one-line marker naming where its obligations migrated (the numbered REQ-IDs and their closing leaf already mapped in the *Numbered REQ-IDs* table). Drop the multi-sentence retirement rationale, the `(GOV-22 drain)` process labels, and the `*(numbered above)*` closing-leaf placeholders — none of that prose is consumed by any gate.

Content constraints the fixer must honour:

- The retired subsection must still list every retired `cka-<n>` token so the preamble's append-only rule ("each new row takes the next free `cka-<n>` and existing tokens never change and are never reused") stays computable — the "next free token" determination depends on retired tokens remaining visible. Update the preamble's append-only sentence so it points the reader at the retired-token subsection for the full allocated-token set.
- The retired tokens must end up outside the two tables `H5d` / `H5f` / `H5a`'s facet arm machine-read (the *Numbered REQ-IDs* and *Code-keyed obligation areas (no numbered REQ-IDs)* tables): author the retired-token record as a list/section, not as a third closing-leaf table, so the retired entries no longer feed `H5d`'s closing-leaf tokeniser.
- `conventions.md`'s `// allow:` / `// allow-broad-catch:` predicate and the `H5c` arm resolve a cited token against "exactly one row's *Token* cell in this table"; once the retired rows leave the live table, a retired token no longer resolves as a valid exemption justification, which is correct because a retired area carries no active obligation. Confirm no `src/**` `// allow:` / `// allow-broad-catch:` comment cites a retired cka token before relocating.

`conventions.md` and the gate leaves are read-only for this fix; the change is confined to `coverage-matrix.md`'s table and preamble. The spec corpus is untouched.
