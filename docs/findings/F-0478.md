## Finding

H4b's first `Tests.` bullet asserts the response-programming-surface self-check proves the double drives **deterministically** — "replaying the same scripted assistant turns, `tool_use` results, binder responses/failures, `tool_loop.max_rounds` round-exhaustion, invoke-child completions, subagent-mode callee dispatch, and cancellation-injection points yields the same observable transcript **on every run**."

"On every run" is a universal quantifier over an unbounded set of executions. A finite replay self-check cannot establish it; the most a replay-compare gate observes is that the runs it actually performed produced identical transcripts. The asserted acceptance condition therefore claims more than the mechanism delivers.

This is a wording-precision defect, not a mechanism defect. The double is deterministic by construction and a replay-compare check is the standard, correct verification; the bullet simply states the resulting guarantee in unfalsifiable absolute terms.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H4b-response-programming-surface.md`, first `Tests.` bullet, drop the universal quantifier from the asserted guarantee. Replace the trailing clause

> …and cancellation-injection points yields the same observable transcript on every run.

with

> …and cancellation-injection points reproduces the same observable transcript when the same scripted inputs are replayed.

The enumerated injection-point list and the `deterministically` framing stay; only the unprovable "on every run" absolute is replaced with the replay-conditional claim the finite self-check can actually establish. No `Ships when` change is required — the existing "the determinism gate … passes" condition already reads against the replay-compare mechanism.

## Consequence

**Severity:** cosmetic

The overclaim does not change what the implementer builds — a replay-compare self-check is the obvious and intended mechanism regardless of the phrasing — so no two reasonable implementers diverge. The cost is purely that the leaf's acceptance prose asserts a guarantee (`on every run`) that the gate cannot falsify, which weakens the precision of the self-check contract.
