## Finding

H4a establishes a Class-2 cross-leaf seam whose canonical name is the **session-double fidelity contract** — it is bolded at the owning site (`H4a-factory-shell-and-harness.md` Tests bullet, "the **session-double fidelity contract** is asserted by a harness self-check") and used verbatim at every other consumption site: `H4b-response-programming-surface.md` Adds ("extends [`H4a`](./H4a-factory-shell-and-harness.md)'s session-double fidelity contract"), and the `SLSH-2` bullets in `V12a-slash-dispatch.md` / `V12a-T-slash-dispatch.md` ("fixed by the **session-double fidelity contract** ([`H4a`](./H4a-factory-shell-and-harness.md))").

`H7a-integration-acceptance.md`'s Adds is the one consumption site that names the seam by informal shorthand: "its fidelity is bounded by that double's contract ([`H4a`](./H4a-factory-shell-and-harness.md))". The phrase "that double's contract" is not the canonical term and is not linked back to the bolded definition by name, only by leaf ID.

The consequence is a cross-reference gap: a grep for the canonical seam name when auditing the contract's consumers misses H7a, even though H7a depends on the contract (`H4a` is in H7a's Deps and the Adds explicitly bounds H7a's fidelity by it). The canonical name should be stable at every consumption site so the seam's consumer set is mechanically discoverable.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** e7e51cc (2026-06-10) "pi-loom plan: resolve \"Plan has no terminal end-to-end integration-acceptance leaf\""
**History:** `git log --diff-filter=A` shows `docs/plan_topics/H7a-integration-acceptance.md` was first added in e7e51cc, and `git log -S"double's contract"` on that file returns the same commit as the sole match — the informal shorthand was present in the leaf's first commit and never replaced by the canonical term. The canonical `session-double fidelity contract` term already existed in `H4a-factory-shell-and-harness.md` at that time, so this is a consumption-site naming drift introduced when the H7a leaf was authored, not a later regression.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H7a-integration-acceptance.md`'s Adds paragraph, replace the shorthand "its fidelity is bounded by that double's contract ([`H4a`](./H4a-factory-shell-and-harness.md))" so the seam is named by its canonical term, e.g. "its fidelity is bounded by the **session-double fidelity contract** ([`H4a`](./H4a-factory-shell-and-harness.md))". Use the canonical term verbatim as it appears at the owning H4a site; the `[`H4a`]` link is retained for leaf-ID traceability. No other H7a text changes are required.

## Consequence

**Severity:** advisory

A reviewer auditing the consumers of the `session-double fidelity contract` seam by searching its canonical name does not find H7a, despite H7a consuming the contract. No implementer divergence results; the gap is in traceability and seam-consumer discoverability, not in what gets built.
