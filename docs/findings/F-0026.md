## Problem

The leaf-pair files `docs/plan_topics/V9b-registration-drain-state.md` and its paired tests file `V9b-T-registration-drain-state.md` carry the slug token `drain-state`, but the leaf's heading is *"`V9b` — Registration steps and reload-wiring seams"* and its **Adds** field delegates the `LoomRegistry` drain-state contract to V9m (`V9m-drain-state-contract.md`). The slug therefore names a subject the leaf no longer owns. The result: two files appear to claim the drain-state contract, while V9b's actual reload-wiring / registration scope is invisible to path-based navigation and grep across `plan_topics/`.

## Issue introduction

**Verdict:** introduced-by-later-edit
**Introducing commit:** `cb8737a` — "pi-loom plan: resolve \"Systemic leaf over-bundling across the plan corpus\"" (2026-06-12)
**History:** The file was created in `c6a664e` ("pi-loom plan: build/update plan for spec.md + review") with heading *"`V9b` — Registration steps and drain-state contract"* and an Adds that genuinely owned the `LoomRegistry` drain-state contract — at that point the `drain-state` slug matched the leaf's scope and was correct. Commit `cb8737a` performed an over-bundling split: it created the new `V9m-drain-state-contract.md` leaf, moved the drain-state contract to it, rewrote V9b's Adds to *"The `LoomRegistry` drain-state contract is owned by V9m,"* and retitled V9b's heading to *"Registration steps and reload-wiring seams."* That commit renamed the heading and reassigned ownership but did not rename the file (or update its inbound link paths), leaving the slug stranded on a subject the leaf no longer owns. `git log --follow` shows no rename of the file across its history.

## Solution approach

Rename the V9b leaf pair so the slug reflects the retitled scope and drops `drain-state` — e.g. `V9b-registration-reload-wiring.md` and `V9b-T-registration-reload-wiring.md`. Then retarget every path-bearing markdown link to the leaf to the new path: the V9 slice-list entry in `docs/plan.md` (line 115), and the **Adds** links in `V6a-frontmatter-contract.md`, `V9m-drain-state-contract.md`, `V10c-settings-merge.md`, and `V11a-binder-model-resolution.md`.

## Solution constraints

- The leaf ID stays `V9b`; cross-references that cite the bare ID `V9b` (Deps and prose in V4e, V4e-T, V10c, V10c-T) are out of scope — only path-bearing links move.
