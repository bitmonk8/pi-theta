## Finding

`host-prerequisites.md` opens by declaring itself the page that owns the **Pi-supplied** contract — the four numbered items are Pi SDK pin, binder model, binder credentials, and Pi-supplied `AbortSignal`, all things the host runtime must provide — and the paragraph immediately above the offending block explicitly relegates *loom's own* npm-package dependencies (`semver`, `chokidar`, `yaml`) to `implementation-notes.md#loom-package-implementation-dependencies-loom-1-0` on the grounds that they are "implementation-side choices rather than part of the Pi-supplied contract this page owns."

The very next paragraph, anchored `tsc-toolchain-floor`, then pins a `>=5.4.0` floor on the TypeScript compiler loom uses to build its own extension package. TypeScript is not a Pi host capability: Pi does not consume `tsc`, the loom *consumer* does not need TypeScript installed, and the H1a plan leaf declares `typescript` in loom's own `devDependencies` (the `tsc` provider behind `npm run typecheck`) — exactly the same `devDependencies`/build-side footing the relocated `semver`/`chokidar`/`yaml` block occupies in `implementation-notes.md`. The block's own opening clause ("Unlike the version-unpinned `semver` / `chokidar` choices above") signals it knows it is sitting beside its natural neighbours yet has been parked on the wrong page.

The floor is genuinely load-bearing — `V18c`'s `loom/typecheck/session-shutdown-reason-snapshot` brand-string gate depends on `tsc` quoting the brand string verbatim, and a future TypeScript release that reshaped literal-type diagnostics could silently disable that gate — so the block must keep its anchor, its load-bearing framing, and its editorial-review obligation under the version-bump procedure. The defect is the placement, not the content.

## Solution Space

**Shape:** single

### Recommendation

Relocate the entire `<a id="tsc-toolchain-floor"></a>**Build-toolchain floor (TypeScript brand-string surfacing).**` block out of `host-prerequisites.md` and into `implementation-notes.md`, placed adjacent to (preferably immediately after) the existing **Loom-package implementation dependencies (loom 1.0)** bullet so the four loom-side build/runtime dependencies (`semver`, `chokidar`, `yaml`, `tsc`) sit together.

Mechanical rules:

- Preserve the anchor literal `tsc-toolchain-floor` on the relocated block so every cross-reference can be re-pointed by file-path change alone, with no anchor-rename sweep. (The H1a leaf, `version-bump-intro.md` step 1, and `version-bump-triggers.md` step 5 all cite the anchor by name — anchor-stability keeps the diff small.)
- Re-point the two PIC cross-references (`version-bump-intro.md` step 1's two `[Host prerequisites — Build-toolchain floor](./host-prerequisites.md#tsc-toolchain-floor)` links, `version-bump-triggers.md` step 5's `[Host prerequisites — Build-toolchain floor]` link) to `../implementation-notes.md#tsc-toolchain-floor`, and update the H1a leaf's two citations (`**Adds.**` and the `**Tests.**` `typescript`-devDependency bullet) from `../spec_topics/pi-integration-contract/host-prerequisites.md#tsc-toolchain-floor` to `../spec_topics/implementation-notes.md#tsc-toolchain-floor` in the same commit.
- Keep the block's body verbatim — the load-bearing framing ("Unlike the version-unpinned `semver` / `chokidar` choices above" naturally improves on the new page, since those choices now genuinely *are* above), the `>=5.4.0` literal, the brand-string rationale, and the editorial-review-on-bump obligation all remain. The only prose edit is to drop the now-redundant "above" anchor and replace it with a same-page reference to the `loom-package-implementation-dependencies-loom-1-0` entry.
- Leave the `**Loom-package implementation dependencies.**` summary paragraph on `host-prerequisites.md` in place — it already correctly delegates `semver`/`chokidar`/`yaml` to implementation-notes and can absorb `tsc` into the same `e.g.` list with a single-word addition, so a reader looking for the floor on the host page still finds the pointer.

Implementer watch-outs: the anchor `tsc-toolchain-floor` must not be silently renamed during the move (an off-by-one rename would break the four citations above without surfacing in any build-time gate); and the four citation-bearing files must all land in the same commit so a partial landing does not leave a dangling `host-prerequisites.md#tsc-toolchain-floor` link in the corpus.

## Consequence

**Severity:** cosmetic

The block is correctly framed and load-bearing wherever it lives; no implementer reading either page would build the wrong thing. The cost is that the page named "Host prerequisites" continues to mis-advertise a loom-side build-toolchain pin as a Pi-supplied prerequisite, which dilutes the page's own scope rule and weakens the precedent the immediately preceding `semver`/`chokidar` delegation set.
