## Finding

The third paragraph of `coverage-matrix.md` §"Code-keyed obligation areas (no numbered REQ-IDs)" restates the closing-gate behaviour for `<new>` placeholder rows — the H5e un-anchored-MUST gate failing on any MUST mapped only to a `<new>` row, the live-corpus footing, and the seeded-mid-authoring `<new>` row staying green until the H6a footing flip. That gate-reddening and seeded-footing behaviour is already stated authoritatively in `conventions.md` §REQ-ID discipline, whose passage on `<new>` placeholder rows ("…does **not** satisfy the 'enumerated with a named closing leaf' condition… the gate therefore reddens for any un-anchored MUST mapped only to a `<new>` row at the loom 1.0 release gate, while a seeded mid-authoring `<new>` placeholder stays green until the `H6a` activation flips this footing to hard-fail") owns the rule. The coverage-matrix.md paragraph even closes by citing `conventions.md` §REQ-ID discipline as the authority for the same clause it restates.

The recogniser *definition* — that the placeholder is exactly the literal bare unbackticked `<new>` token, that it tokenises to the empty set under the same backtick-delimited tokeniser the *Closing-leaf column authoring* rule defines, and that any other closing-leaf token not resolving to an existing plan leaf-ID reddens the gate as a defect — is correctly homed in coverage-matrix.md: `conventions.md` §REQ-ID discipline explicitly defers to "the placeholder convention defined in that file's *Code-keyed obligation areas* section, which pins the recogniser." That portion is not duplicative and must stay.

The defect is the two authoritative statements of the same gate-reddening / footing rule, one in each file, with no single owner. A future edit to the rule in its `conventions.md` home need not be mirrored here, and the restatement can drift into contradiction with the authoritative text.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/coverage-matrix.md` §"Code-keyed obligation areas (no numbered REQ-IDs)", third paragraph, strike the sentences that restate the closing-gate behaviour already owned by `conventions.md` §REQ-ID discipline and retain the recogniser definition that conventions.md defers to this section:

- **Strike** the gate-failure clause "…the [`H5e`](./H5e-un-anchored-must-gate.md) closing-gate arm fails on any such MUST absent from this table with a closing leaf, on the live-corpus footing that binds at the loom 1.0 release gate once the terminal [`H6a`](./H6a-live-corpus-activation.md) activation leaf lands."
- **Strike** the reddening restatement "A `<new>` placeholder row names no real closing leaf, so it does **not** satisfy that closing-leaf condition: on the live-corpus footing the gate reddens for any un-anchored MUST whose only entry here is a `<new>` row, exactly as if the MUST were absent from this table — the placeholder marks the obligation as still uncovered, not as closed."
- **Strike** the seeded-footing parenthetical "(A seeded mid-authoring `<new>` row stays green until the [`H6a`](./H6a-live-corpus-activation.md) footing flip, per [`conventions.md`](./conventions.md) *REQ-ID discipline*.)"
- **Replace** the struck gate-behaviour text with a single cross-reference of the form: the reddening, live-corpus-footing, and seeded-mid-authoring-green behaviour of a `<new>` placeholder row is owned by `conventions.md` §REQ-ID discipline.
- **Retain** the recogniser-definition sentences: "The closing-leaf placeholder is **exactly the literal bare (unbackticked) `<new>` token** — the same form [`H5d`](./H5d-transitive-completeness-gate.md)'s backtick-delimited tokeniser reads as the empty set… so both recognisers select the same cell." and "The recogniser resolves each closing-leaf token against the **real plan leaf-ID universe**: the literal `<new>` is the uncovered-obligation placeholder marker… reddens the gate as a defect rather than being read as a benign placeholder or as naming a real closing leaf." `conventions.md` §REQ-ID discipline explicitly defers the recogniser definition to this section, so this text is the authority and must stay.
- The opening enumeration sentence introducing the third closing-gate surface and the row-shape convention ("Each such MUST is one rule-driven row here with a named closing leaf (a `<new>` placeholder where the obligation is currently uncovered…), not a one-off patch") establish this table's local row convention and may stay; only the gate-failure clause appended to it is struck.

`conventions.md` is read-only for this fix — the authoritative gate-semantics statement there is not edited. The fix is confined to `coverage-matrix.md`.
