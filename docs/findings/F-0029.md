## Finding

V18d and V18d-T refer to the H4a in-process test double using three different non-canonical names. V18d's Tests bullet writes "`H4a` session double"; V18d-T's Tests bullet writes both "`H4a` harness double" and "`H4a` double". The canonical term for this artefact is **Pi session double**, established by and anchored to H4a (the leaf that defines the harness, the double, and the session-double fidelity contract).

None of the three variants matches the canonical term, and no inline equivalence links them to it. A reader or maintainer who searches the corpus for "Pi session double" — for example to enumerate every consumer of the H4a session-double fidelity contract — misses both V18d leaves, even though V18d's acceptance gate is explicitly bounded by that double's fidelity to the bumped pin.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 8af3204 — pi-loom plan: resolve "V18c bundles static gates with runtime-evidence acceptance gate" (2026-06-11, Thomas Andersen)
**History:** Both V18d leaves were created in 8af3204 when the runtime-evidence acceptance gate was split out of V18c. `git log -S` on each of "session double", "harness double", and the "`H4a` double" form returns only 8af3204 for these two files, so all three non-canonical variants were present at the leaves' first commit and have not been edited since; the divergence from the canonical "Pi session double" has existed since inception.

## Solution Space

**Shape:** single

### Recommendation

Use the canonical "Pi session double" at every occurrence in the leaf pair:

- In `docs/plan_topics/V18d-version-bump-acceptance.md`'s Tests bullet, replace "`H4a` session double" with "Pi session double" (append "(`H4a`)" on first occurrence if leaf-ID traceability is wanted).
- In `docs/plan_topics/V18d-T-version-bump-acceptance.md`'s Tests bullet, replace both "`H4a` harness double" and "`H4a` double" with "Pi session double".

The occurrences index (`index/terms.md`) is a regenerated projection, not a hand-maintained plan file; it picks up the V18d/V18d-T occurrences once the leaf text uses the canonical term, so no manual edit to the transient index artefact is needed.

## Consequence

**Severity:** advisory

A cross-reference search for the canonical "Pi session double" silently omits V18d and V18d-T, so the leaf pair is invisible to anyone auditing consumers of the H4a double or its fidelity contract. Implementers can still build the leaves; the cost is weakened traceability and an invitation to further naming drift.
