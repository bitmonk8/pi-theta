## Finding

The manual real-host smoke gate (`real-host-smoke-gate.md`) reuses `H7a`'s committed multi-feature fixture `.loom` and scores a live run against five pass criteria. Those criteria presuppose the fixture exercises a fixed set of observable behaviours: (b) a binder pass whose output validates against the three-arm `ok | needs_info | ambiguous` envelope schema; (c) a spawned subagent-mode `AgentSession` with committed turns and an injected mid-stream cancel point; (d-i) streamed-token order relative to `ctx.waitForIdle()` resolution; (d-ii) single-turn prompt-mode append; (d-iii) a `pi.on` cancel-forward subscription; and (d-iv) cancellation propagation.

`H7a`'s leaf imposes none of these as a fixture-content requirement. Its `Adds`/`Tests`/`Ships when` describe the integrated pipeline as "typed query â†’ tool loop â†’ code-tool invoke â†’ schema lowering/validation â†’ binder â†’ cancellation" and assert turn/diagnostic goldens plus a co-occurring ceiling breach â€” but never state the fixture must spawn a subagent, inject a cancel point, or surface the four fidelity axes. The subagent presupposition is the sharpest gap: `H7a` carries no reference to subagents at all, and `V9i` (subagent isolation) is absent from `H7a`'s `Deps`, so the fixture has no obligation â€” and currently no dependency basis â€” to spawn the subagent-mode `AgentSession` that criterion (c) drives a mid-stream cancellation inside.

If `H7a`'s committed fixture omits any of these behaviours, the corresponding smoke criteria (b)â€“(d) have nothing to observe and pass vacuously. The real-host smoke gate is the only mechanism that detects a double-vs-real divergence on the production seam; criteria that cannot run against the committed fixture silently void that detection.

## Consequence

**Severity:** correctness

An implementer authoring `H7a`'s fixture has no instruction to include a subagent spawn, an injected cancel point, or the four fidelity-axis behaviours, so two reasonable implementers produce fixtures that exercise different subsets. A fixture lacking those behaviours leaves smoke criteria (b)â€“(d) with nothing to score, so the live-host divergence gate â€” loom's only real-host fidelity check â€” passes vacuously on the axes it is supposed to defend.
