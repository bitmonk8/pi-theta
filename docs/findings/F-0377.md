## Finding

`V9b`'s **Adds** legitimately establishes that `V9b` is the single declaration site of the test-only `ReloadFailureInjector` interface for the watcher-time reload failure-injection seam, and that `V9b` wires the registry-swap and `.loom`/`.warp` re-parse arms (with `V10d` contributing the settings-re-merge arm). That is all producer-side ownership content and is correctly placed.

The same Adds paragraph then appends a consumer-side clause: "`V4e` imports the interface from `V9b` (via `Deps`) to exercise `ERR-7`." This describes what a downstream leaf does with the seam, not what `V9b` provides. The consumption relationship is already documented at the consumer end — `V4e`'s **Deps** field states "`V9b` declares the `ReloadFailureInjector` interface (`injectReloadFailure`) that `V4e` imports", and `V4e`'s `ERR-7` Tests bullet repeats it ("the single interface declared by `V9b`, from which `V4e` imports it"). The `V9b` Adds clause is therefore a duplicate of information that lives in two places on `V4e`.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 44a96f5 — pi-loom plan: resolve "V9b filename slug contradicts retitled scope" (2026-06-16, Thomas Andersen)
**History:** The parent revision of `V9b`'s Adds contained no reference to `V4e`, imports, or the consumption relationship. Commit 44a96f5 rewrote the Adds paragraph and introduced the clause "`V4e` imports the interface from `V9b` (via `Deps`) to exercise `ERR-7`" (confirmed by pickaxe `git log -S "imports the interface from \`V9b\`"` returning only this commit, and by inspecting the parent blob). The defect entered with that single later edit.

## Solution Space

**Shape:** single

### Recommendation

Strike the consumer-side clause "`; \`V4e\` imports the interface from \`V9b\` (via \`Deps\`) to exercise \`ERR-7\``" from `V9b`'s **Adds** field, leaving the preceding producer-side sentence ("A caller supplies a synthetic registry-swap or re-parse failure to `ReloadFailureInjector.injectReloadFailure` and the seam routes it onto the `loom-system-note` surfacing path without standing up a live watcher") intact. `V4e`'s Deps field and `ERR-7` Tests bullet already record that `V4e` imports the interface, so no edit to `V4e` is needed; the consumption relationship remains documented at the consumer end.

## Consequence

**Severity:** cosmetic

No implementer is misled: ownership of the seam is unambiguous from `V9b`'s producer-side statement, and `V4e` already documents that it imports the interface. The only cost of leaving it is a maintenance-drift hazard — the consumer relationship is now stated in three places, so a future change to how `V4e` consumes the seam must be remembered in `V9b`'s Adds as well.
