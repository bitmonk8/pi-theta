## Problem

`H4a`'s third Tests bullet requires the session-double fidelity self-check to assert all four fidelity-contract axes — (i) streamed tokens before `ctx.waitForIdle()` resolves, (ii) one streamed response appended as a single prompt-mode turn, (iii) the `pi.on` cancel-forward subscription, and (iv) cancellation propagation — and that self-check is `H4a`'s own Ships-when gate. The Adds, however, scopes the **minimal response-emission capability** as "sufficient only to drive self-check items (i) and (ii)" and defers the full scripted-injection surface to `H4b`, while saying nothing about what mechanism fires axes (iii)/(iv). A fresh implementer cannot tell whether to build extra emission machinery, dispatch through the cancel-forward subscription, or wrongly defer (iii)/(iv) to `H4b` — which would leave `H4a`'s four-axis Ships-when self-check unsatisfiable when `H4a` lands.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** `32ea081` — "pi-loom spec-plan: fix F-0477 — define H4a minimal emission capability + H4a/H4b boundary" (2026-06-20)
**History:** The plan corpus is git-tracked. At `32ea081`'s parent the `H4a` Tests bullet already enumerated all four self-check axes (i)–(iv), including (iii) cancel-forward and (iv) cancellation propagation, while the Adds simply listed the four fidelity-contract behaviours and assigned the response-programming surface to `H4b` with no emission-scope claim. Commit `32ea081` introduced the **minimal response-emission capability** wording into the Adds — "sufficient only to drive self-check items (i) and (ii)" — without reconciling it against the unchanged four-axis Tests self-check. That narrowing created the present contradiction: the Adds now reads as bounding the double's deliverable at (i)/(ii) while the Tests (and Ships-when) still require (iii)/(iv) to pass at this leaf.

## Solution approach

Reword the `H4a` Adds so the "(i) and (ii)" bound is explicit that it scopes the **response-emission** capability alone and no longer reads as excluding the cancel axes. Make explicit that self-check axes (iii) and (iv) are exercised by dispatching a constructed cancel event through the factory-time `pi.on` cancel-forward subscription `H4a` already establishes as part of the fidelity contract, so they require no scripted assistant emission and do not depend on `H4b`. State that axes (iii)/(iv) are therefore self-checkable from `H4a`'s own deliverables, keeping the four-axis Ships-when self-check satisfiable when `H4a` lands.

## Solution constraints

- Out of scope: the scripted-injection response-programming surface (categories (a)–(e)) owned by `H4b`; do not add `H4b` to `H4a`'s `Deps`.
- The cancel event exercising (iii)/(iv) must drive the factory-time cancel-forward subscription path (the contract behaviour), not a test-only shortcut that bypasses the subscription.
