## Finding

The spec enumerates four off-surface `execute()` dispositions in `tool-calls.md` as a normative bulleted list, and the fourth bullet's category name is **"Post-cancel resolution"** (`*Post-cancel resolution.*` in `docs/spec_topics/tool-calls.md`, the *Outcome enumeration (normative)* block). `V14c` is the leaf that closes that enumeration, but it labels the same outcome **"post-cancel late settlement"** in its `Adds`, `Tests`, and `Ships when` fields, and the paired `V14c-T` repeats the divergent label in its `Tests` field. The plan's phrasing borrows from the spec bullet's *body* ("any later settlement of the underlying Promise … is discarded") rather than the bullet's *category name*.

The other three off-surface outcomes in `V14c` track the spec's category names closely ("pre-eval setup throw", "non-conforming return shape", "non-settling promise"), so the fourth is the lone label that does not round-trip back to the spec category a reader would grep for. The behavioural content of the bullet is correct — the discard-per-CNCL-1/CNCL-2/CNCL-3 mechanics match the spec — so this is purely a label-consistency divergence, not a semantic error.

## Issue introduction

**Verdict:** introduced-by-specific-commit (token introduced, then propagated)

When the off-surface routing was split out of `V14a` into the dedicated `V14c` / `V14c-T` leaves in `5a1915d` ("pi-loom plan: resolve 'V14a bundles eight independently-testable concerns'", 2026-06-17), the divergent label was carried verbatim into the new files; `V14a` no longer carries the term. `git log -G "post-cancel late settlement"` over `docs/plan_topics/` confirms these two commits are the only touch points. The divergence has existed since the wording was first authored — at no point did the plan use the spec's "Post-cancel resolution" category name for this outcome.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V14c-tool-calls-off-surface-routing.md`, replace the literal text `a post-cancel late settlement` with `post-cancel resolution` in the `Adds` bullet, and replace `a post-cancel late settlement → discarded per CNCL-1/CNCL-2/CNCL-3 (no \`internal-error\`)` with `post-cancel resolution → discarded per CNCL-1/CNCL-2/CNCL-3 (no \`internal-error\`)` in both the `Tests` bullet and the `Ships when` bullet, aligning the fourth outcome's label with the spec's `*Post-cancel resolution.*` bullet category name in `docs/spec_topics/tool-calls.md`. Apply the identical substitution to the matching `Tests` bullet in `docs/plan_topics/V14c-T-tool-calls-off-surface-routing.md` so the paired test leaf stays in lockstep. The discard mechanics text (`discarded per CNCL-1/CNCL-2/CNCL-3 (no internal-error)`) is correct and stays unchanged; only the outcome's category label changes.

## Consequence

**Severity:** cosmetic

A reader cross-referencing the leaf against the spec's normative outcome enumeration cannot match the fourth outcome by its category name, weakening the spec↔plan traceability the other three labels provide. No implementer divergence results — the discard mechanics are fully specified inline — so nothing ships wrong; only the naming round-trip is lost.
