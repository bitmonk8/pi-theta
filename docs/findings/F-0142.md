## Problem

`PIC-10` (the `Checkpoint` seam) is the only REQ-ID the `V8a` / `V8a-T` leaf pair implements, but its normative definition lives in `docs/spec_topics/pi-integration-contract/host-interfaces-services.md` (anchor `#pic-10`). Both leaves' **Spec.** fields cite `host-interfaces-core.md` and `implementation-notes.md`, neither of which contains PIC-10. An implementer following the read-discipline (read only the leaf and its listed Spec topics) cannot locate the single obligation the leaf exists to close.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e — pi-loom plan: build/update plan for spec.md + review (2026-06-10, Thomas Andersen)
**History:** Both `V8a-checkpoint-validator-seams.md` and `V8a-T-checkpoint-validator-seams.md` were created in commit c6a664e, and the **Spec.** field cited `host-interfaces-core.md` + `implementation-notes.md` from that first commit. The two later commits touching these files (d36eb03/5dd842a Deps fixes, cb8737a over-bundling resolution) never revised the Spec citation. The defect is original to the files.

## Solution approach

In the **Spec.** field of both `V8a-checkpoint-validator-seams.md` and `V8a-T-checkpoint-validator-seams.md`, replace the `host-interfaces-core.md` citation (link text and href) with `host-interfaces-services.md`, so the field points at the page defining PIC-10. Retain the `implementation-notes.md` citation. This is a citation correction; the defect is the wording itself.

## Solution constraints

- The spec is read-only for this fix — PIC-10 already exists at `host-interfaces-services.md#pic-10`; only the plan citation is wrong.
