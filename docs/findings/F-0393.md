## Finding

The first `SLSH-2` Tests bullet in `V12a-slash-dispatch.md` ends with a
sentence that is not a test assertion: "The real-Pi behaviour this models —
Pi's TUI streaming tokens into the transcript before the terminal `agent_end`
settles `ctx.waitForIdle()` — is an unpinned host presupposition audited by
editorial review per [bump-checklist item (ak)](../spec_topics/pi-integration-contract/version-bump-step2.md#bump-checklist-user-visible-streaming-ordering)."
This is a risk/assumption note about a host invariant and its editorial-audit
routing, lodged inside the description of a concrete test case.

A Tests bullet states what the test asserts; the executable content here is the
ordering assertion (streamed tokens observable before `ctx.waitForIdle()`
resolves, driven through the session double whose ordering is fixed by the H4a
fidelity contract). The trailing sentence instead documents *why* the assertion
rests on an unpinned presupposition and where that presupposition is audited —
material that belongs with the leaf's dependency/spec cross-references, not in
the test-case prose. The cross-referenced bump-checklist anchor
(`version-bump-step2.md#bump-checklist-user-visible-streaming-ordering`) and the
H4a fidelity contract both exist, so the note is accurate; only its placement is
wrong.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** `85fc906` (2026-06-10) "pi-loom plan: resolve \"streamed-token-before-waitForIdle() ordering has no version-bump checklist item\""
**History:** `git log -S` on the distinctive tokens ("unpinned host presupposition", "audited by editorial review", "The real-Pi behaviour this models", and the `(ak)` cross-reference) all converge on `85fc906` as the sole commit that introduced this prose into `V12a-slash-dispatch.md`. That commit resolved a prior review finding by adding bump-checklist item (ak) and wiring a cross-reference to it; the cross-reference was authored directly into the SLSH-2 streaming Tests bullet rather than into a dependency/spec field, which is the misplacement reported here. The defect therefore entered as a placement side-effect of an otherwise-substantive fix, not at plan inception.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V12a-slash-dispatch.md`, strike the trailing sentence of
the first `SLSH-2` Tests bullet (the one beginning "The real-Pi behaviour this
models —" and ending at the `bump-checklist item (ak)` link). The executable
assertion preceding it — streamed tokens observable before
`ctx.waitForIdle()` resolves, with the session-double ordering fixed by the H4a
fidelity contract — stays in the bullet unchanged.

Relocate the struck cross-reference to the `Spec.` field as a secondary
citation, since `version-bump-step2.md#bump-checklist-user-visible-streaming-ordering`
is where the unpinned-presupposition and editorial-audit routing already live.
A parenthetical on the `H4a` entry in the `Deps.` field is an acceptable
alternative home. Either way the note must remain discoverable from V12a; the
edit removes it from the test-case description, it does not delete the
cross-reference.

## Consequence

**Severity:** cosmetic

The note's content is correct and load-bearing context survives elsewhere (the
bump-checklist item and the H4a contract). Leaving it in the Tests bullet only
makes the test case harder to read and blurs the line between an assertion and
an editorial-audit rationale; no implementer is misled into a divergent
implementation.
