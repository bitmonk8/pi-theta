## Problem

Coverage-matrix row `cka-33` (swallowing-handler attachment / total three-side-channel suppression) is a multi-leaf row whose *Closing leaf(s)* cell lists four per-site routing facets — `V13f`, `V14f`, `V15h`, `V9o` — none carrying a `co-witness` annotation, so each is a distinct facet keyed by its own leaf-ID. The `H5f` per-facet citing-test gate reddens on any facet leaf whose test corpus has no facet-naming citing test (one citing both the row's `cka-33` token and that facet's closing-leaf-ID inline). Each facet's swallowing-handler Tests bullet — in both the implementation leaf and its `-T` tests leaf — describes the assertion only by spec section name and cites neither `cka-33` nor its own leaf-ID, so an implementer authoring the test emits neither token, leaving each facet with no facet-naming citing test. `H5f` then reddens on each at the `H6a` release gate even though `npm test` stays green per leaf.

**Verdict:** multi-commit-interaction
**Introducing commits:** 26a8794 — pi-loom plan: resolve "split coverage-matrix rows can ship a facet untested" (2026-06-18, Thomas Andersen); 7259777 — pi-loom plan: resolve "code-keyed obligation areas lack a canonical citable token" (2026-06-18, Thomas Andersen)
**History:** The four facet leaves were authored before the per-facet citing-test requirement existed — `V13f` at eaa5000 (2026-06-16), `V14f` at 5a1915d (2026-06-17), `V15h` at 5f4d271 (2026-06-17), `V9o` at 6d7661d (2026-06-17) — and none has ever carried a `cka-33` citation (`git log -S 'cka-33'` over the four files is empty). Commit 7259777 (2026-06-18) made `cka-33` a citable code-keyed-area token; commit 26a8794 (2026-06-18) added the `H5f` per-facet citing-test gate and the multi-leaf `cka-33` row that lists these four leaves as facets, but did not back-propagate the now-required `cka-33` + facet-leaf-ID inline citation into the facet leaves' Tests bullets. The defect is the interaction between the pre-existing facet leaves and the later gate-introducing commit.

## Solution approach

In each facet's swallowing-handler Tests bullet, add the row token `cka-33` and the facet's own closing-leaf-ID inline, in the established inline code-keyed-token citation style (`V14b` cites `(code-keyed-area token `cka-13`)`). Apply to both the implementation leaf and its paired `-T` tests leaf so the two stay consistent; the leaf-ID to cite is the facet key from the `cka-33` cell (the implementation-leaf ID), so each `-T` bullet cites the implementation-leaf ID, not the `-T` ID: `V13f`/`V13f-T` → cite `cka-33` and `V13f`; `V14f`/`V14f-T` → `cka-33` and `V14f`; `V15h`/`V15h-T` → `cka-33` and `V15h`; `V9o`/`V9o-T` → `cka-33` and `V9o`. Add the tokens to the existing assertion text without altering what each bullet asserts.
