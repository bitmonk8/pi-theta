## Finding

`docs/spec_topics/binder/defaulting-system-note-echo.md` §Defaulting installs the post-default-merge validation hook with the sentence "the `SchemaValidator.validate()` call that AJV-validates the merged `args` object against the lowered `params` schema after the runtime has filled the defaults above." The shard names a method `SchemaValidator.validate()` and never defines it, never cross-links to where it is defined, and never states the input/output contract the binder integration relies on.

The normative validator surface in `docs/spec_topics/pi-integration-contract/host-interfaces-services.md` [PIC-11](../spec_topics/pi-integration-contract/host-interfaces-services.md#pic-11) actually exposes two interfaces — `SchemaValidator` carries `compile(schema): CompiledValidator` and `invalidate(schemaSlug)`, while the `validate(value)` method lives on the returned `CompiledValidator`. There is no `SchemaValidator.validate()` method anywhere in the corpus; the in-shard token is a shorthand for the compile-then-validate pipeline. A first-time reader landing on the binder shard has no link to follow, no method definition to inspect, and no statement of the behavioural pre/postconditions the binder depends on at this site (most importantly: the validator MUST NOT convert values and MUST NOT fill defaults, so the merged `args` object the runtime built by fill-if-absent survives the call unchanged).

The binder shard's silence is consequential because the *whole point* of "post-default-merge" is that the defaults have already been filled by the runtime and the validator runs against that merged object as-is. If a reader assumed `.validate()` could coerce or re-default — a reasonable assumption for an AJV-backed validator, which can do exactly that under different option flags — the fill-if-absent contract immediately above would silently break. PIC-11 forbids this, but the binder shard does not say so and does not link out.

## Solution Space

**Shape:** single

### Recommendation

In `docs/spec_topics/binder/defaulting-system-note-echo.md` §Defaulting, rewrite the *"post-default-merge AJV validation"* sentence to:

1. **Cross-link the hook to PIC-11.** Replace the bare token `SchemaValidator.validate()` with a hyperlink to `../pi-integration-contract/host-interfaces-services.md#pic-11`, and clarify that the call is the `compile(loweredParamsSchema).validate(mergedArgs)` pipeline through the injected [`SchemaValidator`](../pi-integration-contract/host-interfaces-services.md#schemavalidator-interface) seam (`SchemaValidator.validate()` is the shorthand this section uses for that pipeline at this site). The compile result is cached per the PIC-11 slug-cache rule, so this is conceptually a single call from the binder's vantage.
2. **State the input/output contract the binder integration relies on.** Add one sentence: the validator inspects the merged `args` object — it MUST NOT mutate, coerce, or re-default it (per [PIC-11](../pi-integration-contract/host-interfaces-services.md#pic-11)), so the fill-if-absent merge from the paragraph above survives the call byte-identical, and the return is the PIC-11 issue-list shape (`{ ok: true } | { ok: false; errors: readonly ValidationError[] }`) that the [AJV-on-`args` failure class](./determinism-cancellation-failure.md#failure-class-taxonomy) consumes.

Implementer-relevant edge case: the cross-link must point at PIC-11's behavioural contract list (the `#pic-11` anchor), not at the non-normative `interface SchemaValidator { … }` declaration block — per [GOV-18 arm (a)](../governance/corpus-direction-and-scope.md#gov-18-arm-a) the member signatures of that DI seam are illustrative, and the normative obligations (one-pass multi-error, no-convert/no-defaults, deterministic) are what binds the binder integration.

## Consequence

**Severity:** advisory

A diligent implementer reaches PIC-11 via `implementation-notes.md` §Runtime and recovers the validate-only / no-coercion / no-defaulting contract, so loom 1.0 can still be built correctly. A less diligent one wires the post-default-merge step through an AJV instance with `useDefaults: true` or `coerceTypes: true`, silently overwrites binder-supplied values for defaulted fields (defeating fill-if-absent), and produces a conforming-looking implementation that diverges on the echo's `(default)` tagging and on the bytes the depth-walk fast-fail short-circuits.
