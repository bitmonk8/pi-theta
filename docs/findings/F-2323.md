## Finding

`V18b`'s **Adds** owns a land-green `src/` sweep: in the same commit that introduces the inventory-closure audit, every existing `src/` Pi-side reference the audit would surface must be deleted, promoted to an `SDK_SURFACE_INVENTORY` entry, marked with an `// allow-pi-surface:` exemption, or **rewritten into a recognised shape** so the audit lands green on `main` rather than as a land-red gate. The "rewrite into a recognised shape" branch is a behaviour-bearing edit across the whole production tree.

The audit that gates the sweep enforces *surface-set closure* — every reference resolves to an inventory entry, the typebox allow-lists, or an exemption — i.e. it checks the **shape** of each reference, not its **semantics**. An incomplete sweep reds immediately (a still-unrecognised reference fails the closure check). But a sweep that rewrites a reference into a recognised shape while silently changing what that reference does lands **green**: the audit cannot tell a faithful shape-preserving rewrite from one that alters behaviour. Nothing in `V18b` requires the sweep to be behaviour-preserving, and nothing surfaces the set of references the sweep touches for review before the rewrite commits.

The result is a single large, non-idempotent rewrite commit whose only automated gate (the closure audit) is blind to behavioural regressions introduced by the rewrite itself.

## Solution Space

**Shape:** single

### Recommendation

Strengthen `V18b`'s **Adds** (and reflect the exit condition in **Ships when**) so the land-green `src/` sweep is required to be behaviour-preserving and reviewable, without weakening the spec's same-commit closure mandate:

- Require that the pre-existing feature-test suite (the non-audit tests already run by `npm test`) stays green across the sweep commit, so a rewrite that alters a reference's behaviour is caught rather than masked by the shape-only closure audit. Make this an explicit exit condition of the sweep in **Ships when** (the suite is green on the sweep commit, in addition to the closure audit running green on `main`).
- Require the sweep to surface the full set of `src/` references it deletes / promotes / exempts / rewrites for human review before the rewrite is committed, so the "rewritten into a recognised shape" branch is auditable for behaviour change rather than landing unseen inside a large mechanical diff.

Author this as content on `V18b` only — name the requirement, not its presentation. The spec (`inventory-audit-intro.md` §Inventory-closure audit) is read-only for this fix: it already mandates the same-commit, land-green sweep, so do not restate or relax that mandate; the behaviour-preservation requirement is additive plan discipline layered on top of it and adds no new spec rule (the green feature-test suite is already required by `npm test`).

## Consequence

**Severity:** advisory

A rewrite-into-recognised-shape edit that changes a reference's behaviour passes the closure audit and lands green, shipping a silent behavioural regression in the audit-introducing commit; because the sweep is a single non-idempotent pass across all of `src/`, the regression has no narrowing automated detector and no pre-commit review surface. The leaf remains pickable and an implementer can still produce a working gate, so this is a risk-mitigation gap rather than a blocker.
