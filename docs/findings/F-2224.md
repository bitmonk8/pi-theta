## Problem

The non-gated *Drain-state tag write (state-independent)* paragraph in `session-only-degraded-state.md` asserts both the `LoomRegistry.markRuntimeDegraded` setter contract and a firing site/timing ("the handler writes … immediately after sub-step 4 returns … **unconditionally**"), but the actual firing predicate, `capturedEventReason ∉ {"quit","reload"}`, lives in the **gated** *Predicate split* clause. An implementer building only the explicitly-buildable non-gated carve-out therefore sees a write that fires after sub-step 4 on every reason with no gate, while the gated clause says it must fire only when `capturedEventReason ∉ {"quit","reload"}`. The word "unconditionally" conflates the state-independent-overwrite sense with the unconditional-firing sense, so the non-gated portion cannot in fact be implemented in isolation as the gate banner promises.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** `8bcd52a` (pi-loom spec-plan: fix F-1163 — session-only-degraded-state gate scopes by clause not page)
**History:** The corpus is git-tracked. `session-only-degraded-state.md` was created at `f5e89f4` carrying both the *Drain-state tag write … unconditionally* paragraph and the `capturedEventReason`-keyed predicate content, but with a **file-level** gate ("The obligations on this page are blocked pending resolution of the open contradiction…") — both clauses were gated together, so no cross-gate dependency existed. `8bcd52a` re-scoped the gate "by clause, not by page," carving out the *Drain-state tag write (state-independent)* paragraph as non-gated while leaving the *Predicate split* (which holds the `∉ {"quit","reload"}` firing predicate the write depends on) gated. That clause-level partition is the edit that placed a non-gated clause's firing condition behind the gate boundary.

## Solution approach

Narrow the non-gated *Drain-state tag write (state-independent)* paragraph to state only the `LoomRegistry.markRuntimeDegraded` setter contract — a state-independent overwrite of the drain-state tag to `"degraded-needs-reload"` without reading the prior value — and rewrite its opening so "unconditionally" reads unambiguously as the state-independent-overwrite sense. Remove the firing-site/timing assertion from this paragraph, replacing it with a reference that the call site and its firing predicate (`capturedEventReason ∉ {"quit","reload"}`) are defined by the gated *Predicate split* clause. Add a cross-reference in the gate banner clarifying that the non-gated carve-out is the setter contract, not the call-site firing logic.

## Solution constraints

- The firing predicate (`capturedEventReason ∉ {"quit","reload"}`) and the branch-behaviour three-arm analysis MUST stay in the gated *Predicate split* clause; do not move gated content to the non-gated side of the banner.
- Out of scope: the cross-file gate-scope coherence owned by F-2225 and the gated/non-gated visual-separation owned by F-2226.
