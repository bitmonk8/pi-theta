## Finding

In the renderer-registration signature block of `extension-bootstrap-and-per-loom.md`, `Theme` is a load-bearing `MessageRenderer` parameter type, but the section preamble pins only `Component` to `@earendil-works/pi-tui`'s `dist/tui.d.ts`; `Theme` carries no declaration pointer despite being a sibling pi-tui type in the same code block.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** 78a2f94 ‚Äî resolve "Misplaced sections within the pi-integration-contract pages" (2026-06-04); f5e89f4 ‚Äî docs: enforce 100KB hard / 30KB recommended size cap on spec set (2026-06-04); 44782e9 ‚Äî resolve "TransportError catch-all / pi-ai provider-error surface" (2026-06-06); 15c0f17 ‚Äî fix F-1284, add uniform Pi-owned-snapshot markers to the four host-interfaces-core.md member-pinning surfaces (2026-06-24); 31debf1 ‚Äî fix F-1578, re-anchor execute() outcome routing on AgentToolResult (2026-06-27)
**History:** When the renderer-registration section was placed (78a2f94) it pinned `Component` to pi-tui's `dist/tui.d.ts` but never pinned the sibling `Theme`. The bald `CompactOptions` equality entered host-interfaces-core.md during the size-cap split (f5e89f4) and `ProviderResponse` entered the provider-error pages soon after (44782e9), both without a snapshot marker. The marker convention was later standardized across four host-interfaces-core.md surfaces (15c0f17, F-1284) but not extended to these inline equalities, and a still-later edit re-authored the `AgentToolResult` bald equality (31debf1, F-1578) after the convention existed without applying it ‚Äî so the defect is the convention's uneven roll-out across these sites rather than any single commit.

## Solution Space

**Shape:** single

### Recommendation

Extend the preamble's pointer clause to name `Theme` as also declared at `@earendil-works/pi-tui`'s `dist/tui.d.ts`, mirroring the existing `Component` pointer (e.g. "the `Component` and `Theme` interfaces live at `@earendil-works/pi-tui`'s `dist/tui.d.ts`").

## Consequence

**Severity:** advisory

A reader cannot resolve where `Theme` is declared (a navigability gap that the sibling `Component` does not have), and the three bald equalities invite an implementer to read them as loom owning/redefining the external type's exact member set rather than as re-validated snapshots. Implementers can still build a working system ‚Äî the surrounding prose already says loom redefines none of these ‚Äî so the gap is one of clarity and convention consistency, not divergence.
