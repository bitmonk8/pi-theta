## Finding

`CIO-5` (`ceilings-3-and-4.md#cio-5`) states that ceiling #3 (binder per-class retry budget) never interleaves with ceiling **#1, #2, or #4** — it is a binder-bypass / load-time-only property spanning three distinct runtime-class ceilings. `V16a` explicitly defers the cross-site, never-interleaves witness for the whole of CIO-5 to `H7a` ("CIO-5's 'ceiling #3 never interleaves with #1/#2/#4' … is a binder-bypass / load-time-only property whose end-to-end witness is `H7a`'s co-occurring-breach integration assertion"), and `coverage-matrix.md` records `H7a` as CIO-5's "cross-site never-interleaves integration witness".

`H7a`'s CIO-5 test bullet, however, drives exactly one pairing: a binder-retry-class breach (ceiling #3, `V11f`) co-occurring with the runtime-class `tool_loop.max_rounds` round boundary (ceiling **#2**, `V13c`). The #3-vs-#1 (`invoke`-entry depth) and #3-vs-#4 (AJV-boundary) pairings are never exercised by the fixture, yet the bullet closes with the unqualified claim that this single run is "closing the live-site integration gap that `V16a`'s synthesised-candidate suite leaves open." The `Ships when` block carries the same single-pairing assertion.

The claim and the coverage-matrix attribution both assert complete closure of CIO-5's cross-site property while only one of its three sibling pairings is witnessed end-to-end. Whatever the right resolution, the prose currently overstates what the gate observes.

## Solution Space

**Shape:** multiple

### Option A — Scope the claim to the representative pairing

**Approach:** Treat CIO-5's #3-vs-#1 and #3-vs-#4 non-interleave as a structural binder-bypass consequence (`invoke(...)` does not invoke the binder; binder is load-time only) for which no positive co-fire event can be constructed, and downgrade the prose from "closes the whole gap" to "witnesses the #3/#2 co-occurrence as the representative runtime-class pairing."

**Plan edits:**
- In `H7a` CIO-5 Tests bullet and `Ships when`, replace "closing the live-site integration gap that `V16a`'s synthesised-candidate suite leaves open" with text scoping the assertion to the #3/#2 (binder-retry vs runtime-class round-boundary) co-occurrence as the representative load-time-vs-runtime pairing, and naming where the #3-vs-#1 / #3-vs-#4 non-interleave is otherwise witnessed (binder-bypass leaves) or routing it to the release-time residue inspection in `conventions.md`.
- In `coverage-matrix.md` CIO-5 row and closure-rationale paragraph, narrow the `H7a` annotation from "cross-site never-interleaves integration witness" to the #3-vs-runtime-class pairing it actually exercises.
- In `V16a`, align the "Not witnessable at the stateless seam" and CIO-5 Tests prose so it no longer attributes the whole cross-site property to `H7a`.

**Spec edits:** None.

**Pros:** Cheapest; matches the binder-bypass reality (a #3-vs-#1/#4 positive co-fire is structurally unconstructable because invoke bypasses the binder); leaves the DAG unchanged.

**Cons:** Leaves the #3-vs-#1 / #3-vs-#4 non-interleave as a structural argument rather than an executed assertion; relies on the residue inspection or binder-bypass leaves to carry it.

**Risks:** If binder-bypass is not independently gated, the structural argument is the only backstop for those facets.

### Option B — Add the missing co-occurrence vectors

**Approach:** Extend `H7a` so the fixture (or a sibling fixture) also drives a #3-vs-#1 (invoke-entry) and/or #3-vs-#4 (AJV-boundary) scenario, so all three CIO-5 pairings are witnessed end-to-end and the "closes the whole gap" claim becomes true.

**Plan edits:**
- Add Tests bullet(s) to `H7a` exercising the invoke-path (#1/#4) sites against a binder-retry context, asserting the binder ceiling is not consulted on the invoke/AJV path (the binder-bypass observable).
- Confirm `H7a` Deps cover the invoke-depth (`V15b`) and AJV-boundary (`V5e`) enforcement sites that the new vectors consult.

**Spec edits:** None.

**Pros:** Makes the coverage-matrix attribution accurate; gives each CIO-5 facet an executed end-to-end witness.

**Cons:** A #3-vs-#1/#4 co-fire cannot occur (invoke bypasses the binder, binder is load-time only), so the added test can only witness the negative "binder not consulted on this path" — a different shape from the #3/#2 positive arbitration; enlarges `H7a`'s Deps and per-pass critique surface.

**Risks:** Forces `H7a` to model an event that is structurally absent, which can mislead the implementer into constructing an artificial co-fire.

### Recommendation

Take Option A. The #3-vs-#1 and #3-vs-#4 non-interleave is a binder-bypass structural fact (`invoke(...)` never invokes the binder, which runs only at slash-load), so no positive co-fire event exists for `H7a` to drive; the honest fix is to scope `H7a`'s CIO-5 claim to the #3-vs-runtime-class (#2) representative pairing, narrow the `coverage-matrix.md` CIO-5 annotation to match, and route the un-driven pairings either to the binder-bypass leaves that already cover them or to the release-time residue inspection. Edge case: keep `V16a`'s deferral prose in lockstep — if it still says `H7a` witnesses the whole CIO-5 cross-site property, the overclaim re-enters through `V16a`.

## Consequence

**Severity:** correctness

`coverage-matrix.md` and `V16a` both attribute the full CIO-5 cross-site never-interleaves witness to `H7a`, but `H7a` only exercises the #3-vs-#2 pairing; the #3-vs-#1 (invoke-bypass) and #3-vs-#4 (AJV-boundary) non-interleave facets ship with no end-to-end witness while the coverage matrix reads green for them. An auditor trusting the matrix concludes CIO-5 is fully closed end-to-end, so a binder-bypass regression on the invoke or AJV path would pass the gate undetected.
