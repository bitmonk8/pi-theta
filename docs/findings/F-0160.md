## Finding

The `V9i` / `V9i-T` swallowing-handler **Tests** bullet ends with the rationale clause "… so a build that bypasses the substrate reddens this leaf's tests." The token **"the substrate"** is never defined inside the leaf. The spec settles the referent — `cancellation.md` ("Race semantics — swallowing-handler attachment on every abandonable Promise") names the `Checkpoint` seam as "the deterministic-test substrate for landing the late settlement … without relying on JS microtask scheduling" — but the leaf bullet uses the bare noun with no link or qualifier. A reader who has not chased that spec sentence can plausibly map "the substrate" either to the `Checkpoint` seam (`V8a`) deterministic-test injection (correct) or to the JS Promise / microtask substrate (the thing the spec explicitly contrasts the seam against), and only the former matches the test the leaf actually prescribes.

The clause is a verbatim templated fragment: the identical "… so a build that bypasses the substrate reddens this leaf's tests" sentence appears in eight leaf files — `V9i`, `V9i-T`, `V13c`, `V13c-T`, `V14a`, `V14a-T`, `V15a`, `V15a-T` — each describing the swallowing-handler suppression at its own abandonable-Promise site. Naming the referent in `V9i` alone while leaving the same bare token in the seven siblings would leave the template inconsistent, so the clarification should land uniformly across every leaf carrying the clause.

The bare token was carried verbatim into the template rather than named when broadcast.

## Solution Space

**Shape:** single

### Recommendation

In the swallowing-handler **Tests** bullet of `V9i` and `V9i-T`, replace the bare "the substrate" with its explicit referent so the clause reads "… so a build that bypasses the `Checkpoint` seam (`V8a`) deterministic-test substrate reddens this leaf's tests." (`cancellation.md`'s "Race semantics" paragraph is the authority that "the deterministic-test substrate" denotes the `Checkpoint` seam; it is read-only for this fix — no spec edit.)

Apply the identical substitution to the same templated clause in the six sibling leaves that carry it verbatim — `V13c`, `V13c-T`, `V14a`, `V14a-T`, `V15a`, `V15a-T` — so the template stays uniform across all eight abandonable-Promise sites. The substitution is purely internal to the plan leaf files; no leaf ID, Deps, or Ships-when field changes.

## Consequence

**Severity:** advisory

The bare token is recoverable: the same sentence already cites "the `Checkpoint` seam (`V8a`)" and the spec defines "the deterministic-test substrate" as that seam, so a careful implementer resolves it correctly. The risk is a reader taking "the substrate" as the JS Promise/microtask layer and writing a fixture that relies on real microtask timing instead of the deterministic `Checkpoint` (`V8a`) injection — a flaky test rather than a wrong contract.
