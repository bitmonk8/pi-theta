## Finding

`V9l` (session-only degraded-state branch) is **blocked** on host-prerequisites clause (a), whose resolution is owned by the `V18c` version-bump editorial-review checklist item (a). The leaf is designed with two mutually-exclusive resolution arms. The *reachable* arm is fully asserted: `V9l`'s **Ships when** names a concrete `npm test` run that exercises the `markRuntimeDegraded` drain-state tag write, the `"degraded-needs-reload"` transition, the `loom/host/session-shutdown-runtime-degraded` emission, the recovery-path prohibition, and the predicate split, with the failing-test partner authored in `V9l-T`.

The *unreachable* arm carries no equivalent verification. `V9l`'s **Ships when** defers it entirely to the `plan.md` §Blocked obligations fallback ("If clause (a) instead resolves *branch unreachable*, the pre-designed fallback … governs what ships"), and that fallback only states the *outcome* — `V9l` "collapses to full teardown + `/reload` with no degraded tag write, and its degraded-branch obligations are reworked spec-side (`session-only-degraded-state.md` and `SM-4`/`SM-5`/`SM-6`/`SM-3b`) before the leaf ships." It names neither a check that proves the branch is genuinely unreachable, nor a recorded descope of the `SM-4`/`SM-5`/`SM-6`/`SM-3b` obligations. Because the unreachable arm authors no degraded-path tests (those live in `V9l-T`, which is reworked or stood down in this arm), nothing observable backs the claim that the collapse is correct.

The coverage matrix maps `cka-27` (the full session-only-degraded-state obligation set) to `V9l` as its closing leaf. If clause (a) resolves *branch unreachable* and `V9l` ships as a bare teardown collapse with the degraded obligations silently reworked away, the `cka-27 → V9l` mapping passes vacuously: the matrix still claims `V9l` closes those obligations while the leaf no longer asserts them. One of the two resolution paths can therefore leave acceptance criteria unasserted with no gate firing.

## Consequence

**Severity:** advisory

If clause (a) resolves *branch unreachable*, the implementer can ship `V9l` as a teardown-and-`/reload` collapse with no recorded verification that the branch is actually unreachable and no recorded descope of the `SM-4`/`SM-5`/`SM-6`/`SM-3b` obligations — leaving the `cka-27 → V9l` coverage mapping to pass vacuously over criteria nothing asserts. The reachable arm is unaffected, and the leaf is unpickable until `V18c` resolves regardless, so no current build breaks; the gap is in the contingent fallback's guidance.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan.md` §Blocked obligations, extend the V9l "Pre-designed fallback" clause so the branch-unreachable arm carries an explicit verification obligation, parallel to how the reachable arm's **Ships when** names a concrete `npm test`. State that, when clause (a) resolves *branch unreachable*, shipping `V9l` requires a recorded verification that closes the obligations one of two ways:

- a recorded demonstration that the degraded branch is genuinely unreachable as written (the artefact that proves the collapse is correct), or
- a recorded formal descope of the `SM-4`/`SM-5`/`SM-6`/`SM-3b` degraded-branch obligations spec-side, with the `cka-27` row in `docs/plan_topics/coverage-matrix.md` retargeted or struck so it no longer maps the descoped obligations to a `V9l` that does not assert them.

The verification names what backs the collapse (an `npm test` assertion or the editorial-review record that performs the descope), so neither resolution arm can ship with the degraded-branch obligations silently unasserted. The spec stays read-only for this plan-side fix — the spec-side rework is performed by the clause-(a) resolution owner (`V18c`), not by this edit.
