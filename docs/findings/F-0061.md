## Problem

The Tests-task bullet of the §Per-phase TDD ritual in `docs/plan_topics/conventions.md` instructs authors to write "One assertion per rule where practical," but never says what makes the one-assertion-per-rule default impractical. The "where practical" qualifier carries the entire exception with no stated boundary. Because the ritual is inherited by every MVP and vertical `-T` tests task, authors will resolve "where practical" differently — some splitting each REQ-ID into its own assertion, others folding several rules into one combined assertion. An over-combined assertion can red out on its first failing rule and mask the others, eroding the per-rule red isolation that the surrounding "fail red for the intended reason" requirement depends on.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 288f191 ("Add implementation plan with horizontal/MVP/vertical-slice phases", 2026-05-04)
**History:** The sentence "One assertion per rule where practical." entered the corpus verbatim in the plan's first commit (288f191, then in `plan.md`'s "Tests first" ritual step). It moved unchanged when the corpus was split into per-topic / per-phase files (fecb504) and again when the docs were relocated under `docs/` (31ff060), carrying the bare "where practical" qualifier the whole way. No later commit attempted to define the exception, so the gap is original to the ritual rather than introduced by a subsequent edit.

## Solution approach

In the §Per-phase TDD ritual Tests-task bullet, rewrite the "One assertion per rule where practical" sentence to state the one-assertion-per-rule default and the single condition under which rules may share a test. Keep the per-rule red-isolation property that the surrounding "fail red for the intended reason" definition relies on, and preserve the inline `PREFIX-N` citation discipline so a combined test's failure still identifies the offending rule.

## Solution constraints

- Convention-text clarification only — do not introduce a new mechanical gate or enforcement check.
- Out of scope: the §REQ-ID discipline citation rules, which are owned elsewhere in `conventions.md`; this fix relies on them but does not edit them.
