## Finding

The spec's canonical noun for the closed enumeration of newline-continuation triggers is the **set**: `grammar.md` §Newline continuation states "The trigger set is closed", and `lexical.md` §Statement terminators states "The continuation-trigger set is closed and is normatively tabulated in [Grammar Appendix — Newline continuation]". The closed concept is the *set*; the Markdown table at `grammar.md#newline-continuation` is merely how that set is presented.

`V1a`'s **Adds** already follows the spec — it names "the closed statement-termination / newline-continuation trigger **set**". But the **Tests** sections of both `V1a` and `V1a-T` name the same concept "closed trigger **table**" in two places each: the `single-line-if`/`stray-backslash` violation bullet ("termination/continuation violations fire per the closed trigger table") and the positive-path continuation bullet ("(closed trigger table, positive path)").

The result is an internal inconsistency within each leaf (Adds says "set", Tests say "table") and a divergence from the spec's normative term. "Table" also reads as a reference to the Markdown table rendering rather than to the closed trigger concept, weakening any grep-based cross-reference between the leaf acceptance criteria and the spec term.

## Consequence

**Severity:** cosmetic

Pure wording: an implementer reading either leaf still arrives at the same closed trigger enumeration, so no behaviour diverges. The cost is term drift — the Tests contradict their own Adds and the spec's normative noun, and the "table" phrasing defeats a grep that keys on the spec term "trigger set".

## Solution Space

**Shape:** single

### Recommendation

In both `docs/plan_topics/V1a-lexer-core.md` and `docs/plan_topics/V1a-T-lexer-core.md`, replace every occurrence of "closed trigger table" in the Tests bullets with "closed trigger set" (matching V1a's Adds and the spec). The two occurrences per file are:

- The `loom/parse/single-line-if`, `loom/parse/stray-backslash` bullet: "termination/continuation violations fire per the closed trigger set".
- The positive-path continuation bullet's parenthetical: "(closed trigger set, positive path)".

"closed continuation-trigger set" is equally acceptable and matches `lexical.md`'s phrasing; whichever is chosen, use the same phrasing in all four spots so the Adds, both Tests files, and the spec agree. Leave the `grammar.md#newline-continuation` link target unchanged — only the link text / surrounding prose changes.
