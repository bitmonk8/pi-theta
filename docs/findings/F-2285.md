## Problem

The plan coins "always-log channel" (and "always-log `loom-system-note` channel"), fusing two concepts the spec keeps distinct: the **always-log set** (a closed enumeration of failure kinds whose membership guarantees exactly-once emission) and the **`loom-system-note` channel** (the delivery surface). The coined term implies channel membership equals always-log-set membership, which is false — the spec routes many non-always-log codes through the same `loom-system-note` channel while excluding them from the always-log set. QRY-20 states the discarded-`Err` case as "preserved as a runtime event on the always-log set ... delivered through the same `loom-system-note` channel"; the plan collapses both halves into one term. The phrase is absent from the spec, so a term-grep audit will not surface it.

## Issue introduction

**Verdict:** present-since-inception (per-site)
**Introducing commits:**
- `4a04570` (2026-06-17) — `V13g` Adds "operator-facing always-log channel", authored when `V13a` was split into five leaves (the resolve commit for "V13a bundles five independent query surfaces into one leaf").
- `4e395ef` (2026-06-18) — `V13g` Tests + Ships-when and `V13g-T` Tests "always-log `loom-system-note` channel", authored when the discard-observability payload/channel assertions were added (the resolve commit for "V13g discard-observability tests assert only that the event fires, not payload or channel").
- `1db2787` (2026-06-17) — `coverage-matrix.md` `cka-33` row "always-log channel", authored when the swallowing-handler row was added (the resolve commit for "cancellation.md swallowing-handler attachment MUST has no coverage-matrix row").

**History:** Each site introduced the conflated phrasing at the moment its text was first authored; `git log -G "always-log channel"` and `--follow` show no earlier spec-correct revision that a later commit regressed. There is no single introducing commit — the term entered independently at the leaf split, the payload-assertion enrichment, and the cka-33 row addition. The durable fix is therefore a terminology pass across all three sites, not a revert.

## Solution approach

Run a terminology pass replacing the coined "always-log channel" / "always-log `loom-system-note` channel" at each of its three sites with the spec term matching each sentence's intent: **`loom-system-note` channel** for the delivery surface and **always-log set** for set-membership / the always-fire guarantee. In the `coverage-matrix.md` `cka-33` row, rewrite the suppression clause so it names the `loom-system-note` channel as the delivery surface. In `V13g` (Adds, Tests, Ships-when), keep the two concepts distinct so the discarded-`Err` event is an always-log-set runtime event delivered on the `loom-system-note` channel. Apply the identical edit to the `V13g-T` Tests bullet so it stays verbatim-aligned with `V13g`.
