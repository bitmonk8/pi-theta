## Problem

`H2a`'s **Adds.** prose describes the blocking-runtime `no-restricted-syntax` check as flagging "`fs.readFileSync`, `execSync`, and the `*Sync` family" over `src/**` ŌĆö an "A, B, and C" enumeration in which A (`fs.readFileSync`) is itself a member of C (the `*Sync` family). The triple therefore admits two incompatible recognisers: match every callee whose property name ends in `Sync` (named methods being examples), or match a finite literal set plus an undefined "`*Sync` family" residual that has no machine-recognisable definition. The same ambiguous triple is the authoritative source of the phrasing in `conventions.md` ┬¦Sequential by default ("Enforcement posture"), the ┬¦Per-phase TDD ritual self-review check, and ┬¦loom 1.0 release-time residue inspection item 3, so two reasonable implementers would build gates that accept and reject different `src/**` call sites (e.g. `writeFileSync`, `readdirSync`).
