## Problem

PIC-43 and PIC-42 in `subagent.md` defer to prompt-mode driver material using a bare positional "above" in three places: "the same two short-circuits the prompt-mode driver pins above", "the same assistant-text rule the prompt-mode driver pins above", and "the same behavioural contract as prompt-mode typed queries above". None of that material lives on `subagent.md` — the two short-circuits and the trailing-assistant-text rule are pinned on `conversation-drive.md`. A reader of `subagent.md` who follows "above" finds nothing, so the referenced rules are unresolvable from where they are cited. The adjacent transport-mapping clause already demonstrates the correct pattern by cross-linking to `#prompt-mode-error-detection`.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `f5e89f4` ("docs: enforce 100KB hard / 30KB recommended size cap on spec set"), `222aacd` ("pi-loom spec-plan: fix F-1227 — subagent untyped-query transport-failure short-circuit")
**History:** Before the size-cap split, prompt-mode drive and subagent mode both lived on the single page `spec_topics/pi-integration-contract.md` (confirmed at `f5e89f4~1`, which contained both `assistant-text rule the prompt-mode driver pins above` and `same behavioural contract as prompt-mode typed queries above` on the same page as the prompt-mode driver). At that point "above" was a valid same-page back-reference. Commit `f5e89f4` (2026-06-04) split that page into the index + child layout, moving subagent mode to `pi-integration-contract/subagent.md` and the prompt-mode driver to `pi-integration-contract/conversation-drive.md`; its map-driven rewrite repointed link anchors but not these prose "above" references (the commit message notes residual dangling refs), so the two pre-existing references silently became cross-document. Commit `222aacd` (2026-06-23, F-1227) then added the third reference — "the same two short-circuits the prompt-mode driver pins above" — written fresh after the split yet echoing the now-broken same-page "above" idiom (`git log -S` localises this phrase to `222aacd`, and the other two to the `f5e89f4` split). The defect is therefore the interaction of the page split with the "above" prose convention, compounded by one later edit replicating it.

## Solution approach

Rewrite the three bare "above" references in PIC-43/PIC-42 as explicit anchored cross-references into `conversation-drive.md`, reusing the anchors the adjacent transport-mapping clause already cites. Point the two short-circuits at [`#prompt-mode-error-detection`](./conversation-drive.md#prompt-mode-error-detection), the assistant-text rule at [`#untyped-query-ok-extraction`](./conversation-drive.md#untyped-query-ok-extraction), and PIC-42's typed-query reference at the prompt-mode typed-query material on `conversation-drive.md`. Only the locator changes from positional to anchored.

## Solution constraints

- `conversation-drive.md` remains the single owner of the prompt-mode rules; do not relocate or inline them into `subagent.md`.
- Change only the locators; do not alter the "same … rule on both sides" invariant semantics the references assert.
