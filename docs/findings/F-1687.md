## Finding

PIC-43 (`agent_end` query-result extraction) in `subagent.md` defines the subagent query-result path by repeatedly deferring to the prompt-mode driver with a bare positional "above":

- "the runtime applies **the same two short-circuits the prompt-mode driver pins above**, in the same order" — the cancellation short-circuit and the `stopReason: "error"` transport short-circuit;
- "assembled … by **the same assistant-text rule the prompt-mode driver pins above**" — the trailing-assistant-text concatenation;
- PIC-42 then adds "Typed queries against the spawned session honour **the same behavioural contract as prompt-mode typed queries above**".

None of that prompt-mode driver material lives on `subagent.md`. The cancellation short-circuit and the `stopReason: "error"` transport short-circuit are pinned in the *Error detection* section of `conversation-drive.md` (anchor `prompt-mode-error-detection`, with PIC-51), and the trailing-assistant-text rule is PIC-53 *Untyped-query `Ok(string)` trailing-turn extraction* (anchor `untyped-query-ok-extraction`) on the same page. A reader of `subagent.md` who follows "above" finds nothing — these are cross-document references mislabelled as same-page ones, so the two short-circuits and the assistant-text rule are referenced but never resolvable from where they are cited.

The page already demonstrates the correct pattern one clause earlier: the transport-mapping shape is cited as "the same shape as the prompt-mode mapping at [prompt-mode error detection](./conversation-drive.md#prompt-mode-error-detection)". The three "above" references should be repointed the same way; the destination anchors already exist.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `f5e89f4` ("docs: enforce 100KB hard / 30KB recommended size cap on spec set"), `222aacd` ("pi-loom spec-plan: fix F-1227 — subagent untyped-query transport-failure short-circuit")
**History:** Before the size-cap split, prompt-mode drive and subagent mode both lived on the single page `spec_topics/pi-integration-contract.md` (confirmed at `f5e89f4~1`, which contained both `assistant-text rule the prompt-mode driver pins above` and `same behavioural contract as prompt-mode typed queries above` on the same page as the prompt-mode driver). At that point "above" was a valid same-page back-reference. Commit `f5e89f4` (2026-06-04) split that page into the index + child layout, moving subagent mode to `pi-integration-contract/subagent.md` and the prompt-mode driver to `pi-integration-contract/conversation-drive.md`; its map-driven rewrite repointed link anchors but not these prose "above" references (the commit message notes residual dangling refs), so the two pre-existing references silently became cross-document. Commit `222aacd` (2026-06-23, F-1227) then added the third reference — "the same two short-circuits the prompt-mode driver pins above" — written fresh after the split yet echoing the now-broken same-page "above" idiom (`git log -S` localises this phrase to `222aacd`, and the other two to the `f5e89f4` split). The defect is therefore the interaction of the page split with the "above" prose convention, compounded by one later edit replicating it.

## Solution Space

**Shape:** single

### Recommendation

Replace the three bare "above" cross-document references in `subagent.md` with explicit anchored cross-references to `conversation-drive.md`, reusing the anchors the adjacent transport-mapping clause already cites:

- "the same two short-circuits the prompt-mode driver pins above, in the same order" → name the two short-circuits and cite them at [prompt-mode error detection](./conversation-drive.md#prompt-mode-error-detection) (cancellation short-circuit + PIC-51 `stopReason: "error"` transport short-circuit). This is the same anchor the immediately-following transport-mapping clause already links to.
- "by the same assistant-text rule the prompt-mode driver pins above" → cite PIC-53 *Untyped-query `Ok(string)` trailing-turn extraction* at [`conversation-drive.md#untyped-query-ok-extraction`](./conversation-drive.md#untyped-query-ok-extraction), which pins the trailing-assistant-text concatenation rule (text-only, `\n`-joined, `thinking`/`toolCalls` omitted, empty string on no assistant text).
- PIC-42's "honour the same behavioural contract as prompt-mode typed queries above" → repoint to the prompt-mode typed-query material on `conversation-drive.md` rather than "above".

Keep "the same … rule on both sides" invariant wording intact; only the locator changes from positional ("above") to an explicit cross-page anchor. Do not relocate the prompt-mode rules — `conversation-drive.md` remains their single owner; `subagent.md` only references them.

## Consequence

**Severity:** advisory

An implementer reading `subagent.md` in isolation cannot resolve "the same two short-circuits" or "the same assistant-text rule" from the page itself; discoverability of the subagent query-result contract depends on already knowing the prompt-mode driver lives on `conversation-drive.md`. The underlying rules are fully and unambiguously pinned there (and the adjacent transport-mapping clause already cross-links to them), so a working implementation is still achievable — the defect is navigational, not a behavioural ambiguity.
