## Finding

The paired `V14a` / `V14a-T` leaves declare a four-page `**Spec.**` field, and two of the four page links carry trailing obligation-scope parentheticals: `conversation-drive.md` is annotated `(the denial-surface MUST under *No additional access channels*)` and `ceilings-3-and-4.md` is annotated `(the code-driven-tool-args row of the ceiling-#4 per-boundary table)`.

Per `conventions.md` §Leaf format and `leaf-template.md`, the `**Spec.**` field is a *page listing* — the set of `spec_topics/*.md` pages an implementer may restrict their reading to, closed under normative cross-link. Obligation-level scoping (which MUST, which table row) is the job of the `**Tests.**` REQ-ID / obligation bullets, which carry the leaf's binding obligations. Both parentheticals' content is already stated, more precisely, in the Tests bullets: the denial-surface MUST is the `[conversation-drive.md — No additional access channels]` bullet, and the ceiling-#4 code-driven-tool-args row is the `[ceilings-3-and-4.md — Per-boundary destination/surface table (ceiling #4)]` bullet.

The parentheticals are therefore a second, non-authoritative restatement of obligations that already live in Tests. They add no information the field's page-listing role requires and can drift from the Tests wording on later edits.

The second parenthetical, `ceilings-3-and-4.md (the code-driven-tool-args row …)`, was introduced later at `3d1ffd7`: that commit resolved a different finding by adding the `ceilings-3-and-4.md` page to the Spec field and attached the inline scope qualifier alongside the page link. The defect as framed (the Spec field carrying inline obligation-scope qualifiers) is thus the union of an inception-era pattern and a later fix that reinforced it.

## Solution Space

**Shape:** single

### Recommendation

In both `docs/plan_topics/V14a-tool-calls.md` and `docs/plan_topics/V14a-T-tool-calls.md`, edit the `**Spec.**` line to strike the two trailing parentheticals, leaving the four page links as a bare listing. Strike `(the denial-surface MUST under *No additional access channels*)` after the `conversation-drive.md` link and `(the code-driven-tool-args row of the ceiling-#4 per-boundary table)` after the `ceilings-3-and-4.md` link. The resulting field lists `tool-calls.md`, `conversation-drive.md`, `cancellation.md`, and `ceilings-3-and-4.md` with no qualifiers. Do not remove the page links themselves — only the parenthetical prose. The obligation scoping is already carried by the `conversation-drive.md — No additional access channels` and `ceilings-3-and-4.md — Per-boundary destination/surface table (ceiling #4)` Tests bullets, which remain authoritative.

Edge cases the implementer must watch: keep the two leaves in lockstep — their `Spec.` fields are identical by design, so apply the identical edit to both files. The strike does not affect the `conventions.md` §Sequential-by-default carve-out, which keys off cited REQ-IDs / code-keyed obligation areas in Tests, not Spec-field parentheticals.

## Consequence

**Severity:** cosmetic

The `Spec.` field's role is a closed page listing; the inline parentheticals duplicate obligation-scoping that already lives verbatim in the Tests bullets, creating a second, non-authoritative home for the same obligation that can drift from the Tests text on later edits. No implementer is misled today — the cited pages are still listed and the binding obligations remain in Tests.
