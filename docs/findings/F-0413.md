## Problem

`V18c`'s build-time literal-read assertion (authored by `V18c-T`) reads the cancellation-runtime constant `SHUTDOWN_AWAIT_CAP_MS` directly from `V9a`'s single source-of-truth pinned-constants block, and both leaves state the producer→consumer relationship explicitly in their `Adds.` fields. Yet `V18c`'s `Deps.` is `{V18c-T, V18a, V18b}` and `V18c-T`'s `Deps.` is `{V18a, V18b}` — neither lists `V9a`. `V9a` is reachable only transitively through `V18a`, whose `V9a` edge exists for a *different* symbol (`FACTORY_PROBABLE_CAPABILITIES`), so the direct edge for `SHUTDOWN_AWAIT_CAP_MS` is undeclared. The co-consumer `V17a`, which imports the same symbol, declares its `V9a` edge explicitly.

## Issue introduction

**Verdict:** single introducing commit
**Introducing commit:** `622d0d3` — "pi-loom plan: resolve \"SHUTDOWN_AWAIT_CAP_MS physical home contradicts pinned-constants-block placement\"" (2026-06-11)
**History:** `V18c`'s `Deps.` line has read `{V18c-T, V18a, V18b}` since plan inception (`c6a664e`); it was briefly expanded with runtime-evidence feature deps (`81ab342`, `c42f13d`, `ce32225`) and trimmed back to `{V18c-T, V18a, V18b}` in `8af3204` when the runtime-evidence acceptance gate was split out to `V18d`. At none of those points did `V18c` consume `SHUTDOWN_AWAIT_CAP_MS`, so omitting `V9a` was correct. `git log -S 'SHUTDOWN_AWAIT_CAP_MS'` shows the constant's build-time literal-read first entered `V18c` in `622d0d3`; that commit added the consumption without extending `Deps.`, converting a consistent leaf into a consume-but-don't-declare one. `V18c-T` has never listed `V9a` (`git log -S 'V9a' -- V18c-T` returns nothing) and does not name the constant in its own text, so its half of the gap is present-since-inception, exposed by the same `622d0d3` consumption add on the paired implementation leaf.

## Solution approach

Add `V9a` to the `Deps.` field of both `docs/plan_topics/V18c-version-bump-checklist.md` and `docs/plan_topics/V18c-T-version-bump-checklist.md`, declaring the direct producer→consumer edge for `SHUTDOWN_AWAIT_CAP_MS`. Mirror the explicit `V9a` edge the co-consumer `V17a` already carries.

## Solution constraints

- `V9a` owns the `SHUTDOWN_AWAIT_CAP_MS` pinned-constants block as the single source of truth; do not redeclare the constant or its value in `V18c`/`V18c-T`.
- The fix is confined to plan `Deps.` fields; the spec pages are read-only for this fix.
