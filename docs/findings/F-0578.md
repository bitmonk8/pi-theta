## Problem

`V18b` introduces a fail-closed build-time gate over the whole audited source tree (`src/**/*.ts` minus the enumerated exclusions) and, in the same commit, sweeps every existing off-inventory `src/` Pi-side reference so the audit lands green on `main`. A walker false positive, or a wrong rewrite during the sweep, reddens the entire `npm test` suite for every contributor on the shared CI substrate until the commit is corrected or reverted. Unlike the sibling gate leaves `H6a` (which carries a **Rollback** block) and `V18d` (which carries a **Blast radius** block plus a one-commit revert path), the `V18b` leaf documents no recovery path and no blast-radius statement for this event.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e ("pi-loom plan: build/update plan for spec.md + review")
**History:** `git log --follow` over `docs/plan_topics/V18b-inventory-audit.md` shows two commits. The leaf was created in c6a664e (2026-06-10) already carrying the fail-closed audit with no Rollback or Blast-radius annotation; the recovery-annotation gap is present from inception. The later commit 402bdbe (2026-06-11, "resolve … V18b Adds omits broader-inventory population, taxonomy, and land-green sweep") made the same-commit `src/` sweep ownership explicit in the Adds field — amplifying the blast-radius surface — but did not add a recovery or blast-radius note, so it did not introduce the gap.

## Solution approach

Add a recovery annotation and a blast-radius annotation to the **Adds** prose of the `V18b` leaf, mirroring `H6a`'s **Rollback** block and `V18d`'s **Blast radius** block. The recovery annotation states that the gate and its same-commit `src/` sweep revert together by reverting the single introducing commit, with no data, schema, or registry migration to unwind. The blast-radius annotation states that, because the audit is fail-closed over the whole audited source tree and the sweep edits `src/` in the same commit, a walker false positive or wrong sweep rewrite reddens the entire `npm test` suite for every contributor until the commit is corrected or reverted.

## Solution constraints

- Do not add a warn-only canary or staged "land red, fix incrementally" stage: `inventory-audit-intro.md` §Inventory-closure audit mandates the same-commit land-green sweep and forbids a staged rollout, so the annotation must record fail-closed-by-spec-design recovery rather than soften the gate.
- The spec is read-only for this fix; confine the edit to the `V18b` leaf.
