## Finding

`V3d` (and its tests partner `V3d-T`) carry a RET final-value-contract assertion that the function body's produced value "is its final value at the function-result seam on success **and is absent on fail/cancel**", and `V3d`'s **Ships when** repeats it ("the function-result-seam final value (produced on success, absent on fail/cancel)"). `V3d`'s declared **Deps** are `V3d-T`, `V3a`, `V4a`.

Per the spec, the final value's *absence* on fail/cancel is observable only at a caller surface: `functions.md` §"Final value (language definition)" states the final value is observable to programmatic callers in exactly two places — an `invoke` parent receives it as the `Ok` payload (`invocation.md` §Typed return) and a subagent-mode parent receives it across the subagent boundary — and that "On failure (`?` propagation, panic, exhausted runtime ceiling) and on cancellation, NO final value flows," with the trichotomy and per-outcome `Err` envelope owned by `errors-and-results/error-model.md` §Terminal outcomes and `cancellation.md`. The plan already routes these caller-side observations to the right leaves: `V15a` (invoke return) and `V9i` (subagent return) each carry a delegated witness "against the function-result seam `V3d` defines."

None of those observation surfaces is reachable from `V3d`'s dep scope. `V15a`/`V9i` (the only surfaces where an absent final value is observable) depend on `V3d`, not the reverse; the cancellation primitive lives in `V17a` and the cancelled-outcome envelope in `V4c`, neither in `V3d`'s Deps. The `?`/Err short-circuit half of "fail" is reachable (via `V4a`), but the caller-observed *absence* and the cancel facet are not. A `V3d-T` cancel test therefore cannot red out for the intended reason at this scope, so an implementer invents a local seam abstraction and a cancel sentinel that may diverge from the real `V15a`/`V9i` return surfaces and the `V17a`/`V4c` cancelled-outcome model.

## Issue introduction

**Verdict:** introduced-by-specific-commit
**Introducing commit:** `7a8565a` (2026-06-11) — "pi-loom plan: resolve \"V3d-T over-asserts final-value propagation\""
**History:** At plan inception (`c6a664e`, 2026-06-10) `V3d-T`'s final-value bullet read "the final value propagates to an `invoke`/subagent caller on success and is absent on fail/cancel" — i.e. the absence claim was anchored to the caller surface where it is observable, and `V3d`'s Ships when said only "final-value propagation." Commit `7a8565a`, a prior review-fix targeting the over-assertion of final-value *propagation* at `V3d`, removed the caller-surface anchoring and replaced it with the "function-result seam" coinage at `V3d`'s own scope, while retaining the "absent on fail/cancel" clause and adding it to `V3d`'s Ships when. `git log -S "function-result seam"` confirms that commit is the sole origin of the coinage across `V3d`, `V3d-T`, `V15a`, and `V9i`. The over-correction stripped the only construct that made the absence assertion observable, producing the present unreachable-at-dep-scope defect.

## Solution Space

**Shape:** single

### Recommendation

Re-scope `V3d`/`V3d-T`'s final-value-contract assertion to what is observable at `V3d`'s dep scope, and leave the absent-on-fail/cancel observation to the leaves that already carry it.

- In `docs/plan_topics/V3d-T-functions-and-return.md` and `docs/plan_topics/V3d-functions-and-return.md`, change the final-value-contract Tests bullet so it asserts only the success-path production: the function body's produced value (tail expression, or the operand of a short-circuiting `return`, or the implicit `null` of an empty-tail body) is its final value on the success path, including the `?`/Err early-return short-circuit that `V4a` makes reachable. Drop "and is absent on fail/cancel" from this bullet — that observation requires a caller surface and a cancellation primitive `V3d` does not depend on.
- In `docs/plan_topics/V3d-functions-and-return.md`, change **Ships when** to assert "the function-result-seam final value produced on success" only, dropping "absent on fail/cancel."
- Do not add `V15a`/`V9i` to `V3d`'s **Deps** — they depend on `V3d`, so the edge would be circular. The absent-on-fail/cancel observation is already closed by the `V15a` (invoke return) and `V9i` (subagent return) delegated witnesses, which is where the caller surface, the `V17a` cancellation primitive, and the `V4c` terminal-outcome envelope become reachable.
- Confirm the `cka-7` row in `docs/plan_topics/coverage-matrix.md` keeps the RET final-value contract jointly closed after the trim: `V3d` for production-on-success, with `V15a`/`V9i` recorded as the witnesses for the absent-on-fail/cancel facet, so the facet is not left without a matrix-recognised closer.

Edge case the implementer must watch: the `?`/Err short-circuit ("fail" via `Err` propagation) *is* reachable at `V3d` via `V4a` and stays in `V3d`'s scope; only the caller-observed *absence* of a final value and the *cancel* facet move out.

## Consequence

**Severity:** correctness

`V3d-T`'s cancel/absence arm has no observable surface at the leaf's dep scope, so the TDD ritual's "red for the intended reason" gate is unsatisfiable there; two reasonable implementers diverge — one invents a function-local seam abstraction and a private cancel sentinel, the other defers the facet — and either may not match the `V15a`/`V9i` return surfaces or the `V17a`/`V4c` cancelled-outcome model the absence is actually observed against. `cka-7` (RET) maps the final-value contract solely to `V3d`, so the divergent local assertion is the only matrix-recognised closure for a facet whose real witnesses live downstream.
