## Problem

`V16a`'s **Adds** field carries two clauses that prescribe the implementation topology of the cross-ceiling arbitration seam in a mandatory register: that the function is "imported and *consulted* directly at each call site, never constructor-injected", and that the No-globals/statics/singletons rule "must not be misread" as a directive to wire it as an injected interface or module-level singleton. Both prescribe *how the seam is wired* rather than *what it must do* — a Class-3 shape mandate, which `conventions.md` §Leaf format treats as illustrative and not binding through Adds prose alone. The Class-2 seam contract this leaf owns is already fully expressed without them: the `arbitrate(candidate) → { surfaced, masked? }` signature, the stateless/pure characterisation, the input/output bullets, and the consumer Deps edges.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 6826ca0 — pi-loom plan: resolve "V16a arbitration seam reuses spec's reserved seam noun" (2026-06-17, Thomas Andersen)
**History:** Both topology clauses entered V16a's Adds in 6826ca0, the single commit that resolved a different finding — the arbitration seam reused the spec's reserved "seam" noun. To disambiguate the Class-2 arbitration seam from the spec's construction-time-injected host DI seams, that edit inserted the prescriptive wiring prose ("imported and consulted directly … never constructor-injected"; "must not be misread … as an injected interface or a module-level singleton"). The pre-6826ca0 Adds carried no wiring-topology prose, so this is an over-correction introduced by that one disambiguating edit.

## Solution approach

In `docs/plan_topics/V16a-ceiling-order-masked.md`'s **Adds** field, strike the two wiring-topology clauses: the parenthetical "— the arbitration function is imported and *consulted* directly at each call site, never constructor-injected —", and the trailing "; the No-globals/statics/singletons rule must not be misread as a directive to wire this stateless pure function as an injected interface or a module-level singleton". Retain the Class-2-vs-DI disambiguation the same sentence already carries (that "seam" means a Class-2 named cross-leaf seam per `conventions.md`, distinct from the spec's PIC-10…PIC-16 host DI seams), the `arbitrate(candidate) → { surfaced, masked? }` signature, and the stateless/pure characterisation.
