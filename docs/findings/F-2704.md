## Finding

The real-host smoke gate's pass/fail criterion (d) requires that "the four session-double fidelity-contract axes … hold." The four axes are enumerated under a single verb "hold," but only the first two carry an inline invariant a scorer can check: axis (i) is "streamed-token order relative to `ctx.waitForIdle()` resolution" (an ordering observable) and axis (ii) is "each streamed assistant response is appended as exactly one prompt-mode turn (a per-response invariant, independent of total turn count)." Axes (iii) "the `pi.on` cancel-forward subscription" and (iv) "cancellation propagation" are bare noun phrases — they name a behaviour but state no observable that must be true for the axis to "hold" on a live host.

A contributor running the manual smoke against real Pi cannot determine what to observe to score axes (iii) and (iv). For axis (iii) there is no observable text anywhere in the criterion. For axis (iv), criterion (c) of the same gate already defines an observable for cancellation ("cancellation propagation is observed at the injected cancel point, and a mid-stream cancellation inside the subagent-mode `AgentSession` the fixture spawns leaves that session's committed turns unmutated"), so axis (iv) restates a concept criterion (c) already scores without saying whether (d-iv) adds anything or merely points back at (c).

The fidelity-contract axes are authored at `H4a`, whose self-check items (iii)/(iv) carry the same bare-phrase form but are at least pinned to per-axis spec cross-references (`conversation-drive.md` PIC-18, `cancellation.md` "Forwarding into `loomAbort`", CNCL-4). The smoke-gate criterion (d) carries neither the observable nor those cross-references, leaving the live-host scorer with nothing concrete to assert.

## Consequence

**Severity:** correctness

Two contributors scoring criterion (d) against a live Pi host would diverge on what makes axes (iii) and (iv) "hold": one might assert nothing observable for the `pi.on` cancel-forward subscription and pass it vacuously, while another tries to construct an ad-hoc check. A genuine double-vs-real divergence on the cancel-forward subscription — the production seam underlying every host-driven cancellation — could pass the gate unflagged, which defeats the smoke run's purpose of catching that exact class of divergence.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/real-host-smoke-gate.md`, criterion (d), give axes (iii) and (iv) inline observables parallel to the ordering observable on (i) and the single-turn invariant on (ii), so each axis names what must be true on a live host.

- Axis (iii) — `pi.on` cancel-forward subscription: state the observable that a host-side cancel signal delivered through the `pi.on` subscription forwards into loom-side cancellation (the behaviour `conversation-drive.md` PIC-18 and `cancellation.md` "Forwarding into `loomAbort`" define — the same cross-references `H4a`'s fidelity-contract sentence cites).
- Axis (iv) — cancellation propagation: state the live observable per `cancellation.md` CNCL-4. Criterion (c) already defines a cancellation observable ("cancellation propagation is observed at the injected cancel point … committed turns unmutated"); either point axis (iv) at criterion (c)'s observable or restate it, so the axis is no longer a bare phrase and its relationship to (c) is explicit rather than implicit.

The fix is internal to the smoke-gate note; the spec cross-references are read-only sources for the observable text, and no leaf acceptance criterion changes.
