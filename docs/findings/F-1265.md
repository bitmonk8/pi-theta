## Finding

`docs/spec_topics/pi-integration-contract/host-interfaces-core.md` introduces the touched `ExtensionContext.compact(options?: CompactOptions): void` row and immediately follows it with a free-standing prose paragraph that restates the full `CompactOptions` member set inline: `CompactOptions = { customInstructions?: string; onComplete?: (result: CompactionResult) => void; onError?: (error: Error) => void }`. The same paragraph also discloses that "Loom does not construct either value in loom 1.0".

The restatement carries no non-normative marker. Every other instance of a member-pinning TypeScript surface on this page either belongs to the loom-load-bearing subset (the three `interface … { … }` blocks for `ExtensionContext` / `ExtensionCommandContext` / `AgentSession`, each prefaced with a "MUST be re-validated against [the cited declaration] on each Pi minor bump" lead-in that defines them as point-in-time snapshots of the Pi-owned shape), or is an explicitly labelled non-normative orientation passage (the `estimateTokens` "**Non-normative.**" character-quarter rationale). The `CompactOptions` literal falls in neither category: it reads as authoritative member-pinning prose for a Pi-owned construction surface loom never instantiates, and the only "this is just a snapshot" signal a reader gets is the trailing re-validation sentence.

The practical consequence is a maintenance and reader-clarity gap. A reader cannot tell from the typography or framing whether loom is asserting a contract on the `CompactOptions` shape (it is not) or merely recording what the SDK declared at the pin; a contributor watching for sites that the Pi version-bump procedure must sweep has no marker to recognise this paragraph as one of them; and the paragraph's inline literal will silently drift from the SDK declaration between bumps with no visible "this is provisional" framing.

## Solution Space

**Shape:** single

### Recommendation

Prefix the paragraph that opens "`CompactOptions` and `CompactionResult` (referenced by the `compact` row above) are declared …" with an inline `**Non-normative snapshot.**` marker (matching the wording the related fix introduces on the three Pi-owned `interface { … }` blocks elsewhere on the page), and keep the existing trailing "re-validated on each Pi minor bump per [Pi version bump procedure]" sentence as-is.

Do not drop the inline literal: it is the page's snapshot of the Pi-owned shape at the SDK pin, and the version-bump procedure's surface-inventory re-validation reads from it. The non-normative marker keeps the snapshot present for that purpose while signalling, in the same typographic vocabulary the page already uses for `ExtensionContext` / `ExtensionCommandContext` / `AgentSession` once the related fix lands, that loom does not assert a contract on the shape.

Implementer notes:

- The marker phrase must be the same one chosen by the related finding's fix on the three `interface { … }` blocks ("Pi-owned interface signature blocks lack the non-normative-snapshot marker…"); both fixes share one convention on this page and should land together or in adjacent edits.
- Leave the "Loom does not construct either value in loom 1.0" sentence in place — it is the only place on the page that disclaims construction of these two types, and removing or paraphrasing it would lose information the marker does not carry.
- The marker is editorial only; do not relocate the paragraph, do not retitle the section, and do not change which file it lives in.

## Consequence

**Severity:** advisory

The spec is not wrong about the `CompactOptions` shape, and the surrounding sentence already says loom does not construct either value, so no implementer would mistakenly try to instantiate the type or read a member loom does not consume. The gap is editorial: a reader scanning the page for Pi-owned snapshot surfaces has no consistent marker, the inline literal will drift silently between Pi minor bumps without a "this is a snapshot of Pi at the pin" framing, and the version-bump checklist has one more site that depends on the contributor remembering it rather than a typographic cue.
