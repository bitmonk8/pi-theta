## Finding

The Trust boundary orientation bullet in `overview-and-orientation.md` ends with: "Any callable a loom declares exposes the full underlying capability of that callable to the loom's model; the runtime applies no privilege classification or filtering. `` `bash` `` and `` `read` `` are illustrative — the rule does not depend on a callable being deemed 'privileged.'" The two example tokens are presented bare, with no marker that they are **Pi tool names** drawn from Pi's built-in tool registry.

In this orientation page — which is establishing loom's own concepts — a reader has no local cue that `bash`/`read` are externally-owned Pi tools rather than loom built-ins or reserved loom identifiers. The fact that these are Pi tool names is only established elsewhere (`frontmatter/frontmatter-fields-a.md`: "Pi tool names (`read`, `bash`, `grep`, …) resolve against Pi's tool registry"; `pi-integration-contract/subagent.md` lists `read`/`bash`/`edit`/`write` as Pi default built-ins). A one-word qualifier at the point of use removes the ambiguity without disturbing the surrounding normative prose.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** f27c1bd — pi-loom spec: resolve "High-privilege callable qualifier is undefined and conceals a universal rule" (2026-05-08, Thomas Andersen)
**History:** Commit f27c1bd rewrote the Trust boundary bullet to remove the undefined "high-privilege callable" qualifier, replacing the earlier text (which named only `` `bash` `` as an example) with the current universal-rule phrasing plus the new sentence "`` `bash` `` and `` `read` `` are illustrative …". That edit introduced the unlabelled `read` example and the "illustrative" framing without identifying either token as a Pi tool name. The phrasing then migrated verbatim into `spec_topics/overview-and-orientation.md` when the spec was split out of `spec.md` (81bf52a, 2026-06-11).

## Solution Space

**Shape:** single

### Recommendation

In the Trust boundary bullet, qualify the example tokens as Pi tools at the point of use, e.g.: "`` `bash` `` and `` `read` `` (Pi tools) are illustrative — the rule does not depend on a callable being deemed 'privileged.'" No cross-reference is required; the parenthetical is sufficient to prevent the built-in misreading. Leave the surrounding universal-rule sentence unchanged.

## Consequence

**Severity:** cosmetic

A reader may momentarily misread `bash`/`read` as loom built-ins rather than Pi tools. No implementer divergence in behaviour results — the bullet is informative orientation and the rule explicitly does not depend on a callable being "privileged."
