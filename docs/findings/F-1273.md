## Problem

`tool-calls.md` is registered as a non-narrative page under the `TOOL` prefix in the REQ-ID prefix table but carries zero `TOOL-N` REQ-ID anchors. Its defining obligation sites — the model-driven parallel-batch settle-all rule, the `CodeToolError` cause surface, the off-surface outcome-routing rules, the `.loom`-adapter-layer MUSTs in the outcome enumeration, and the code-driven tool-argument rules — therefore expose no `#prefix-n` fragment. GOV-9's `#prefix-n` cross-link contract is consequently unsatisfiable for every page that depends on these obligations, forcing local restatement (a GOV-8 drift hazard) or a heading-anchor link GOV-9 sanctions only for narrative pages.

## Issue introduction

**Verdict:** present-since-inception-with-progressive-extension

**Introducing commits:**

- `c1e5d64` (2026-05-05, "spec: REQ-ID infrastructure + Phase 12a heading promotion on five named pages (Decision #12, partial)") — the foundational commit. It introduced the REQ-ID prefix table on `spec.md` with rows for `LEX`, `SCHM`, `DESC`, `FN`, `TOOL`, `BNDS` (then `BNDG`), `RET`, `RVM`, `IMPL`, and `GRAM`, and explicitly deferred the per-rule coinage pass: *"Phase 12b (per-rule REQ-ID assignment) … remain as the mechanical sweep across the rest of spec_topics/ and plan_topics/."* The Phase 12a heading-promotion pass touched only five named pages (`binder.md`, `query.md`, `schemas.md`, `expressions.md`, `frontmatter.md`); the eleven pages in this finding were registered but not anchored, by design.
- `1c7f550` (2026-05-07, "Hard ceilings block does load-bearing definitional work inside informative orientation") — extended the same pattern when the new `hard-ceilings.md` page was registered under `CEIL` without coining `CEIL-N` IDs.
- `c4ad105` (2026-05-08, "HC3-a…HC3-e and NOCEIL-N labels escape GOV-3/GOV-8") — re-registered `hard-ceilings.md` to additionally own `CIO`, and `CIO-1…CIO-6` were subsequently coined while `CEIL` remained empty.
- `4f588e0` (2026-06-03, "Non-narrative pages with zero coined REQ-IDs non-referenceable") — recognised the residue and *codified* the progressive-drainage discipline by adding GOV-22 to `governance.md`. GOV-22 explicitly pins the per-commit, on-edit coinage trigger and disclaims any corpus-wide coinage commit.

**History:** The gap is structurally intentional — the eleven prefixes were registered up front (`c1e5d64`) as a placeholder commitment that per-page coinage would land later, page-by-page, as edits to those pages flow through. `4f588e0` later codified that intention as GOV-22, which means the current state is not a regression but the steady-state operating mode of the rule. The finding's force is that "later" has never arrived for these eleven pages despite ~13 months of spec evolution, leaving GOV-9 cross-links permanently unsatisfiable for the bulk of the normative corpus.

## Solution approach

Coin `TOOL-N` REQ-ID anchors at the defining obligation sites on `tool-calls.md`, in the dual-form layout GOV-1 mandates, under the page's already-registered `TOOL` prefix with numeric tails allocated per GOV-3. This is GOV-22's progressive-coinage obligation, drawing the page's standing residue against GOV-9's `#prefix-n` cross-link contract.
