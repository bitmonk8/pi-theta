## Problem

The closing-gate machinery has no arm that verifies per-facet citing-test coverage for a multi-leaf coverage-matrix row. The `H5a` per-REQ-ID citing-test scan certifies only that *a* citing test exists somewhere for the whole REQ-ID, and the `H5d` transitive-completeness arm is a per-cell at-least-one check that reddens only when *none* of a cell's listed leaves reaches `H5b`'s `Deps.` — neither inspects whether each individual facet-closer in a split row carries its own facet-naming citing test. So a split row such as `cka-33 | V17a, V14f, V13f, V15h, V9o` can ship with one facet tested and a sibling failure-path / cancellation-suppression facet (e.g. `V9o`'s subagent-`AgentSession.abort()` suppression) carrying no citing test, with `H5a`, `H5d`, and `H5e` all green. The only backstop is the manual *Loom 1.0 release-time residue inspection* item 7, which has no mechanical enforcement and no reproducible per-row record.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 953e3fa — pi-loom plan: resolve "closing gate checks numbered-REQ-ID matrix mapping, not test assertion" (2026-06-10, Thomas Andersen); cda161b — pi-loom plan: resolve "split REQ-IDs pass closing gate on one facet citing test" (2026-06-17, Thomas Andersen)
**History:** The citing-test reconciliation arm was added to `conventions.md` by 953e3fa (2026-06-10) as an existence-only, per-REQ-ID scan, so it never gated per-facet completeness of multi-leaf rows. The resulting split-row gap was explicitly recognised by cda161b (2026-06-17), which introduced the split-row joint-completeness wording and routed the residue to manual item 7 rather than mechanising it; later commits (2ad25ab, 685e45b, cf1bb3e) refined item 7 (per-cell at-least-one phrasing, co-witness vs facet distinction, code-keyed-table sweep) but kept the backstop manual. The mechanical gap therefore dates to the gate's existence-only design and has been documented-but-unmechanised since.

## Solution approach

Establish a machine-parseable facet-annotation convention in `coverage-matrix.md` so each facet of a multi-leaf row is individually associable to a citing test, and add a closing-gate arm that derives each row's facet partition by the item-7 rule (drop co-witness-annotated leaves, keep the rest) and asserts every remaining facet leaf carries a facet-naming citing test, across both coverage-matrix tables. Bind the arm at the `H6a` activation alongside the other citing-test surfaces, seed its violation / no-violation fixtures in the citing-test owner `H5a-closing-gate-automation.md` or the facet-partition owner `H5d-transitive-completeness-gate.md`, and record the arm in `conventions.md` §REQ-ID discipline. Give the code-keyed rows (e.g. `cka-33`) explicit attention, since the diagnostic-code↔asserting-test parity gate gives them no incidental coverage.

## Solution constraints

- The facet-annotation convention must keep the closing-leaf cell's annotation prose backtick-free — only leaf-ID tokens may be backtick-delimited — because `H5d` tokenises closing-leaf cells by their backtick-delimited spans (`coverage-matrix.md` closing-leaf-column authoring rule).
