## Problem

The off-surface routing bullet in `V14c` (**Adds**, **Tests**, **Ships when**) and `V14c-T` (**Tests**) states the pre-eval setup throw outcome ("a pre-eval setup throw → `{isError:true}` + one `loom/runtime/internal-error` diagnostic") with no scope qualifier. The spec's `tool-calls.md` §Outcome enumeration scopes this outcome narrowly to a *pre-evaluation setup throw inside the `.loom`-callable adapter* on a model-driven parallel tool call. As written, the bullet reads as though any code-side tool pre-evaluation throw routes to `{isError:true}` + `internal-error`, whereas a code-side Pi-tool `execute()` throw lowers to `Err(CodeToolError { cause: "execution" })`. A red-first author could test V14c against the unscoped reading, gating the implementation on a disposition the spec never requires.

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** The off-surface routing bullet first appeared in `V14a-tool-calls.md` at `c6a664e`, already unscoped ("a pre-eval setup throw → `{isError:true}` + `internal-error`"). `bd2eaf4` (2026-06-16, "resolve 'V14a internal-error Tests bullet conflates four off-surface outcomes'") reworded the bullet to its current four-outcome form, still unscoped. `5a1915d` (2026-06-17, "resolve 'V14a bundles eight independently-testable concerns'") split the off-surface routing out of V14a into the new `V14c` / `V14c-T` leaves, copying the bullet verbatim. The `.loom`-callable-adapter scope qualifier was never present at any revision; the gap is an original authoring omission carried through both later edits.

## Solution approach

Qualify the pre-eval setup throw clause in `V14c` (**Adds**, **Tests**, **Ships when**) and `V14c-T` (**Tests**) to name its `.loom`-callable parallel-batch adapter scope, distinguishing it from a code-side Pi-tool `execute()` throw. Anchor the scope to `tool-calls.md` §Outcome enumeration, "Pre-evaluation setup throw inside the `.loom`-callable adapter", and preserve the further obligations that scoped form carries (the co-emitted `loom-system-note` and the bare callable-set name in the `{isError:true}` message).

## Solution constraints

- Qualify only the pre-eval setup throw clause; leave the other three off-surface outcomes (non-conforming return shape, non-settling promise, post-cancel late settlement) unqualified.
- Do not edit the post-cancel late settlement clause's naming — it is owned by the same-cluster naming sibling finding.
