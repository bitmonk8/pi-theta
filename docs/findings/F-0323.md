## Finding

The Pass/fail criterion paragraph in the manual real-host smoke gate note ends with the unqualified claim: "so a genuine divergence still produces a falsifiable fail that drives the revert / merge-block path below." This overstates what the gate guarantees. The same paragraph scopes the run to *model-output-invariant* observables (criteria (a)–(e)) and, in its very next clause, classifies "benign live-model variance in turn order, turn count, or which permitted codes fire" as explicitly **not** a fail.

A genuine behavioural regression whose only observable manifestation falls inside that variant space — for example, a real fidelity defect that changes turn count or shifts which permitted-code fires without violating any of (a)–(e) — is indistinguishable from benign live-model variance and therefore produces no fail. The summary sentence promises a guarantee broader than the criteria deliver: it asserts detection of *any* genuine divergence, when the gate only detects divergences observable on the (a)–(e) invariant criteria.

The downstream H6a leaf already frames its reference to this gate correctly ("where pass is the model-output-invariant criterion `real-host-smoke-gate.md` defines" and "narrowed model-output-invariant criterion"), so the inconsistency is confined to this note's own self-summary.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/real-host-smoke-gate.md`, within the Pass/fail criterion paragraph, narrow the summary clause so it claims only what (a)–(e) deliver. Replace the text `so a genuine divergence still produces a falsifiable fail` with `so a divergence observable on the (a)–(e) model-output-invariant criteria still produces a falsifiable fail`. The rest of the clause (`that drives the revert / merge-block path below; benign live-model variance in turn order, turn count, or which permitted codes fire is **not** a fail`) is unchanged and already states the complementary caveat. This brings the self-summary into line with the surrounding scoping language and with H6a's "narrowed model-output-invariant criterion" reference.

## Consequence

**Severity:** advisory

A contributor or the H6a release-gate auditor reading the summary sentence may over-trust the smoke gate to catch any genuine double-vs-real-host divergence, when a real regression that manifests only as a turn-order, turn-count, or which-permitted-code-fires change is scored benign and passes. The residual gap is real but the gate still functions as defined by (a)–(e); no leaf is implemented incorrectly.
