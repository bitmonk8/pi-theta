## Finding

`V9g-T`'s sub-step-3 Tests bullet bounds the `Promise.allSettled` await by `SHUTDOWN_AWAIT_CAP_MS`, a `V9a` export, but `V9g-T.Deps` lists only `V9e, V9h, V17a`; the impl partner `V9g` declares `V9a`. The edge is reachable only transitively (`V9g-T → V9h → V9a`) and breaks if that chain is restructured.

- `V11j` / `H4b` asymmetry — `66b3815` (2026-06-16, "pi-loom plan: resolve \"V11f bundles binder retry budget with in-flight cancellation forwarding\""), the commit that created the `V11j` / `V11j-T` pair (born with `V11j-T.Deps = V11f, V17a, H4b` and `V11j.Deps = V11j-T, V11f, V17a`).
**History:** `git log -S` over each cited token shows the missing/one-sided edge has existed in each file since that file's authoring; no commit ever declared the edge and later removed it. Each asymmetry is original to its leaf, not a later regression, and the four leaves were authored in separate commits.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V9g-T-session-shutdown.md`, add `V9a` to the `**Deps.**` line so the tests-task declares the leaf whose constant it consumes, matching `V9g.Deps`.

## Consequence

**Severity:** advisory

A tests-task whose missing edge is only transitively satisfied (`V9g-T`, `V9h-T`) stays pickable today but turns into a compile/pickability failure the moment the intervening chain is re-routed, with no local signal of the dependence. The one-sided `H4b`/`V2c` edges (`V11j`, `V11h-T`) leave a reader unable to tell a deliberate test-only / impl-only dependency from an omission, and `V11h-T`'s `V2c` gap is not even transitively covered. No leaf is unpickable now, so implementers can still produce working leaves.
