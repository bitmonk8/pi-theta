## Finding

`H7a` checks in a **committed permitted-code list** alongside its fixture `.loom`: a manually-maintained, best-effort union of every `loom/...` code each `Deps` slice *can* emit, "kept current by a per-Deps-slice provenance obligation." The only mechanical gate over it is the `golden ŌŖå permitted` containment test (one of `H7a`'s `Tests` bullets). The list's *superset* invariant ŌĆö that it captures every code each slice can emit ŌĆö is asserted by no step: `H7a`'s `Adds` and the smoke-gate note both explicitly concede "its completeness ŌĆ” is not mechanically verified, so a stale union can omit a benign code a slice emits."

That conceded residue is routed nowhere. The list is the single committed reference set the `real-host-smoke-gate.md` pass criterion (e) (`emitted ŌŖå permitted`) and the `H6a` release-gate evidence record score the live run against. When a slice gains an emittable code that the maintainer does not add to the union, the union goes stale; a benign live emission of that code then trips criterion (e) and scores as a confirmed behavioural-divergence finding ŌĆö a merge-blocker / revert trigger ŌĆö even though nothing actually diverged. In the other direction, a permanently over-broad union dulls (e)'s ability to flag a genuinely new emitted code. `real-host-smoke-gate.md` defines un-runnable-gate carve-outs (window-deferral, missing binder model / credentials) that do not score as fails, but it has no carve-out for a stale-union (e) failure: criterion (e) failure on any emitted code outside the permitted set is unconditionally "a fail (a confirmed behavioural-divergence finding)."

The `conventions.md` *Loom 1.0 release-time residue inspection* already carries the matching precedent: item 9 sweeps the `H3a` ambient timing-primitive list ŌĆö "a manually-maintained derivation with no mechanical reconciliation ŌĆ” required to be a superset of the spec ban," reconciled by a reviewer at the release gate. The permitted-code list has the identical shape (manually-maintained superset, completeness not mechanically verified) but no equivalent sweep item, so its provenance obligation has no surface that ever forces the reconciliation.

## Solution Space

**Shape:** single

### Recommendation

Route the permitted-code-list completeness residue to the existing `conventions.md` *Loom 1.0 release-time residue inspection*, mirroring how item 9 already routes the structurally-identical `H3a` timing-primitive superset list:

- In `conventions.md` ┬¦*Loom 1.0 release-time residue inspection*, add a sweep item directing the release-gate reviewer to reconcile each `H7a` `Deps` slice's emittable `loom/...` codes against the committed permitted-code list and confirm the union is a true superset (every code each slice can emit appears in the list). State its clean condition (the union covers every slice-emittable code) and that a missing code is a dispositionable residue defect, exactly as item 9 records superset coverage of the spec ban. The codes the reviewer reconciles against are the slices in `H7a`'s `Deps` (`H4a`, `H4b`, `V5d`, `V8a`, `V11a`, `V11b`, `V11i`, `V11c`, `V11d`, `V11e`, `V11f`, `V13a`, `V13b`, `V13c`, `V13d`, `V13f`, `V14a`, `V14b`, `V16a`, `V17a`).
- Update the release-time residue-inspection item count where it is stated ŌĆö `H6a`'s `Adds` and `Ships when` both name the sweep as the "nine-item" checklist ŌĆö so the count matches the added item, and ensure `H6a`'s residue-inspection recording obligation covers the new item's clean-or-dispositioned outcome alongside the other items.
- In `real-host-smoke-gate.md` criterion (e) (or `H7a`'s permitted-code-list `Adds`), state the recovery path for a criterion-(e) failure traced to a stale union: an emitted code the owner confirms is a benign slice-emittable code absent from the committed list is dispositioned by adding it to the list (and re-running), distinct from a genuine new-code divergence which remains a fail. This closes the gap that `real-host-smoke-gate.md` leaves by scoring every out-of-permitted emission unconditionally as a confirmed divergence.

Edge case the implementer must watch: this finding interacts with the proposal to relocate the release-time residue-inspection checklist out of `conventions.md` (see Related Findings) ŌĆö the new sweep item lands wherever that checklist ultimately lives, and the anchor `#release-time-residue-inspection` referenced by `H6a` must stay valid.

## Consequence

**Severity:** advisory

A `Deps` slice can gain an emittable code with no surface forcing the maintainer to add it to the committed permitted-code list. The stale union then makes the manual real-host smoke gate's criterion (e) (`emitted ŌŖå permitted`) fire spuriously ŌĆö a benign live emission scores as a confirmed behavioural-divergence finding and a merge-block / revert trigger ŌĆö while in the over-broad direction it dulls (e)'s power to flag a real new-code divergence. The structurally-identical item 9 list is swept by the release-time residue inspection; this one is not, so the conceded completeness residue is never reconciled.
