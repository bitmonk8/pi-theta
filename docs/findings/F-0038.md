## Problem

`H1a`'s **Adds** exhaustively enumerates loom's own `devDependencies` (`vitest`, the four `@earendil-works/pi-*` packages, the lint toolchain) but never declares the standalone `typescript` package, even though the same **Adds** defines `npm run typecheck` as the build-time `tsc` runner and **Ships when** requires a fresh checkout to run it green. Without `typescript` declared, `tsc` is not provisioned by `npm install`, so an implementer must invent the declaration off-script. The version is also load-bearing: the spec pins a TypeScript floor of `>=5.4.0` at `host-prerequisites.md#tsc-toolchain-floor`, because `V18c`'s `loom/typecheck/session-shutdown-reason-snapshot` brand-string gate relies on `tsc` surfacing that literal verbatim and grep-ably. `H1a` cites no such anchor and carries no Tests bullet asserting either the `typescript` key or the floor, unlike its existing spec-anchored assertions for `engines.node` and the four-peer pin.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` ("pi-loom plan: build/update plan for spec.md + review", 2026-06-10) — the commit that first authored `H1a-scaffold-and-toolchain.md`.
**History:** The `typescript` package was never present in H1a's devDependency enumeration; `git log -S'typescript'` against the leaf matches only `@typescript-eslint/parser` (added by `fe9d03a`, the lint-toolchain identities pass), never the standalone `typescript` key. The load-bearing `>=5.4.0` floor has never appeared in any plan file (`git log -S'>=5.4.0' -- docs/plan_topics/` is empty). The gap survived every subsequent pass that extended exactly this enumeration — `fe9d03a` (lint toolchain), `c8564bb` (Pi-SDK devDeps provisioning), `ca5c2fc` (ajv-formats runtime dep) — and the `765aff2` pass that introduced the `npm run typecheck` script that depends on `tsc`, none of which added the package or its floor.

## Solution approach

Declare `typescript` in `H1a`'s **Adds** under loom's own `devDependencies`, pinned to the `>=5.4.0` floor and sourced from `host-prerequisites.md#tsc-toolchain-floor` rather than restated as a fresh plan-side literal, matching how `H1a` already sources the `engines.node` floor from its spec anchor. Note that `typescript` is the `tsc` provider behind the `npm run typecheck` script `H1a` already defines. Add a **Tests** bullet asserting, by build-time literal-read of `package.json#devDependencies`, that the `typescript` key is present and its declared range satisfies the spec floor.

## Solution constraints

- The spec is read-only for this fix; the `>=5.4.0` floor is owned by `host-prerequisites.md#tsc-toolchain-floor` and is not the plan's to define.
