## Problem

The typed depth-6 co-fire `Tests` bullet on `V13c-T` omits the loom-owned-depth-walk-before-AJV mechanism clause that its paired implementation leaf `V13c` carries; the `-T` bullet asserts only the `maxDepth` error surface, not the ordering. Because `V13c-T` is the authoritative authoring of the failing tests (its `Ships when` is "the tests above exist, compile, and fail red for the intended reason"), a test written from it alone is satisfied by a non-conforming AJV-only implementation that emits a coincidentally-identical `schema_keyword:"maxDepth"` error without ever running the loom-owned depth walk that `V5e` exists to own. `V13c` could then ship with the `V5e` before-AJV ordering guarantee unverified while every gate stays green.

## Issue introduction

**Verdict:** single-commit-introduced
**Introducing commit:** `9d880b8` — "pi-loom plan: resolve \"V5e before-AJV Tests bullet asserts an ordering its isolated harness cannot witness\"" (2026-06-17)
**History:** `git log -S "trips the loom-owned depth walk"` / `-S "before AJV"` shows the clause entered `docs/plan_topics/V13c-query-tool-loop.md` only in `9d880b8` and never entered `docs/plan_topics/V13c-T-query-tool-loop.md`. `git show --stat 9d880b8` confirms that commit edited `V13c`, `V5e`, and `V5e-T` but did not touch the paired `V13c-T`. Before `9d880b8` the two leaves' depth-6 bullets were symmetric (both lacked the clause); the asymmetric one-sided edit — made while resolving a different finding — created the divergence. The durable fix is to restore symmetry by adding the clause to `V13c-T`, not to revert `9d880b8`.

## Solution approach

Add the loom-owned-depth-walk-before-AJV mechanism clause to `V13c-T`'s typed depth-6 co-fire `Tests` bullet so it reads identically to `V13c`'s: assert that a depth-6 typed-query response trips the loom-owned depth walk (`V5e`) before AJV, and that ceiling #4 then surfaces as `Err(QueryError { kind: "validation", cause: "schema_validation" })` (`schema_keyword: "maxDepth"`). Restoring the clause makes the witnessable `V5e` ordering property part of the red-first test the leaf authorises.
