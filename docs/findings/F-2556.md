## Finding

In `grammar.md`, the block-expressions paragraph explaining the empty-tail return behaviour ends with:

> When `FnBody` or `LoomBody` is parsed without a trailing `Expr` … per [Function Definitions — Empty-tail body](./functions.md#empty-tail-body). Authors who want the function or loom to be syntactically required to produce a value should declare an explicit non-`void` / non-`null` return type; `void` is the explicit "intentionally produces no value" annotation and **is governed by the same page**.

"the same page" is an implicit back-reference to `functions.md#empty-tail-body`. That anchor *was* named explicitly earlier in the same paragraph, but an intervening clause ("Authors who want … non-`null` return type") separates the antecedent from the pronoun, so a reader must backtrack to resolve which page governs `void`. The companion Type-grammar sentence (line 87 of the same file) already states the same ownership with a proper anchored link (`whose semantics are owned by [Function Definitions — Empty-tail body](./functions.md#empty-tail-body)`); this trailing reference should match that form rather than relying on a deictic phrase.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 52219c4 — Address all 41 spec-review findings (2026-05-06, Thomas Andersen)
**History:** The entire empty-tail `void` paragraph — including the trailing "is governed by the same page" phrasing — was added to `grammar.md` in commit 52219c4. The defect was born with the prose; no later edit touched the phrase (the only subsequent hit, 31ff060, merely relocated the file into `docs/`).

## Solution Space

**Shape:** single

### Recommendation

Replace "is governed by the same page" with an explicit anchored link, mirroring the Type-grammar sentence above it:

> `void` is the explicit "intentionally produces no value" annotation, governed by [Function Definitions — Empty-tail body](./functions.md#empty-tail-body).

## Consequence

**Severity:** cosmetic

Pure wording. The governing page is still discoverable by re-reading the sentence, and no implementer behaviour diverges; the cost is reader friction only.
