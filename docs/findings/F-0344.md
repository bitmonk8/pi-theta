## Finding

`H6a`'s **Ships when** clause names the manual real-host smoke pass criterion as "no-error completion, binder output that validates against the binder envelope schema …, observed cancellation propagation, **the four fidelity-contract axes**, and emitted `loom-system-note` codes …". The phrase "the four fidelity-contract axes" is never enumerated in `H6a` and carries no cross-reference to where the four members are defined. The release-gate auditor reading `H6a` alone cannot determine which four behaviours the manual smoke run must observe.

The set is in fact owned by `H4a`, which establishes the **session-double fidelity contract** as a set of four pinned-SDK *behaviours*: (i) streamed assistant tokens observable in the transcript before `ctx.waitForIdle()` resolves, (ii) one streamed assistant response appended as a single prompt-mode turn, (iii) the `pi.on` cancel-forward subscription, and (iv) cancellation propagation. `H6a` calls them "axes" where `H4a` calls them "behaviours", and the two leaves are not linked at this point, so a reader cannot mechanically connect `H6a`'s term to `H4a`'s contract.

A second wrinkle compounds the gap: `H6a`'s criterion already lists "observed cancellation propagation" as a standalone item immediately before "the four fidelity-contract axes" — but cancellation propagation is itself member (iv) of `H4a`'s four, so the criterion double-counts it.

## Issue introduction

**Verdict:** single introducing commit
**Introducing commit:** `3911733` ("pi-loom plan: resolve \"Live-host smoke pass criterion assumes a non-deterministic LLM reproduces a transcript\"", 2026-06-11)
**History:** Prior to `3911733`, `H6a`'s release-gate acceptance criterion was an exact-reproduction test ("appended turns match `H7a`'s committed golden transcript in order and count and the run emits exactly `H7a`'s committed golden `loom-system-note` diagnostics list") — the phrase "fidelity-contract axes" did not exist. Commit `3911733` replaced that exact-reproduction criterion with a model-output-invariant criterion and introduced the unenumerated phrase "the four fidelity-contract axes" referencing `H4a`, while `H4a` describes the same set as "behaviours" — so the axes-vs-behaviours mismatch and the missing enumeration both entered in this single edit. The later commits touching this paragraph (`eca63cf` 2026-06-11, adding the falsifiable-evidence-record wording; `0bd1978` 2026-06-17, moving the acceptance from Tests to Ships when) preserved the phrase verbatim and did not enumerate or cross-reference it.

## Solution Space

**Shape:** single

### Recommendation

In `H6a`'s **Ships when** field, replace the bare term "the four fidelity-contract axes" with an enumeration of the four behaviours and a cross-reference to `H4a`'s session-double fidelity contract (`H4a`'s Tests self-check enumerates them as items (i)–(iv)). Use `H4a`'s canonical label "behaviours" rather than "axes" so the two leaves name the set identically. The four behaviours to enumerate, per `H4a`:

- streamed assistant tokens observable in the transcript before `ctx.waitForIdle()` resolves;
- one streamed assistant response appended as a single prompt-mode turn;
- the `pi.on` cancel-forward subscription;
- cancellation propagation.

While editing, resolve the double-count: `H6a` currently lists "observed cancellation propagation" as a standalone item immediately before the four — cancellation propagation is the fourth behaviour, so the standalone mention should be folded into the enumerated set (or the enumeration phrased to make clear it subsumes the standalone item) rather than listed twice.

The fix is confined to `H6a`'s **Ships when** prose plus a link to `H4a`; no spec edit and no `H4a` acceptance-criteria change is required. If aligning the term requires touching `H4a`, limit that edit to confirming the "behaviours" label remains the single canonical name.

## Consequence

**Severity:** advisory

The release-gate auditor reading `H6a`'s manual smoke criterion cannot identify the four behaviours the live-host run must observe without independently discovering that they live in `H4a` under a different label ("behaviours", not "axes"). A mis-identified criterion risks the manual gate being checked against the wrong observations; the standalone "cancellation propagation" item duplicating one of the four invites further confusion about how many distinct observations the gate requires.
