## Finding

H6a's **Tests** section lists three automated `npm test` assertions (live-corpus green, removed-mapping reddens, removed-citation reddens) followed by a fourth bullet, **"Release-gate acceptance (manual real-host smoke)."** That fourth bullet is not an automated assertion — it never runs under `npm test`. It is a manual process obligation that *defines* the falsifiable evidence record the release gate requires: the Pi-SDK pin literal the run executed against, the named owner who ran the H4a pre-merge real-host smoke gate, the run date, and the observed result against H4a's narrowed model-output-invariant criterion, all recorded in this leaf's **gate-activation commit message**.

The leaf's **Ships when** clause already carries this obligation as part of the exit gate — but only by forward-reference: it says "the manual real-host smoke checklist item **above** has recorded its evidence record in this leaf's gate-activation commit message (Pi-SDK pin literal, named owner, run date, and observed result …)." So the authoritative definition sits in Tests while the exit gate that depends on it points back up at it. The definitional weight belongs at the exit gate (Ships-when), where the release-time / bump audience observes the gate; a manual record obligation parked among automated assertions both misrepresents H6a's automated coverage and splits one obligation across two sections with a fragile "above" pointer.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H6a-live-corpus-activation.md`, relocate the manual evidence-record definition out of **Tests** and into **Ships when**, then remove the forward-reference pattern:

- Strike the fourth **Tests** bullet ("Release-gate acceptance (manual real-host smoke). …"). Leave the three automated `npm test` assertions (live-corpus green; removed-mapping reddens; removed-citation reddens) as the entire Tests body.
- In **Ships when**, replace the back-reference phrase — currently "the manual real-host smoke checklist item **above** has recorded its evidence record in this leaf's gate-activation commit message (Pi-SDK pin literal, named owner, run date, and observed result against `H4a`'s narrowed model-output-invariant criterion)" — with the obligation stated directly at the exit gate: that the manual real-host smoke run (driving H7a's committed multi-feature fixture `.loom` against a live Pi host, passing H4a's narrowed model-output-invariant criterion) was executed and passed against the shipping Pi-SDK pin, and that its falsifiable evidence record — the Pi-SDK pin literal at `host-prerequisites.md#pi-sdk-pin`, the named owner who ran the H4a pre-merge real-host smoke gate, the run date, and the observed result against H4a's criterion — is recorded in this leaf's gate-activation commit message, with the release not passing until that commit message records the run.

Keep H4a as the single source of the pass/fail criterion (cross-reference, do not restate it). The "manual gate — loom 1.0 has no mechanical real-host fidelity gate" framing moves with the obligation into Ships-when.

## Consequence

**Severity:** advisory

An implementer reading H6a's Tests sees a manual process obligation interleaved with three automated `npm test` assertions and may treat the evidence-record check as automatable, or misjudge H6a's automated coverage. The release-time evidence obligation is also stated definitionally in Tests but referenced (not restated) in the Ships-when exit gate, so a reader scanning only the exit gate gets the obligation by an "above" pointer rather than in place. The obligation remains observable, so this does not block pickup or cause spec divergence.
