## Finding

The spec rule the V9q leaf pair closes, PIC-55 (`registration-steps.md#pic-55`), is labelled "Watcher post-`error`/post-throw continuation and recovery posture" — a single posture governing two distinct failure routes: the chokidar adapter's `error` route and a *synchronous throw* out of the watcher. The V9q leaf body honours this: its **Adds** opens with "The watcher post-`error`/post-throw terminal recovery posture" and its **Tests** assert "when the chokidar `error` route fires **or the watcher throws**".

Both leaf-file H1 headings, however, read "Watcher post-error terminal recovery posture" — dropping the `/post-throw` half. The same truncated text is repeated in the plan.md §V9 table-of-contents link to V9q. A reader skimming headings (or running a grep-based heading audit) sees only the `error`-route half and could conclude the leaf handles `error` events and not the distinct synchronous-throw route — a conclusion the Adds and Tests immediately contradict.

The defect is a heading/body divergence on the same leaf, not a behavioural gap: the acceptance criteria are correct and complete; only the titling is narrowed.

## Solution Space

**Shape:** single

### Recommendation

Rename the H1 heading in both leaf files and the plan.md TOC link text so the title carries the full PIC-55 posture, restoring the `/post-throw` route:

- `docs/plan_topics/V9q-watcher-terminated-recovery.md` line 1 — set the heading to `` # `V9q` — Watcher post-`error`/post-throw terminal recovery posture ``.
- `docs/plan_topics/V9q-T-watcher-terminated-recovery.md` line 1 — set the heading to `` # `V9q-T` — Watcher post-`error`/post-throw terminal recovery posture (tests) ``.
- `docs/plan.md` line 140 — set the V9q TOC link text to `` `V9q` — Watcher post-`error`/post-throw terminal recovery posture `` (leave the `./plan_topics/V9q-watcher-terminated-recovery.md` target unchanged; only the link label changes).

The leaf filenames and the `V9q`/`V9q-T` IDs stay as they are; only the human-readable heading/link text changes. The spec PIC-55 label is the authority for the corrected wording and is read-only for this fix.

## Consequence

**Severity:** cosmetic

The leaf's Adds and Tests cover both the `error`-route and synchronous-throw arms, so no implementer building from the body diverges. The narrowed heading is a misnomer that can mislead a heading-level skim or a grep audit into thinking the synchronous-throw route is out of scope, but it does not change what the leaf ships.
