## Finding

`docs/spec_topics/pi-integration-contract.md` is a 37-line index page consisting of the `# Pi Integration Contract` heading, a single intro paragraph, and a 30-entry **Contents** bullet list pointing at child files under `pi-integration-contract/` (e.g. `patch-skew-degradation.md`, `runtime-event-channel.md`, `drain-state-contract.md`, `extension-bootstrap-and-per-loom.md`, `tool-registration-lifetime.md`). It carries none of the normative rules its link text names. Inbound links throughout the corpus nevertheless point at this bare index — 20 occurrences in `docs/spec_topics/diagnostics/` alone (8 in `code-registry-runtime.md`, 5 in `diagnostic-shape.md`, 4 in `code-registry-host.md`, 2 in `code-registry-load.md`, 1 in `placeholder-rendering-b.md`), with the same pattern repeated in 32 files across the spec corpus (`glossary.md`, `cancellation.md`, `implementation-notes.md`, `discovery/*`, `binder/*`, `query/*`, `errors-and-results/*`, `frontmatter/*`, `hard-ceilings/*`, etc.) for a total of ~80 occurrences.

The link text routinely names a specific sub-rule whose definition lives in a known child file: `[Pi Integration Contract — Extension entry point](../pi-integration-contract.md)` is defined at `pi-integration-contract/extension-bootstrap-and-per-loom.md#extension-entry-point`; `[PIC's **Per-step isolation** rule](../pi-integration-contract.md)` is in `pi-integration-contract/patch-skew-degradation.md`; `[PIC's **idempotent** clause](../pi-integration-contract.md)` is in `pi-integration-contract/drain-state-contract.md#idempotent-handler-clause`; `[Pi Integration Contract — System notes](../pi-integration-contract.md)` is in `pi-integration-contract/runtime-event-channel.md`; `[Pi Integration Contract — Tool-registration lifetime and visibility](../pi-integration-contract.md)` is in `pi-integration-contract/tool-registration-lifetime.md`; `[Pi Integration Contract — Subagent session lifecycle](../pi-integration-contract.md)` is in `pi-integration-contract/subagent.md`; `[PIC — Runtime event channel](../pi-integration-contract.md)` is in `pi-integration-contract/runtime-event-channel.md`. A reader following the link arrives at the 37-line TOC and must guess which child file holds the rule whose verbatim name the link text just gave them — even though the spec already demonstrates, in adjacent links, that the precise anchored target is known.

The inconsistency is internal: the same shard frequently has the precise form next to the bare one (e.g. `code-registry-host.md` row for `session-shutdown-runtime-degraded` cites `../pi-integration-contract/patch-skew-degradation.md#interstitial-seam-tag-write-swallow` and `../pi-integration-contract/session-only-degraded-state.md#session-only-reason-degraded-state` in the same cell as four bare `../pi-integration-contract.md` references). An implementer treating the bare link as authoritative is misdirected to the index; one who guesses the wrong child file implements timing, delivery-channel, or payload-shape against a contract that does not say what they think it says.

## Issue introduction

**Verdict:** single-commit

**Introducing commits:** `f5e89f4` (2026-06-04, *docs: enforce 100KB hard / 30KB recommended size cap on spec set*).

**History:** Before `f5e89f4` the spec corpus contained a single 576 KB `docs/spec_topics/pi-integration-contract.md` that held the whole contract in one file; all cross-references targeted that file (with or without a fragment) and resolved to the normative paragraph the link text named. `f5e89f4` split the file into a 37-line index page plus 30 children under `pi-integration-contract/`, preserving every `<a id>` anchor on the child that received the originating paragraph. The split tool (`docs/_tools/split_spec.py`) recomputed link paths via a global `(target_rel, frag) → owner` map, but the map keys on `(file, fragment)` pairs — a bare `(file, None)` link with no fragment falls through to the *"unchanged target file"* arm and is only path-recomputed, never repointed. Every `[…](pi-integration-contract.md)` without a fragment therefore survived the split rewriting the path but still pointing at the (now-emptied) index; the same commit message explicitly notes "2 dangling refs remain" but does not flag the much larger class of bare links that resolve syntactically yet land on a TOC. The defect is the predictable side-effect of one commit's rewrite policy; no later commit has touched these specific links, and the corpus before the split did not have the failure mode.

## Solution Space

**Shape:** single

### Recommendation

Sweep every `](../pi-integration-contract.md)` and `](./pi-integration-contract.md)` occurrence in `docs/spec_topics/` and repoint each to the precise `pi-integration-contract/<child>.md#<anchor>` target whose normative rule the surrounding link text already names. Apply the same fix to the four spec-internal links that drop the leading `./` (`[…](pi-integration-contract.md)`). The corpus already provides the lookup table: for every distinct link text that currently points at the bare index, an adjacent occurrence in the same shard (and almost always in the same paragraph) points at the precise child anchor. Concretely:

| Link text the bare link uses | Repoint to |
|---|---|
| `Pi Integration Contract — Extension entry point` | `pi-integration-contract/extension-bootstrap-and-per-loom.md#extension-entry-point` |
| `Pi Integration Contract — Runtime event channel` (and the bare `Pi Integration Contract` form when the cited claim is the always-log / runtime-event surface) | `pi-integration-contract/runtime-event-channel.md` (or the in-file anchor the precise claim names — `#always-log-set`, `#pic-1`, etc.) |
| `Pi Integration Contract — System notes` | `pi-integration-contract/runtime-event-channel.md` (the `System notes` paragraph at the head of that file) |

| `Pi Integration Contract — Extension-bootstrap SDK failures` | `pi-integration-contract/extension-bootstrap-and-per-loom.md` (with the in-file anchor the claim names) |

For each bare link, the repointing is mechanical: take the verbatim phrase after the em-dash in the link text, find the child file whose `## ` / `### ` heading or `**…**` paragraph opener matches it, and substitute its path plus the anchor `<a id>` already present at that paragraph (every anchor `f5e89f4` preserved is still live on the child that received the source paragraph). If no anchor exists at the destination paragraph, add one inline in the destination file in the same edit (the corpus convention is `<a id="kebab-case">` at the start of the paragraph) and link to it. The edit touches inbound-link text only — no rule moves, no behavioural prose changes, no normative content is added or removed, and the targets already exist on the child pages.

Implementer-relevant edge cases:

- A small number of `pi-integration.md`-style links (the high-level extension overview, the abbreviation glossary entry for "PIC") genuinely refer to "the whole contract" and should be left pointing at the index. The discriminator is whether the link text names a specific sub-rule (repoint) or names the contract as a whole (leave).
- The `docs/_tools/check_links.py` validator will not catch this class of defect because the bare links resolve to a valid file; preventing regression requires either (a) a new audit that flags `]([./]*pi-integration-contract.md)` followed by a `—` segment in the link text whose target paragraph lives on a child file, or (b) discipline at review time. Adding the audit is out of scope for this finding but is the natural follow-up.
- The same defect shape exists for other split parents (`diagnostics.md`, `errors-and-results.md`, `binder.md`, `query.md`, `governance.md`, `hard-ceilings.md`, `frontmatter.md`, `future-considerations.md`, `discovery.md`, `spec.md`) — all 11 files split by `f5e89f4`. This finding is scoped to `pi-integration-contract.md` because that is what the original heading names; the same mechanical sweep against the other ten parents is a separate finding the reviewer should raise if it has not been raised already.

## Consequence

**Severity:** correctness

A first-time implementer reading any of the eighty-odd bare links lands on a 37-line table of contents and must guess which of the 30 child files holds the rule whose name the link text just promised them. Two reasonable implementers will land on different files, or on the index and silently give up on cross-checking, and will implement delivery channel, emission timing, or payload shape against the wrong contract. The misdirection is silent — the link resolves, the file exists, no validator catches it — and is most concentrated on the highest-stakes pages (`session_shutdown` teardown, diagnostic delivery sink, tool-registration lifetime).
