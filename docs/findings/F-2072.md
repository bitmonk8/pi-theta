## Problem

The `createRequire` clause under `id="audit-recognised-shapes"` in `audit-recognised-shapes.md` mandates that the inventory-closure audit detect "any call to a function whose source binding transitively resolves to the `createRequire` named export of the Node `"module"` package" — reaching through re-export chains, intermediate barrel modules, and cross-module captured bindings. That is a whole-program def-use / symbol-resolution problem. But `inventory-audit-intro.md` (anchor `id="sdk-capability-inventory"`) frames the audit as satisfiable by a pure static-AST walker and asserts the behavioural contract is "invariant under tooling choice." A pure static-AST walker that processes one file at a time without a program load or symbol table cannot follow a binding across module boundaries back to the `"module"`-package export, so the obligation as written is only satisfiable by the type-aware variant the spec frames as optional — contradicting the static-AST floor and the tooling-invariance claim.

The broad transitive `createRequire`/rebinding detection obligation ("any other re-binding shape … transitively resolves to the `createRequire` named export") was added to the recognised-shapes paragraph by d8d5ad0 (2026-06-20, `git log -G 'transitively resolves'` / `git show d8d5ad0` confirm the clause is new in that commit), which reconciled the Step 0(d) peer-dep probe against audit family-(4). The contradiction arises from the interaction: the newer transitive mandate was added without reconciling it against the pre-existing static-AST-sufficient claim.

## Solution approach

Narrow the `createRequire` detection obligation under `id="audit-recognised-shapes"` from transitive cross-module resolution to a within-module, intra-module def-use scope, so the clause stays statically decidable by the static-AST floor in `inventory-audit-intro.md` (`id="sdk-capability-inventory"`). Note in the clause that cross-module transport of a `createRequire` binding is reached via the separately-prohibited peer-package re-export and barrel-module shapes rather than by transitive resolution.

## Solution constraints
