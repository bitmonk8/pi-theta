## Finding

The `SchemaValidator` seam leaf pair cites the slug-cache-collision diagnostic as the bare token `validator-cache-collision`. The canonical code in the spec is `loom/runtime/validator-cache-collision` — defined in `code-registry-runtime.md` (the runtime-namespace registry row) and bound by the PIC-11 validator-cache rule in `host-interfaces-services.md`. The bare citation drops the `loom/<phase>/` namespace.

The namespace is load-bearing for this code specifically: the spec deliberately keeps `loom/runtime/validator-cache-collision` (per-query AJV cache) distinct from the load-time `loom/load/schema-slug-collision` (`$defs` hoist) and the `loom/runtime/registration-cache-collision` (tool registration) so a collision's originating cache is identifiable from the code alone. Stripping the prefix discards the load-vs-runtime phase signal the spec considers significant, and it makes V8c the only diagnostic citation in its slice that is not prefixed (`V8c`'s sibling leaves cite `loom/runtime/registry-swap-failed`, `loom/runtime/internal-error`, `loom/runtime/custom-type-unsafe`, etc., all fully namespaced).

The bare form appears three times in the leaf pair: the `PIC-11` Tests bullet in both `V8c-schema-validator-seam.md` and `V8c-T-schema-validator-seam.md`, and the **Ships when** line of `V8c-schema-validator-seam.md`.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e ("pi-loom plan: build/update plan for spec.md + review", 2026-06-10)
**History:** At the original plan build (c6a664e) the `SchemaValidator` seam was bundled into the `V8a-checkpoint-validator-seams.md` / `V8a-T-checkpoint-validator-seams.md` leaf pair, where the `PIC-11` bullet already cited the bare `validator-cache-collision` (no `loom/runtime/` prefix). The over-bundling-resolution commit cb8737a (2026-06-12) split the `SchemaValidator` seam out into the new `V8c-schema-validator-seam.md` / `V8c-T` leaf pair, carrying the bare citation forward verbatim. The missing prefix has therefore been present since the corpus's first plan build; no later edit introduced it, and the V8c split relocated rather than originated the defect.

## Solution Space

**Shape:** single

### Recommendation

Replace the bare `validator-cache-collision` with the canonical `loom/runtime/validator-cache-collision` at every occurrence in the leaf pair:

- `docs/plan_topics/V8c-schema-validator-seam.md` — the `PIC-11` Tests bullet ("…on a slug-cache byte mismatch fires `loom/runtime/validator-cache-collision`") and the **Ships when** line ("…and a `loom/runtime/validator-cache-collision` on a slug-cache byte mismatch").
- `docs/plan_topics/V8c-T-schema-validator-seam.md` — the `PIC-11` Tests bullet, identically.

The canonical token is fixed by `code-registry-runtime.md` and PIC-11 in `host-interfaces-services.md`; no spec edit is required.

## Consequence

**Severity:** correctness

Two implementers writing the PIC-11 assertion can diverge: one asserts the runtime's actual emitted code `loom/runtime/validator-cache-collision`, the other the bare `validator-cache-collision` literal as written, producing a test that never matches the emitted diagnostic. The dropped namespace also erases the load-vs-runtime phase distinction the spec relies on to keep the three collision codes separable.
