## Problem

The phase-2 enumeration in the `**Conversation drive — prompt mode.**` typed-query bullet defines the *forced respond turn* as firing only "once the model emits a plain text turn" — i.e. after a free-phase plain-text turn. Read in isolation, that enumeration makes the forced respond turn unreachable at `max_rounds: 0` (where no free-phase turn is ever issued) and on CIO-4's `max_rounds`-final branch, which dispatches the forced respond turn at `slot_count == max_rounds` regardless of any plain-text turn. The same bullet's follow-up "exempt-routed terminator" sentence does pin both entry points, so the bullet is internally consistent — but an implementer who treats the enumeration as the operative trigger contract and the follow-up sentence as commentary will skip the `max_rounds: 0` and `max_rounds`-final dispatches.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `82be125` (2026-05-04, "spec: adopt P1 typed-query two-phase loop (Option A)") established the two-phase enumeration with the phase-2 trigger pinned solely as "Once the model emits a plain text turn". `22619b0` (2026-05-25, "pi-loom spec: resolve \"T11d / T11a — forced-respond exempt-routed terminator\"") added the CIO-4 `max_rounds`-final / `max_rounds: 0` entry point to the same bullet, as a follow-up sentence after the enumeration, without harmonising the phase-2 trigger clause itself.
**History:** `git log -S "two-phase tool-loop conversation"` returns `82be125`, `f5e89f4` (the later `Move spec, plan, and review docs into docs/` consolidation `31ff060` and the size-cap edit `f5e89f4` only relocated the text), and `11ee94f` (re-review tooling, no normative change). `git show 22619b0 -- docs/spec_topics/pi-integration-contract.md` shows the phase-2 trigger clause untouched and the second-trigger language appended as a separate sentence: `"The forced respond turn is the exempt-routed terminator dispatched by CIO-4's max_rounds-final branch … including max_rounds: 0 where it is the only turn"`. The phrasing trap is therefore the residue of a two-step evolution: a single-trigger enumeration authored before the CIO-4 entry point existed, then a second entry point grafted on without rewriting the enumeration.

## Solution approach

Rewrite the phase-2 trigger clause in the typed-query bullet so the enumeration itself names both entry points — the free-phase plain-text turn (provider stop reason `end_turn` / `stop`) and CIO-4's `max_rounds`-final branch, degenerating to the only turn at `max_rounds: 0`. Demote the existing "exempt-routed terminator" follow-up sentence to a pure cross-reference carrying the facts it uniquely holds: the CIO-4 link and that the turn is not debited against `tool_loop.max_rounds`.
