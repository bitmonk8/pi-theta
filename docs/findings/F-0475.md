## Finding

`H4b` defines the shared response-programming surface as seven scripting categories (a)–(g). Categories (a)–(e) script behaviours that are fully verifiable against `H4a`'s session-double fidelity contract at `H4b` build time (assistant turns and streamed fragments, `tool_use` results including `isError`/mixed-success batches, binder/provider responses and failures, `tool_loop.max_rounds` round-exhaustion, abort/cancellation injection). Categories (f) and (g) are different in nature: (f) models the `V4f` ERR-13 completed-invoke-child vector and (g) models the `V9i` subagent-mode session contract (the `V4c` ERR-12 subagent-loom vector) — both forward-modelling slices that are not authored when `H4b` runs (`H4b`'s `Deps` are `H4a` only).

`H4b`'s self-checks can only confirm the (f)/(g) models are internally consistent and deterministic, and that each scripted input yields its own scripted observable. They cannot confirm the model matches what `V4f`/`V9i` actually need, because the authoritative behaviour (`error-model.md` §No rollback completed-callee finality; `subagent.md` / `return.md` final-value and lifecycle contract) is transcribed into the harness rather than asserted against the spec. The plan resolves the forward-reference by declaring the harness contract authoritative ("the owning slices consume this surface rather than redefining it"), which fixes *naming ownership* but not *fidelity*: nothing reddens if the (f)/(g) model diverges from the spec vector it claims to model.

The surface is consumed by ten downstream leaves (`H7a`, `V4c`, `V4f`, `V9i`, `V11f`, `V13c`, `V13d`, `V13f`, `V14b`, `V17a`). A wrong (f)/(g) model is therefore discovered only when `V4f`/`V9i`/`V4c` are authored against it, by which point the surface is load-bearing for the whole harness-driven cohort and a correction propagates as rework.

## Solution Space

**Shape:** multiple

### Option A — Pin the (f)/(g) self-checks to the spec vectors they model

**Approach.** Keep `H4b` as the single authoritative harness contract — a harness-first design requires the scripting surface to exist before the slices that script through it, so forward-modelling of (f)/(g) is inherent and cannot be ordered away. Close the validation gap by making the (f)/(g) functional-effect self-checks assert conformance to the authoritative spec vectors, not merely internal determinism/consistency.

**Plan edits.** In `H4b`'s second `Tests.` bullet, strengthen the (f) assertion so the scripted completed-invoke-child outcome conforms to the `error-model.md` §No rollback completed-callee-finality vector (the produced final value persists; no compensating turn), and the (g) assertion so the scripted subagent-loom outcome conforms to the `subagent.md` final-value/isolation and `return.md` final-value contract. Reference those spec pages in `H4b` so the model is checked against the spec surface that `V4f`/`V9i` will close.

**Spec edits.** None (spec pages referenced read-only).

**Pros.** No dependency cycle; preserves the required harness-first ordering; closes the actual fidelity gap at the owning leaf where it is cheapest to catch.

**Cons.** `H4b` still carries all seven categories (the bundling itself is unchanged); fidelity is only as good as the transcribed spec vectors.

**Risks.** Importing too much of `V4f`/`V9i`'s internal contract into `H4b` couples the harness to slice-owned detail; scope the assertions to the observable outcome each vector requires.

### Option B — Split (f)/(g) into a new leaf `H4c`

**Approach.** Separate the verifiable-now (a)–(e) from the forward-modelled (f)/(g) by relocating (f)/(g) into a new leaf, isolating the blast radius of a wrong forward-model. `H4c`'s `Deps` must stay `H4a` only — it cannot depend on `V4f-T`/`V9i-T`, because those tests-tasks script through the (f)/(g) points and would then depend on `H4c`, forming a cycle (`H4c` → `V4f-T`/`V9i-T` → `H4c`).

**Plan edits.** Add `docs/plan_topics/H4c-...md` carrying categories (f)/(g) with `Deps` `H4a`; strike (f)/(g) from `H4b`; retarget the `Deps` of the (f)/(g) consumers (`V4f`, `V4f-T`, `V9i`, `V9i-T`, `V4c`, `V4c-T`) to cite `H4c`; link `H4c` into `docs/plan.md` Horizontal phases; update `coverage-matrix.md` citations that reference `H4b` for the `V4f`/`V9i` vectors.

**Spec edits.** None.

**Pros.** Cleaner atomicity; a wrong (f)/(g) model is confined to `H4c`'s consumers; (a)–(e) ship independently.

**Cons.** Does not close the validation gap — `H4c` still forward-models the same contracts; the source finding's "codify the contracts before modelling" intent is unachievable (the cycle blocks it); large propagation diff across ≥6 leaves plus the coverage matrix; this is a split-plus-retarget (bimodal) change whose breadth risks fix-loop instability.

**Risks.** A naive implementer adds `V4f-T`/`V9i-T` to `H4c`'s `Deps` and reintroduces the cycle; a mis-retargeted consumer loses its (f)/(g) source.

### Recommendation

Take **Option A**. The forward-modelling is inherent to a harness-first design — the scripting surface must exist before the slices that consume it, so a split cannot codify the slice contracts first and only relocates the same forward-model while adding a broad propagation diff and a cycle hazard. Option A closes the operative defect (the (f)/(g) self-checks cannot detect divergence from the spec) at the owning leaf without restructuring the dependency graph. Watch one edge case: scope the new (f)/(g) spec-conformance assertions to the observable outcome each vector requires, so `H4b` does not absorb internal contract detail that `V4f`/`V9i` rightfully own.

## Consequence

**Severity:** correctness

Two reasonable implementers can model the (f) completed-invoke-child and (g) subagent-loom scripting points differently — and neither divergence reddens any `H4b` gate, because the self-checks test only internal consistency against the transcribed model, not conformance to `error-model.md`/`subagent.md`/`return.md`. A model that does not match what `V4f`/`V9i` need surfaces only when those slices are authored, forcing rework through the surface that ten harness-driven leaves already depend on.
