## Finding

GOV-30 (aggregator lock-step convention) and GOV-31 (integer-count literal-preservation invariant) are filed in `governance/req-id-prefix-table-active-b.md`, whose every other rule governs the REQ-ID prefix table; the two aggregator rules are arm-(b) `spec.md`-maintenance invariants unrelated to the prefix → page binding and are undiscoverable from the aggregator-editing site.

## Issue introduction

**Verdict:** present-since-inception (aggregator misfiling) / multi-commit-interaction (co-edit cross-reference staling)
**Introducing commits:** `126ff8a` (2026-05-06, "Step 0 capability probe rewrite + V1.0 deferrals + Choice B"); `f5e89f4` (2026-06-04, "docs: enforce 100KB hard / 30KB recommended size cap on spec set"); `fe8aa5c` (2026-06-05, "resolve GOV-12 bundles ≥3 independently-testable sub-obligations")
**History:** The corpus is git-tracked. `git log -S` shows the aggregator rule was born as GOV-12 in `126ff8a`, authored inside the monolithic `spec_topics/governance.md` that also held the REQ-ID prefix table — the aggregator rule has been co-resident with prefix-table material since inception and was never given its own home. `f5e89f4` split that governance file under the size cap, landing the rule in the page literally named `req-id-prefix-table-active-b.md`, which sharpened the mismatch; `fe8aa5c` then split GOV-12 into GOV-30/GOV-31 in place, inheriting the location. For the co-edit obligation, `git log -S 'Five-site list co-edit obligation'` and `git grep` against `f5e89f4^` show the obligation, the ceiling-#4 table, and CIO-3 all lived together in the monolithic `hard-ceilings.md`, where "above (in this page)" was correct. The same `f5e89f4` size-cap split severed them — the obligation went to `ceiling-invariants-and-audit.md`, the table and CIO-3 to `ceilings-3-and-4.md` — without repointing the now-stale intra-page cross-references. Both vectors thus trace through the `f5e89f4` mechanical file split; neither was a deliberate placement decision.

## Solution Space

**Shape:** single

### Recommendation

Move the GOV-30 and GOV-31 paragraphs (with their `<a id="gov-30">` / `<a id="gov-31">` anchors intact) out of `req-id-prefix-table-active-b.md` into `governance/corpus-direction-and-scope.md` (which already hosts the arm-(a)/(b) classification GOV-30/31 depend on), or into a dedicated aggregator-governance page under `governance/`. Update any inbound cross-references that point at `req-id-prefix-table-active-b.md#gov-30` / `#gov-31` to the new path (e.g. the GOV-30 citation in `hard-ceilings/ceiling-invariants-and-audit.md` and `governance.md`'s hub links). The `GOV` prefix → `governance/` subtree binding (GOV-24) is unaffected because the subtree-named row covers every file under `governance/`.

## Consequence

**Severity:** advisory

A future editor changing a `spec.md` aggregator, or adding/removing/splitting a ceiling-#4 table row, can complete the edit without ever encountering the obligation that binds it, because the rule lives on an unrelated page (GOV-30/31) or names its referents as "in this page" when they are on another page (co-edit obligation). The result is silent lock-step drift — exactly the failure these editorial-convention MUSTs exist to prevent — caught, if at all, only by chance during unrelated review.
