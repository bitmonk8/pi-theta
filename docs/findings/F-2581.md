## Finding

In the **Final-value propagation across callees** block of `invocation.md` (`#final-value-propagation`), the prompt-mode sentence reads:

> A `prompt`-mode child attaches to the caller's current conversation, but the final value still propagates through the same return surface.

"the same return surface" has no resolvable antecedent. Nothing earlier in the paragraph names a "return surface" the prompt-mode path could be the *same* as: the preceding sentence states only that a final value flows to the `invoke` parent on success, and the *following* sentence (subagent-mode) is the one that actually names the mechanism — "the final value crosses the subagent boundary as the `Ok` payload." A reader hitting the prompt-mode sentence first cannot tell whether "same" means the same surface as a default (non-`prompt`, non-`subagent`) invoke, the same `Ok` payload the subagent sentence later describes, or the same conversation the clause just contrasted against.

The intended meaning is recoverable from the whole paragraph — across all three modes the callee's final value reaches the parent identically, as the `Ok` payload of the `invoke` result, and the only per-mode difference is conversation attachment — but the sentence relies on the reader having already read the subagent sentence to back-fill the referent.

## Solution Space

**Shape:** single

### Recommendation

Make the comparison explicit by naming the return surface and stating it is identical across modes. Rewrite the prompt-mode sentence so the referent does not depend on the later subagent sentence — e.g.:

> A `prompt`-mode child attaches to the caller's current conversation, yet the final value reaches the `invoke` parent through the same return surface as any other mode — the `Ok` payload of the `invoke` result. Conversation attachment is the only per-mode difference; the value-return path does not vary.

Equivalently, hoist a single lead sentence stating that in every mode the final value is returned as the `Ok` payload, then describe each mode's conversation handling as the sole variation. Either phrasing removes the dangling "same" by giving it a named antecedent (the `Ok`-payload return surface) before the per-mode contrasts.

## Consequence

**Severity:** cosmetic

A reader must scan ahead to the subagent sentence to resolve what "same" refers to; the behaviour itself (final value reaches the parent as the `Ok` payload regardless of conversation mode) is unambiguously specified by the surrounding sentences, so no implementer would diverge on behaviour. Pure reading-friction in normative prose.
