## Finding

- **Description:** The `file:` plugin specifier and layout-dependent V18a/V18b/V18c reads assume npm's flat/hoisted linker; a pnpm/yarn-PnP checkout resolves differently and could fail or mis-resolve the V18c live read. No detection or fallback when the assumption is violated.
- **Suggested fix:** Add a `preinstall`/`engines` guard (or architectural test) asserting npm + hoisted layout, failing loudly at H1a rather than producing a misleading downstream resolution.
