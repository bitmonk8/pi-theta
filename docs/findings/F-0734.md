## Finding

DISC-5 says: "When no plain (un-prefixed) include pattern is present, the starting set is every path under the package root; otherwise the starting set is the union of the paths matched by the plain include patterns." The phrase "every path under the package root" does not say whether the enumeration is recursive (every transitive descendant) or flat (only the root's immediate children). The rest of the spec is uniformly non-recursive — the global `.pi/looms/` / `~/.pi/agent/looms/` walks, the `looms/` fallback, and a directory-shaped match inside DISC-5 itself are each explicitly scanned **non-recursively** — so the absence of any qualifier reads as a deliberate departure, but the paragraph does not state which direction.

The two readings produce materially different registered sets for a nested package. Under recursion, a package shipping `subdir/foo.loom` with `pi.looms: ["!subdir/**"]` has `subdir/foo.loom` enter the starting set and then be excluded; under the flat reading the same exclude is vacuous because the file was never in the set. The starting set is the input to the `!` / `+` / `-` override pipeline; DISC-5 pins the four-step order but never pins what the steps act *over*, and a `**/*.loom` plain include only meaningfully selects nested files if the enumeration is itself recursive.

The Pi reference symbol DISC-5 cites (`applyPatterns` in `dist/core/package-manager.js`) operates on a `collectFilesFromPaths` output that walks each directory **recursively**. If "parity with Pi" is the intent, the recursive reading is implied — but DISC-5 does not name parity here, and recursion contradicts the corpus-wide non-recursion stance, so an implementer cannot tell which premise wins. The defect silently propagates to `loomPaths`, whose entry schema delegates its glob contract back to DISC-5.

## Solution Space

**Shape:** multiple

### Option A — Flat (match the corpus-wide non-recursion stance)

**Approach.** Pin the no-plain-includes default to the package root's immediate children only, mirroring the non-recursive walks already specified for `.pi/looms/`, `~/.pi/agent/looms/`, and the `looms/` fallback. A `**/*.loom` plain include is then the only way to register `.loom` files in nested subdirectories.

**Spec edits.** Replace "the starting set is every path under the package root" with "every immediate child of the package root (the package root scanned non-recursively, matching the global non-recursion rule at the top of this file)". Add: "Patterns such as `**/*.loom` reach nested files by being authored as plain include patterns, which take their matches from a recursive enumeration scoped by the pattern itself rather than from the default starting set."

**Pros.** Internally consistent with every other walk; with-vs-without-manifest behaviours stay symmetric (the `looms/` fallback is also non-recursive); an author cannot accidentally surface deep `.loom` files by writing `pi.looms: ["!secret.loom"]`.

**Cons.** Diverges from Pi's `applyPatterns` recursive default; packages relying on Pi-like recursion (none yet exist — loom 1.0 unreleased) would need an explicit `**/*.loom`.

**Risks.** Future "behavioural parity with Pi" claims in DISC-5 become harder to make truthfully.

### Option B — Recursive (match Pi's `applyPatterns` reference)

**Approach.** Pin the no-plain-includes default to a recursive enumeration of every descendant under the package root, matching Pi's `collectFilesFromPaths` → `collectResourceFiles` → `collectFiles` chain DISC-5 already cites.

**Spec edits.** Replace "every path under the package root" with "every path under the package root, enumerated recursively across all subdirectories (departing from the corpus-wide non-recursion rule because DISC-5 inherits its starting-set semantics from Pi's `applyPatterns`; the non-recursion rule still applies to the per-match contribution step and to the `looms/` fallback path)". The recursive enumeration MUST honour the `lstat` / symlink rule pinned for root-level child classification (PIC-13) so a symlinked subdirectory does not leak out-of-package paths.

**Pros.** Preserves behavioural parity with Pi's resource-array dialect, which DISC-5 already names as its reference; override patterns work uniformly regardless of nesting depth.

**Cons.** The discovery walk's depth differs from every other walk, requiring a callout each time the non-recursion rule is invoked; the DISC-6 bounded-walk caps were framed against `package.json` reads, not `.loom` enumerations.

**Risks.** Authors of small packages may accidentally register `.loom` files buried in nested test fixtures; the `!` / `-` machinery is the only escape hatch.

### Recommendation

This finding consolidates two reviewer findings that disagreed on direction — one preferring **flat** (Option A) for corpus consistency, the other preferring **recursive** (Option B) for Pi parity — so the fork is genuinely contested and both options are retained for resolution.

The corpus-consistency case for **Option A** is the stronger: every other walk is explicitly non-recursive, the `looms/` fallback is non-recursive (keeping with-vs-without-manifest symmetric), and authors wanting deep coverage can write `pi.looms: ["**/*.loom"]`, the explicit audit-able opt-in. The Pi-parity argument is the only force toward recursive, and DISC-5 already hedges its Pi reference as "pins loom's *use* of that matching, not minimatch's glob dialect", which legitimises diverging on the starting-set shape. Whichever is chosen, state the depth once in the existing sentence and note the `pi.looms`-vs-`looms/` asymmetry in a parenthetical.

Edge case (Option A): a `pi.looms` entry like `["!secret.loom"]` (override-only) produces an *empty* starting set and registers no looms — the same observable as a package shipping no `pi.looms` field and no `looms/` directory; "no `.loom` files registered" is silent today and should remain silent.

## Consequence

**Severity:** correctness

Two reasonable implementers diverge on every package that ships `.loom` files outside its root's immediate level. The `!`/`+`/`-` override machinery becomes partially inert under the flat reading and fully active under the recursive one, and the parity-with-Pi reading directly contradicts the surrounding non-recursive ethos, so the choice is non-obvious. The defect also silently propagates to `loomPaths`, whose entry schema delegates its glob contract back to DISC-5.
