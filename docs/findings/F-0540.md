## Finding

In the `## Vertical slices` → `### V9` list in `docs/plan.md`, `V9n` (Prompt-mode transport-error mapping) is listed immediately after `V9c`, near the top of the slice. Its actual `Deps` (`docs/plan_topics/V9n-transport-error-mapping.md`: `V9n-T`, `V9c`, `V9j`, `V17a`) make it buildable only after `V9j`, which in turn depends on `V11a` (`docs/plan_topics/V9j-binder-inference-provider-mapping.md`: `V9j-T`, `V4d`, `V11a`). So `V9n`'s true build position is after `V9j`/`V11a` — at the tail of the V9 build seam — not adjacent to `V9c`, which itself only depends on `V9a`/`V8a`/`V17a`.

The plan supplies interleave notes for its other out-of-band V9 build positions: the V9 interleave blockquote documents the `V9b → V11a → V9c`/`V9i`/`V9j` seam and the `V18c` deferrals for `V9g`/`V9h`/`V9l`. `V9n`'s deferred position receives no such note. The build order is still correct under the global "sequence by `Deps`, not slice number" rule the plan states repeatedly, so nothing builds out of order; the gap is that the index clustering misrepresents `V9n`'s build position and is inconsistent with the interleave-note treatment given to the other deferred V9 leaves.

The misrepresentation has existed since `V9n` first entered the index; no later commit added the missing note.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan.md`, extend the V9 slice interleave note (the `> **Interleave note.**` blockquote above the V9 leaf list) with a sentence recording `V9n`'s true build position: `V9n` depends on `V9j` (and therefore transitively on `V11a`), so it lands at the tail of the V9 seam after `V9j`, not adjacent to `V9c` where it is listed — sequence by `Deps`, not slice number. This mirrors the note's existing treatment of the `V11a` and `V18c`-deferred leaves and keeps `V9n` editorially clustered with the `V9c` conversation-drive material it extends. No `Deps` edge changes; the leaf files are already correct.

## Consequence

**Severity:** advisory

An implementer reading the V9 list top-to-bottom would attempt `V9n` right after `V9c`, find its `Deps` (`V9j`, and transitively `V11a`) unsatisfied, and have to defer it — minor friction. No leaf builds out of order (the canonical order is `Deps`-driven, as the plan states), but the omission is inconsistent with the interleave notes the plan provides for the `V11a`/`V18c` deferrals in the same slice.
