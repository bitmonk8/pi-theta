## Problem

The `CNCL-4` (session-shutdown synthesised-reason facet) Tests bullet, present verbatim in both `V9g-T-session-shutdown.md` and `V9g-session-shutdown.md`, ends "assert **this** is the observed `loomAbort.signal.reason` at a downstream checkpoint." The antecedent of "this" can bind either to the synthesised `Error` object or to its byte-exact `message` string, and the two readings drive incompatible assertions and production shapes (`signal.reason instanceof Error` vs `signal.reason === "loom cancelled by session shutdown"`). The spec resolves this unambiguously: `cancellation.md` CNCL-4 pins `loomAbort.signal.reason` to the synthesised `Error` object passed to `loomAbort.abort(reason)`. A test author working from the ambiguous bullet can encode the string-equality form, which passes against a non-conforming `abort("…")` production shape and silently diverges from the spec contract.

## Issue introduction

**Verdict:** introduced-by-later-edit
**Introducing commits:** `4dde482` (2026-06-10, "pi-loom plan-review: park T27 …") added the bullet to `V9g-session-shutdown.md`; `9081369` (2026-06-10, "pi-loom plan: resolve \"CNCL-4 session-shutdown reason facet not authored red or gated\"") added the identical bullet to `V9g-T-session-shutdown.md`.
**History:** `git log -S "assert this is the observed"` over both leaf files attributes the exact ambiguous phrasing to these two commits; the bullet did not exist at the plan's `c6a664e` inception. The defect entered as a side effect of authoring the `CNCL-4` reason-facet Tests bullet during a prior plan-review fix pass — the bullet correctly pinned the synthesised `Error` and message but introduced the "assert this is …" pronoun whose antecedent was never disambiguated in the Tests context.

## Solution approach

Rewrite the ambiguous "assert this is …" clause in the `CNCL-4` Tests bullet of both `V9g-T-session-shutdown.md` and `V9g-session-shutdown.md` so the `Error` object — not the bare message string — is named as the subject of the `loomAbort.signal.reason` assertion, naming both observable facets (`instanceof Error` and the byte-exact `message`). Align the assertion with `cancellation.md` CNCL-4's `signal.reason === source.reason` contract. Apply the identical observable wording to `V9g`'s Ships-when clause so the Tests bullet and Ships-when state the same thing.
