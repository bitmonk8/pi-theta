## Problem

`spec_topics/pi-integration-contract/runtime-event-channel.md` carries normative MUST/MUST-NOT obligations that bear neither a numbered `PIC-N` anchor nor a `loom/...` registry code (the renderer `content === ""` tolerance, the no-`await`/no-`.catch` emission discipline, the two fallback re-entry guards, the panic-message-in-final-`console.error` MUST, the Group-B `console.error`-sink exclusion-list co-edit MUST, the `details`-shape disjointness MUST, and the tool-host non-cross-correlation MUST). None of these are enumerated in `coverage-matrix.md`'s *Code-keyed obligation areas (no numbered REQ-IDs)* table — the page has no `cka-<n>` row and no row records a closing leaf for them. The page is non-narrative, so these un-anchored MUSTs sit inside the `H5e` un-enumerated-MUST gate surface. At the `H6a` live-corpus footing flip the `H5e` arm reddens against each MUST and `H6a` cannot go green, even though the behaviours are implemented by `V9d` (channel shape, group-A/B routing) and `V7d` (delivery envelope, renderer registration, fallback chain).
