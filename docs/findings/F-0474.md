## Problem

`H4a`'s Adds field defines an in-memory fixture-supply mechanism whose constraints are pinned (in-memory, no ambient `src/**` read, no `FileSystem` seam) but whose integration point is not: the plan never states how the factory's discovery receives the harness-provided `.loom` content, nor whether doing so requires a collaborator outside the `H3a` runtime seam set. Production discovery is filesystem-backed (the walk deepened by `V10a` reads through the `FileSystem` seam, `V8b`), while `H4a` forbids the harness path from using that seam, so an implementer must invent both a source-provider collaborator and a harness-vs-production selection rule. Two reasonable implementers diverge — one adds a new injected seam to the `H3a` graph (changing the ambient-access scan surface), another bolts an ad-hoc parameter onto the discovery entry point, a third trips the `H3a` ambient-access scan that the "clean scan" outcome exists to avoid. `M` and `M-T` are the immediate consumers, so the ambiguity propagates into the narrowest end-to-end vertical.

The resolution pinned the mechanism's *constraints* (in-memory, no ambient read, no `FileSystem` seam) but did not pin its *integration contract* — leaving the injection-point gap this finding records. The relevant sentence has not been reworded since (no later commit appears in the `-S` history), so the gap has been present in its current form since `2d52ad1`.

## Solution approach

In `H4a`'s Adds field, pin the injection contract for the in-memory fixture-supply mechanism: state that the factory's discovery obtains its `.loom` source through an injected source-provider collaborator it consumes, rather than by reading the `FileSystem` seam or a host primitive directly. Pin that this collaborator is a discovery-layer collaborator distinct from the `H3a` host-seam set (so the `H3a` seam graph and its ambient-access scan are unchanged — the clean-scan property becomes a construction, not an aspiration), and that production binds it to a `FileSystem`-backed implementation while the harness binds it to the in-memory implementation, with selection made at the factory's construction site rather than by a runtime branch inside discovery. Confirm the "harness-supplied source" wording in `M`'s and `M-T`'s Adds references this same injected collaborator.
