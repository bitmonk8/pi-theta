## Finding

The Grammar Appendix (`grammar.md`) is declared normative only for "the few surface-syntax forms that no single topic page owns" plus the literal sublanguage. Its productions therefore lean on nonterminals whose definitions live on other pages, but several of those nonterminals are used with no inline "defined in `<page>`" pointer:

- **`Stmt`** тАФ used in `BlockExpr ::= "{" Stmt* Expr "}"`, `FnBody`, `LoomBody`, and `StmtBlock`. `Stmt` is never defined as a production anywhere in the corpus, and the appendix gives no pointer to where the statement forms are enumerated (the set is scattered: `let` on this page, assignment/`break`/`continue` under Bindings and Control Flow, the statement-form `if`/`while`/`for` under Control Flow, and expression-statements under the Expression Sublanguage). A reader cannot determine the membership of `Stmt*` from the appendix alone.
- **`Expr`** тАФ used pervasively (block expressions, `let`, the control-flow heads, `MatchArm`) with no pointer to the Expression Sublanguage page that owns the expression grammar.
- **`EnumDecl`** тАФ referenced in the `///` placement list ("`EnumDecl` (the entire enum declaration)") with no pointer to `schemas.md#enum-declarations`, where the enum grammar is owned.

The two other nonterminals named in the original heading already carry pointers and are not part of the defect: `Pattern` is cross-referenced to `expressions.md#pattern-grammar` in the `let` form section, and `Field` carries the inline note "Field per Schema Declarations" plus a link to `schemas.md#object-schema`. The delegation pattern is intentional and correct; the gap is that it is applied unevenly тАФ some externally-owned nonterminals get a pointer and some do not.

The literal-position dependency raised in the heading's second clause is weaker than the nonterminal gap: the appendix intro already points the `params:`-default position at `frontmatter.md` and the Pi-tool-argument position at `tool-calls.md`. The one place that could still use a local anchor is the **Position rules** paragraph, where "a Pi-tool call argument supplies it via the Pi tool's registered input schema" repeats the dependency without re-citing the owning page.

The same window and subsequent edits *did* selectively add pointers for the sibling nonterminals тАФ `Field` ("Field per Schema Declarations") at `232a020` (2026, "Type grammar omits inline anonymous object types") and `Pattern` (`expressions.md#pattern-grammar`) reaffirmed at `87dfa1d` тАФ which is why the delegation is uneven today. The defect is thus best characterised as present since the appendix was authored and accreted as each new production reused the undefined/un-pointed nonterminals.

## Solution Space

**Shape:** single

### Recommendation

Add a one-line "defined in `<page>`" pointer at each externally-owned nonterminal use in the Grammar Appendix, matching the convention already applied to `Pattern` and `Field`:

- For `Expr`: add a single note (e.g. alongside the Notation paragraph, where `Ident`/`STRING`/тАж are already delegated to Lexical Structure) stating that `Expr` is the full expression grammar owned by the [Expression Sublanguage](./expressions.md). One note covers every `Expr` use in the appendix.
- For `Stmt`: add an inline enumeration at the Block-expressions section noting that `Stmt` is any Loom statement тАФ `let` (this page), assignment / `break` / `continue` (see [Bindings](./bindings.md)), the statement-form `if` / `while` / `for` (see [Control Flow](./control-flow.md)), and expression-statements (see [Expression Sublanguage](./expressions.md)). A pointer alone is insufficient because no single page owns `Stmt`; the enumeration is the definition.
- For `EnumDecl`: in the `///` placement list, link "`EnumDecl` (the entire enum declaration)" to [Schema Declarations тАФ Enum declarations](./schemas.md#enum-declarations).
- For the Pi-tool input-schema dependency: in the **Position rules** paragraph, re-cite [Tool Calls тАФ Argument shape](./tool-calls.md) at the "Pi tool's registered input schema" clause so the dependency is anchored locally and not only in the section intro.

Leave the existing `Pattern` and `Field` pointers as-is. The edit is prose-only; do not alter any production, diagnostic, or semantics.

## Consequence

**Severity:** advisory

An implementer reading the appendix in isolation cannot resolve the membership of `Stmt*` or the grammar of `Expr`/`EnumDecl` without already knowing which sibling page owns each one; this is a navigation/guidance gap that slows comprehension and invites guessing about whether `Stmt` has a normative definition (it has none). It does not by itself cause two implementers to diverge on parser behaviour, because the forms are recoverable by cross-page reading.
