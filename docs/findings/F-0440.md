## Problem

In `docs/plan_topics/conventions.md`, the *Sequential by default* cross-cutting rule permits a banned `Promise` combinator in `src/**` only when the calling leaf's `Adds.` field "names the construct together with that REQ-ID or code-keyed obligation area." The phrase "names the construct" is unqualified: it does not say whether `Adds.` must contain the literal combinator token (e.g. `Promise.allSettled`) or may merely describe the concurrent behaviour in prose. The ambiguity is sharpened because the §Leaf format **Adds.** definition explicitly makes the neighbouring Class-2 seam naming test token-free, leaving the reader unsure whether the Class-1 concurrency construct here requires the token. Every existing combinator-using leaf (`V14b`, `V9g`, `V9i`) already names the literal token, so the de facto convention is token-naming, but the rule text does not state it.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `861e935` (2026-05-05, "pi-loom plan: resolve \"Promise.all/race exemption rule has no operational definition\"") — first authored the `Adds.`-names-the-construct requirement as "names the construct and the REQ-ID together"; re-established verbatim at `15f69aa` (2026-05-26, "finish scaffold/template re-pivot") after the intervening plan reset.
**History:** The unqualified word "construct" has been the operative wording of this requirement since it was first authored at `861e935`. The example it shipped with ("fans tool-use blocks out via `Promise.all` per QUERY-N") shows the intent was to name the literal combinator, but the normative phrase never tied "construct" to the token. The wording survived the plan reset/re-pivot (`657ee76` → `15f69aa`) intact and the carve-out rework at `75b6a9b` (2026-06-10), which broadened the carve-out to code-keyed obligation areas but kept "names the construct" unqualified. No later commit introduced the ambiguity; it is original to the rule.

## Solution approach

In `docs/plan_topics/conventions.md`, rewrite the *Sequential by default* clause "names the construct together with that REQ-ID or code-keyed obligation area" so it requires `Adds.` to name the specific banned `Promise` combinator token used at the site (one of `Promise.all` / `Promise.race` / `Promise.allSettled` / `Promise.any`) alongside the mandating REQ-ID or `cka-<n>` token. Align the §Leaf format **Adds.** definition's Class-1 parenthetical to the same token-naming wording so the cross-reference does not reintroduce the unqualified "construct"; the *Sequential by default* rule is the authoritative home and the **Adds.** definition only points at it, so both must agree.
