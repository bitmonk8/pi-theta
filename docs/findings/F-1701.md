## Problem

The step-2 editorial-review checklist in `version-bump-step2.md` splits its audit items into a MUST tier (items (a)–(e)) and a SHOULD tier (items (f)–(aq)). Within the SHOULD tier every item is a flat peer with identical escalation boilerplate, and the recording convention permits a contributor to record `not-performed` on any item with a one-line rationale. The SHOULD items span a wide stakes range — some guard correctness/privacy-adjacent guarantees (e.g. (i) overflow-signature wording, (aj) in-memory-transcript non-retention, (ac)/(ak)/(al) prompt-mode ordering preconditions) while others are observational drift-detectors — but the tier surfaces no signal of that spread, so a time-constrained contributor skipping a subset has no basis for distinguishing the high-stakes items from the conformance-only ones.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** 79fa3b9 — pi-loom spec: resolve "Binder model / model: resolution — registry-population timing" (2026-06-04, Thomas Andersen); 803106d — pi-loom spec: resolve "Pi behavioural presuppositions lack authoritative behavioural pointers" (2026-06-07, Thomas Andersen); db69f2b — pi-loom spec-plan: fix F-0981 — promote loom-flag-namespace to host-prerequisite presupposition + SHOULD-level bump checklist item (ao) (2026-06-22, Thomas Andersen)
**History:** The checklist was built incrementally: 79fa3b9 first introduced the "SHOULD re-confirm … via editorial review" boilerplate when checklist item (k) was coined; 803106d added the authoritative-source / known-fragile-evidence framing across the existing presupposition paragraphs; subsequent commits (db69f2b is the most recent example, adding (ao)) extended the list to its current ~42 items, each new item reusing the same uniform-SHOULD shape. No commit ever introduced a within-SHOULD tier or audit-priority annotation; the MUST(a–e) / SHOULD(f–ap) cut is the only structural distinction the list has ever carried. The defect — uniform weighting across the SHOULD subset — is therefore not localised to any single commit but emerged as each addition preserved the homogeneous shape.

## Solution approach

Add a sentence to the step-2 editorial-review checklist preamble in `version-bump-step2.md`, alongside the existing MUST/SHOULD distinction, that names the correctness/privacy-adjacent SHOULD items — (i), (ac), (aj), (ak), (al) — a time-constrained contributor should prioritize when skipping any item, citing them by their stable letters. Extend the existing same-edit obligation governing new checklist items so any future SHOULD item guarding a correctness- or privacy-adjacent guarantee is added to that callout in the same commit that introduces it.

## Solution constraints

- Out of scope: changing existing item letters or anchors — the (f)–(aq) items are cross-referenced by letter elsewhere, so reordering or regrouping them breaks those references.
