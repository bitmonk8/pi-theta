## Finding

The step-2 editorial-review checklist in `version-bump-step2.md` partitions its 43 audit items into a MUST tier (items (a)–(e), whose detection routes to mandatory editorial review) and a SHOULD tier (items (f)–(aq), 38 items audited "per each item's SHOULD-level scope and escalation clause"). Within the SHOULD tier there is no further ranking: every item is a flat peer bullet, each closing with the identical "SHOULD-level only at loom 1.0 … escalates to a build-time pin" boilerplate, and the recording convention explicitly permits a contributor to skip any SHOULD item by recording `not-performed` with a one-line rationale.

The SHOULD items span a wide stakes range, but the checklist surfaces no signal of that spread at the tier level. Some encode correctness- or privacy-adjacent guarantees — e.g. (i) provider overflow-signature wording (a silent rewording downgrades a real overflow to `TransportError` with null token fields), (aj) `SessionManager.inMemory` transcript non-retention (a regression is a transcript leak), (ac)/(ak)/(al) the prompt-mode error-detection and `Ok(string)` extraction ordering preconditions. Others are largely observational drift-detectors — e.g. (ae) per-extension privilege-scoping facet detection, (ao) `pi.registerFlag('loom', …)` namespace behaviour, (o)-style satellite re-validations that loom does not even consume. A contributor who must skip a subset under time pressure has no guidance distinguishing the high-stakes items from the conformance-only ones; the flat presentation invites skipping a transcript-leak or overflow-misclassification check on the same footing as a no-op drift probe.

The defect is the absence of any intra-tier prioritization signal — not the SHOULD level itself, which is appropriate for editorial audits that cannot yet be mechanically gated.

**History:** The checklist was built incrementally: 79fa3b9 first introduced the "SHOULD re-confirm … via editorial review" boilerplate when checklist item (k) was coined; 803106d added the authoritative-source / known-fragile-evidence framing across the existing presupposition paragraphs; subsequent commits (db69f2b is the most recent example, adding (ao)) extended the list to its current ~42 items, each new item reusing the same uniform-SHOULD shape. No commit ever introduced a within-SHOULD tier or audit-priority annotation; the MUST(a–e) / SHOULD(f–ap) cut is the only structural distinction the list has ever carried. The defect — uniform weighting across the SHOULD subset — is therefore not localised to any single commit but emerged as each addition preserved the homogeneous shape.

## Solution Space

### Recommendation

Keep items (f)–(aq) as flat peers and add a single sentence to the step-2 editorial-review checklist preamble in `version-bump-step2.md`, alongside the existing MUST/SHOULD distinction, that names the highest-stakes SHOULD items a time-constrained contributor SHOULD prioritize when skipping any item. Cite the prioritized items by their stable letters — the correctness/privacy-adjacent set (i), (ac), (aj), (ak), (al) — with a one-clause rationale that these guard against silent correctness or privacy regressions (overflow-signature misclassification, prompt-mode error-detection / `Ok(string)` extraction ordering, and in-memory-transcript non-retention). Do not add per-item tier labels and do not reorder or regroup the bullets, so no item letters or anchors change.

Extend the existing same-edit obligation that governs adding new checklist items so that the callout is maintained in lockstep: any future SHOULD item that guards a correctness- or privacy-adjacent guarantee MUST be added to the prioritized callout in the same commit that introduces it, preventing the callout from going stale.

## Consequence

**Severity:** advisory

A time-constrained contributor performing a Pi minor bump, allowed to record `not-performed` on any SHOULD item, has no basis for choosing which to skip and may drop a correctness- or privacy-adjacent audit (e.g. overflow-wording or in-memory-transcript non-retention) as readily as a purely observational one. The procedure still yields a working system — full coverage remains available and the per-item bodies describe individual stakes — so the gap is a prioritization-guidance deficiency rather than a contradiction or a missing rule.
