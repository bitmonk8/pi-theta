## Finding

The `FN-5` Tests bullet on `V3d` (and verbatim on its `V3d-T` partner) asserts that the function body's produced value "is absent on fail/cancel" at the function-result seam, without naming what "absent" looks like to a programmatic caller. Read on its own, "absent" admits several mutually-incompatible observables: a JS `undefined` returned through the value channel, a rejected seam promise, or a distinct sentinel value. A test author writing directly from the bullet has no basis to pick one.

The spec pins exactly one. `FN-5` (`functions.md#final-value-language-definition`) states that on failure (`?` propagation, panic, exhausted runtime ceiling) and on cancellation NO final value flows — the caller observes only the corresponding `Err` envelope (per `errors-and-results/queryerror-variants.md` and `cancellation.md`), with the success / fail / cancelled trichotomy and per-outcome envelope contents owned by `errors-and-results/error-model.md#terminal-outcomes`. That is neither a promise rejection nor `undefined` nor a sentinel — it is a terminal `Err` outcome on the result channel. A leaf that asserts promise-rejection or `undefined` would therefore ship green against a behaviour the spec does not specify.

The bullet does carry the disambiguating spec link, but the prose `absent on fail/cancel` reads as self-contained, and the `Ships when` clause repeats the same loose phrasing with no link at all. The identical phrasing also propagates to the `FN-5` re-citations in `V15a` and `V9i`, which re-cite the seam `V3d` defines.

## Consequence

**Severity:** correctness

Two reasonable implementers authoring the `FN-5` assertion diverge: one asserts the function-result seam surfaces a terminal `Err` envelope on fail/cancel, another asserts the value channel returns `undefined`/a sentinel or that the seam promise rejects. The leaf can ship green having asserted an observable the spec does not specify, leaving the spec's actual `Err`-envelope contract unverified.

## Solution Space

**Shape:** single

### Recommendation

Append a parenthetical to the `FN-5` Tests bullet on `V3d` naming the concrete observable: on fail/cancel the function-result seam yields the corresponding terminal `Err` envelope and no value flows through the success (`Ok`-payload) channel — the seam does not return a value/sentinel and does not reject. Cite `functions.md#final-value-language-definition` (the language definition) and `errors-and-results/error-model.md#terminal-outcomes` (the per-outcome envelope owner). Mirror the same parenthetical on the `V3d-T` `FN-5` bullet (which holds the identical text) and bring the `Ships when` "absent on fail/cancel" clause into line with the named observable.

The `FN-5` re-citations on `V15a` and `V9i` reference the seam `V3d` defines, so they inherit the disambiguation once `V3d` carries it; mirror the parenthetical there only if the re-citation bullets are touched.
