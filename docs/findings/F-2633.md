## Problem

The patch-skew teardown-witness obligation in `#patch-skew-degradation-contract` (the (P-a)/(P-b-1)/(P-c-1) fixtures) asserts the fixture must witness "each of the **eight** call sites enumerated in the closed `details.call` label set of the **Per-step isolation** paragraph". This restates a cardinality whose single source of truth is that `details.call` label set in `session-shutdown-semantics.md`. There is no skew-detection mechanism, so if the owning set's membership changes the duplicated literal "eight" drifts stale silently.
