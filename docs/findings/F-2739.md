## Finding

The `V15a` invocation-core leaf (and its paired `V15a-T`) bundles eight distinct, independently red-testable obligations under one impl+tests pair:

1. **INV-1** â€” discovery-root containment: identical `realpath` + segment-boundary re-checks at load-time and invocation-time, surfacing an escape on both channels (diagnostic + `InvokeInfraError{load_failure}`).
2. **INV-2** â€” the AST arg-list `style:"positional"|"named"` shape (only positional defined in 1.0).
3. **INV-3** â€” the invoke-options open-struct record (additive per-call-timeout seam).
4. The **static-resolution per-pass parse cache** and its transitive parse/lower walk over `invoke` paths and `.loom` `tools:` entries (closes `cka-44`).
5. **FN-5** â€” final-value propagation of the callee's produced value to the `invoke` caller on success, absent on fail/cancel (a re-citation against `V3d`'s function-result seam).
6. The **fresh-vs-attach cross-mode matrix** selection by callee mode.
7. The **`cka-47` cancellation checkpoint** firing immediately before each `invoke` (distributed off `V17c`, exercised on the live `V15a` surface via the `V8a` `Checkpoint` seam).
8. The **ERR-13 completed-callee-finality witness** (delegated live-carrier for `V4f`'s deferral): a completed `invoke` child stays final after a downstream `?`/panic/cancel.

These span four unrelated verification concerns â€” load-time path containment, static (parse-time) resolution, runtime value/mode behaviour, and cancellation/concurrency â€” each with its own setup, fixtures, and seam dependencies. A single agent cannot hold all eight in scope in one pass, and the leaf's `Ships when` gate fires only when all eight assert together, so a partial implementation cannot ship a verified subset. The two cancellation facets (7, 8) are red-testable in complete isolation from the core invocation path and carry their own seam dependencies (`V8a`, `V17c`, `V4f`), making the bundle's seams especially clean to cut.

## Consequence

**Severity:** correctness

The `Ships when` gate only fires when all eight facets assert together, so an agent that cannot hold the full leaf in scope is likely to under-implement or under-verify a facet â€” most easily the two cancellation obligations (`cka-47`, ERR-13), which were appended last and sit furthest from the core invocation path. A dropped or weakly-asserted facet ships silently because no smaller gate isolates it, and two reasonable implementers would partition the work differently, producing divergent partial leaves.
