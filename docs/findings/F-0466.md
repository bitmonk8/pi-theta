## Problem

In `real-host-smoke-gate.md`, the manual real-host smoke gate's pass/fail criterion (e) tests that the emitted `loom-system-note` codes are a subset of the committed permitted-code list. The left-hand (scored) side is qualified by the `loom-system-note` *delivery channel* (the `sendMessage` envelope owned by `V7d`), but the right-hand permitted-code list is defined over `loom/...` *diagnostic codes* regardless of how they are delivered — the two scopes are not the same set. Some `loom/...` codes travel a `console.error` isolation path rather than the `loom-system-note` channel (`V9g`'s session-shutdown-teardown-step-failed at coverage row `cka-29`, and `V7d`'s own delivery-failure fallback tail). An auditor therefore has two readings of this merge-blocking criterion: score only channel-delivered codes (a real divergence confined to the `console.error` path escapes the gate entirely) or score every emitted `loom/...` code (a benign isolation-channel code merely absent from the hand-maintained union scores as a spurious fail).

## Issue introduction

**Verdict:** present-since-inception (carried through multiple commits without disambiguation)
**Introducing commits:**
- `3911733ff40bb4e32534fdf2b663980a85d3108c` (Thomas Andersen, 2026-06-11, "resolve 'Live-host smoke pass criterion assumes a non-deterministic LLM reproduces a transcript'") — reworked the smoke pass condition into the model-output-invariant criterion and introduced the "emitted `loom-system-note` codes within the permitted set" phrasing in `H6a-live-corpus-activation.md`. The channel-qualified "`loom-system-note` codes" wording (predating this in the "committed golden `loom-system-note` diagnostics list" framing, e.g. `e7f14dd`) was carried forward without disambiguating channel-scope vs all-`loom/...`-codes.
- `052b019b35264047f3f4601d06672d90202bb35a` (Thomas Andersen, 2026-06-11, "resolve 'Real-host smoke pass criterion (e) names a permitted code set with no committed source'") — replaced "permitted set" with the "**committed permitted-code list** … union of the `loom/...` codes the slices … can emit." This introduced the scope mismatch in its current form: the LHS stayed channel-qualified (`loom-system-note` codes) while the new RHS is defined over `loom/...` codes irrespective of channel. The mismatch was not noticed or resolved.
- `fd22a3dd5d83ecaad73db7351b7cb3a8200535a8` (Thomas Andersen, 2026-06-17, "park F-0209 (… H7a stale ref to moved criterion)") — moved the criterion (a)–(e) block from `H6a`/`H4a` into `real-host-smoke-gate.md`, carrying the ambiguous "emitted `loom-system-note` codes are a **subset**" phrasing verbatim into its current home.

**History:** The criterion has named the scored codes by the `loom-system-note` delivery channel since the smoke pass condition was first authored, while the underlying `loom/...` diagnostic set has always included `console.error`-delivered codes (`V9g` isolation path, `V7d` fallback tail). No commit in the chain disambiguated the LHS channel-scope against the all-`loom/...`-codes RHS; the defect is structural and present since the criterion existed, refined but never fixed across the reworks above.

## Solution approach

Rewrite the left-hand (scored) side of the subset test in criterion (e) so the scored set matches the right-hand permitted-code list — all `loom/...` diagnostic codes the run emits regardless of delivery channel (the `loom-system-note` `sendMessage` envelope, the `V7d` delivery-failure fallback, and the `V9g`/`cka-29` `console.error` isolation path alike), not only codes that reached the `loom-system-note` channel. The RHS is already authored over channel-agnostic `loom/...` codes, so the channel-qualified LHS is the side to realign.

## Solution constraints

- Whether the permitted-code-list union is a complete superset of the emittable `loom/...` codes is owned by F-0322; do not widen or audit the union here — this fix realigns only what criterion (e) scores.
- The smoke gate's "benign live-model variance … is **not** a fail" carve-out must continue to hold: an isolation-channel code already in the permitted list must not become a fail.
