## Finding

The manual real-host smoke gate's pass/fail criterion (e) reads: "the emitted `loom-system-note` codes are a **subset** of the **committed permitted-code list** … a manually-maintained, best-effort union of the `loom/...` codes the slices in `H7a`'s **Deps** can emit." The left-hand side of the subset test is qualified by the `loom-system-note` *delivery channel* (the `sendMessage` envelope owned by `V7d`), but the right-hand permitted-code list is defined over `loom/...` *diagnostic codes* regardless of how they are delivered. The two scopes are not the same set.

Not every `loom/...` code the run can emit travels the `loom-system-note` channel. The diagnostic-emission-isolation path emits via `console.error` rather than `sendMessage`: `V9g`'s `loom/host/session-shutdown-teardown-step-failed` (coverage row `cka-29`) is delivered through the wrapped `console.error` isolation sequence, and `V7d`'s own delivery-failure fallback chain terminates in `console.error` (`loom/runtime/system-note-delivery-failed`) when `pi.sendMessage` throws. An auditor scoring criterion (e) therefore faces two readings: (1) score only codes that actually reached the `loom-system-note` channel, ignoring isolation-channel diagnostics; or (2) score every `loom/...` code the run emits regardless of delivery channel. The two readings produce different pass/fail verdicts whenever an isolation-channel diagnostic fires during the smoke run — under reading (1) such a code is invisible to (e); under reading (2) it must appear in the permitted-code list or the run is scored a confirmed behavioural-divergence **fail**.

Because (e) is a falsifiable release-/merge-blocking criterion, the ambiguity matters: under reading (1) a genuine real-host divergence confined to the `console.error` isolation path escapes the gate entirely (the exact divergence class the smoke exists to catch), while under reading (2) a benign isolation-channel code merely missing from the hand-maintained union scores as a spurious divergence and blocks the merge.

## Solution Space

**Shape:** single

### Recommendation

In `real-host-smoke-gate.md` criterion (e), make the scored set match the right-hand permitted-code list, which is already defined over `loom/...` codes irrespective of channel. Rewrite the left-hand side of the subset test so it reads, in substance, that **all `loom/...` diagnostic codes the run emits — regardless of delivery channel (the `loom-system-note` `sendMessage` envelope, the `ctx.ui.notify`/`console.error` delivery-failure fallback in `V7d`, and the `console.error` diagnostic-emission-isolation path in `V9g`/`cka-29` alike) — are a subset of the committed permitted-code list**. Strike the bare channel qualifier "emitted `loom-system-note` codes" as the LHS of the subset test wherever it appears in the criterion-(e) sentence.

This is the correct disambiguation because the all-channels reading is the only one that lets criterion (e) catch a real-host divergence confined to the `console.error` isolation path — the divergence class the smoke gate exists to detect — and because the RHS permitted-code list is already authored as the union of `loom/...` codes the `H7a`-Deps slices can emit, so the channel-scoped LHS is the side that is out of alignment.

Edge cases the implementer must watch:
- `H7a`'s committed golden diagnostics list is itself described as `loom-system-note` diagnostics; if (e)'s scope widens to all-channels, confirm the permitted-code-list union actually enumerates the `console.error`-delivered codes (`V9g` `cka-29`, `V7d` fallback) so a legitimate isolation-channel emission during the smoke does not score as a false divergence. (This containment-of-the-union concern is the subject of a separate finding; the channel-scope fix here only widens what (e) reads, it does not by itself guarantee the union is complete.)
- Keep this consistent with criterion (e)'s "benign live-model variance … is **not** a fail" carve-out: widening the scope must not turn an isolation-channel diagnostic that is already in the permitted list into a fail.

## Consequence

**Severity:** correctness

Two reasonable auditors scoring the same smoke run reach opposite pass/fail verdicts on criterion (e) whenever an isolation-channel `console.error` diagnostic fires. The channel-scoped reading silently lets a real-host divergence on the `console.error` isolation path through the gate; the all-codes reading spuriously fails a benign isolation-channel code that is merely absent from the hand-maintained union — either way a release-/merge-blocking decision rests on an unspecified scope.
