## Finding

`V5c-descriptions.md` carries three Tests bullets: (1) byte-for-byte `///` lowering into `description:`, (2) the `loom/parse/doc-comment-misplaced` diagnostic, and (3) the multi-line `///` join with common-leading-whitespace strip and `//`-non-propagation. Its **Ships when** line — "`npm test` asserts byte-for-byte `description:` lowering and `doc-comment-misplaced`" — names only the first two. The third bullet describes a distinct observable behaviour (DESC multi-line join / whitespace strip / `//` not propagated) that the externally observable exit condition silently drops.

The multi-line-join behaviour is independent of byte-for-byte single-line lowering: an implementation can satisfy bullets (1) and (2) while mis-joining multi-line comments, failing to strip common leading whitespace, or wrongly propagating `//`. The Ships-when line, as written, would read as observably satisfied with that defect present. The Tests bullet remains binding under the per-phase TDD ritual, so a careful implementer still closes all three; the gap is that the leaf's stated exit gate under-describes what must hold before `V5c-complete` is tagged.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V5c-descriptions.md`, extend the **Ships when** line so it also names the third Tests bullet's observable. The line currently reads:

> **Ships when.** `npm test` asserts byte-for-byte `description:` lowering and `doc-comment-misplaced`.

Add the multi-line-join observable so all three Tests bullets are represented, e.g.:

> **Ships when.** `npm test` asserts byte-for-byte `description:` lowering, `doc-comment-misplaced`, and multi-line `///` join with common-leading-whitespace strip and `//` not propagated.

The fix is confined to V5c's Ships-when field; no spec edit and no change to the Tests bullets or to `V5c-T` is required.

## Consequence

**Severity:** advisory

The Tests bullet is binding under the TDD ritual, so the leaf can still be implemented correctly; the risk is that the leaf's externally observable exit gate is incomplete, so an implementer reasoning from Ships-when alone could tag `V5c-complete` with multi-line-join / whitespace-strip / `//`-propagation behaviour unverified.
