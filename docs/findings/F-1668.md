## Problem

`docs/spec_topics/hard-ceilings/ceiling-invariants-and-audit.md` carries the
H1 `# Ceiling invariants and audit`, and `docs/spec_topics/hard-ceilings.md`
links to it under that same label in its `## Contents` list. The page's
actual `##` sections are `Worked consequences` and `No additional loom 1.0
runtime ceiling applies` (the NOCEIL-1 … NOCEIL-4 claims plus audit
methodology). The ceiling-set invariants the title promises are not on this
page — they live on the sibling `hard-ceilings/ceilings-3-and-4.md` under
`#ceiling-set-invariants`. A reader following the parent-index link for
"ceiling invariants" lands on a page that never defines an invariant.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** f5e89f4 — docs: enforce 100KB hard / 30KB recommended size cap on spec set (2026-06-04, Thomas Andersen)
**History:** Before the size-cap split, `docs/spec_topics/hard-ceilings.md`
was a single page whose `##` sections — in source order — were
`Per-class retry budget`, `Per-boundary destination/surface table`,
`Interaction between ceilings`, `Ceiling-set invariants`, `` `masked` field ``,
`Worked consequences`, and `No additional loom 1.0 runtime ceiling applies`.
Commit f5e89f4 split that page into two children: `ceilings-3-and-4.md`
received the first five sections (including `Ceiling-set invariants`) and
`ceiling-invariants-and-audit.md` received the last two (`Worked
consequences` and the NOCEIL block). The split-driver chose the new H1
`Ceiling invariants and audit` for the second file despite having moved the
invariants section to the first; subsequent commits (last touch
2026-06-11 81bf52a, a path-only re-collapse) preserved the mismatch
unchanged.

## Solution approach

Rename the H1 in
`docs/spec_topics/hard-ceilings/ceiling-invariants-and-audit.md` to describe
the page's actual content — the NOCEIL claims and audit methodology — rather
than the invariants it no longer holds. Update the matching link label in
`docs/spec_topics/hard-ceilings.md`'s `## Contents` list so it tracks the new
H1.

## Solution constraints

- Out of scope: the page filename and every on-page anchor
  (`#no-additional-ceilings`, `#audit-methodology`, `#four-axis-closure-list`,
  `#release-gate`) — inbound cross-references resolve through these anchors,
  the `NOCEIL` inline-label registration in
  `governance/stable-inline-labels.md` keys off the file path, and plan
  leaves cite the path.
