## Finding

`V18c`'s **Adds** field opens by enumerating the leaf's build-time static gates (the SDK surface-inventory tests, the `engines.node` three-way floor gate, the `peerDependencies` pin, the capability-probe / `SHUTDOWN_AWAIT_CAP_MS` literal-read assertions, the reason-snapshot brand string, the provider seed-field table, and the strict-capability probe) and then appends a standalone trailing sentence describing the contributor's editorial-review checklist (items `(a)`–`(al)`): a manual per-bump audit of each host presupposition against the candidate Pi minor, MUST-level for `(a)`–`(e)` and SHOULD-level-audit-but-MUST-record for `(f)`–`(al)`.

That checklist is a manual contributor workflow whose obligations are defined and owned by the spec's `version-bump-step2.md` — already cited in `V18c`'s **Spec** field. `V18c` adds nothing in code for it: the sentence is explicitly flagged "*not* a build-time gate," and the paired test leaf `V18c-T`'s Adds/Tests enumerate only the automated gates with no editorial-review test. **Adds** is the field that states what the leaf contributes to the codebase; a description of an external manual audit procedure is workflow narration, not a contribution of this leaf.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** f1cda48 — pi-loom plan: resolve "V18c mislabels editorial-review checklist as a static build-time gate" (2026-06-11, Thomas Andersen)
**History:** The editorial-review checklist has been mentioned in `V18c`'s Adds since the leaf's first commit (c6a664e, 2026-06-10), but originally as a list item folded into the build-time-gate enumeration — the mislabel a later review flagged. Resolving that finding, f1cda48 removed the list item and introduced the current standalone "Separately — and *not* a build-time gate — the contributor's editorial-review checklist … a manual per-bump audit" sentence; that descriptive workflow note is the prose this finding flags.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V18c-version-bump-checklist.md`, in the **Adds** field, strike the trailing sentence beginning "Separately — and *not* a build-time gate — the contributor's editorial-review checklist (items `(a)`–`(al)`):" through its end ("…SHOULD-level-audit-but-MUST-record for `(f)`–`(al)`."). Leave the preceding build-time-gate enumeration and the "These gates need only the inventory leaves; the runtime-evidence acceptance gate and revert path travel with `V18d`." sentence intact. The editorial-review checklist's obligations are defined and owned by `version-bump-step2.md` (already cited in `V18c`'s Spec field); the spec is read-only for this fix.

Edge case for the implementer: `V18c`'s **Ships when** field still references the editorial-review checklist's acceptance as a ship gate. That fragment is the subject of a separate finding and is out of scope for this Adds edit — do not touch Ships-when here.

## Consequence

**Severity:** advisory

`V18c`'s Adds conflates the leaf's automated build-time gates with a manual contributor-audit procedure the leaf does not implement, overstating the leaf's code scope. An implementer must parse and discard the workflow narration to find the leaf's actual contributions; the "*not* a build-time gate" qualifier and the absence of any `V18c-T` test signal it is out of code scope, so no implementer is blocked or driven to implement the wrong thing.
