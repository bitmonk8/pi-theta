## Finding

PIC-13's case-canonicalisation guarantee binds the production `PiFileSystem` adapter, but the property is unobservable on case-sensitive CI and no editorial-review fallback is stated; the seam-owning plan leaf tests only `.code` mapping.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** e9b0597 — pi-loom spec: resolve "invocationId has no injectable id-source seam" (2026-06-05); a765f6ca — fix F-0977: pin PIC-13 realpath case-canonicalisation guarantee for DISC-3 dedup (2026-06-22); 4556623 — fix F-1178: PIC-12 timing-primitive ban gets observable build-time conformance criterion (2026-06-25)
**History:** PIC-20 was introduced whole by e9b0597 (the only commit touching the `any other ambient UUID source` clause, per `git log -S`), already lacking any conformance criterion. a765f6ca added the PIC-13 `realpath` case-canonicalisation guarantee binding the production adapter with no stated test path. The PIC-20/PIC-12 asymmetry then crystallised on 2026-06-25 when 4556623 retrofitted the build-time-check conformance criterion onto PIC-12 (and PIC-13's `homedir` bullet by reference) but not PIC-20 — even though PIC-20 was edited the same day by 0c6c347 (fix F-1396, toolCallId minting) — leaving PIC-20 the only ambient-primitive ban in the section without a criterion.

## Solution Space

**Shape:** single

### Recommendation

State explicitly that the production-adapter case-canonicalisation guarantee is verified by editorial review / known-fragile evidence (it is unverifiable by automated test on a case-sensitive host), following the pattern used for the `cwd == project root` presupposition — add a Pi version-bump checklist item analogous to item (w). Pin an automated conformance test for the **Fake** half (`FakeFileSystem`) via injected case-variant symlink-table aliases that MUST resolve to one byte-identical canonical path, so the testable half is exercised on every CI run regardless of host case-sensitivity.

## Consequence

**Severity:** correctness

Two reasonable implementers diverge on whether PIC-20's UUID-source ban is build-time-verified and against which identifiers, and the open "any other ambient UUID source" clause cannot be turned into a falsifiable check; one ships without the scan, the other guesses the identifier set. Independently, the PIC-13 `realpath` case-canonicalisation MUST binds the production adapter but cannot be exercised on case-sensitive CI, so a leaf-case canonicalisation bug ships undetected and breaks the byte-for-byte discovery-root containment comparison the guarantee exists to protect.
