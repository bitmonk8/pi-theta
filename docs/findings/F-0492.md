## Finding

The `H7a` leaf names its single gate with two distinct QA terms inside one `Adds.` paragraph. Sentence 1 calls it "A terminal **integration-acceptance** gate"; sentence 2 then calls the same gate "a cross-slice **integration-regression** gate that exercises the composition the per-leaf gates verify only in isolation." Integration-acceptance and integration-regression denote different QA roles (proving the composition is correct at first commit vs guarding against later regressions), so a reader cannot tell whether the leaf defines one gate with two roles or conflates two concepts.

Every other reference to this gate uses "integration-acceptance" consistently: the H7a heading, the `Ships when.` field, the `H6a` dependency note (`H6a-live-corpus-activation.md`), and `real-host-smoke-gate.md` ("the `H7a` integration-acceptance fixture"). The "integration-regression" variant is unique to this one sentence.

The gate genuinely plays both roles — it accepts the cross-slice composition and thereafter guards it against regression — so the fix is to pick the primary classifier ("integration-acceptance", matching every cross-reference) and demote the regression role to a secondary descriptor rather than a competing name.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H7a-integration-acceptance.md`, `Adds.` field, sentence 2, replace the competing name with the primary classifier and recast the regression role as a secondary descriptor. Change `It closes no new spec REQ-ID; it is a cross-slice integration-regression gate that exercises the composition the per-leaf gates verify only in isolation.` to text that keeps "integration-acceptance" as the gate's name and frames regression as an additional role it serves — for example: `It closes no new spec REQ-ID; it is the cross-slice integration-acceptance gate that exercises the composition the per-leaf gates verify only in isolation, and thereby also serves as a cross-slice regression guard.` Leave the heading, `Tests.`, `Ships when.`, and all cross-references unchanged — they already use "integration-acceptance".
