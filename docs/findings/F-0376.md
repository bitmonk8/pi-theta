## Finding

`V11a` (Adds and the single-matcher cross-resolution reconciliation Tests bullet) and `V6a` (Adds) assert that the loom `model:` load-time resolution and the binder-model resolution "cannot diverge on the 'reference matches no available model' condition," and `V11a` pins that guarantee to a witness of *single-source-of-construction (instance identity), not equivalence-of-outcome*. The inference does not hold. A shared matcher instance witnesses only that both call sites run the **same matching implementation**; it says nothing about whether the two sites feed that matcher **equivalent inputs**, nor whether they **interpret a no-match return identically**.

Concretely, the `V6a` parser hook could pre-normalise the `model:` reference string (trim, case-fold, provider-qualify) before calling the shared matcher while `V11a`'s binder-model path passes the reference through unmodified — or the two could branch differently on the same no-match verdict (one fails the load via `loom/load/model-unresolved`, the other routes `loom/load/binder-model-unresolved`). Both behaviours are fully compatible with one shared matcher instance, so the "cannot diverge" claim overstates what the test (`expect` the same instance) actually proves.

The witness statement itself — "the same instance services both resolutions (instance identity, not equivalence-of-outcome)" — is accurate. The defect is the conclusion drawn from it.

## Solution Space

**Shape:** single

### Recommendation

Soften the conclusion in both leaves so it claims only what the instance-identity witness proves.

- In `docs/plan_topics/V11a-binder-model-resolution.md` **Adds**, replace the clause "so the two paths cannot diverge on the \"reference matches no available model\" condition" with wording that scopes the guarantee to the shared implementation, e.g. "so both paths bind the same matcher instance and therefore share one matching implementation (not, by itself, equivalent inputs or identical no-match handling)".
- In `docs/plan_topics/V11a-binder-model-resolution.md`, the single-matcher cross-resolution reconciliation **Tests** bullet, strike the trailing "— so the two paths cannot diverge on the \"reference matches no available model\" condition" and stop at the instance-identity witness clause ("observed by single-source-of-construction (instance identity), not equivalence-of-outcome against a shared fake").
- In `docs/plan_topics/V6a-frontmatter-contract.md` **Adds**, replace "so this `model:` resolution and `V11a`'s binder-model resolution cannot decide the same \"reference matches no available model\" condition differently" with the same scoped wording (both bind the same matcher instance / share one matching implementation).

Leave `V9b`'s "(instance identity, not equivalence-of-outcome)" witness statements unchanged — they are accurate as written. If the stronger no-divergence property is actually wanted, that is a separate testable obligation (assert both call sites pass the matcher the same normalised reference and react identically to a no-match return) and should be stated as such rather than asserted from instance identity.

## Consequence

**Severity:** advisory

The wording asserts a stronger guarantee than the test establishes, so an implementer may rely on instance identity as proof that the two paths cannot produce different no-match verdicts and therefore omit any test for input-normalisation or verdict-interpretation divergence between the `V6a` `model:` path and the `V11a` binder-model path. A working leaf still ships; the risk is a silent divergence that the overclaim discourages anyone from testing for.
