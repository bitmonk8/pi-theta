## Finding

The `system:` interpolation *Path grammar* in `frontmatter-fields-b-and-templates.md` opens its prose with "There is no depth bound — arbitrary chains of `.Ident` are accepted", then immediately constrains acceptance: "The head `Ident` must name a declared `params` entry; each subsequent `.Ident` must name a reachable field of an *object* schema in the loom-side `params:` declaration." The first sentence's unqualified verb "accepted" reads as syntactic *and* semantic acceptance, when it is really only describing the absence of a chain-length cap. The per-step reachability rule that actually gates each `.Ident` is presented as a separate, following statement rather than as an inseparable part of the same acceptance claim.

The constraint is not absent — it is fully specified two clauses later and is backed by dedicated parse diagnostics (`loom/parse/system-interp-unknown-param` for the head, `loom/parse/system-interp-bad-field` for a step that does not name a reachable object field). The defect is presentational: a reader who anchors on "arbitrary chains of `.Ident` are accepted" may momentarily treat path acceptance as unconditional. Scoping the "no depth bound" statement to syntactic chain length and folding the head/step validation into the same acceptance rule removes the ambiguity without changing any behaviour.

## Solution Space

**Shape:** single

### Recommendation

Rewrite the depth statement so the field-reachability constraint is inseparable from it. Scope "no depth bound" to syntactic chain length and present head/step validation as part of the same acceptance rule, e.g.: "The `Path` production places no limit on chain length, but every step is validated: the head `Ident` must name a declared `params` entry, and each subsequent `.Ident` must name a reachable field of an *object* schema in the loom-side `params:` declaration (`loom/parse/system-interp-unknown-param` and `loom/parse/system-interp-bad-field` respectively)." Keep the existing arrays/discriminated-unions/scalars-terminate-the-path sentence and the diagnostics list unchanged.

## Consequence

**Severity:** cosmetic

The complete per-step validation rule and its two diagnostics are present in the same paragraph, so two careful implementers converge on the same behaviour. The only exposure is a transient misreading of the opening sentence; no observable behaviour or acceptance criterion is at stake.
