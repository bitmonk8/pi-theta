## Finding

`V9k`'s fifth Tests bullet (the `session_start`-time `pi.getCommands()` read-failure case) ends "— asserted distinctly from **the four write-side surfaces**." That phrase has no explicit antecedent: it relies on the reader counting the four bullets that physically precede it (`pi.registerMessageRenderer`, `pi.registerCommand`, `pi.registerFlag`, `pi.on`) and inferring that those four are the "write-side surfaces" being contrasted with the `getCommands()` read.

The count is correct today, but it is bound to bullet ordering rather than to the surface names. Any later edit that adds, removes, or reorders one of the preceding registration-call bullets silently invalidates "four" while leaving the prose unchanged — the bullet would then assert distinctness from a miscounted set with nothing flagging the drift. The four surface names are already spelled out verbatim in the four preceding bullets, so naming them at the contrast site costs nothing and removes the positional dependency.

The same phrasing appears verbatim in the paired `V9k-T` tests leaf (where it carries an additional trailing rationale clause tracked as a separate finding), so the fix applies to both leaves to keep the paired bullets aligned.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V9k-extension-bootstrap-failures.md`, in the `pi.getCommands()` read-failure Tests bullet, replace the phrase "the four write-side surfaces" with an explicit enumeration of the four surfaces it contrasts against: `pi.registerMessageRenderer`, `pi.registerCommand`, `pi.registerFlag`, and `pi.on`. Apply the identical replacement to the matching `pi.getCommands()` read-failure bullet in `docs/plan_topics/V9k-T-extension-bootstrap-failures.md` so the paired test bullet stays aligned. The four names are already used verbatim in the preceding bullets of both leaves, so this introduces no new identifiers.

## Consequence

**Severity:** advisory

Today an implementer can recover the four write-side surfaces by reading the four preceding bullets, so no leaf is blocked and no divergence is forced. The exposure is future drift: a later bullet edit makes "four" wrong while the assertion still reads as if it were right, weakening the very distinctness check the bullet exists to enforce.
