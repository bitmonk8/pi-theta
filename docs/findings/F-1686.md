## Finding

PIC-41 ("No `signal` field on the spawn call") states that the subagent `createAgentSession(...)` call **MUST NOT** include a `signal` field. The same sentence concedes the basis for the rule: the pinned Pi `CreateAgentSessionOptions` type (declared at `dist/core/sdk.d.ts` at the loom 1.0 Pi-SDK pin) has **no** `signal` field, "so passing one would be silently ignored." That concession is precisely what makes the `MUST NOT` unverifiable as a black-box conformance obligation: because Pi ignores an unrecognised `signal` field, an implementation that includes one and an implementation that omits one are observationally identical — there is no runtime behaviour, diagnostic, or surface that distinguishes the conformant case from the non-conformant case. A normative requirement whose violation produces no observable consequence has no test oracle.

The genuinely testable cancellation contract sits in the same paragraph and is already normative: the runtime registers a one-shot listener on `loomAbort.signal` that calls `AgentSession.abort()`, and — for the already-aborted case — invokes `session.abort()` synchronously before the listener is registered and before the first user turn. Those obligations have clear oracles (cancel a spawned subagent and assert it aborts; pre-abort the signal and assert the spawn-then-immediate-cancel path terminates). PIC-41 adds nothing testable on top of them; it restates the absence of an SDK field as a behavioural prohibition.

The fix is to stop carrying the prohibition as a numbered normative `MUST NOT` and demote it to the non-normative observation it actually is (the SDK type has no `signal` field; cancellation is forwarded by the listener), leaving the listener-and-synchronous-abort rules as the normative, testable content.

## Solution Space

**Shape:** single

### Recommendation

Demote PIC-41 from a numbered normative `MUST NOT` to a non-normative implementation note. Concretely:

- Drop the `**PIC-41. No `signal` field on the spawn call.** The spawn call MUST NOT include a `signal` field.` sentence as a normative obligation. Retain the factual observation — the pinned `CreateAgentSessionOptions` (`dist/core/sdk.d.ts`) declares no `signal` field, so a `signal` would be silently ignored — as descriptive prose or a parenthetical note, since it justifies *why* the listener is the forwarding mechanism.
- Keep as the normative, testable content the two rules already in the paragraph: (1) the runtime registers a one-shot listener on `loomAbort.signal` calling `AgentSession.abort()`, detached in the per-invocation `finally`; and (2) if `loomAbort.signal.aborted` is already true when the spawn resolves, the runtime calls `session.abort()` synchronously before registering the listener and before the first user turn.
- If the PIC-41 anchor is referenced elsewhere, preserve the `#pic-41` anchor on the relocated note (per the GOV anchor-stability convention) so inbound links do not break; the surrounding identifiers PIC-40/PIC-42/PIC-43 are unaffected.

Edge case for the implementer: the only way to give the "no `signal` field" rule an oracle is a *compile-time* one — constructing the spawn options as an object literal typed against the pinned `CreateAgentSessionOptions` makes TypeScript's excess-property check reject a stray `signal`. That is a type-checking artefact of consuming the pinned type, not a runtime conformance test, so it does not warrant keeping PIC-41 as a runtime `MUST NOT`; mention it (if at all) only in the non-normative note.

## Consequence

**Severity:** advisory

PIC-41 is a normative `MUST NOT` that no black-box conformance test can exercise, because the field it forbids is one the pinned Pi options type does not declare and silently discards. It inflates the conformance surface with an unfalsifiable obligation and invites a reviewer to author a fixture that cannot fail. No working system is endangered — the real cancellation guarantee (listener → `session.abort()`) is separately normative and testable — so the impact is a spec-hygiene/testability gap, not a behavioural divergence.
