## Finding

`V15c`'s `IMP-1` test asserts the resolver-failure contract: an unresolvable `.warp` path throws тЖТ `loom/load/unresolvable-warp-path`, where "unresolvable" includes "no byte-exact final-segment entry." The spec rule that defines this comparison (`imports.md` ┬зIMP-1) is explicit that the byte-exactness is taken **against the bytes [`FileSystem.readdir`](../spec_topics/pi-integration-contract/host-interfaces-services.md#pic-13) returns for the resolved parent directory** тАФ entry names only, UTF-8, case-sensitive, never against a Unicode-normalised path literal. The non-normative note in the same spec section further directs a conformant resolver to enumerate the parent via `readdir` once rather than relying on `FileSystem.exists` / `readText` (which would falsely succeed on a case-insensitive host).

That `readdir` return-shape contract lives in `host-interfaces-services.md` under PIC-13 (`readdir(path: string): Promise<readonly string[]>` тАФ entry names only, no full paths). `V15c` and `V15c-T` both name only `imports.md` in their `Spec.` field. An implementer who reads strictly the cited `Spec.` page has the cross-link to PIC-13 available inside `imports.md` text but no pin instructing them to read it, and the leaf does not name `V8b` (the `FileSystem` seam that owns the `readdir` member) in its `Deps.`. The risk is an implementer building the byte-exact comparison without the `readdir` return-shape and case-sensitivity contract тАФ exactly the `exists`/`readText` mis-implementation the spec warns against.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commit:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** `V15c-imports.md` was created at `c6a664e` with `**Spec.** [imports.md]` as its sole spec reference, and `git log -S "host-interfaces-services"` over both `V15c-imports.md` and `V15c-T-imports.md` returns no commits тАФ the `host-interfaces-services.md` / PIC-13 page has never appeared in either leaf's `Spec.` field. The dependency the omission concerns was already live at leaf-creation time: the byte-exact-against-`FileSystem.readdir` rule entered `imports.md` earlier, at `9aa73b1` (2026-06-07, "pi-loom spec: resolve \"Filesystem case-sensitivity for .warp basenames and invoke/tools containment\"", confirmed via `git log -S "FileSystem.readdir" -- docs/spec_topics/imports.md`). Thus when `V15c` was authored on 2026-06-10 the spec already required the `readdir`-byte-exact comparison, and the leaf's `Spec.` field has carried the incomplete reference set since its first commit.

## Solution Space

**Shape:** single

### Recommendation

Add `host-interfaces-services.md` to the `Spec.` field of both `docs/plan_topics/V15c-imports.md` and `docs/plan_topics/V15c-T-imports.md`, alongside the existing `imports.md` entry тАФ e.g. append `, [`../spec_topics/pi-integration-contract/host-interfaces-services.md`](../spec_topics/pi-integration-contract/host-interfaces-services.md)` so the PIC-13 `readdir` return-shape and case-sensitivity contract is pinned for the implementer who builds the byte-exact final-segment comparison.

Additionally, name `V8b` (the `FileSystem` seam leaf that owns the `readdir` member) in `V15c`'s `Deps.` field so the resolver's dependency on the seam is observable in the dependency graph rather than implicit; keep `V15c-T`'s `Deps.` unchanged unless the tests-task itself exercises the seam. `V8b` and the cited spec pages are read-only for this fix; the spec is correct as written and must not be edited.

## Consequence

**Severity:** advisory

An implementer scoping to `V15c`'s cited `Spec.` page can author the `IMP-1` byte-exact comparison without the PIC-13 `readdir` return-shape and case-sensitivity contract, reproducing the `exists`/`readText`-on-case-insensitive-host failure mode the spec explicitly warns against. The in-spec cross-link from `imports.md` to PIC-13 mitigates for a careful reader, so a working leaf is still reachable, but the leaf's own `Spec.` pin does not close the dependency.
