## Finding

The QRY-6 "Runtime short-circuit" bullet in `docs/spec_topics/query/query-forms.md` ends with a `Rationale:` clause appended to the MUST NOT statement that suppresses respond-repair on an empty-template short-circuit:

> … `ValidationError.attempts` is 0 regardless of `respond_repair.attempts` and `respond_repair.methodology`. **Rationale:** respond-repair repairs a malformed model response (see Schema-validation respond-repair); the short-circuit is the runtime refusing input it constructed itself, before any model response exists, so there is nothing for a follow-up turn to repair.

The MUST NOT rule is fully normative on its own — it specifies the zero follow-up turns, the unchanged conversation history, and the fixed `attempts: 0` outcome. The `Rationale:` sentence is decision-log prose that justifies the choice to a reviewer; it adds no obligation an implementer can act on, and it explicitly cross-references the very section the MUST NOT lives next to (`Schema-validation respond-repair`).

This pattern recurs at one other site in the same shard (the non-validation-follow-up bullet later in the page) and is one instance of the broader inline-decision-log cruft that the review groups as a cross-shard pattern.

## Issue introduction

**Verdict:** present-since-inception (of the MUST NOT block)
**Introducing commits:** `4ac4297` ("pi-loom spec: resolve \"empty-template short-circuit suppresses coercion\"", 2026-05-05)
**History:** `git log -S` on the rationale sentence ("the short-circuit is the runtime refusing input it constructed itself, before any model response") returns three commits: `4ac4297` (the original addition, then at `spec_topics/query.md`), `31ff060` (the bulk move under `docs/`), and `f5e89f4` (the topic split into `docs/spec_topics/query/query-forms.md`). `git show 4ac4297 -- spec_topics/query.md` confirms the entire "An empty-template short-circuit … MUST NOT trigger the coercion path: … Rationale: coercion repairs a malformed model response …" addition entered in one diff — i.e. the `Rationale:` clause was authored as part of the MUST NOT block, not appended later. Commit `708a42f` ("coercion overloaded: schema-validation repair vs. type conversion", 2026-05-06) renamed "coercion" → "respond-repair" throughout the clause but did not prune it; commit `f14a45e` (2026-06-05) added the suppression hatch language elsewhere in QRY-6 without touching this sentence. The defect therefore dates from the moment the MUST NOT block existed.

## Solution Space

**Shape:** single

### Recommendation

Delete the `Rationale: respond-repair repairs a malformed model response (see Schema-validation respond-repair); the short-circuit is the runtime refusing input it constructed itself, before any model response exists, so there is nothing for a follow-up turn to repair.` sentence at the end of the QRY-6 Runtime short-circuit bullet. The preceding sentence ends "… regardless of `respond_repair.attempts` and `respond_repair.methodology`." — that is the bullet's terminal sentence after the edit. Do not replace the deleted text with a "see also" pointer; the bullet already names `respond_repair.attempts` / `respond_repair.methodology`, which makes the relevant section trivially greppable. Confirm afterward that the bullet still contains both the `MUST NOT` line and the unchanged `cause: "empty_template"` short-circuit envelope — those are the normative payload and must not be disturbed by the deletion.

## Consequence

**Severity:** cosmetic

A reader who needs the QRY-6 contract gets it from the MUST NOT statement; the trailing `Rationale:` clause is non-normative decision-log narrative that does not affect observable behaviour or implementer obligations. Its only cost is that it normalises an inline-rationale style the rest of the spec avoids, making the pattern harder to clean up at the cluster level.
