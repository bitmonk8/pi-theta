## Problem

`V7a` specifies the assembled `Diagnostic[]` as ordered by `(file, line, col)` in its **Adds.** sentence, its multi-error **Tests.** bullet, and its **Ships when.** gate. Read lexicographically the leading `file` component sorts files alphabetically by name, but the same Tests bullet cites implementation-notes.md's Static-resolution load pass, which aggregates each visited file's diagnostics in depth-first visitation (import) order — a different sequence whenever non-alphabetically-named `.warp` modules are imported. The leaf gives no criterion to choose between the two readings, and `V7a-T` pins whichever reading its author adopts into a failing test that the `V7a` Ships-when gate then enforces — risking an alphabetical contract shipping green against the spec's visitation order.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 249cec5 — pi-loom plan: resolve "implementation-notes.md (IMPL) unmapped in coverage matrix" (2026-06-11, Thomas Andersen)
**History:** `V7a` carried no diagnostic-ordering language at its creation commit (c6a664e). Commit 249cec5 added the `(file, line, col)` ordering clause together with the parenthetical pointing at implementation-notes.md's "aggregates each visited file's diagnostics … in this order"; the tuple shorthand and the visitation-order prose entered in the same edit, embedding the file-dimension ambiguity. The later reword 692cc8c preserved both phrasings without resolving the conflict.

## Solution approach

In `V7a`, rewrite the `(file, line, col)` ordering phrasing in the **Adds.** sentence, the multi-error **Tests.** bullet, and the **Ships when.** gate to state an explicit two-level order: first by static-resolution visitation order (entry `.loom` first, then transitively-imported `.warp` modules in depth-first import order), then within each file by ascending `(line, col)`. Apply the identical ordering language to `V7a-T`'s multi-error **Tests.** bullet so the paired tests assert the same order. Retain the existing cross-reference to implementation-notes.md — Static-resolution load pass.
