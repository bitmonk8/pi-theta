## Problem

The `H3a` second `Tests.` bullet and the `conventions.md` *No globals, statics, singletons* rule both ban direct ambient-primitive access using the clause "the timing primitives the spec's WallClock ban enumerates outside the `WallClock` adapter". The phrase "outside the `WallClock` adapter" has indeterminate attachment: it can scope only the trailing timing-primitive sub-list, or qualify the whole assertion (all primitives scanned only outside the WallClock adapter). Neither reading matches the comment-keyed exemption mechanism both surfaces then describe, where the sole exempt site is one carrying a same-line `// allow-ambient: <primitive> — <seam>` comment and exemption applies uniformly to all primitive categories at their owning adapters. `V8d-clock-id-seams.md` (PIC-12 / PIC-20) already states the WallClock adapter's own timing sites each carry their own `// allow-ambient:` comment, contradicting any location-keyed reading.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** c6824f2 — pi-loom plan: resolve "ambient-primitive scan omits spec-banned clock primitives" (2026-06-12, plan author)
**History:** Before c6824f2 the H3a Tests bullet and the conventions.md rule listed the banned primitives as a flat enumeration (`process.env`, `process.cwd`, `crypto.randomUUID`, `Date.now`, or `setTimeout`) with no location qualifier — unambiguous. Commit c6824f2 expanded the timing set to the full five-primitive list to match the spec's WallClock ban and, in doing so, introduced the relative clause "the timing primitives the spec's WallClock ban enumerates outside the `WallClock` adapter" into both surfaces, creating the attachment ambiguity. The same edit landed identically in `H3a-di-seam-skeleton.md` and `conventions.md`.

## Solution approach

Rewrite the dangling clause in the `H3a` second `Tests.` bullet so the five timing primitives sit as banned members of the same `src/**`-wide list with no location qualifier on the list itself. Move the WallClock-adapter reference into a note that the timing primitives' intended exempt site is the `WallClock` adapter, exempted by the same `// allow-ambient:` comment that exempts every other primitive at its owning adapter. Apply the parallel edit to the `conventions.md` *No globals, statics, singletons* rule, which carries the identical clause. Use `V8d-clock-id-seams.md` PIC-12 / PIC-20 as the consistency target.

## Solution constraints

- Exemption remains keyed solely by the `// allow-ambient: <primitive> — <seam>` comment — the reworded text must not introduce a location-based exemption for the WallClock adapter.
- The spec (`host-interfaces-services.md` PIC-12) and `V8d-clock-id-seams.md` are read-only references for this fix.
