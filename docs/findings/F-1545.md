## Problem

The named Pi behavioural presupposition `<a id="cwd-project-root-presupposition">` *Factory-time `cwd == project root` presupposition* lives inline inside the PIC-13 `cwd()` bullet of the `FileSystem` DI seam on `host-interfaces-services.md`, instead of on `host-prerequisites.md` alongside the other named host-behaviour presuppositions. `host-prerequisites.md` is the canonical inventory page an enumerating reviewer or version-bump contributor consults, and it does not list this entry — so the presupposition is invisible to an inventory-driven audit. The block is framed identically to its inventory siblings (peer cross-cites, *Authoritative source* paragraph, version-bump re-audit hook); content, anchor, cross-cites, and bump-checklist coverage are all sound. The defect is purely placement.

## Solution approach

Add a forward-link under the PIC-13 `cwd()` bullet pointing to the relocated paragraph so a reader walking the `cwd()` seam definition still reaches the launch-time presupposition. Rewrite the file portion of the inbound link in `version-bump-step2.md` checklist item `(w)` (`#bump-checklist-cwd-project-root`) from `./host-interfaces-services.md#cwd-project-root-presupposition` to `./host-prerequisites.md#cwd-project-root-presupposition`, and repoint any other corpus reference to the same anchor.
