## Problem

The spec's extension-discovery contract at `host-prerequisites.md` §"Extension entry point" pins a single module — `extensions/index.ts`, the file Pi auto-discovers from `package.json#pi.extensions = ["./extensions"]`. Neither half has a closing leaf: `H1a`'s `package.json` field enumeration declares every other manifest surface but omits the `pi.extensions` key, and `H4a`'s factory description never names the file the factory lives in. Because the end-to-end harness imports the factory directly rather than through Pi's package-discovery path, every shard gate and the harness stay green while a build a real Pi host cannot discover ships silently.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commit:** `c6a664e` (2026-06-10) — "pi-loom plan: build/update plan for spec.md + review"
**History:** The plan corpus is git-tracked. The current `H1a` and `H4a` leaf files both first appear in `c6a664e` (`git log --diff-filter=A --follow`), and neither has ever carried `pi.extensions` or `extensions/index` (`git grep -c` against `c6a664e` returns zero for both files; `git log -S "pi.extensions" -- docs/plan_topics/` shows no commit touching the current leaf paths). The contract was not always absent from the wider corpus: a prior plan generation owned it in `docs/plan_topics/h4-extension-shell.md`, whose `Adds.` pinned *"`extensions/index.ts` exporting `default function (pi: ExtensionAPI)`; `package.json`'s `pi.extensions` manifest entry…"*. That generation was deleted wholesale in the scaffold reset `657ee76` (2026-05-25, "reset to scaffold + template"), and the rebuild in `c6a664e` re-authored the scaffold and extension-factory leaves as `H1a`/`H4a` without re-establishing the discovery contract. The gap has therefore been present in every revision of the current `H1a`/`H4a` leaves since they were authored.

## Solution approach

Add the `pi.extensions` key with value `["./extensions"]` to `H1a`'s `package.json` field enumeration, and add a matching build-time literal-read assertion to `H1a`'s `Tests.`, citing `host-prerequisites.md` §"Extension entry point". In `H4a`'s `Adds.`, name the factory's on-disk file as `extensions/index.ts` and cross-reference the `pi.extensions` directory entry that resolves to it, citing the same anchor. Keep the manifest directory value (`["./extensions"]`) and the file path (`extensions/index.ts`) consistent — the manifest entry is a directory, not a file path.

## Solution constraints

- Out of scope: `docs/spec_topics/` (including `host-prerequisites.md`) is read-only for this fix.
- If the "H4a — too large" split lands, the factory-path pin belongs to the reduced factory-shell leaf, not the governance/self-check leaf.
