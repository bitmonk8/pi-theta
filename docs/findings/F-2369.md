## Problem

`H5e`'s un-rowed-page recogniser reddens on any `spec_topics/**` page absent from the REQ-ID prefix table "other than a GOV-24 table-of-contents hub stub", but states only the exclusion, never the mechanical test that separates a benign hub stub (stay green) from a defective un-rowed page (redden). The naive "no row and no anchors" reading fails to discriminate: a hub stub and a genuinely defective un-rowed page are both rowless and anchorless, so an implementer with only the leaf text cannot tell them apart. The actual discriminator lives in GOV-24 (a hub stub's subtree is named by a trailing-slash subtree row, and the stub itself anchors no REQ-IDs) but is never surfaced on the leaf. The corpus this gate binds against at `H6a` activation contains real hub stubs (`binder.md` under `binder/`, `governance.md` under `governance/`), so the divergence manifests concretely.

## Solution approach

In the **Adds** sentence of `docs/plan_topics/H5e-un-anchored-must-gate.md` that introduces the un-rowed-page recogniser, replace the bare exclusion with GOV-24's trailing-slash discriminator: an un-rowed `spec_topics/**` page `X.md` is an excluded hub stub when the prefix table carries a subtree row whose `Page` cell names `X`'s subtree with a trailing slash (e.g. `binder/`) and the page anchors no `PREFIX-N` REQ-IDs; every other un-rowed page reddens. Cite GOV-24 (`docs/spec_topics/governance/req-id-prefix-table-active-b.md#gov-24`) as the source of the criterion. Apply the same discriminator to the **Tests** un-rowed-page bullet and the **Ships when** seeded-fixture description so the green hub-stub fixture's subtree is stated as covered by a trailing-slash row while the red non-hub-stub fixture's is not, making the two un-rowed fixtures mechanically distinguishable rather than separated only by stated intent.

## Solution constraints

- The spec is read-only: GOV-24 already defines the discriminator — surface it on the `H5e` leaf, do not edit `req-id-prefix-table-active-b.md` or `conventions.md`.
