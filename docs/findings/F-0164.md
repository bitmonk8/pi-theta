## Finding

`V9k-T`'s fifth Tests bullet (the `session_start`-time `pi.getCommands()` read-failure case) ends with the clause `— asserted distinctly from the four write-side surfaces, since conflating this read with them produces a wrong test`. The paired implementation leaf `V9k` carries the same bullet but ends cleanly at `— asserted distinctly from the four write-side surfaces`, without the `since conflating this read with them produces a wrong test` rationale tail.

The trailing clause is explanatory prose about *why* the assertion is structured a certain way, not part of the observable the test must assert. It adds no testable content over what the preceding clause already states, and it introduces a wording asymmetry between the `-T` tests bullet and the implementation bullet that should otherwise read in lockstep.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `6df646a` (2026-06-12) "pi-loom plan: resolve \"extension-bootstrap SDK-failure rule has no closing leaf\""
**History:** `docs/plan_topics/V9k-T-extension-bootstrap-failures.md` was created in `6df646a` (`git log --diff-filter=A --follow`), and `git show 6df646a:docs/plan_topics/V9k-T-extension-bootstrap-failures.md` confirms the `since conflating this read with them produces a wrong test` clause was present in that creating revision. `git log -S` for the clause over `docs/plan_topics/` returns only `6df646a`; the subsequent edit `cb8737a` (over-bundling resolution) did not touch it. The defect therefore entered with the leaf at inception, not via a later edit.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V9k-T-extension-bootstrap-failures.md`, in the fifth Tests bullet (the `pi.getCommands()` read-failure case), strike the trailing clause so the bullet ends at `— asserted distinctly from the four write-side surfaces`. Specifically, delete ` since conflating this read with them produces a wrong test` (and the comma preceding it), leaving the bullet's terminal phrasing identical to the corresponding bullet in `V9k-extension-bootstrap-failures.md`.

## Consequence

**Severity:** cosmetic

The rationale tail does not change what the test asserts, so no implementer diverges on behaviour; the residue is a wording asymmetry against the paired `V9k` bullet. Shipping unfixed leaves a stylistic inconsistency between the tests leaf and its implementation leaf.
