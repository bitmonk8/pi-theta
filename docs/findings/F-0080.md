## Finding

The session-context truncation walk bullet in [`V11b-T`](../../../docs/plan_topics/V11b-T-bind-context-transcript.md) closes its lead clause with the parenthetical *"(do not couple to Pi's real estimation heuristic)"*. The same clause already names the mechanism that makes coupling impossible: the walk is driven by a `FakeTokenEstimator` (PIC-16) stub that "assigns chosen per-message counts." Once the test substitutes a fake estimator with author-chosen counts, there is no path by which it could consume Pi's real estimation heuristic; the parenthetical restates a constraint the preceding clause already enforces.

The aside is also methodology guidance — it tells the test author how to avoid a wrong implementation rather than stating an additional behaviour the system must exhibit. The four boundary vectors that follow are the substantive content; the parenthetical adds nothing a reader needs.

The defect is confined to `V11b-T`. The paired implementation leaf `V11b` describes the same truncation walk in its **Tests** bullet but does not carry the parenthetical, so no edit to `V11b` is required.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 3763100 — pi-loom plan: resolve "V11b session-context truncation boundary has no asserting test" (2026-06-11, Thomas Andersen)
**History:** The session-context truncation walk bullet did not exist in `V11b-T` until commit 3763100, which added the entire bullet — the `FakeTokenEstimator` lead clause, the four boundary vectors, and the parenthetical aside — in one edit while resolving a prior review finding about the boundary having no asserting test. The redundant methodology aside entered the corpus with that bullet.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V11b-T-bind-context-transcript.md`, strike the parenthetical `(do not couple to Pi's real estimation heuristic)` from the session-context truncation walk **Tests** bullet, so the lead clause reads `… each vector asserts the expected included-turn set/count:` directly before the four boundary sub-bullets. Leave the `FakeTokenEstimator` (PIC-16) clause and the four vectors unchanged. No edit to `V11b` is needed — it does not carry the parenthetical.

## Consequence

**Severity:** cosmetic

A test author following the bullet still produces the correct test: the `FakeTokenEstimator` is already specified, so the parenthetical changes no behaviour and misleads no one. It is pure wording cruft that adds methodology noise to an otherwise content-only acceptance bullet.
