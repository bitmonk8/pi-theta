## Problem

The fragment id `no-additional-ceilings` is defined on two pages: the canonical `<a id="no-additional-ceilings"></a>` in `hard-ceilings/ceiling-invariants-and-audit.md` (which owns the NOCEIL-1 … NOCEIL-4 non-existence claims and the four-axis audit closure) and a content-free duplicate on the compound anchor line of the "Hard ceilings" aggregator paragraph in `overview-and-orientation.md`, which only summarises and forward-links the ceiling set. Because the two ids live in different files, inbound NOCEIL-3 citations have drifted across them: `diagnostics/code-registry-runtime.md` and `errors-and-results/error-model.md` cite the overview copy and land readers on the summary, while `binder/determinism-cancellation-failure.md` and `pi-integration-contract/subagent.md` cite the canonical page. A reviewer auditing the NOCEIL closure through the former citations never reaches the normative claim text.

**Verdict:** single-commit

**History:** Commit cc240be first added a single `<a id="no-additional-ceilings"></a>` directly on the "No additional V1 runtime ceiling applies" definition in the then-monolithic `spec.md`. Commit 1c7f550 moved that definitional content out to the dedicated `hard-ceilings.md` page (adding the canonical anchor there) but deliberately retained a forwarding `<a id>` stub on the orientation paragraph's compound anchor line in `spec.md` — explicitly weighing "preserve them as forwarding `<a id>` stubs … or sweep all citing leaves" and choosing to keep the stub — which created the duplicate; inbound citations were never consistently re-aimed. Later restructuring (81bf52a split the page into `overview-and-orientation.md`; f5e89f4 split the owner into `ceiling-invariants-and-audit.md`) only relocated the two coexisting copies.

## Solution approach

Delete the `<a id="no-additional-ceilings"></a>` stub from the compound anchor line in `overview-and-orientation.md` so the id is defined only in `hard-ceilings/ceiling-invariants-and-audit.md`. Repoint the two NOCEIL-3 links that currently target the overview copy — the `loom/runtime/internal-error` row link in `diagnostics/code-registry-runtime.md` and the NOCEIL-3 partition link in `errors-and-results/error-model.md` — to `ceiling-invariants-and-audit.md#no-additional-ceilings`, adjusting the relative path per each file's location.
