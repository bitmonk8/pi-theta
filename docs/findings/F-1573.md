## Finding

The fragment id `no-additional-ceilings` is defined in two different spec pages. The canonical definition is `<a id="no-additional-ceilings"></a>` in `hard-ceilings/ceiling-invariants-and-audit.md` (the "No additional loom 1.0 runtime ceiling applies" section that owns the NOCEIL-1 … NOCEIL-4 non-existence claims and the four-axis audit closure). A second, content-free copy rides a compound anchor line in `overview-and-orientation.md` — `<a id="ceiling-interaction-order"></a><a id="no-additional-ceilings"></a>…` — at the head of the "Hard ceilings" orientation aggregator paragraph, which only *summarises and forward-links* the ceiling set; it does not state the NOCEIL claims.

Because the two ids live in different files, inbound references resolve to whichever page-path the link uses, and they have drifted apart. `diagnostics/code-registry-runtime.md` and `errors-and-results/error-model.md` both cite NOCEIL-3 via `../overview-and-orientation.md#no-additional-ceilings`, landing the reader on the orientation summary rather than the normative NOCEIL-3 text; `binder/determinism-cancellation-failure.md` (NOCEIL-1) and `pi-integration-contract/subagent.md` (NOCEIL-3) correctly cite `…/hard-ceilings/ceiling-invariants-and-audit.md#no-additional-ceilings`. A reviewer auditing the load-bearing NOCEIL closure claims by following the code-registry-runtime / error-model citations never reaches the actual claim text, while one following the binder / subagent citations does — both citing "the same" id.

The overview paragraph's NOCEIL anchor is a forwarding stub left behind when the definitional content was relocated to the dedicated ceilings page; the overview paragraph already forward-links the canonical `ceiling-invariants-and-audit.md#no-additional-ceilings` itself, confirming the dedicated page is the intended owner.

## Solution Space

**Shape:** single

### Recommendation

Delete the `<a id="no-additional-ceilings"></a>` stub from the compound anchor line in `overview-and-orientation.md` (leaving `<a id="ceiling-interaction-order"></a>` to be handled by its sibling finding), so the id is defined only in `hard-ceilings/ceiling-invariants-and-audit.md`. Repoint the two inbound references that currently target the overview copy — the `loom/runtime/internal-error` row's NOCEIL-3 link in `diagnostics/code-registry-runtime.md` and the NOCEIL-3 partition link in `errors-and-results/error-model.md` — to `../hard-ceilings/ceiling-invariants-and-audit.md#no-additional-ceilings` (adjusting the relative path per each file's location). The `determinism-cancellation-failure.md` and `subagent.md` citations already target the canonical page and need no change. Edge case: if any future link must point at the overview "Hard ceilings" aggregator paragraph specifically, give that paragraph its own distinct anchor rather than re-coining the owner page's id.

## Consequence

**Severity:** advisory

Citations of NOCEIL-3 from `code-registry-runtime.md` and `error-model.md` land readers on the orientation summary paragraph instead of the normative NOCEIL-1 … NOCEIL-4 claims, so anyone auditing or implementing against those claims through those links must make an extra hop or may mistake the summary for the definition. The spec stays implementable; the harm is navigational and weakens the traceability of the load-bearing closure claims.
