## Finding

The phrase *delegated live-carrier witness* (and its bare *live-carrier* / *live carrier* variants) is used as if it were an established plan convention. It appears in nine leaf files — `V4c`, `V4e`, `V4f`, `V9i`, `V13c`, `V14a`, `V14e`, `V15a`, `V15j` — in the coverage-matrix `ERR-13` closing-leaf annotations (`coverage-matrix.md` line 20), and in two `plan.md` slice-link titles (`V14e`, `V15j` tagged "live carrier"). The construction "delegated live-carrier witness for `<leaf>`'s `<routing row>`" reads as a citation to a defined term.

No definition exists. `conventions.md` — the home of the cross-cutting test/witness vocabulary (`## Cross-cutting rules (every leaf)`, `## Leaf format`) — has no entry for it, and neither does `plan.md` or the spec. The reader is left to infer the pattern's contract: whether a live-carrier witness must re-assert the originating leaf's full observable outcome on the live production surface, or merely witness reachability of that surface. The current usage sites are in fact internally consistent — each spells out the concrete observable assertion inline (e.g. `V14a`'s `ERR-13` bullet pins "leaves the completed callee's side effect in place with no compensating turn injected"; `V14e` pins the `Err(CodeToolError { cause: "validation" })` / `schema_keyword: "maxDepth"` surface) — so the contract is recoverable today by reading each site. The gap is that the term carries no anchoring definition, so a future leaf that cites the pattern without restating the full observable assertion has nothing to bind against, and a reader must reconstruct the contract from the union of existing sites rather than from one authoritative statement.

## Solution Space

**Shape:** single

### Recommendation

Add a definition of the *delegated live-carrier witness* pattern to `docs/plan_topics/conventions.md` under `## Cross-cutting rules (every leaf)` (alongside the existing test/witness vocabulary), so every citing leaf binds against one authoritative statement. The definition must establish that a delegated live-carrier witness is a leaf whose live-integration test carries the production-path re-assertion the originating (delegating) leaf deferred — asserting the same observable outcome on the live production surface, not merely witnessing that surface's reachability. State explicitly that this is the full-observable-outcome reading, since that is the contract the existing `V14a` / `V13c` / `V15a` / `V14e` / `V15j` / `V9i` / `V4e` usage sites already follow.

Cross-reference the definition from the place co-witness annotations are authored: `coverage-matrix.md`'s `ERR-13` row and closing-leaf-column authoring note already use the descriptor "delegated … live-carrier" inside `co-witness` annotations, so pointing those annotations (or the matrix's closing-leaf authoring preamble) at the new `conventions.md` definition keeps the two vocabularies reconciled.

Implementer notes: the term is distinct from the `coverage-matrix.md` `co-witness` annotation token (a co-witness drops a leaf from the facet partition for the `H5d` transitive-completeness gate; a live-carrier witness is a property of the witnessing test's scope) — keep the two concepts separable in the definition. The existing usage sites need no edit once the definition lands; they already carry their concrete per-site assertions.

## Consequence

**Severity:** advisory

The undefined term forces every reader of an affected leaf to reconstruct the live-carrier witness contract from the surrounding inline prose rather than from one authoritative definition, and gives a future leaf that cites the pattern (without restating the full observable assertion) nothing to bind its test scope against — admitting a weaker reachability-only test where a full observable re-assertion was intended. No current leaf is blocked: each present usage spells out its concrete assertion, so today's implementers can still produce working leaves.
