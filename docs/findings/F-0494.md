## Finding

`H7a`'s **Adds** paragraph carries the full definition of the committed permitted-code list: it is a manually-maintained, best-effort union of each Deps slice's emittable codes, kept current by a per-Deps-slice provenance obligation; with no emittable-code registry to verify against, its completeness is not mechanically verified, so a stale union can omit a benign code a slice emits; only the `golden ⊆ permitted` containment is mechanically gated.

The third **Tests** bullet (the "**committed permitted-code list**" bullet) restates the same definition almost verbatim — the manually-maintained/best-effort framing, the per-Deps-slice provenance model, the deliberately-broader/superset rationale, the not-mechanically-verified completeness limitation, and the cross-gate role (the reference set `H4a`'s real-host smoke criterion (e) and `H6a`'s release-gate record check against). The two passages now state the same provenance model and the same limitation sentence in two places on the same page, with no single authoritative home.

The genuine test obligation of that third bullet is narrow: that a committed permitted-code list is checked in alongside the fixture `.loom`, adjacent to the golden transcript and golden diagnostics list, and is the shared reference set criterion (e) / `H6a` read. The remaining definitional prose is a duplicate of the Adds definition and is a divergence surface — a future edit to the limitation or provenance wording in one place silently leaves the other stale. (The `golden ⊆ permitted` containment assertion lives in the fourth Tests bullet, which is unaffected.)

## Solution Space

**Shape:** single

### Recommendation

Keep the full definition in `H7a`'s **Adds** paragraph (the authoritative home), and in the third **Tests** bullet retain only that bullet's test obligation — that a committed permitted-code list is checked in alongside the fixture `.loom`, adjacent to the committed golden transcript and golden diagnostics list, and is the single committed reference set `H4a`'s real-host smoke pass criterion (e) and `H6a`'s release-gate evidence record check the live run's emitted codes against.

Strike from the third Tests bullet the prose the Adds paragraph already owns: the "manually-maintained, best-effort union … kept current by a per-Deps-slice provenance obligation rather than verified against an emittable-code registry" definition, the "deliberately broader … intended as a **superset**" rationale, and the "Its completeness … is **not** mechanically verified, so a stale union can omit a benign code a slice emits; only the `golden ⊆ permitted` containment is gated …" limitation sentence. Where the bullet needs to invoke the definition, point to the Adds paragraph rather than re-stating it.

Leave the fourth Tests bullet (the `golden ⊆ permitted` containment check) and the Adds paragraph unchanged.

## Consequence

**Severity:** cosmetic

The permitted-code list's definition is stated twice on the H7a page; an edit to the provenance model or the completeness-not-verified limitation in one location can leave the other stale and contradictory. No implementer is blocked and no two implementers diverge today — the duplicated text is currently consistent — so the cost is a maintenance divergence surface, not a correctness gap.
