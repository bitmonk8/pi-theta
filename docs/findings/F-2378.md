## Problem

`V9b-T`'s "model-reference-matcher production wiring" Tests bullet asserts that the load pass injects the concrete matcher into the model-reference-matcher injection seam `V6a` defines, and that `V6a`'s `loom/load/model-unresolved` resolution binds that same instance (instance identity). Compiling that assertion requires `V6a`'s seam interface to exist, yet `V9b-T`'s `Deps.` line lists only `V9a`, `V10a`, `V8e` — `V6a` is absent. The pickability predicate (`conventions.md` §Leaf format **Deps**) therefore reads `V9b-T` as pickable before `V6a` ships, so the assertion would fail to compile rather than fail red for its intended behavioural reason. The paired `V9b` implementation leaf already carries `V6a` in its Deps, so the omission is unique to the tests task.

**Verdict:** present-since-inception

That same commit's diff added both the "model-reference-matcher production wiring" Tests bullet (which references the injection seam `V6a` defines and asserts instance identity) and the `**Deps.** V9a, V10a, V8e` line. `V6a` was never present in `V9b-T`'s Deps at any point (`git log -S "V6a" -- V9b-T-registration-reload-wiring.md` returns only `44a96f5`). Later commits (`3424d54`, `2424003`, `e084853`) refined the matcher wording but did not touch the Deps omission. The defect is original to the file.

## Solution approach

Add `V6a` to the `**Deps.**` line in `docs/plan_topics/V9b-T-registration-reload-wiring.md`, mirroring the paired `V9b` implementation leaf which already lists it.
