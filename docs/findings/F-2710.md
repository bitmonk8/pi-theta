## Finding

`H2a` ("Cross-cutting lint and architectural gates") packages four mechanically distinct enforcement surfaces into a single leaf, each with its own rule body, its own fixture, and its own Tests bullet:

1. the `no-broad-catch` ESLint rule (the *Specific exception types only* convention);
2. the `no-restricted-syntax` sequential-by-default ban on `Promise.all`/`race`/`allSettled`/`any` (the *Sequential by default* convention);
3. the `no-restricted-syntax` blocking-runtime ban on `*Sync`-suffixed callees (also *Sequential by default*);
4. the `src/**` architectural AST test that fails on a module-level mutable binding (the *No globals, statics, singletons* convention).

These are independent mechanisms wired into one `npm test` invocation behind one `Ships when` gate. They share no implementation: each has a separate ESLint rule or AST scan and a separate fixture. A red `npm test` on one of the four does not implicate the others, and the leaf cannot ship any one gate independently of the rest. The four collapse into a single execution unit only by editorial grouping, not by a shared mechanism that forces them to land together.

## Consequence

**Severity:** advisory

An implementer working `H2a` under the per-phase TDD ritual must land four unrelated lint/architectural mechanisms and four fixtures in one task behind one `Ships when` gate, and a `npm test` failure does not isolate which gate broke. The four gates still get built and wired, so the leaf produces a working result — this is an atomicity/organisation gap, not a pickability or correctness defect.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review"); bundle widened from three gates to four at `c33d309` (2026-06-17, "resolve 'Blocking-runtime ban concedes enforcement to manual review'")
**History:** The first plan-build commit `c6a664e` already authored `H2a` as a single leaf bundling three distinct gates (`no-broad-catch`, the `no-restricted-syntax` sequential ban, and the singleton architectural test). The fourth gate — the `*Sync` blocking-runtime ban — was added later at `c33d309` (refined at `8561a7c`, 2026-06-18), widening the bundle but not introducing the multi-gate bundling premise, which was present from inception. Subsequent commits on the file (`d56d65f`, `3d64f7f`, `506d71c`, `cf47e6f`, `30ee609`, `33f9ee5`) refined individual gate bodies and never revisited the bundling.

## Solution Space

**Shape:** single

### Recommendation

Decompose `H2a` into separate leaves, one per cohesive convention, so each gate ships and reds independently:

- A leaf owning the `no-broad-catch` ESLint rule body, its broad-catch fixture and Tests bullet, and a `Ships when` covering the broad-catch rejection (the *Specific exception types only* convention).
- A leaf owning **both** `no-restricted-syntax` rules — the `Promise.all`/`race`/`allSettled`/`any` sequential-by-default ban and the `*Sync` blocking-runtime ban — with their two fixtures and a `Ships when` covering both (both belong to the *Sequential by default* convention and share the `no-restricted-syntax` engine).
- A leaf owning the module-level-mutable-binding architectural AST test, its fixture, and a `Ships when` covering the mutable-binding rejection (the *No globals, statics, singletons* convention).

Use `<new>` placeholder IDs for the net-new leaves (the implementer assigns final IDs per the plan's leaf-ID scheme); the implementer may instead repurpose `H2a` itself as one of the three groups to reduce reference churn. Each resulting leaf keeps `Deps. H1a` and wires its own assertions into `npm test`.

Retarget every existing reference to `H2a` to whichever split leaf now owns the cited gate: `H3a`'s `Deps. H2a`; the `docs/plan.md` §Horizontal-phases listing entry; the `conventions.md` prose that attributes each gate to `H2a` (the *Specific exception types only*, *Sequential by default*, and *No globals, statics, singletons* bullets, the §"mechanical gates scoped to `src/**`" sentence, and residue-inspection items 1/3/6); and the `H1a` lint-toolchain test bullet.

Edge cases for the implementer: the spec corpus is read-only for this fix — no spec edit is needed. The `// allow-broad-catch:` and `// allow-sync:` allow-list semantics and the `H5c` broad-catch allow-list reconciliation read `src/**` directly and are unaffected by which leaf owns the rule. `coverage-matrix.md` carries no `H2a` row (horizontal leaves cite a Convention, not a REQ-ID), so no matrix edit is required.
