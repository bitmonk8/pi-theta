## Problem

The spec page `binder/determinism-cancellation-failure.md` §Failure-mode templates (normative) (anchor `#failure-mode-templates-normative`) carries an un-anchored normative MUST — renderers MUST emit the surrounding template text verbatim, interpolating only the `<…>` placeholders — that owns the six verbatim user-facing system-note templates, the `<provider>` / `<ajv-summary>` placeholder-derivation pins, and the depth-walk fast-fail clause for the non-cancelled failure classes (transport / malformed-envelope / AJV-on-`args` / context-overflow). It is a GOV-22 un-anchored-MUST residue, but `coverage-matrix.md`'s *Code-keyed obligation areas* table has no row for it: `cka-42` covers §Determinism (→ `V11e`) and `cka-43` covers §Cancellation plus the cancelled-binder row (→ `V11j`), neither of which names the templates MUST or the depth-walk synthesis clause. `V11f` already implements the obligation but is associated only by markdown link rather than a citable `cka-<n>` token, so the `H5e` un-anchored-MUST closing-gate arm reddens against the live corpus once `H6a` activates.

**History:** The `§Failure-mode templates` section has existed on `determinism-cancellation-failure.md` since at least the 2026-06-04 spec size-cap refactor (`f5e89f4`). The plan's coverage-matrix mapped the page's obligations in two later passes — `12f7933` added the `§Determinism` row (→`V11e`) and `66b3815` added the `§Cancellation + §Failure modes (cancelled-binder row)` row (→`V11j`) — each covering an adjacent slice of the same page. Neither pass mapped the `§Failure-mode templates` verbatim-rendering MUST or the depth-walk fast-fail synthesis clause for the non-cancelled classes, so the gap is the cumulative result of two partial-mapping commits leaving the templates obligation uncovered.

## Solution approach

Add one append-only row to the *Code-keyed obligation areas* table in `coverage-matrix.md` for `binder/determinism-cancellation-failure.md` §Failure-mode templates — the verbatim-template-rendering MUST plus the depth-walk fast-fail `<ajv-summary>` synthesis clause for the non-cancelled classes — with closing leaf `V11f`. Then add a `V11f` Tests-bullet citation of the new `cka-<n>` token so the closing-gate citing-test reconciliation associates `V11f` to the row.

## Solution constraints

- Out of scope: `cka-42` (§Determinism → `V11e`) and `cka-43` (§Cancellation + cancelled-binder row → `V11j`) stay unchanged — the new row covers only the non-cancelled-class template-rendering obligations; the spec is read-only for this fix.
- The *Token* column is append-only: take the next free `cka-<n>` and never reuse a token; the new row's closing-leaf cell is the bare backtick-delimited `` `V11f` `` token only (no other backtick span), per the *Closing-leaf column authoring* rule.
