## Finding

The `H5a` closing-gate **citing-test reconciliation** arm passes a `coverage-matrix.md`-mapped numbered REQ-ID as soon as some test source contains the inline `PREFIX-N` token. `conventions.md` §REQ-ID discipline and `H5a` both correctly and explicitly scope this arm to *existence*, not *fidelity* ("it certifies that a citing test exists for each mapped REQ-ID, not that the cited test's assertion is semantically faithful to the obligation"). The gate therefore does **not** overclaim — it states its own limit. So the live-corpus release gate (`H6a`) can stay green over a cell whose citing test is vacuous or mis-asserting, by design.

The plan does route the residual faithfulness obligation: it states faithfulness "stays a TDD and self-review obligation under the per-phase ritual." The gap is in *where* that routing lands. The per-phase self-review (`conventions.md` §Per-phase TDD ritual, step 3 — "any rule unverified?") is performed by the same agent that authored the (possibly vacuous) test, so it is a self-check, not an independent one. The plan's designated **independent** release-time residue surface — the `governance.md` GOV-15 reviewer-inspection step, to which the *No globals*, *Sequential by default* blocking-runtime, *Doc updates*, and un-anchored-MUST residue classes all route — does **not** currently list citing-test assertion faithfulness among the classes it samples.

Net: the only backstop against a token-citing-but-non-asserting test is the authoring agent inspecting their own work. No independent sampling of mapped REQ-IDs for assertion faithfulness occurs at the release gate. (The original concern's other half — "require each Tests bullet to name its observable assertion" — is already satisfied: `leaf-template.md` and `conventions.md` §Leaf format already require each Tests bullet to state what its test asserts, one line per REQ-ID.)

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/conventions.md` §REQ-ID discipline, extend the residue routing of the citing-test reconciliation paragraph so that citing-test assertion faithfulness is sampled by the independent `governance.md` GOV-15 release-time reviewer inspection, not only by the authoring agent's per-phase self-review. Concretely: alongside the existing "faithfulness stays a TDD and self-review obligation under the per-phase ritual" sentence, add that the GOV-15 reviewer inspection samples `coverage-matrix.md`-mapped numbered REQ-IDs and confirms each cited test actually asserts its obligation rather than merely citing the token — placing this residue on the same independent surface the *No globals*, *Sequential by default* blocking-runtime, *Doc updates*, and un-anchored-MUST residues already route to.

Keep the existing self-review sentence as the authoring-time obligation; the addition supplies the independent backstop. Do not weaken or re-scope the citing-test gate arm or the transitive-completeness MUST — both are correct as written, and the gate's existence-not-fidelity scope is deliberate. No leaf-format change is needed: `leaf-template.md` and §Leaf format already require each Tests bullet to state what its test asserts.

This item is one entry of the GOV-15 checklist enumeration requested by the related "GOV-15 manual reviewer-inspection residue has no in-plan checklist" finding; resolve them together so the checklist is enumerated once.

## Consequence

**Severity:** advisory

A vacuous or mis-asserting test that cites the `PREFIX-N` token keeps its coverage-matrix cell green at the loom 1.0 release gate, and the only backstop is the authoring agent's own self-review — so a faithfulness defect (test present, obligation not actually verified) can ship undetected, giving the coverage signal more apparent assurance than any independent check delivers. Implementers can still produce working leaves; this is a defence-in-depth assurance gap, not a blocker.
