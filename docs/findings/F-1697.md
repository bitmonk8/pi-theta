## Problem

PIC-8(b) and its `loom/runtime/active-set-restore-failed` registry row both require the diagnostic's `hint` to list the snapshot tool names, but the `Diagnostic` shape pins `hint?: string` and neither site says how the `string[]` of tool names is serialised into that single line. Plausible encodings (comma-and-space-joined, newline-joined, JSON-array-stringified, whitespace-joined) are all unprivileged, so two conforming implementations emit divergent `loom-system-note` transcript bytes for the same restore failure and diverge on any test asserting `hint`. PIC-8(b) is the only row in the registry whose `hint` is a multi-valued list with no encoding rule; the sibling rows each pin their `hint` content concretely.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 2d2fc8e (2026-05-06, "pi-loom spec: resolve \"Pi SDK capability calls have no failure contract\"")
**History:** Commit 2d2fc8e added both the PIC-8 paragraph and the `loom/runtime/active-set-restore-failed` registry row in their original form, including the verbatim phrase "`hint` carrying the snapshot tool names so an operator can manually restore via Pi's `/tools` interface". The defect — naming the field's contents but not its encoding — was present from that introduction. Commit bef37f3 (2026-06-25, "fix F-1228 — pin registry-persistence presupposition, narrow PIC-8(b) /tools hint") trimmed the trailing "/tools interface" rationale clause but kept the encoding gap. No intermediate commit altered the field's specification.

## Solution approach

Clarify the `hint` encoding in PIC-8(b) and mirror it in the `loom/runtime/active-set-restore-failed` registry row's *Trigger* cell, aligning the delimiter with the existing `reload-teardown-timeout` `<list>` comma-and-space convention already pinned in the same registry so consumers parse both rows under one rule. Pin enough of the encoding (delimiter, snapshot iteration order, empty-snapshot rendering) that the resulting `hint` byte-string is deterministic. Add a worked rendering vector in `diagnostics/placeholder-rendering-b.md` alongside the existing `active-set-restore-failed` Message vector, and carry a V9f fixture asserting the `hint` byte-string for a multi-element snapshot.
