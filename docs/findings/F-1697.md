## Finding

PIC-8(b) in `pi-integration-contract/tool-registration-lifetime.md` requires the runtime to emit `loom/runtime/active-set-restore-failed` with "`hint` listing the snapshot tool names", and the registry row in `diagnostics/code-registry-runtime.md` repeats the same phrase. The `Diagnostic` shape pins `hint?: string` (`diagnostics/diagnostic-shape.md`), so the producer has to serialise a `string[]` into a single line — but neither site says how. Plausible encodings include comma-and-space-joined (`"foo, bar, baz"`, the convention `loom/runtime/reload-teardown-timeout` uses for its `<list>` placeholder), newline-joined, JSON-array-stringified (`'["foo","bar","baz"]'`), or whitespace-joined; none is privileged.

Three observable consequences follow. (1) `hint` is rendered into the serialised `loom-system-note` `content` as `"\n  hint: <hint>"` (`diagnostics/diagnostic-shape.md`, *Serialised content format*), so two conforming implementations produce divergent transcript bytes for the same restore failure. (2) Plan leaf V9f's `PIC-8` test asserts the diagnostic's emission; per DIAG-4 the Message template is byte-stable, but `hint` is left to the test author's reading of an unpinned phrase, so equivalent implementations diverge on test outcome. (3) Tool names containing the chosen delimiter (Pi extension tool names are content-addressed and may include `_` and digits, but a future Pi minor or a foreign extension's name with a comma, space, or newline is not excluded by the active-set-membership rule) silently corrupt an operator's reconstruction of the snapshot.

The sibling rows in the same registry pin their `hint` content concretely: `reload-teardown-timeout` mandates "a bare decimal integer with no unit suffix" with a worked example; `subagent-dispose-failure`, `system-note-delivery-failed`, and `internal-error` each pin `hint` to a single named underlying string (the dispose error's message, the `sendMessage` error's message, the `error.stack`). PIC-8(b) is the only row in the table whose `hint` is a multi-valued list with no encoding rule.

## Solution Space

**Shape:** single

### Recommendation

Pin the `hint` encoding inline in PIC-8(b) and mirror it in the `loom/runtime/active-set-restore-failed` registry row's *Trigger* cell. Specify: `hint` is the snapshot tool names joined by the two-byte literal `", "` (comma-then-space), in the iteration order returned by the step-1 `pi.getActiveTools()` call (i.e. the verbatim `string[]` order, not a re-sort), with no surrounding brackets and no quoting of individual names. An empty snapshot renders `hint` as the empty string (the `hint` field is still emitted; this is distinct from omitting it). This matches the existing `reload-teardown-timeout` `<list>` convention already pinned in the same registry, so consumers parsing both rows use one delimiter rule.

Add the corresponding worked vector in `diagnostics/placeholder-rendering-b.md` alongside the existing `active-set-restore-failed` Message vector: a snapshot of `["alpha", "beta", "gamma"]` renders `hint` as `"alpha, beta, gamma"`. Carry one fixture in V9f-T / V9f that asserts the `hint` byte-string for a multi-element snapshot.

Edge cases the implementer must watch:

- A tool name containing `", "` would corrupt a delimiter-splitting reader. The Pi SDK does not constrain tool-name characters at the type level, but the loom-synthesised names follow `__loom_callee_<slug>__<post-rename-name>` / `__loom_respond_<slug>` (`tool-registration-lifetime.md`) and the slug is a 64-bit SHA-256 truncation, so a delimiter byte cannot appear in a loom-synthesised name. A foreign tool name containing `", "` is not excluded; the encoding rule accepts the resulting ambiguity (consistent with `reload-teardown-timeout`'s identical posture for invocation ids).
- The `hint` field is emitted as the empty string for an empty snapshot rather than omitted; this preserves the wire-shape invariant that `active-set-restore-failed` always carries `hint`, and the *Serialised content format* renderer produces `"\n  hint: "` (trailing space) in that case — accept that rendering rather than special-casing the empty list.
- The captured snapshot is the `string[]` produced by step-1 `pi.getActiveTools()`; the runtime MUST NOT re-sort or deduplicate before joining (the post-probe SDK-shape drift rule already requires preserving the exact returned array).

## Consequence

**Severity:** correctness

Two reasonable implementers will pick different list encodings; their `loom-system-note` content lines differ byte-for-byte for the same restore failure, and any test asserting `hint` content (which V9f's `PIC-8` leaf is set up to do) will pass under one implementation and fail under the other. Operators piping the transcript through tooling that splits `hint` on a presumed delimiter will silently lose entries when a tool name contains that delimiter.
