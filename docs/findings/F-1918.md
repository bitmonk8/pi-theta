## Finding

`V4f`'s **Ships when** field carries two sentences. The first states the testable exit gate: "`npm test` proves the enumerated `ERR-13` cases hold — a completed callee stays final after a downstream terminal event across the sampled authoring sites, with no side effect unwound and no compensating turn injected." The second sentence — "The no-rollback guarantee is architectural: it rests on the runtime containing no compensating/rollback path, and the tests witness it on the enumerated cases rather than exhaustively." — is a near-verbatim repeat of the architectural rationale already stated in `V4f`'s **Adds** field ("The no-rollback guarantee is architectural: the runtime contains no compensating path, so a completed callee's side effect survives a downstream terminal event by construction rather than by an undo being suppressed.").

`conventions.md` §Leaf format defines **Ships when** as "A concrete, externally observable change." Architectural rationale is not an observable change; it belongs in **Adds**, where it already lives. The trailing sentence is duplicated rationale occupying the exit-gate field.

The redundancy was authored with the leaf; no later commit introduced or amplified it.


## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V4f-no-rollback.md`, strike the trailing sentence from the **Ships when** field — "The no-rollback guarantee is architectural: it rests on the runtime containing no compensating/rollback path, and the tests witness it on the enumerated cases rather than exhaustively." — leaving the **Ships when** field as the single observable-gate sentence ("`npm test` proves the enumerated `ERR-13` cases hold — a completed callee stays final after a downstream terminal event across the sampled authoring sites, with no side effect unwound and no compensating turn injected."). The architectural rationale is retained, unchanged, in the **Adds** field; do not move or restate it. The paired `V4f-T` is not touched — its **Ships when** ("The tests above exist, compile, and fail red for the intended reason.") carries no rationale.

## Consequence

**Severity:** cosmetic

The actual exit gate (the first sentence) is intact and observable, so no implementer is misled about when the leaf ships. The duplicated rationale is pure cruft in a field that `conventions.md` reserves for an externally observable change.
