## Finding

The step-5 sweep (`version-bump-triggers.md`) greps the fixed superset `src/`/`test/`/`docs/` and justifies it only against future subdirectories and `docs/` siblings, never asserting that all runtime references live under those three roots; a new top-level root is silently missed.

**Spec edits:** In the *Enumeration mechanism — grep target* clause of `version-bump-triggers.md` step 5, either (a) state the invariant explicitly ("all loom runtime/source/test/doc references MUST live under `src/`, `test/`, or `docs/`; introducing a new top-level root that holds such a reference MUST extend this grep-root set in the same edit"), or (b) redefine the grep root as repo-root-relative minus a named vendored/excluded set (e.g. minus `node_modules/`, `dist/`), so a new top-level root is covered by construction. Pick one and apply it to both the contributor-side recipe and the optional post-1.0 audit-pass description.

**Relationship:** same-cluster — resolves independently at `version-bump-triggers.md` step 5.

## Consequence

**Severity:** correctness

Each precondition admits silent divergence or a vacuous pass: two implementers disagree on what an absent `engines.node` field does at the floor gate; a runtime reference under a future top-level root is silently elided by the step-5 sweep; the step-2(b) tie-break apparatus is inert under contributor self-merge; and an empty fixture corpus turns a re-validation gate green without exercising anything. None blocks the procedure from being implemented, but each lets a real regression land undetected.
