## Finding

The REQ-ID projection (`ids.md`) registers a definition site for every bold `**PREFIX-N**` span it finds, with no way to distinguish a *definition* (a bolded label co-located with an `<a id>` anchor) from a *back-reference* (a bolded mention that cites an obligation another page owns). The result is that prose back-references are recorded as canonical anchor locations and reviewers auditing conformance via the index are sent to the wrong section.

Three witnesses confirm the defect:

- **GOV-7 / GOV-8.** The index lists both IDs as defined in `docs/spec_topics/diagnostics/placeholder-rendering-a.md:7` and `placeholder-rendering-b.md:131`. Both are bold back-references (`… governed by **GOV-7** and **GOV-8** …` and `… spec-versioned per **GOV-7** and **GOV-8** …`), not anchors. The real anchors — `<a id="gov-7"></a> **GOV-7 (mutation procedures).**` and `<a id="gov-8"></a> **GOV-8 (REQ-ID lifecycle).**`, both in `docs/spec_topics/governance/req-id-prefix-table-active-b.md` (lines 22 and 36) — are absent from the index because the parenthetical-titled label form does not match the witness's bare-canonical extractor. The only GOV-7/GOV-8 rows in `ids.md` therefore point exclusively at the two placeholder-rendering back-references.
- **ERR-7.** The index lists ERR-7 twice: once at its canonical anchor `docs/spec_topics/errors-and-results/error-model.md:21` (`<a id="err-7"></a> **ERR-7.**`) and once at the bold back-reference `docs/spec_topics/discovery/package-and-settings.md:98` (`… surfaces its rebuild failures as the **ERR-7** pre-evaluation failure defined in [Errors and Results — ERR-7](../errors-and-results/error-model.md#err-7) …`). The back-reference is already hyperlinked to the canonical anchor, yet is still recorded as a second definition site.

The underlying cause is a convention gap: GOV-1 reserves the bold-with-trailing-period form (`**PREFIX-N.**`) for anchored definition sites, but nothing prohibits authors from using the bare bold form (`**PREFIX-N**`) for prose back-references, so the two are typographically indistinguishable and the projection conflates them.

## Solution Space

**Shape:** single

### Recommendation

De-bold the three cited back-references so the projection no longer reads them as definition sites, and close the convention gap that lets the pattern recur.

- In `placeholder-rendering-a.md:7`, `placeholder-rendering-b.md:131`, and `package-and-settings.md:98`, replace each bold `**GOV-7**` / `**GOV-8**` / `**ERR-7**` mention with a plain (non-bold) token, and make each a hyperlink to the canonical anchor: GOV-7 → `../governance/req-id-prefix-table-active-b.md#gov-7`, GOV-8 → `../governance/req-id-prefix-table-active-b.md#gov-8`, ERR-7 → `../errors-and-results/error-model.md#err-7` (the ERR-7 site already carries this link; only the bold span is removed). Reserve the bold form for anchor sites.
- Add one normative clause to the REQ-ID cross-link contract (GOV-9, in `req-id-prefix-table-active-b.md`) stating that REQ-ID *back-references* (citations of an obligation owned by another page) MUST use a non-bold, hyperlinked token, and that the bold label form is reserved for the co-located definition anchor. This makes the projection's definition-vs-reference distinction enforceable and prevents reintroduction.

Edge cases the editor must watch: GOV-7/GOV-8 also appear as bold back-references elsewhere (e.g. `release-version-naming.md`, `stable-inline-labels.md` use the non-bold `GOV-7 / GOV-8` form already — confirm those stay non-bold); sweep for any other bold `**PREFIX-N**` mention that is not a definition anchor in the same pass so the index is clean after regeneration. This fix is independent of, but must stay consistent with, the witness-regex/label-format decision in the related systemic finding (see below): whichever extractor change that finding adopts must continue to exclude non-bold back-references.

## Consequence

**Severity:** advisory

A reviewer or auditor using `ids.md` to confirm a cited REQ-ID has a definition is directed to a back-reference instead of the canonical anchor, and for GOV-7/GOV-8 never sees the real definition site at all. The spec prose itself remains unambiguous, so no implementer diverges on behaviour; the damage is confined to conformance/traceability tooling.
