## Finding

The paired V12a / V12a-T leaves each carry two parallel `SLSH-2` edge-case bullets that describe the same kind of post-prefix-append artifact on two failure paths. The `Err`-propagation bullet names it formally — "the failure `loom-system-note` is appended *after* the prefix" — while the mid-stream-cancellation bullet names it informally — "the cancellation note is appended *after* the prefix". The asymmetry leaves it ambiguous whether "cancellation note" denotes the same `loom-system-note` channel artifact (the spec-owned diagnostics channel defined in `V7a`) or a distinct surface.

The spec resolves the ambiguity in loom's favour: `slash-invocation.md` SLSH-2 describes both edge cases, and the cancellation row `SNK-f` (`cancelled` → "loom `/<name>` cancelled") confirms the cancellation note is emitted on the `loom-system-note` channel. The plan's informal phrasing therefore drops the canonical channel name on one of two structurally identical bullets, where the adjacent bullet uses it. The same informal phrasing is carried a third time into V12a's **Ships when** gate text.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (2026-06-10) — "pi-loom plan: build/update plan for spec.md + review"
**History:** The V12a / V12a-T leaf pair was first authored in `c6a664e`; both the formal "failure `loom-system-note`" bullet and the informal "cancellation note" bullet arrived in that same commit, so the naming asymmetry has existed since the leaves were created. A later commit `1abb15a` (2026-06-11, "resolve 'V12a Ships-when gate omits the SLSH-2 failure-path assertions'") propagated the informal "cancellation note" wording into V12a's Ships-when gate but did not introduce the inconsistency.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V12a-slash-dispatch.md` and `docs/plan_topics/V12a-T-slash-dispatch.md`, change the `SLSH-2` cancellation Tests bullet so the artifact is named on the canonical channel: replace "the cancellation note is appended" with "the cancellation `loom-system-note` is appended", matching the adjacent failure-path bullet's "failure `loom-system-note`". Apply the same replacement to the cancellation clause of V12a's **Ships when** line ("the cancellation note is appended *after* the prefix" → "the cancellation `loom-system-note` is appended *after* the prefix"). The spec text in `slash-invocation.md` is the authoritative source for the canonical channel name and is not edited.

## Consequence

**Severity:** advisory

An implementer reading the cancellation bullet cannot tell from the leaf alone that the cancellation note is the same `loom-system-note` channel artifact as the failure note, and a cross-search for `loom-system-note` misses the cancellation assertion. The spec is unambiguous, so a careful implementer still produces a correct test; the cost is the lost grep traceability and the reader having to infer channel identity.
