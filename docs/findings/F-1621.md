## Finding

Throughout the PIC corpus "seam" is a load-bearing term of art: it names a DI-injected interface that the runtime reads through and that tests substitute against — `Checkpoint` seam (PIC-10), `SchemaValidator` (PIC-11), `Clock` / `FakeClock` (PIC-12), `FileSystem` (PIC-13), `FileWatcher` (PIC-14), `TokenEstimator` (PIC-16), `IdSource` (PIC-20), plus the `Resolver` seam in imports. The vocabulary is so stable that *Seam minimality (normative)* in `session-only-degraded-state.md` uses it again in exactly this sense (the rule forbids "handler-scoped state seams" beyond the two named captured variables).

In `patch-skew-degradation.md` §**Per-step isolation** the phrase **"the interstitial seam between sub-steps 4 and 5"** — and the anchor `#interstitial-seam-tag-write-swallow` it carries — instead labels a *point in the teardown sequence* at which the runtime performs the `"shutting-down" → "degraded-needs-reload"` drain-state tag write (`LoomRegistry.markRuntimeDegraded`). No new DI interface is introduced; nothing is injected; tests do not substitute against it. The label conflates two unrelated concepts: a *DI interface that exists across the runtime's lifetime* and a *control-flow transition that exists for one statement of one handler*.

The collision then propagates: `session-only-degraded-state.md` calls the same site the "interstitial-seam swallow rule" and the "interstitial-seam `try`/`catch`"; `drain-state-contract.md` refers to "the interstitial-seam `LoomRegistry.markRuntimeDegraded` call" and the "interstitial-seam transition reference"; `code-registry-host.md` describes the "seam-tag-write-throw recovery path". Each of these readings would, if "seam" carried its corpus-wide meaning, imply a DI surface the implementation has to expose and the spec has to enumerate — but no such surface exists or is intended. A "tag write between sub-steps 4 and 5" describes the same site without inventing a phantom DI interface.

## Solution Space

**Shape:** single

### Recommendation

Rename "interstitial seam between sub-steps 4 and 5" to "interstitial tag-write between sub-steps 4 and 5" at the definition site in `patch-skew-degradation.md` §Per-step isolation, and rename the anchor `interstitial-seam-tag-write-swallow` → `interstitial-tag-write-swallow`. Update every cross-reference site that still spells "seam" to drop the term where it labels this transition:

- `patch-skew-degradation.md` — the bolded phrase, the anchor, and the later sentence "The interstitial seam's tag write per the **`LoomRegistry` drain-state contract** below carries the fixed label `"loomRegistry.markRuntimeDegraded"` for prose cross-reference only" → "The interstitial tag write per ..."; also "the seam's own dedicated `try`/`catch` swallows silently" → "the tag write's own dedicated `try`/`catch` swallows silently".
- `session-only-degraded-state.md` — "the [interstitial-seam swallow rule]" → "the [interstitial tag-write swallow rule]"; "the [interstitial-seam `try`/`catch`]" → "the [interstitial tag-write `try`/`catch`]"; "on the same handler-scoped seam as the tag write" → "at the same handler-scoped point as the tag write" (or simply "between sub-steps 4 and 5"); §**Isolation** "the tag write by the [interstitial-seam swallow rule]" → "... [interstitial tag-write swallow rule]"; acceptance (f) "interstitial-seam tag write" → "interstitial tag-write"; acceptance (k) heading "Interstitial-seam tag-write throw" → "Interstitial tag-write throw" and the body's "the interstitial-seam `markRuntimeDegraded` write throws" → "the interstitial `markRuntimeDegraded` tag-write throws", and the closing "the [interstitial-seam swallow rule] accepts" → "the [interstitial tag-write swallow rule] accepts".
- `drain-state-contract.md` — "by the [interstitial-seam swallow rule]" → "by the [interstitial tag-write swallow rule]"; "transitioned to `"degraded-needs-reload"` by the interstitial-seam `LoomRegistry.markRuntimeDegraded` call" → "transitioned to `"degraded-needs-reload"` by the interstitial `LoomRegistry.markRuntimeDegraded` tag-write call"; "Called from the interstitial seam between sub-steps 4 and 5" → "Called from the interstitial tag-write between sub-steps 4 and 5"; "the seam's dedicated `try`/`catch`" → "the tag-write's dedicated `try`/`catch`" (twice); the four-method-surface paragraph's "interstitial-seam transition reference" → "interstitial tag-write transition reference"; the **Read-failure fallback** sentence "the interstitial-seam swallow of `LoomRegistry.markRuntimeDegraded`'s own throw" → "the interstitial tag-write swallow of `LoomRegistry.markRuntimeDegraded`'s own throw".
- `code-registry-host.md` — "on the seam-tag-write-throw recovery path" → "on the tag-write-throw recovery path"; "the [interstitial-seam swallow rule]" → "the [interstitial tag-write swallow rule]"; "the same handler-scoped seam as the [`"shutting-down" → "degraded-needs-reload"` drain-state tag transition]" → "the same handler-scoped point as the [...]".

Preserve the legacy anchor by adding `<a id="interstitial-seam-tag-write-swallow"></a>` alongside the new `<a id="interstitial-tag-write-swallow"></a>` at the renamed site, so any downstream link the rename misses (or an external bookmark) still resolves; loom 1.0 anchors are durable wire-style surface and a one-line dual-anchor is cheaper than a missed link.

Edge cases the implementer should watch:
- The DI-seam term is used legitimately elsewhere in the same files (e.g. `Clock` seam references in sub-step 3, the `Checkpoint` seam reference in sub-step 2, the `FileWatcher` seam, *Seam minimality (normative)* in `session-only-degraded-state.md`). Do **not** rename those — only the sites that refer to the post-sub-step-4 tag write.
- The closed `details.call` set in **Per-step isolation** already names the tag write `"loomRegistry.markRuntimeDegraded"`; that label is part of the wire-contract prose cross-reference and must not change.
- The acceptance-criterion bullet identifier (k) is referenced by its letter, not by its heading text; the heading rename is safe.

## Consequence

**Severity:** cosmetic

Mis-using a term of art widens the implementer's interpretation surface: a reader who has internalised "seam = DI interface" from PIC-10 through PIC-20 has to either invent a new DI surface (and then discover it does not exist) or learn that "seam" is overloaded in this one location. The behavioural contract — the dedicated `try`/`catch` around the tag write, the silent swallow, the call-site label `"loomRegistry.markRuntimeDegraded"` carried for cross-reference only — is fully specified and unambiguous, so no observer diverges.
