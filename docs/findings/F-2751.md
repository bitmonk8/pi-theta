## Finding

`V2b`'s **Adds** promises "a runtime AJV safety-net for statically-unresolvable operands," and its **Ships when** condition states `npm test` "leaves statically-unresolvable operands to the runtime AJV safety-net." This branch is normatively required: `type-system.md` §"Unresolvable operands" specifies that when either side of a `T₁ ⊑ T₂` check is past the parser's static view (an inferred binding whose RHS depends on a Pi-tool call schema not visible at parse time, or an `invoke` against a callee that produced `loom/load/callee-has-errors`), the parse-time `*-type-mismatch` check is skipped and the runtime AJV check is the safety net.

The eleven test bullets `TYPE-1…TYPE-11` all assert *static* compatibility outcomes — reflexivity, numeric narrowing, union widening, array covariance, per-site mismatch codes, nominal/alias behaviour. None exercises the unresolvable-operand path: that a statically-unresolvable operand bypasses the parse-time `*-type-mismatch` codes and a violating value is instead caught at the runtime AJV check.

Consequently the leaf can satisfy its **Ships when** gate with the entire safety-net branch unimplemented or silently broken — `npm test` is green because every authored assertion is about the static path it does implement.

## Consequence

**Severity:** correctness

The safety-net branch the leaf claims to deliver has no observable gate, so two reasonable implementers diverge: one wires the skip-and-defer-to-AJV path, another never implements it and still ships green against `TYPE-1…11`. A loom whose unresolvable-operand path is missing would surface no diagnostic where the spec requires the runtime AJV check to catch a type violation, and the gap is invisible until a downstream slice (or a real `.loom`) hits an operand past the parser's static view.

## Solution Space

**Shape:** single

### Recommendation

Add a test bullet to `V2b`'s **Tests** list asserting the safety-net path: a statically-unresolvable operand — e.g. an inferred binding whose RHS depends on a Pi-tool-call schema not visible at parse time, or an `invoke` against a callee that produced `loom/load/callee-has-errors` — bypasses the parse-time `*-type-mismatch` codes, and a value violating the expected type is instead caught at the runtime AJV check. Cite `type-system.md` §"Unresolvable operands" (and its cross-reference to the schema-subset lowering algorithm) as the closed obligation.

If the implementer determines this new citing test closes a coverage-matrix-mapped obligation or an un-anchored normative MUST, the existing `conventions.md` §REQ-ID discipline (*Transitive-completeness plan-maintenance*) and How-to-use step 2 already govern updating `coverage-matrix.md`'s `V2b` row and `H5b`'s **Deps**; no new rule is needed.
