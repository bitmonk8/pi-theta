## Finding

The *Sequential by default* cross-cutting rule in `conventions.md` forbids `Promise.all` / `Promise.race` / `Promise.allSettled` / `Promise.any` in production code (`src/**`, excluding `**/*.test.ts`) "unless the calling leaf's `Spec.` field cites an obligation whose normative text mandates concurrency at the specific operation the combinator call implements — either (i) a spec REQ-ID, or (ii) a **code-keyed obligation area** … — and the leaf's `Adds.` field names the construct together with that REQ-ID or code-keyed obligation area." The exemption is thus gated on the presence of a `Spec.` field.

Horizontal leaves do not have a `Spec.` field. `plan.md` §Horizontal phases states it directly — "their leaves cite a **Convention.** field instead of a **Spec.** field" — and every horizontal leaf header confirms it (`H1a`, `H2a`, `H3a`, `H4a`, `H4b`, `H4c`, `H5a`–`H5f`, `H6a`, `H7a` all open with `**Convention.**`). Several horizontal leaves contribute `src/**` production code (`H3a` the DI seam skeleton; `H4a` the extension factory shell; `H5a` the closing-gate automation), so the rule's scope reaches them.

The rule therefore has no defined exemption path for the horizontal leaf class. Two readings stand: (A) a horizontal leaf may never use a Promise combinator in `src/**` (the exemption is unconditionally unavailable to it), or (B) a `Convention.`-field citation path exists but is unstated. The sibling *Specific exception types only* rule already faced the analogous "no spec obligation to cite" problem and resolved it with a structural `pi-sdk-boundary` token plus comment-token resolution that is independent of any `Spec.` field — so the asymmetry is conspicuous: only *Sequential by default* couples its exemption to the `Spec.` field specifically. The mechanical closing gate resolves the `// allow:` token against `coverage-matrix.md` regardless of which leaf field cites it, so the `Spec.`-field coupling is an authoring-discipline constraint, not a gate constraint, and it locks horizontal leaves out at the authoring layer.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `fecb504` (2026-05-04, "Split spec.md and plan.md into per-topic / per-phase files" — first authoring of the *Sequential by default* rule as "No `Promise.all` / `Promise.race` outside slices that have a documented spec reason"); hardened to the explicit `Spec.`-field coupling around `c6a664e` (2026-06-10, "build/update plan for spec.md + review") and `75b6a9b` (2026-06-10, "resolve 'Sequential by default carve-out admits only a numbered REQ-ID'", which extended the cited obligation from a REQ-ID to a REQ-ID-or-code-keyed-area but left the `Spec.`-field coupling untouched).
**History:** `git log -S`/`-G` over `docs/plan_topics/conventions.md` shows the rule keyed its exemption to a spec reason from the file's first authoring (`fecb504`), and the `Convention.`-field-for-horizontal-leaves concept coexisted in the same *Leaf format* rule throughout. No commit ever closed the gap between the two: every later refinement (`c6a664e`, `75b6a9b`, `d31eeef`) tightened the obligation/scope of the cited token but never addressed the horizontal-leaf field-coupling. The defect is original to the rule, not introduced by a specific later edit.

## Solution Space

### Recommendation

In `conventions.md` §Sequential by default, add one sentence to the *Sequential by default* bullet stating that the Promise-combinator exemption requires a `Spec.`-field citation and is therefore unavailable to horizontal leaves, which carry a `Convention.` field instead. Production `src/**` code introduced by a horizontal leaf must be sequential; a horizontal leaf that genuinely needs concurrency is spec-drift to resolve first per the *Spec drift* rule. Make no leaf edits, no `H2a` closing-gate edit, and no `coverage-matrix.md` edit — the conservative sequential reading is already what an implementer does and no horizontal leaf currently uses a combinator in `src/**`.

Word the added sentence so it does not contradict the existing `Adds.`-field clause, which already presumes a `Spec.`-field REQ-ID or code-keyed-area citation accompanies any exempted combinator.

## Consequence

**Severity:** advisory

If a horizontal leaf ever needs a Promise combinator in `src/**`, the rule gives no exemption path it can satisfy: one implementer writes the code sequentially (always correct), another invents a `Convention.`-field citation whose `// allow:` token cannot resolve at the closing gate and reddens CI. No current horizontal leaf is blocked — the conservative sequential reading works — so the defect is a latent guidance gap rather than an active blocker.
