## Problem

The `bind_context: session` truncation walk sources its message list from `buildSessionContext(ctx.sessionManager.getEntries(), ctx.sessionManager.getLeafId()).messages` in the *Session-context truncation* paragraph, but the spec never pins whether the user message carrying the triggering slash command is present in `getEntries()` when the slash-command handler enters and runs the walk. The silence is doubly observable: it changes the binder's prompt bytes (the slash text appears once via the *User-arguments line* of `#system-prompt-structure-normative`, or twice — also as the trailing transcript turn) and it changes which turns count toward the 8000-token / 20-turn caps. As a result `#bndr-7`'s reproduce-exactly contract and the adjacent input-reproducibility contract cannot be exercised against the production Pi runtime, and two conformant implementations can diverge. The dependency is host-behavioural — whether Pi commits the inbound slash-trigger message to `sessionManager` before invoking the handler — yet it is not inventoried under Host prerequisites.

## Solution approach

Clarify the *Session-context truncation* paragraph in `binder-model-and-context.md` to pin whether the triggering slash-command user message is present in the walked `getEntries()` snapshot at handler entry, grounded against the fact that the slash text already reaches the binder via the `#system-prompt-structure-normative` User-arguments line. Add a companion host-behaviour presupposition in `host-prerequisites.md`, modelled on the `#driven-turn-commit-ordering-presupposition` paragraph in `conversation-drive.md`, naming the Pi slash-handler-entry commit ordering the rule depends on. Append a checklist item in `version-bump-step2.md` routing detection to editorial review, with the same structure as items (ac) and (ak).

## Solution constraints

- The clarification MUST NOT change `#bndr-7c`'s handling of the historical `/lookup foo` turn — only the currently-triggering slash turn is in scope, not past slash invocations recorded in the session.
