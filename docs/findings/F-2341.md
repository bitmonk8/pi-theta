## Problem

PIC-11 (`spec_topics/pi-integration-contract/host-interfaces-services.md#pic-11`) defines six behavioural MUSTs for a conforming `SchemaValidator`. The `V8c` / `V8c-T` Tests bullet asserts only three: one-pass multi-error reporting, no coercion/defaulting, and the `validator-cache-collision` byte-mismatch path. Three further MUSTs carry no asserting vector — silent acceptance of unknown `format` keywords, in-document-only `$ref` resolution, and determinism for a given `(schema, value)` pair. None emit a diagnostic, so the DIAG-2 reverse-gate cannot cover them either; a validator violating all three passes the leaf's gate.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e ("pi-loom plan: build/update plan for spec.md + review", 2026-06-10)
**History:** The PIC-11 Tests bullet has asserted only the one-pass / no-coercion / cache-collision facets since the plan's initial build commit (c6a664e); `git log -S`/`-G` on the bullet text shows the silent-unknown-`format`, in-doc-`$ref`, and determinism MUSTs were never given a Tests vector. The bullet carried verbatim through the V8a→V8c over-bundling split (cb8737a) and the spec-page-reference fix (abc5d40); neither narrowed the coverage — the gap predates both.

## Solution approach

Add asserting test vectors to `V8c-T`'s Tests for the three currently-unasserted PIC-11 MUSTs: silent acceptance of an unknown / non-loom `format` keyword (validator returns its normal result rather than raising), in-document-only `$ref` resolution (external `$ref`s left unresolved), and determinism across repeated `validate` calls on the same `(schema, value)` pair. Mirror the additions in `V8c`'s Tests and widen `V8c`'s Ships-when to recap them. Retain the existing one-pass / no-coercion / cache-collision assertions.

## Solution constraints

- Confine the change to the `V8c` / `V8c-T` leaf pair; the `PIC-11 → V8c` coverage-matrix row already exists and needs no edit.
