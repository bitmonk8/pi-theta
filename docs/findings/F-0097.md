## Problem

The `loom/runtime/internal-error` Tests bullet in `V14a` and `V14a-T` is led by a single `loom/runtime/internal-error` label and the phrase "route correctly," reading as a claim that all four listed off-surface outcomes route to `internal-error`. Per `tool-calls.md` (*Outcome enumeration (normative)*) and `cancellation.md`, only the pre-evaluation setup throw and the non-conforming return shape route through that runtime-defect surface: a non-settling promise blocks at its `await` until `loomAbort.signal` fires and surfaces via the `cause: "cancelled"` path (no `internal-error`), and a post-cancel late settlement is silently discarded per CNCL-1/CNCL-2/CNCL-3 (promoting it to `internal-error` is forbidden). The bullet also says "three off-surface outcomes" while listing four. A test author reading the bullet literally would assert that a non-settling promise and a post-cancel settlement each fire `loom/runtime/internal-error`, contradicting the `cancelled`-path and silent-discard requirements.

The defect entered with the initial plan-topic build and has not been edited by any later commit (`git log -S "three off-surface outcomes route correctly"` returns only c6a664e).

## Solution approach

Rewrite the `loom/runtime/internal-error` Tests bullet in both `V14a-tool-calls.md` and `V14a-T-tool-calls.md` so each off-surface outcome is attributed to its actual routing channel rather than to a shared `internal-error` label: pre-evaluation setup throw → `{ isError: true }` plus one `internal-error` diagnostic; non-conforming return shape → `internal-error{tool-return-shape}`; non-settling promise → blocks at `await` and surfaces via `cause: "cancelled"` (no `internal-error`); post-cancel late settlement → discarded per CNCL-1/CNCL-2/CNCL-3 (no `internal-error`). Reconcile the count word so the Tests bullet, V14a's Adds field, and both leaves' Ships-when references state a count matching the enumeration. Keep the `V14a` and `V14a-T` bullet byte-aligned.

## Solution constraints
