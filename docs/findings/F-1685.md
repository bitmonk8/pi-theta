## Problem

The all-three-throw corner case (sub-step 1's `LoomRegistry.drain`, sub-step 1's `LoomRegistry.initDrainStateTag`, and the post-sub-step-4 `LoomRegistry.markRuntimeDegraded` all throwing on a single session-only `session_shutdown`, leaving the registry at `(drained: false, tag: undefined)`) has two observable, loom-1.0-accepted outcomes: a subsequent slash invocation routes through arm (a) of `readDrainState` and silently dispatches the loom into the half-torn-down runtime (the no-suppression prohibition PIC-30 pins), and a follow-up session-only `session_shutdown` re-runs the handler and emits an additional `loom/host/session-shutdown-runtime-degraded` row. Neither acceptance set pins these: the `## Acceptance criteria` block (a)–(m) in `session-only-degraded-state.md` and the arm-(b) block in `drain-state-contract.md` cover every other branch but no fixture exercises the all-three-throw tuple. Fixture (b) explicitly defers multi-site failure to "the all-three-throw corner case", naming it as the coverage vehicle that is never instantiated. The case is rare and silent, so the gap is unlikely to be caught incidentally.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 52b9101 ("pi-loom spec: resolve T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state", 2026-05-11)
**History:** Commit 52b9101 introduced both the all-three-throw corner case (the first appearance of the `all-three-throw` token in the corpus) and the per-branch acceptance-criteria fixture list (the "Positive — clean session-only reason" set) in the same edit, inside the then-monolithic `pi-integration-contract.md`. The corner case was documented as a loom-1.0 acceptance with observable consequences but received no fixture at inception, while every sibling branch did. Later edits extended the fixture set without closing the gap: f5e89f4 (2026-06-04, the spec-set size-cap split that created `session-only-degraded-state.md`) added fixture (b)'s "covered only by the all-three-throw corner case" out-of-scope marker — naming the corner case as the coverage vehicle — and 2775382 (2026-06-05) added the read-failure fixtures (l)/(m) citing the corner case as their shared acceptance footing. Across all three commits the named coverage vehicle was never instantiated.

## Solution approach

Add one acceptance fixture for the all-three-throw corner case to `session-only-degraded-state.md`'s `## Acceptance criteria` block, alongside (a)–(m) — the section that already owns the corner case's prose. Drive a session-only `session_shutdown` with `LoomRegistry.drain`, `LoomRegistry.initDrainStateTag`, and the post-sub-step-4 `LoomRegistry.markRuntimeDegraded` all throwing, and assert the two observable outcomes: a slash invocation arriving before the next `session_shutdown` routes through arm (a) of `readDrainState` — dispatching the loom and returning neither the shutting-down nor the degraded system note; and a follow-up session-only `session_shutdown` re-runs the handler and emits an additional `loom/host/session-shutdown-runtime-degraded` row. Reference PIC-30 from the fixture so the prohibition carries a witnessing criterion.
