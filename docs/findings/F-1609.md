## Finding

The always-log set partitions by routing channel: group A emits `details: { event: RuntimeEvent }`, group B (runtime panics — every `loom/runtime/*` row except the five `console.error`-routed teardown codes) emits `details: { diagnostics: Diagnostic[] }`. In `runtime-event-channel.md` the **Deduplication and lifetime rules** give group A an observable, falsifiable handle on its exactly-once guarantee: the dedup tuple `(kind, query_site, message, occurred_at)`, a worked `?`-propagation derivation ("emits at its origin, not at each rethrow"), the cascade-twin re-emission rule, and — in PIC-1 clause (e) — an explicit conformance fixture ("a conformance fixture that drives the same failure under two implementations and compares the emitted `RuntimeEvent` stream").

Group B's counterpart rule — "A panic emits exactly one `loom-system-note` per top-level panic … the cascade-twin dedup key above does not apply to group B" — carries no analogous acceptance fixture, and the page explicitly states the `RuntimeEvent` dedup tuple "applies to group A only and ha[s] no analogue for group B." Three group-B-specific behaviours are normative but unexercised by any pinned fixture:

1. **Direct raise** — a directly-raised panic emits exactly one `loom-system-note` carrying a single-element `details: { diagnostics: [Diagnostic] }`, with zero `details: { event }` emissions.
2. **`?`-propagation non-multi-emit** — a panic surfaced through intermediate frames emits once at its origin, not once per frame (panics do not flow through `?` as values, but the emission-site rule must still be witnessed).
3. **No second note at the cascade boundary** — a panic reaching the top-level slash-dispatch boundary emits no boundary twin, in deliberate contrast to group A's cascade-twin re-emission.

The only emission-count assertion anywhere is the generic V9d/V9d-T plan bullet "Each always-log kind emits exactly once", which neither enumerates the three group-B scenarios nor pins the no-cascade-twin asymmetry that distinguishes group B from group A.

## Solution Space

**Shape:** single

### Recommendation

Add a fixture-obligation block under the panic-emission bullet in `runtime-event-channel.md` (mirroring the clause-(e) falsifiable-fixture style on the same page and the lettered fixture-obligation lists on `session-only-degraded-state.md`), pinning three acceptance fixtures against one representative `loom/runtime/*` panic code:

1. A directly-raised panic emits exactly one `loom-system-note` with `details: { diagnostics: [Diagnostic] }` carrying a single diagnostic and `content` set to the `"loom /<name> aborted: <message>"` framing, and **zero** `details: { event }` emissions.
2. A panic surfaced through ≥2 intermediate frames emits exactly one note at the origin frame — none re-emitted per frame.
3. A panic cascading to the top-level slash-dispatch boundary emits **no** second note, witnessing that the group-A cascade-twin re-emission rule does not apply to group B.

Pin these to one representative code rather than per `loom/runtime/*` code: the emission/dedup machinery is code-independent, and the per-code message templates are already covered by `V4b` (Runtime panics). Then sharpen the V9d/V9d-T "Each always-log kind emits exactly once" bullet to cite the three group-B scenarios explicitly.

Edge cases for the implementer: keep the `loom/runtime/internal-error` content variant (`"aborted with internal error:"`) out of scope — it is a `content`-string concern, not an emission-count one — and ensure a fixture does not mistake a `loom/runtime/system-note-delivery-failed` fallback emission (independent and explicitly outside any dedup key) for a second panic note.

## Consequence

**Severity:** advisory

The group-B panic-emission behaviour is pinned normatively, so a careful implementer can build it correctly, but the conformance suite cannot witness a regression: an implementation that double-emits on `?`-propagation, or re-emits a cascade twin at the slash-dispatch boundary, would still satisfy the generic "each always-log kind emits exactly once" test. The asymmetry with group A (which has a worked derivation plus a falsifiable clause-(e) fixture) leaves the group-B guarantee the weakest-tested rule on the page.
