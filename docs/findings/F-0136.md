## Problem

The `FRNT-1` Tests bullet in `V6e` and the byte-identical bullet in its paired `V6e-T` end with the unqualified clause "exhaustion produces `QueryError{tool_loop_exhausted}`". That outcome holds only on the untyped-query path: on a typed query, ceiling #2's `max_rounds`-final branch (CIO-4) dispatches the forced respond turn as the exempt-routed terminator and never surfaces `tool_loop_exhausted`. An implementer authoring the FRNT-1 test from this bullet can write a typed-query exhaustion assertion that never fires, producing a vacuous or red test that contradicts the runtime contract owned by `V13c`/`V13d`.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e ("pi-loom plan: build/update plan for spec.md + review", 2026-06-10)
**History:** Both `V6e-respond-repair-tool-loop.md` and `V6e-T-respond-repair-tool-loop.md` were created in c6a664e, and the FRNT-1 Tests bullet carrying the unscoped "exhaustion produces `QueryError{tool_loop_exhausted}`" clause is present in that initial authoring (`git log -S 'tool_loop_exhausted'` and `git log -L /FRNT-1/` on both files resolve to c6a664e against `/dev/null`). The only later commit touching the file (4c76a7c) resolved an unrelated diagnostic-code defect and did not alter this clause. The defect has existed since the leaf pair was first authored.

## Solution approach

Narrow the trailing exhaustion clause of the `FRNT-1` Tests bullet in both `V6e` and `V6e-T` so it scopes `tool_loop_exhausted` to the untyped-query path and points at the runtime owners (`V13c`/`V13d`) rather than restating the ceiling rule. Note the typed-query exemption (the forced respond turn is the exempt-routed terminator, so the variant is unreachable). Apply the identical edit to both the implementation leaf and its `-T` twin so they stay aligned.

## Solution constraints

- Out of scope: the runtime exhaustion behaviour and the CIO-4 ceiling rule, owned by `V13c`/`V13d`; cite the owning behaviour rather than re-derive it in `V6e`.
- The spec is read-only for this fix; `frontmatter-fields-b-and-templates.md` FRNT-1 and `ceilings-3-and-4.md` CIO-4 already carry the exemption.
