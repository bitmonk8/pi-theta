## Finding

`runtime-value-model.md` §Wire-name translation declares that translation happens in **exactly two places**: *inbound* (model output → loom value, post-AJV rebuild to loom-side names with enum-tag reattach) and *outbound* (loom value → JSON, walking the loom-side value to produce wire-named JSON before AJV validation, when constructing tool input, query response payloads, or `invoke` arguments). `V2e`'s **Adds** field claims both directions and explicitly names "outbound wire-name JSON".

`V2e`'s sole wire-translation **Tests** bullet, however, asserts only the inbound direction: "inbound translation rebuilds loom-side names so loom code never sees wire names; defaults bypass inbound translation". The paired `V2e-T` tests leaf carries the identical inbound-only bullet. Nothing in either leaf asserts the outbound path — that a `field as "Wire"` rename serialises to the wire name on the way out, or that a non-renamed field is left unchanged.

Because `cka-2` covers `runtime-value-model.md` (RVM) as a code-keyed area assigned to `V2c` and `V2e`, the outbound rule sits inside `V2e`'s coverage remit but has no asserting vector. The outbound half of the boundary `V2e` exists to implement can therefore ship with a broken rename — emitting loom-internal names to tools, query responses, or `invoke` arguments — and the red/green gate stays green.

## Solution Space

**Shape:** single

### Recommendation

Add an outbound assertion to the wire-translation coverage of the paired leaves so the **Tests** vectors mirror the two-direction scope `V2e`'s **Adds** already claims:

- In `docs/plan_topics/V2e-T-wire-name-translation.md`, add a **Tests** bullet under the existing inbound bullet, citing `runtime-value-model.md` §Wire-name translation (RVM code-keyed area, Outbound row), asserting that constructing the outbound JSON from a loom-side value whose schema declares `field as "Wire"` produces the wire name `Wire` in the emitted JSON, and that a field with no rename is emitted unchanged.
- In `docs/plan_topics/V2e-wire-name-translation.md`, add the same outbound bullet so the implementation leaf's green gate mirrors the failing test, and extend the **Ships when** sentence so `npm test` covers the outbound vector alongside the inbound one.

Edge cases the implementer must keep in the outbound vector: a renamed field nested inside an object schema (translation map applies recursively), and confirmation that enum values still serialise to the bare wire string with the interpreter-private tag stripped (per the §value representation row), since the outbound walk is where tag-stripping must hold.

## Consequence

**Severity:** correctness

An implementer satisfying `V2e`'s red gate only has to make the inbound walk pass; the outbound rebuild can be absent or buggy and still ship green, leaking loom-internal field names to tool input, query response payloads, and `invoke` arguments. Two reasonable implementers diverge on whether the outbound direction is even built, and a regression in outbound rename is undetectable from the leaf's own gate.
