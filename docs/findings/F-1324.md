## Problem

`cka-33` is a multi-leaf coverage-matrix row whose five facets — `V17a`, `V14f`, `V13f`, `V15h`, and `V9o` — each carry no `co-witness` annotation, so under the *Multi-leaf-row per-facet citing tests* convention `H5f` requires every facet to have a facet-naming citing test that cites both the row's `cka-33` token and that facet's closing-leaf-ID inline. The `V17a` / `V17a-T` "Swallowing-handler side-channel suppression" Tests bullet describes the suppression assertion and links `cancellation.md` §*Race semantics*, but cites neither the `cka-33` token nor the `V17a` facet leaf-ID inline. An implementer working the leaf under the single-leaf reading model has no instruction to emit either token, so the test ships without them and `H5f`'s per-facet arm reddens at the `H6a` release-gate activation.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `cf1bb3e` (resolve "item 7 never sweeps the code-keyed obligation table") — split `cka-33` into a five-facet multi-leaf row with `V17a` as the Checkpoint-seam-substrate facet; `26a8794` (resolve "split coverage-matrix rows can ship a facet untested") — authored `H5f`, the per-facet citing-test gate that mechanically requires each facet's test to cite both the `cka-<n>` token and the facet's leaf-ID inline.
**History:** The `V17a` "Swallowing-handler side-channel suppression" Tests bullet was authored 2026-06-10 in `52a6819` ("cancellation test bullet conflates four obligations"), before `cka-33` was a multi-leaf row and before any per-facet inline-citation requirement existed — so it was well-formed when written. The defect crystallised when `cf1bb3e` (2026-06-18 10:10) made `V17a` a `cka-33` facet and `26a8794` (2026-06-18 19:38) imposed the `H5f` per-facet inline-citation gate; neither commit retrofitted the pre-existing `V17a` bullet, so the omission became a release-gate-reddening gap.

## Solution approach

Add the `cka-33` code-keyed-area token and the `V17a` facet leaf-ID inline to the "Swallowing-handler side-channel suppression" Tests bullet, in the same inline-citation form `V14b` already uses for `cka-13`. Apply the edit in both `docs/plan_topics/V17a-cancellation-core.md` and `docs/plan_topics/V17a-T-cancellation-core.md`, keeping the `-T` mirror consistent with the implementation leaf.
