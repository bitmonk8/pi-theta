## Finding

Registration step 1 asserts that the `resources_discover` event payload `{ cwd, reason }` "supplies the project root used for the project-local source," and that on every event `event.cwd` is preferred over the seam value "so a future per-session cwd change is honoured on reload." The neighbouring factory-time `FileSystem.cwd()` arm of the same sentence is already covered by the named *Factory-time `cwd == project root` presupposition* under [PIC-13](./host-interfaces-services.md#cwd-project-root-presupposition) (bump-checklist item (w)), which pins the assumption that Pi launches the extension factory with `process.cwd()` set to the project root and routes a future-minor regression through editorial review. The symmetric assumption — that on every `resources_discover` event Pi populates `event.cwd` with the project root rather than some other directory (host process cwd, a per-session sandbox, the previous session's cwd, an empty string, …) — is just consumed in prose and never named as a presupposition.

The two `cwd` sources are load-bearing for the same project-local discovery walk (`.pi/looms/` and project `.pi/settings.json`, plus the same resolution base for the deferred `--loom` relative-path branch tracked in F-1154). Both are unpinned by Pi's typed surface — the `ExtensionContext` declaration types `cwd: string` but pins no semantics — and both require an editorial-review item on each Pi minor bump to remain trustworthy. Treating the seam arm as a named presupposition while the event arm is treated as background fact is inconsistent on its face and leaves the event arm with no audit hook: a Pi minor that scoped `event.cwd` to a per-session working directory (or to the host process cwd while leaving `FileSystem.cwd()` alone) would silently misroute the reload-time project-local walk and the reload-time settings read with no build-time, load-time, or editorial-review signal.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** b987cb4 (2026-05-04), c6e464a (2026-06-04), 4ea0544 (2026-06-05)
**History:** Commit b987cb4 ("resources_discover is a contribution event, not an inbound path feed") rewrote step 1 to consume `event.cwd` as the reload-time project root, with no presupposition framing. Commit c6e464a ("Registration step 1 — factory-time working-directory source unstated") then introduced the `FileSystem.cwd()` seam member and rewrote step 1 to split the two sources, but pinned only the seam-side semantics. Commit 4ea0544 ("Factory-time cwd == project root premise is unpinned") added the named `#cwd-project-root-presupposition` block and bump-checklist item (w) for the factory-time arm only; its resolution scope was confined to the seam, so the symmetric event-time assumption was carried forward unpinned. The defect is the asymmetry the three commits jointly produced — neither b987cb4 (which simply consumed `event.cwd`) nor 4ea0544 (which scoped its pin to the seam) is wrong in isolation; the gap is in their interaction.

## Solution Space

**Shape:** single

### Recommendation

Pin the `event.cwd == project root` assumption on the same footing as the factory-time arm.

1. In `host-prerequisites.md`, add a new presupposition block — adjacent to the existing list (degraded-state branch, binder-model load-time resolution, settings write-back, interactive-session dispatch, `pi.getCommands()` completeness, etc.) — anchored `#event-cwd-project-root-presupposition`. The block names the assumption that on every `resources_discover` event Pi populates `event.cwd` with the project root the loom-extension instance is bound to (the same directory the factory-time `FileSystem.cwd()` arm is pinned to resolve against), states that no SDK surface pins this behaviour, names the failure mode (project-local looms and project `.pi/settings.json` resolve against the wrong directory on every reload), and routes re-confirmation through editorial review under [Pi version bump procedure](./version-bump-intro.md#pi-version-bump-procedure) via a new checklist item. Mirror the *Authoritative source* sub-paragraph convention used by the neighbouring presuppositions: name where the behaviour is observed at the pin (the `resources_discover` event-emission site in `dist/core/extensions/runner.js` / `dist/core/extensions/loader.js`) and label it **known-fragile evidence** routed to the same textual-diffability recovery path.

2. In `registration-steps.md` step 1, repoint the `event.cwd is preferred over that seam value` clause to cite this new anchor, on the same model as the existing inline cross-reference to `#pic-13` for the factory-time arm. No prose rewrite beyond the cross-reference.

3. In `version-bump-step2.md`, add a new lettered checklist item (next free letter in the enumeration) cross-linked to the new anchor, SHOULD-level on the same footing as item (w). The item's text MUST confirm against the candidate Pi minor that every `resources_discover` event still carries `event.cwd` equal to the project root the extension instance is bound to.

4. Update `V18c`'s editorial-review checklist coverage in `docs/plan_topics/V18c-version-bump-checklist.md` to include the new item letter in the `(f)`–`(al)` SHOULD-level range (or whatever range the new letter falls into).

Implementer note: keep the new presupposition block at the same anchor-citation cadence as `#cwd-project-root-presupposition` so a single sentence in step 1 can cite both arms without re-shaping. The new item is independent of F-1154 (the `--loom` CLI relative-path branch) — that finding resolves at the CLI bullet and merely consumes whichever `cwd` source step 1 is currently using — but landing this one first lets F-1154's fix cite a stable two-source pin.

## Consequence

**Severity:** advisory

A reader looking up the project-root assumption on the reload path finds the seam-side pin and infers, wrongly, that the event-side arm carries the same audit guarantee. Two implementations stay observationally identical at the loom 1.0 pin, so the gap does not divide implementers today; it is a guidance-and-audit-completeness defect rather than a divergence-producer. The cost lands on a future Pi minor that re-shapes `event.cwd`: no item on the bump checklist names it, so the regression slips past the editorial-review gate that catches every other behaviourally-unpinned host assumption on this surface.
