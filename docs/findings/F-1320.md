## Finding

`V11d`'s sole `Tests.` bullet ("System-prompt structure …") folds eleven independently-observable obligations into one assertion description: the eight structural system-prompt items (1–8) — with both trigger-present and trigger-absent arms for the conditional items 2, 3, 4, 6 — plus three orthogonal reference-rendering tables (the *Type display* declared-Loom-type→rendered-string table, the *Default-literal rendering* forms, and the four *Parameter-line reference renderings*). Each of these can fail while the others pass: a wrong *Type display* row does not implicate the *Parameter-line* renderings, and a missing conditional-item arm does not implicate the literal-default forms.

The per-phase TDD ritual (`conventions.md` §"Three kinds of phase", tests-task rule) directs the author to "default to one assertion per REQ-ID so each rule reds out in isolation; fold multiple rules into a single assertion only when they describe one indivisible behaviour that cannot fail rule-by-rule." These eleven obligations are not one indivisible behaviour — they are the distinct un-anchored MUSTs that `coverage-matrix.md` row `cka-45` enumerates for `binder-bypass-and-envelope.md` §"System-prompt structure (normative)". Carrying them as one undifferentiated bullet is the plan-level expression of an against-default bundling: a reader cannot read off the plan which sub-obligations get their own red, and the single bullet invites the implementer to write one coarse assertion that greens on partial coverage.

The verbatim bullet also lives in the paired tests leaf `V11d-T`, so any decomposition must be applied to both leaves in lockstep.

## Solution Space

**Shape:** single

### Recommendation

In `V11d-binder-system-prompt.md`, separate the single "System-prompt structure" `Tests.` bullet into distinct bullets so the structural-item obligations and the reference-rendering-table obligations red independently:

- one bullet covering the eight structural items (1–8), retaining the trigger-present and trigger-absent arms for the conditional items 2, 3, 4, 6;
- one bullet covering the three reference-rendering tables — *Type display*, *Default-literal rendering* (`default=Severity.High`, `default="hello"`, `default=[1, 2, 3]`, `default=[]`), and the four *Parameter-line reference renderings* including the description-omitted form `  language (string) required`.

Each resulting bullet keeps the existing `binder-bypass-and-envelope.md#system-prompt-structure-normative` citation; the `cka-45` coverage-matrix mapping is unchanged. Apply the identical decomposition to the verbatim bullet in `V11d-T-binder-system-prompt.md` so the tests leaf and its impl partner stay in lockstep. `V11d`'s `Ships when` already enumerates the same sub-obligations and needs no change.

## Consequence

**Severity:** advisory

If shipped as-is the leaf is still buildable — the bullet enumerates every sub-obligation — but the plan-level test specification under-documents which obligations red in isolation, and the single bullet nudges the tests-task author toward one coarse assertion that can green on partial coverage of `cka-45`, weakening the later impl-side green. The cost is reduced traceability and a thinner failing-test gate, not implementer divergence on behaviour.
