## Problem

The `loom/runtime/watcher-terminated` row's *Trigger* cell and the preamble paragraph above the table in `code-registry-runtime.md` both name the npm package `chokidar` and one of its internal event names (`error`) in normative prose inside the closed diagnostic registry the DIAG-2 gate consumes. Everywhere else in the corpus the watcher abstraction is the `FileWatcher` seam pinned by PIC-14, with `chokidar` deferred to implementation choice; the trigger condition is observable as a post-condition over that seam. Naming the production adapter and its private event conflates the seam contract with one conforming implementation and is inconsistent with sibling rows, which key their trigger on the abstract operation.

## Issue introduction

**Verdict:** present-since-inception

**Introducing commits:** `0b195eb` (2026-06-25) ŌĆö *pi-loom spec-plan: fix F-1299+F-1308 ŌĆö coin PIC-55 watcher post-error posture + loom/runtime/watcher-terminated*

**History:** Commit `0b195eb` introduced both occurrences of the chokidar-naming text in `code-registry-runtime.md` in one diff: it added the new `loom/runtime/watcher-terminated` row (whose *Trigger* cell opens with "The chokidar file watcher over the discovered roots terminated ŌĆö its `error` route fired or ŌĆ”") and the preamble paragraph that introduces the code ("emitted when the chokidar file watcher's `error` route fires or it throws ŌĆ”"). `git log -L` on the relevant lines shows no subsequent edits. The same commit also added the PIC-55 prose in `registration-steps.md` from which both passages were transcribed; the prescription was carried over from the PIC-55 source text rather than introduced in a later editorial pass.

## Solution approach

Rewrite the runtime-registry preamble sentence and the `watcher-terminated` row's *Trigger* cell in `code-registry-runtime.md` to key on the `FileWatcher` seam's (PIC-14) observable post-condition ŌĆö one or more watched roots stop delivering events and the adapter is left torn down rather than re-armed ŌĆö instead of naming `chokidar` and its `error` route. Keep the row's existing back-reference to PIC-55.

## Solution constraints

- Out of scope: the *Message template* column of the `watcher-terminated` row, asserted character-for-character by DIAG-4 tests under V7b.
- Out of scope: PIC-55's prose in `registration-steps.md`, which is owned by a separate finding.
