## Problem

`discovery/package-and-settings.md` DISC-5 places normative weight on the `minimatch` engine — it pins the glob matcher loom uses for `pi.looms` and grounds the dialect by asserting parity with the package Pi applies to its own resource arrays. The owning inventory, the `**Loom-package implementation dependencies (loom 1.0).**` bullet (anchor `loom-package-implementation-dependencies-loom-1-0`) in `implementation-notes.md`, enumerates `semver`, `chokidar`, and `yaml` but omits `minimatch`. Because the inventory is silent, the spec leaves unresolved whether loom reuses Pi's `minimatch` instance or carries its own, so the dialect-parity claim has no inventory entry to attach version-skew discipline to, and an implementer re-deriving the dependency set from the spec alone would miss `minimatch`.

## Issue introduction

**Verdict:** single-commit

**Introducing commits:**

- `1cb54c7` (2026-06-06) — *pi-loom spec: resolve "Glob !/+/- precedence and matcher engine unspecified for pi.looms / loomPaths"*

**History:** Commit `1cb54c7` introduced the normative `minimatch` reliance into DISC-5, pinning both the engine and the dialect-parity claim against Pi's `applyPatterns`. The `Loom-package implementation dependencies (loom 1.0)` bullet on `implementation-notes.md` already existed at that point — added by `1ea0455` (2026-06-04) for `semver` and `chokidar`, later extended by `1064946` (2026-06-10) for `yaml` in response to a parallel "YAML frontmatter parsing mechanism is never declared as a dependency" finding — so the inventory site that this finding asks to be backfilled was a known landing surface; the defect is that `1cb54c7` added the DISC-5 reliance without the symmetric inventory entry the earlier and later sibling additions modelled.

## Solution approach

Add a `minimatch` entry to the `**Loom-package implementation dependencies (loom 1.0).**` bullet (anchor `loom-package-implementation-dependencies-loom-1-0`) in `implementation-notes.md`, on the same footing as the existing `semver` / `chokidar` / `yaml` entries, stating that loom carries its own `minimatch` instance distinct from Pi's and pinning the version-skew posture at that entry. Add a forward-link from DISC-5 (anchor `disc-5`) so its dialect-parity claim anchors to the new inventory entry rather than floating on the bundler-output symbol citation.

## Solution constraints

- Out of scope: hardening DISC-5's `dist/core/package-manager.js` / `applyPatterns` citation (the bundler-output symbol path / known-fragile-evidence hedge) — owned by a sibling finding.
