## Problem

`V9a`'s **Adds** field attaches the clause "the probe iterates" to `FACTORY_PROBABLE_CAPABILITIES`, "the closed five-element list of factory-probable capability identifiers (inventory items 1/2/3/4/6)". This asserts the probe walks five targets, contradicting the spec: `capability-probe.md` step (c) directs the probe to check the nine named function members in listed order (capability 1 → 1, capability 2 → 1, capabilities 3 and 6 → 2 each, capability 4 → 3). The five-element list is the capability-identifier partition `V18a`/`V18c` reconcile against, not the probe's iteration target. An implementer building from this clause produces a five-target loop, silently omitting four function members and forfeiting the spec's normative first-failure short-circuit determinism on which the `details.member` conformance contract depends.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commit:** `b9de3ee` — "pi-loom plan: resolve \"V18a partition assertion imports undeclared V9a probe constant\"" (2026-06-11)
**History:** `git log -S 'the probe iterates'` and `-S 'five-element list'` on `docs/plan_topics/V9a-capability-probe.md` both point to `b9de3ee` as the sole introducing commit; the file's earliest version (`c6a684e`) carried neither the `FACTORY_PROBABLE_CAPABILITIES` export nor the "five-element list" prose. `b9de3ee` added the export and the descriptive clause "the closed five-element list of factory-probable capability identifiers (inventory items 1/2/3/4/6) the probe iterates" while resolving an unrelated finding (V18a's import of the probe constant); the misleading attachment of "the probe iterates" to the five-element list was introduced there. A later edit (`2f4ecea`) trimmed an adjacent clause but left the defect intact. The defect is not present-since-inception; it entered with the export's introduction.

## Solution approach

Rewrite the **Adds** clause in `docs/plan_topics/V9a-capability-probe.md` so "the probe iterates" no longer attaches to the five-element identifier list. Name `FACTORY_PROBABLE_CAPABILITIES` (inventory items 1/2/3/4/6) as the capability-identifier partition that `V18a`/`V18c` reconcile against, and separately state that the step-(c) loop iterates the nine function members those five capabilities contribute, in the spec's fixed order.
