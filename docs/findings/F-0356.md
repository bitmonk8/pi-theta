## Problem

`V16a` defines the cross-ceiling arbitration seam as a stateless function `arbitrate(candidate) → { surfaced, masked? }`. Its **output** is fully pinned (`surfaced` as a single `"ceiling#1"`…`"ceiling#4"` string, `masked` as a closed-set enumeration omitted when empty, spec-anchored to `ceilings-3-and-4.md#masked-field`), but its **input** ceiling-candidate is described only in prose with no field names or struct shape. Five producer leaves (`V5e`, `V4e`, `V11f`, `V13c`, `V15b`) each construct a candidate and consult this seam via `Deps: V16a`, so each must guess the argument struct it builds. The producers and the seam author can settle on divergent shapes that each satisfy an isolated harness yet fail to compose at the `H7a` co-occurring-breach integration run.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** e2b7e81 — "pi-loom plan: resolve 'isolated cross-ceiling unit interface/authority undefined'" (2026-06-10)
**History:** At c6a664e (initial plan build) `V16a` described only an abstract "unit that computes the cross-ceiling order," with no function signature and no producer consult contract. Commit e2b7e81 resolved the undefined-interface review item by introducing the `arbitrate(candidate) → { surfaced, masked? }` signature and the consult-at-first-enforcement model, and in the same commit added the "consult … binds the seam via Deps" wording to `V5e`, `V4e`, `V11f`, `V13c`, and `V15b`. That commit pinned the output (`surfaced`/`masked`) but left the `Input` ceiling-candidate prose-only. The output/input asymmetry — and the cross-shard shape assumption it forces on the five producers — dates to e2b7e81; the contract has not been touched on the input axis in any later commit.

## Solution approach

Pin the ceiling-candidate input contract in `V16a-ceiling-order-masked.md`'s **Adds** `Input —` bullet so it is as authoritative as the already-pinned output — either by naming the candidate's fields (the check-site / ceiling-class tag each producer sets, per the `invoke`-entry → #1 / round-boundary → #2 / slash-load → #3 / AJV-boundary → #4 mapping the bullet already lists) or by declaring `V16a`'s exported TypeScript candidate type the binding contract every `Deps: V16a` consult resolves against.

## Solution constraints

- Out of scope: the five producer leaves `V4e`/`V5e`/`V11f`/`V13c`/`V15b` — their existing `Deps: V16a` binding already resolves against the seam owner; touch a producer's Adds only to strike a candidate-field description that contradicts the pinned `V16a` contract.
- Out of scope: the existing `Output —` bullet's `surfaced`/`masked` wording, which is correct and spec-anchored to `ceilings-3-and-4.md#masked-field`.
