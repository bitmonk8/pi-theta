## Finding

GOV-22 (`docs/spec_topics/governance/req-id-prefix-table-active-b.md`) defines the per-commit coinage trigger over the set of *defining obligation sites*. The trigger's exclusion list mixes two kinds of predicate:

> excluding tokens inside fenced code blocks, inline code spans, and HTML comments (the GOV-3 exclusion list) and **excluding back-references to obligations another page owns**.

The first three exclusions (fenced code blocks, inline code spans, HTML comments) are structural — a line-level scanner can decide them from the post-diff text alone. The fourth — "back-references to obligations another page owns" — is semantic: it asks the committer/reviewer to recognise that a `MUST`/`MUST NOT` token, in normative prose, on a non-narrative page, is *re-stating* an obligation whose owner lives elsewhere. There is no surface signal on the line that partitions a new obligation from a back-reference: a back-reference rendered as a link with anchor `#prefix-n` looks the same as a co-located restatement; a co-located restatement is in fact what GOV-9 explicitly permits ("MUST either state the rule locally or reference it by a markdown link…"). Two diligent committers reading the same edit will disagree about whether a given MUST is "owned here" or "owned elsewhere and re-stated for local readability", and the disagreement lands on opposite sides of the coining MUST.

The undecidability matters even though GOV-22 commits to reviewer-and-committer enforcement ("no CI gate is reserved"): the exclusion is the *boundary* of the obligation, not a CI-only concern. A reviewer asked to enforce GOV-22 on a PR has no rubric for the borderline case — restating a rule locally per GOV-9 is by construction a `MUST` on a non-narrative page that another page owns. Under a strict reading of the exclusion this is exempt; under a loose reading every local restatement requires its own coined REQ-ID, generating spurious anchors and noise. The spec leaves both readings open.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `4f588e0` (Wed 3 Jun 2026, "pi-loom spec: resolve \"Non-narrative pages with zero coined REQ-IDs non-referenceable\"")
**History:** GOV-22 was authored in `4f588e0` with the disputed clause already in place — the *defining obligation site* definition's closing `…and excluding back-references to obligations another page owns` appeared in the original add to `docs/spec_topics/governance.md`. The file split commit `f5e89f4` (Thu 4 Jun 2026) relocated the paragraph verbatim into `docs/spec_topics/governance/req-id-prefix-table-active-b.md` without touching the exclusion text. No commit between authorship and the current `main` has edited the predicate. The defect is therefore an authoring oversight at GOV-22's inception rather than a regression introduced by a later edit.

## Solution Space

**Shape:** single
**State:** shaped

### Recommendation

Keep the semantic predicate and pin a reviewer rubric plus worked examples. GOV-22's enforcement model is already reviewer-and-committer ("no CI gate is reserved"), and the rule sits in a cluster of governance MUSTs (GOV-8 *Pure rewording*, GOV-30 lock-step, GOV-31) that the spec elsewhere acknowledges are reviewer-judgement; a structural predicate would single GOV-22 out for a stricter regime than its siblings and conflict with GOV-9's express permission to restate rules locally.

Spec edits: append a parenthetical "(reviewer judgement: a MUST is a back-reference iff every normative claim it makes is sourced from a `PREFIX-N`-anchored paragraph on another page cited in the surrounding paragraph)" after the disputed clause, and add a short *Worked examples* block at the GOV-22 paragraph's tail covering three cases: one in-scope new obligation, one out-of-scope cross-link restatement, and one borderline restatement resolved by adding a `per PREFIX-N` citation. Reaffirm that the predicate is reviewer-judgement on the same footing as GOV-8's *Pure rewording* boundary.

Edge cases the implementer must watch: the worked examples must show the *borderline-restatement-resolved-by-adding-`per PREFIX-N`* case explicitly (this is the recurring pattern that motivates the predicate); the rubric must not imply that the presence of any `#prefix-n` link on the page is sufficient to discharge a MUST as a back-reference — the link must sit in the surrounding paragraph and the cited prefix must be owned by a non-current page.

## Consequence

**Severity:** advisory

The rule is reviewer-enforced (no CI gate), so the spec does not break in the mechanical sense, but the boundary of the obligation is ambiguous: a reviewer cannot tell from the GOV-22 text whether a co-located restatement permitted under GOV-9 also triggers coining under GOV-22. The likely failure mode is silent inconsistency across PRs — some restatements acquire fresh REQ-IDs, others do not — eroding the prefix table's signal value over time and leaving GOV-22's residue-draining intent half-met.
