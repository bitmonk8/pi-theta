## Finding

The no-rollback leaf pair states its architectural by-construction rationale once where it belongs — in `V4f` **Adds** ("The no-rollback guarantee is architectural: the runtime contains no compensating path, so a completed callee's side effect survives a downstream terminal event by construction…") — and then restates the same rationale a second and third time inside fields that are supposed to carry only an observable criterion or a test assertion.

`V4f` **Ships when** opens with the observable criterion ("`npm test` proves the enumerated `ERR-13` cases hold — a completed callee stays final after a downstream terminal event across the sampled authoring sites, with no side effect unwound and no compensating turn injected") and then appends a second sentence repeating the architectural claim plus the "rather than exhaustively" scope-limitation. Per the leaf template, **Ships when** is the externally observable change; the appended rationale is not observable and merely echoes Adds.

Both the `V4f` **Tests** ERR-13 bullet and the `V4f-T` **Tests** ERR-13 bullet close with the same methodological sentence ("The guarantee rests on the runtime having no compensating/rollback path, so the tests witness it on these enumerated authoring sites rather than proving the absolute absence of a compensating path exhaustively"). That sentence is rationale for the test design, not part of the per-vector assertion, and it duplicates the Adds claim a further time. (The source finding flagged only the `V4f-T` bullet; the identical clause is present in the `V4f` Tests bullet as well.)

## Issue introduction

**Verdict:** introduced by a single commit (the V4c → V4f/V4f-T split)
**Introducing commits:** `098ec80` (2026-06-17) — "pi-loom plan: resolve \"V4c bundles the ERR-8...ERR-12 trichotomy and the ERR-13 no-rollback proof\""
**History:** The `V4f` / `V4f-T` leaf pair was carved out of `V4c` at `098ec80`. `git log -S` over the Ships-when scope-limitation clause ("tests witness it on the enumerated cases rather than exhaustively") and over the Tests-bullet rationale sentence ("witness it on these enumerated authoring sites rather than proving the absolute absence") both return `098ec80` as the sole introducing commit. The duplicated architectural rationale entered at carve-out time, copied into Ships-when and the Tests bullets alongside the same prose placed in `V4f` Adds. Git `--follow` traces the file back to `V4c-T-terminal-outcomes.md` (created at `c6a664e`, 2026-06-10), but the no-rollback-specific content is wholly a product of the `098ec80` split.

## Solution Space

**Shape:** single

### Recommendation

Strip the duplicated rationale so each field carries only what it is for; the architectural by-construction rationale and the enumerated-sites scope-limitation are retained in `V4f` **Adds**, which already states both.

- In `docs/plan_topics/V4f-no-rollback.md` **Ships when**, strike the trailing sentence "The no-rollback guarantee is architectural: it rests on the runtime containing no compensating/rollback path, and the tests witness it on the enumerated cases rather than exhaustively." Keep the first sentence (the observable criterion) unchanged.
- In `docs/plan_topics/V4f-no-rollback.md` **Tests** ERR-13 bullet, strike the trailing sentence "The guarantee rests on the runtime having no compensating/rollback path, so the tests witness it on these enumerated authoring sites rather than proving the absolute absence of a compensating path exhaustively."
- In `docs/plan_topics/V4f-T-no-rollback.md` **Tests** ERR-13 bullet, strike the same trailing sentence.

No relocation is required: `V4f` Adds already carries the architectural rationale and the "enumerated authoring sites rather than … exhaustively" scope-limitation, so striking the duplicates loses no information.

## Consequence

**Severity:** cosmetic

The observable criterion and the per-vector assertions are both already present and complete; the repeated rationale neither obscures them nor invites divergent implementation. The defect is pure field hygiene — rationale that belongs to (and already lives in) Adds is echoed into the criterion and assertion fields.
