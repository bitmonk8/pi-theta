## Finding

The **Delivery surface** note near the top of `runtime-event-channel.md` carries two forward references of the form "under **Runtime event channel** below" with no specific rule anchor:

- *Subagent-mode `display: false` cascades* — "…noted under **Runtime event channel** below — use the same `pi.sendMessage` call…".
- The directly-dispatched subagent path — "…per the `display` rule under **Runtime event channel** below and [Slash-Command Invocation — SLSH-3](../slash-invocation.md#slsh-3)".

The page is titled `# Runtime event channel`, and there is *also* a bold inline subsection `**Runtime event channel.**` further down (the always-log-set definition). A reader following "under Runtime event channel below" is therefore pointed at a target whose name collides with the page they are already on, with no anchor and no rule identifier — forcing a linear scan of a long section to locate the always-log group-A cascade rule and the `display: false` visibility rule respectively. The two intended targets exist (the always-log set definition and the `display: false` gates-only-renderer-visibility paragraph), so the references resolve in principle; the defect is purely that they are unanchored self-references rather than precise cross-links.

**History:** Commit 78a2f94 relocated the Delivery-surface block onto `runtime-event-channel.md`, introducing the first self-reference ("the subagent top-level `Err` case noted under **Runtime event channel** below") — a pointer that now names the page it sits on. Commit 0033898 later expanded the cascade prose for the directly-dispatched subagent path and added the second instance of the same shape ("per the `display` rule under **Runtime event channel** below"). The defect is the cumulative result: the relocation created the same-page-name collision, and the later edit added a second unanchored pointer into the same section.

## Solution Space

**Shape:** single

### Recommendation

Replace both unanchored "under **Runtime event channel** below" references with anchored cross-references to their specific target rules:

- The subagent-mode `display: false` cascade reference should point to the always-log group-A cascade rule (the `**Runtime event channel.**` paragraph / its group-A enumeration).
- The "`display` rule" reference should point to the `display: false` gates-only-renderer-visibility paragraph.

Neither target currently has an `<a id="…">` anchor, so the edit must also add an anchor immediately before each target paragraph (e.g. `runtime-event-channel-emission` on the `**Runtime event channel.**` paragraph and `display-false-visibility` on the `display: false` paragraph) and link to them as `#…`. Use a label distinct from the bare section name so the link text does not re-collide with the page title (e.g. "the always-log emission rule" / "the `display: false` visibility rule"). Keep the existing `slsh-3` link alongside the second reference.

## Consequence

**Severity:** cosmetic

A reader resolving either reference must scan the long Runtime event channel section to find the always-log cascade rule and the `display: false` rule. No behaviour is at stake; the cost is reader time and the ambiguity between the page title and the same-named inline subsection.
