## Finding

- **Description:** A clone that has not fetched tags reads a satisfied Deps entry as unsatisfied (or an unpushed local tag as satisfied), so a contributor either stalls or starts a leaf whose deps are not complete. Acknowledged but unmitigated.
- **Suggested fix:** Add a How-to-use instruction: evaluate pickability only against a clone that has run `git fetch --tags`, never a shallow/no-tags clone.
