## Finding

`V17c-checkpoint-granularity.md`'s **Adds** field describes the checkpoint set as firing immediately before each of the five sites "and at no other node kinds." That trailing clause is a universal no-false-negative assertion: it claims the implementation emits a checkpoint at *no* node kind outside the fixed five-site set.

The verification backing it cannot prove that universal. The same leaf's **Tests** bullet and **Ships when** condition describe only a *best-effort negative arm* that asserts the seam witnesses no checkpoint at the two non-checkpoint categories the test actually drives (inside a primitive operation, and at a straight-line statement boundary), and explicitly states "a checkpoint emitted at a node kind outside the test corpus is not caught by this assertion." A spurious checkpoint at any unsampled node kind ships green.

The Adds prose therefore overclaims relative to its own acceptance criteria. The positive arm (a checkpoint fires before each of the five sites) is genuinely exhaustive and checked; only the negative direction is best-effort. The fix is to align the Adds wording with the qualifier the Tests bullet already carries.

## Solution Space

**Shape:** single

### Recommendation

In the **Adds** field of `docs/plan_topics/V17c-checkpoint-granularity.md`, soften the universal clause so it stops asserting no-false-negative coverage the best-effort negative arm cannot provide. Strike "and at no other node kinds" and restate the property to mark the positive arm as exhaustive and the negative arm as best-effort over the driven categories, matching the Tests bullet already in the same leaf.

Suggested replacement text for the clause: state that the interpreter checks the cancellation signal immediately before each of the five sites (positive arm, exhaustive) and emits no checkpoint at the non-checkpoint node kinds the negative arm drives (best-effort over the driven categories, per the Tests bullet). The Tests bullet and Ships-when condition already carry the correct qualifier and need no change.

## Consequence

**Severity:** advisory

An implementer reading the Adds clause as a hard guarantee may believe the leaf must structurally prove no checkpoint fires at any node kind, while the actual acceptance gate only samples two driven categories. The mismatch between the asserted property and the verified property is a guidance defect; the leaf still ships a working checkpoint surface, so no gate is blocked.
