## Finding

`cancellation.md` *Race semantics — swallowing-handler attachment on every abandonable Promise* imposes two separable obligations. First, an **attachment-timing** MUST: "The handler MUST be attached at the same site that constructs the Promise, before the first microtask boundary; a `.catch(() => {})` registered lazily after cancellation fires will not catch a rejection that has already been queued for `unhandledRejection`." Second, a **late-settlement suppression** obligation: a rejection arriving after the checkpoint has surfaced `cause: "cancelled"` is absorbed along all three side channels.

The four per-site routing leaves — `V14f` (code-side `execute()`), `V13f` (`@`-query provider), `V15h` (`invoke` child top-level), `V9o` (subagent `AgentSession.abort()`) — each carry a Tests bullet that claims to assert *both*: "assert the … Promise attaches its swallowing handler at the Promise-construction site (before the first microtask boundary), and that a late settlement landed via the `Checkpoint` seam (`V8a`) after the checkpoint has surfaced `cause: "cancelled"` is suppressed along all three side channels." But the only mechanism the leaf describes — and the only one its Ships-when reproduces — is the *late* settlement. A late settlement is caught by a swallowing handler attached at any point before it lands, including one attached lazily after one or more awaits. The early/synchronous-rejection window the timing wording exists to guard (a rejection already queued for `unhandledRejection` before a lazily-attached handler runs) is never exercised.

The result is an overclaim: the Tests assert a property the mechanism cannot distinguish. Worse, the attachment-timing MUST is then verified by no leaf at all — `V17a`'s substrate-level suppression test is likewise a late-settlement test and explicitly defers per-site routing to these four leaves.

## Solution Space

**Shape:** multiple

### Option A — Narrow the claim to match the mechanism

- **Approach:** Drop the construction-site/first-microtask attachment clause from the Tests bullet (and the parallel Ships-when phrasing) in all four leaves, leaving the assertion to state only that a post-cancellation late settlement is suppressed along the three side channels.
- **Plan edits:** In `V9o`, `V13f`, `V14f`, `V15h`, strike "attaches its swallowing handler at the Promise-construction site (before the first microtask boundary), and that" from the Tests bullet, and reword Ships-when symmetrically so it claims only late-settlement three-channel suppression.
- **Spec edits:** None.
- **Pros:** Smallest diff; removes the overclaim so the assertion matches what the `Checkpoint`-seam late-settlement mechanism actually exercises.
- **Cons:** The spec's construction-site/first-microtask attachment MUST is then verified by no leaf (`V17a`'s substrate test is also late-settlement-only), leaving a coverage gap for that obligation.
- **Risks:** A future lazily-attached-handler regression in the early-rejection window ships green; if `cka-33` treats a per-site leaf as the closer for the attachment-timing facet, that facet becomes vacuously green.

### Option B — Strengthen the mechanism to exercise the early-rejection window

- **Approach:** Keep the existing late-settlement arm and add an early/synchronous-rejection arm: construct an abandonable Promise that rejects synchronously / at the first microtask boundary before any handler-attachment await, and assert the rejection is still swallowed (no Node `unhandledRejection`), so a lazily-attached `.catch` reddens the test. Apply to all four leaves.
- **Plan edits:** In `V9o`, `V13f`, `V14f`, `V15h`, add to the Tests bullet and Ships-when an assertion that a rejection landing before the handler-attachment point is still absorbed with no `unhandledRejection`, distinguishing construction-site attachment from lazy attachment.
- **Spec edits:** None — `cancellation.md` *Race semantics* already states the attachment-timing MUST.
- **Pros:** Actually verifies the spec's construction-site/first-microtask MUST; closes the coverage gap; consistent with the project's *No silent test skipping* rule.
- **Cons:** More test machinery; the `Checkpoint` seam is designed to land *late* settlements, so the early-rejection arm needs a separate fixture (a Promise that rejects before the handler-attach point).
- **Risks:** Constructing a deterministic early-rejection window without depending on JS microtask scheduling is fiddly; the negative `unhandledRejection` assertion needs a defined quiescence boundary (see the related `V15h`/`V17a` observation-window finding).

### Recommendation

Take **Option B** for all four leaves (`V9o`, `V13f`, `V14f`, `V15h`), keeping the existing late-settlement arm alongside the new early-rejection arm. The spec mandates the attachment timing as a MUST; Option A leaves that MUST untested everywhere, which is the deeper defect the overclaim masks. Resolve the four leaves uniformly so the per-site routing set stays symmetric. Edge cases the implementer must watch: the early-rejection fixture must reject before the handler-attachment await and the negative `unhandledRejection` assertion must drain to a defined quiescence boundary (coordinate with the `V15h`/`V17a` observation-window finding); leave `V17a`'s substrate-level late-settlement test as-is, since it legitimately covers suppression at the substrate.

## Consequence

**Severity:** correctness

Two reasonable implementers diverge while both pass: one attaches the swallowing handler at the construction site (spec-compliant), the other registers a lazy `.catch` after an await (spec-violating but green against a late settlement). The construction-site/first-microtask MUST is therefore verified nowhere, so a regression that drops handler attachment into the early-rejection window ships green and can leak a Node `unhandledRejection` — the exact failure the timing rule forbids. The finding names `V13f`/`V14f`; the identical wording in `V15h` and `V9o` carries the same defect, so the fix must sweep all four.
