## Finding

`H6a`'s **Adds.** paragraph carries a **Rollback.** claim: that recovery from a bad live-corpus flip is "a single revert" — reverting the activation commit returns `H5a`'s closing gate to its seeded-fixture footing, with "no data, schema, or registry migration to unwind, so the revert is the complete recovery step." This recovery path is the leaf's safety valve: it is what makes activating the hard-fail gate at the terminal step low-risk.

The leaf's three inline `Convention:` tests all exercise the **forward** flip only — that the gate runs green on the live corpus, reddens on an unmapped executable REQ-ID, and reddens on a coverage-matrix-mapped REQ-ID with no citing test. None of them, and nothing in **Ships when**, verifies the reverse direction: that after the activation commit is reverted the gate is actually back on its seeded-fixture footing (runs green against the seeded fixtures and does **not** reconcile the live corpus). The safety valve is therefore documented but unverified — a future refactor of the flip mechanism could leave the reverted state silently reconciling the live corpus (or failing the seeded footing) with nothing to catch it.

## Solution Space

**Shape:** single

### Recommendation

Add to `H6a` a reproducible verification of the reverse (revert) direction, alongside the existing forward-flip checks. `H6a` is a horizontal, non-test-paired leaf that carries inline tests, so the natural home is a fourth inline `Convention:` test bullet under **Tests.** (mirroring the existing live-corpus-footing bullets), with the corresponding acceptance condition reflected in **Ships when.**

The check must assert that, with the activation toggle reverted (the activation commit's change undone), the closing gate runs the **seeded-fixture footing green** and does **not** reconcile the live corpus — i.e. the gate behaves exactly as it does at `H5a` before activation. Phrase it to exercise the same toggle the forward tests flip (assert-then-restore, as the existing reddening bullets do), so the seeded↔live footing is shown to round-trip rather than only flip one way.

Keep the existing **Rollback.** prose in **Adds.**; this fix supplies the reproducible evidence the prose currently only asserts.

## Consequence

**Severity:** advisory

The asserted single-revert recovery path is the documented safety valve for activating the terminal hard-fail gate, but nothing reproducibly confirms a revert restores the seeded-fixture footing. A regression in the flip toggle could make a reverted activation leave `main` still reconciling the live corpus (or failing the seeded footing), defeating the recovery path exactly when it is needed. Implementers can still ship a working `H6a`; the gap is a missing verification of the recovery direction.
