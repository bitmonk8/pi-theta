## Finding

`V4b`'s `Adds.` field catalogues the runtime-defect surface as "the closed six-source runtime-panic set with its normative message templates, the `internal-error` surface for unexpected throws, and the rule that panics bypass `?`/`match`." It describes the *diagnostic-emitting* half of the NOCEIL-3 surface (catchable host allocation failures routing through `loom/runtime/internal-error`) but not the inverse half: that a host-fatal *uncatchable* condition emits no diagnostic at all.

That inverse behavioural rule appears only in a Tests bullet ("A host-fatal uncatchable condition (NOCEIL-3 carve-out) emits no diagnostic at all — in particular, no `loom/runtime/internal-error` is delivered."). The leaf's prose surface inventory and its binding test bullet are therefore asymmetric — the no-diagnostic carve-out is a distinct observable of the same NOCEIL-3 (`ceiling-invariants-and-audit.md#noceil-3`) axis that the `Adds.` catalogue silently drops.

Per `conventions.md` §Leaf format, `Adds.` is descriptive by default and the binding obligation is the Tests REQ-ID bullet, so this is an organisational gap in the leaf's surface catalogue rather than a missing obligation — the carve-out is still asserted where it binds.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V4b-runtime-panics.md`, extend the `Adds.` sentence so its surface catalogue also names the NOCEIL-3 host-fatal carve-out. Append a clause stating that host-fatal uncatchable conditions (the NOCEIL-3 carve-out) emit no diagnostic — for example, ending the existing sentence with "…, the rule that panics bypass `?`/`match`, and the NOCEIL-3 carve-out that a host-fatal uncatchable condition emits no diagnostic." The existing Tests bullet then asserts a rule the `Adds.` catalogue states.

## Consequence

**Severity:** cosmetic

A reader scanning `V4b`'s `Adds.` catalogue for the leaf's runtime-defect surfaces sees the `internal-error` emission path but not the inverse no-diagnostic carve-out. Because `Adds.` is descriptive-by-default and the carve-out is already carried by the binding Tests bullet, no implementer is led to diverge; the defect is a surface-catalogue asymmetry.
