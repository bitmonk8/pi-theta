## Problem

In `docs/plan_topics/real-host-smoke-gate.md`, the manual real-host smoke run is scored against a five-part pass criterion (a)–(e). Sibling criteria (a), (b), (c), and (e) each state an observable pass condition, but criterion (d) lists the four session-double fidelity-contract axes (streamed-token order relative to `ctx.waitForIdle()` resolution, single-turn prompt-mode append, the `pi.on` cancel-forward subscription, and cancellation propagation) and requires only that they "hold". A contributor running the smoke, and the `H6a` auditor reading the recorded run, have no shared per-axis test for whether (d) passed, so the same run can be scored pass by one reader and fail by another. Observable phrasing for the same four axes already exists in `H4a`'s session-double fidelity-contract self-check (`docs/plan_topics/H4a-factory-shell-and-harness.md` line 12).

## Issue introduction

**Verdict:** present-since-inception (of the criterion block)
**Introducing commit:** `3911733` (2026-06-11) "pi-loom plan: resolve 'Live-host smoke pass criterion assumes a non-deterministic LLM reproduces a transcript'"
**History:** The plan corpus is git-tracked. The (a)–(e) pass criterion, including criterion (d)'s "the four session-double fidelity-contract axes — … — hold" formulation, entered the corpus in `3911733` inside `H4a-factory-shell-and-harness.md`. The bare verb "hold" was present from that first authoring and was carried verbatim through every later edit, including the move of the criterion block into `real-host-smoke-gate.md` (`fd22a3d`, 2026-06-17, the file's only commit). Sibling criteria were progressively sharpened over the same window (e.g. `78bbee7` sharpened (b); `dd875e1` sharpened axis (ii)'s wording) while criterion (d)'s scoring verb was never given an observable.

## Solution approach

In `docs/plan_topics/real-host-smoke-gate.md`, rewrite criterion (d) so its trailing verb "hold" is replaced by a per-axis observable pass condition for each of the four axes, drawing the observable phrasing from `H4a`'s session-double fidelity-contract self-check (line 12) so the smoke criterion and the harness self-check state the same four axes the same way. De-duplicate the cancellation-propagation axis against criterion (c) so the criterion does not require the same observable twice.

## Solution constraints

- Confine the edit to `docs/plan_topics/real-host-smoke-gate.md`; `H4a` is the read-only source of the observable wording and its self-check is owned by F-0334.
