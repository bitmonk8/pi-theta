## Problem

In `audit-target-categories.md`'s *Target surface categories* bullet (`#audit-target-surface-categories`), the category-(3) paragraph already pins the SDK `.d.ts` declarations as the single source of truth for category (3)'s closed in-scope set and mandates that `SDK_SURFACE_INVENTORY` category-(3) entries be derived from those declarations rather than from any prose enumeration on the page. It then immediately enumerates a point-in-time list of 13 `ExtensionContext` / `ExtensionCommandContext` members and disclaims that the listing is illustrative only and MUST NOT seed the closed set. The sentence is structurally self-defeating: it forbids the only use a prose enumeration could have, yet ships an enumeration an implementer scanning the dense bullet will naturally anchor on. The list also drifts silently against the authoritative `.d.ts` on every Pi SDK minor that renames, adds, or removes a member, with no gate catching the drift.

**Verdict:** present-since-inception (in this file)
**Introducing commits:** `e5cd514` (2026-05-09, "pi-loom spec: resolve \"T35 SDK capability inventory closed-set negative-direction audit\"")
**History:** The category-(3) paragraph — including both the `.d.ts`-as-source-of-truth pin and the illustrative-members sentence that disclaims itself against that pin — was added in a single commit (`e5cd514`) to the predecessor monolithic file `docs/spec_topics/pi-integration-contract.md`. The contradiction shipped in the same edit that introduced the authoritative pin: there was no prior state in which the prose enumeration was the canonical source. Later commits on this surface (`23d28a1` shard split into `audit-target-categories.md`; `d5a714a` carrier-annotation literal-token tightening; `bf92fe6` literal-read absence-arm relocation) carried the sentence forward verbatim without revisiting it. `git log -S "Illustrative members" --follow` against the file and its predecessor confirms no intermediate state.

## Solution approach

Delete the sentence beginning "Illustrative members at the loom 1.0 pin include …" through "… MUST NOT be seeded from it." within the category-(3) paragraph of the *Target surface categories* bullet (`#audit-target-surface-categories`). The preceding `.d.ts`-as-source-of-truth sentence already pins derivation from the declarations and carries the disclaimer's substance; no replacement prose is needed.
