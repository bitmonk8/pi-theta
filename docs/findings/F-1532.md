## Finding

In `audit-target-categories.md`'s *Target surface categories* bullet, the category-(3) paragraph already pins the SDK `.d.ts` declarations as the single source of truth for the closed in-scope set ("the SDK type declarations are the single source of truth for category (3)'s closed in-scope set, so a Pi minor that adds, removes, or renames a member of either type extends or contracts category (3) automatically and the implementation's `SDK_SURFACE_INVENTORY` category-(3) entries â€¦ MUST be derived from those declarations rather than from any prose enumeration on this page"). It then immediately enumerates a point-in-time list of 13 members (`waitForIdle`, `cwd`, `ui`, `model`, `modelRegistry`, `isIdle`, `getContextUsage`, `hasPendingMessages`, `compact`, `shutdown`, `getSystemPrompt`, `sessionManager`, `signal`) and tags it with the disclaimer "this listing is illustrative only and the audit's category-(3) closed set MUST NOT be seeded from it."

The sentence is structurally self-defeating. It pre-emptively forbids the only use a prose enumeration could have (seeding `SDK_SURFACE_INVENTORY`), yet ships an enumeration that an implementer scanning a dense single-paragraph bullet will naturally anchor on â€” exactly the failure mode the previous sentence already pins authoritatively against. Beyond that, the list is high-maintenance: any Pi SDK minor that renames, adds, or removes a member of `ExtensionContext` / `ExtensionCommandContext` silently drifts this prose against the `.d.ts` that the same paragraph pins as source of truth, with no gate catching the drift (the version-bump step-5 list-of-pinned-constants does not include this prose enumeration). Because it constrains nothing and risks contradicting the authoritative source on every Pi bump, it is pure self-disclaimed cruft inside a normative paragraph.

## Solution Space

**Shape:** single

### Recommendation

Delete the sentence beginning "Illustrative members at the loom 1.0 pin include `waitForIdle`, `cwd`, `ui`, â€¦" through "â€¦ MUST NOT be seeded from it." in its entirety. The immediately preceding sentence already pins the `.d.ts` declarations as the single source of truth and mandates derivation from them; no replacement prose is needed.

Edge cases for the implementer to watch:

- The deletion is purely within the category-(3) paragraph of the *Target surface categories* bullet (anchored at `#audit-target-surface-categories`); the surrounding sentences (the `.d.ts`-as-source-of-truth pin before, and the `ExtensionContext` / `ExtensionCommandContext` export-site / declaration-site sentence after) stay verbatim.
- No cross-reference target is removed: nothing else in the corpus links to the illustrative list. `grep` for any of the 13 member names across `docs/spec_topics/` and `docs/plan_topics/` to confirm before committing.
- The disclaimer's substance (that prose on this page MUST NOT seed the closed set) is already carried by the preceding `.d.ts`-derivation MUST and does not need a separate restatement.

## Consequence

**Severity:** advisory

An implementer who anchors on the illustrative list (whether for an inventory seed, a code-comment enumeration, or a test fixture) ships a list that drifts on every Pi minor that touches `ExtensionContext` / `ExtensionCommandContext`, and the same paragraph's authoritative `.d.ts`-derived rule silently disagrees. The self-disclaimer reduces but does not eliminate the risk; the sentence costs the spec more than it pays.
