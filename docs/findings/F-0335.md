## Finding

The second paragraph of `H4a`'s **Adds** section is a standalone disclaimer about the manual real-host smoke governance protocol: it enumerates the protocol's components ("its two triggers, named owners, activation window, live-host precondition, pass/fail criterion, and merge-blocker / revert path"), states the protocol is "addressed to the bump/merge contributor and the `H6a` release-gate auditor, not to the `H4a` implementer," and notes the protocol "lives in `real-host-smoke-gate.md` and consumes this leaf's harness only as a downstream reader."

The paragraph conveys no implementation constraint on the `H4a` implementer — it describes content that lives entirely in another file and names topics it does not itself define. Every fact it carries is already stated elsewhere. The negative-scope point (the smoke run is not part of `H4a`'s self-check, never runs under `npm test`, and is defined in `real-host-smoke-gate.md`) is carried by `H4a`'s third **Tests** bullet. The audience claim ("not to the `H4a` implementer") and the harness-consumption relationship ("`H4a` end-to-end harness ... reference") are both stated verbatim in `real-host-smoke-gate.md`'s own opening paragraphs.

The paragraph is residue from the extraction that moved the full governance protocol body out of `H4a` into `real-host-smoke-gate.md`: after the body relocated, the pointer/disclaimer prose left behind in **Adds** duplicates the Tests bullet and the destination file. It enlarges the leaf's prose surface without adding load-bearing scope.

## Solution Space

**Shape:** single

### Recommendation

Strike the standalone disclaimer paragraph from `H4a`'s **Adds** section — the paragraph beginning "The manual real-host smoke governance protocol that gates a double-vs-real divergence ..." and ending "... consumes this leaf's harness only as a downstream reader."

Deletion loses no load-bearing scope: the negative-scope fact (the manual smoke run is not part of the harness self-check, never runs under `npm test`, and is defined in `real-host-smoke-gate.md`) is already carried by `H4a`'s third Tests bullet, and both the audience pin ("not to the `H4a` implementer") and the harness-as-downstream-reader relationship are already stated in `real-host-smoke-gate.md`'s opening paragraphs. The fix is internal to `docs/plan_topics/H4a-factory-shell-and-harness.md`; `real-host-smoke-gate.md` is read-only for this change and the spec is untouched.

## Consequence

**Severity:** cosmetic

The duplicated disclaimer does not mislead the implementer or change what `H4a` ships — the negative-scope it asserts is already carried by `H4a`'s third Tests bullet and by `real-host-smoke-gate.md` itself. It is redundant prose that widens the leaf's read surface and invites drift if the smoke protocol's owner/scope wording is later edited in one location but not the other.
