## Problem

`V9d`'s **Adds** claims ownership of the entire `loom-system-note` runtime-event shape — the single `pi.sendMessage` call with its four `details` variants (`{ diagnostics }`, `{ event }`, `{ structural }`, `{ recovery }`), the group-A/group-B routing partition, the dedup tuple, and the per-variant `display`/`content` matrix from `spec_topics/pi-integration-contract/runtime-event-channel.md`. Its **Tests** assert only PIC-1 and that each always-log kind emits exactly once with the success-side null-policy honoured. The normative content that distinguishes one note shape from another is therefore unasserted: nothing witnesses that a given failure emits the correct `(details-key, display, content)` triple, nor the spec's load-bearing rule that a single always-log failure routes to exactly one `details` shape (group A → `details: { event }`, group B → `details: { diagnostics }`) with no fan-out. An implementation that misroutes a failure, sets the wrong `display` flag, or fans one failure across both variants passes V9d's exit gate green.

**Verdict:** present-since-inception

**History:** `V9d-runtime-event-channel.md` and `V9d-T-runtime-event-channel.md` entered the corpus together in the single plan-build commit c6a664e. The Tests sections of both leaves have carried only the PIC-1 bullet and the always-log-once / null-policy bullet from that first commit; assertions for the `details`-shape / `display`-`content` matrix and the group-A/B routing partition were never present. The defect is original to the leaf, not introduced by a later edit.

## Solution approach

Add to the **Tests** of `docs/plan_topics/V9d-T-runtime-event-channel.md` assertions covering the per-variant `(details-key, display, content)` triple for each row of the spec's matrix and the group-A/B single-shape routing rule (a single always-log failure emits exactly one `details` shape with no fan-out across variants). Mirror the new assertions into the **Tests** of `docs/plan_topics/V9d-runtime-event-channel.md` and add a corresponding clause to its **Ships when**.

## Solution constraints

- The matrix rows and the group-A/B membership lists are spec-owned in `spec_topics/pi-integration-contract/runtime-event-channel.md`; the plan references them and must not restate or add normative rules.
