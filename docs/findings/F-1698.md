## Finding

PIC-9 (`subagent.md`, `id="pic-9"`) pins a complete observable disposition for the case where the cancellation listener's synchronous `AgentSession.abort()` call throws: the runtime traps the error, routes it through `loom/runtime/internal-error`, and does not alter the invocation result on any disposal trigger — on an `Err`/panic teardown the original error the surrounding `finally` was protecting is not masked, and on a normal-return teardown the loom's `Ok` final value is preserved (the trapped throw never promotes `Ok` to `Err`). The parent-observable surface is also fixed: a slash-command surface emits the `"loom /<name> aborted with internal error: <message>"` system note, while an `invoke` parent observes `Err(InvokeInfraError { kind: "invoke_infra", cause: "internal_error", ... })`.

Every element of that disposition is observable, yet no acceptance fixture pins it. The spec never states how a throwing synchronous `abort()` is injected (no fake-`AgentSession` hook is named for it, unlike the `sendUserMessage`-blocks fake that PIC-22 explicitly calls out), which disposal triggers a conformance suite must cover (normal-return / `Err` / panic), nor which surface assertions distinguish a conformant runtime from one that masks the original result or mis-routes the diagnostic. The sibling `dispose()`-throw path (`loom/runtime/subagent-dispose-failure`) has the same gap. The PIC-9 test leaf in the plan (`V9i-T`) lists the dispose-in-`finally`/idempotence/no-mask invariants but contains no bullet for the synchronous `abort()`-throw route, so the gap is mirrored in the plan's coverage rather than caught by it.

This is the `AgentSession.abort()` synchronous-throw path specifically — distinct from the forwarding-listener `loomAbort.abort()` throw (owned by Cancellation and plan leaf `V17b`) and from the abandonable-Promise *late rejection* of the discarded `abort()` Promise (owned by the swallowing-handler rule and plan leaf `V9o`), both of which already have their own fixtures.

That commit pinned the routing prose but added no accompanying acceptance fixture. Three later commits elaborated the same clause — 46e848b (late-rejection vs synchronous-throw split), 61b4063 (dispose-mid-abort presupposition), and 7238428 (Ok-path no-mask wording) — and none added a fixture (`git show` on each touched no `fixture`/`conformance`/`witness` text for this path). The testability gap has therefore co-existed with the clause since its introducing commit.

## Solution Space

**Shape:** single

### Recommendation

Add an acceptance fixture to PIC-9's `AgentSession.abort()` paragraph (and propagate it into `V9i-T`'s PIC-9 test bullet) covering the synchronous `abort()`-throw route:

- **Injection:** drive the throw through a fake `AgentSession` whose `abort()` throws synchronously, reusing the existing fake-`AgentSession` test seam (the same seam PIC-22 and `V9i-T` already exercise for `sendUserMessage`).
- **Triggers to cover:** one normal-return teardown and one `Err`/panic teardown, so both arms of the result-preservation invariant are asserted.
- **Assertions:** exactly one `loom/runtime/internal-error` diagnostic is emitted; the parent observation matches the surface — slash-command parent sees the `"loom /<name> aborted with internal error: <message>"` system note, `invoke` parent sees `Err(InvokeInfraError { kind: "invoke_infra", cause: "internal_error", ... })`; and the original result is unmasked — the protected `Err`/panic still surfaces on the error-teardown trigger, and the loom's `Ok` final value still surfaces (never promoted to `Err`) on the normal-return trigger.

Edge cases the fixture must not blur: this is the synchronous-throw path only — keep it distinct from the late-rejection swallowing fixture (`V9o`, no diagnostic, three-channel suppression) and the forwarding-listener `loomAbort.abort()` throw fixture (`V17b`), which assert different dispositions on adjacent surfaces.

## Consequence

**Severity:** advisory

The runtime behaviour is fully specified, so implementers can build a correct teardown path without the fixture; what is missing is pinned conformance coverage. Two conformance suites may diverge on whether the `abort()`-throw route is exercised at all, and a regression that masks the original `Err`/`Ok` or mis-routes the diagnostic could ship undetected because no fixture forces the assertion. The cost is degraded test coverage, not divergent runtime behaviour.
