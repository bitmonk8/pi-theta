## Problem

`V4e`'s **Adds** and **Tests** fields both lean on `CIO-1` to fix the `ERR-16` load-time cross-route ("consults `V16a`'s cross-ceiling arbitration seam at slash-load per CIO-1"; "cross-routed via CIO-1 / ceiling #3 no-retry"), but its **Spec.** field links only `errors-and-results/error-model.md`. `CIO-1` is a REQ-ID defined elsewhere — `spec_topics/hard-ceilings/ceilings-3-and-4.md#cio-1` — whose normative text (ceiling #3 evaluated at slash-load before any runtime-class ceiling, the ceiling-#4 slash-load `params` arm routed by ceiling #3's failure-mode templates) is what actually fixes the routing. With no Spec link to that anchor, "per CIO-1" is unresolved at the leaf, and two implementers can produce incompatible `ERR-16` load-time routing logic.

**Verdict:** present-since-inception

**Introducing commits:** `c6a664e` (`pi-loom plan: build/update plan for spec.md + review`, 2026-06-10) — the commit that first added `docs/plan_topics/V4e-pre-evaluation-failures.md`.

**History:** At the file's first commit the **Spec.** field already linked only `error-model.md`, while the `ERR-16` test bullet already cited "cross-routed via CIO-1" — so the cite-without-Spec-link gap is present-since-inception. A later commit `e2b7e81` (`resolve "isolated cross-ceiling unit interface/authority undefined"`, 2026-06-10) added the Adds sentence "consults `V16a`'s cross-ceiling arbitration seam at slash-load per CIO-1", broadening the `CIO-1` surface in the body but not closing the Spec-link gap. `git log -S 'CIO-1'` and `git log -S 'error-model.md'` over the leaf file corroborate both points.

## Solution approach

In `docs/plan_topics/V4e-pre-evaluation-failures.md`, add the `CIO-1` definition page to the **Spec.** field alongside the existing `error-model.md` link, pointing at the anchored target `../spec_topics/hard-ceilings/ceilings-3-and-4.md#cio-1`. The `V4e-T` tests task shares this leaf page, so the single Spec-field edit covers both tasks.
