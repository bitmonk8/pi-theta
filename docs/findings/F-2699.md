## Finding

The normative authoring rule "every leaf-ID in a closing-leaf cell MUST therefore be backtick-delimited" lives only in `docs/plan_topics/coverage-matrix.md`'s *Closing-leaf column authoring* preamble. `conventions.md` — which holds the *Leaf format* fields, the cross-cutting rules, and the *Transitive-completeness plan-maintenance* clause that describes the `H5d` gate reading the closing-leaf column — never restates or points to this authoring MUST. `grep -i backtick docs/plan_topics/conventions.md` returns nothing.

A contributor who edits `conventions.md` to understand coverage-matrix obligations (for example, when extending the *Transitive-completeness plan-maintenance* clause, or adding a new matrix row referenced from a leaf's `Deps`) is told that `H5d` "reads ... the right-hand closing-leaf column" but is never told that the column's tokeniser keys on backtick-delimited spans, nor that authoring a cell therefore carries a hard backtick MUST. The rule is discoverable only by opening `coverage-matrix.md` itself.

This is a discoverability / placement gap, not a correctness break: the matrix preamble does state the MUST prominently, so a contributor actually authoring a cell in `coverage-matrix.md` encounters it in the same file. The defect is the one-directional link — `conventions.md` references the matrix and `H5d`'s read behaviour without cross-referencing the authoring constraint that read behaviour depends on.

## Consequence

**Severity:** cosmetic

A contributor reading `conventions.md` for coverage-matrix-related rules is not pointed to the backtick-delimit authoring MUST and could edit or add a closing-leaf cell without learning the constraint that `H5d`'s tokeniser depends on. The risk is low because the canonical rule sits in the matrix preamble where cells are authored; the gap is purely the missing cross-reference from `conventions.md`.

## Issue introduction

**Verdict:** single-commit introduction

**History:** The backtick-delimit authoring MUST and its enclosing *Closing-leaf column authoring* preamble were first authored in `8b69cd9` (2026-06-18, "pi-loom plan: resolve \"H5d tokeniser depends on unstated backtick authoring convention\""). `git log -S "Closing-leaf column authoring" -- docs/plan_topics/coverage-matrix.md` and `git log -G "MUST therefore be backtick-delimited" -- docs/plan_topics/coverage-matrix.md` both report `8b69cd9` as the earliest commit containing the text. That commit touched only `coverage-matrix.md`, `H5d-transitive-completeness-gate.md`, and finding files — it added no corresponding rule or pointer to `conventions.md`. `grep -i backtick docs/plan_topics/conventions.md` returns nothing today, confirming the authoring MUST has lived solely in the matrix preamble since it entered the corpus. The placement and the absent `conventions.md` cross-reference both date from that single fix commit; no later edit altered the situation.

## Solution Space

**Shape:** single

### Recommendation

Keep the canonical normative MUST in `coverage-matrix.md`'s *Closing-leaf column authoring* preamble (its natural home, alongside the cells it governs) and add a cross-reference to it from `conventions.md` so a contributor working in `conventions.md` is routed to the authoring constraint. The natural anchor is the *Leaf format* §Deps coverage-matrix paragraph in `conventions.md` (the sentence beginning "In the **coverage matrix**, each spec REQ-ID maps to the **implementation** leaf..."), which already discusses the matrix; append a pointer noting that authoring a closing-leaf cell is governed by the backtick-delimit MUST in `coverage-matrix.md`'s *Closing-leaf column authoring* section, with a link to that section.

Prefer a cross-reference over copying the MUST verbatim into `conventions.md`: duplicating a normative MUST across two files invites drift between the two statements. If a restatement is used instead of a pointer, the matrix preamble must remain the single canonical source and the `conventions.md` copy must defer to it by link.

The spec corpus is read-only for this fix — the change is confined to the two plan documents named above.
