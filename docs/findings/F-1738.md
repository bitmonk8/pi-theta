## Problem

The schema-inference algorithm in `docs/spec_topics/query/query-forms.md` §Schema inference algorithm partitions enclosing AST constructs into a **Crossed (transparent)** list and a **Stopped (opaque)** list, but classifies the postfix error-propagation `?` operator (ERR-18) in neither. Postfix `?` sits between the query and the sink in the page's flagship typed-query idiom `let x: ReviewScore = @\`...\`?`. A literal reader can take `?` as an unenumerated opaque operator — stopping the walk, leaving the query untyped, violating QRY-2 and rejecting the page's own first worked example — or as transparent, where the binding annotation supplies the schema. The algorithm states neither reading, so two implementers diverge on the typed-query idiom.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `fae85d3` (Mon May 4 11:20:46 2026, "Tighten spec for implementation-plan readiness")
**History:** `git log -G "Crossed \(transparent\)"` on the spec corpus identifies `fae85d3` as the first commit to introduce the **Crossed / Stopped** two-list partition (then in `spec.md`); the partition migrated unchanged into `docs/spec_topics/query/query-forms.md` at `f5e89f4` (Thu Jun 4 10:38:09 2026, the topic-split commit). The page's flagship worked example `let x: ReviewScore = @\`...\`?` was introduced in the same `fae85d3` commit, so the algorithm has been silent on postfix `?` from the moment the algorithm was authored. Subsequent edits to the bullet — `10e3f6d` (forward-link to the explicit-ascription override rule) and `6406600` ("F-1201: opaque set" — expanded the operator enumeration to name `+ - * / % == != < <= > >= && ||` and the unary `!` and `-`) — touched the Stopped list without adding postfix `?` to either side; `6406600` in particular tightened the unary enumeration without addressing the postfix operator. The defect is structural to the partition's authoring, not a regression from any later edit.

## Solution approach

Classify the operand position of postfix `?` as transparent in the **Crossed (transparent)** bullet of §Schema inference algorithm, so the walk continues from `?` into its operand; ERR-18 grounds the unwrap (the operand is `Result<T, QueryError>` and `?` yields `T`). Update the `let x: ReviewScore = @\`...\`?` worked example so the crossing of `?` is observable against the algorithm.
