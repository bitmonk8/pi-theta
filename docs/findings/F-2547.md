## Finding

GOV-20 cites `docs/plan_topics/conventions.md:9` for the `V1`…`Vn` reservation, but that text is now on line 11; the line-number suffix is stale and will drift again on the next edit to `conventions.md`.

## Issue introduction

**Verdict:** specific-commit (per vector); the `pending` vector is a regression of a previously-fixed defect
**Introducing commits:** `fe8aa5c`, `5a772e3` (the `pending` cells); `8ee4a42` (the GOV-20 line-number drift)
**History:** The corpus is git-tracked. The two vectors entered independently.

- *Retired-in `pending` cells.* Commit `3f00256` (2026-06-05) resolved an earlier review finding, "Governance retirement tables ship unresolved commit-SHA placeholders": it replaced the `<retirement commit>` placeholders with concrete SHAs (e.g. GOV-11 → `64cdc60`) and removed the parenthetical that had licensed placeholders, hardening the column rule to "7-character abbreviated commit SHA or a release tag". The two `pending` cells were then added *after* that hardening — GOV-12's row by `fe8aa5c` (2026-06-05, "GOV-12 bundles ≥3 independently-testable sub-obligations") and GOV-14's row by `5a772e3` (2026-06-05, "GOV-14 two SHOULD-NOTs admit no mechanical witness"), per `git blame` of the two rows. They reintroduced the same defect class `3f00256` had just closed, under a new token (`pending` instead of `<retirement commit>`).
- *GOV-20 line-number drift.* The `conventions.md:9` citation was introduced by `f5e89f4` (2026-06-04, "enforce 100KB hard / 30KB recommended size cap on spec set"); at that revision the `V1`…`Vn` reservation sentence was on line 9 of `conventions.md` (`git show f5e89f4:docs/plan_topics/conventions.md`), so the citation was correct when written. Commit `8ee4a42` (2026-06-11, "Release gate is a fourth top-level section") later inserted the "Horizontal leaves may be presented…" paragraph at line 9, pushing the reservation sentence down to line 11, without updating the citing line number in `release-version-naming.md`.

## Solution Space

**Shape:** single

### Recommendation

In `release-version-naming.md`, either update the suffix to `:11`, or — preferably — drop the brittle line-number suffix and cite the file (and, if available, the section/anchor that owns the reservation) so the reference cannot silently drift when `conventions.md` is re-edited.

## Consequence

**Severity:** advisory

A governance page asserts a value-space rule for the `Retired in` column and then breaks it on two rows, leaving an internal inconsistency that an audit reader must reconcile; the GOV-20 citation sends a reader to the wrong line of the plan conventions file. Neither defect changes runtime behaviour or causes two implementers to diverge, but both degrade the trustworthiness of the bookkeeping the governance corpus exists to provide.
