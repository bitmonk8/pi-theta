## Finding

The single seam blockquote under `## Options surface` in `query/query-failure-and-repair.md` couples two future extensions with materially different timelines and anchoring stories:

- **Per-call timeouts**, which already have standalone, REQ-ID-anchored seam callouts on the sibling primitive pages (`invocation.md` INV-3, and the `loom 1.0 seam — per-call timeout` blockquote in `tool-calls.md`). The Surface-extensions inventory in `future-considerations/surface-extensions.md` lists this extension as a single bullet whose *Anchored at* line cites all three primitive pages.
- **Per-query overrides for `model`, `tools`, and `system`**, which are a query-only extension. The Surface-extensions inventory lists them as a separate bullet whose *Anchored at* line points at this same Query Options surface as their sole anchor anywhere in the corpus.

Because the two extensions ride a shared blockquote, "Per-query overrides" has no independent seam anchor — a reviewer or fixer who edits or relocates the timeout half necessarily disturbs the overrides half, and a reader auditing the per-query-overrides seam against the inventory cannot point at a paragraph that is *only* about overrides. The inventory page already treats the two as independent items; the topic-page anchor should match that shape.

**History:** Commit 09c42c0 added the `## Options surface` section to `spec_topics/query.md` for the first time, and the very first version of the paragraph already bundled both extensions in a single sentence ("…the deferred per-call timeout and per-query `model` / `tools` / `system` override extensions in Future Considerations…"). The sibling `invocation.md` and `tool-calls.md` "Options surface" paragraphs added in the same commit each named only the per-call-timeout extension, so the bundling was query-specific from the outset rather than a uniform corpus-wide pattern. Commit f5e89f4 (2026-06-04, the 30 KB / 100 KB size-cap split) moved the section verbatim from `query.md` into the new child page `query/query-failure-and-repair.md` and reflowed the prose into a `> **loom 1.0 seam — per-call timeout / per-query overrides.**` blockquote, but did not alter the bundling. Subsequent revisions (a6d82db prose `V1 → loom 1.0` rename; 8362824 demotion of the consumer obligation to a non-normative implementation note) likewise preserved the coupled shape. The defect has therefore been continuously present since the section first existed.

## Solution Space

**Shape:** single

### Recommendation

Replace the single `> **loom 1.0 seam — per-call timeout / per-query overrides.**` blockquote with two adjacent callouts under the same `## Options surface` heading, each describing the same open-struct seam from the angle of one extension:

- `> **loom 1.0 seam — per-call timeout.**` — phrased to match the existing `invocation.md` / `tool-calls.md` per-call-timeout callouts (open-struct rationale + "allows the deferred per-call timeout extension in [Future Considerations] to land additively"), so the three primitive pages share one consistent shape.
- `> **loom 1.0 seam — per-query `model` / `tools` / `system` overrides.**` — same open-struct fact, but anchoring the query-only overrides extension. Give it its own anchor (e.g. `<a id="v1-seam-per-query-overrides"></a>`) so the Surface-extensions inventory bullet for "Per-query overrides for `model`, `tools`, and `system`" can point at a paragraph that is exclusively about overrides.

Both callouts can state the open-struct fact (consumers tolerate unknown fields) once, with the second deferring to the first by short cross-reference, to avoid duplicating the non-normative implementation note. Then update the *Anchored at* line for "Per-query overrides for `model`, `tools`, and `system`" in `future-considerations/surface-extensions.md` from `[Query — Options surface](../query.md)` to the new overrides-only anchor. The per-call-timeout *Anchored at* line already cites three primitive pages and needs no change beyond optionally pointing at the new timeout-only anchor instead of the section heading. No REQ-ID needs to be coined — neither half currently carries one, and the surrounding `query-failure-and-repair.md` seam blockquotes (e.g. the pre-flight token-count nullability seam) follow the same un-numbered convention.

## Consequence

**Severity:** advisory

Per-query overrides currently have no independently-citable seam anchor: their *Anchored at* line in the inventory points at a paragraph whose first half is about a different extension on different primitive pages. Future edits to the timeout half (renaming, anchor moves, REQ-ID coining, relocation onto the primitive pages alongside INV-3) cannot be done without touching prose that documents a different deferral, and the two extensions cannot be revised on independent cadences without first decoupling them. No implementer behaviour diverges today.
