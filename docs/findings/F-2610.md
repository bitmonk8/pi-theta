## Finding

In `query/query-failure-and-repair.md`, `## Options surface` is a standalone H2 whose entire body is a single non-normative seam blockquote. The blockquote states that the runtime-internal options record is an **open struct** (consumers tolerate unknown fields) so the deferred per-call-timeout and per-query `model`/`tools`/`system` override extensions can land additively. It is explicitly labelled "Non-normative implementation note" and pins no loom 1.0 obligation an implementer must satisfy now.

The heading "Options surface" reads as a present feature of the query primitive, but the section delivers no current capability — it is purely a forward-compatibility note for deferred work that is already enumerated in `future-considerations/surface-extensions.md` (the "Per-call timeouts" and "Per-query overrides for `model`, `tools`, and `system`" bullets, both anchored back at this surface).

The same file already demonstrates the house style for seam notes: the `ContextOverflowError` pre-flight-token-count seam is a `> **loom 1.0 seam — …**` blockquote nested inside a real normative section ("Detection of `ContextOverflowError`"), not a standalone heading. The Options-surface seam is the odd one out — a top-level heading whose only content is a deferral note, inconsistent with its sibling seam's treatment and overstating the section's normative weight.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commit:** `09c42c0` (2026-05-05) — "pi-loom spec: resolve \"Surface-extension Seam annotations carry V1 obligations from non-goals page\""
**History:** The `## Options surface` H2 was authored net-new into `spec_topics/query.md` by `09c42c0` while resolving an earlier review finding about surface-extension seam annotations; that commit created parallel `## Options surface` standalone sections across the query, tool-calls, and invocation pages, each body being a single open-struct seam paragraph. The paragraph was later reshaped into the `> **loom 1.0 seam — per-call timeout / per-query overrides.**` blockquote form (`52219c4`, 2026-05-06) and relocated into `query/query-failure-and-repair.md` by the topic-file split (`f5e89f4`, 2026-06-04). The standalone-heading-over-a-pure-deferral structure persisted unchanged across every reshaping — the defect was present from the section's first authoring, not introduced by the later reshapes.

## Solution Space

**Shape:** single

### Recommendation

Remove the standalone `## Options surface` heading and fold its seam note into the existing `## Failure modes` section as a `> **loom 1.0 seam — per-call options.**` blockquote, matching the treatment of the sibling pre-flight-token-count seam in the same file. The open-struct / tolerate-unknown-fields contract and the forward-reference to Future Considerations stay verbatim; only the framing changes from a feature-styled heading to a clearly-labelled seam note.

Preserve a stable anchor at the seam's new location (e.g. an explicit `<a id="options-surface">` or a `v1-seam-`-prefixed id) so the `surface-extensions.md` cross-references that point at "Query — Options surface" retain a target. Note those cross-references currently link to `../query.md` (the aggregator, with no fragment) rather than to `query/query-failure-and-repair.md#options-surface`; the fold does not require touching them, but retaining the anchor keeps a future link-repair pass viable.

## Consequence

**Severity:** cosmetic

A standalone heading whose body is a non-normative deferral note overstates the section's weight and is inconsistent with the file's other seam (a nested `> **loom 1.0 seam — …**` blockquote). The open-struct seam contract is unaffected either way, so no implementer diverges; the cost is reader confusion and organisational drift only.
