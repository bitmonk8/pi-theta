## Finding

The subagent-mode bullet under `### Scope of a loom file` in `docs/spec_topics/overview.md` ends with the clause "the intermediate transcript is private to the loom and is not retained by the runtime after the loom returns." Two distinct problems compound:

1. **Ambiguous referent.** "Not retained by the runtime" reads two ways. Read narrowly, it is the GC/lifetime claim that no reference to the intermediate-turn data is held after the per-invocation `finally` returns — an internal property no observer can witness. Read broadly, it asserts that the spawned `AgentSession`'s on-disk / in-memory transcript is actively torn down. The Pi Integration Contract pins the actual obligation as `PIC-9` — the `AgentSession.dispose()` call in the per-invocation `finally` (per `docs/spec_topics/pi-integration-contract/subagent.md` and the `dispose()` member declared in `host-interfaces-core.md`) — but the overview clause does not point at it and does not say which component is responsible for any teardown.

2. **No observability surface.** The conformance harness has no way to prove "the runtime is not holding a reference." `V9i` tests `dispose()`-in-`finally` idempotency and isolation (no shared transcript / no shared tool table) against a fake `AgentSession`; nothing tests, or can test, a retention-after-return predicate. The observable property the spec actually needs in the overview is the propagation-and-isolation property — that **only the return value crosses back to the caller**, that no intermediate turn appears in the caller's session, and that the parent's `match` / `?` arm observes the result *after* `dispose()` has run (as `subagent.md` already pins under "Disposal must precede the parent observing the result").

The polished overview clause should state the observable propagation-and-isolation property and forward-link to PIC-9 for the disposal obligation, rather than restate a retention claim no test can witness.

## Solution Space

**Shape:** single

### Recommendation

Rewrite the subagent-mode bullet to pin only the observable propagation-and-isolation property and forward-link the disposal obligation to its owning page. Replace

> the intermediate transcript is private to the loom and is not retained by the runtime after the loom returns

with

> only the loom's return value crosses back to the caller; no intermediate turn appears in the caller's session or in any value reachable from loom code, and the parent observes the result only after the spawned `AgentSession` has been disposed per [Pi Integration Contract — PIC-9 Subagent session lifecycle](./pi-integration-contract/subagent.md).

Edge cases the implementer must watch:

- Keep the future-work parenthetical's removal under the sibling finding ("Scope of a loom file — `mode: subagent` future-work parenthetical") in scope; the recommended rewrite already drops it, but do not re-introduce the `loom test` aside.
- Do not add a new normative obligation in the overview; the rewrite forward-links to PIC-9 and adds no requirement of its own.
- Mirror the wording in the `glossary.md` `prompt mode` / `subagent mode` entry if the "discarded when the loom returns" phrasing reads as a retention claim after this edit; the glossary entry should converge on the same propagation-and-isolation framing, not on a retention claim.

## Consequence

**Severity:** advisory

A reader of the conceptual overview comes away believing the runtime makes a retention/disposal guarantee that PIC-9 alone owns, and a conformance author looking for the test surface to lock down "not retained" finds none. The defect does not block implementation — `V9i` proceeds against the PIC obligation — but the overview's framing weakens the spec's input-reproducibility discipline by pinning a property no test can witness.
