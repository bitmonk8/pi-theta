## Finding

The `### V8 — Pi host seams` section lists `V8a … V8e` in letter order with no interleave note, yet `V8a` (`Checkpoint` seam) declares `V8d` in its **Deps** (`V8a-T`, `H3a`, `V8d`). The dependency is real: `V8a`'s macrotask-yield-for-`loop-iter` checkpoint semantics rest on the `setTimeout`/`clearTimeout` members the `Clock` seam owns, and `Clock` is built by `V8d`. So the section presents `V8a` before the sibling it depends on.

Every other vertical slice in `plan.md` that contains a non-letter-order dependency edge carries an explicit interleave note (V9, V10, V11, V12, V14, V15, V17 all close with "Sequence by **Deps**, not slice number" and name the inverting edge). V8 is the only slice with an intra-slice letter-order inversion that flags nothing. A reader scanning the V8 section in listed order sees no signal that `V8d` must precede `V8a`, even though the rest of the plan trains readers to treat these notes as the canonical navigation aid for exactly this situation.

## Consequence

**Severity:** advisory

The Deps-driven pickability predicate (How-to-use step 3) still enforces correct build order — `V8a` is not pickable until `V8d-complete` is tagged — so this does not block or mis-sequence the build. It is a navigation-consistency gap: V8 alone among slices with a letter-order inversion gives no "Sequence by Deps" cue, so a reader following section order is briefly misled into reading letter order as build order.

## Solution Space

**Shape:** single

### Recommendation

Add an interleave note to the `### V8 — Pi host seams` section of `docs/plan.md`, placed directly under the section heading in the blockquote form the sibling slices use (e.g. the V10/V11/V12 notes), naming the inverting edge and closing with the standard "Sequence by **Deps**, not slice number." sentence. The note must state that `V8d` (`Clock` and `IdSource` seams) lands before `V8a` (`Checkpoint` seam) because `V8a`'s `loop-iter` macrotask-yield checkpoint semantics depend on the `setTimeout`/`clearTimeout` members `V8d`'s `Clock` seam owns.

Author the note as content matching the sibling slices' phrasing; do not reorder the V8 leaf list (every other inverting slice resolves this with a note, not a re-listing, and a reorder would make V8 the lone reordered slice). The leaf files themselves need no edit — `V8a`'s `Deps` already carries `V8d`.
