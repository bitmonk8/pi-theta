## Finding

`V12a` and its paired tests task `V12a-T` both assert `SLSH-2` against the in-process Pi session double, and their SLSH-2 bullet pins the streamed-token-vs-`ctx.waitForIdle()` ordering to the **session-double fidelity contract** — naming `H4a` inline ("whose `ctx.waitForIdle()`-vs-streaming ordering is fixed by the **session-double fidelity contract** (`H4a`) … the ordering asserted here is the one that contract requires the double to model"). That contract is a Class-2 cross-leaf seam: `H4a`'s Adds establishes it as "the enumerated set of pinned-`@earendil-works/pi-coding-agent` behaviours the in-process double must reproduce," axis (i) of which is the exact streamed-token-before-`ctx.waitForIdle()` ordering SLSH-2 relies on.

Per `conventions.md` §Leaf format, a Class-2 seam binding is consummated only when "a leaf listing this leaf in its Deps names the seam in its own Adds or Tests." `V12a` names the seam in Tests but lists `V12a-T, V9c, V11f, V4a, V13c` in Deps; `V12a-T` lists `V9c, V11f, V4a, V13c`. Neither declares `H4a`. The binding is therefore one-sided — the seam is consumed without the matching declared edge.

This is also a local inconsistency: every other leaf that drives the `H4a` harness / session double declares `H4a` directly in its Deps — `M`, `M-T`, `H5a`, `H7a`, `V4c`, `V4c-T`, `V7a`, `V7a-T`, `V9a`, `V9a-T`, `V9k`, `V9k-T`, `V17a`, `V17a-T`, `V17b`, `V17b-T`, `V18d`, `V18d-T`. The sibling pattern the source notes — `V13c`/`V13d` declaring `H4b` for the response-programming surface they bind — is the same discipline applied to the other half of the harness contract. `V12a`/`V12a-T` are the only harness-driven leaves missing the edge.

## Solution Space

**Shape:** single

### Recommendation

Add `H4a` to the Deps of both leaves so the declared edge matches the SLSH-2 seam binding, exactly as the other harness-driven leaves do.

- In `docs/plan_topics/V12a-slash-dispatch.md`, change the Deps line to `**Deps.** \`V12a-T\`, \`V9c\`, \`V11f\`, \`V4a\`, \`V13c\`, \`H4a\``.
- In `docs/plan_topics/V12a-T-slash-dispatch.md`, change the Deps line to `**Deps.** \`V9c\`, \`V11f\`, \`V4a\`, \`V13c\`, \`H4a\``.

No SLSH-2 prose change is needed — the binding text is already correct; only the declared edge is missing. Edge case for the implementer: if the "H4a — too large" split lands first and relocates the session-double fidelity contract to a new leaf, point the new Deps edge at whichever leaf then owns the contract rather than at `H4a`.

## Consequence

**Severity:** advisory

The seam binding is undeclared, breaking the Class-2 seam discipline `conventions.md` defines and diverging from every other harness-driven leaf. `H4a` is currently reachable from `V12a-T` only transitively (`V12a-T → V13c → H4b → H4a`), so the ordering accident is masked today; but the plan invites free re-sequencing "as long as the deps DAG is respected," and if `V13c`'s chain to `H4b`/`H4a` is later pruned or restructured, `V12a-T` could be sequenced before the harness and session double exist — its SLSH-2 tests would then fail at setup (a harness/fixture red), which the per-phase TDD ritual explicitly disqualifies from satisfying the "fail red for the intended reason" gate.
