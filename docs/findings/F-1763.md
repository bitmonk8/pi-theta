## Problem

Version-bump step 5's "PIC owns the routing semantics" sentence pins its routing contract for the two snapshot/probe sub-triggers on a human-judgment predicate: each failure-output entry MUST be "unambiguously routable by a reader … without an out-of-band translation table." No deterministic test a conformance reviewer or build gate can run decides whether a set of discriminator strings satisfies this; two reviewers could disagree on whether a pair of discriminators is routable. The step's genuinely mechanical obligations (orthogonal individually-discriminable failures, implementation-pinned literal strings as permanent test invariants, the grep-able `tsc` brand string for trigger (ii)) sit beside the predicate without being elevated to its normative role.

## Solution approach

Rewrite step 5's "PIC owns the routing semantics" routability MUST to demote the reader-judgment phrasing to non-normative rationale and pin a mechanical distinctness predicate over the per-trigger discriminators as the normative conformance criterion, assertable by a permanent test invariant. The predicate must span the full discriminator set across both sub-triggers, including the `tsc`-emitted brand string `loom/typecheck/session-shutdown-reason-snapshot` (trigger [(ii)](#bump-step-5-trigger-ii)'s type-equality discriminator, which is not implementation-owned).

## Solution constraints

- Out of scope: pinning the literal discriminator strings — the implementation owns them under the existing PIC ownership clause; the new predicate fences discriminator distinctness, not the token values.
