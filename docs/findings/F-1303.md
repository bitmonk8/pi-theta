## Problem

DISC-5's final sentence — "When two patterns of the same class match one path, insertion order is preserved" — has no falsifiable conformance scenario in any of the four override classes the same paragraph defines. Plain includes union into a set, `!` removes, `+` re-admits, and `-` removes with final precedence; in every class two patterns of that class matching one path produce the same membership outcome regardless of order. There is no observable effect "insertion order is preserved" could pin, so an implementation that hashes the patterns into an unordered structure satisfies the MUST vacuously.

## Solution approach

Demote the final within-class insertion-order sentence of DISC-5 (`id="disc-5"`) from a normative MUST to a non-normative note, grounding the demotion in the four override classes' idempotence under same-class overlap.
