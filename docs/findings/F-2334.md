## Finding

`H5b`'s **Deps** field is a bare list of leaf IDs followed immediately by a multi-paragraph italic parenthetical explaining *why* the listed set is the complete coverage-producing set (which leaves supply which gate arm, the standing transitive-completeness plan-maintenance obligation, and the note that `H6a` inherits the set by depending on `H5b`). `conventions.md` §Leaf format defines **Deps** as "Other leaf IDs that must be complete first" — a list of leaf IDs — and places descriptive prose in the **Adds** field. The rationale block sits in the Deps region between the ID list and **Ships when**, so the Deps field is no longer a pure leaf-ID list as the leaf format prescribes.

The rationale is load-bearing, not cruft: `plan.md` (Release gate section) states that `H5b`'s Deps "carry the single statement of the transitive-completeness obligation," and `conventions.md` §REQ-ID discipline points at it. It must be preserved, just relocated out of the Deps field.

Note that the fixed leaf format (Spec/Convention, Adds, Tests, Deps, Ships when) has no **Notes** field; relocating the prose into a new field would change the leaf-format convention rather than apply it.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H5b-warn-only-canary.md`, relocate the coverage-producing-set rationale parenthetical (the `_(The dependency set is the complete coverage-producing set … cover an identical set of leaves.)_` block) out of the **Deps** field so that **Deps** holds only the leaf-ID list, matching the `conventions.md` §Leaf format definition. Move the rationale into the leaf's **Adds** field, which §Leaf format already designates as the descriptive-prose field, preserving the text — it is the single statement of the transitive-completeness obligation that `plan.md` (Release gate) and `conventions.md` §REQ-ID discipline reference; do not delete or summarise it. Do not introduce a new "Notes" field — the leaf format is a fixed field set and **Adds** is the sanctioned home.

The spec is read-only for this fix; the edit is confined to the `H5b` leaf file.

## Consequence

**Severity:** cosmetic

A reader or tool treating the Deps field as the pure leaf-ID list `conventions.md` §Leaf format prescribes encounters multi-paragraph prose mixed in; any tokenizer over `H5b`'s Deps (e.g. `H5d`'s expansion of `H5b`'s named singletons) must skip rationale text rather than parse a clean list. No implementer produces a divergent leaf from this; it is pure organisation.
