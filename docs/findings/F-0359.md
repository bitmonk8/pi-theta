## Problem

`V5e` (JSON document depth enforcement, hard ceiling #4) owns the per-boundary routing contract: the mapping from each of the five depth-walk enforcement sites to its destination surface class. In the leaf, this mapping is enumerated in full only inside the second **Tests** bullet; the **Adds** bullet references it obliquely as "emitting the `maxDepth` error with the per-boundary routing," with no enumeration and no forward-link to the canonical owner. The routing's authority is therefore lodged in test prose rather than in the **Adds** contract the implementer reads to build the leaf. The canonical owner of the mapping is the spec's per-boundary destination/surface table at `../spec_topics/hard-ceilings/ceilings-3-and-4.md#ceiling-4-table`, which `V14e`, `V15h`, and `V4e` already cite directly.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e — pi-loom plan: build/update plan for spec.md + review (2026-06-10, plan author)
**History:** The `V5e` leaf's first commit (c6a664e) already placed the per-boundary routing enumeration in a Tests bullet ("Per-boundary routing maps each site to its surface …") while the Adds bullet said only "with the per-boundary routing"; the misplacement was present at creation. A later commit (3fa39a9, 2026-06-11) elaborated the Tests-bullet routing into the decision-vs-carrier split but preserved the same placement, leaving the contract still absent from Adds.

## Solution approach

State the per-boundary routing contract in the **Adds** bullet of `docs/plan_topics/V5e-depth-enforcement.md`, forward-linking the canonical owner — the spec per-boundary table at `#ceiling-4-table` — rather than leaving the enumeration to a Tests bullet. Demote the second **Tests** bullet so it asserts against the Adds-stated contract rather than originating the enumeration. Apply the same Adds-vs-Tests correction to the paired `V5e-T` leaf so its Tests bullet references the contract rather than re-stating it as the authority.

## Solution constraints

- The spec table at `#ceiling-4-table` is the canonical owner; cite it, do not edit the spec.
- Keep the two `params` arms distinguishable: runtime `invoke(...)` `params` routes to `InvokeInfraError`, while slash-load `params` cross-routes to ceiling #3 per `CIO-1`.
- Out of scope: the carrier-wrapping assertions owned by `V13c`, `V14e`, `V15h`, and `V4e`; this fix touches only `V5e` and `V5e-T`.
