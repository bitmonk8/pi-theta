## Finding

`V4e` packages all eight closed pre-evaluation failure causes — `ERR-1`…`ERR-7` plus `ERR-16` — under a single `Ships when` gate ("`npm test` proves all eight pre-eval causes route to `loom-system-note` without firing a turn"). Seven of these (`ERR-1`…`ERR-6`, `ERR-16`) are self-contained routing assertions: each drives one synthetic pre-eval failure and checks that it surfaces on the `loom-system-note` channel with `triggerTurn:false`. `ERR-16` additionally consults `V16a`'s cross-ceiling arbitration seam and wraps a `V5e` depth-walk breach, but it remains a single load-time routing check.

`ERR-7` is a different complexity class. It is a watcher-time reload integration scenario that injects a synthetic failure through `V9b`'s `ReloadFailureInjector.injectReloadFailure` interface and must exercise both the `V9b`-owned arms (the `loom/runtime/registry-swap-failed` registry-swap arm and the `.loom`/`.warp` re-parse arm) and `V10d`'s settings-re-merge arm simultaneously. It is the sole reason `V4e` carries `Deps` on `V9b` and `V10d`; the other seven causes need none of that wiring.

Because one `Ships when` gate covers all eight causes, progress on the seven routing checks is unobservable until the `ERR-7` reload-injection harness is also stood up, and a single-pass review must hold the routing taxonomy and the multi-arm reload-injection seam in scope at once.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commit:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** `V4e` has bundled the full pre-evaluation failure set (`ERR-1`…`ERR-7` + `ERR-16`) under the "all eight pre-eval causes" `Ships when` since the leaf was first authored in `c6a664e`. Later commits (`49e3837`, `66acde6`, `b2d5a1f`, `d64dce5`, `fdce2a2`, all 2026-06-11) did not introduce the bundling; they progressively deepened `ERR-7`'s integration burden — pinning the `ReloadFailureInjector` seam, its single owner, and the registry-swap / re-parse / settings-re-merge arms — which sharpened the heterogeneity between `ERR-7` and the seven routing checks but left the single-leaf structure unchanged.

## Solution Space

### Recommendation

Split the `V4e` leaf-pair into two implementation leaves, then propagate the split through the coverage matrix and `H5b`. Perform the leaf split first to bound the new leaf IDs and the `ERR` partition, then retarget the coverage references on that stable baseline.

**Leaf split.** Separate the seven routing checks from the `ERR-7` reload-integration scenario, each with its `-T` partner. Retain `V4e`'s existing ID for the routing leaf to minimize coverage-matrix and `H5b` churn.
- In `V4e-pre-evaluation-failures.md` and `V4e-T-pre-evaluation-failures.md`: keep `ERR-1`…`ERR-6` and `ERR-16`; remove the `ERR-7` Tests bullet; drop `V9b` and `V10d` from `Deps` (they exist only for `ERR-7`), retaining `V9a`, `V6a`, `V11f`, `V10a`, `V16a`, `V5e`. Reword `Ships when` from "all eight pre-eval causes" to the seven routing causes it now owns (`ERR-1`…`ERR-6`, `ERR-16`).
- Create a new `<new>` implementation leaf (and `<new>-T`) owning `ERR-7`, with `Deps` `V9b`, `V10d` (plus its `-T`), `Spec.` citing `error-model.md`, and a `Ships when` scoped to the single watcher-time reload-injection cause. Add the navigation bullet for it in `docs/plan.md` §V4. The new leaf ID must honour the project's leaf-ID scheme; leave it as the literal `<new>` placeholder until the implementer allocates a real ID (`no-invented-ids`).

**Coverage propagation.** Retarget every coverage-matrix and `H5b` reference of `V4e` to the post-split leaf set so the transitive-completeness gate stays green.
- `coverage-matrix.md`: split `ERR-1 … ERR-7 | V4e` into `ERR-1 … ERR-6 | V4e` (routing leaf) and `ERR-7 | <new>` (reload leaf). `ERR-16 | V4e` and the `V4e` member of `CIO-1 | V16a, V4e, V11f` stay the routing leaf, since `ERR-16`'s slash-load consult lives there.
- `H5b-warn-only-canary.md`: ensure both post-split leaf IDs are reachable from `Deps` so `H5d`'s per-cell at-least-one check stays green for the new `ERR-7 | <new>` row. If the `ERR-7` reload leaf's ID falls outside `H5b`'s `V4a`–`V4f` range, add it to `H5b`'s `Deps` explicitly (extend the range or add the new leaf as a named entry, depending on the allocated ID). The new leaf ID must be backtick-delimited in the matrix cell, since `H5d` tokenises the closing-leaf cell by backtick-delimited spans and otherwise silently drops it from the gate's set.

**Spec edits.** None.

## Consequence

**Severity:** advisory

A single `Ships when` over eight heterogeneous causes makes partial completion unobservable — the seven routing checks cannot be demonstrated done until the `ERR-7` watcher-time reload-injection harness (spanning `V9b`'s two arms and `V10d`'s settings-re-merge arm) is also wired, and the leaf cannot be picked up incrementally. The leaf is still implementable, so this degrades reviewability and incrementality rather than blocking it.
