## Finding

`V4c`'s `ERR-12` Tests bullet ends with a long scope-exclusion / governance aside: it re-states what the live re-assertion would test (cancellation does not mutate the real subagent `AgentSession`'s committed turns), labels it a "real-host-only behaviour … not asserted under `npm test`", names the smoke-gate criterion `(c)`, links `real-host-smoke-gate.md`, and attributes the live surface to `V9i` ("the live-surface re-assertion `V9i` carries off the `H4a` double"). A Tests bullet's job is to name the assertion the leaf proves; this aside instead documents work `V4c` explicitly does *not* do.

Every clause of that aside is already owned elsewhere. The paired test leaf `V4c-T` states the same boundary in one crisp phrase — "`ERR-12`: ERR-8 holds inside a subagent loom, exercised via the `H4a` harness modelling a subagent-mode callee — not the live `V9i` surface." `V9i`'s own `ERR-8` Tests bullet (the "delegated live-carrier witness for `V4c`'s ERR-8/ERR-12 deferral") carries the full real-host-only / smoke-criterion-`(c)` detail, and `real-host-smoke-gate.md` criterion `(c)` carries it again as the authoritative governance home. The `V4c` aside is therefore pure duplication of detail whose canonical owners (`V9i` and the smoke-gate note) already exist.

The redundancy was introduced incrementally: the crisp `V4c` `ERR-12` bullet acquired the delegated-witness sentence in one plan fix and was then expanded with the real-host-only / smoke-criterion-`(c)` text in the `F-1363` relocation that moved `V9i`'s real-session witness to the smoke gate — the same relocation that placed the authoritative copy in `V9i` and the smoke-gate note. The `V4c` copy is the residue of that move.

## Consequence

**Severity:** cosmetic

The duplicated aside is noise in a gate/assertion field; it does not mislead the `V4c` implementer (the bullet, and `V4c-T`, already make clear the live re-assertion is not `V4c`'s job). The cost is maintenance drift: three copies of the same smoke-criterion-`(c)` detail (`V4c`, `V9i`, the smoke-gate note) can diverge when only one is updated.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V4c-terminal-outcomes.md`, trim the `ERR-12` Tests bullet so it states only the test assertion and its scope boundary, matching the phrasing the paired `V4c-T` leaf already uses. Concretely, replace the current bullet body with the `V4c-T` form:

`- `ERR-12`: ERR-8 holds inside a subagent loom, exercised via the `H4a` harness modelling a subagent-mode callee — not the live `V9i` surface.`

Delete (do not relocate to `V4c` Adds) the "real-host-only behaviour … not asserted under `npm test` … manual real-host smoke gate (`real-host-smoke-gate.md` criterion (c)) … `V9i` carries off the `H4a` double" detail: `V9i`'s `ERR-8` Tests bullet and Ships-when, and `real-host-smoke-gate.md` criterion (c), already own it, so no information is lost. Moving the aside to `V4c` Adds would re-create the same duplication in a different field; the deletion is the fix. The `V9i` and `real-host-smoke-gate.md` files are read-only for this fix — leave their copies of the detail untouched.
