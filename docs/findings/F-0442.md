## Finding

In `conventions.md`'s cross-cutting rules, the *Sequential by default* rule explicitly ties itself to the external project-config file: "This rule subsumes the CLAUDE.md *"Never block the async runtime"* directive…". The adjacent *Specific exception types only* rule ("No `catch (e)` … — bind to a specific subtype or let the exception propagate") is the analogous pi-loom TypeScript operationalisation of CLAUDE.md's "Catch specific types or let crash" exception-handling directive, but it carries no corresponding note tying it back to that CLAUDE.md principle.

The result is an asymmetry within one bullet list: of the two cross-cutting rules that each restate a CLAUDE.md directive in pi-loom-specific terms, only one declares the relationship. A reader auditing plan-to-CLAUDE.md alignment sees one rule cross-reference CLAUDE.md and the structurally parallel rule stay silent, and cannot tell whether the omission is deliberate (the exception rule is *not* meant to be a CLAUDE.md reconciliation) or an oversight.

This is purely a documentation-consistency gap: the *Specific exception types only* rule's normative content — the banned `catch` forms, the rethrow-on-mismatch prohibition, the `no-broad-catch` ESLint backing, and the five enumerated exempt-broad-catch-site categories — is complete and unambiguous without the note. Nothing an implementer does changes whether or not the note is present.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** `c70ebce` — "pi-loom plan: resolve \"Sequential by default omits Never block the async runtime\"" (2026-06-12)
**History:** `conventions.md` entered the corpus at `31ff060` ("Move spec, plan, and review docs into docs/", 2026-05-08) with both the *Specific exception types only* and *Sequential by default* rules already present and **neither** referencing CLAUDE.md — a symmetric baseline. `git log -G "subsumes the CLAUDE.md"` and `git show c70ebce -- docs/plan_topics/conventions.md` confirm that commit added the "This rule subsumes the CLAUDE.md *Never block the async runtime* directive" clause to *Sequential by default* alone, while leaving *Specific exception types only* untouched. The asymmetry this finding flags therefore did not pre-exist `conventions.md`; it was introduced precisely when `c70ebce` gave one of the two parallel rules a CLAUDE.md note and not the other.

## Solution Space

**Shape:** multiple

There are two coherent ways to restore symmetry between the two CLAUDE.md-derived rules, pulling in opposite directions. They are mutually exclusive and must be decided jointly with the related cruft finding (see Related Findings), which proposes the removal direction directly.

### Option A — Add the reconciliation note (symmetry by addition)

**Approach:** Append a one-sentence note to the *Specific exception types only* bullet naming it as the pi-loom TypeScript reconciliation of CLAUDE.md's "Catch specific types or let crash" directive, mirroring the existing *Sequential by default* CLAUDE.md clause.

**Plan edits:** In `docs/plan_topics/conventions.md`, extend the *Specific exception types only* bullet with text to the effect of: "This rule is the pi-loom TypeScript reconciliation of the CLAUDE.md *Catch specific types or let crash* directive." No CLAUDE.md edit.

**Spec edits:** None.

**Pros:** Both CLAUDE.md-derived rules then declare their lineage; the alignment is explicit and traceable.

**Cons:** Adds a second cross-artifact dependency on an external project-config file; the note carries no normative content the rule lacks.

**Risks:** Drift if the CLAUDE.md directive wording later changes — now two rules reference it instead of one.

### Option B — Remove the CLAUDE.md reference from *Sequential by default* (symmetry by removal)

**Approach:** Strike the "This rule subsumes the CLAUDE.md …" clause from *Sequential by default*, so neither rule cross-references CLAUDE.md and the baseline symmetry is restored.

**Plan edits:** In `docs/plan_topics/conventions.md`, delete the sentence "This rule subsumes the CLAUDE.md *"Never block the async runtime"* directive: …" from the *Sequential by default* bullet (preserving the blocking-runtime ban text that follows it, which is independently load-bearing). No CLAUDE.md edit.

**Spec edits:** None.

**Pros:** Removes the lone cross-artifact dependency rather than doubling it; co-resolves with the cruft finding in a single edit; the blocking-runtime constraint stands without the CLAUDE.md aside.

**Cons:** Loses the explicit CLAUDE.md traceability that `c70ebce` deliberately added.

**Risks:** If a future reader expects plan conventions to name their CLAUDE.md provenance, the relationship becomes implicit only.

### Recommendation

Resolve via Option B, co-ordinated with the cruft finding. Removal restores the symmetric baseline with one edit, avoids adding a second external-file dependency, and the blocking-runtime ban in *Sequential by default* is fully stated without the CLAUDE.md aside. Watch one edge case: when striking the clause, keep the "synchronous blocking of the event loop in `src/**` … is forbidden" text that currently follows it — that normative content must survive the deletion. If the project instead values explicit CLAUDE.md provenance for plan conventions, adopt Option A for *both* rules so the two stay symmetric; do not adopt Option A for only one rule, which reproduces the asymmetry.

## Consequence

**Severity:** cosmetic

If the plan ships unfixed, the two CLAUDE.md-derived cross-cutting rules remain inconsistent in whether they declare their CLAUDE.md lineage. A reader cannot tell from the text whether the missing note is intentional. No implementer goes off-script and no gate is affected — the exception rule is mechanically enforced and self-contained regardless.
