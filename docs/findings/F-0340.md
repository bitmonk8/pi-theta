## Problem

`H5b`'s `Deps.` field declares itself "the complete coverage-producing set," but its V13-group enumeration reads `V13a`–`V13d`, `V13g` and omits `V13f`. `V13f` is a coverage-producing leaf: `coverage-matrix.md` lists it as a closing co-witness for the `cancellation.md` swallowing-handler-attachment un-anchored MUST, alongside `V14f`, `V15h`, `V9o`, and `V17a`, all of which `H5b` already pulls in. The omission falsifies two stated invariants — `H5b`'s own "complete coverage-producing set" claim and `H6a`'s "identical set by construction" inheritance claim — and violates the standing transitive-completeness plan-maintenance obligation in `conventions.md` *REQ-ID discipline*. It is not caught mechanically: `H5d`'s per-cell at-least-one arm stays green on the four present co-witnesses, so the warn-only canary can pre-flight before `V13f`'s coverage lands, defeating its purpose.

## Issue introduction

**Verdict:** single-commit introduction (with a later missed correction)
**Introducing commit:** `1db2787` — "pi-loom plan: resolve 'cancellation.md swallowing-handler attachment MUST has no coverage-matrix row'" (2026-06-17)
**History:** The plan corpus is git-tracked. `V13f` was created by `eaa5000` (2026-06-16, the "V13c bundles four independent spec obligations" split); at that point `H5b`'s `Deps.` read `V13a`–`V13d` and the coverage matrix did not yet name `V13f`. `1db2787` (2026-06-17 16:11) added the swallowing-handler MUST row listing `V17a, V14f, V13f, V15h, V9i` (the co-witness later renamed to `V9o`), making `V13f` coverage-producing per the matrix — but without the paired addition to `H5b`'s `Deps.` that conventions.md *Transitive-completeness plan-maintenance* requires; that is the commit at which the defect entered. `4a04570` (2026-06-17 18:23, "V13a bundles five independent query surfaces") then edited the same V13 region of `H5b`'s `Deps.` to insert `V13g` but did not add `V13f`, perpetuating the omission. `V13f` has never appeared in `H5b`'s `Deps.` (`git log -S '`V13f`' -- H5b-warn-only-canary.md` returns no commits).

## Solution approach

Add `V13f` to the V13 group of `H5b`'s `Deps.` field in `docs/plan_topics/H5b-warn-only-canary.md`. This reconciles the explicit enumeration with the inherited `M, V1a`–`V18d` set the parenthetical already describes, restores the transitive-completeness invariant, and makes `H6a`'s "identical set by construction" claim true transitively without any `H6a` edit.

## Solution constraints

- `V13e` does not exist, so the V13 group is legitimately non-contiguous; the addition must keep `V13f` and `V13g` as discrete entries and must not produce a range that implies `V13e`.
