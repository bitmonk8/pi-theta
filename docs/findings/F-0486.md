## Finding

`H5d`'s **Adds** describes the membership test the transitive-completeness arm performs: it requires at least one leaf-ID from each `coverage-matrix.md` closing-leaf cell to be a member of `H5b`'s `Deps.` "after expanding both sides' contiguous within-group `<group><letter>` ranges by letter suffix â€¦ and `H5b`'s named singletons (`H1a`, `H5a`, `M`)". The parenthetical enumerates the non-range (standalone) entries of `H5b`'s `Deps.`.

That enumeration is stale. `H5b`'s actual `Deps.` standalone singletons are `H1a`, `H5a`, `H5e`, `M` â€” four, not three. `H5e` (the un-anchored normative-MUST recogniser arm) was added to `H5b`'s `Deps.` as a coverage-producing singleton, but `H5d`'s mirror enumeration was never updated to match.

The arm's primary instruction reads membership against `H5b`'s `Deps.` field directly, and `H5e` is not itself a closing-leaf token in any coverage-matrix cell, so the gate behaves correctly today regardless of the parenthetical. The defect is that the parenthetical now misrepresents `H5b`'s `Deps.`: an implementer who treats it as the authoritative non-range membership set (rather than reading `H5b`'s `Deps.` verbatim) would hardcode `{H1a, H5a, M}` and silently drop `H5e` from the membership target. The drift is latent rather than live-breaking, but it is exactly the kind of hand-mirrored restatement that desyncs whenever `H5b`'s `Deps.` singleton set changes.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `617a3ad` (authored the mirror enumeration), `5f33832` (desynced it)
**History:** `617a3ad` (2026-06-16, "H5a bundles gate arms spanning four data sources") created `H5d` with the parenthetical `(\`H1a\`, \`H5a\`, \`M\`)`; at that commit `H5b`'s `Deps.` standalone singletons were exactly `H1a, H5a, M`, so the enumeration was correct on authoring. `5f33832` (2026-06-18, "H5a bundles seven gate arms behind a single Ships-when") created `H5e` and added it to `H5b`'s `Deps.` (singletons â†’ `H1a, H5a, H5e, M`), touching H5a/H5b/H6a/conventions.md/coverage-matrix.md but not `H5d-transitive-completeness-gate.md`. The defect is the divergence opened by `5f33832` editing the source-of-truth (`H5b` `Deps.`) without propagating to `H5d`'s hand-copied mirror.

## Solution Space

**Shape:** multiple

### Option A â€” Correct the enumeration to all four singletons

**Approach:** Update the parenthetical to match `H5b`'s current `Deps.` singletons.
**Plan edits:** In `H5d`'s **Adds**, replace `H5b`'s named singletons (`H1a`, `H5a`, `M`) with `` `H5b`'s named singletons (`H1a`, `H5a`, `H5e`, `M`) ``.
**Spec edits:** None.
**Pros:** Minimal touch; restores accuracy immediately.
**Cons:** Re-establishes the same hand-mirror that drifted here; the next singleton added to `H5b`'s `Deps.` reopens the identical divergence with no mechanical guard.
**Risks:** Recurrence on the next `H5b` `Deps.` singleton change.

### Option B â€” Make the membership target `H5b`'s `Deps.` verbatim, demoting the list to illustrative

**Approach:** State that the membership set is the whole of `H5b`'s `Deps.` (standalone entries are members directly; only contiguous `<group><letter>` ranges expand), so no second enumeration is maintained.
**Plan edits:** In `H5d`'s **Adds**, replace the `and \`H5b\`'s named singletons (\`H1a\`, \`H5a\`, \`M\`)` clause with text that the arm reads the full `H5b` `Deps.` set â€” every standalone entry is a member as written, range entries expand by letter suffix â€” and that any singleton list shown is illustrative of the `H5b` `Deps.` field rather than a maintained copy.
**Spec edits:** None.
**Pros:** Removes the drift surface entirely; the source of truth (`H5b` `Deps.`) becomes the only authority, matching how the arm actually operates.
**Cons:** Slightly more rewording than a one-token correction.
**Risks:** The reword must still make clear that non-`V` standalone entries (e.g. `H1a`, which closes `cka-21`/`cka-44`; `M`) participate in membership, so the at-least-one check still recognises closing leaves that are horizontal/MVP singletons.

### Recommendation

Take Option B. The issue introduction shows this exact divergence was opened by a later edit to `H5b`'s `Deps.` that did not propagate to `H5d`'s hand-copied list; pinning the membership target to `H5b`'s `Deps.` verbatim closes the drift class rather than resetting the same trap. Ensure the reword preserves the existing two facts the prose carries: contiguous `<group><letter>` ranges expand by letter suffix, and standalone entries (including non-`V` singletons like `H1a` and `M`) are members directly.

## Consequence

**Severity:** advisory

A reader who takes `H5d`'s parenthetical as the authoritative enumeration of `H5b`'s non-range `Deps.` entries omits `H5e` from the membership set the arm reconciles against. No coverage-matrix cell currently maps to `H5e` as a closer, so the gate is correct today; the hazard is that the hand-mirrored list will keep drifting as singletons are added to `H5b`'s `Deps.`, and a future singleton that *is* a closing leaf would then be silently dropped from the membership target, reddening a genuinely-covered cell.
