## Finding

`H1a`'s acceptance criteria describe the same "nothing has been implemented yet" precondition twice, with two phrasings of different scope. The first **Tests.** bullet requires `npm run build`, `npm test`, and `npm run typecheck` to "all run green on an empty `src/**` tree", while **Ships when.** requires the fresh-checkout `npm install && npm run build && npm run typecheck && npm test` to pass "with zero production source files".

These are not interchangeable. "An empty `src/**` tree" is a concrete, scoped predicate over one directory — and `src/**` is exactly the scope H2a's lint gates later key on. "Zero production source files" is an unscoped phrase that could be read to also cover production source outside `src/**`: the `extensions/index.ts` factory module (a directory value `["./extensions"]` is pinned in H1a's own manifest, though the file itself is authored by `H4a`) is production source that does not live under `src/**`. H1a also legitimately creates a non-production source file outside `src/**` (the `./tools/eslint-plugin-loom-local` plugin-skeleton entry module), so a reader reconciling the two phrasings cannot tell from the leaf alone which paths must be empty for the gate to be satisfiable.

In practice both phrasings resolve to the same satisfiable state at H1a sequencing, so this is an internal-consistency wording defect rather than a behavioural one.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (`pi-loom plan: build/update plan for spec.md + review`, 2026-06-10)
**History:** `git log -S` on both literals (`zero production source files` and `` empty `src/**` tree ``) over `docs/plan_topics/H1a-scaffold-and-toolchain.md` returns the single commit `c6a664e`, which is also the first commit to touch the leaf file. Both phrasings were authored together when the leaf was first created; the inconsistency has existed since inception and was not introduced by any later edit.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H1a-scaffold-and-toolchain.md`, in the **Ships when.** sentence, replace the phrase `zero production source files` with `an empty `src/**` tree` so it reads "… runs `npm install && npm run build && npm run typecheck && npm test` green with an empty `src/**` tree …". This aligns Ships when with the first **Tests.** bullet (already worded "on an empty `src/**` tree") on the more precise, directory-scoped predicate — the same `src/**` scope H2a's gates key on — and removes the ambiguity about whether production source outside `src/**` (e.g. `extensions/`) must also be absent. The `./tools/eslint-plugin-loom-local` plugin skeleton H1a creates lives outside `src/**`, so the `src/**`-scoped phrasing remains correct for the leaf's own deliverables.

## Consequence

**Severity:** cosmetic

A reader cannot tell from H1a alone whether the empty-tree precondition is scoped to `src/**` or to all production source paths (including `extensions/`). Because both phrasings are satisfiable in the same state at H1a sequencing, no implementer divergence in the shipped leaf results; the cost is reviewer friction reconciling the two phrasings.
