## Problem

`docs/plan_topics/conventions.md` routes five conceded correctness-residue classes to a single release-time manual review it calls the GOV-15 reviewer-inspection step — closure-captured / lazy-cache / DI-container singletons (*No globals, statics, singletons*), the indirect ambient reads the `H3a` identifier-keyed scan cannot witness, blocking-runtime calls (*Sequential by default*), the *Doc updates* artifacts, and the un-anchored-MUST residue under GOV-22 (*REQ-ID discipline*). The *No globals, statics, singletons* bullet further asserts the mechanical scans and that review "partition the space with no third gap between them," but no plan file enumerates the inspection's procedure or per-class checklist. The spec page the plan cites, `docs/spec_topics/governance/source-language-stability.md`, scopes its GOV-15 reviewer-inspection step exclusively to cross-release source-language equivalence and enumerates none of these implementation-residue classes. The review is therefore not reproducible, and a conceded-residue defect can pass while the plan reports the partition gap-free.

**History:** The named-but-unenumerated review surface accreted across four commits, each of which resolved an earlier plan-review finding by routing one more residue class to the GOV-15 reviewer-inspection step without any commit enumerating that inspection's checklist. `1035d0b` ("un-anchored-MUST closing-gate recogniser overclaims precision/recall") first routed token-less / narrative-ambiguous un-anchored-MUST residue to the GOV-15 step. `f005760` ("Architectural / ambient-access scan blind spots have no named compensating review gate") then routed the closure-captured / lazy-cache / DI-container singleton and the indirect ambient-read residue to the same step, and introduced the "mechanical scans and that named review step partition the space with no third gap between them" claim — the overclaim this finding targets. `c06749d` ("Doc-updates convention unenforced / mislabeled Tests bullets") routed the doc-updates residue to the same surface. `7ba3c93` ("Un-anchored-MUST recogniser seam-name exclusion arm") routed seam-name-dependent obligation residue there. Each fix correctly named a review surface but none defined it; the gap is the interaction of the four routings against an undefined inspection procedure.

## Solution approach

In `docs/plan_topics/conventions.md`, add an enumerated checklist for the GOV-15 release-time reviewer inspection that covers each residue class the conventions already route to it — the singleton and indirect-ambient-read classes under *No globals, statics, singletons*, the blocking-runtime class under *Sequential by default*, the doc-artifact class under *Doc updates*, and the un-anchored-MUST / GOV-22 class under *REQ-ID discipline*. Reconcile the "no third gap" partition claim in *No globals, statics, singletons* so it names the enumerated checklist as its backing. If a name disambiguation is wanted, state that loom's release-time residue review is a plan-side surface distinct from (or a superset of) the spec's source-language-equivalence GOV-15 step.

- Out of scope: `docs/spec_topics/governance/source-language-stability.md` is read-only for this fix — do not add the residue classes to its GOV-15 source-language-equivalence step.
