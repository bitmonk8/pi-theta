## Finding

ERR-8/ERR-9 forbid mutation/compensating-injection of the Pi-committed conversation but are stated entirely against Pi-internal state, with no loom-side observable; a conformance test has nothing concrete to assert against and risks being vacuous.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 02d7ca5 — pi-loom spec: resolve "T02 surface-extension seam obligations not consistently marked" (2026-05-08, Thomas Andersen); 2180f59 — pi-loom spec: resolve "Mid-stream cancellation: no observable rule for loom-runtime conversation behaviour" (2026-05-08, Thomas Andersen)
**History:** The two obligations entered the corpus independently, each in its own same-day commit, and each was authored without a loom-side acceptance criterion from the outset. `git log -S` localises the ERR-15 type-openness seam MUST to 02d7ca5, which added the seam blockquote ("`kind` … MUST be typed as `string`") carrying only the runtime-emission conformance assertion. The mid-stream non-mutation MUST NOTs trace to 2180f59, which introduced the "Mid-stream cancellation, conversation state" paragraph (later split into ERR-8…ERR-12 by cd0e82c and relocated into `error-model.md` by the f5e89f4 size-cap split); that paragraph stated the prohibition against Pi-committed conversation state but never named a loom-side observable. Neither defect is an interaction between the two commits — they are two independent single-commit introductions bundled under one review heading.

## Solution Space

**Shape:** single

### Recommendation

In `error-model.md`, add a conformance-oracle clause to ERR-8/ERR-9 specifying the loom→Pi conversation API surface whose *absence of invocation* after an abort constitutes conformance. State that loom drives the conversation only via `pi.sendUserMessage` (prompt mode) / `session.sendUserMessage` (subagent mode) and that conformance is witnessed by a spy/double over the Pi conversation interface recording **zero** truncate / rewrite / replace / remove / compensating-append calls (and zero further `sendUserMessage` injections) on the path between the cancelled streaming turn and the next driver send. Tie the oracle to the offline harness double for prompt/subagent modes and reference the real-host smoke gate for the live re-assertion (`V9i` / `real-host-smoke-gate.md`).

## Consequence

**Severity:** advisory

Both MUSTs are clearly worded, so an implementer can produce a working system — but the stated acceptance criteria do not bind the obligations. An ERR-15-conformant runtime that nonetheless types `kind` as a closed enum passes the named runtime test while silently foreclosing the deferred variant seam; an ERR-8/9 conformance test has no named observable and risks being written as a vacuous always-green check. Two reasonable implementers could ship different (and in the closed-enum case, non-conformant) results while both believing they passed.
