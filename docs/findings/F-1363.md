## Finding

`V9i`'s final Tests bullet — the `ERR-8` delegated live-carrier witness — asserts that "a mid-stream cancellation driven inside the real subagent `AgentSession` does not mutate the subagent session's committed turns," and `V9i`'s **Ships when** makes that an `npm test` gate ("asserts the `ERR-8` live-surface outcome — a mid-stream cancellation inside the real subagent `AgentSession` leaves its committed turns unmutated"). To observe a *mid-stream* cancellation, the spawned session must be producing a token stream that can be interrupted in flight.

Every other streaming-shaped test in this area is driven against a named double: `PIC-22` uses "a fake `AgentSession` whose `sendUserMessage` blocks," and the modeled-behaviour subagent-callee category (g) in `H4c` explicitly models the subagent outcome "through the double rather than the live `V9i` surface." This bullet alone switches to a *real* session spawned via `createAgentSession` + `SessionManager.inMemory`. Per `subagent.md`, that real session is driven by the loom's resolved `model` value passed through unchanged to a live provider; the only model source in the spawn path is `ctx.modelRegistry.getAvailable()`. Nothing in `V9i`'s Adds or Deps names an offline stub `Model<Api>` / transport (or any other seam) that would feed a real `createAgentSession` session a scripted, cancellable token stream under `npm test`.

The consequence is that the Ships-when gate as written is unsatisfiable offline without infrastructure the plan never enumerates. An implementer either invents an undeclared stub-model seam (guessing its shape and owning leaf) or cannot make the test fire, because a real session with no offline stream cannot be driven to mid-stream cancellation under `npm test`.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `7bd65a7` (pi-loom plan: resolve "V4c/V4f defer ERR-8 to ERR-13 live-surface confirmation to downstream owners that never re-assert it", Thu Jun 18 2026) introduced the `ERR-8` delegated live-carrier witness bullet on `V9i`, naming "the real subagent `AgentSession`" and adding `V4c` to Deps but no stream-source seam; `f892019` (pi-loom spec-plan: fix F-1243 — add ERR-8 red-first witness to V9i-T + Ships-when gate, Tue Jun 23 2026) added the paired `V9i-T` bullet and rewrote `V9i`'s **Ships when** to assert the real-session ERR-8 outcome under `npm test`.
**History:** `git log --follow` / `git log -S` over `V9i-subagent-isolation.md` show the bullet did not exist before `7bd65a7`. That commit's purpose was to give `V4c`'s deferred ERR-8/ERR-12 live-surface confirmation a downstream owner on the real `V9i` surface; the latent gap — a real-session witness with no enumerated offline stream — entered there. `f892019` then promoted the latent assumption into an explicit `npm test` Ships-when obligation, which is the point the gap became an unsatisfiable gate. Neither commit added a stub-model/transport seam to Adds or Deps.

## Solution Space

### Recommendation

Treat the real-`AgentSession` mid-stream-cancellation outcome (ERR-8) as a real-host-only behaviour and remove its `npm test` obligation, relocating the live-surface confirmation to the manual real-host smoke gate. Offline `npm test` coverage of ERR-8 continues to run against the `H4a`/`H4c` modeled subagent callee (category (g)) together with `V4c`'s ERR-12 double-harness assertion.

Make these plan edits:

- In `V9i-subagent-isolation.md`, remove the real-session ERR-8 outcome from **Ships when**, and re-cast the `ERR-8` Tests bullet so the live-surface confirmation is asserted at the manual real-host smoke gate (`real-host-smoke-gate.md`) instead of under `npm test`. Mirror the same change in the paired `V9i-T-subagent-isolation.md` `ERR-8` Tests bullet.
- Re-state `V4c`'s ERR-12 prose so its deferred live-surface re-assertion points at the smoke-gate witness rather than a `V9i` `npm test` witness.
- Reconcile the `coverage-matrix.md` `ERR-8 … ERR-12` row's `V9i` co-witness annotation with the relocated assertion, so the matrix no longer claims a live-carrier witness under `npm test`.

Do not edit `subagent.md`: the spec's `model` pass-through contract is read-only here. No spec edits are required. Carry the `coverage-matrix.md` `ERR-8` co-witness annotation and `V4c`'s ERR-12 prose in lockstep with the relocated smoke-gate witness, so the matrix never claims a `V9i` `npm test` live-carrier witness that no longer exists.

## Consequence

**Severity:** blocking

`V9i`'s **Ships when** demands an `npm test` assertion that a real `AgentSession` survives mid-stream cancellation, but no offline source of a cancellable token stream for that real session is declared, so the gate cannot fire correctly as written. A `V9i` implementer cannot pick the leaf up without first deciding where the stream comes from, and two implementers would diverge — one inventing an undeclared stub-model seam, another reading it as a real-host-only assertion that does not belong under `npm test`.
