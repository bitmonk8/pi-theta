## Problem

V18d's **Ships when** packs several sentences of conditional revert-path merge-discipline on top of its single externally observable exit condition: restore the prior pin before merge on a red runtime-evidence run or a confirmed real-host smoke divergence; the bump MUST NOT merge at the candidate pin; re-run V18c's static build-time gates green before merging the revert; and widen the revert to restore any co-edited operand left red. Per `conventions.md` §Leaf format, **Ships when** must be a single concrete, externally observable change that gates `<id>-complete` — an implementer or orchestrator reading this gate cannot tell whether the opening observable or the trailing MUST/MUST-NOT clauses gate completion. Because **Ships when** is not a binding field the way **Adds** Class-1 mechanisms are, the merge-discipline parked there risks not binding the contributor at all. The same revert prose is mirrored into V18d-T's **Ships when**, propagating the placement issue to the paired tests task whose gate is defined as the red-state condition.

## Issue introduction

**Verdict:** present-since-inception (with multi-commit amplification)
**Introducing commits:** `8af3204` (2026-06-11, "resolve 'V18c bundles static gates with runtime-evidence acceptance gate'") created `V18d-version-bump-acceptance.md` with revert-path obligations ("the prior pin is restored before merge — reverting step 4's edit in one commit") already inside Ships-when. Amplified by `eed7975` (2026-06-11, "resolve 'revert path re-asserts gates return green'"), which added the "re-run V18c's static build-time gates … green before the revert is merged" and "the revert MUST be widened" clauses, and by `15eb44c` (2026-06-11, "resolve 'real-host divergence pre-merge gate (posture i)'"), which added the real-host-smoke "MUST NOT merge at the candidate pin" pre-merge obligation.
**History:** `git log --follow` on the leaf file shows the placement defect existed in the file's first revision (V18d was split out of V18c with revert prose already in Ships-when) and grew across three same-day plan-fix commits, each appending another normative sentence to the exit gate rather than placing it in a binding field. The defect is the cumulative shape of those edits, not a single later regression.

## Solution approach

Reduce V18d's **Ships when** to the single externally observable exit condition — the `H4a` end-to-end harness passing against the bumped pin on `main` — and move the conditional revert-path obligations into **Adds**, where V18d's revert path is already the declared subject, citing `version-bump-triggers.md`'s "MUST NOT be merged at the candidate pin" so they bind as Class-1. Apply the same reduction to V18d-T's **Ships when**: keep its tests-task red-state condition and drop the mirrored revert prose.

## Solution constraints

- The spec is read-only for this fix — `version-bump-triggers.md` already carries the merge-blocking MUST; cite it rather than restating it in the spec.
- Do not relocate the revert-path obligations into **Tests** — they are manual merge-discipline, not automatable assertions.
