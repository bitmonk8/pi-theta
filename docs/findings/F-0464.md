## Finding

The pass/fail criterion in `real-host-smoke-gate.md` concludes that "a failure of any one of (a)–(e) … is a **fail** … so a genuine divergence still produces a falsifiable fail that drives the revert / merge-block path below." This asserts the no-false-negative direction unconditionally: it reads as a guarantee that every genuine double-vs-real divergence will surface as a scored fail.

The protocol elsewhere states the run is driven by "a live, non-deterministic LLM." A single non-deterministic run observes only the code paths it actually exercises. A divergence confined to a conditionally-reached branch — `respond_repair`, tool-loop retries, error/transport mapping, or cancellation-timing paths the particular run never enters — produces no fail on that run, even though it is a genuine divergence. The note already caveats the *other* axis (benign live-model variance in turn order, turn count, or which permitted codes fire is "not a fail"), but it carries no caveat for the single-run path-coverage limit, so the no-false-negative claim reads stronger than the mechanism supports.

## Solution Space

**Shape:** single

### Recommendation

In `real-host-smoke-gate.md`, in the Pass/fail-criterion sentence, scope the no-false-negative claim to the paths the run actually exercises. Change the clause "so a genuine divergence still produces a falsifiable fail" to read that a genuine divergence **on a path the single run exercises** produces a falsifiable fail, and add a caveat (alongside the existing benign-variance caveat) that the single non-deterministic run is best-effort against divergences confined to conditionally-reached branches — naming the example branches `respond_repair`, tool-loop retries, error/transport mapping, and cancellation timing. The revert / merge-block consequence for a confirmed fail is unchanged; only the universality of the detection claim is qualified.

## Consequence

**Severity:** advisory

An auditor or contributor reading the criterion may over-trust a green smoke run, treating it as evidence that no double-vs-real divergence exists, when a divergence on a branch the single run did not exercise goes undetected. The gate still functions as a best-effort pre-merge check; only the stated guarantee is broader than what one non-deterministic run delivers.
