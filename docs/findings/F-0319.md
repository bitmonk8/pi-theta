## Finding

`conventions.md` release-time residue-inspection item 7 (*Split-row joint-completeness*) sweeps every `coverage-matrix.md` *Numbered REQ-IDs* row whose closing-leaf cell lists two or more leaves and derives the row's **facet partition** as: where the cell carries parenthetical facet annotations each annotated leaf is one facet; *otherwise the partition is the cell's closing-leaf set minus any co-witness leaf*. The "minus any co-witness leaf" branch presupposes the reviewer can tell which leaf in the cell is a co-witness, but an un-annotated cell carries no such marker — the co-witness/facet distinction for these rows lives only in the matrix's *CIO closure rationale* prose paragraph, which item 7 does not read.

Two such rows exist: `CIO-1 | V16a, V4e, V11f` and `CIO-5 | V16a, H7a`. Per the closure rationale, both are single-facet rows — `V16a` closes the facet (the cross-ceiling precedence decision for CIO-1, the seam-local single-surface arbitration for CIO-5) while `V4e`/`V11f` (CIO-1, temporal load-time placement) and `H7a` (CIO-5, cross-site never-interleaves property) merely co-witness and are dropped. Neither cell marks this. Item 7 hardcodes the answer for `CIO-5` in its own example text, but says nothing about `CIO-1`, whose three-leaf cell defaults under the un-annotated branch to treating all of `V16a`, `V4e`, `V11f` as facets — directly contradicting the closure rationale, which makes `V16a` the sole facet.

The derivation is therefore not mechanically reproducible: a reviewer who reads only item 7 and the cell partitions `CIO-1` into three facets and demands a facet-naming citing test on each of `V16a`, `V4e`, `V11f`; a reviewer who consults the closure-rationale prose partitions it into one facet (`V16a`) with two co-witnesses. The same un-annotated branch also leaves the genuinely-multi-facet rows (e.g. `ERR-17 | V4d, V13d`) reading correctly only because they happen to have no co-witness — the rule cannot distinguish "all leaves are facets" from "one facet plus co-witnesses" without a cell-level marker.

**Verdict:** multi-commit-interaction

**Introducing commits:** `cda161b` — "pi-loom plan: resolve \"split REQ-IDs pass closing gate on one facet citing test\"" (2026-06-17, proximate); `8ef7055` — "pi-loom plan: resolve \"V16a over-claims all CIO bullets provable at stateless seam\"" (2026-06-11, latent precondition)

**History:** The `CIO-1` row has existed since the initial plan build (`c6a664e`, 2026-06-10) and was refined in `2565ddd` (2026-06-10). `8ef7055` (2026-06-11) introduced the co-witness *concept* into `coverage-matrix.md`: `git log -S 'co-witness' -- docs/plan_topics/coverage-matrix.md` shows that commit added the *CIO closure rationale* paragraph describing `V16a` as the facet closer and `V4e`/`V11f`/`H7a` as co-witnesses — but only in narrative prose, never as inline cell annotations. The mechanical defect was then introduced by `cda161b` (2026-06-17), which authored item 7's *Split-row joint-completeness* sweep (`git log -S 'minus any co-witness leaf' -- docs/plan_topics/conventions.md` and `git log -S 'Split-row joint-completeness'` both resolve to `cda161b`): its "otherwise the partition is the cell's closing-leaf set minus any co-witness leaf" branch assumes a cell-level co-witness marker that `8ef7055` had only ever placed in prose, and item 7 hardcoded the `CIO-5` example while leaving `CIO-1` underivable. The gap is the interaction of the two — the prose-only co-witness record (`8ef7055`) and the mechanical rule that presumes it is readable from the cell (`cda161b`).

## Solution Space

**Shape:** single

### Recommendation

Make the split-row facet partition fully derivable from each matrix cell, with no recourse to the *CIO closure rationale* prose, through two coordinated plan edits applied in order.

First, in `conventions.md` item 7 (*Split-row joint-completeness*), restate the facet-partition derivation so co-witness leaves are identified by an explicit cell-level marker rather than by the un-readable "minus any co-witness leaf" default. Define the marker semantics explicitly: a leaf carrying a co-witness marker is dropped from the partition; a leaf carrying a facet annotation is a facet; and every leaf in a multi-leaf cell must carry exactly one classification so no leaf is left ambiguous. Keep the co-witness marker distinct from the parenthetical facet-annotation form so the two read as opposite classifications and a co-witness marker is never swept into the "parenthetical facet annotations → each annotated leaf is one facet" branch. Keep the `DISC-4` / `PIC-17,PIC-18` facet-annotation example and the `CIO-5` co-witness example, but make their reading follow from the stated rule rather than from a hardcoded instance.

Then, in `coverage-matrix.md`'s *Numbered REQ-IDs* table, mark the co-witness leaves inline so each cell names which leaves are dropped, using the `(co-witness)` marker: `CIO-1 | V16a, V4e (co-witness), V11f (co-witness)` and `CIO-5 | V16a, H7a (co-witness)`. Apply the marker only to these two rows unless the item 7 revision surfaces another un-annotated multi-leaf cell that carries co-witnesses.

Edge cases to watch: confirm the genuine all-facets rows (`ERR-17 | V4d, V13d`, `ERR-19 | V4d, V13c`) still partition into all-facets under the revised rule; and verify the change leaves `H5d`'s tokeniser (backtick-delimited leaf IDs only, parenthetical prose dropped) and the `H5a` per-REQ-ID citing-test scan untouched.

## Consequence

**Severity:** correctness

Two reviewers running item 7 over `CIO-1` (and, absent the hardcoded example, `CIO-5`) derive different facet partitions: the un-annotated default counts `V4e`/`V11f`/`H7a` as facets requiring their own facet-naming citing tests, while the closure rationale counts them as droppable co-witnesses. The release-time joint-completeness inspection therefore either raises false residue defects against co-witnesses that need no facet-naming test, or — under the opposite misreading — masks a genuinely-uncovered facet. The check is a manual GOV-15-adjacent reviewer step, not a CI gate, so the divergence ships as inconsistent review outcomes rather than a hard failure.
