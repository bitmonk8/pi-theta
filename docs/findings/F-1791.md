## Finding

In `session-only-degraded-state.md`, the prose justifying the broad tag-transition predicate states: "The complement form `∉ {"quit","reload"}` is the **value-independent seam** pinning the conservative tag write across all three arms." The phrase "value-independent seam" is never defined and is not a glossary term.

The word "seam" is heavily overloaded on this page, which makes the coinage actively confusing rather than merely vague. Elsewhere in the same file "seam" denotes a handler-scoped state variable (`pinnedConstantReadOk`, `capturedEventReason` — "the two handler-scoped seams", "MUST NOT introduce any handler-scoped state seam") and a code site between teardown sub-steps ("interstitial-seam tag-write swallow"). At line 57 it is pressed into a third, unrelated meaning: the predicate's complement *form* itself. A reader cannot tell whether "value-independent seam" names a new state variable, a code site, or the predicate, and "value-independent" is left to inference.

The intended meaning is plain once decoded: the complement `∉ {"quit","reload"}` fires on any reason not confirmed as a tear-down value, so the conservative `"degraded-needs-reload"` tag write happens regardless of *which* specific value `capturedEventReason` holds — including the two `"<unreadable>"`-carrying arms. The normative behaviour is already fully pinned by the predicate table and the "Seam minimality (normative)" clause; this sentence is rationale, so the fix is a wording change only.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `f5e89f4` ("docs: enforce 100KB hard / 30KB recommended size cap on spec set", 2026-06-04)
**History:** The corpus is git-tracked. `session-only-degraded-state.md` was created (`--diff-filter=A`) in `f5e89f4`, and `git log -S'value-independent seam'` across the entire repository returns only that commit — the phrase entered the corpus as net-new prose at file creation, not as text relocated verbatim from a predecessor file. The defect is therefore an authoring artefact of the page-split commit rather than a pre-existing term carried forward.

## Solution Space

**Shape:** single

### Recommendation

Replace the jargon with a plain restatement that names the property directly. Substitute the clause:

> The complement form `∉ {"quit","reload"}` — which fires on any reason not confirmed as a tear-down value, so the conservative tag write happens regardless of the specific value `capturedEventReason` holds — pins the tag write across all three arms, so an operator on a stdio-capturing host … still gets the in-band `"degraded; /reload to recover"` signal on the next slash invocation.

Drop the word "seam" from this sentence entirely; reserve "seam" on this page for the handler-scoped state variables and the interstitial tag-write site so the term keeps a single meaning. Leave the predicate table and the "Seam minimality (normative)" clause untouched.

## Consequence

**Severity:** cosmetic

A reader of the rationale paragraph must guess what "value-independent seam" means, and the local overloading of "seam" invites mis-association with the handler-scoped state variables or the interstitial code site. No implementation behaviour is affected — the tag-transition predicate and the seam-minimality constraint are pinned normatively elsewhere on the page.
