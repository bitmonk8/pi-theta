## Problem

`V4b` and `V4b-T` each carry a Tests bullet asserting that a host-fatal uncatchable condition (the NOCEIL-3 carve-out) emits no diagnostic at all — in particular, no `loom/runtime/internal-error`. Per the spec NOCEIL-3 carve-out (`../spec_topics/hard-ceilings/ceiling-invariants-and-audit.md`), that uncatchable arm (the V8 `OOMErrorCallback` / `abort()` path) terminates the host process; the runtime cannot observe it and no `npm test` assertion can run after it to verify absence of a diagnostic. The bullet adds no coverage — NOCEIL-3 is already closed via `V4b`'s catchable `loom/runtime/internal-error` carve-out — but states an acceptance criterion two implementers will resolve differently: one silently omitting it, one writing a `RangeError` stand-in that actually exercises the *catchable* arm and mislabels it.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commit:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** The plan corpus is git-tracked. `git log -G "uncatchable" -- docs/plan_topics/V4b-runtime-panics.md docs/plan_topics/V4b-T-runtime-panics.md` traces the NOCEIL-3 "emits no diagnostic" carve-out bullet to the plan's inception commit `c6a664e`; at that revision the bullet read "`loom/runtime/internal-error` (NOCEIL-3 uncatchable carve-out): a host-fatal uncatchable condition emits no diagnostic." A later commit `2ff06f2` (2026-06-16, "resolve 'V4b internal-error Tests bullet inverts lead-code-fires pattern'") reworded the bullet to its current form but did not address its realizability. The defect — asserting an observable "no diagnostic" outcome for a path that terminates the test runner — has existed since the bullet was first authored.

## Solution approach

Strike the NOCEIL-3 uncatchable-fatal Tests bullet from both `V4b-runtime-panics.md` and `V4b-T-runtime-panics.md`. Record the uncatchable carve-out instead as a bounded-scope annotation in `V4b`'s **Adds**, noting it is unobservable by the runtime and out of `npm test` scope, mirroring the existing non-`npm-test` deferral pattern used for `V4c`'s ERR-12 real-host carve-out. Cite the spec NOCEIL-3 carve-out (`../spec_topics/hard-ceilings/ceiling-invariants-and-audit.md`) in the annotation.

## Solution constraints

- The spec is read-only for this fix; do not add or restate any NOCEIL-3 obligation spec-side.
- Leave the catchable-arm `loom/runtime/internal-error` Tests bullet and the coverage-matrix NOCEIL-3 → `V4b` closure unchanged.
- Do not substitute a `RangeError`-based test for the struck bullet — that exercises the catchable arm and asserts the wrong behaviour.
