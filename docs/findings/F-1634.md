## Problem

The *Malformed-marker discriminator* paragraph — the definitional rule fixing when an exemption attempt is syntactically invalid via clauses (a)–(g) — lives in `audit-failures.md`, while every other clause of the exemption-marker grammar (*Exemption mechanism*, *Separator tolerance*, *Minimum justification*, *Citation-token grammar*, *Stale-marker discriminator*) lives in `audit-resolution.md`. The seven clauses are exemption-grammar content extending those sibling clauses, yet they sit one shard away from them. Several intra-page directionals consequently resolve cross-shard: *Stale-marker discriminator*, *Multi-line surface placement*, and *Citation-token grammar* in `audit-resolution.md` each cite the *Malformed-marker discriminator* "below" or by clause letter, but the target is on another page.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `e5cd514` (2026-05-09, *pi-loom spec: resolve "T35 SDK capability inventory closed-set negative-direction audit"*) and `f5e89f4` (2026-06-04, *docs: enforce 100KB hard / 30KB recommended size cap on spec set*)
**History:** Commit `e5cd514` introduced the *Malformed-marker discriminator* paragraph in the then-single `docs/spec_topics/pi-integration-contract.md` file, placed adjacent to the *Exemption mechanism* / *Citation-token grammar* / *Stale-marker discriminator* paragraphs (verified at line 854 of that file pre-split, immediately following the *Stale-citation limitation* sub-bullet at line 852). At that point the paragraph's "above" / "below" directionals to its sibling clauses resolved on-page and the placement was internally consistent. Commit `f5e89f4` then split the 576KB single file into the `pi-integration-contract/<child>.md` subdirectory layout; the splitter placed the *Malformed-marker discriminator* paragraph in `audit-failures.md` (chosen because it interacts with the *Failure-surface contract* sentence immediately following it) while routing the *Exemption mechanism* / *Citation-token grammar* / *Stale-marker discriminator* block to `audit-resolution.md`, severing the on-page adjacency. The cross-shard "below" directionals were carried over textually unchanged. Neither commit individually authored a misplaced paragraph; the defect is the interaction — the original on-page co-location was load-bearing, and the split chose its shard boundary one paragraph too high.

## Solution approach

Move the definitional clauses (a)–(g) of the *Malformed-marker discriminator* from `audit-failures.md` into the exemption-grammar cluster on `audit-resolution.md`, beside *Citation-token grammar* and *Stale-marker discriminator*, so the cross-shard "below" / "above" directionals to those clauses resolve on-page. Split the family-(5) routing and dual-emission obligation off so it stays in `audit-failures.md` — that content is failure-shard content and moving the paragraph wholesale would re-create the defect in mirror form. Update the now-cross-shard directionals on both shards, the *Failure-surface contract* parenthetical, and the `version-bump-step2b.md` branch (4) link to reference the clauses' new home.
