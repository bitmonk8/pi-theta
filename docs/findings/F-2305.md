## Problem

`conventions.md` §"Doc updates" instructs each completed leaf to explicitly `git add` the three repository-root artifacts (`README.md`, `CHANGELOG.md`, `notes.md`), since `commitAddPaths` auto-stages only `docs/`. The instruction's correctness depends on an unstated premise: that the orchestrator's per-leaf commit records the whole staged index, not only the `commitAddPaths`-scoped paths. If the orchestrator instead committed pathspec-limited to `docs/`, the explicitly-staged root files would be silently dropped, surfacing only at the after-the-fact residue inspection (checklist item 4), since the rule's own *Enforcement posture* concedes it has no mechanical gate. The plan rests on this commit behaviour without stating it, while describing `commitAddPaths` as producing "scoped commits" — a phrasing a reader can misread as a pathspec-limited commit that would drop the explicit stages.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commits:** `a9f8409` — "pi-loom spec: resolve \"commitAddPaths omits root files the plan conventions mandate updating\"" (Thomas Andersen, 2026-06-06)

That commit resolved a prior finding by adding the explicit-`git add` instruction together with the "`commitAddPaths` … auto-stages only `docs/`" rationale. The unstated commit-semantic dependency entered the corpus as a side-effect of that resolution and has not been reworded since (no later commit touches those tokens). The gap is therefore present since `a9f8409`, not introduced by a multi-commit interaction.

## Solution approach

In §"Doc updates", extend the staging clause to state the relied-upon commit semantic where the rule depends on it: that the orchestrator's per-leaf commit records the full staged index (every explicitly `git add`-ed path), not only the `commitAddPaths`-scoped `docs/` paths — `commitAddPaths` governs what is auto-staged, not what is committed. Optionally note in the *Enforcement posture* sentence that, absent this semantic, the residue inspection (checklist item 4) is the sole backstop.

## Solution constraints

- Keep the three artifacts on their current repository-root paths and change no leaf; this fix states an assumption in `conventions.md`, it does not alter the staging mechanism.
- Out of scope: relocating the residue-inspection checklist is owned by F-2348; the *Enforcement posture* cross-reference to "checklist item 4" follows that relocation if it lands.
