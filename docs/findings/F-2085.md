## Problem

*Type display* (under *System-prompt structure (normative)* item 4) governs how each Parameters-block field's `<type>` is rendered and pins a closed eight-row reference table marked "conforming implementations MUST reproduce these exactly." The Loom type grammar (`grammar.md#type-grammar`) admits compound forms that are legal in `params:` positions — inline anonymous object types, nested generics, literal types, and unions over them — whose interior whitespace the table never pins: brace-interior spacing, post-comma spacing, trailing-comma emission, and spacing inside nested generic brackets are all left free. The sibling token-order rule constrains whitespace *between* `<wire-name>`, `(<type>)`, and `<requirement>` but says nothing about whitespace *inside* the `<type>` rendering itself. Two conforming renderers therefore emit different `<type>` bytes for the same compound-typed loom, breaking the binder's byte-identical-input contract (`binder-model-and-context.md#why-the-transcript-bytes-are-pinned`).

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** 5a4ec51 — pi-loom spec: resolve "Binder system prompt information content lacks verifiable field set" (2026-05-06, Thomas Andersen)
**History:** Commit 5a4ec51 introduced both item 4's "rendered per *Type display* below" rule and the *Type display* table with its "conforming implementations MUST reproduce these exactly" clause into the predecessor `spec_topics/binder.md`. The table enumerated only the same closed reference set then in place (`string`, `int`, `boolean`, enum, named schema, `array<int>`, `string | null`, `Cat | Dog`) with no general canonical-whitespace rule for compound types, so the gap was present at the rule's introduction. The later token-order/indentation commit (4188412, 2026-05-07) pinned whitespace *between* the three tokens but not *within* `<type>`, and the size-cap split (f5e89f4, 2026-06-04) moved the rule verbatim into `binder/binder-bypass-and-envelope.md`; neither closed the gap.

## Solution approach

Rewrite *Type display* in `binder/binder-bypass-and-envelope.md` from a closed reference table into a normative canonical-rendering rule that pins the whitespace the grammar leaves free for every compound type form the Loom grammar admits, recursing into element, field, and union-arm types. Add worked reference rows covering at least an inline-object and a nested-generic rendering.

## Solution constraints

- Out of scope: *Default-literal rendering* (the `default=<literal>` rule in the same item 4), owned by F-2089.
