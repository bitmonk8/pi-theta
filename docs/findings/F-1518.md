## Finding

The version-bump step-2 editorial checklist defines a shared preparatory step — the **Baseline acquisition** block (anchor `bump-baseline-acquisition`) — that materialises the loom 1.0 Pi-SDK pin's `dist/*.js` tree via `npm pack` and extraction. That extracted `package/dist/` tree is the canonical "from" side for every `dist/*.js` diff on the page. The block is the explicit upstream dependency of four checklist items: (e) (per-session slash-dispatch serialisation), (r) (`convertToLlm` conversion), (aj) (`SessionManager.inMemory` transcript-non-retention), and (am) (`createAgentSession` tools-allowlist suppression). Each of those items links to `#bump-baseline-acquisition` for its baseline.

Despite serving four items, the block is physically nested as a sub-paragraph inside item (e)'s body, between item (e)'s heading and item (f). Items (r), (aj), and (am) appear much later in the list (item (am) is near the end of a list running to (ap)), so a contributor working those items in sequence encounters a forward/backward anchor jump into the middle of an unrelated, very large checklist item to find a procedure that is logically a precondition for several items, not a detail of (e). The cross-references are working markdown anchors, so navigation is link-assisted; the defect is structural — a shared preparatory step is sited under one specific consumer's heading rather than as a peer preparatory sub-step.

The fix is to hoist the Baseline acquisition block to a stand-alone preparatory sub-step ahead of the checklist items (its own heading), preserving the existing `bump-baseline-acquisition` anchor so the (e)/(r)/(aj)/(am) citations keep resolving.

## Solution Space

**Shape:** single

### Recommendation

Hoist the Baseline acquisition block out of item (e)'s body to a stand-alone preparatory sub-step placed before checklist item (a) (or immediately after the step-2 preamble), under its own descriptive heading (e.g. **Baseline acquisition (preparatory step)**). Carry the existing `<a id="bump-baseline-acquisition"></a>` anchor onto the relocated heading verbatim so the four citing items (e), (r), (aj), (am) continue to resolve without edit. Leave item (e)'s own `dist/*.js` diff prose (the `AgentSession.prototype.prompt` body diff and the `session.prompt(...)` call-graph enumeration) in place under item (e); only the shared materialisation paragraph moves.

Edge cases to watch:
- The block's text references "every `dist/*.js` diff on this page" and "before running the `dist/*.js` diffs items (e), (r), and (aj) prescribe" — update that enumeration to also name (am) (currently understated) when relocating, so the prose matches the four actual consumers.
- If the "Outcome recording conventions" block is also relocated to the front of the section (a sibling placement finding), order the two preparatory blocks deterministically and keep their anchors intact.

## Consequence

**Severity:** cosmetic

A shared baseline-acquisition prerequisite is sited under item (e)'s heading rather than as a peer preparatory step, so a contributor reading items (r)/(aj)/(am) sequentially must follow an anchor back into item (e)'s body to find it. The cross-references resolve via working anchors, so no procedural step is lost and no implementation behaviour diverges; the cost is purely document organisation and sequential readability.
