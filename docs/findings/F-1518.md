## Problem

The version-bump step-2 editorial checklist's Baseline acquisition block (anchor `bump-baseline-acquisition`) is a shared preparatory step that materialises the loom 1.0 Pi-SDK pin's `dist/*.js` tree — the canonical "from" side for every `dist/*.js` diff on the page — and is the explicit upstream dependency of four checklist items: (e), (r), (aj), and (am). Despite serving four items, it is physically nested as a sub-paragraph inside item (e)'s body, between item (e)'s heading and item (f). Items (r), (aj), and (am) appear much later in the list, so a contributor working those items sequentially must follow an anchor jump back into the middle of an unrelated, very large checklist item to find a procedure that is a precondition for several items rather than a detail of (e). The cross-references resolve via working anchors, so no procedural step is lost; the defect is structural placement.

## Issue introduction

**Verdict:** present-since-inception (of the block)
**Introducing commit:** `ccc6e38` (2026-06-07) — *pi-loom spec: resolve "Item (e)/(r) dist/*.js baseline source unspecified"*
**History:** `git log -S 'bump-baseline-acquisition' -- docs/spec_topics/pi-integration-contract/version-bump-step2.md` and `git show ccc6e38` confirm the Baseline acquisition block did not exist before `ccc6e38`; that commit created it and placed it nested inside item (e)'s body, already cross-referenced from the remote item (r). The shared-prerequisite-buried-under-one-consumer placement was therefore present at the block's inception. Two later commits widened the consuming set but did not introduce the placement defect: `9ced9f1` (2026-06-08, subagent transcript-discard) added item (aj)'s reference, and `077afb5` (2026-06-20, F-0806) added item (am)'s reference — these compounded the back-tracking surface from two consumers to four but left the structural siting unchanged.

## Solution approach

Move the Baseline acquisition block out of item (e)'s body to a stand-alone preparatory sub-step ahead of the checklist items, under its own heading, carrying the existing `bump-baseline-acquisition` anchor verbatim so the (e)/(r)/(aj)/(am) citations keep resolving. Leave item (e)'s own `dist/*.js` diff prose (the `AgentSession.prototype.prompt` body diff and the `session.prompt(...)` call-graph enumeration) in place under item (e). Correct the relocated block's consumer enumeration, which names only items (e), (r), and (aj), to also name item (am).

## Solution constraints

- Out of scope: item (e)'s own `dist/*.js` diff prose stays under item (e); only the shared materialisation paragraph moves.
