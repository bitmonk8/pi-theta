## Problem

`runtime-value-model.md` §Wire-name translation makes both the *Inbound* and *Outbound* directions normative MUSTs, and `V2e`'s **Adds** claims the whole boundary including outbound wire-name JSON. But the single **Tests** bullet in both `V2e-wire-name-translation.md` and `V2e-T-wire-name-translation.md` asserts only the inbound direction (rebuild + defaults-bypass); nothing asserts the outbound walk. Because the outbound path emits no diagnostic and the RVM area is code-keyed in `coverage-matrix.md`, no `DIAG-2` reverse-gate or numbered-REQ-ID gate binds it. `V2e` can be tagged complete with outbound translation untested.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** The wire-name translation Tests bullet was authored in `c6a664e` on the then-combined `V2c` leaf ("Runtime value model, equality, and wire-name translation"), covering only the inbound rebuild + defaults-bypass case while that leaf's Adds already listed outbound JSON and enum-tag reattach — so the assertion gap dates to the bullet's first authoring. Commit `d4f6e5c` (2026-06-17, "split V2c value model from wire-name translation boundary") moved the bullet verbatim into the newly created `V2e` leaf without broadening it, preserving the gap rather than introducing it.

## Solution approach

Add a Tests entry asserting the *Outbound* direction (loom-side value → wire-named JSON before AJV validation) to `V2e-T-wire-name-translation.md`, and mirror the same entry into the Tests of `V2e-wire-name-translation.md`, keeping the existing inbound bullet. Cite `runtime-value-model.md` §Wire-name translation as the source. Update `V2e`'s **Ships when** so its "wire-translation vectors above" reference resolves to both the inbound and the outbound assertion.

## Solution constraints

- The spec is read-only for this fix — the outbound MUST already exists in `runtime-value-model.md`; this is an internal plan-coverage fix.
