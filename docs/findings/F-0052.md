## Problem

H7a's terminal integration-acceptance gate asserts that the run's appended turns "match — in order and count — the committed golden transcript," and the Ships-when sentence repeats the same as "matching the committed golden transcript (order and count)". Both formulations enumerate only turn count and sequence; neither asserts that each turn's content equals the golden. A run that produces the right number of turns in the right order but with drifted content (wrong text, wrong tool arguments, wrong rendered envelope) therefore passes the gate, defeating the cross-slice regression backstop the leaf's `Adds.` framing depends on.

## Issue introduction

**Verdict:** introduced by a specific later commit (not present at inception)
**Introducing commit:** `39593c3` — "pi-loom plan: resolve 'H7a Tests bullets 1-2 lack a defined referent'" (2026-06-11)
**History:** The plan corpus is git-tracked. `H7a-integration-acceptance.md` was created in `e7e51cc` ("Plan has no terminal end-to-end integration-acceptance leaf") with the first Tests bullet reading only "the run produces the expected appended turns" — no golden transcript, no order/count/content dimensions. Commit `39593c3` resolved a separate finding that those bullets lacked a defined referent; it introduced the committed-golden-transcript referent but phrased the match as "in order and count" in both the Tests bullet and the Ships-when sentence (`git log -S "in order and count"` and `git log -S "order and count"` on the file both return only `39593c3`). The content-equality omission therefore entered as a side effect of that referent-defining fix, not at the leaf's inception.

## Solution approach

In `docs/plan_topics/H7a-integration-acceptance.md`, rewrite the golden-transcript assertion in the Tests first bullet so the match includes per-turn content equality, not merely turn order and count, removing the narrowing carried by the "in order and count" qualifier. Apply the same change to the Ships-when sentence so its "(order and count)" parenthetical no longer restricts the match to structural dimensions, keeping the two surfaces consistent.
