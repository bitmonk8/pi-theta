## Problem

`V3a-T`'s Tests list carries four stdlib-surface bullets (`String.replace` reference vectors / `concat`, `string` members, `array<T>` members, `object` members) that are byte-identical to the four Tests bullets of `V3e-T`. The paired implementation leaf `V3a` carries only three core-evaluator Tests bullets and its `Adds.` explicitly defers the stdlib members to `V3e`. Because `V3a` does not implement the stdlib surface, it cannot turn those four bullets green at its exit gate (`conventions.md` ┬¦"Per-phase TDD ritual"), so `V3a`'s "Ships when" is unsatisfiable and the stdlib surface is double-owned between `V3a-T` and `V3e-T`.

## Issue introduction

**Verdict:** introduced (multi-commit interaction culminating in an incomplete leaf split)
**Introducing commits:** `cb8737a` (`pi-loom plan: resolve "Systemic leaf over-bundling across the plan corpus"`, 2026-06-12); contributing prior state from `20b4689` (`pi-loom plan: resolve "Stdlib members beyond replace/concat have no named assertion"`, 2026-06-11)
**History:** `20b4689` added the three `string`/`array<T>`/`object` stdlib Tests bullets to **both** `V3a` and `V3a-T` (the `String.replace`/`concat` bullet predated it in both), which was internally consistent at that point ŌĆö `V3e`/`V3e-T` did not yet exist. `cb8737a` then split the stdlib surface into the new `V3e`/`V3e-T` leaves, removed the four stdlib bullets from the `V3a` implementation leaf, and rewrote `V3a`'s `Adds.`/`Ships when` to defer stdlib to `V3e` ŌĆö but left the paired `V3a-T` tests leaf untouched, still carrying the four bullets. The defect is the incomplete split: the impl leaf was updated and its paired test leaf was not, leaving `V3a-T` asserting a surface `V3a` no longer owns.

## Solution approach

Delete the four stdlib Tests bullets from `V3a-T` (`docs/plan_topics/V3a-T-expression-evaluator.md`), keeping its three core-evaluator bullets that match `V3a`'s three Tests bullets 1:1. The struck bullets remain owned by `V3e-T`, so EXPR's stdlib coverage is preserved and `V3a` can turn its `V3a-T` suite green at its exit gate.

## Solution constraints

- Out of scope: `V3e-T` is the sole `-T` owner of the four stdlib bullets ŌĆö do not remove or weaken them there.
