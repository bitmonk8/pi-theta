## Finding

The `host-interfaces-core.md — *Tool execution from loom code*` Tests bullet in both `V14a-tool-calls.md` and `V14a-T-tool-calls.md` carries the parenthetical `(un-anchored; GOV-22 residue)` immediately after the spec link. This is coverage-matrix audit metadata: it records *why* the cited spec section has no numbered REQ-ID anchor (a known GOV-22 residue), a fact that belongs to the matrix's machine-read code-keyed-area rows, not to an implementer-facing acceptance-criterion bullet.

The obligation after the colon — the five accepted-path `execute()` envelope-lowering mechanics — is complete and unambiguous on its own. The bullet's spec link already resolves to a section anchor (`#tool-execution-from-loom-code`), so the parenthetical constrains nothing the implementer must do; it is internal governance history embedded in a normative Tests bullet. The same parenthetical appears verbatim in both paired files.
