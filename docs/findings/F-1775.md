## Problem

Sub-step 1's unset-drain-state-tag fallback clause (anchor `#substep-1-unset-tag-fallback`) carries a normative requirement that the slash-handler reader treat an unset tag as the `"shutting-down"` arm and `emit the same legacy template` ŌĆö but the noun phrase `legacy template` is never defined. No prior sentence in the paragraph or in the cross-referenced `drain-state-contract.md` introduces it; the only other appearance of `legacy` is the unrelated `legacy single-template surface` phrase, which characterises external operator tooling rather than naming a runtime-emitted template. The intended referent (the `"loom /<name>: extension shutting down"` content string fixed one clause earlier) is reachable only by inference, so a MUST whose payload string depends on the reader recovering it from context invites divergent implementations.

**Verdict:** later-commit-introduced
**Introducing commits:** `52b9101a` (2026-05-11, "pi-loom spec: resolve \"T24 ŌĆö Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state\"")
**History:** `git log -S "emit the same legacy template" --all -- docs/` returns exactly two commits ŌĆö `52b9101a` (initial introduction, into `docs/spec_topics/pi-integration-contract.md`) and `f5e89f4f` (2026-06-04, mechanical re-shard splitting the monolithic file into `pi-integration-contract/*.md`, which carried the line verbatim into `patch-skew-degradation.md`). `52b9101a` is the substantive edit: it introduced the second `"degraded-needs-reload"` arm and, as part of distinguishing the two arms, labelled the pre-existing `"shutting down"` template `legacy` ŌĆö both in the `legacy single-template surface` phrase that characterises external operator tooling and in the load-bearing `emit the same legacy template` MUST. The MUST was written in a paragraph where the preceding sentence had just bound the `"loom /<name>: extension shutting down"` literal, so the local context made the referent obvious to the author at the time; the term was never given a stand-alone definition, and no later edit has audited the MUST against the lookup `legacy template ŌåÆ ?`.

## Solution approach

Rewrite the `#substep-1-unset-tag-fallback` clause to name the literal `"loom /<name>: extension shutting down"` content string in place of the undefined `legacy template` noun phrase. Reword the downstream `legacy single-template surface` phrase (in the same paragraph's operator-tooling sentence), since once `legacy template` is gone the bare `legacy` there is left without an antecedent.
