## Finding

Sub-step 1 of the `session_shutdown` teardown contains a normative requirement on the slash-handler reader for the *unset drain-state tag* fallback — `the slash-handler reader MUST also treat an *unset* drain-state tag … as the "shutting-down" arm and emit the same legacy template` — but the noun phrase `legacy template` is never defined. No prior sentence in the same paragraph (or in the cross-referenced `drain-state-contract.md`) introduces or anchors a definition for `legacy template`; the only earlier appearance is the unrelated phrase `legacy single-template surface`, which characterises *external operator tooling* rather than naming a template the runtime emits.

The intended referent is inferable — it is the `"shutting-down"`-arm `content` string `"loom /<name>: extension shutting down"` that the same paragraph fixes one sentence earlier — but a MUST whose payload string is reachable only by inference invites two divergent implementations: one that emits the literal `"loom /<name>: extension shutting down"` and one that emits some other "legacy" candidate (for instance the binder-cancellation note this arm is described as "shared with"). The drain-state contract's arm-(b) clause and the existing acceptance fixtures both expect the exact `"loom /<name>: extension shutting down"` string, so the inferred reading is the right one — but the MUST itself should not depend on the reader recovering it from context.

## Issue introduction

**Verdict:** later-commit-introduced
**Introducing commits:** `52b9101a` (2026-05-11, "pi-loom spec: resolve \"T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state\"")
**History:** `git log -S "emit the same legacy template" --all -- docs/` returns exactly two commits — `52b9101a` (initial introduction, into `docs/spec_topics/pi-integration-contract.md`) and `f5e89f4f` (2026-06-04, mechanical re-shard splitting the monolithic file into `pi-integration-contract/*.md`, which carried the line verbatim into `patch-skew-degradation.md`). `52b9101a` is the substantive edit: it introduced the second `"degraded-needs-reload"` arm and, as part of distinguishing the two arms, labelled the pre-existing `"shutting down"` template `legacy` — both in the `legacy single-template surface` phrase that characterises external operator tooling and in the load-bearing `emit the same legacy template` MUST. The MUST was written in a paragraph where the preceding sentence had just bound the `"loom /<name>: extension shutting down"` literal, so the local context made the referent obvious to the author at the time; the term was never given a stand-alone definition, and no later edit has audited the MUST against the lookup `legacy template → ?`.

## Solution Space

**Shape:** single

### Recommendation

Replace `emit the same legacy template` with `emit the same `"loom /<name>: extension shutting down"` cancelled-binder-style content` in the unset-tag fallback clause, so the MUST quotes the exact template string. The replacement keeps the existing "shared with binder cancellation" framing the next clause carries (the rationale for *why* this template is the safe fallback is unchanged) and aligns the wording with the `drain-state-contract.md` arm-(b) clause, which already cites the literal verbatim. Also delete or rephrase the downstream `legacy single-template surface` phrase (in the same paragraph's "operator tooling that grepped …" sentence) — once `legacy template` is gone, the only remaining occurrence of `legacy` is that one phrase, where it now lacks any antecedent; rewording it to `pre-two-arm single-template surface` (or simply `single-template surface`) closes the loop without expanding scope.

Implementer-visible edge cases: the substitution is one paragraph in one file with one anchor (`#substep-1-unset-tag-fallback`); no cross-reference repointing is required. Check that no other shard greps for the bare token `legacy template` (it does not at the time of this finding) before landing the edit.

## Consequence

**Severity:** advisory

Two reasonable implementers will land on the same `"loom /<name>: extension shutting down"` string because every other site that pins this arm's content uses that literal verbatim, but the MUST does not enforce that on its own terms — a careful reader following only the unset-tag fallback sentence has to backtrack one clause to find what `legacy template` refers to. No runtime behaviour is at risk; the cost is reader effort and a small surface for misreading.
