## Finding

The silent-precision-loss overflow rule invokes IEEE-754 double precision but pins no output vector, so a tester cannot distinguish correctly-lossy from incorrectly-wrong-but-non-panicking.

**Suggested fix:** Add one normative overflow reference vector.
