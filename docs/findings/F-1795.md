## Problem

The `.warp` file rules' `May call invoke(...)` bullet in `imports.md` describes an `invoke(...)` from inside a `.warp` `fn` body as executing "exactly like a `@`...`` query inside a warp function." That comparison is intended only to fix the conversation-routing rule, but its surface reading equates the invoke with a query on every axis — including queries' non-participation in `INV-4` invoke-depth accounting. An implementer who reads the equivalence as load-bearing can omit the per-chain depth-frame increment for invokes originating in `.warp` bodies, leaving `INV-4`'s ceiling enforced for `.loom`-originating invokes but silently bypassed for `.warp`-originating ones.

## Issue introduction

**Verdict:** present-since-the-introducing-feature-commit
**Introducing commit:** `8eea523` (2026-05-04, *spec: fill implementation-blocking gaps*) — the same commit that first allowed `invoke` from inside `.warp` `fn` bodies; its commit message lists "Allow invoke from .warp function bodies" as a bullet, and its diff against the prior `spec.md` adds the bullet verbatim with the "exactly like a `@`...`` query inside a warp function" comparison already in place.
**History:** The "exactly like" wording is the only wording the rule has ever had. The May 4 split (`fecb504`, *Split spec.md and plan.md into per-topic / per-phase files*) moved the bullet into `docs/spec_topics/imports.md` byte-for-byte. The only subsequent edit (`be0dba6`, 2026-06-25, *pi-loom spec-plan: fix F-1280 — forward-ref INV-4 depth-frame rule from functions.md + imports.md*) appended a *cross-file `.warp fn` call* depth-frame sentence to the same bullet; it left the "exactly like" clause untouched and only addressed the orthogonal cross-file `fn` arm, not the direct `invoke` arm whose semantics this finding flags. `git log -L` on the bullet (run via `git log -L /May call \`invoke/,+1:docs/spec_topics/imports.md`) confirms two commits: the June 25 cross-file-frame append and, before the topic split, the original May 4 introduction (`git log -S "exactly like a \`@\`...\`\`" --all` returns exactly `8eea523` and `fecb504`). The defect entered with the feature; no prior shape exists to revert to.

## Solution approach

Rewrite the trailing comparison in the `May call invoke(...)` bullet of `imports.md` so the conversation-routing rule and the `INV-4` invoke-depth applicability are stated as separate clauses instead of equating the invoke with a query. State that `INV-4`'s depth accounting applies to invokes originating in `.warp` `fn` bodies on the same terms as invokes from `.loom` bodies, cross-linking `INV-4` by anchor (`./invocation.md#inv-4`).

## Solution constraints

- Out of scope: `INV-4`'s wording in `invocation.md` — do not modify it; this edit is local to the `imports.md` bullet.
- Preserve the existing trailing sentence about cross-file `.warp` `fn` calls being countable frames — it covers the orthogonal `fn`-call arm.
