## Problem

In `docs/plan_topics/real-host-smoke-gate.md`, axis (ii) of the four session-double fidelity-contract axes is worded two different ways. Scored pass/fail criterion (d) spells it out as the per-response invariant "each streamed assistant response is appended as exactly one prompt-mode turn (a per-response invariant, independent of total turn count)", while trigger (2)'s enumeration of fidelity-axis-touching diffs reduces it to the bare label "single-turn prompt-mode append semantics". The bare label admits a session-total reading that contradicts both the per-response criterion (d) and the file's closing turn-count carve-out. Because trigger (2) is the recognition heuristic a contributor uses to decide whether their diff obliges a manual smoke run, the two wordings let the same diff be judged in- or out-of-scope for the gate depending on which is consulted.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** dd875e1 — pi-loom plan: resolve "Axis (ii) single-turn prompt-mode append semantics ambiguity" (2026-06-12, Thomas Andersen)
**History:** Commit 3911733 (2026-06-11) reframed the smoke pass criterion for a non-deterministic LLM, introducing the (a)–(e) list and the turn-count carve-out; at that point criterion (d) and the trigger enumeration both used the same bare "single-turn prompt-mode append semantics" phrase and so were mutually consistent. Commit dd875e1 (2026-06-12) disambiguated criterion (d) to the per-response invariant but consciously left trigger (2)'s axis (ii) as the bare label (its own fix note called the trigger-(2) occurrence an "axis label, not the scored predicate" that "may stay as short"), introducing the present divergence. Commit fd22a3d (2026-06-17) moved the content verbatim from `H4a-factory-shell-and-harness.md` into the new `real-host-smoke-gate.md`, carrying the divergence forward unchanged.

## Solution approach

Rewrite axis (ii) in trigger (2)'s enumeration in `real-host-smoke-gate.md` so its label asserts the per-response single-turn append invariant, semantically agreeing with criterion (d) and the trailing turn-count carve-out rather than admitting a session-total reading.
