## Finding

The `V9g` leaf heading reads `# `V9g` — Session-shutdown teardown and emission isolation` (and the plan.md §V9 link text repeats it). The leaf's load-bearing second spec source is `diagnostic-emission-isolation.md`, whose page title and normative rule are both named **Diagnostic-emission isolation**. The heading drops the `Diagnostic-` qualifier.

The abbreviation is ambiguous within the leaf's own vocabulary: the `Adds.`/`Tests.`/`Ships when.` fields use **per-step isolation** (the `patch-skew-degradation.md` §Per-step isolation rule) as a distinct concept from the diagnostic-emission swallow. A reader scanning the heading's bare "emission isolation" cannot tell which of the two isolation flavours the leaf's title is pointing at, where the spec keeps them as separately-named rules ("Per-step isolation" vs "Diagnostic-emission isolation").

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (pi-loom plan: build/update plan for spec.md + review, 2026-06-10)
**History:** `git log -L'1,1:docs/plan_topics/V9g-session-shutdown.md'` shows the heading line `# `V9g` — Session-shutdown teardown and emission isolation` was authored verbatim in the leaf file's first commit `c6a664e`; `git log -S` on the plan.md link text shows the same phrasing entered `docs/plan.md` in that same initial plan-build commit. The abbreviated form has never differed from its inception — no later edit introduced or widened the divergence.

## Solution Space

**Shape:** single

### Recommendation

Expand both occurrences of the heading to name the diagnostic-emission flavour:

- In `docs/plan_topics/V9g-session-shutdown.md`, change the H1 heading text to `# `V9g` — Session-shutdown teardown and diagnostic-emission isolation`.
- In `docs/plan.md` (§Vertical slices, V9 slice), change the `V9g` link text to `V9g` — Session-shutdown teardown and diagnostic-emission isolation` so the index entry matches the leaf title.

This re-uses the spec page's own term verbatim, so the heading resolves unambiguously to the diagnostic-emission rule and is distinct from the leaf's separate "per-step isolation" usage.

## Consequence

**Severity:** cosmetic

The leaf still cites `diagnostic-emission-isolation.md` explicitly under its `Spec.` field, so no implementer is blocked or misdirected on behaviour; the cost is a reader momentarily unsure whether the title's "emission isolation" denotes per-step isolation or diagnostic-emission isolation. Pure wording.
