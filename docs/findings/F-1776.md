## Problem

The normative lexical restrictions on `@`...`` query templates — that `@` has only one lexical role and never appears inside `${...}`, and that there is no bash-style argument-slice sugar (`${@:N}`, `$1`, `$@`, `$ARGUMENTS`) — live only in `docs/spec_topics/frontmatter/frontmatter-fields-b-and-templates.md` §"Template Interpolation". The Query topic (`query-forms.md` + `query-escapes-stringification.md`, which carries QRY-17 and QRY-18) states neither rule and has no forward link to the frontmatter page. An implementer reading the Query subtree for the full `@`-template interpolation contract cannot tell which lexical forms are admissible inside `${...}`, and a lexer author building from the Query topic alone may treat `@` as significant inside the slot or admit bash-style sugar because nothing there forbids it.

**History:** `git log -S 'no bash-style'` shows the phrase first appearing in `83802db`, which deleted the "Argument Styles" and "Template Interpolation Disambiguation" sections that previously defined the bash sugar and replaced them, in-place, with the current "Template Interpolation" paragraph carrying the prohibition. `git show 83802db -- spec.md` confirms the prohibition was written into what was then a frontmatter-region section, not into the Query section. `fecb504` later split the monolithic `spec.md` into per-topic files and carried the prohibition into `frontmatter/frontmatter-fields-b-and-templates.md` verbatim. The placement defect is therefore not a regression — it has been in the corpus from the moment the prohibition was first written, inheriting the location of the deleted bash-sugar text rather than moving to its post-rewrite natural home in the Query topic.

## Solution approach

Move the two lexical rules — `@` has no role inside `${...}`, and there is no bash-style argument-slice sugar — into `docs/spec_topics/query/query-escapes-stringification.md` adjacent to QRY-17, which already owns the `@`-template body lexical rules. Add a forward-link from `frontmatter-fields-b-and-templates.md` §"Template Interpolation" to the relocated rules.
