## Problem

In `docs/spec_topics/cancellation.md`, the late-settlement discard is split into per-REQ-ID clauses CNCL-1/CNCL-2/CNCL-3 (anchor `cncl-3`), a split that invites citing one clause in isolation. CNCL-2 reads "MUST NOT emit a second `Err` **for the same invocation**", but CNCL-3 reads only "MUST NOT emit a second `RuntimeEvent`" — the scope qualifier is dropped. Cited alone, CNCL-3 reads as a session-wide prohibition rather than per-invocation suppression of the late settlement, so both a session-wide and a per-invocation implementation are literally compliant. The intended scope is unambiguous elsewhere in the same section (the swallowing-handler paragraph and the `V17a` test both carry "for the same invocation" / "per invocation"); the clause text is the only place the qualifier is missing.

## Issue introduction

**Verdict:** introduced-by-commit (single commit; defect present since the rule's first authoring)
**Introducing commit:** `11ef3b9c26ee5dcc9a1ba1e5970af9912f2e1c07` — "pi-loom spec: resolve \"Tool failure modes beyond throw / isError: true unspecified\"" (2026-05-07)
**History:** `git log -G 'second .Err. for the same invocation'` and `git show 11ef3b9 -- spec_topics/cancellation.md` show this commit first added the three-obligation discard sentence to `cancellation.md`. From that first appearance the text read "...MUST NOT emit a second `Err` **for the same invocation**, and MUST NOT emit a second `RuntimeEvent`." — clause (b) carried the qualifier, clause (c) did not. The phrase was absent from the parent tree (`git grep` against `11ef3b9^`), confirming this commit is the origin. The asymmetry was then carried through unchanged by `d9f42d0` (2026-05-08, "split four cancellation MUSTs into per-obligation paragraphs") and `482098b` (2026-06-04, "split into three CNCL-N sub-anchors"); neither refactor corrected it. The defect is therefore present since the rule was introduced, not injected by a later edit.

## Solution approach

Append the scope qualifier "for the same invocation" to the CNCL-3 clause (anchor `cncl-3`) in `docs/spec_topics/cancellation.md`, so its scope matches the qualifier CNCL-2 already carries and the swallowing-handler paragraph's existing phrasing in the same section. The defect is the missing wording, so the specific qualifier is the fix.

## Solution constraints

- The plan is read-only for this fix: `V17a` / `V17a-T` already assert per-invocation scope and require no edit.
