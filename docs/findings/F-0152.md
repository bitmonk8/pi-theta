## Problem

`V9c`'s Adds names the `stopReason:"error"` probe as a first-class deliverable of the prompt-mode driver, and `conversation-drive.md` §Error detection (`#prompt-mode-error-detection`) pins the full transport-failure surface: the post-`waitForIdle()` probe of the trailing `assistant` message mapping `stopReason:"error"` to a synthesised `transport` `QueryError`, the empty-`errorMessage` fallback to `"provider transport failure"`, the synchronous-`sendUserMessage`-throw secondary mapping, and the cancellation short-circuit's precedence over both the error probe and `Ok(string)` extraction. `V9c-T`'s Tests assert only `PIC-2`, `PIC-17`, `PIC-18`, and the happy-path trailing-turn `Ok(string)` extraction — no bullet witnesses any part of the transport-failure surface, the primary failure path of every driven prompt-mode turn. Compounding the gap, the §Error-detection MUSTs have no row in `coverage-matrix.md`, so the H5a closing gate has no mapped obligation to reconcile `V9c`'s closure against.

**Verdict:** present-since-inception

Later commits added but never closed the gap: `2ce483b` added `PIC-2` and the trailing-turn `Ok(string)` extraction bullet, and `3462575` / `ecedd5f` adjusted V17a Deps. The Adds-names-it-but-Tests-omit-it defect is therefore original to the leaf pair, not introduced by a later edit.

## Solution approach

Add a code-keyed row to `coverage-matrix.md` citing `conversation-drive.md` §`prompt-mode-error-detection` with `V9c` as the closing leaf, covering the four transport-failure-detection MUSTs. Add failing-test bullets to `V9c-T`'s Tests asserting the `stopReason:"error"` → synthesised `transport` `QueryError` mapping, the empty-`errorMessage` → `"provider transport failure"` fallback, the synchronous-`sendUserMessage`-throw secondary mapping, and the cancellation short-circuit firing — and taking precedence over the error probe and `Ok(string)` extraction — even when `waitForIdle()` resolved cleanly with no error written to session state. Mirror the same four assertions into `V9c`'s Tests and extend `V9c`'s Ships when to drive a `stopReason:"error"` turn and observe the synthesised `transport` error.

## Solution constraints

- The §Error-detection behaviours are already pinned in `conversation-drive.md`; the spec is read-only for this fix (no spec edits, no new spec anchors).
- The `provider` value in the synthesised `transport` error is owned by V9j's provider-error-mapping surface — reference it, do not re-derive it here.
- Keep the new coverage row distinct from the existing §`untyped-query-ok-extraction` row (which covers the separate `Ok(string)` extraction obligation); do not fold into or duplicate it.
