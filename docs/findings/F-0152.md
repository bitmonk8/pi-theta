## Finding

`V9c`'s **Adds** names the `stopReason:"error"` probe as a first-class deliverable of the prompt-mode driver, and `conversation-drive.md` §Error detection (`#prompt-mode-error-detection`) pins that surface in full: because `pi.sendUserMessage` returns `void` and `waitForIdle()` never rejects, the **primary** transport-failure surface is the post-`waitForIdle()` probe of the driven turn's trailing `assistant` message. A `stopReason` of `"error"` MUST map to `Err(QueryError { kind: "transport", message: <errorMessage>, http_status: null, provider: <resolved-model-provider>, retryable: false })`; when `errorMessage` is absent, `message` MUST be the fixed string `"provider transport failure"`. A synchronous throw from `pi.sendUserMessage` is the narrow **secondary** mapping to the same `transport` shape. And the cancellation short-circuit (`loomAbort.signal.aborted` ⇒ `Err(QueryError { kind: "cancelled" })`) MUST take precedence over the error-state probe and the `Ok(string)` extraction, even when Pi tore down cleanly and `waitForIdle()` resolved with no error written to the session.

`V9c-T`'s **Tests** assert only `PIC-2`, `PIC-17`, `PIC-18`, and the happy-path trailing-turn `Ok(string)` extraction. No bullet asserts the `stopReason:"error"` → `TransportError` mapping, the empty-`errorMessage` fallback, the synchronous-throw secondary mapping, or the cancellation-precedence ordering. The entire transport-failure detection surface — the primary failure path of every driven prompt-mode turn — ships with no executable witness.

Compounding the gap: these §Error-detection MUSTs are un-anchored GOV-22 residue, and the coverage matrix has no row mapping them to `V9c`. The only conversation-drive.md row for `V9c` (besides PIC-2 / PIC-17 / PIC-18) covers §`untyped-query-ok-extraction` (the `Ok(string)` extraction), which references the `stopReason:"error"` short-circuit only as ordering context, not as the obligation under audit. So even a newly-added citing test would have no mapped obligation for the H5a closing gate to reconcile against.

## Solution Space

**Shape:** multiple

This finding carries two independent obligations — adding the asserting tests, and giving those assertions a mapped coverage-matrix home — that touch different files with different review surfaces. They are presented as separate options and applied in order.

### Option A — Assert the prompt-mode transport-failure surface in the V9c test pair

**Approach:** Add failing-test bullets to `V9c-T` (mirrored into `V9c`'s Tests, and reflected in `V9c`'s Ships when) covering the §Error-detection MUSTs.

**Plan edits:** In `V9c-T`'s Tests, add bullets for:
- (a) post-`waitForIdle()` trailing-`assistant` `stopReason:"error"` → `Err(QueryError { kind: "transport", message: <errorMessage>, http_status: null, provider: <resolved-model-provider>, retryable: false })`;
- (b) absent `errorMessage` → `message` is the fixed string `"provider transport failure"`;
- (c) synchronous throw from `pi.sendUserMessage` → `Err(QueryError { kind: "transport", message: <error.message>, http_status: null, provider: <resolved-model-provider>, retryable: false })` (the secondary surface);
- (d) `loomAbort.signal.aborted` true after `waitForIdle()` resolves cleanly → `Err(QueryError { kind: "cancelled" })`, taking precedence over the error-state probe and the `Ok(string)` extraction.

Mirror the same assertions into `V9c`'s Tests and extend `V9c`'s Ships when to drive a `stopReason:"error"` turn and observe the synthesised `transport` error.

**Spec edits:** None — `conversation-drive.md` §Error detection already pins all four behaviours.

**Pros:** Closes the actual assertion gap; reuses the existing PIC-bullet style already in the leaf.

**Cons:** On its own, the new bullets assert un-anchored MUSTs with no coverage-matrix row, so the H5a citing-test reconciliation cannot witness them — Option B closes that.

**Risks:** Bullet (d) depends on V17a's `loomAbort` surface, already declared in `V9c`/`V9c-T` Deps.

### Option B — Map the §Error-detection transport-failure MUSTs in the coverage matrix

**Approach:** Add a code-keyed row to `coverage-matrix.md` citing `conversation-drive.md` §`prompt-mode-error-detection` with `V9c` as the closing leaf.

**Plan edits:** Add a row whose obligation text names the driven-turn transport-failure detection MUSTs — `stopReason:"error"` → synthesised `TransportError`; absent `errorMessage` → `"provider transport failure"`; synchronous `sendUserMessage` throw → `transport`; cancellation short-circuit precedence over the error probe and `Ok(string)` extraction — marked `(un-anchored; GOV-22 residue)`, closing leaf `V9c`.

**Spec edits:** None.

**Pros:** Gives the new assertions a traceable closing-leaf mapping and satisfies conventions §REQ-ID discipline's requirement that every un-anchored MUST has a closing-leaf home; the row is what H5a's transitive-completeness arm and H5b's Deps audit witness against.

**Cons:** Marginal; widens the `V9c` row set.

**Risks:** Must not duplicate or fold into the existing §`untyped-query-ok-extraction` row, which covers a distinct obligation (the `Ok(string)` extraction).

### Recommendation

Apply both, Option B first, then Option A: add the coverage-matrix row so the §Error-detection MUSTs have a mapped obligation, then add the `V9c-T` test bullets so they cite a row the H5a citing-test reconciliation is already green against. Watch one edge case in bullet (d): the cancellation short-circuit must be observed firing even when `waitForIdle()` resolved cleanly with no error in the session state (the spec's explicit case), so it cannot collapse into a generic cancelled-mid-flight test.

## Consequence

**Severity:** correctness

The primary transport-failure path of every driven prompt-mode turn is unverified: a regression that mis-synthesises the `TransportError` shape, drops the empty-`errorMessage` fallback, mis-classifies a synchronous throw, or lets the error probe run ahead of the cancellation short-circuit passes `npm test` green and reaches the release gate undetected. Because the §Error-detection MUSTs are also unmapped in the coverage matrix, the H5a closing gate cannot witness `V9c`'s closure of them, so the obligation can silently ship unimplemented.
