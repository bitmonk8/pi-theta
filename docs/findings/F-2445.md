## Finding

The closing edge-case bullet's "Production wiring is a no-op (an already-resolved promise per checkpoint) … The seam is purely a test surface and imposes no observable behaviour on production code beyond the always-`await`ed no-op" contradicts both `cancellation.md`'s own Granularity rule (the `loop-iter` checkpoint yields a macrotask turn) and the precise PIC-10 definition (`host-interfaces-services.md#checkpoint-seam`), which scopes the production effect to a macrotask yield for `loop-iter` and an already-resolved promise for all other kinds.

## Solution Space

**Shape:** single

### Recommendation

In the closing edge-case bullet of `cancellation.md`, drop the blanket "already-resolved promise per checkpoint" / "purely a test surface" framing and cross-reference PIC-10's production-wiring rule (macrotask yield for `loop-iter`, microtask-resolved no-op for the other kinds), keeping the precise "no observable production effect beyond …" scoping only at the PIC-10 definition site. Retain the bullet's substantive point — that the seam is the deterministic-test substrate for the two race rules.

## Consequence

**Severity:** correctness

An implementer reading the closing edge-case bullet in isolation would wire every checkpoint — including `loop-iter` — as an already-resolved promise, defeating cancellation of compute-bound loops (the exact defect the macrotask-yield rule was added to fix); the page contradicts itself on observable production behaviour. The duplicated SDK-wiring prose additionally invites drift between `cancellation.md` and `host-interfaces-core` as either copy is edited.
