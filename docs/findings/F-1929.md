## Finding

Each of the four Tests bullets in both `V9l` and `V9l-T` appends a `(un-anchored; GOV-22 residue)` parenthetical to its `session-only-degraded-state.md §…` citation (`§degraded branch`, `§Recovery-path prohibition`, `§Predicate split`, and `§Seam-minimality + *Inline triplet is normative*`).

The annotation is spec-corpus governance metadata: it records that the cited spec section carries no stable numbered REQ-ID anchor and is therefore swept by the GOV-22 un-anchored-normative-MUST residue gate (`conventions.md` §"Loom 1.0 release-time residue inspection" item 5; `governance/req-id-prefix-table-active-b.md#gov-22`). That fact already lives where it is consumed — the `coverage-matrix.md` `cka-27` row carries the same `(un-anchored; GOV-22 residue)` marker for this page.

On the acceptance-criteria surface the parenthetical constrains nothing the implementer needs: the obligation after the colon (the drain-state tag write, the `"degraded-needs-reload"` transition, the recovery-path prohibition, the predicate split, the seam-minimality prohibition) stands whether or not the spec section is anchored. Repeated eight times across the leaf pair (four per file), it is pure noise on the Tests bullets.

**History:** The annotation was not present at plan inception. It entered incrementally: each commit resolved a prior plan-review finding that a particular un-anchored MUST was unnamed by any leaf, and in each remediation the author both assigned the obligation a closing-leaf Tests bullet and copied the coverage-matrix `GOV-22 residue` classification into that bullet. The two slightly different tag spellings (`(GOV-22 un-anchored residue)` in `V9f` vs `(un-anchored; GOV-22 residue)` elsewhere) reflect the separate remediation events rather than a single authoring pass.


## Solution Space

**Shape:** single

### Recommendation

Strike the literal text ` (un-anchored; GOV-22 residue)` from each of the four Tests bullet labels in both `V9l-session-only-degraded-branch.md` and `V9l-T-session-only-degraded-branch.md`, leaving the `session-only-degraded-state.md §…` citation and the obligation text after the colon intact. For example, the first bullet becomes `session-only-degraded-state.md §degraded branch: on new/resume/fork with a successful snapshot read, …`.

The spec is read-only for this fix. The `coverage-matrix.md` `cka-27` row legitimately retains its `(un-anchored; GOV-22 residue)` marker — that is where the GOV-22 residue gate consumes it — and is out of scope here.

## Consequence

**Severity:** cosmetic

The parentheticals are governance annotation only; they change nothing an implementer must build or assert, and no gate reads them. Leaving them in clutters the Tests bullets and duplicates metadata already carried by `coverage-matrix.md` `cka-27`, but no implementer is misled and no closing gate fires incorrectly.
