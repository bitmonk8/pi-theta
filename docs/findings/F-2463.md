## Finding

The `loom/runtime/reload-teardown-timeout` row in `code-registry-runtime.md` defines how each `ActiveInvocationRegistry` entry in `<list>` renders as `/<slash-name>:<invocation-id>`. Describing the leading `/`, it states the `loom` field is read verbatim "immediately following the literal `/` prefix that the surrounding template contributes **per category 3 above**."

That cross-reference is wrong. Placeholder category 3 (`placeholder-rendering-a.md` §3, *Syntactic-construct placeholder*) governs only `<construct>` and `<expr>`; it contains no leading-`/` or `<slash-name>` convention. The leading-`/` rule actually lives in `placeholder-rendering-b.md` §7's identifier-shaped sub-rule, which pins: "The leading `/` of `<slash-name>` is contributed by the surrounding template (e.g. `/<slash-name>`), not by the placeholder." §7 is also the page that owns the `<slash-name>`, `<invocation-id>`, and comma-space-join sub-rules into which the bespoke `<list>` rendering decomposes (per `placeholder-rendering-a.md` §Closure clause (c)).

An implementer chasing "category 3 above" lands on the syntactic-construct rule and finds nothing about the `/` prefix.

## Issue introduction

**Verdict:** introduced by a single later commit
**Introducing commit:** `52b9101` — "pi-loom spec: resolve \"T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state\"" (2026-05-11)
**History:** `git log -S 'per category 3 above' --all` returns exactly two commits. Before `52b9101`, the `reload-teardown-timeout` row read "…`<slash-name>` is the registered slash command name (without its leading slash) immediately following the literal `/` prefix…" with no category citation. `52b9101` rewrote the row to read the canonical-key `entry.loom` form verbatim and, in that rewrite, appended the erroneous "per category 3 above" pointer; the reference was wrong at the moment it was added, because category 3 has always been the syntactic-construct placeholder rule. The later commit `f5e89f4` (2026-06-04, "docs: enforce 100KB hard / 30KB recommended size cap on spec set") only split `diagnostics.md` into the `diagnostics/` subpages, carrying the defective text verbatim into `code-registry-runtime.md`; it did not introduce the defect.

## Solution Space

**Shape:** single

### Recommendation

In the `loom/runtime/reload-teardown-timeout` row, replace "per category 3 above" with a pointer to `placeholder-rendering-b.md` §7's `<slash-name>` leading-`/` convention — the identifier-shaped sub-rule stating the leading `/` is contributed by the surrounding template, not by the placeholder. Leave the rest of the sentence (the `entry.loom` verbatim-read clause and the slash-registered / callee-only arms) unchanged.

## Consequence

**Severity:** advisory

The misdirected cross-reference sends an implementer to an unrelated placeholder rule, costing search time. It does not cause output divergence: the row's Message template literally writes `/<slash-name>:<invocation-id>`, §7 pins the leading-`/` convention, and §7's test vector renders `/plan:aaaaaaaa-...`, so the produced bytes are unambiguous regardless of the wrong pointer.
