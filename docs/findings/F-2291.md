## Finding

`V8a` and its paired test leaf `V8a-T` both carry the same `PIC-10` test bullet: "a checkpoint awaits at each defined cancel site with the correct yield kind and adds no extra sites." Two of those three clauses — "awaits at each defined cancel site" and "adds no extra sites" — are whole-interpreter properties: they can only be observed once the five wired checkpoint sites exist (each `for`/`while` iteration, each `@`-query dispatch, each tool call, each `invoke`, and the binder LLM call). At `V8a`'s build position its `Deps` are `V8a-T`, `H3a`, `V8d` — the DI-seam skeleton and the `Clock`/`IdSource` seams. No interpreter and no call-sites exist yet, so there is nothing against which "each defined cancel site / no extra sites" can be asserted.

The coverage matrix confirms the site-coverage closure is owned elsewhere: the `cka-47` row distributes the closed five-site set (the "no extra sites" / whole-set obligation) and the two cycle-free site arms to `V17c`, and the per-site presence arms to `V13c` (`@`-query dispatch), `V14a` (tool call), and `V15a` (invoke). The `V8a` `Checkpoint` seam is the testability hook those leaves drive; `V8a` itself owns only the seam mechanics.

What `V8a` *can* assert in isolation via the seam (and `FakeClock`) is the seam-mechanics contract from `PIC-10` in `host-interfaces-services.md#checkpoint-seam`: the yield-kind-per-`CheckpointKind` mapping (a macrotask turn for `loop-iter`, observable through `FakeClock.advance(0)`; an already-resolved microtask for every other kind) and the test-fake call-once-per-checkpoint / no-skip rules. The site-presence and no-extra-sites obligations cannot be asserted here. As worded, `V8a-T`'s `PIC-10` test cannot "fail red for the intended reason," and `V8a`'s `Ships when` over-claims a coverage closure it cannot reach.

## Issue introduction

**History:** The `PIC-10` bullet "a checkpoint awaits at each defined cancel site with the correct yield kind and adds no extra sites" is present verbatim in `V8a` at the plan's first build (`c6a664e`), where `V8a`'s `Deps` were `V8a-T`, `H3a` — already free of any interpreter or wired call-sites. The over-scoped claim therefore predates the per-site distribution. The later commit `849d926` (`fix F-1404 — V17c/V17c-T checkpoint-site deps + per-site distribution`, 2026-06-25) introduced the `cka-47` distribution onto `V13c`/`V14a`/`V15a`/`V17c`, making the ownership-elsewhere relationship explicit, but it did not touch or introduce the `V8a` position-vs-claim mismatch, which has been present since inception.

## Solution Space

**Shape:** single

### Recommendation

In `V8a` and `V8a-T`, rescope the `PIC-10` test to the seam-in-isolation contract and drop the whole-interpreter site claim. The `PIC-10` bullet should assert only what the `Checkpoint` seam exposes without a wired interpreter: the yield-kind-per-`CheckpointKind` mapping — a macrotask turn for `loop-iter` (witnessed via `FakeClock.advance(0)` between the abort-flip and the next signal-check) and an already-resolved microtask for every other kind — together with the test-fake call-once-per-checkpoint and no-skip rules from `host-interfaces-services.md#checkpoint-seam`. Strike "awaits at each defined cancel site … and adds no extra sites" from the `V8a`/`V8a-T` `PIC-10` bullet and from `V8a`'s `Ships when` clause.

Add a cross-reference in `V8a` (and `V8a-T`) noting that `V8a` asserts the seam mechanics only, and that the wired-site coverage — the closed five-site set / "no extra sites" obligation on `V17c` and the per-site presence arms on `V13c` (`@`-query dispatch), `V14a` (tool call), and `V15a` (invoke) — is the `cka-47` distribution, not a `V8a` obligation. Do not add or restate the site-coverage arms on `V8a`; they already exist on those leaves per the `cka-47` row. The spec is read-only for this fix — `PIC-10` in `host-interfaces-services.md` already separates the seam-mechanics contract from the Granularity site set, so no spec edit is required.

## Consequence

**Severity:** correctness

At `V8a`'s build position the "each defined cancel site / no extra sites" clauses have no wired sites to observe, so two reasonable implementers diverge: one rescopes the test to the seam mechanics, while the other satisfies the wording against zero sites — a vacuous gate that passes without verifying anything. The genuine site-coverage closure already lives on `V17c`/`V13c`/`V14a`/`V15a`, so the defect is a misleading/vacuous `V8a` gate rather than missing coverage, but the `V8a-T` "fail red for the intended reason" gate cannot hold as written.
