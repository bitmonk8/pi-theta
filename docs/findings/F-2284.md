## Problem

`H7a`'s **Adds** says the committed golden transcript and golden diagnostics list are derived (not snapshotted from an unreviewed pipeline run): each golden turn and `loom/...` code "is sourced from that leaf's already-established expected output." That phrase names no concrete artifact or procedure — each Deps leaf (`V5d`, `V8a`, `V11a`–`V11f`, `V11i`, `V13a`–`V13f`, `V14a`, `V14b`, `V16a`, `V17a`) expresses its expected behaviour as inline `Tests.` bullet assertions, and no committed per-leaf "expected output" file exists. A fresh implementer cannot tell whether to transcribe those `Tests.` assertions, run each leaf's tests, or snapshot the integrated pipeline — and the last reading directly contradicts the Adds's own "not snapshotted from an unreviewed pipeline run" prohibition.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** c6e2a8b — pi-loom plan: resolve "golden transcript/diagnostics initial-correctness establishment" (2026-06-11, Thomas Andersen)
**History:** `H7a` was created at `e7e51cc` without any golden-derivation prose. The "derived, not snapshotted … is sourced from that leaf's already-established expected output" clause was added at `c6e2a8b` (originally as a `Tests.` bullet) while resolving the initial-correctness-establishment finding; the fix introduced the undefined "already-established expected output" phrase. A later commit (`0904502`) moved the methodology from the `Tests.` bullet into `Adds`, carrying the ambiguous phrase verbatim, where it remains.

## Solution approach

In `H7a` **Adds**, rewrite the clause "is sourced from that leaf's already-established expected output" into a concrete derivation procedure that names the Deps leaves' inline `Tests.` bullet assertions as the source for each behaviour-gated golden turn and each `loom/...` code. Make clear the derivation reads the leaves' `Tests.` assertions rather than executing the pipeline to obtain the goldens. Retain the existing "not snapshotted from an unreviewed pipeline run" prohibition and the existing first-commit human review of the composed cross-leaf turn ordering against `docs/spec.md`.
