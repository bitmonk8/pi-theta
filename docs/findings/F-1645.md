## Problem

In the `### Rules` list of `docs/spec_topics/descriptions.md`, the `**No transformation.**` bullet lists `dedenting` among the transforms it excludes ("no escaping, dedenting, or wrapping is performed beyond …"). But the adjacent `**Multi-line.**` bullet performs exactly that operation — "common leading whitespace inside the description is stripped (same algorithm as query-template dedent)" — and the spec uses *dedent* as the canonical name for the performed transform. Naming `dedenting` in the exclusion list of one bullet while the same operation is named as *performed* two lines above is a surface self-contradiction. A reader anchoring on the `No transformation.` headline can conclude no whitespace stripping happens, diverging from the byte-for-byte lowering behaviour the V5c leaves assert.

## Issue introduction

**Verdict:** single-commit-introduction
**Introducing commits:** 7f217a8 ("pi-loom spec: resolve \"Language-core hidden assumptions: enum-tag sidecar, AJV-config silence, Markdown claim\"", 2026-06-07)
**History:** Before 7f217a8 the bullet read `**Markdown.** Description text is treated as Markdown by providers; no transformation is performed.` — a one-sentence Markdown-provenance claim with no enumeration of excluded transforms, and therefore no opportunity to contradict the Multi-line bullet. The fix-loop commit 7f217a8 rewrote the bullet to its current form (`**No transformation.** Loom emits description text byte-for-byte into the lowered schema; no escaping, dedenting, or wrapping is performed beyond the multi-line join and common-leading-whitespace strip defined above. *(Non-normative provenance.)* …`). The Multi-line bullet that performs the strip predates the rewrite (introduced in 53f5831, 2026-05-03, "spec: rewrite query primitive, system prompts, and schema syntax"), so the contradiction is wholly attributable to the 7f217a8 rewrite — that single commit chose the word "dedenting" for the exclusion list while the same paragraph already used the same operation in the carve-out.

## Solution approach

Remove `dedenting` from the exclusion list of the `**No transformation.**` bullet in `docs/spec_topics/descriptions.md`, leaving the byte-for-byte-emission claim and the remaining exclusions (escaping, wrapping) intact. Keep the `beyond the multi-line join and common-leading-whitespace strip defined above` carve-out, which continues to pin which transforms are performed and preserves the dedent-algorithm link via the Multi-line bullet.
