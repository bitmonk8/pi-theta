## Finding

The `coverage-matrix.md` intro asserts, for every REQ-ID's mapped implementation leaf, that "its green tests are the closure evidence." This overstates the strength of the only mechanical backing the plan provides. The `H5a` closing-gate automation's citing-test arm is, by its own Tests bullet, "a best-effort inline-`PREFIX-N`-citation scan over the test sources [that] certifies citing-test existence, not the assertion's semantic fidelity." A green run therefore proves only that *a* test naming the REQ-ID exists and passes — not that the test's assertions faithfully exercise the spec obligation.

The overclaim is sharper for multi-leaf (split) rows. For a row such as `ERR-17 | V4d, V13d` or the five-leaf `cka-33` row, the H5a per-REQ-ID scan is satisfied by one citing test anywhere across the listed leaves, and the H5d per-cell at-least-one rule keeps the gate green even when a facet-closing leaf carries no facet-specific assertion. The only backstop for split-row joint-completeness is the manual residue-inspection item 7 in `conventions.md`. "Green tests are the closure evidence" reads as a categorical guarantee where the mechanism delivers an existence check plus deferred manual/TDD obligations.

The identical sentence appears verbatim in `conventions.md` §Leaf format (the `Deps.` coverage-matrix paragraph), so softening only one location leaves the two copies divergent.

## Solution Space

**Shape:** single

### Recommendation

Soften the overclaiming clause in both files that carry it, replacing the categorical "its green tests are the closure evidence" with a qualified statement that scopes the mechanical guarantee to existence-checking and routes semantic fidelity and split-row joint-completeness to their actual obligations.

- `docs/plan_topics/coverage-matrix.md`, intro paragraph (line 3): in the parenthetical `(its green tests are the closure evidence; the paired \`<id>-T\` tests task is not listed separately)`, replace `its green tests are the closure evidence` with text to the effect of `its green citing tests are the best-effort closure evidence (existence-checked by \`H5a\`; semantic faithfulness and split-row joint-completeness are TDD / self-review and residue-inspection item-7 obligations)`.
- `docs/plan_topics/conventions.md` §Leaf format (line 51): apply the same softening to the identical clause `its green tests are the closure evidence`, so the two copies stay consistent.

Edge case: the two occurrences are byte-identical today; fixing only one reintroduces the divergence the cruft/placement lenses would flag. Keep both edits in the same change. The clause sits in prose, not in a machine-read closing-leaf cell, so the backtick-tokeniser constraints in *Closing-leaf column authoring* do not apply here.

## Consequence

**Severity:** advisory

A reviewer or auditor reading the intro can over-trust a green gate as proof of faithful, complete spec closure, lowering diligence on the self-review / residue-inspection obligations (item 7) that actually carry semantic-fidelity and split-row joint-completeness. No leaf's output diverges, but the prose mischaracterises the evidentiary strength of the closing gate.
