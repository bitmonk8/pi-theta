## Finding

`V9f`'s **Adds.** field describes the materialised `ToolDefinition.label` as "the loom file's **basename** with interior hyphens preserved and only the leading character capitalised," illustrated by `code-review.loom → "Code-review"`. The worked example strips the `.loom` extension, but `path.basename(...)` (and the ordinary meaning of "basename") returns the filename *including* its extension — `path.basename("code-review.loom")` is `"code-review.loom"`, which would capitalise to `"Code-review.loom"`. The term names the wrong derivation; the intended input is the filename **stem** (the basename with the extension removed).

The leaf is internally self-correcting — the `code-review.loom → "Code-review"` example and the matching Tests / Ships-when assertions pin the exact expected output, so an implementer who follows the example reaches the right result. The defect is the load-bearing word in the prose contradicting its own example. The same imprecise term appears in the upstream spec prose (`extension-bootstrap-and-per-loom.md` §Per-loom registration, `label` field), which the plan leaf mirrors; the plan can state the precise derivation without a spec edit.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V9f-tool-registration-lifetime.md`, **Adds.** field, replace "the loom file's **basename** with interior hyphens preserved" with "the loom file's **stem** (filename without the `.loom` extension) with interior hyphens preserved". The `code-review.loom → "Code-review"` example, the `extension-bootstrap-and-per-loom.md` Tests bullet, and the Ships-when assertion already encode the correct behaviour and need no change; this aligns the load-bearing term with them. The fix is internal to the plan leaf — the spec is read-only context for this change.

## Consequence

**Severity:** advisory

The prose names `path.basename` semantics (extension retained) while the example and acceptance assertions require the stem (extension stripped). An implementer reading the prose literally without anchoring on the example could derive `"Code-review.loom"`; the example and Ships-when assertions catch that, so the gap is a guidance imprecision rather than a divergence that ships.
