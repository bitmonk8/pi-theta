## Problem

`V9b` is the single declaration site of the test-only `ReloadFailureInjector` interface, and all three watcher-time reload failure-injection arms — registry-swap (`loom/runtime/registry-swap-failed`), `.loom`/`.warp` re-parse, and settings re-merge — inject through its one `injectReloadFailure` method. The leaf describes the method only in prose, pinning no parameter signature and no arm-discriminating argument, yet three surfaces call it: `V9b` (registry-swap + re-parse), `V10d` (settings-re-merge), and `V4e` ERR-7 (imports the interface). With no pinned call shape, the callers can adopt incompatible argument conventions and fail to agree on how to select an arm. The leaf is also silent on the `triggerTurn` value for the registry-swap and re-parse arms — only settings-re-merge has a stated value (`triggerTurn:false` in `V10d`) — leaving those arms' turn-firing contract unobservable for tests.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `d64dce5` — "pi-loom plan: resolve \"Watcher-time reload failure-injection seam under-specified and ungated\"" (2026-06-11)
**History:** The `injectReloadFailure` method entered the corpus in `d64dce5`, the commit that created the watcher-time reload failure-injection seam. From inception the method was described only in prose ("whose `injectReloadFailure` method a caller invokes to hand it a synthetic failure"), with no parameter signature, no arm discriminator, and no `triggerTurn` value for the registry-swap / re-parse arms. Subsequent commits that reshaped seam ownership — `fdce2a2` (consolidated the interface to a single declaration site at `V9b`) and `f14e929` (split the settings-re-merge arm out of `V10c` into `V10d`) — rewrote the surrounding prose but never pinned a signature. The gap has therefore been present since the seam was introduced.

## Solution approach

In `V9b`'s **Adds**, pin `injectReloadFailure`'s parameter as a closed, arm-discriminated argument covering the registry-swap, re-parse, and settings-re-merge arms, and state the `triggerTurn` value each arm routes with (settings-re-merge is already `triggerTurn:false` in `V10d`; supply the registry-swap and re-parse values too). Update `V10d`'s **Adds** and `V4e`'s ERR-7 **Adds/Tests** bullets to cite the discriminator each uses against the pinned shape. Update `V9b-T`, `V10d-T`, and `V4e-T` to assert against the pinned shape and the per-arm `triggerTurn` values.

## Solution constraints

- `V9b` remains the sole declaration site: `V10d` and `V4e` (and their `-T` partners) reference the pinned shape rather than declare a second interface or arm-selection convention.
- The seam is test-only and plan-internal; the spec is read-only for this fix.
