## Finding

The `tools` field bullet in `frontmatter-fields-a.md` names the enforcing SDK calls (`customTools` + `tools` allowlist on `createAgentSession`; `pi.setActiveTools` snapshot/restore) in the same sentence that delegates the mechanism to PIC — duplicating PIC-owned content in the field page.

## Issue introduction

**Verdict:** specific-commit (two distinct introducing commits, one per obligation)
**Introducing commits:**
- `7ce0ffa` (2026-05-04) "pi-loom spec: resolve \"Tool registry mutation during execution and model-driven parallel tool calls\"" — authored the Resolution-snapshot block; its "Sub-decision A — registry snapshot" explicitly chose the "frozen table … holds a strong reference" framing as the resolution.
- `27ec53d` (2026-05-04) "Resolve Cluster-A: per-mode tool registration" — added the `customTools`/`createAgentSession`/`pi.setActiveTools` enforcement parenthetical to the `tools` field bullet while pointing to PIC for the full mechanism.

**History:** The spec corpus is git-tracked. `git log -S` on the defect tokens ("strong reference", "frozen per-loom table", "setActiveTools snapshot/restore", "customTools") shows neither passage is present-since-inception: each entered as part of a resolution commit that fixed a genuine prior ambiguity (registry mutation semantics; per-mode tool registration) but over-specified mechanism on the way. The 2026-05-08 `docs/`-move (`31ff060`) and the 2026-06 topic split (`f5e89f4`) only relocated the existing prose; they did not introduce the defect.

## Solution Space

**Shape:** single

### Recommendation

Keep the behavioural statement (the Pi session's ambient tools are not inherited; opt in by listing each callable explicitly) and the cross-reference to Pi Integration Contract — Tool-registration lifetime and visibility. Delete the SDK-call parenthetical so the enforcement mechanism is owned solely by PIC.

## Consequence

**Severity:** advisory

Implementers can still build a conforming, correct system — the observable contract is fully stated alongside the over-pinned wording. The cost is constrained design freedom (the "strong reference" term forecloses otherwise-conforming reference disciplines) and a duplicated SDK-call mechanism in the field page that can drift from its PIC owner.
