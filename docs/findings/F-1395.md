## Problem

The terminal `console.error` step of the System-notes fallback chain — defined in `runtime-event-channel.md`'s closing fallback paragraph as "if both channels fail, the failure is logged to `console.error` and execution continues" — is not wrapped in `try`/`catch` at its definition site. The only rule that wraps a terminal `console.error` of that chain, the *Pi-side stdio visibility* sentence in `diagnostic-emission-isolation.md` (`#pic-emission-visibility`), narrows its scope to the per-invocation `finally` clean-cancel reach-path. The bootstrap-failure reach-path (the **Extension-bootstrap SDK failures** rule) and the capability-probe `host-incompatible` reach-path are therefore uncovered: a `console.error` that throws in an extension host escapes the surrounding per-call `try`/`catch`, unwinds the factory body or the `session_start` handler, and breaches both the factory MUST-NOT-throw contract and the `session_start` swallow obligation.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `11b9199` (2026-05-04, "pi-loom spec: resolve \"pi.sendMessage failure leaves system notes silently dropped\"") established the System-notes fallback chain with `console.error` as the unwrapped terminal step and the wording "if both channels fail, the failure is logged to `console.error` and execution continues"; `2d2fc8e` (2026-05-06, "pi-loom spec: resolve \"Pi SDK capability calls have no failure contract\"") added the **Extension-bootstrap SDK failures** paragraph and routed `loom/load/extension-bootstrap-failed` through that same fallback chain under a per-call `try`/`catch` while asserting the factory MUST-NOT-throw contract; `6085192` (2026-05-07, "pi-loom spec: resolve \"Session-shutdown teardown: console.error is the unguarded last resort\"") added the **Diagnostic-emission isolation** rule that wraps the teardown-handler `console.error` sites and, in a single trailing sentence, narrowed the analogous wrap of "the terminal `console.error` of the `sendSystemNote` fallback chain" to the per-invocation `finally` reach-path only; `035d82e` (2026-06-07, "pi-loom spec: resolve \"diagnostic-emission-isolation opening scope omits per-invocation surfaces\"") later widened the opening scope to three site classes but kept the same reach-path narrowing on the `sendSystemNote` sentence.
**History:** The defect is the interaction of three commits. `11b9199` left the chain's terminal `console.error` un-wrapped at its definition site. `2d2fc8e` added a caller (the bootstrap-failure paragraph) whose surrounding contract — factory MUST-NOT-throw — requires that terminal step to be wrapped on this reach-path, but did not add the wrap. `6085192` added the swallow rule that would have closed the gap, then scoped it narrowly to the per-invocation `finally` reach-path, leaving the bootstrap-failure reach-path (and the capability-probe reach-path that pre-dates `2d2fc8e`'s explicit routing) uncovered. No single commit introduced the defect alone; each individually preserved its local invariant, and the gap emerged where their scopes meet.

## Solution approach

Move the swallow obligation to the fallback-chain definition site. Amend `runtime-event-channel.md`'s closing fallback paragraph so the terminal `console.error` step is itself defined as wrapped and silently swallowing, regardless of the reach-path from which the chain is entered. Rewrite the *Pi-side stdio visibility* sentence in `diagnostic-emission-isolation.md` (`#pic-emission-visibility`) to cross-reference that definition-site obligation rather than restate the per-invocation narrowing.

## Solution constraints

- Strengthening the fallback paragraph's normative-modal content with the wrap MUST requires coining a co-located `PIC-N` REQ-ID anchor in the same commit per GOV-22 (`governance/req-id-prefix-table-active-b.md#gov-22`).
