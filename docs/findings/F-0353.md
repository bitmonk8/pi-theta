## Finding

The `ERR-8` Tests bullet in the `V4c` / `V4c-T` leaf pair reads "a mid-stream cancellation does not mutate Pi-committed surfaces." The phrase "Pi-committed surfaces" is a loose label: read on its own it could denote either the conversation turns Pi has appended (assistant tokens, tool-call cards, system notes) or any durably-written Pi-side state such as registrations or the registry. The leaf gives no inline disambiguation.

The authoritative spec the leaf links is precise. `error-model.md` ERR-8 binds the runtime against truncating, re-writing, replacing, or removing assistant tokens, tool-call cards, or system notes Pi has committed to the conversation; `cancellation.md` similarly spells out "Pi-committed conversation surfaces (assistant tokens, tool-call cards, system notes)." The plan leaf's summary drops the parenthetical enumeration the spec carries, leaving the leaf prose narrower-looking and more ambiguous than the rule it summarises.

The defect is confined to the summary label; the `ERR-8` test ID anchors the implementer to the precise spec rule, so the practical risk is small. The tidy is to make the leaf's Tests bullet name the same conversation surfaces the spec enumerates rather than the bare "Pi-committed surfaces" shorthand.

## Solution Space

**Shape:** single

### Recommendation

In both `docs/plan_topics/V4c-terminal-outcomes.md` and `docs/plan_topics/V4c-T-terminal-outcomes.md`, edit the `ERR-8` Tests bullet so its label names the same conversation surfaces the spec enumerates rather than the bare "Pi-committed surfaces" shorthand. Replace the current line

`- `ERR-8`: a mid-stream cancellation does not mutate Pi-committed surfaces.`

with a bullet that points at the assistant tokens, tool-call cards, and system notes Pi has committed to the conversation, e.g.

`- `ERR-8`: a mid-stream cancellation does not mutate the conversation Pi has committed — no truncation, re-write, replacement, or removal of assistant tokens, tool-call cards, or system notes (per `error-model.md` ERR-8).`

Keep both leaves' bullets in sync. The spec files stay read-only; this is a plan-side wording alignment to the existing ERR-8 rule, not a spec change.

## Consequence

**Severity:** cosmetic

A reader skimming the `ERR-8` Tests bullet without following the test ID to the spec could momentarily read "Pi-committed surfaces" as durable Pi-side state (registrations, registry) rather than appended conversation turns. The `ERR-8` ID anchors the implementation to the precise spec rule, so no working leaf is actually blocked; the issue is summary wording that is looser than the rule it stands for.
