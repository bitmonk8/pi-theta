## Finding

The `engines.node` literal-read assertion bullet in `H1a`'s **Tests** section describes its comparison operand as "the Node-floor literal (currently `>=22.19.0`) transcribed inline into this `H1a` test directly from `capability-probe.md` §(a)". The bullet's design is sound — it deliberately transcribes the floor literal into the test code so the population-time check is not a vacuous self-comparison, and it states that a version-bump moves that transcribed literal in lockstep with `capability-probe.md` §(a).

The defect is the descriptive prose parenthetical `(currently >=22.19.0)`, distinct from the load-bearing literal that lives in the test code. The parenthetical restates the current value of a floor whose canonical owner (`capability-probe.md §(a)`) is already named in the same sentence. It is a second, ungated copy of the floor: after the next Node-floor bump, the version-bump procedure updates `package.json#engines.node` and the test's transcribed literal, but nothing forces this prose snapshot to move with them, so it silently drifts to a wrong value.

The load-bearing transcription inside the test (and the `H1a`-resident-copy rationale) is not the target — only the redundant prose snapshot is. Removing the parenthetical leaves the sentence pointing readers at the authoritative anchor for the actual value.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H1a-scaffold-and-toolchain.md`, in the **Tests** bullet anchored to `capability-probe.md §(a)`, strike the parenthetical ` (currently \`>=22.19.0\`)` so the clause reads "The comparison operand is the Node-floor literal transcribed inline into this `H1a` test directly from `capability-probe.md` §(a) — an independently-sourced copy resident in the test itself."

Leave the rest of the bullet intact — in particular the transcribed-literal design, the version-bump-alignment sentence, and the `H1a`-resident-copy rationale. The literal `>=22.19.0` that resides in the test code itself is load-bearing and stays; only the redundant descriptive snapshot in the leaf prose is removed.

## Consequence

**Severity:** cosmetic

After a Node-floor bump the prose snapshot `(currently >=22.19.0)` becomes a stale value with no gate to catch it. Reader impact is bounded: the authoritative anchor `capability-probe.md §(a)` is named in the same sentence and the test's own transcribed literal (not this parenthetical) is what the assertion compares, so an implementer is already directed to the correct source.
