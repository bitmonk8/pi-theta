## Finding

`V17a` (Cancellation core) asserts the absence of a *second* `RuntimeEvent` in two places: the `CNCL-3` test (`no second RuntimeEvent is produced per invocation`) and the swallowing-handler side-channel suppression test (`no Node unhandledRejection, no second RuntimeEvent, and no diagnostic of any severity`). Its `Ships when` re-states both. The paired tests leaf `V17a-T` carries the same two assertions.

The `RuntimeEvent` emission shape and channel are owned by `V9d` (Runtime-event channel and `masked` co-fire) — its `Adds` defines the single `sendMessage` runtime-event shape, the routing, and the emission policy. Neither `V17a` (`Deps.` = `V17a-T`, `V8a`, `V4d`, `V9a`, `H4a`, `H4b`) nor `V17a-T` (`Deps.` = `V8a`, `V4d`, `H4a`) lists `V9d`. The sibling leaf `V16a`, which makes an analogous `masked`-on-`RuntimeEvent` assertion, correctly lists `V9d` in its `Deps.`; `V17a`/`V17a-T` do not, and `V9d` has never appeared in either file's history.

Because the pickability predicate is satisfied without `V9d`, an implementer can pick `V17a-T`/`V17a` before the runtime-event channel exists. With no channel to emit a first `RuntimeEvent`, the "no second `RuntimeEvent`" arms have nothing real to observe: they either pass vacuously (reporting success while verifying nothing — the `No silent test skipping` failure mode the plan forbids) or fail against a missing emission. No earlier seam supplies the channel; `V9d` is its sole producer.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (pi-loom plan: build/update plan for spec.md + review, 2026-06-10)
**History:** `V17a-cancellation-core.md` was created in `c6a664e` already carrying the `CNCL-3` "no second `RuntimeEvent`" assertion, and its `Deps.` line has never contained `V9d` (`git log -S 'V9d' -- plan_topics/V17a-cancellation-core.md` returns no commits). Later commits `52a6819` and `dc56e9a` expanded the `RuntimeEvent` wording in the test/Ships-when bullets but never added the `V9d` dependency edge. The gap is original to the leaf, not introduced by a later edit; `V17a-T` (created in the same inception lineage) carries the identical omission.

## Solution Space

**Shape:** single

### Recommendation

Add `V9d` to the `Deps.` field of both `docs/plan_topics/V17a-cancellation-core.md` and `docs/plan_topics/V17a-T-cancellation-core.md`, mirroring `V16a`, which already lists `V9d` for its own `RuntimeEvent`/`masked` assertions.

- `V17a-cancellation-core.md` `Deps.`: change `` `V17a-T`, `V8a`, `V4d`, `V9a`, `H4a`, `H4b` `` to include `` `V9d` ``.
- `V17a-T-cancellation-core.md` `Deps.`: change `` `V8a`, `V4d`, `H4a` `` to include `` `V9d` ``.

`V9d` is the sole producer of the runtime-event channel (no earlier seam emits it), so the "or state the producer" alternative resolves to the same edge.

Edge case: if the separate `V17a` atomicity split (core/race) lands, the `V9d` edge belongs on whichever child carries the `CNCL-3` and swallowing-handler arms — apply it there rather than to a `V17a` that no longer holds those assertions.

## Consequence

**Severity:** correctness

Two reasonable implementers diverge: one builds `V9d` first and the no-second-`RuntimeEvent` arms assert against a real channel; the other picks `V17a`/`V17a-T` earlier (Deps allow it) and the same arms assert against a channel that does not exist, passing vacuously or reddening on a missing emission. The vacuous-pass path silently certifies `CNCL-3` and the swallowing-handler suppression contract as verified when nothing was exercised.
