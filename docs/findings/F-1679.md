## Finding

The "Acceptance criteria — PIC-17 active-set install vector" block at the end of `docs/spec_topics/pi-integration-contract/tool-registration-lifetime.md` enumerates exactly two fixtures: **(a)** empty callable set on an untyped/free turn (install vector `[]`) and **(b)** forced-respond on a typed-query response turn (install vector `[...loomCallableSetNames, respondToolName]`). Both pin the boundary cases — the empty-set degenerate and the typed-query injection — but neither exercises the production-nominal shape: a loom whose declared callable set is non-empty, on an untyped free turn (no respond tool), with a non-empty step-1 `pi.getActiveTools()` snapshot containing tools that are **not** in the callable set.

That nominal shape is the one PIC-17's payoff invariant — `[Parameters and Frontmatter]` "ambient tools are deliberately *not* inherited" — most directly witnesses. Under the empty-callable-set fixture (a) the install vector is `[]`, so the "no snapshot member appears in the install vector" check is vacuously satisfied by emptiness alone and cannot distinguish a correct implementation from one that silently unions or replaces. Under the forced-respond fixture (b) the install vector is dominated by the synthesised respond tool, and the "exactly `[...loomCallableSetNames, respondToolName]`" assertion does not separately witness non-leakage of step-1 snapshot members on the free-turn path. The leaf-owned `V9c-T` test cited against PIC-17 (`a query snapshots active tools, sets them, and restores in finally; ambient tools are not inherited`) therefore inherits a coverage gap from the spec: there is no normative fixture that would catch an implementation that, on a free untyped turn with a non-empty callable set, installs `[...callableSetNames, ...snapshotNames]` instead of exactly `[...callableSetNames]`.

The block's preamble already commits to fixture-completeness ("decomposes into the fixture obligations below, **all of which** the implementation MUST satisfy"), so the omission is straightforwardly an enumeration gap rather than a deliberate scoping choice.

## Solution Space

**Shape:** single

### Recommendation

Add a third fixture **(c)** to the `## Acceptance criteria — PIC-17 active-set install vector` block, between fixtures (a) and (b), reading approximately:

> **(c) Non-empty callable set, untyped/free turn.** For a loom whose declared callable set is non-empty on an untyped query (no respond tool active), with the step-1 `pi.getActiveTools()` snapshot non-empty and disjoint from the callable set: assert the step-2 install vector is exactly `[...loomCallableSetNames]` (with `loomCallableSetNames` in the order pinned at the PIC-17 install-vector site) and contains no member of the step-1 snapshot.

Edge cases the implementer must witness:

- The step-1 snapshot must be **non-empty** and must contain at least one name **not** present in the callable set; a snapshot equal to or contained in the callable set leaves the non-leakage invariant under-witnessed (the assertion collapses to a tautology when the two sets coincide).
- Disjointness of the snapshot from the callable set must be a precondition of the fixture's setup, not an inferred property of the implementation under test; the fixture controls both the snapshot Pi returns and the loom's declared `tools:` set.
- The fixture must inspect the step-2 install vector exactly (set equality plus order, not membership-only); set-only checking would silently accept an implementation that prepends or appends snapshot members.

Cite the new fixture from `V9c-T`'s `PIC-17` test bullet so the coverage-matrix per-facet scan witnesses it.

## Consequence

**Severity:** correctness

The PIC-17 acceptance enumeration is the spec's sole closing surface for the "ambient tools not inherited" invariant on the prompt-mode free-turn path. As written, two conformant test suites can both pass while differing on whether step-1 snapshot members leak into the step-2 install on the most common production shape — the central invariant becomes unfalsifiable on its nominal path. Coverage-matrix machinery (`H5f` per-facet citing-test gate) cites `V9c` against this acceptance block, but a green facet test under the current two fixtures does not witness non-leakage; the omission silently widens what `V9c-T` is permitted to assert and survives the closing gate.
