## Problem

`NOCEIL-1` in `hard-ceilings/ceiling-invariants-and-audit.md` asserts the absence of a per-call wall-clock timeout is enforced at parse time by rejecting a `timeout:` field at `every site where a future per-call timeout could land`, and the four-axis closure list's `time` row repeats the same universal. The bullet does not name the rule that enforces the rejection — the `loom/parse/timeout-field-rejected` registry row — nor the closed set of sites the universal covers (query, tool-call, invoke); both must be reconstructed by cross-walking the diagnostic registry, the cancellation-page prose, and three `loom 1.0 seam — per-call timeout` blockquotes that never name `NOCEIL-1`. The audit-methodology closure immediately below leans on this universal, so the gap propagates to the `time` axis and leaves a future ceiling-introducing leaf without a canonical site checklist to keep in lock-step.
