## Problem

Two normative load-time rules pivot on the binary "bypass-eligible / non-bypass loom" predicate — the `bind_model` field-contract row in `frontmatter/frontmatter-fields-a.md` and the `looms.binderModel` keys-read entry in `discovery/package-and-settings.md` — but neither anchors the reader at the predicate's definition. One links only to the whole-page index `../binder.md` (a sub-directory table of contents that does not name the bypass cases); the other omits a cross-reference entirely. The canonical predicate is the two-case enumeration (no-params bypass and single-string bypass) at `binder/binder-bypass-and-envelope.md`. The risk is navigation friction rather than divergent semantics, but the obvious shortcut ("no `params:`") covers only one of the two cases and would mis-fire `loom/load/binder-model-unresolved` on a legitimate single-string-bypass loom.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 0fa81fa — spec: pin binder model resolution; document envelope two-arm rationale (Decisions #5, #6) (2026-05-05, Thomas Andersen); f5e89f4 — docs: enforce 100KB hard / 30KB recommended size cap on spec set (2026-06-04, Thomas Andersen)
**History:** Commit `0fa81fa` introduced the bypass-eligibility predicate into both the `bind_model` field-contract row and the binder-model prose in the original monolithic `docs/spec.md` as part of Decision #5 (binder-model resolution pinned to a two-step chain with no further fallback). The row cited only `[Slash-Command Argument Binding](./binder.md)`; the predicate's definition lived on the same large page so the reference resolved to the same document. The 100KB shard split (`f5e89f4`, 2026-06-04) carried the row verbatim into `docs/spec_topics/frontmatter/frontmatter-fields-a.md` and the bypass-cases definition into `docs/spec_topics/binder/binder-bypass-and-envelope.md`, repointing the link to `../binder.md` — now a four-child sub-directory index page that no longer names the predicate. The "Required when any non-bypass loom is in scope" rule in `discovery/package-and-settings.md` was added without any cross-reference. The defect has therefore been present at the cited sites since each was first written; no later commit introduced or amplified it (`b0ae34f` rewrote the same row's strict-capability clause but left the `../binder.md` link unchanged).

## Solution approach

Add an anchored forward-link to the bypass-case definition at `binder/binder-bypass-and-envelope.md` (anchor `#binder-bypass`) at both load-bearing sites: the `bind_model` field-contract row in `frontmatter/frontmatter-fields-a.md` and the `looms.binderModel` keys-read entry in `discovery/package-and-settings.md`. Reuse the same anchor that `binder/binder-model-and-context.md` already targets so the prose sites converge on one anchor name.

## Solution constraints

- Do not redefine the bypass predicate inline at either site; the two-case enumeration has a single owning definition at `binder/binder-bypass-and-envelope.md`.
