## Finding

Two normative load-time rules pivot on the binary "bypass-eligible / non-bypass loom" predicate:

- `frontmatter/frontmatter-fields-a.md` line 40 — the `bind_model` field-contract row: "When neither `bind_model:` nor `looms.binderModel` resolves and the loom is not bypass-eligible, load fails with `loom/load/binder-model-unresolved`."
- `discovery/package-and-settings.md` line 64 — the `looms.binderModel` keys-read entry: "**Required when any non-bypass loom is in scope** — a non-bypass loom whose `bind_model:` is also absent fails to load with `loom/load/binder-model-unresolved`."

Neither site links to the predicate's definition. Both either omit a cross-reference entirely (`package-and-settings.md`) or link only to the whole-page index `../binder.md` (`frontmatter-fields-a.md`), which is a four-child sub-directory table of contents that does not name the bypass cases. The canonical predicate exists — it is the two-case enumeration at `binder/binder-bypass-and-envelope.md#binder-bypass` (anchor also reachable as `#bypass-cases`): the no-params bypass and the single-string bypass. Sibling pages that depend on the same predicate already cite it correctly: `binder/binder-model-and-context.md` line 5 links to `binder-bypass-and-envelope.md#binder-bypass`, and `pi-integration-contract/capability-inventory-items.md` item 7 links to `binder-bypass-and-envelope.md#bypass-cases`.

An implementer reading the `bind_model` row in isolation has no anchored path to the definition and must navigate via the `../binder.md` index page to discover which of the four binder sub-pages owns "bypass". The risk is navigation friction rather than divergent semantics — the predicate has one definition site — but the load-bearing rule should not silently presume the reader has already read the binder bypass page, especially when the obvious guess ("no `params:`") covers only one of the two cases and would mis-decide single-string-bypass looms.

## Solution Space

**Shape:** single

### Recommendation

At both citation sites, replace the unanchored (or absent) cross-reference with an anchored link to the bypass-case definition, matching the form already used by `binder-model-and-context.md` and `capability-inventory-items.md`:

- In `docs/spec_topics/frontmatter/frontmatter-fields-a.md` (the `bind_model` row of the field-contract table), replace "the loom is not bypass-eligible" with a link such as "the loom is not [bypass-eligible](../binder/binder-bypass-and-envelope.md#binder-bypass)" (the `#binder-bypass` anchor is the section heading; `#bypass-cases` is an equivalent in-section anchor — pick whichever matches the surrounding house style, but reuse the same anchor that `binder-model-and-context.md` line 5 already targets so the two prose sites converge on one anchor name).
- In `docs/spec_topics/discovery/package-and-settings.md` (the `looms.binderModel` keys-read entry), apply the same anchored link to the two occurrences of "non-bypass loom" in the "**Required when any non-bypass loom is in scope**" rule.

Do not redefine the predicate inline at either site — duplicating the two-case enumeration would create a second definition surface that would have to be kept in sync with `binder-bypass-and-envelope.md` and that the *Glossary bypass-eligible predicate conflicts with the diagnostics registry* finding (commit `94e292f`) has already paid down once. The link form keeps the single owning definition.

Implementer edge case: the anchor target page contains both `<a id="binder-bypass">` (the section heading at line 3) and `<a id="bypass-cases">` (the immediate sub-anchor at line 5 that introduces the two-case enumeration). Either resolves the reference correctly; choose `#binder-bypass` to match the existing `binder-model-and-context.md` citation so a future anchor-stability audit sees one citation pattern, not two.

## Consequence

**Severity:** advisory

An implementer reading the `bind_model` row or the `looms.binderModel` entry must traverse the unanchored `../binder.md` index page and choose the right sub-page to discover what "bypass-eligible" means. The likely shortcut — assuming the predicate is "no `params:`" — silently omits the single-string-bypass case, which would mis-fire `loom/load/binder-model-unresolved` on a single-string-param loom that legitimately skips the binder. The risk is contained because the canonical predicate has exactly one definition site, but the load-bearing rule should anchor the reader at it directly.
