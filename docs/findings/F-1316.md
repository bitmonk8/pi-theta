## Finding

The single `V9l` bullet in `docs/plan.md` §Blocked obligations uses the bare label "(a)" for two different things in the same paragraph. The first is the **host-prerequisites open contradiction** — "host-prerequisites clause (a)" (`host-prerequisites.md#degraded-state-host-prerequisites`), the unresolved spec question that blocks the leaf. The second is the **resolution owner** — "version-bump editorial-review checklist item (a)" (`version-bump-step2.md#bump-checklist-instance-survival`), the checklist item whose verdict closes that question.

Both end in the same parenthetical "(a)", and the paragraph places them in close proximity: "…owned by version-bump editorial-review checklist item (a) (…), tracked … by `V18c`'s Deps edge … Pre-designed fallback: if the version-bump editorial-review checklist item (a) (…) resolves the host-prerequisites clause (a) open contradiction…". A reader scanning for the bare "(a)" sees "item (a) … resolves … clause (a)" and can read it as circular or self-referential — conflating the open question with the item that resolves it. The disambiguating noun phrases ("host-prerequisites clause" vs "version-bump editorial-review checklist item") are present but the shared bare letter undercuts them on a quick scan.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `55ed974` (2026-06-12, "pi-loom plan: resolve \"degraded-state obligations green over open spec contradiction\"")
**History:** `git log -S` on both "host-prerequisites clause (a)" and "editorial-review checklist item (a)" over `docs/plan.md` reports the same two commits — `55ed974` and `9c42f2d`. `git show 55ed974 -- docs/plan.md` shows the §Blocked obligations section was created whole in that commit, already carrying both bare-"(a)" labels in the one `V9l` paragraph. The later `9c42f2d` (2026-06-16) reworded only the "Pre-designed fallback" clause; the dual-"(a)" labeling predates it. The clarity defect therefore entered with the section's authoring in `55ed974`, not via any later edit.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan.md` §Blocked obligations, disambiguate the two "(a)" references in the `V9l` bullet so the shared bare letter no longer reads as self-referential. Give each reference a descriptor that distinguishes the open question from its resolution owner — for example, render the spec-question reference as "host-prerequisites **degraded-state** clause (a)" (anchor `#degraded-state-host-prerequisites`) and the resolution-owner reference as "version-bump editorial-review **checklist** item (a)" (anchor `#bump-checklist-instance-survival`), and keep both anchors so the link targets are unchanged. The edit is confined to the `V9l` paragraph in plan.md §Blocked obligations; the spec anchors and the `V18c` Deps edge that actually enforces the block are not touched.

## Consequence

**Severity:** cosmetic

A scanning reader may conflate the spec open question (host-prerequisites clause (a)) with its resolution owner (version-bump checklist item (a)), reading the fallback sentence as circular. Pickability is unaffected — `V9l`'s block is enforced by its `V18c` Deps edge, not by this prose — so no implementer is misdirected; the cost is reader friction in the blocked-status record.
