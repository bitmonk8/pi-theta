## Finding

`V4f` (no-rollback guarantee) defers the *live-surface* re-assertion of its `ERR-13` completed-callee-finality vector to three production-surface leaves, which it names as "delegated live-carrier witnesses": `V14a` (code-side tool call), `V13c` (query tool loop), and `V15a` (`invoke` core). Each of those impl leaves carries a Tests bullet stating the witness — e.g. V13c's "a query-tool-loop callee driven to completion on the live `V13c` surface, then a downstream `?`/panic/cancel fired, leaves the completed callee's side effect in place with no compensating turn injected".

The witness is, however, wired into neither half of the project's TDD gate. The paired test-first leaves `V13c-T`, `V14a-T`, and `V15a-T` do not carry the `ERR-13` bullet, so the per-phase TDD ritual's red-first obligation (the `<id>-T` task must hold the failing test before the impl task) never covers this assertion. And none of the three impl leaves' **Ships when** conditions names the finality assertion: V13c's Ships-when stops at the tool loop / forced-respond / `max_rounds:0` / exhaustion paths; V14a's at the argument codes, `CodeToolError` enum, return-type rows, `.loom`-callable failure, and the five envelope-lowering mechanics; V15a's at the cross-mode matrix, `INV-1` containment, the static-resolution walk, and final-value propagation.

The witness therefore sits in an impl-leaf Tests block that no `-T` leaf seeds red and no Ships-when condition gates. A reviewer can observe every Ships-when condition green and tag `V13c`/`V14a`/`V15a` complete with the finality assertion never written. Although the finding heading names only V13c/V14a, the defect is identical at V15a — the same enriching commit introduced all three witnesses the same way — so a complete fix addresses the pattern across all three carriers.

That commit added the `ERR-13` live-carrier witness bullet to the Tests blocks of `V13c`, `V14a`, and `V15a` and rewrote `V4f` to delegate the live-surface re-assertion to those three production surfaces, but it did not add the corresponding bullet to the paired `-T` leaves nor name the assertion in any impl-leaf Ships-when. The witness thus entered the corpus red-first-unbacked and ungated in the same edit that created it; the term is absent from `V13c-T` / `V14a-T` / `V15a-T` at every revision.

## Solution Space

**Shape:** single

### Recommendation

Thread the existing `ERR-13` live-carrier witness through both ends of the TDD gate for each of the three carrier leaves:

- Add the `ERR-13` delegated live-carrier witness bullet to the **Tests** field of `V13c-T-query-tool-loop.md`, `V14a-T-tool-calls.md`, and `V15a-T-invocation-core.md`, matching the surface-specific wording already present in each paired impl leaf (query-tool-loop callee for V13c, code-side tool call for V14a, `invoke` child for V15a) so each `-T` leaf carries the failing test red-first per the per-phase TDD ritual.
- Add a clause naming the `ERR-13` completed-callee-finality assertion to the **Ships when** field of `V13c-query-tool-loop.md`, `V14a-tool-calls.md`, and `V15a-invocation-core.md` — e.g. "asserts the completed callee's side effect persists with no compensating turn injected after a downstream `?`/panic/cancel on the live surface" — so the witness is an observable gate condition, not an ungated Tests-block entry.

The `-T` leaves' existing "fail red for the intended reason" Ships-when wording already covers the new bullet, so no further `-T` Ships-when edit is needed. If the related V14a atomicity split lands first (see Related Findings), the V14a witness wiring lands on whichever post-split leaf owns the envelope-lowering / return-type concerns.

## Consequence

**Severity:** correctness

The whole point of the delegation — confirming `ERR-13` finality (a completed callee's side effect survives a downstream terminal event with no compensating turn) on a *live* production surface rather than only through the `H4a` session double — goes unenforced: two reasonable implementers diverge, one writing the live-carrier test and one omitting it, and the gate catches neither. `ERR-13`'s authoring-site witness still runs inside `V4f`, so the spec rule is not entirely untested, but the live-surface confirmation that `V4f` explicitly hands off ships with no gated assertion on any of the three carriers.
