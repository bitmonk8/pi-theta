## Problem

The single per-slash-invocation structured-output completion the binder issues through `@earendil-works/pi-ai`'s `complete()` is named three different ways across the spec — *binder inference call*, *binder LLM call* / *binder's LLM call*, and *binder `complete()` call* — with no equivalence statement linking them. The load-bearing pin is *binder inference call*: it is the page title, heading, and `id="binder-inference-call"` anchor at `pi-integration-contract/binder-inference.md`. A reader hitting HC3-d's *"3 binder LLM calls"*, the cancellation checkpoint set, or the `options.onResponse` bullet has nothing in the immediate text confirming these denote the same event, and the third form is coined inside the page that owns the canonical name.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `7cb4cc3` (2026-05-04, "Binder LLM call has no cancellation checkpoint") · `fb3474f` (2026-06-04, "Binder inference call — no pi-ai entry point pinned") · `44782e9` (2026-06-06, "TransportError catch-all / pi-ai provider-error surface")
**History:** Each name entered the corpus in a separate, self-contained spec-fix commit and none reconciled against the names already in place. `7cb4cc3` added the *binder's LLM call* checkpoint to `cancellation.md`'s *Granularity* list (and the cancelled-binder *Surfacing* arm), with HC3-d picking up *"3 binder LLM calls"* in the same family of edits — so *binder LLM call* settled in at the ceiling and cancellation pages before any per-call entry point was pinned. `fb3474f` later created `pi-integration-contract.md`'s *Binder inference call* anchor + heading as the load-bearing entry-point pin, coining the second term without renaming the first; the determinism page's retry-budget paragraph was rewritten to link to the new anchor but kept HC3-d's *"3 binder LLM calls"* wording verbatim, locking the two-name surface in. `44782e9` then added the `options.onResponse` bullet inside the now-canonical `binder-inference.md`, phrased as *"every binder `complete()` call"* — coining the third term inside the page that owns the canonical one, again without reconciliation. None of the three commits' subject lines or diffs flag the prior names; the drift is the cumulative residue of three independently scoped fixes against a shared concept that had no single naming owner.

## Solution approach

Standardise on `binder inference call` as the canonical noun phrase for the per-slash-invocation `complete()` invocation. Rename the surviving *binder LLM call* / *binder's LLM call* / *binder `complete()` call* surfaces to it: HC3-d's label and the `params`-validation row in `hard-ceilings/ceilings-3-and-4.md`, the `id="per-invocation-retry-budget"` paragraph in `binder/determinism-cancellation-failure.md`, the *Granularity* bullet and cancelled-binder *Surfacing* arm in `cancellation.md`, the checkpoint enumeration in `pi-integration-contract/host-interfaces-services.md`, and the `options.onResponse` bullet in `pi-integration-contract/binder-inference.md`.

## Solution constraints

- Preserve the literal `complete()` token wherever it names the function symbol or contract (the lead-in entry-point pin, the `onResponse`-fires-multiple-times language) rather than the per-invocation call event — only the call-event noun phrase is being reconciled.
- Out of scope: plan-side files (`coverage-matrix.md` rows, `V11j`/`V17c` leaves) — they re-render naturally and are not part of the spec edit.
