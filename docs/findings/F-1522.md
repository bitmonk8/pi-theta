## Finding

The single per-slash-invocation structured-output completion that the binder issues through `@earendil-works/pi-ai`'s `complete()` is named three different ways across the spec, with no equivalence statement linking them:

- **`binder inference call`** — the load-bearing pin. `pi-integration-contract/binder-inference.md` is titled *Binder inference* and carries the `<a id="binder-inference-call"></a>` anchor with the bold lead-in *"**Binder inference call.**"*. `binder/determinism-cancellation-failure.md` cross-references the call by exactly this name (`[binder inference call](../pi-integration-contract/binder-inference.md#binder-inference-call)`) in both the Cancellation paragraph and the retry-budget paragraph.
- **`binder LLM call` / `binder's LLM call`** — used by `hard-ceilings/ceilings-3-and-4.md` HC3-d (*"3 binder LLM calls per slash invocation"*), by the same wording in `binder/determinism-cancellation-failure.md`'s per-invocation-retry-budget paragraph, by `cancellation.md`'s *Granularity* bullet (*"immediately before issuing the slash-command argument binder's LLM call"*), and by `pi-integration-contract/host-interfaces-services.md`'s checkpoint enumeration (*"and the binder LLM call"*). The plan picks the term up at `coverage-matrix.md` row `cka-47` and at `V17c-checkpoint-granularity.md`'s checkpoint-site description.
- **`binder `complete()` call`** — `pi-integration-contract/binder-inference.md`'s `options.onResponse` bullet says *"the runtime registers it on every binder `complete()` call"* — i.e. a third noun-phrase form coined *inside the page that owns the canonical name*, with no signal that it denotes the same event.

Three concrete consequences fall out of the drift. (1) A reader who hits HC3-d's *"3 binder LLM calls"* and the adjacent finding's ambiguity about whether the budget counts `complete()` invocations or HTTP attempts (see *Related findings*) has nothing in the immediate text to tell them HC3-d's "binder LLM call" is the same event as `binder-inference.md`'s "binder inference call" — the only way to confirm it is to follow the link in the determinism page's retry-budget paragraph, which in turn equates the two only by reference. (2) The `binder-inference-call` anchor at `binder-inference.md` is the inbound link target for several cross-shard cites; a future reader landing there finds the page's own `onResponse` bullet using a different noun phrase from the heading they just clicked. (3) `V17c-T-checkpoint-granularity.md`'s test description cites *"the binder's LLM call (the binder-inference site `V9j` introduces)"* — leaf prose that has to reconcile two names to point at one site, with the reconciliation left implicit.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `7cb4cc3` (2026-05-04, "Binder LLM call has no cancellation checkpoint") · `fb3474f` (2026-06-04, "Binder inference call — no pi-ai entry point pinned") · `44782e9` (2026-06-06, "TransportError catch-all / pi-ai provider-error surface")
**History:** Each name entered the corpus in a separate, self-contained spec-fix commit and none reconciled against the names already in place. `7cb4cc3` added the *binder's LLM call* checkpoint to `cancellation.md`'s *Granularity* list (and the cancelled-binder *Surfacing* arm), with HC3-d picking up *"3 binder LLM calls"* in the same family of edits — so *binder LLM call* settled in at the ceiling and cancellation pages before any per-call entry point was pinned. `fb3474f` later created `pi-integration-contract.md`'s *Binder inference call* anchor + heading as the load-bearing entry-point pin, coining the second term without renaming the first; the determinism page's retry-budget paragraph was rewritten to link to the new anchor but kept HC3-d's *"3 binder LLM calls"* wording verbatim, locking the two-name surface in. `44782e9` then added the `options.onResponse` bullet inside the now-canonical `binder-inference.md`, phrased as *"every binder `complete()` call"* — coining the third term inside the page that owns the canonical one, again without reconciliation. None of the three commits' subject lines or diffs flag the prior names; the drift is the cumulative residue of three independently scoped fixes against a shared concept that had no single naming owner.

## Solution Space

**Shape:** single

### Recommendation

Standardise on **`binder inference call`** as the canonical noun phrase for the per-slash-invocation `complete()` invocation, and rewrite the surviving *binder LLM call* / *binder's LLM call* / *binder `complete()` call* surfaces to it. That term is already the load-bearing pin: it is the page title, section heading, and `binder-inference-call` anchor at `pi-integration-contract/binder-inference.md`, and is the cross-page link text the determinism page already uses. The mechanical edits are:

- `hard-ceilings/ceilings-3-and-4.md` — HC3-d: *"3 binder LLM calls per slash invocation (1 initial attempt + 1 transport-class retry + 1 malformed-envelope-class retry)"* → *"3 binder inference calls per slash invocation (…)"*. Same rename in the `params`-validation row of the per-boundary table (*"The binder LLM call that produced the depth-violating envelope still counts toward HC3-d's worst-case 3-call budget …"*).
- `binder/determinism-cancellation-failure.md` — retry-budget paragraph: *"the runtime MUST issue at most **3** binder LLM calls per slash invocation"* → *"the runtime MUST issue at most **3** binder inference calls per slash invocation"*. The paragraph's existing link to `[binder inference call](../pi-integration-contract/binder-inference.md#binder-inference-call)` already supplies the cross-reference at first mention; no new link is needed.
- `cancellation.md` — *Granularity* bullet: *"immediately before issuing the slash-command argument binder's LLM call"* → *"immediately before issuing the slash-command argument binder's inference call"*. Cancelled-binder *Surfacing* arm: *"abort observed before or during the binder's LLM call"* → *"… before or during the binder's inference call"*.
- `pi-integration-contract/host-interfaces-services.md` — checkpoint enumeration: *"and the binder LLM call (the binder checkpoint fires even though …)"* → *"and the binder inference call (…)"*.
- `pi-integration-contract/binder-inference.md` — `options.onResponse` bullet: *"the runtime registers it on every binder `complete()` call"* → *"the runtime registers it on every binder inference call"*. The bullet's later phrase *"when `onResponse` fires more than once before `complete()` resolves"* stays as-is — there *"`complete()`"* names the function symbol, not the call event, so it remains the right token. Apply the same distinction wherever the page mentions `complete()` as a symbol vs. the call event.
- Plan-side follow-up (not part of the spec edit; the leaves above flip naturally on re-render): rename in `coverage-matrix.md` rows `cka-34`, `cka-43`, `cka-47` and in `V11j`, `V11j-T`, `V17c`, `V17c-T` to match.

Implementer edge cases. (i) Do *not* delete the literal `complete()` token where it names the function — `binder-inference.md`'s lead-in pin (*"through `@earendil-works/pi-ai`'s `complete` free function"*), the `onResponse`-fires-multiple-times language, and `conversation-drive.md`'s *`complete()` retry-timing and cancellation behavioural presupposition* anchor all describe the symbol/contract, not the per-invocation call event, and the rule under repair is the noun phrase for the *event*. (ii) HC3-d's anchor `<a id="hc3-d"></a>` stays; only the prose flips. (iii) An optional polish — adding a *binder inference call* entry to `glossary.md` pointing at the `binder-inference-call` anchor — is consistent with the existing *binder* / *binder model* entries and would close the naming surface, but is not load-bearing for this finding.

## Consequence

**Severity:** cosmetic

The drift does not create a divergence between implementations — every reference, followed transitively, resolves to the same `complete()` invocation — but it forces every reader of HC3-d, the cancellation checkpoint set, and the `onResponse` bullet to re-prove that *binder LLM call* ≡ *binder inference call* ≡ *binder `complete()` call*. The cost compounds with the adjacent HC3-d budget-counting ambiguity, where readers most need a single unambiguous noun phrase for "the thing being counted."
