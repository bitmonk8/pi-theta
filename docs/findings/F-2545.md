## Finding

GOV-16 introduces *stable inline label* then uses the bare "inline label" throughout without declaring the abbreviation.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:**
- Stem collision (vector 1): `dual-form layout` predates the collision (introduced `ce1af28`, 2026-05-06, "resolve GOV-1 leaves anchor placement ambiguous"); the colliding `dual-anchor convention` / `dual-anchored` terminology entered later with the V1→loom 1.0 rename cluster (`e23f5ba` → `a4feafb` → `0c0434b`, 2026-05-27, GOV-25–GOV-29 authoring). The collision arose when the later family reused the `dual-` stem already held by GOV-1.
- "Ceilings 3 and 4" misnomer (vector 2): `f5e89f4` (2026-06-04, "enforce 100KB hard / 30KB recommended size cap on spec set"), which split the hard-ceilings page into the subdirectory+index layout and assigned the title/filename `ceilings-3-and-4.md` to the page that already carried the four-ceiling CIO interaction order and set-level invariants.
- GOV-3 missing short name (vector 3): present since GOV-3's inception relative to the parenthetical-short-name convention; peer REQ-IDs acquired short names progressively (e.g. the prefix-table rewrite `27ea061`, 2026-05-06) without GOV-3 ever being retrofitted.
- "stable inline label" abbreviation (vector 4): `c4ad105` (2026-05-08, "resolve HC3-a–HC3-e and NOCEIL-N labels escape GOV-3/GOV-8"), the commit that authored GOV-16 and introduced both the formal term and the undeclared bare form.

**History:** The four vectors entered through four unrelated commits across 2026-05 to 2026-06; there is no single introducing commit. The cluster is an accretion of independent naming choices, not a regression from one change.

## Solution Space

**Shape:** single

### Recommendation

Declare the short form once at first use in GOV-16 (`stable-inline-labels.md`), e.g. "a *stable inline label* (hereafter *inline label*)", since the bare form is already pervasive; alternatively standardise on the full term. Do not change the `## Stable inline labels` heading or the `gov-16` anchor.

## Consequence

**Severity:** advisory

A reader can conflate the two `dual-` conventions or look for the four-ceiling interaction order under a page named for only two ceilings, and the GOV-3 / inline-label inconsistencies erode the otherwise-uniform identifier conventions. No runtime behaviour or testable obligation is wrong; implementers can still produce a working system.
