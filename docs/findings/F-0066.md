## Finding

`leaf-template.md`'s **Ships when** bullet lists `"running \`pi /<name> …\` in a real Pi session produces …"` as one of the three canonical externally-observable exit-gate forms a leaf author may copy. The template never states that such a session exists, how it is stood up, or which leaf provides it.

In practice no leaf has a runnable *real* Pi session at exit-gate time. Every automated exit gate in the corpus runs against the **in-process Pi session double** built by the `H4a` end-to-end harness: `H7a`'s Ships-when ("…through the `H4a` harness against the in-process Pi session double"), `M`/`M-T`, and `V18d`'s runtime-evidence gate all drive that double, and `H4a` itself states the harness "deliberately avoids a live Pi session." The only place a live Pi host appears is `H4a`'s **manual real-host smoke run** — a contributor obligation gated on the `host-prerequisites.md` live-host precondition (Pi install at the pinned SDK, a structured-output-capable binder model, credentials), not an automated leaf exit gate and not observable under `npm test`.

A leaf author copying the template phrasing verbatim would write a Ships-when that cannot be observed in CI, because no provisioned real Pi session backs it. The template's third example contradicts the double-based footing every existing leaf actually uses.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/leaf-template.md`, edit the **Ships when** bullet's middle example so it names the runnable host the corpus actually provides for automated exit-gate observation. Strike the bare `"running \`pi /<name> …\` in a real Pi session produces …"` and replace it with an example that points at the `H4a` end-to-end harness and its in-process Pi session double — e.g. `"dispatching \`pi /<name> …\` through the \`H4a\` end-to-end harness against the in-process Pi session double produces …"`.

Add one clause to the same bullet stating that a *live* Pi host is not available to automated leaf exit gates: it exists only via `H4a`'s manual real-host smoke run (a contributor obligation under its live-host precondition), so a leaf's Ships-when MUST be observable against the harness double, not a real session. Cross-reference `H4a` for both the harness/double and the manual-smoke distinction.

Keep `conventions.md` read-only for this fix — the "real Pi session" example lives only in the template; conventions.md's Ships-when bullet states only "A concrete, externally observable change" and does not repeat the example.

## Consequence

**Severity:** advisory

A leaf author who copies the template's "real Pi session" example writes an exit gate that no automated harness can observe (CI runs against the H4a in-process double; a live Pi host exists only in H4a's manual smoke). The author either has to silently reinterpret it as "the H4a harness double" or stalls on an unsatisfiable gate. Existing leaves already use the double-based phrasing, so the defect is a guidance gap in the template rather than a broken leaf today.
