## Problem

`V13g`'s **Adds** names two substantive properties for the runtime discard-observability event: it preserves a discarded `Err`, and it lands on the operator-facing always-log channel. The Tests bullet in `V13g-T` — carried verbatim into `V13g`'s Ships-when — asserts only that the event fires on an explicit `Err` discard. Neither the payload (the discarded `Err`'s `kind`/`message`/`discard_site`, per `runtime-event-channel.md`) nor the channel routing (the group-A `loom-system-note` always-log emission) is witnessed, so an implementation that fires a bare or wrong-payload event on any channel passes the gate. There is also no negative arm asserting that a path which does not discard an `Err` emits no event, leaving over-emission undetected.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commit:** `c6a664e` (pi-loom plan: build/update plan for spec.md + review, 2026-06-10)
**History:** The discard-observability test obligation entered the corpus in `c6a664e` inside `V13a-T`/`V13a` as "`discarded-query-result` (with its runtime event) fire" — firing-only from the start. `6698d91` (2026-06-16) split the parse diagnostic from the runtime event and expanded the bullet to "the runtime discard-observability event fires on an explicit `Err` discard", but added no payload/channel/negative assertions. `4a04570` (2026-06-17) split `V13g`/`V13g-T` out of `V13a` and carried the same firing-only bullet forward verbatim. The under-assertion has been present at every revision of this test obligation; no commit ever introduced the payload-preservation or channel assertions.

## Solution approach

Strengthen the discard-observability Tests bullet in `V13g-T` (and the matching Ships-when clause in `V13g`) so the event's two Adds-named properties are asserted, not just its firing: that the emitted `RuntimeEvent` preserves the discarded `Err`'s `kind`/`message` and stamps `discard_site`, and that it routes through the operator-facing always-log (`loom-system-note`) channel. Add a negative arm asserting that a path which does not discard an `Err` emits no discard-observability event.

## Solution constraints

- The negative-arm trigger predicate (whether a discarded `Ok` emits nothing, versus firing on any discard) is owned by `V9d`'s trigger condition and the sibling finding on this leaf; phrase the negative arm against whichever predicate they settle on rather than encoding trigger semantics here.
- The spec is read-only for this fix; these are test-assertion additions in the `V13g`/`V13g-T` plan leaves only.
