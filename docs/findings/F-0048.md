## Finding

H6a's **Adds** carries a `**Rollback.**` paragraph asserting that recovery from a bad live-corpus flip is a single revert: reverting the activation commit returns H5a's closing gate to its seeded-fixture footing, so `main` stops reddening on live coverage later work is still landing, with "no data, schema, or registry migration to unwind." This is a recovery-correctness property the leaf relies on, but neither H6a's **Tests** bullets nor its **Ships when** condition assert it. The Tests/Ships-when surfaces witness only the forward direction — the live-corpus footing passes green and reddens on a removed mapping / removed citation, plus the manual smoke-evidence record — and say nothing about the revert restoring the green seeded-fixture footing.

The property is structurally sound (the revert simply removes the flip, after which H5a's already-tested seeded gate runs green again), so this is a witnessing gap rather than a contradiction. But as written the rollback claim is unfalsifiable from the leaf's exit gate: an implementer landing H6a has no acceptance criterion that exercises the revert path, so a regression in the seeded-fixture footing that makes revert non-green would not be caught at H6a.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H6a-live-corpus-activation.md`, add a verification clause that ties the documented rollback to an observable: state — in **Ships when** (or as a Tests bullet) — that reverting H6a's activation commit re-runs H5a's seeded-fixture closing-gate suite green, naming H5a's existing seeded-fixture gate as the witness so no new gate machinery is introduced. The clause should make the green-after-revert footing the observable that closes the `**Rollback.**` claim, consistent with how the existing removed-mapping / removed-citation Tests bullets witness the forward live-corpus failure modes. The `**Rollback.**` prose in **Adds** stays as the rationale; the new clause supplies its acceptance criterion. Edge case for the implementer: the witness must reconcile the live corpus exclusive of the dedicated seeded test-fixtures root (per H5a's Adds), so the revert check observes H5a's seeded gate, not a residual live-corpus reconciliation.

## Consequence

**Severity:** advisory

The rollback recovery property is documented but not observable from H6a's exit gate, so the leaf can be tagged complete without ever exercising the revert path. An implementer cannot distinguish "rollback works" from "rollback is asserted" at the gate; a latent regression in H5a's seeded-fixture footing that breaks the green-after-revert assumption would surface only in production rather than at H6a.
