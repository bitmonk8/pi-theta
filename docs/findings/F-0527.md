## Finding

`V9e`'s **Adds** field enumerates six behaviours the `ActiveInvocationRegistry` leaf owns: the five-field entry, **the canonical-name resolver (three insertion sites)**, **the setup wrap**, the per-entry `disposeBarrier`, insertion-order iteration, and `invocationId` allocation via `IdSource`. Its three **Tests** bullets — replicated verbatim in the paired `V9e-T` tests task — assert only insertion-order iteration, `disposeBarrier` settlement, and `invocationId`-from-`IdSource.newInvocationId()`. The canonical-name resolver and the dispatch-site setup wrap have no paired asserting test, so neither is TDD-driven and neither reddens on regression.

Both untested behaviours are load-bearing in the spec (`active-invocation-registry.md`). The **canonical-name resolver** is the rule that the entry's `loom` field is *read* from the resolved catalog entry's slash-name field (the final-derived name — post-`pi.looms` mapping, post-`as` rename, stem verbatim, no leading `/`) at each of the three insertion sites and is **not** re-derived at the insertion site from the authored identifier; the spec calls out that two implementations converging on different canonical forms would split operator dedup buckets on `details.event.loom`. The **Dispatch-site setup wrap** carries an explicit normative MUST: each insertion site MUST wrap the pre-evaluation setup sequence (`AbortController` construction, `Promise.withResolvers()`, the registry `Set.add`, the forwarding-listener attach, and — subagent mode — the awaited `createAgentSession(...)` plus its abort-listener attach) in a `try`/`catch` whose `catch` routes the throw through the runtime-defect surface, with the `Set.add` insertion happening before any awaitable work.

A self-consistent-but-wrong resolver (wrong canonical form) or a missing/incorrect setup wrap (throw not routed to the runtime-defect surface, or insertion ordered after awaitable work) would pass `V9e` green today, because no test exercises either path.

## Solution Space

**Shape:** single

### Recommendation

Add two paired test bullets to `V9e-T`'s **Tests** field (the failing tests land first per the TDD ritual) and mirror the same two bullets into `V9e`'s **Tests** field:

1. **Canonical-name resolver bullet.** Assert that, at each of the three insertion sites (slash-command handler entry, `tool.execute(...)` adapter entry, `invoke` spawn-site entry), the entry's `loom` field equals the resolved catalog entry's slash-name field in final-derived form (post-`pi.looms` mapping, post-`as` rename, stem verbatim, no leading `/`) and is not re-derived from the authored call-site identifier. Because the registry symbol is internal (per the spec's "The registry name is internal" rule), assert through an observable channel — e.g. the `details.event.loom` value surfaced in the per-invocation `finally` emission, or a probe/audit seam exposing the entry's `loom` field — rather than on the registry symbol directly.

2. **Setup-wrap bullet.** Assert that the pre-evaluation setup sequence runs inside the `try`/`catch` with the registry `Set.add` insertion occurring before any awaitable body work, and that a throw raised within the setup sequence is routed through the runtime-defect surface (`loom/runtime/internal-error`) per [Errors and Results — Runtime panics](../spec_topics/errors-and-results/error-model.md#runtime-panics), with no registry-entry leak (throw before `Set.add` leaves nothing to remove; throw after insertion is cleaned by the per-invocation `finally`).

Optionally extend `V9e`'s **Ships when** to reference these two observables alongside the existing teardown/insertion-order condition. Cite only existing leaf IDs; no new leaf is required. The fix is internal to the `V9e`/`V9e-T` pair — do not edit the spec (the behaviours are already normative there) and do not edit `coverage-matrix.md` (cka-26 remains the un-anchored-MUST residue row).

## Consequence

**Severity:** advisory

The implementer is told (via `Adds`) to build the canonical-name resolver and the setup wrap, but nothing reds out if either is built wrong or later regresses: a divergent canonical-name form ships silently (splitting operator dedup buckets on `details.event.loom`), and the setup-wrap MUST — try/catch routing to the runtime-defect surface, insertion before awaitable work — is unverified. The behaviours remain implementable from the spec, but two of `V9e`'s six enumerated `Adds` behaviours are outside the leaf's red-test coverage, contrary to the per-phase TDD ritual.
