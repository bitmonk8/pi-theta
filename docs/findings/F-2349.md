## Problem

The *Numbered REQ-IDs* preamble of `docs/plan_topics/coverage-matrix.md` carves out *retired interior IDs* from the mapped REQ-ID set, naming "the prefix's `## Retired REQ-IDs` table" as the authority for which interior IDs are retired, but supplies no cross-reference to where that table lives or to the rule that governs it. The governing rule is GOV-8 §Per-ID retirements (`req-id-prefix-table-active-b.md#gov-8`), which states the table is a trailing `## Retired REQ-IDs` section on each non-narrative page. An implementer building the `H5a` per-prefix-numbering-hole / citing-test reconciliation must locate that table per prefix to exclude retired interior IDs from the mapped set, and with the location and its governing rule un-cited, must rediscover both by corpus search.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commits:** `7103dca`
**History:** The retired-interior-ID carve-out sentence — including the un-anchored reference to "the prefix's `## Retired REQ-IDs` table" — was first authored in `7103dca` (2026-06-18, "pi-loom plan: resolve \"range expansion excludes retired interior REQ-ID from citing-test gate\"") as a one-line addition to the *Numbered REQ-IDs* preamble of `docs/plan_topics/coverage-matrix.md` (`git log -S 'Retired REQ-IDs\` table accounts for' -- docs/plan_topics/coverage-matrix.md` reports only `7103dca`). The cross-reference was missing at that authoring; no later commit has touched the sentence. The defect is coeval with the carve-out feature it documents.

## Solution approach

In the *Numbered REQ-IDs* preamble of `docs/plan_topics/coverage-matrix.md`, where the carve-out sentence first names "the prefix's `## Retired REQ-IDs` table", add a cross-reference resolving that table to the GOV-8 §Per-ID retirements rule at `../spec_topics/governance/req-id-prefix-table-active-b.md#gov-8`. Match the existing sibling governance-link relative-path form already used in this file.
