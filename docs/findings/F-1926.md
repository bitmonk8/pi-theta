## Problem

Both interleave notes in `docs/plan.md` — under the `### V9 — Extension host integration` heading and under `### V11 — Binder` — assert that `V11a` (Binder-model resolution) is a prerequisite of `V9c`, `V9i`, and `V9j`, describing the seam as `V9b → V11a → V9c`/`V9i`/`V9j`. The authoritative leaf `Deps` fields disagree: only `V9j` lists `V11a`; `V9c` and `V9i` neither name it nor reach it transitively, and neither leaf consumes binder-model resolution. The narrative therefore names `V9c`/`V9i` as gated on `V11a` when the leaf graph says they are independent, misleading on first read and contradicting the authoritative signal.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `b9b427d` (2026-06-10, "pi-loom plan: resolve \"slice-ordering narrative overpromises DAG order\""); `8490e2c` (2026-06-25, "pi-loom spec-plan: fix F-1327 — relabel V9i pre-spawn model guard, swap V11a->V6a dep")
**History:** `b9b427d` authored both interleave notes, introducing the `V11a … is a prerequisite of \`V9c\`/\`V9i\`/\`V9j\`` claim as added lines. `git log -S 'V11a'` over `docs/plan_topics/V9c-conversation-drive.md` returns no commits, so `V9c` never carried `V11a` in its `Deps` — the `V9c` strand of the claim was wrong at authoring. The `V9i` strand became wrong later: `8490e2c` (F-1327) swapped `V9i`'s `V11a` dependency to `V6a` when relabelling its pre-spawn model guard, but did not propagate that correction into the interleave notes, leaving the `V9i` reference stale. The current full defect is the join of both: an overbroad claim at authoring (`V9c`) plus an un-propagated dep correction (`V9i`).

## Solution approach

Narrow both interleave notes in `docs/plan.md` so the only V9 leaf named as gated on `V11a` is `V9j`, making the documented seam `V9b → V11a → V9j`. Drop `V9c` and `V9i` from the prerequisite list in both notes, reconciling the narrative to the leaf `Deps`. Leave the unrelated clauses of each note (the `V9h`/`V9g`/`V18c`/`V9l` material and the `V11e`/`V11h` and "Sequence by Deps" material) intact.

## Solution constraints

- Do not add `V11a` to the `Deps` of `V9c`/`V9i` (or their `-T` leaves) — those leaves are correct as-is; only the narrative is wrong.
- The spec is read-only for this fix.
