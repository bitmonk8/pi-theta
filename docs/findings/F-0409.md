## Finding

The closing parenthetical that follows `V17a`'s **Tests.** bullets reads
`_(The forwarding-listener throw-trap — a distinctly complex, independently-verifiable sub-mechanism — is owned by [V17b](./V17b-forwarding-listener-throw-trap.md).)_`.
The em-dash-delimited middle clause — "a distinctly complex,
independently-verifiable sub-mechanism" — is rationale for *why* the
throw-trap was carved into its own leaf. It imposes no implementation
constraint on `V17a`; the only load-bearing content of the note is the
scope delimiter: the forwarding-listener throw-trap is out of `V17a`'s
scope and is owned by `V17b`.

The paired test leaf `V17a-T` carries the leaner form of the same note —
`_(The forwarding-listener throw-trap is owned by `V17b`.)_` — so the
impl leaf and its `-T` twin state the same scope boundary in two
different wordings, the impl side carrying surplus split-decision prose.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** cb8737a — pi-loom plan: resolve "Systemic leaf over-bundling across the plan corpus" (2026-06-12, Thomas Andersen)
**History:** Commit cb8737a split the forwarding-listener throw-trap out of `V17a` into the new `V17b` leaf, deleting `V17a`'s former throw-trap Tests bullet and Ships-when arm and replacing them with the closing parenthetical. The same commit introduced the em-dash-delimited "distinctly complex, independently-verifiable sub-mechanism" rationale clause in `V17a` while giving `V17a-T` the leaner phrasing, creating the surplus-prose / wording-divergence defect in one change.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V17a-cancellation-core.md`, replace the closing
parenthetical that follows the Tests bullets with the form already used
in `V17a-T`, striking the em-dash-delimited clause
`— a distinctly complex, independently-verifiable sub-mechanism —` and
retaining the `V17b` ownership/scope delimiter, i.e.
`_(The forwarding-listener throw-trap is owned by [`V17b`](./V17b-forwarding-listener-throw-trap.md).)_`.
The spec is read-only for this fix; no other leaf is touched.

## Consequence

**Severity:** cosmetic

No implementer behaviour changes either way; the clause is
design-rationale prose rather than an acceptance constraint, and it
diverges in wording from the identical scope note on the paired `V17a-T`
leaf. The cost is reader noise and impl/`-T` wording drift.
