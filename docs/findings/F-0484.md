## Problem

`H6a`'s single gate-machinery flip reconfigures `H5c`'s broad-catch (`no-broad-catch` allow-list) closing-gate arm from its seeded-fixture footing to its live-corpus footing — `H5c`'s **Adds** explicitly enrolls itself in that flip. But `H6a`'s **Deps.** field is `H5b, H7a`, and `H5c` is named in no leaf's **Deps.** anywhere in the plan. Because `H5c` closes no spec REQ-ID, it is not a member of `H5b`'s coverage-producing set and cannot be reached transitively through `H5b` (the same reason `H7a` is pinned directly to `H6a`). The order holds only incidentally today, so `H6a` can be picked and shipped with `H5c` unbuilt — activating the release gate while the broad-catch arm is never flipped to its live-corpus footing.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** cb8737a
**History:** `H5c` was created in `cb8737a` (2026-06-12, "pi-loom plan: resolve 'Systemic leaf over-bundling across the plan corpus'"). At birth its **Adds** already declared "both footings are reconfigured by the single `H6a` gate-machinery flip", yet the same commit did not add `H5c` to `H6a`'s **Deps** (which was `H5b` at that point, becoming `H5b, H7a` one day earlier). On 2026-06-11 commit `1cab2ae` ("H6a consumes H7a's golden artifacts but no dependency edge") had added the `H7a` edge under precisely the governing principle here — a leaf that H6a reconfigures/consumes but which closes no REQ-ID, and so is not a member of `H5b`'s coverage-producing set, must be pinned directly in `H6a`'s **Deps**. `H5c`, split out the next day, matches that precedent but was overlooked. The omission has been present in `H5c` since its inception; `H6a` has never named `H5c`.

## Solution approach

Add `H5c` to `H6a`'s **Deps.** field so it reads `H5b, H5c, H7a`. Extend the existing Deps rationale parenthetical (the `_(This leaf depends on the warn-only canary …)_` block) to record the `H5c` edge alongside the `H7a` justification, noting that `H5c` closes no spec REQ-ID and so is pinned directly here rather than threaded through `H5b`.
