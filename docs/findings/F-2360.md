## Finding

Impl `V11h` declares and uses `V2c` (its value model supplies the `integer`-vs-`number` kind discriminator from the bound value's static type); the paired `V11h-T.Deps` (`V11g, V2d`) omits `V2c`, and `V2c` is not reachable transitively from `V11h-T`. If `V11h-T`'s fixtures construct or inspect value-model values the edge is genuinely undeclared.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V11h-T-argument-echo.md`, add `V2c` to the `**Deps.**` line if the tests-task consumes a `V2c` value-model symbol; otherwise record on `V11h` that `V2c` is an impl-only edge so the asymmetry reads as intentional.

## Consequence

**Severity:** advisory

A tests-task whose missing edge is only transitively satisfied (`V9g-T`, `V9h-T`) stays pickable today but turns into a compile/pickability failure the moment the intervening chain is re-routed, with no local signal of the dependence. The one-sided `H4b`/`V2c` edges (`V11j`, `V11h-T`) leave a reader unable to tell a deliberate test-only / impl-only dependency from an omission, and `V11h-T`'s `V2c` gap is not even transitively covered. No leaf is unpickable now, so implementers can still produce working leaves.
