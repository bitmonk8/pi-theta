## Finding

`V6a`'s **Adds** is a single dense paragraph that mixes three distinct concerns: (1) the YAML frontmatter parser itself (defaults, required `mode:`, model/`bind_*` resolution hooks, unknown-key tolerance); (2) the normative declaration that `V6a` defines the **model-reference-matcher injection seam** — the interface the parser's `model:` resolution hook calls; and (3) the shared resolution contract that `V11a` owns and that the concrete matcher must honour (exact-match resolution over `ctx.modelRegistry.getAvailable()` against `Model<Api>.id` / `Model<Api>.provider`), plus the note that `V9b` owns the production wiring point.

The seam declaration is a real design commitment — it is what lets `V6a` close `model:` resolution in-leaf while carrying no forward `Deps` edge onto the downstream binder machinery — but it is buried mid-paragraph inside the parser deliverable rather than presented as the separate interface declaration it is. A reader scanning the Adds for "what does this leaf own" has to disentangle the parser implementation from the seam-interface contract and from the inherited `V11a` contract clause.

This is an organisational defect, not a correctness one: the seam is exercised by the `loom/load/model-unresolved` Tests bullet (which drives the hook through the seam `V6a` defines), and the leaf's **Ships when** is focused on the parser. No structural leaf split is warranted; the concern is readability of the Adds.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V6a-frontmatter-contract.md`'s **Adds**, present the model-reference-matcher injection-seam declaration as a clause/element distinct from the YAML-parser deliverable, so the seam reads as the separate interface-declaration commitment it is rather than being interleaved with the parser description. Keep the existing seam content intact: that `V6a` defines the seam, that the leaf therefore carries no forward `Deps` edge onto the binder machinery, that the concrete matcher binds `V11a`'s shared exact-match resolution contract over `ctx.modelRegistry.getAvailable()` against `Model<Api>.id` / `Model<Api>.provider`, and that `V9b` owns the production wiring point. No structural leaf split, no change to **Deps**, **Tests**, or **Ships when**, and the `V11a`/`V9b` cross-references stay. The `model-unresolved` Tests bullet's "see Adds" reference remains valid because the seam content is retained, only reorganised.

## Consequence

**Severity:** cosmetic

The model-reference-matcher injection seam is a normative interface declaration but is embedded inside the parser-deliverable prose, so a reader cannot quickly see it as a distinct commitment of the leaf. No implementer is blocked or misled — the seam is named in bold and exercised in Tests — so the impact is confined to Adds readability.
