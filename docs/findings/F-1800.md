## Problem

The `Seam minimality (normative)` paragraph in the *Predicate split* section forbids specific internal handler-scoped state seams — `branchTakenWasClosedSet`, a per-sub-step success roll-up, a `!pinnedConstantReadOk` disjunct, a `"<unreadable>"` sentinel-equality check — whose observable consequences are already pinned by acceptance criteria (b)/(d)/(e). This elevates a non-observable implementation choice to a normative MUST NOT.

**Verdict:** introduced-by-commit
**Introducing commits:** 52b9101
**History:** The entire session-only degraded-state branch — the *Predicate split* / "Seam minimality (normative)" prose and the per-branch acceptance fixtures — was authored wholesale in `52b9101` (2026-05-11, "resolve T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state"). `git log -S` on `handler-scoped state seam beyond`, `captured-string triplet`, and `Predicate split` against the then-current `docs/spec_topics/pi-integration-contract.md` shows none of these tokens existed before `52b9101`; the internal-seam-prohibition MUST NOT and the acceptance list that omits a captured-value-vs-fresh-read fixture both entered the corpus in that single commit. The later split commit `f5e89f4` (2026-06-04) only condensed and relocated this content from `pi-integration-contract.md` into the standalone `session-only-degraded-state.md` page; it preserved both the prohibition and the fixture omission verbatim and so did not introduce the defect.

## Solution approach

Rewrite the `Seam minimality (normative)` paragraph to state the requirement observably: the diagnostic-emission and tag-transition predicates are evaluated from the handler-entry-captured `pinnedConstantReadOk` and `capturedEventReason` values, producing the per-branch outcomes acceptance (a)–(e) assert. Retain as a normative observable point that the discriminator reads the captured value rather than performing a fresh `event.reason` property access, since a fresh access bypasses the catch-arm classification and is non-conformant. Demote the enumerated forbidden-variable list to a non-normative implementation note.
