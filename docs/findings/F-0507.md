## Finding

`V4d`'s **Adds** asserts: "`ContextOverflowError` is the sole token-domain failure surface (NOCEIL-2): loom 1.0 imposes no per-query response-token cap and no cumulative token budget." The normative content behind that claim — the *only token-domain failure surface* assertion and the no-cap / no-budget rules — is owned by `NOCEIL-2` in `hard-ceilings/ceiling-invariants-and-audit.md`, which `V4d`'s **Spec** field does not link (the field cites only `queryerror-variants.md` and `error-model.md`).

`queryerror-variants.md` (which V4d does link) defines the `ContextOverflowError` wire shape but does not carry the "sole token-domain surface / no cap / no budget" assertion; that lives solely under `NOCEIL-2`. An implementer or auditor reading only V4d's linked Spec pages therefore cannot confirm the "sole" claim — i.e. cannot verify that no other token-budget gate is required of the leaf.

`coverage-matrix.md` records `V4d` as the distributed observable seam for `NOCEIL-2` (`NOCEIL-2 → V4d's ContextOverflowError / ERR-14/15/17`), so V4d genuinely owns NOCEIL-2's observable surface — which makes the missing link to the owning spec page a real traceability/clarity gap rather than incidental prose.

## Solution Space

**Shape:** single

### Recommendation

Add the NOCEIL-2 owning page to `V4d`'s **Spec** field in `docs/plan_topics/V4d-queryerror-variants.md` so the "sole token-domain failure surface (NOCEIL-2)" claim is backed by a followable link. Append a third link to the existing field:

```
[`../spec_topics/hard-ceilings/ceiling-invariants-and-audit.md`](../spec_topics/hard-ceilings/ceiling-invariants-and-audit.md#no-additional-ceilings)
```

Use the section anchor `#no-additional-ceilings` (the heading that contains the four `NOCEIL-*` inline labels); do not use `#noceil-2`, which is not a real anchor in that file (the `NOCEIL-2` label is an inline bold list item, not a heading). For parity, the same link may be added to `V4d-T`'s Spec field, though `V4d-T` does not itself reference NOCEIL-2 and is not strictly required.

## Consequence

**Severity:** advisory

V4d's load-bearing obligation is the `QueryError` variant wire shape, which its linked pages already cover, so a leaf can still ship correctly. But the unlinked NOCEIL-2 reference leaves the "sole token-domain failure surface" claim unverifiable from V4d's own Spec field: an implementer cannot confirm no additional token-budget gate is owed without chasing down the owning ceiling page by hand, and a self-review cannot trace the NOCEIL-2 seam V4d closes back to its source.
