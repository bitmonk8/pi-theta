## Problem
The first leaf that writes the status table destructively replaces the existing prose `## Status` section — a one-time non-idempotent rewrite with no dry-run, backup, or recovery note (recovery is git history, unstated).

_Suggested direction (lens hint):_ Add a one-line note that prior `## Status` prose is recoverable from git history (or that the first writer seeds the rows from it).
