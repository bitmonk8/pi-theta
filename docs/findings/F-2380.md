## Problem

`V9i` bundles `PIC-22` — the N≥2 parallel-spawn path that dispatches `createAgentSession` for all N callables before any returns via `Promise.all` over the per-call spawns — into the single-session subagent lifecycle leaf (spawn, pre-spawn model guard, dispose-in-`finally`, isolation; PIC-9 / PIC-23 / PIC-40..43). `PIC-22` is a distinct concurrency mechanism with its own test harness (a fake `AgentSession` whose `sendUserMessage` blocks until released) that builds on, but ships and tests independently of, the single-spawn lifecycle. The plan already draws this line for tool calls — the single code-side call (`V14a`) is separate from the parallel batch (`V14b`) — so bundling parallel-spawn dispatch into `V9i` is inconsistent and forces every `V9i` consumer to inherit the parallel path as a prerequisite they do not need.

An earlier 6d7661d ("resolve V9i bundles three separable concerns") split the swallowing-handler concern out to `V9o` but did not touch the parallel-spawn path, so the concurrency-mechanism atomicity break survived.

## Solution approach

Split the `PIC-22` parallel-spawn path out of `V9i` into its own paired leaf blocked on `V9i`, mirroring the `V14a` / `V14b` single-vs-parallel structure. Move the `PIC-22` Tests bullet out of `V9i.Tests` and `V9i-T.Tests` and the parallel-spawn clause out of `V9i.Adds` into the new leaf pair, striking `PIC-22` from the `V9i.Adds` anchor enumeration. Retarget the `coverage-matrix.md:51` `PIC-22` mapping from the `V9i` row to a new row owned by the new leaf, and link the new implementation leaf into `docs/plan.md`'s V9 slice listing.

## Solution constraints

- `V9i`'s remaining facets (PIC-9, PIC-23, PIC-40..43, isolation, RET final-value, ERR-8) are one coherent single-session lifecycle and stay in `V9i`; out of scope for further splitting.
- The new leaf must be a paired tests/implementation leaf per the plan-rules ritual: a `-T` tests leaf and a bare implementation leaf listing its `-T` partner in `Deps`, with a fresh `V9`-phase letter (no reused ID).
