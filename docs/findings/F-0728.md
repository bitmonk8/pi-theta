## Finding

The glossary pins two foundational concepts whose only distinguishing mark is letter case: `loom` (the file unit — what a slash command resolves to, what `invoke(...)` targets, what `mode:` is declared on) and `Loom` (the scripting language whose grammar/semantics the spec defines, shared by `.loom` and `.warp`). Case is a structurally unreliable disambiguator in CommonMark prose: every sentence-initial occurrence, every list-bullet-initial occurrence, and every heading word is capitalised by typographic convention, irrespective of which sense the author intends. The corpus already exhibits this collapse:

- `spec_topics/influences.md` opens "Loom borrows from two languages…" (sentence-initial; the language sense is intended and only inferable from the surrounding "Rust / TypeScript / Loom" enumeration) and immediately follows with the third bullet `**Original to loom**:` (lowercase mid-sentence) whose content — `schema` / `enum` declarations, the `@`...`` query primitive, frontmatter-declared mode, the `.loom`/`.warp` split, the unified callable set — is unambiguously language-level material that should bind to the `Loom` sense.
- `spec_topics/grammar.md` carries `## Loom literal sublanguage` (heading-initial capitalisation forced) and refers to "a legal Loom expression", "the Loom AST", "Loom distinguishes…" throughout — uses that are individually correct under the glossary rule but indistinguishable in form from the same words appearing sentence-initially when the file-unit sense is intended.
- `spec_topics/runtime-value-model.md` carries `| Loom type |` (table-cell capitalisation), "Loom values are represented…", and "Loom code observes `Result` only…" interleaved with body prose that uses lowercase `loom code` for the same referent.

A reader cannot recover the intended sense from a citation, a quoted fragment, or a heading-anchored cross-link, and `grep` over either spelling silently misses the other. The glossary's case-only convention therefore degrades to "rely on the surrounding paragraph", which is exactly what the rest of the glossary is structured to avoid for `caller`, `arm`, and the other ambiguous-when-bare terms — those carry distinct alternative names. `loom` / `Loom` is the only remaining overload that relies on case alone.

## Issue introduction

**Verdict:** present-since-inception (for the file-unit / language split as a *two-entry, case-distinguished* pair)

**Introducing commit:** `53b8970` — *pi-loom spec: resolve "loom / Loom / pi-loom — three referents, no glossary disambiguation"* (Tue May 5 22:33:43 2026 +0200). This commit added the `loom (file unit)` and `Loom (language)` entries to `spec_topics/glossary.md` in their current form, including the rule "Capitalised when used as a proper noun … the lowercase form is reserved for sense 1 above." The commit also retitled `overview.md`'s `### Scope of a Loom File` → `### Scope of a loom file` to enforce the new rule, demonstrating that the case-only disambiguator was load-bearing from its first commit.

**History:** Before `53b8970` the glossary contained no entries for either sense (`git show 53b8970^:spec_topics/glossary.md` lists `binder`, `callable set`, `coercion`, `loom-side name`, `prompt mode`, `Pi tool` / `.loom callable`, `top-level loom` — no `loom` or `Loom` entry). The two glossary entries and the rule that distinguishes them were introduced together by `53b8970`; the prose collision documented above all pre-dates the rule and was inherited rather than created, so the rule did not introduce the collision so much as fail to resolve it. No later commit has attempted a stronger disambiguator on either sense.

## Solution Space

**Shape:** single
**State:** shaped

### Recommendation

Rename the language sense to a heading-stable form. Adopt a fixed phrase for the language sense — `the Loom language` when standing alone, `Loom-language <noun>` (e.g. `Loom-language expression`, `Loom-language AST`, `Loom-language value`) when used adjectivally — and reserve bare `loom` / `Loom` for the file-unit sense in every position (including sentence-initial, heading-initial, and table-header). The disambiguator is then structural, not positional: it survives headings, sentence-initial use, table cells, and quoted fragments; `grep "Loom"` returns only language-sense hits and `grep "loom"` only file-unit-sense hits once the rename completes; and the pattern matches the existing two-name resolution used for `slash caller` / `invoke parent`. The asymmetry of frequency favours this direction — language-sense references concentrate in a few topic pages (`grammar.md`, `expressions.md`, `bindings.md`, `runtime-value-model.md`, `influences.md`), while file-unit references are spread across the whole corpus, so renaming the language sense moves the edit surface to the smaller, more localised set.

Spec edits:

- `docs/spec_topics/glossary.md`: rewrite the `Loom (language)` entry to record the rename; mark the bare `Loom` spelling as a non-canonical alias retained for one release under the existing alias machinery (GOV-20 / GOV-26 supply the prose-token-alias and dual-anchor patterns); add a usage note immediately under the `loom (file unit)` entry: "If a heading, table cell, or sentence-initial position would force capitalisation of the file-unit sense, restructure or pluralise (`looms`) to keep the bare word lowercase." Add an alias-retirement table row under GOV-20/GOV-26 recording bare `Loom` as a release-bounded alias for `Loom language`.
- `docs/spec_topics/influences.md`: "Loom borrows…" → "The Loom language borrows…"; `**Original to loom**:` → `**Original to the Loom language**:`; "Loom is much smaller than either parent" → "The Loom language is much smaller than either parent".
- `docs/spec_topics/grammar.md`: `## Loom literal sublanguage` → `## Loom-language literal sublanguage`; "a legal Loom expression" → "a legal Loom-language expression"; "the Loom AST" → "the Loom-language AST"; "Loom distinguishes…" → "The Loom language distinguishes…"; "Statements … are not expressions in Loom" → "Statements … are not expressions in the Loom language".
- `docs/spec_topics/runtime-value-model.md`: table header `| Loom type |` → `| Loom-language type |`; "Loom values are represented…" → "Loom-language values are represented…"; "Loom code observes `Result`…" → "Loom-language code observes `Result`…".
- `docs/spec_topics/expressions.md`, `bindings.md`, `lexical.md`: apply the same rule to each `Loom <verb>` / `Loom <noun>` site (a closed list under one editorial pass).

Edge cases the implementer must watch:

- The edit must be human-driven case-by-case: a mechanical find-and-replace cannot distinguish "Loom" (proper-noun language) from "Loom" (sentence-initial file-unit), and an over-eager pass would rewrite the few legitimate sentence-initial file-unit uses and quietly change the referent.
- Sentence-initial file-unit uses ("Loom files are discovered…", "Loom code never sees wire names…") must be restructured or pluralised (`looms`), not capitalised — the usage note pinned under `loom (file unit)` covers this.
- The `pi-loom (extension)` entry is unaffected — it is already a distinct lexical form (hyphenated, lowercase).
- The fixed compound `loom code` is a per-author choice; after the rename prefer "Loom-language code" when the language sense is intended and "code in a loom" / "the loom's body" when the file-unit sense is intended.
- The diagnostic-code namespace `loom/...` is governance-locked and out of scope of the rename.

## Consequence

**Severity:** correctness

Two reasonable implementers — or two reviewers reading the same paragraph — can disagree on whether a heading, a list bullet, or a cross-page citation is binding the file-unit or the language sense; for normative prose (e.g. `runtime-value-model.md`'s "Loom code observes `Result` only…", which constrains language-level semantics) the divergence is observable in what they implement and what they test. The defect is not localised to the glossary: it propagates anywhere either sense is referenced, and it forecloses `grep`-driven audits over the corpus because both spellings collide on the same token under case-insensitive matching and miss each other under case-sensitive matching.
