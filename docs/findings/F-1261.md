## Finding

The two trailing paragraphs of `host-prerequisites.md` — *Extension ES-module declaration* (anchor `#extension-esm-type-module`) and *Extension entry point* (the `extensions/index.ts` + `package.json#pi.extensions = ["./extensions"]` factory pin) — describe loom's *own* extension manifest, not a Pi-side prerequisite. They belong on `pi-integration.md` (the parent integration page that already lists slash-command discovery and the file-watcher contract for the same audience) or a dedicated extension-manifest topic page. The ESM paragraph was relocated here on 2026-06-22 next to the wrong neighbours.

## Solution Space

**Shape:** single

### Recommendation

Relocate both paragraphs to `pi-integration.md` (or a dedicated `extension-manifest.md` topic page paired with the existing `package.json#pi.extensions` build-time gate). Preserve `#extension-esm-type-module` (cited twice from `H1a-scaffold-and-toolchain.md`). Leave a pointer where *Extension entry point* sat, because PIC-18 (`conversation-drive.md`), `binder-inference.md`, and the *System notes* + factory-bind paragraphs reference "the **Extension entry point** above" — re-point every such anaphor in the same commit (reuse the `f5e89f4` anchor/anaphor-rewrite recipe under `docs/_tools/`).
