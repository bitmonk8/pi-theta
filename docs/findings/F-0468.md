## Finding

`real-host-smoke-gate.md`'s H1 heading reads `# Manual real-host smoke gate (release-gate / version-bump governance)`, but the document's own body establishes a different canonical name for the thing it defines: lines 3 and 5 both bold-introduce it as the **manual real-host smoke governance protocol**, and `plan.md` §Release gate references it by that same bolded term. The heading drops "governance protocol" and substitutes the bare noun "gate".

The mismatch matters because the document is at pains to say it is *not* a mechanical gate — it is "a cross-cutting governance note, not a leaf" and "a contributor obligation, not a CI-enforced gate". The corpus uses "gate" elsewhere for mechanically-enforced CI checks (release gate, closing gate); titling this manual protocol "smoke gate" invites a reader to mis-file it as one of those. The title should match the canonical term the document and the plan index already agree on.

The same commit's `plan.md` §Release gate addition and the new file's body (lines 3 and 5) both use the canonical bolded term "manual real-host smoke governance protocol", but the file's new H1 heading coined the shortened variant "Manual real-host smoke gate". `git log -S"Manual real-host smoke gate"` over `docs/` returns only `fd22a3d`, while `git log -S"smoke governance protocol"` shows the canonical term predates the extraction — so the heading drift was born with the file, not inherited from an earlier revision.

## Solution Space

**Shape:** single

### Recommendation

Replace the H1 heading on line 1 of `docs/plan_topics/real-host-smoke-gate.md` with `# Manual real-host smoke governance protocol (release-gate / version-bump governance)`, so the title matches the bolded canonical term used in the same file's body (lines 3 and 5) and in `plan.md` §Release gate.

No other file requires editing: `H4a`, `H6a`, `V18d`, and `H7a` already refer to the document as the "manual real-host smoke governance protocol" or "manual real-host smoke run", never "smoke gate". The filename and the `real-host-smoke-gate.md` link text need not change — Markdown links resolve by path, not by heading text.

## Consequence

**Severity:** cosmetic

A reader reconciling the H1 title ("smoke gate") against the bolded canonical term used in the same document's body and in `plan.md` ("smoke governance protocol") must decide which name is authoritative, and the "gate" wording risks mis-classifying a manual contributor obligation as a mechanical CI gate. No implementer goes off-script and no leaf's acceptance criteria depend on the title.
