## Problem

`V9a` constructs the full `loom/load/host-incompatible` payload on every probe-failure route, but the only payload assertion in `V9a-T` (mirrored in `V9a`) is the final Tests bullet asserting "the correct closed `details.kind` discriminator". The spec's finer `details.*` payload and routing contract in `capability-probe.md` — kind-selection routing (e.g. a non-SemVer `process.versions.node` MUST route to `probe-failed`, not `node-floor`) and the `details.member`/`details.package`/`details.step`/`details.observed`/`details.cause` field population — is asserted on no leaf. Because the closed diagnostic-code reverse-gate (DIAG-2) only fires at the `H6a` live-corpus footing and is keyed on emitted codes rather than payload-field contents, `V9a`/`V9a-T` can ship green with the routing implemented incorrectly; `V7c`'s `host-incompatible` bullet asserts only the rendered `node-floor` Message and does not close the gap.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commit:** `c6a664e` — "pi-loom plan: build/update plan for spec.md + review" (2026-06-10)
**History:** The failure-emission Tests bullet has scoped the payload assertion to the closed `details.kind` discriminator since `docs/plan_topics/V9a-T-capability-probe.md` (and its `V9a` mirror) were first authored in `c6a664e`. The bullet text at inception is byte-equivalent to its current form for this clause; the finer `details.*` payload and routing rules were never added. Subsequent commits touching the leaf pair (`b9de3ee`, `692cc8c`, `2f4ecea`, `f68f8c2`, `c3de47a`) reworked the capability-list/constant exports and Deps, not the failure-payload test scope. The defect is an original-authoring omission, not a regression introduced by a later edit.

## Solution approach

Add Tests bullets to `V9a-T` — and mirror them in `V9a` — that pin the per-route `details.*` payload and routing contract alongside the existing closed-`details.kind` bullet, citing `capability-probe.md` steps (a)–(e), *On failure: refusal and diagnostic*, and *Self-failure*. The spec already pins every payload value and routing rule (including the invalid-version routing distinction and the first-failure-in-order `details.member` selection), so source those vectors from `capability-probe.md` rather than re-deriving them. Update `V9a`'s **Ships when** so its `npm test` condition references the payload and routing assertions in addition to the `details.kind` refusal.

- Out of scope: the spec is read-only for this fix — `capability-probe.md` and `placeholder-rendering-b.md` already specify every value, so the change is internal to the `V9a`/`V9a-T` leaf pair.
