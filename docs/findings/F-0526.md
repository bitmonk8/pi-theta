## Finding

`V9c` (Prompt-mode conversation drive) produces and asserts a `Result.Ok(string)` value: its **Adds** names "trailing-turn `Ok(string)` extraction", a Tests bullet asserts "an untyped query resolves to the trailing turn's `Ok(string)`", and **Ships when** requires `npm test` to assert "the trailing-turn `Ok(string)` extraction". The `Ok`/`Err`/`Result` value model is owned by `V4a` (`match`, `?`, and `Result`).

`V9c`'s **Deps** are `V9c-T`, `V9a`, `V8a`, `V17a`. None of these is `V4a`, and none reaches it transitively: `V9a`'s Deps are `V9a-T`, `H4a`, `V7d`; `V8a`'s are `V8a-T`, `H3a`, `V8d`; `V17a`'s are `V17a-T`, `V8a`, `V4d`, `V9a`, `H4a`, `H4b`. The only `V4*` leaf reachable from `V9c` is `V4d` (via `V17a`), and `V4d` is *upstream* of `V4a` (`V4a`'s own Deps list `V4d`), so reaching `V4d` does not satisfy `V4a`.

Under the pickability predicate in `conventions.md` Â§Leaf format (**Deps**) â€” a Deps entry is satisfied exactly when its completion tag is present â€” `V9c` becomes pickable once `V9c-T`, `V9a`, `V8a`, and `V17a` are complete, none of which forces `V4a-complete`. An implementer could therefore pick `V9c` before the `Ok`/`Err`/`Result` constructor exists. This is a pervasive pattern, not a `V9c`-only slip: many runtime-facing leaves construct or consume `Ok`/`Err`/`Result` without declaring `V4a` in their Deps.

## Solution Space

**Shape:** multiple

### Option A â€” Pin V4a as an explicit Deps edge on V9c

**Approach:** Add `V4a` to `V9c`'s **Deps** field so the Result-type owner is a declared, completion-tag-gated prerequisite.
**Plan edits:** In `docs/plan_topics/V9c-conversation-drive.md`, change the Deps field from `` `V9c-T`, `V9a`, `V8a`, `V17a` `` to include `` `V4a` ``.
**Spec edits:** None.
**Pros:** Mechanically closes the pickability gap for `V9c` under the existing predicate; no convention change.
**Cons:** Treats one instance of a corpus-wide pattern; the same omission remains on every other leaf that constructs or consumes `Ok`/`Err`/`Result` (e.g. `V13a`, `V13c`, `V14a`, `V15b`, `V17a`, `V2c`, `V5d`, `V6d`, `V9n`). Listing `V4a` on each becomes Deps noise for a foundational type.
**Risks:** A partial sweep leaves sibling leaves with the same latent gap, giving a false impression the class is closed.

### Option B â€” Declare the Result value model an ambient foundation in conventions.md

**Approach:** State once in `conventions.md` that the `V4a` Result/`Ok`/`Err` value model is an ambient foundation that leaves may construct or consume without listing `V4a` in their Deps, and pin where the foundation is guaranteed available.
**Plan edits:** In `docs/plan_topics/conventions.md` Â§Leaf format (Deps) / pickability, add a statement that the `V4a` Result value model (`Ok`/`Err`/`Result`) is an ambient foundation; leaves using it need not declare a `V4a` Deps edge. Leave `V9c` (and siblings) unchanged.
**Spec edits:** None.
**Pros:** Resolves the entire class in one place; avoids decorating every runtime-facing leaf with a `V4a` edge.
**Cons:** Relaxes strict pickability for one named foundation, so the convention must make the ambient guarantee true â€” i.e. that `V4a` is reliably complete before any consumer is picked, or that consumers tolerate its absence.
**Risks:** If a consumer is genuinely pickable before `V4a-complete`, the ambient assumption is violated and the gap re-opens silently. `V4a`'s own Deps (`V2b`, `V3a`, `V4d`) place it in early vertical work, ahead of the V9 slice, which supports the assumption.

### Recommendation

Adopt **Option B**: add the ambient-foundation statement for the `V4a` Result value model to `conventions.md` Â§Leaf format (Deps). The omission is corpus-wide, so a per-leaf edge (Option A) is both noise and an incomplete sweep. The convention text must name `V4a` as the owner of `Ok`/`Err`/`Result` and state that consuming leaves need not declare a `V4a` Deps edge; confirm while writing it that `V4a`'s position (Deps `V2b`/`V3a`/`V4d`, early vertical work) precedes the earliest leaf that constructs a Result value, so the ambient guarantee holds. If any consumer is found that could be picked before `V4a` completes, that specific leaf still needs an explicit `V4a` edge.
