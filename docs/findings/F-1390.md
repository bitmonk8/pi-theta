## Finding

The `> **loom 1.0 seam — automatic context escalation.**` blockquote in `docs/spec_topics/binder/binder-model-and-context.md` (the paragraph between the subagent-mode warning and the `### Session-context truncation` subheading) runs five sentences. Only the first two carry the normative loom 1.0 binder-path content:

- S1 — the path is re-entrant per loom turn; the loom 1.0.0 runtime issues exactly one binder call per slash invocation;
- S2 — the binder's input record (parameter table, raw slash text, optional session-context block) and the resolved binder-model handle are constructed afresh on every binder call, with no cached state that would prevent a second call from observing a different `bind_context` snapshot.

The other three sentences are about the deferred *automatic context escalation* extension and a disclaimer chain around it: S3 motivates the seam via the deferred extension's `needs_info` retry, S4 asserts the re-entrancy invariant is settled regardless of how the open design decision resolves, and S5 restates that the pin commits loom 1.0 only to the binder-path property and not to the extension's surface. S4 and S5 are residue from an earlier spec-review fix that pushed the blockquote in the opposite direction (away from sounding like a normative MUST for the deferred feature), and they now drown the actual loom 1.0 obligation in deferred-feature framing.

The two anchors `loom-1-0-seam-automatic-context-escalation` and `v1-seam-automatic-context-escalation`, the `> **loom 1.0 seam — <name>.**` blockquote framing, and the seam heading text are load-bearing: `future-considerations/surface-extensions.md` cites the `#v1-seam-automatic-context-escalation` fragment as the seam's *Carrier*, and `overview-and-orientation.md` references "automatic context escalation" by name in the 12-seam-count exclusion list under GOV-30. Any rewrite must keep both anchors, the seam blockquote shape, and the seam name intact.

## Solution Space

**Shape:** single

### Recommendation

Compress the blockquote to three sentences while preserving the seam blockquote framing and both `<a id>` anchors verbatim. Suggested replacement body (verbatim except for the two anchors, which stay where they are):

> **loom 1.0 seam — automatic context escalation.** <a id="loom-1-0-seam-automatic-context-escalation"></a><a id="v1-seam-automatic-context-escalation"></a> The binder-invocation path is **re-entrant per loom turn**: the loom 1.0.0 runtime issues exactly one binder call per slash invocation (and `bind_context` is therefore observed at most once per invocation), but the path makes no assumption that `bind_context` is set at most once per loom over the loom's lifetime. The binder's input record (parameter table, raw slash text, optional session-context block) and the resolved binder-model handle are constructed afresh on every binder call, with no cached state that would prevent a second call from observing a different `bind_context` snapshot. This invariant is the loom 1.0 commitment of the seam; the deferred extension that motivates it (`needs_info` → automatic retry with `bind_context: session`) and the open design decision around it live at [Future Considerations — Surface extensions gated on an open design decision](../future-considerations/surface-extensions.md#surface-extensions-open-design).

Implementer-relevant edge cases:

- Keep the blockquote shape (`> **loom 1.0 seam — <name>.**`) — `overview-and-orientation.md` describes seams uniformly via this blockquote framing.
- Keep the seam name "automatic context escalation" — it is the token referenced from the GOV-30 12-seam-count exclusion list in `overview-and-orientation.md`.
- Keep both anchors (`loom-1-0-seam-automatic-context-escalation` and `v1-seam-automatic-context-escalation`) on the same blockquote — the `v1-`-prefixed one is the active inbound from `future-considerations/surface-extensions.md`, and the `loom-1-0-`-prefixed one is the post-rename canonical anchor.
- Cross-ref hygiene: the link target `../future-considerations/surface-extensions.md#surface-extensions-open-design` is the same one the current S3 uses; re-use it verbatim so the anchor resolver stays satisfied. No outbound link to the BNDR-1/2/3 area is required (this seam stands on its own; the refinement-loop seam has its own blockquote).
- Do **not** drop sentence S2 (the input-record / no-cached-state clause). It is the second half of the normative invariant — without it the seam reduces to "exactly one binder call per invocation", which does not capture the re-entrancy property the seam exists to pin.

## Consequence

**Severity:** advisory

A reader skimming the blockquote for "what does loom 1.0 commit to on this path?" must extract the normative invariant from the first two sentences while the surrounding three sentences pull attention toward a deferred feature and an open design decision. No implementer will produce a wrong runtime; some will spend extra effort distinguishing the load-bearing pin from the disclaimer chain.
