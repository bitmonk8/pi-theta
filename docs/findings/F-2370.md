## Problem

`V11i-T`'s Tests list carries a `BNDR-10` bullet asserting that, for a `mode: subagent` loom declaring `bind_context: session`, the truncation walk is skipped and no *Recent session context* block is emitted (the binder input is constructed as for `bind_context: none`). Observing that absence exercises the binder-input construction / compact-transcript rendering surface owned by `V11b`. `V11i-T`'s `Deps` lists only `V8e`, `V9i`, omitting `V11b`. An implementer who picks `V11i-T` before `V11b` ships authors the `BNDR-10` assertion against a rendering surface that does not yet exist, producing a compile-time red rather than the intended behavioural red — which `conventions.md`'s "fail red for the intended reason" gate does not satisfy.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 6a5852a — pi-loom spec-plan: fix F-1352 — close BNDR-10 at V11i (subagent-mode skip) + citing tests (2026-06-24, Thomas Andersen)
**History:** `V11i-T` was created in d0c127f (2026-06-16) with `Deps` `V8e, V9i` and only walk-selection assertions, for which those deps were complete. Commit 6a5852a appended the `BNDR-10` bullet — an assertion over the rendered binder input that `V11b` owns — without extending `Deps` to include `V11b`, introducing the missing edge.

## Solution approach

Add `V11b` to the `Deps` field of `docs/plan_topics/V11i-T-session-context-truncation.md`. `V11b` owns the compact-transcript renderer / binder-input construction surface the `BNDR-10` bullet observes, so its completion must gate `V11i-T`'s pickability for the suite to red out on the intended behavioural reason. This brings the tests task into line with the paired `V11i`, which already lists `V11b` in its `Deps`.
