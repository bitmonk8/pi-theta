## Finding

The Unknown-reason rule paragraph states "Pin the coercion to `String(event.reason)` explicitly — a template-literal coercion (`` `${event.reason}` ``) re-introduces the symbol-throw bug `String()` is chosen to avoid" as a bare imperative, with no MUST/MUST NOT, unlike the adjacent "The coercion itself MUST be wrapped in `try`/`catch`". The requirement is implementation-determining and load-bearing yet not normatively pinned.

## Issue introduction

**Verdict:** multi-commit-interaction (cross-reference); present-since-inception (coercion modal)

**Introducing commits:**
- `6cd6c56` (2026-05-07, "resolve 'details.event.reason coercion is unspecified for non-string Pi values'") — authored the `String(event.reason)` coercion clause and the template-literal warning; it never carried an RFC-2119 modal.
- `52b9101` (2026-05-11, "resolve 'T24 — Fork-reason watcher closure …'") — introduced the branch-(k) seam-tag-write-throw recovery text and its `… below` cross-reference while the whole Pi-integration contract was a single document, where "below" resolved correctly in reading order.
- `f5e89f4` (2026-06-04, "enforce 100KB hard / 30KB recommended size cap on spec set") — split the contract into per-topic files, separating the reference from its target.
- `c3de47a` (2026-06-27, "fix F-1602 — relocate universal session_shutdown teardown to session-shutdown-semantics.md; sweep spec + plan citations") — relocated the citing prose into `session-shutdown-semantics.md`, leaving branch (k) in `session-only-degraded-state.md` (which the Contents list orders *before* the citing file), so the surviving "below" became both cross-file and directionally wrong.

**History:** The coercion-modal gap has existed since the clause was first authored. The cross-reference was valid when authored in a single-file corpus and was degraded into a dangling reference by the later topic-split and relocation commits that separated reference and target across files without re-resolving the "below" directional word.

## Solution Space

**Shape:** single

### Recommendation

Restate the obligation with a normative modal, e.g.: "The `details.observed` coercion **MUST** use `String(event.reason)` and **MUST NOT** use a template-literal coercion (`` `${event.reason}` ``), which reintroduces the symbol-`toString`-throw bug `String()` avoids." Keep the existing rationale clause.

## Consequence

**Severity:** advisory

A reader of `session-shutdown-semantics.md` who follows "branch (k) under *Acceptance criteria* below" hits a dead end — the section is absent from that file and the real target is in a different, earlier topic file — so the recovery-path rationale is hard to confirm. The missing modal leaves the `String()`-vs-template-literal requirement non-normative and unverifiable. Neither blocks implementation; the underlying behaviour is described elsewhere in the same cluster.
