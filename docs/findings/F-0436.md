## Finding

`docs/plan.md` §Release gate states that the canary's "**Deps** name the **complete** coverage-producing set and carry the single statement of the transitive-completeness obligation." The word *complete* reads as a guarantee that `H5b`'s `Deps.` contain every coverage-producing leaf, mechanically enforced.

The mechanical backing does not support that claim at facet granularity. `H5d`'s transitive-completeness arm is a **per-cell at-least-one** check: for each `coverage-matrix.md` closing-leaf cell it reddens only when *none* of the cell's listed leaves reaches `H5b`'s `Deps.`. For a genuine multi-facet split row — where each listed leaf closes a distinct facet (e.g. `ERR-17 | V4d, V13d`, `DISC-4 | V10a, V9m`) — the arm stays green as soon as any one listed facet-closer is present, so a sibling facet-closer omitted from `H5b`'s `Deps.` is not caught. `conventions.md` §REQ-ID discipline (*Transitive-completeness plan-maintenance*) states this limit explicitly and routes the every-facet-closer residue to the loom 1.0 release-time residue inspection (checklist item 7). The §Release gate overview prose carries the unqualified "complete" word without that caveat, asserting a completeness property the named machinery does not enforce.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan.md` §Release gate, soften the sentence "The canary's **Deps** name the complete coverage-producing set and carry the single statement of the transitive-completeness obligation" so the prose distinguishes what `H5d` mechanically backs from the manual obligation. Convey that the `Deps.` name the coverage-producing leaves with per-cell at-least-one membership mechanically backed by `H5d`, while every-facet-closer completeness for split coverage-matrix rows is the manual authoring obligation swept at the loom 1.0 release-time residue-inspection item 7. The source-suggested replacement clause "name the coverage-producing leaves (per-cell at-least-one, mechanically backed by `H5d`; every-facet-closer completeness is the manual authoring obligation, swept at residue-inspection item 7)" is acceptable literal text. Keep the cross-reference targets (`coverage-matrix.md`, `conventions.md` §REQ-ID discipline) intact; this edit is confined to `docs/plan.md` overview prose and adds no normative content.

## Consequence

**Severity:** cosmetic

A plan reader takes "complete" at face value and assumes the `H5d` mechanical backing guarantees every coverage-producing leaf — including every facet-closer of a split coverage-matrix row — is present in `H5b`'s `Deps.`. In fact `H5d` enforces only per-cell at-least-one, leaving every-facet-closer completeness to the manual residue-inspection item 7. No leaf ships wrong from this; the cost is an overview claim that overstates the mechanical guarantee and can under-cue the manual obligation.
