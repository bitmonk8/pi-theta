## Problem

`cancellation.md` §Granularity states a normative obligation (the fixed five-site checkpoint set plus the loop-iteration macrotask-yield property), and `V17c` is its closing leaf, but no traceability row anywhere in `coverage-matrix.md` maps §Granularity to `V17c`. The *Numbered REQ-IDs* table maps `CNCL-1`…`CNCL-6` to `V17a`, and the *Code-keyed obligation areas* table's two `cancellation.md` rows (`cka-32`, `cka-33`) cover §Forwarding-listener and §Race semantics — none covers §Granularity. Because §Granularity carries no literal `MUST`/`MUST-NOT` token, `H5e`'s un-anchored-MUST scan does not surface it, and `H5d` only validates existing rows, so the obligation has no closing-gate traceability hook and the matrix-driven gates pass vacuously for it.

**History:** The cancellation coverage was first authored in c6a664e with `CNCL-1`…`CNCL-6` rows pointing at `V17a`, but no row was ever created for the §Granularity obligation (`git log -S 'Granularity' -- docs/plan_topics/coverage-matrix.md` is empty across all history); the gap stayed latent while `V17a` was the single cancellation leaf, reachable through the `CNCL` rows. Commit 5b95143 peeled §Granularity onto a new leaf `V17c` (adding `V17c-checkpoint-granularity.md` and editing `V17a`) without touching `coverage-matrix.md`, concretising the gap: the obligation's now-dedicated closing leaf has no traceability row at all.

## Solution approach

Add one row to the *Code-keyed obligation areas (no numbered REQ-IDs)* table in `docs/plan_topics/coverage-matrix.md`, whose spec-area cell names `cancellation.md` (CNCL) §Granularity (the fixed five-site checkpoint set plus the loop-iteration macrotask-yield property) and whose closing-leaf cell is `` `V17c` ``. Take the next free append-only token at the table tail for the *Token* column. No `H5b` `Deps.` change is needed — `V17c` is already reachable through the `V17a`–`V17c` range entry, which `H5d` expands by letter suffix.

## Solution constraints

- Honour the *Closing-leaf column authoring* rule: `V17c` is the only backtick-delimited span in the closing-leaf cell, and any code identifiers in the spec-area annotation prose are written bare.
- The *Token* column is append-only — take the next free token and never reuse or renumber an existing token.
