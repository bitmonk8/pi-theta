## Problem

The *Binder inference call* option enumeration in `binder-inference.md` (anchor `binder-inference-call`) never lists `options.maxRetries` or `options.maxRetryDelayMs`, yet the section's closing sentence delegates per-attempt backoff and any server-requested `Retry-After` wait to "this call's `StreamOptions.maxRetries` / `StreamOptions.maxRetryDelayMs`" ŌĆö presupposing loom configures them without stating whether or to what values. Implementers cannot tell whether loom sets those fields or inherits pi-ai's defaults, which changes how the HC3-d "Ōēż3 binder LLM calls per slash invocation" budget is counted: whether an SDK-internal wire retry counts against the budget, and whether one outer `complete()` call is one budgeted slot. The same ambiguity surfaces at `determinism-cancellation-failure.md#per-invocation-retry-budget`.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `d0bcdb2` (2026-06-05, "resolve 'binder per-invocation retry inter-attempt timing'") ŌĆö introduced the retry-delegation prose without extending the option-bullet enumeration to pin whether/how loom sets the two fields, and without revisiting HC3-d's call-counting unit.
**History:** `git log -S'maxRetries' -- docs/spec_topics/pi-integration-contract/binder-inference.md` returns the single hit `d0bcdb2`. Before that commit the section's closing clause read "inter-attempt timing is out of scope here." `d0bcdb2` replaced it with the current delegation prose ŌĆö the same defect anatomy recorded against `F-0622`. The finding is one facet of a larger `complete()`-call-shape under-pinning across `binder-inference.md` and its peers.

## Solution approach

Add bullets for `options.maxRetries` and `options.maxRetryDelayMs` to the *Binder inference call* option enumeration in `binder-inference.md`, stating loom's disposition of each field (a pinned value versus an inherited pi-ai default). Rewrite the section's closing retry-delegation sentence so the per-`complete()` wire-attempt count it implies is unambiguous against that disposition. At `determinism-cancellation-failure.md#per-invocation-retry-budget`, reconcile the HC3-d call-counting unit with the stated disposition so it is unambiguous whether an SDK-internal wire retry counts against the Ōēż3 budget.

## Solution constraints

- Out of scope: the `options.onResponse` bullet (owned by F-1296) and the `options.signal` citation (owned by F-1297) in the same enumeration.
- The disposition stated for the two fields MUST keep HC3-d's (`hard-ceilings/ceilings-3-and-4.md#hc3-d`) "Ōēż3 outer `complete()` calls per slash invocation" counting unit and NOCEIL-1's no-loom-side-delay invariant intact.
