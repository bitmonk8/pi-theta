## Finding

`docs/spec_topics/language-and-architecture.md` line 3 carries an editor-only HTML comment recording where the cross-cutting NFR dispositions (Source-language stability, Runtime observability) were relocated to:

```html
<!-- The cross-cutting NFR dispositions (Source-language stability, Runtime observability) are recorded in the Non-functional requirements section of overview-and-orientation.md#non-functional-requirements. -->
```

The comment is invisible to rendered-Markdown readers. The relocation it describes is already accomplished by the live link from `spec.md` and from the corpus's `Non-functional requirements` aggregator at `overview-and-orientation.md#non-functional-requirements`, which any reader who needs the dispositions will reach through normal navigation. The comment serves only the editor who is staring at the raw file and wondering where the bullets went — a transient concern from the day of the move, not a durable spec artefact.

Leaving it in place compounds two harms: it is dead text that future editors must read past and reason about ("is this stale? does it bind anyone?"), and it normalises HTML-comment-as-editorial-note as a pattern in the spec corpus — a pattern that, once accepted, accumulates.

## Issue introduction

**Verdict:** introduced-by-specific-commit
**Introducing commits:** 510f74f ("pi-loom spec: resolve \"NFR aggregator delivers one of three announced bullets\" (T13/T14)", 2026-06-05); subsequently carried verbatim through 81bf52a ("docs: collapse spec/ section pages into spec_topics/", 2026-06-11) which only moved the file from `docs/spec/` to `docs/spec_topics/`.
**History:** `git log --oneline -L 1,5:docs/spec_topics/language-and-architecture.md` shows the file is git-tracked. The comment first appears in 510f74f, which removed the two NFR bullets (`Source-language stability`, `Runtime observability`) from this page in favour of consolidating them into the `Non-functional requirements` section of `overview-and-orientation.md`; the diff replaces the two bullets with this single HTML comment as a relocation marker, intended for editors who would otherwise look for the bullets in their old home. The relocation itself was sound; the marker was never removed afterward and is the residue. The earlier creation commit f5e89f4 (2026-06-04, the 100KB-cap split) is the original source of the bullets at this location but is not the source of the comment.

## Solution Space

**Shape:** single

### Recommendation

Delete the HTML comment line (line 3 of `docs/spec_topics/language-and-architecture.md`) and the now-empty blank line that surrounds it, so the file reads:

```markdown
# Language and architecture

<a id="loom-1-0-non-goals-aggregator"></a><a id="v1-non-goals"></a>
### V1 non-goals
```

No replacement contributor-guide entry is needed: the relocation it commemorated is a year-old one-shot event, the canonical home (`overview-and-orientation.md#non-functional-requirements`) is reachable through the spec's normal forward links, and an editor searching for the relocated NFR bullets can grep the corpus in seconds. No anchors, links, or normative text are affected — fingerprint of the file is unchanged for every downstream reader.

## Consequence

**Severity:** cosmetic

No observer is affected — the comment is invisible in any rendered view and binds no implementer. The cost is editorial: a small, recurring source of noise for anyone reading the raw Markdown, and a precedent for further editorial-note comments.
