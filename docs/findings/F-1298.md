## Problem

The "loom 1.0 seam — Pi-owned subagents collision source set" blockquote in `registration-steps.md` step 3 MUSTs that the `SlashCommandSource` candidate list be defined as a single named set inside the runtime and consulted via membership test "rather than open-coded as three separate string comparisons or a hard-coded `switch`". This pins an internal runtime artefact, not behaviour: a passing and a failing implementation are observationally identical — both drop the same looms with the same `loom/load/cross-format-collision` diagnostic against the same inputs. The seam's load-bearing content is the collision-candidate membership rule (`source ∈ {"prompt","extension","skill"}`) and the future-widening invariant; neither needs the named-set / membership-test shape, and no conformance test cites it.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `7a6cf96` (2026-05-07) promoted the "single named set inside the runtime, consulted by the collision check via membership test" MUST into the pi-integration-contract page (lifting the seam pin out of `future-considerations.md` prose). `11cabfe` (2026-06-22) relocated the seam blockquote from `host-interfaces-services.md` into `registration-steps.md` verbatim (move-only; did not soften the named-set MUST).
**History:** The MUST has been present in equivalent form since `7a6cf96` and in its current location since the move-only `11cabfe`; neither commit attached an observable to the structural pin.

## Solution approach

Rewrite the seam blockquote in `registration-steps.md` step 3 (anchor `#loom-1-0-seam-pi-owned-subagents-collision-source-set`) so its normative content is the behavioural collision-candidate membership rule — a pending loom collides when an existing entry's `source` is one of `"prompt"`, `"extension"`, `"skill"` at the loom 1.0 Pi-SDK pin — and the future-widening invariant. Demote the "single named set / membership test rather than three string comparisons or a `switch`" sentence to non-normative guidance rather than a MUST.
