## Finding

`V16a` (the cross-ceiling arbitration seam, slice V16) is a prerequisite of several earlier-listed slices. `V4e`, `V5e`, `V11f`, and `V13c` each declare `V16a` in their **Deps** because they consult the seam at their first-enforcement / load-time point (`V4e-pre-evaluation-failures.md:16`, `V5e-depth-enforcement.md:12`, `V11f-binder-retry-taxonomy.md:16`, `V13c-query-tool-loop.md:17`). Under the plan's dep-driven build order these are forward references: leaves in slices V4/V5/V11/V13 must wait on a leaf listed in slice V16.

The plan carries interleave notes for V9, V10, V11, V12, V14, V15, and V17 that flag exactly this kind of non-letter-order edge with a "Sequence by **Deps**, not slice number" close. The V15 note already documents `V15b → V16a`. But no note covers `V4e → V16a`, `V5e → V16a`, `V11f → V16a`, or `V13c → V16a`: the V4 and V5 sections have no interleave note, the V11 note (`docs/plan.md:153`) addresses only `V11a`/`V11e`, the V13 section has no note, and the V16 section flags no prerequisite role for `V16a`.

The `Deps` edges themselves are present and correct, so this is a navigation/declaration gap rather than a build break — a reader scanning section order sees `V16a` near the end and gets no signal that earlier slices depend on it.

## Consequence

**Severity:** advisory

An implementer scanning slice order encounters `V16a` near the end of the vertical slices with no signal that V4/V5/V11/V13 leaves depend on it; lacking the interleave note carried by sibling slices, they may misread the build as letter-ordered. Because the `Deps` edges are present, the pickability predicate still enforces correct order, so no leaf can actually be started early — the cost is reader/navigation confusion, not a build break.

## Solution Space

**Shape:** single

### Recommendation

Add an interleave note declaring that `V16a` lands before its early-slice consumers. Place it in the §Vertical slices V16 section (the natural shared-prerequisite home, since `V16a` is depended on by multiple earlier slices), mirroring the form and the "Sequence by **Deps**, not slice number." close the sibling notes use.

The note must name the uncovered forward-dep edges so a reader sees the full prerequisite relationship: `V4e → V16a` (slash-load `params` ceiling-#4 consult per CIO-1), `V5e → V16a`, `V11f → V16a`, and `V13c → V16a` (first-enforcement-point consults). `V15b → V16a` is already declared in the V15 interleave note (`docs/plan.md:196`) and need not be repeated, though referencing it for completeness is acceptable. The note should state that `V16a` is built before these consumers because each binds the arbitration seam through its `Deps` on `V16a`.

Equivalently the declaration may be distributed as per-section notes under V4/V5/V11/V13, but a single V16-section note keeps the shared prerequisite in one place and avoids four scattered edits.
