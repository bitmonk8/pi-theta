## Problem

The V9h leaf pair (`V9h-degraded-unknown-reason.md` and `V9h-T-degraded-unknown-reason.md`) carries a `degraded-` component in its filename slug, but both leaves are titled "Unknown-reason rule" and V9h's Adds states the session-only degraded-state branch is split out to V9l. V9h now owns only the unknown-reason partition, the closed-set validation, and the snapshot-failure path, so the `degraded-` slug names a responsibility the leaf no longer holds. The sibling V9l (`V9l-session-only-degraded-branch.md`) is the sole owner of that responsibility and uses an accurate slug, making V9h's residual prefix misleading to a reader browsing or grepping `plan_topics/` for `degraded`.

## Issue introduction

**Verdict:** introduced-by-later-edit
**Introducing commits:** `55ed974` (2026-06-12, "pi-loom plan: resolve \"degraded-state obligations green over open spec contradiction\"")
**History:** The leaf file was created at `c6a664e` (2026-06-10, initial plan build) with the slug `V9h-degraded-unknown-reason.md`, at which point V9h legitimately owned both the unknown-reason rule and the session-only degraded-state branch ŌĆö the slug was accurate. Commit `55ed974` created `V9l-session-only-degraded-branch.md` and rewrote V9h's Adds to "split out to `V9l`" (confirmed via `git log -S "split out to" --follow` and `git log --diff-filter=A`), extracting the degraded-state responsibility but leaving V9h's filename unchanged. The slug became stale at that commit and has never been renamed since (`git log --follow --name-status` shows only content modifications at `a70e2a7`, `cb8737a`, `c47c49b` and no rename).

## Solution approach

Rename the V9h leaf pair to drop the stale `degraded-` component so the slug matches the "Unknown-reason rule" title ŌĆö e.g. `V9h-unknown-reason-rule.md` and `V9h-T-unknown-reason-rule.md`. Update the one filename-bearing reference, the `docs/plan.md` line 121 table-of-contents link, to point at the renamed file.

## Solution constraints

- Leave the leaf ID `V9h` (and `V9h-T`) unchanged ŌĆö only the filename slug changes ŌĆö so existing Deps citations and cross-leaf references stay valid.
