## Finding

The second `ERR-13` bullet (the "completed-callee finality" bullet) in both `V4c-terminal-outcomes.md` and `V4c-T-terminal-outcomes.md` ends with the appositive "— exercising a completed callee distinct from an appended turn". That clause supports two readings:

- (a) the completed callee's committed *side effect* is of a **kind** other than an appended conversation turn — e.g. a tool-call result whose completion is not transcript-visible (per `error-model.md` ERR-13, "Tool-call completion is not transcript-visible"); versus
- (b) the callee is a different **entity** from some previously appended turn.

Reading (a) is the one the spec grounds: ERR-13's "No rollback" rule and the partial-append contract treat the completed-callee finality property as *distinct from* the turn-grain partial-append property — a completed tool call / `invoke` child stays final even though its completion is not an appended turn. The current phrasing leaves (a) and (b) on equal footing, with the appositive as the only signal. The preceding test instruction ("drive a tool call / invoke child to *completion* … assert the completed callee's side effect persists and no compensating turn is injected") does point at reading (a), so the ambiguity is confined to the explanatory clause rather than the test itself, but the clause should not invite the wrong gloss.

## Solution Space

**Shape:** single

### Recommendation

In the second `ERR-13` (completed-callee finality) bullet of both `docs/plan_topics/V4c-terminal-outcomes.md` and `docs/plan_topics/V4c-T-terminal-outcomes.md`, replace the trailing clause "— exercising a completed callee distinct from an appended turn" with phrasing that makes the side-effect-**kind** contrast explicit, e.g.:

> "— exercising a completed callee whose committed side effect is not itself an appended conversation turn (for example a tool-call result that is not transcript-visible), distinct from the turn-grain partial-append property exercised by `ERR-8`…`ERR-11`."

Keep both leaves byte-aligned on this bullet (V4c-T's bullet is currently identical to V4c's). The edit is to the explanatory clause only; the test instruction preceding the dash stays as-is.

## Consequence

**Severity:** advisory

Two test-authoring agents could read the appositive differently when framing the completed-callee finality assertion; the surrounding instruction steers both toward the spec-intended kind-contrast, so a working test is still likely, but the wording is a guidance gap that should be closed before the leaf is picked up.
