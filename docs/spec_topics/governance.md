# Governance

*Audience.* This page is directed at spec maintainers and at automated tooling that maps REQ-IDs to implementation work, not at implementers of the loom runtime. It governs the spec corpus itself — how REQ-IDs are coined, anchored, and retired — and sources no software requirement on the runtime, the binder, the type system, or the Pi integration.

This page owns the spec corpus's REQ-ID governance: the per-page prefix table, the per-prefix retirement registry, the corpus-direction and binding-scope rules (GOV-17 / GOV-18), and the rules (GOV-1, GOV-3 through GOV-9, GOV-12, GOV-14 through GOV-16; GOV-2, GOV-10, GOV-11, GOV-13 retired — see [Retired REQ-IDs](./governance/anchor-scheme-and-retired.md#retired-req-ids)) that govern how REQ-IDs are coined, anchored, and retired. GOV-16 additionally governs a second cross-page-citable identifier class — *stable inline labels* (currently `HC3-a`…`HC3-e` and `NOCEIL-1`…`NOCEIL-4`, both bound to [`hard-ceilings.md`](./hard-ceilings.md)) — on terms parallel to GOV-1 / GOV-3 / GOV-8 / GOV-9. The rules apply to every non-narrative page under `spec_topics/`.
