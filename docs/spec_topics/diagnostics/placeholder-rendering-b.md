# Placeholder rendering b

### 5. Source-derived placeholders

**Placeholders.** `<path>`, `<file>`, `<descriptor>`, `<name>`, `<field>`, `<param>`, `<variant>`, `<keyword>`, `<key>`, `<char>`.

**Rule.** Render verbatim from the source as the offending text would appear there:

- **`<path>`, `<file>`** — the literal text inside the path-literal quotes, with no realpath normalisation, no symlink resolution, no scheme prefixing. This generalises the convention already pinned at `theta/parse/invoke-non-theta-extension`. For the **inline** arm of `theta/load/callee-has-errors` (a broken `subagent fn` body, theta 1.2 — [Functions — FN-6](../functions.md#fn-6)) the "callee" is a same-file function rather than a path literal, so `<path>` renders that `subagent fn`'s identifier (identifier-shaped per [Lexical — Identifiers](../lexical.md)) rather than a quoted `.theta` path.
- **`<name>`, `<field>`, `<param>`, `<variant>`, `<keyword>`** — identifier-shaped per [Lexical — Identifiers](../lexical.md); rendered unquoted. Any quoting in the rendered message comes from the surrounding registry template (e.g. the literal `'<name>'` in `cannot reassign immutable binding '<name>'`), not from the placeholder.
- **`<key>`** — quoted with double quotes only when the key string is *not* identifier-shaped per [Lexical — Identifiers](../lexical.md) (i.e. would not match `[A-Za-z_][A-Za-z0-9_]*`); otherwise rendered bare. The identifier-shape predicate is a runtime check on the key string, not a parse-time grammar production.
- **`<char>`** — the raw character itself when printable (any code point outside the Unicode general categories `Cc` and `Cn`); otherwise the standard `\xNN` (for code points ≤ U+00FF) or `\u{NNNN}` escape per the escape forms accepted in [Lexical — String literals](../lexical.md).
- **`<descriptor>`** — the discovery-source descriptor as a `kind:value` pair, rendered as `<kind>:"<value>"` (the kind unquoted, the value double-quoted). Kinds are drawn from the closed set defined in [Discovery](../discovery.md) (`settings`, `cli-flag`, `package`); the value is the descriptor's source text verbatim (the settings entry, the CLI flag string, the npm package name). The descriptor format is normative.

**Test vectors.**

- An unknown frontmatter field `wibble` renders `unknown frontmatter field 'wibble'` (the registry template supplies the surrounding single quotes; the placeholder itself contributes the bare identifier).
- A non-existent settings-sourced theta directory `~/work/theta` renders `discovery source path does not exist: settings:"~/work/theta"`.
- A `match` on `obj["my-key"]` against a missing key renders `missing object key: "my-key"` (key is not identifier-shaped, so quoted).
- A member access `obj["kind"]` on a missing key renders `missing object key: kind` (key is identifier-shaped, so bare).

### 6. Underlying-error placeholders

**Placeholders.** `<error.message>`, `<original content first line>`, `<dispose error first line>`.

<a id="underlying-error-coercion"></a>
**Underlying-error coercion.** `<error.message>` and `<dispose error first line>` bind a *caught thrown value*, which JavaScript does not guarantee to be `Error`-shaped (`throw <any value>` is legal, and only `Error`-derived values carry a string-valued `.message`). Before the first-line truncation rule below applies — and likewise wherever a producer site populates an error-message field from a caught thrown value (a diagnostic `details.error` or `details.cause`, a `hint` or `message` that carries a caught throw's message, or `CodeToolError.message`) — the caught value `v` is coerced to the underlying string by:

1. when `v` is an object whose `.message` is a string, the underlying string is that `.message`;
2. otherwise the underlying string is `String(v)`, or the literal text `<unreadable>` when the `String(v)` coercion itself throws.

The truncation rule below then operates on this guaranteed string, so a non-Error throw never yields `undefined`, a non-string value that defeats the truncation rule, or a synchronous `TypeError`. (`<original content first line>` binds a string by construction and skips this coercion.)

**Rule.** Render as the underlying string (per the coercion above for the caught-throw placeholders) truncated to its first line:

1. Newline-normalise the underlying string per [Lexical — Encoding](../lexical.md) (`\r\n` and bare `\r` collapse to `\n`).
2. Take the prefix up to (but not including) the first `\n`.
3. Preserve trailing whitespace on the resulting line — no rstrip.
4. When the underlying string is empty, or contains nothing before its first `\n`, render the literal text `<no message>`.

Only `\n` (post-normalisation) is a line break for this rule. The Unicode line separators `\u2028` and `\u2029` are *not* recognised by the lexical newline rule and are *not* recognised here either: a host throwing an `Error` whose message embeds `\u2028` produces a single first-line cut at the first `\n` (or at end-of-string if there is none), with `\u2028` rendered as an ordinary character.

**Test vectors.**

- A `pi.sendMessage` synchronous throw whose underlying error message is `file:5:3: theta/parse/binding-case-mismatch: ...` renders `system-note delivery failed: file:5:3: theta/parse/binding-case-mismatch: ...`.
- A `pi.sendMessage` synchronous throw whose underlying error was constructed without a message (`new Error()`) renders `system-note delivery failed: <no message>`.
- A subagent child-process teardown error whose message is `connection closed\nstack trace ...\n` renders the prefix up to the first `\n` — the literal text `connection closed` — preserving any trailing space on that line.

### 7. Identifier-, descriptor-, and closed-enum placeholders

**Placeholders.** `<callee>`, `<enum>`, `<schema>`, `<method>`, `<model>`, `<provider>`, `<slash-name>`, `<slug>`, `<invocation-id>`, `<uuid>`, `<name1>`, `<name2>`, `<X>`, `<A>`, `<B>`, `<root>`, `<paths>`, `<path-a>`, `<path-b>`, `<source>`, `<higher>`, `<lower>`, `<fields>`, `<ms>`, `<deadline>`, `<cap>`, `<capability>`, `<dotted-key>`, `<reason>`, `<kind>`, `<op>`, `<step>`, `<call>`, `<ctor>`, `<value>` (in the parse-time literal-value position; the runtime-value usage of `<value>` belongs to category 2 and is distinguished by emitting site).

**Rule.** Each placeholder renders by the closed sub-rule below; sub-rules compose with categories 1–6.

- **Identifier-shaped** (`<callee>`, `<enum>`, `<schema>`, `<method>`, `<model>`, `<provider>`, `<slash-name>`, `<slug>`, `<invocation-id>`, `<uuid>`, `<name1>`, `<name2>`, `<X>`) — rendered unquoted in identifier form per [Lexical — Identifiers](../lexical.md). Any surrounding quoting comes from the registry template, not from the placeholder. The leading `/` of `<slash-name>` is contributed by the surrounding template (e.g. `/<slash-name>`), not by the placeholder. `<uuid>` and `<invocation-id>` render in canonical lowercase 8-4-4-4-12 hex form. `<callee>` renders as the registered slash name without leading `/` when one exists; otherwise it falls through to category 5's `<path>` rule (verbatim path-literal source text).
- **Path-shaped** (`<root>`, `<paths>`, `<path-a>`, `<path-b>`, `<source>`, `<A>`, `<B>` — the latter two as the file-path stems used in `theta/load/import-cycle` and `theta/load/invocation-cycle`) — rendered via category 5's `<path>` rule (the literal text inside the path-literal quotes, with no realpath normalisation, no symlink resolution, no scheme prefixing). List-valued forms (`<paths>`) are joined with `, ` (comma-space) in declaration order; for `theta/load/cross-format-collision` the order is: registered candidates first, then the slash-name-deriving `.md` siblings, both internally ordered by discovery-source priority then by absolute path.
- **Descriptor-shaped** (`<higher>`, `<lower>`) — rendered via category 5's `<descriptor>` rule (`<kind>:"<value>"`).
- **Joined-identifier list** (`<fields>`) — identifier-shaped names joined with `, ` (comma-space) in declaration order.
- **Numeric** (`<ms>`, `<deadline>`) — rendered via category 4's integer rule.
- **Closed-enum** — each placeholder draws from a closed value table:
  - `<cap>` ∈ {`theta.scanPackagesMaxFiles`, `theta.scanPackagesTimeoutMs`} (the two scan-package caps in [Discovery — Package discovery](../discovery.md)).
  - `<capability>` ∈ {`pi.registerMessageRenderer`, `pi.registerCommand`, `pi.registerFlag`, `pi.on`, `pi.getCommands`} (per [Pi Integration Contract — Extension-bootstrap SDK failures](../pi-integration-contract.md)).
  - `<dotted-key>` ∈ {`tool_loop.max_rounds`, `respond_repair.attempts`} (the two non-negative-integer frontmatter scalars in [Frontmatter — Field contract](../frontmatter.md); other dotted keys join the closed set only as new frontmatter scalars are added under GOV-7).
  - `<reason>` ∈ {`quit`, `reload`, `new`, `resume`, `fork`} for the closed-arm path; for the unknown-reason path (per PIC's **Unknown-reason rule**), the verbatim observed value coerced to string via `String(event.reason)`, or the literal `<unreadable>` when the property access threw. The literal `<unreadable>` rendering is overloaded across three distinct producing paths — the throwing-property-access path above, the snapshot-failure path where the handler-entry pinned-constant lookup failed before `event.reason` was read, and the sub-step-2 stamp-throw path per PIC's [*Accepted theta 1.0 residual gap — sub-step-2 stamp-throw case*](../pi-integration-contract/session-only-degraded-state.md#substep-2-stamp-throw-residual-gap), where payload-construction substitutes the `<unreadable>` sentinel because the affected invocation's `entry.shutdownReason` field is unset (sub-step 2's stamp threw before populating it) and the per-invocation `finally` later reads that unset field. PIC's [**Unknown-reason rule**](../pi-integration-contract/unknown-reason-rule.md#unknown-reason-rule)'s captured-value enumeration covers the first two paths via its four captured `event.reason` forms (closed-set member; `String(event.reason)`; `<unreadable>` on throwing-access; `<unreadable>` on snapshot-failure); the third path's `<unreadable>` is sourced from the per-entry `entry.shutdownReason` field rather than from the captured value and is owned by the residual-gap paragraph just cited. The three `<unreadable>` causes are disambiguated by orthogonal signals rather than by the rendered `<reason>` value alone: the co-emitted teardown diagnostic (`theta/host/session-shutdown-reason-unknown` versus `theta/host/session-shutdown-pinned-constant-unreadable`) discriminates the throwing-access versus snapshot-failure paths, and the affected invocation's dual appearance in the `theta/runtime/reload-teardown-timeout` `<list>` of the same `session_shutdown` event (per the residual-gap paragraph) discriminates the sub-step-2 stamp-throw path from the other two.
  - `<kind>` is a closed token-name table per emitting site:
    - In `theta/parse/non-string-enum-value` and `theta/parse/non-string-discriminator`: one of `string`, `integer`, `number`, `boolean`, `null` (the type-kind of the offending literal).
    - In `theta/load/settings-invalid-entry` and `theta/load/manifest-invalid`: the JSON kind, one of `string`, `number`, `boolean`, `null`, `array`, `object`.
    - In `theta/load/host-incompatible`: one of `node-floor`, `abortsignal-shape`, `sdk-capability-missing`, `peer-dep-out-of-range`, `peer-dep-malformed-version`, `typebox-shape`, `probe-failed` (the closed Step 0 capability-probe kinds).
  - `<op>` is the source token verbatim, drawn from the closed table in category 3, extended to `++` and `--` for `theta/parse/increment-decrement`.
  - `<step>` ∈ {`1`, `3`, `4`, `5`} (the four `session_shutdown` teardown sub-steps that emit `theta/host/session-shutdown-teardown-step-failed`), rendered via category 4's integer rule. The closed set is pinned by that row's *Trigger* prose in [Code registry — host](./code-registry-host.md), whose source of truth is PIC's **Per-step isolation** paragraph.
  - `<call>` is the static call-site label, drawn from a closed set keyed per `details.step` and rendered verbatim (the label is a fixed wire-contract string, not implementation-chosen). The closed set is pinned by the `theta/host/session-shutdown-teardown-step-failed` row's *Trigger* prose in [Code registry — host](./code-registry-host.md) (source of truth: PIC's **Per-step isolation** paragraph) — for instance `"thetaRegistry.drain"` (step 1) and `"discoveryWatcher.close"` (step 4).
  - `<ctor>` ∈ {`array`, `Result`} (the closed `GenericType` constructor set used by `theta/parse/generic-arity-mismatch`), rendered unquoted in identifier form; any surrounding quoting (the literal `'<ctor>'` in `generic type '<ctor>' expects <expected> type argument(s); got <actual>`) comes from the registry template. The source of truth for the closed set is the *Generic-application constructors* paragraph in [Grammar Appendix — Type grammar](../grammar.md#type-grammar); its existing GOV-7 / GOV-8 closed-enum versioning posture governs any future widening of the set.
- **Parse-time literal-value** (`<value>`) — when used in a parse-time row (`theta/load/unknown-mode-value`, `theta/load/unknown-methodology-value`, `theta/load/unknown-bind-context-value`, `theta/load/model-unresolved`, `theta/parse/duplicate-enum-value`, `theta/parse/duplicate-discriminator-value`), rendered as the literal source text verbatim per category 5's `<key>` (bare-when-identifier-shaped, double-quoted otherwise) — except that YAML scalars with no enclosing source quoting (e.g. `mode:` values) render unquoted regardless of identifier shape.

**Test vectors.**

- A `theta/runtime/reload-teardown-timeout` with two in-flight invocations against slash names `plan` and `review`, UUIDs `aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa` and `bbbbbbbb-bbbb-bbbb-bbbb-bbbbbbbbbbbb`, and elapsed wall time 2003 ms renders `reload teardown timed out after 2003ms; 2 invocation(s) still in flight: /plan:aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa, /review:bbbbbbbb-bbbb-bbbb-bbbb-bbbbbbbbbbbb` and sets `hint` to `2003`.
- A `theta/load/discovery-slow` aborted at root `~/work/theta` because the `theta.scanPackagesMaxFiles` cap fired renders `package-discovery walk aborted at ~/work/theta: theta.scanPackagesMaxFiles cap reached`.
- A `theta/load/cross-source-shadow` where the same slash name `plan` resolves from a higher-priority `--theta` flag pointing at `/proj/a/plan.theta` and a lower-priority settings entry pointing at `~/work/plan.theta` renders `slash name 'plan' shadowed across discovery sources: 'cli-flag:"--theta /proj/a/plan.theta"' wins over 'settings:"~/work/plan.theta"'`.

### 8. Host-derived freeform-tail placeholders

**Placeholders.** `<error>`, `<message>`, `<observed>`.

**Rule.** Render the underlying host-supplied string by category 6's first-line truncation rule (newline-normalise per [Lexical — Encoding](../lexical.md), take the prefix up to but not including the first `\n`, preserve trailing whitespace, render the literal `<no message>` when empty). When the §8 placeholder binds a *caught thrown value* — `<error>` on `theta/load/extension-bootstrap-failed` and `theta/runtime/active-set-restore-failed` — the value is first coerced to the underlying string per the [underlying-error coercion](#underlying-error-coercion) in §6 above, then rendered by the first-line truncation rule. Byte-identicality of the surrounding template still holds: every byte of the *Message* template before the §8 placeholder, and every byte after, is byte-identical across implementations. Only the §8 placeholder's interpolation is implementation-defined-tail at the byte level (the first-line truncation rule still applies to bound the rendering). Tests asserting on a *Message* template containing a §8 placeholder MUST construct partial-match patterns anchored on the byte-identical prefix and suffix; conformance to category 8 is therefore both decidable and testable.

**`<observed>` on `theta/load/host-incompatible`.** This row's `<observed>` is host-derived (and rendered per the rule above) for some `kind`s and a closed theta-constructed literal for others; [the per-`kind` `<observed>` / `<required>` enumeration](#host-incompatible-observed-required) is the source of truth for which. The §8 strict-equality build-time prohibition still applies to the whole row because its *Message* template contains the `<observed>` token; tests therefore assert via anchored partial-match patterns for every `kind` (for the closed-literal kinds the pattern is fully constrained but remains a regex match, not a `===` comparison).

**`<observed>` on the parsed-scalar out-of-range codes.** The `<observed>` token on `theta/load/frontmatter-value-out-of-range` and `theta/load/settings-value-out-of-range` is **not** host-derived freeform tail and is **not** rendered per §8's first-line-truncation rule above: it carries the offending *parsed* value (the post-parse YAML/JSON value, not source text), and renders fully byte-identically by the value's kind — a `number` (including integer-valued numbers) per the `integer` / `number` rows of the [canonical stringification table](../query/query-escapes-stringification.md#stringification-of-interpolated-values); a `boolean` as `true` / `false`; `null` as the literal text `null`; a `string` per category 5's `<key>` rule (rendered bare when identifier-shaped per [Lexical — Identifiers](../lexical.md), double-quoted otherwise — so a stringly-typed `"25"` renders `"25"`, distinct from the number `25`); and — on `theta/load/settings-value-out-of-range` only, whose offending value at a top-level key or a non-object settings root may be a composite rather than a scalar — an `array` value as the literal text `array` and an `object` value as the literal text `object` (the JSON kind token, matching the `array` / `object` members of the `<kind>` closed set in §7; the token renders the offending value's JSON *kind*, not a serialisation of its contents, so the rendering stays bounded regardless of the value's size). Because the rendering is fully byte-identical with no implementation-defined tail, these two rows are excluded from §8's strict-equality build-time prohibition, and tests MAY assert on their full *Message* strings by byte-identical equality.

**Carve-out: `<failure>` is not a §8 placeholder.** The `<failure>` placeholder used by the `theta/host/session-shutdown-pinned-constant-unreadable` row is **not** host-derived freeform tail and is **not** rendered per the §8 rule above. It is a theta-internally-constructed discriminator drawn from a closed set of two byte-determined literal forms (`"missing-entry"`, `"literals-shape-invalid"`) plus a `"throw:<…>"` template family whose `<…>` payload has already been truncated to at most 256 UTF-16 code units and control-character-escaped at the diagnostic-construction site (per [Pi Integration Contract — Unknown-reason rule](../pi-integration-contract/unknown-reason-rule.md#unknown-reason-rule), which owns the grammar). The renderer MUST emit the discriminator string verbatim: it MUST NOT re-truncate or re-escape the `"throw:<…>"` payload, MUST NOT substitute `<no message>` for an empty string (the empty case is unreachable by construction — every discriminator value is non-empty), and MUST NOT apply §8's first-line truncation. Tests asserting on the `pinned-constant-unreadable` row's rendered message MAY use byte-identical strict equality for the two literal forms, and MAY use a `startsWith("throw:")` partial-match for the throw-template family (matching the operator-side parse pinned by PIC's **Unknown-reason rule**). The discriminator set is **not** a `Set<string>`-membership-checkable enumeration: the third arm is a template family with an open-ended payload, not a third literal, so a `Set<string>.has(d)` check against the two literals would reject every legitimate `"throw:..."` value. The canonical membership predicate operator tooling MUST use against a `details.failure` value `d` is `d === "missing-entry" || d === "literals-shape-invalid" || d.startsWith("throw:")` (or an equivalent expression that admits the same three discriminator forms — the two literals by `===` and the throw-template family by `startsWith("throw:")`); the `"throw:<unreadable>"` coercion-failure fallback pinned in the row description below is admitted by the `startsWith("throw:")` arm and needs no separate predicate clause. The §8 prefix/suffix-anchoring rule below does **not** apply to this row, and the build-time prohibition on strict-equality assertions enumerates only rows whose Message templates contain an actual §8 placeholder. A future tightening of §8's rendering rules MUST NOT change `<failure>`'s wire format — that wire format is pinned at the construction site, and operator tooling that parses `details.failure` with `startsWith("throw:")` keys on it directly.

### Edge cases

- **Category 2 boundary.** A test that constructs an exactly-80-code-point string scrutinee MUST observe the full string (no truncation); a test at 81 code points MUST observe the truncated form (77-code-point prefix + literal `...`). Boundary-condition test vectors are required at the 80/81 code-point boundary.
- **Category 5 identifier predicate.** The identifier-shape check for `<key>` is a *runtime* string predicate against the regular expression `^[A-Za-z_][A-Za-z0-9_]*$`; a key like `kind` renders bare, `my-key` renders quoted. Keys whose string value happens to match a reserved keyword from [Lexical — Reserved keywords](../lexical.md) are still identifier-shaped for this rule (the reserved-keyword rule fires at parse time on identifier *positions* in source, not at runtime on string values).
- **Category 6 line-separator scope.** Restating for emphasis: `\u2028` and `\u2029` are ordinary characters for this rule. Implementations MUST NOT split on them, MUST NOT strip them, and MUST NOT promote them into `\n` — the runtime is byte-stable in their presence even though authors cannot introduce them through a regular string literal (see [Lexical — String literals](../lexical.md)).
- **Category 7 `<callee>` fallback.** When `<callee>` falls through to the path form (the parent has no slash-name binding for the callee — e.g. a sibling `.thetalib` `fn`), it renders verbatim per the path-literal rule rather than as an absolute path. This matters for the `theta/parse/invoke-arity-too-few` test vectors: a `.thetalib` `fn` callee renders as the source-text path the `import` declared, not a realpath.
- **Category 7 closed-enum closure.** The closed value tables for `<cap>`, `<capability>`, `<dotted-key>`, `<reason>` (closed-arm path), `<kind>`, `<step>`, `<call>`, and `<ctor>` are themselves spec-versioned per **GOV-7** and **GOV-8**: extending any table is a breaking change to the rendering surface even when the underlying registry row is unchanged. For `<ctor>` the table is owned by grammar.md's *Generic-application constructors* paragraph, so the breaking-change verdict on widening that set is reached there.
- **Category 8 prefix/suffix anchoring.** A test that asserts equality against the full *Message* string of a category-8 row is non-conformant under §8's rule: the placeholder is implementation-defined-tail at the byte level and only the surround is byte-identical. This non-conformance is detected at build time by enumerating registry rows whose Message templates contain a §8 placeholder — excepting `theta/load/frontmatter-value-out-of-range` and `theta/load/settings-value-out-of-range`, whose `<observed>` is the parsed-scalar carve-out documented in §8 above (fully byte-identical, not host-derived freeform tail) and is therefore not subject to this prohibition — and asserting that no test in the suite asserts strict equality against those rows' full Message strings.
