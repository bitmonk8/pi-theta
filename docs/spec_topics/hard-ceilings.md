# Hard Runtime Ceilings

This page is the normative owner of the cross-ceiling content that no single per-ceiling topic page is responsible for: the [ceiling-set invariants](./hard-ceilings/ceilings-3-and-4.md#ceiling-set-invariants) (audience-coverage invariant and ceiling-#1 panic-uniqueness invariant), the binder per-class retry budget's sub-obligations (HC3-a … HC3-e), the JSON-document-depth ceiling's per-boundary destination/surface table, the fixed evaluation order between ceilings (CIO-1 … CIO-6), the `masked` field's closed identifier set and wire-location split (the per-site reachable mask domain, V1 reachable predicate, pure-read MUST, verbatim-copy obligation on cascade-twin re-emission, and dedup-key non-inclusion rule are owned by [Pi Integration Contract — PIC-1](./pi-integration-contract/runtime-event-channel.md#pic-1)), the four enumerated non-existence claims (NOCEIL-1 … NOCEIL-4), the worked consequences of the interaction order, and the four-axis [Audit methodology](./hard-ceilings/ceiling-invariants-and-audit.md#audit-methodology) that closes the NOCEIL-1 … NOCEIL-4 set. The four-item ceiling list itself, with its per-ceiling routing-class one-liner, is owned by [`spec.md` — Hard ceilings](../spec/overview-and-orientation.md#hard-runtime-ceilings) as an aggregator paragraph per [Governance — GOV-30](./governance.md); this page carries the load-bearing definitions that aggregator forward-links into.

*Identifier classes used on this page.* The `HC3-a` … `HC3-e` and `NOCEIL-1` … `NOCEIL-4` markers below are *stable inline labels* in the second governed identifier class introduced at [Governance — GOV-16](./governance/stable-inline-labels.md#gov-16); the `HC3` and `NOCEIL` prefixes are registered on this page in GOV-16's per-page inline-label prefix table. The `CIO-1` … `CIO-6` markers are full REQ-IDs under the `CIO` prefix — the second of two prefixes co-owned by the `hard-ceilings/` subtree (the other being `CEIL`) per the [REQ-ID prefix table](./governance/req-id-prefix-table-active-a.md#req-id-prefix-table) and GOV-24's many-to-one prefix → page binding.

Each per-ceiling rule's *first* enforcement point still lives on the topic page that owns the breach surface — `loom/runtime/invoke-depth-exceeded` on [Invocation](./invocation.md), `Err(QueryError { kind: "tool_loop_exhausted", … })` on [Errors and Results](./errors-and-results.md) and [Query](./query.md), the binder retry classifier on [Slash-Command Argument Binding](./binder.md), the depth-walk on [Schema Subset](./schema-subset.md). This page pins the *cross-ceiling* obligations that no single per-ceiling page can own.
