# `V9r` — Session-swap fail-fast tripwire

**Spec.** [`../spec_topics/pi-integration-contract/session-only-degraded-state.md`](../spec_topics/pi-integration-contract/session-only-degraded-state.md) (the *Session-swap fail-fast tripwire — governed-by-rebind resolution* section), [`../spec_topics/pi-integration-contract/host-prerequisites.md`](../spec_topics/pi-integration-contract/host-prerequisites.md#degraded-state-host-prerequisites) (clause (a), resolved `governed-by-rebind`), [`../spec_topics/pi-integration-contract/drain-state-contract.md`](../spec_topics/pi-integration-contract/drain-state-contract.md), [`../spec_topics/diagnostics/code-registry-host.md`](../spec_topics/diagnostics/code-registry-host.md) (the `loom/host/session-swap-instance-survived` row).

**Adds.** The session-swap fail-fast tripwire — the recovery action that replaces the retired session-only degraded-state branch (`V9l`, never shipped) under the recorded `governed-by-rebind` determination. This leaf: (1) confirms the session-only reasons (`"new"`/`"resume"`/`"fork"`) run the same full teardown as the always-tear-down reasons, wiring **no** `markRuntimeDegraded` transition, **no** `"degraded-needs-reload"` tag write, **no** `loom/host/session-shutdown-runtime-degraded` emission, and **no** degraded slash note; (2) arms a private per-extension-instance `sessionSwapTornDown` tripwire on the closed-over `LoomRegistry` from the `session_shutdown` handler when `capturedEventReason ∈ {"new","resume","fork"}` (the flag is not part of the public `readDrainState` snapshot surface); (3) guards every loom-registered slash `handler` (at entry, before dispatch) and the `session_start` handler with a tripwire check that, if the flag is ever observed set, emits one `loom/host/session-swap-instance-survived` (E, runtime) diagnostic via `console.error` and then terminates the process on the NFR-2.1 fail-fast path — surfacing a future-Pi lifecycle-contract violation loudly at the contradiction point rather than running unspecified degraded logic.
