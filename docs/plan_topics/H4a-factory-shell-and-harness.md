# `H4a` — Extension factory shell and end-to-end harness

**Convention.** [`conventions.md`](./conventions.md) (phase categories — Pi-extension shell, end-to-end harness).

**Adds.** The Pi extension factory entry point — the factory implementation lives under `src/**` (the single production-code root the [`conventions.md`](./conventions.md) cross-cutting gates scope to), and the on-disk module `extensions/index.ts` (the file Pi resolves under the `package.json#pi.extensions = ["./extensions"]` package-discovery directory [`H1a`](./H1a-scaffold-and-toolchain.md) declares, per [`host-prerequisites.md` §"Extension entry point"](../spec_topics/pi-integration-contract/host-prerequisites.md); the manifest entry is a directory and `extensions/index.ts` is the file resolved under it) is a **thin delegating entry shim** that carries no rule-subject logic and re-exports that `src/**` factory as the standard `default function (pi: ExtensionAPI)` export. The factory's declared return type stays `void | Promise<void>` for host-interface conformance, but loom pins it to the **synchronous arm**: the factory body runs synchronously and returns `void` — it is not declared `async` and awaits no work, so there is no returned `Promise` arm that could later reject. It establishes the extension by side-effect registration calls on the injected `pi` handle (`pi.registerMessageRenderer`, `pi.registerCommand`, `pi.registerFlag`, `pi.registerTool`, `pi.on`) under per-call `try`/`catch`, so that — because the body returns synchronously with no awaited work — "MUST NOT throw out of the factory body" is the complete never-fault prohibition for this boundary — and a reusable end-to-end test harness that loads the extension against an in-process Pi session double and drives a slash dispatch. The harness also defines an **in-memory fixture-supply mechanism**: the harness supplies the fixture `.loom` content to the extension in memory so a harness-driven leaf's discovery reads from the harness-provided source rather than the real filesystem — no ambient `src/**` filesystem read, no `FileSystem` seam (`V8b`) dependency, keeping the `H3a` ambient-access scan clean. `M` and `M-T` consume this mechanism for their single-source happy-path discovery. The harness is established together with the **session-double fidelity contract** — the enumerated set of pinned-`@earendil-works/pi-coding-agent` behaviours the in-process double must reproduce: streamed-token order relative to `ctx.waitForIdle()` resolution, single-turn prompt-mode append semantics, the `pi.on` cancel-forward subscription, and cancellation propagation. The full scripted-injection **response-programming surface** the double's input side exposes is owned by [`H4b`](./H4b-response-programming-surface.md), which depends on this leaf. The capability-probe refusal logic is added by `V9a`; this leaf only establishes the never-throw factory boundary, the harness, and the fidelity contract its self-check enforces.

The manual real-host smoke governance protocol that gates a double-vs-real divergence — its two triggers, named owners, activation window, live-host precondition, pass/fail criterion, and merge-blocker / revert path — is a release-gate / version-bump governance obligation addressed to the bump/merge contributor and the [`H6a`](./H6a-live-corpus-activation.md) release-gate auditor, not to the `H4a` implementer; it lives in [`real-host-smoke-gate.md`](./real-host-smoke-gate.md) and consumes this leaf's harness only as a downstream reader.

**Tests.**
- `Convention:` (phase categories) the factory completes its side-effect registrations on the injected `pi` handle and never throws even when a host seam is absent (each host-binding call is `try`/`catch`-wrapped per the exempt-broad-catch sites in `conventions.md`), and the factory returns **synchronously** — its return value is `undefined`, not a thenable, witnessing the synchronous-arm pin (a non-`async`, non-`Promise` return) so the never-fault property covers the returned-value path and not merely the synchronous-throw path.
- `Convention:` (end-to-end harness) the harness loads the extension and asserts a registered command can be dispatched end-to-end against the session double.
- `Convention:` (end-to-end harness) the **session-double fidelity contract** is asserted by a harness self-check: the in-process double reproduces, relative to the pinned `@earendil-works/pi-coding-agent` SDK, (i) streamed assistant tokens observable in the transcript before `ctx.waitForIdle()` resolves, (ii) one streamed assistant response appended as a single prompt-mode turn, (iii) the `pi.on` cancel-forward subscription, and (iv) cancellation propagation. The self-check asserts these behaviours *as modelled by the in-process double* — the harness deliberately avoids a live Pi session, so it cannot detect the double diverging from real Pi, and loom 1.0 has no mechanical real-host fidelity gate (`V18d`'s version-bump runtime-evidence gate also runs against this same double and is explicitly not real-host coverage — see [`V18d`](./V18d-version-bump-acceptance.md)). The manual real-host smoke run that gates a double-vs-real divergence is **not** part of this self-check and never runs under `npm test`; it is defined as a release-gate / version-bump governance protocol in [`real-host-smoke-gate.md`](./real-host-smoke-gate.md).

**Deps.** `H3a`

**Ships when.** `npm test` loads the extension through the harness, dispatches a no-op command end-to-end, and the session-double fidelity-contract self-check passes.
