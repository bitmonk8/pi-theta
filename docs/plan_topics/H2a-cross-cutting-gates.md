# `H2a` — Cross-cutting lint and architectural gates

**Convention.** [`conventions.md`](./conventions.md) — *No globals, statics, singletons*; *Specific exception types only*; *Sequential by default*; *No silent test skipping*.

**Adds.** The ESLint rules (`no-broad-catch` — permitting a broad catch only at an exempt-site shape whose same line carries the `// allow-broad-catch:` comment, the set of those comments across `src/**` being the lint allow-list, with no separate allow-list file read; and the `no-restricted-syntax` sequential-by-default allow-list; and a `no-restricted-syntax` blocking-runtime check that flags the synchronous calls the *Sequential by default* ban names — every call whose callee name ends in the literal suffix `Sync`, whether a member access such as `fs.readFileSync` or a bare identifier such as `execSync` (those two cited as examples of the recognised form, not a finite set standing alongside an undefined residual family) — over `src/**` (excluding `**/*.test.ts`), permitting a call only where its line carries a `// allow-sync: <reason>` comment, the set of those comments across `src/**` being the exemption allow-list with no separate allow-list file read; the check is an approximation that catches every statically named callee whose name ends in `Sync`, not busy-wait loops or dynamically-constructed sync calls) and the `src/**` architectural test that fails on a module-level mutable binding — a top-level `let`/`var`, or a top-level `const` whose initializer is a mutable object or array literal, declared outside any class or function (such a binding is banned because the object it names is then shared and mutated across calls; the AST scan decides the ban on the observable initializer shape, not on runtime reuse, since closure-captured, lazy module-cache, and DI-container singletons are not mechanically detected; that residue is owned by the *Per-phase TDD ritual* self-review step in [`conventions.md`](./conventions.md)), wired into `npm test`.

**Tests.**
- `Convention:` (*Specific exception types only*) a fixture file containing `catch (e: unknown)` with no `// allow-broad-catch:` comment fails lint; an exempt site whose same-line `// allow-broad-catch:` comment cites any token the widened gate predicate admits — a coverage-matrix REQ-ID, a *Code-keyed obligation areas* row's canonical `cka-<n>` token (matching exactly one row's *Token* cell in [`coverage-matrix.md`](./coverage-matrix.md)), a concrete `loom/...` registry code, or the structural `pi-sdk-boundary` token — passes; the same-line comment is itself the allow-list entry and the lint reads no separate allow-list file.
- `Convention:` (*Sequential by default*) a fixture using `Promise.all` in `src/**` without an allow comment fails lint; `**/*.test.ts` is unrestricted.
- `Convention:` (*Sequential by default* — blocking runtime) a fixture using a blocking `fs.readFileSync` or `execSync` call in `src/**` without an `// allow-sync:` comment fails lint; the same call with a same-line `// allow-sync: <reason>` comment passes; `**/*.test.ts` is unrestricted.
- `Convention:` (*No globals, statics, singletons*) a fixture introducing a module-level mutable binding of a flagged form — a top-level `let`/`var`, or a top-level `const` whose initializer is a mutable object or array literal, declared outside any class or function — fails the architectural test.

**Deps.** `H1a`

**Ships when.** `npm test` includes passing lint + architectural assertions that reject a broad catch, an unguarded `Promise.all`, an unexempted blocking `fs.readFileSync`/`execSync` call, and a module-level mutable binding of a flagged form in `src/**`.
