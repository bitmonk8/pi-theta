**Tests.**
- `Convention:` (*Phase 1 — automated non-interactive acceptance*) the suite drives each feature loom via `pi -p --loom <dir> "/<name>"` against a live host and asserts, per loom, its invariant set — **no-error exit**, binder output validates against the binder envelope schema where a binder pass fires, typed-query responses validate against their declared schema (`QRY-22`), observed cancellation propagation, and emitted `loom-system-note` codes ⊆ the committed permitted-code list. The fuller feature-loom set covers, at minimum, distinct areas: (a) prompt-mode sentinel turn; (b) typed query with a **named** `schema` decl; (c) typed query with an **inline** object type; (d) a **params** loom that forces a real **binder** pass (envelope validity); (e) a **subagent-mode** loom exercising spawn + mid-stream **cancellation** with committed turns unmutated; (f) a **code-tool loop**; (g) **imports / invoke** across looms; (h) **error/result `match`** surfacing a `QueryError`; (i) **multi-source discovery** (project + `--loom` CLI source). The suite reds today because the runner and looms are absent and the (b)/(d)/(e) axes are not yet correct.
