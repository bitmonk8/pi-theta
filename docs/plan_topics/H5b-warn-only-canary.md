# `H5b` — Warn-only live-corpus canary (pre-activation pre-flight)

**Convention.** [`conventions.md`](./conventions.md) — *REQ-ID discipline* (the warn-only live-corpus pre-flight of the closing gate) and the [`coverage-matrix.md`](./coverage-matrix.md) closure obligation.

**Adds.** A warn-only live-corpus pre-flight that runs [`H5a`](./H5a-closing-gate-automation.md)'s closing gate in its **warn-only live-corpus mode** immediately before [`H6a`](./H6a-live-corpus-activation.md)'s hard-fail flip. It reconciles the same three live surfaces `H6a` hard-fails on — the live spec REQ-ID set, the live `spec_topics/**` normative-MUST set, and the live test corpus, all against the live [`coverage-matrix.md`](./coverage-matrix.md) — but emits each gap (an unmapped executable REQ-ID, a coverage-matrix-mapped numbered REQ-ID with no citing test, or an un-enumerated un-anchored MUST) as a CI finding **without** reddening `npm test`. Because it runs the identical [`H5a`](./H5a-closing-gate-automation.md) gate machinery over the identical live surfaces, its warn-only reconciliation set stays in lockstep with `H6a`'s hard-fail set by construction — no second copy of the surface definitions is authored here. This canary surfaces live coverage gaps to contributors before the hard flip can redden `main`, so a missing `H6a` dependency or unclosed coverage is visible while CI is still green. Like [`H6a`](./H6a-live-corpus-activation.md)'s hard-fail footing it reconciles the live spec/test corpus exclusive of the dedicated test-fixtures root [`H5a`](./H5a-closing-gate-automation.md) places the seeded gate fixtures under, so the pre-flight emits no spurious finding on a permanent-by-design seeded violation.

**Tests.**
- `Convention:` (*REQ-ID discipline* — warn-only canary) the canary reconciles the live spec REQ-ID set, the live `spec_topics/**` normative-MUST set, and the live test corpus against the live [`coverage-matrix.md`](./coverage-matrix.md) and emits a finding for each gap kind — an unmapped executable REQ-ID, a coverage-matrix-mapped numbered REQ-ID with no citing test, and an un-enumerated un-anchored MUST — while `npm test` stays green (asserted by seeding one gap of each kind, confirming the canary reports it, and confirming the run's exit status stays green, then restoring the seed).

_(The dependency set is the complete coverage-producing set: [`H5a`](./H5a-closing-gate-automation.md) supplies the gate machinery including its warn-only mode, and every MVP and vertical implementation leaf (`V1a`–`V18d`) can introduce an executable REQ-ID, a numbered-REQ-ID citing test, or an un-anchored normative MUST. Keeping this set transitively complete as new coverage-producing leaves are added is the standing plan-maintenance obligation stated in *REQ-ID discipline* on [`conventions.md`](./conventions.md); [`H6a`](./H6a-live-corpus-activation.md) inherits this set by depending on this leaf rather than restating it, so the warn-only and hard-fail footings cover an identical set of leaves.)_

**Ships when.** `npm test` runs the warn-only canary on the live-corpus footing ahead of [`H6a`](./H6a-live-corpus-activation.md) — the live spec REQ-ID set, the live `spec_topics/**` normative-MUST set, and the live test corpus reconciled against the live [`coverage-matrix.md`](./coverage-matrix.md) — and the seeded-gap checks above confirm it emits a finding for each gap kind while `npm test` stays green.
