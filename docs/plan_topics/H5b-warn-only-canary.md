# `H5b` ‚Äî Warn-only live-corpus canary (pre-activation pre-flight)

**Convention.** [`conventions.md`](./conventions.md) ‚Äî *REQ-ID discipline* (the warn-only live-corpus pre-flight of the closing gate) and the [`coverage-matrix.md`](./coverage-matrix.md) closure obligation.

**Adds.** A warn-only live-corpus pre-flight that runs [`H5a`](./H5a-closing-gate-automation.md)'s closing gate in its **warn-only live-corpus mode** immediately before [`H6a`](./H6a-live-corpus-activation.md)'s hard-fail flip. It reconciles the same three live surfaces `H6a` hard-fails on ‚Äî the live spec REQ-ID set, the live `spec_topics/**` normative-MUST set, and the live test corpus, all against the live [`coverage-matrix.md`](./coverage-matrix.md) ‚Äî but emits each gap (an unmapped executable REQ-ID, a coverage-matrix-mapped numbered REQ-ID with no citing test, a multi-leaf coverage-matrix-row facet whose closing leaf carries no facet-naming citing test, or an un-enumerated un-anchored MUST) as an entry in the gate machinery's **returned** warn-only findings collection ‚Äî the in-process representation [`H5a`](./H5a-closing-gate-automation.md)'s Adds fixes, one structured entry per gap tagged with its gap kind ‚Äî **without** reddening `npm test`. Because it runs the identical [`H5a`](./H5a-closing-gate-automation.md) gate machinery over the identical live surfaces, its warn-only reconciliation set stays in lockstep with `H6a`'s hard-fail set by construction ‚Äî no second copy of the surface definitions is authored here. This canary surfaces live coverage gaps to contributors before the hard flip can redden `main`, so a missing `H6a` dependency or unclosed coverage is visible while CI is still green. Like [`H6a`](./H6a-live-corpus-activation.md)'s hard-fail footing it reconciles the live spec/test corpus exclusive of the dedicated test-fixtures root [`H5a`](./H5a-closing-gate-automation.md) places the seeded gate fixtures under, so the pre-flight emits no spurious finding on a permanent-by-design seeded violation.

**Tests.**
- `Convention:` (*REQ-ID discipline* ‚Äî warn-only canary) the canary reconciles the live spec REQ-ID set, the live `spec_topics/**` normative-MUST set, and the live test corpus against the live [`coverage-matrix.md`](./coverage-matrix.md) and emits a finding for each gap kind ‚Äî an unmapped executable REQ-ID, a coverage-matrix-mapped numbered REQ-ID with no citing test, a multi-leaf coverage-matrix-row facet whose closing leaf carries no facet-naming citing test, and an un-enumerated un-anchored MUST ‚Äî while `npm test` stays green (asserted by seeding one gap of each kind, asserting the gate machinery's returned warn-only findings collection contains a structured entry tagged with that gap kind, and confirming the run's exit status stays green, then restoring the seed).
