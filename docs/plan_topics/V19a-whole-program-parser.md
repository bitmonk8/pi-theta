# `V19a` — Loom whole-program parser

**Spec.** [`../spec_topics/implementation-notes.md`](../spec_topics/implementation-notes.md), [`../spec_topics/grammar.md`](../spec_topics/grammar.md), [`../spec_topics/lexical.md`](../spec_topics/lexical.md), [`../spec_topics/bindings.md`](../spec_topics/bindings.md), [`../spec_topics/control-flow.md`](../spec_topics/control-flow.md), [`../spec_topics/functions.md`](../spec_topics/functions.md), [`../spec_topics/return.md`](../spec_topics/return.md), [`../spec_topics/expressions.md`](../spec_topics/expressions.md), [`../spec_topics/frontmatter.md`](../spec_topics/frontmatter.md), [`../spec_topics/descriptions.md`](../spec_topics/descriptions.md), [`../spec_topics/schemas.md`](../spec_topics/schemas.md), [`../spec_topics/imports.md`](../spec_topics/imports.md), [`../spec_topics/invocation.md`](../spec_topics/invocation.md), [`../spec_topics/diagnostics.md`](../spec_topics/diagnostics.md). _(Closed under normative cross-link: `grammar.md`'s statement/block productions are owned by the surface pages above (`bindings.md` `LetStmt`, `control-flow.md` `IfStmt`/`WhileStmt`/`ForStmt`, `functions.md` `FnDecl`, `return.md`, `expressions.md` block/`?` forms, `descriptions.md` `DocComment`, `schemas.md` `SchemaDecl`/`EnumDecl`, `imports.md` import forms) and `implementation-notes.md` §Parser normatively cites `diagnostics.md`; each is a non-narrative page.)_

**Adds.** Introduces the whole-`.loom`/`.warp` parser producing an executable body statement-list AST (the `LoomBody ::= Stmt* Expr?` node types plus the statement/block tokeniser) alongside frontmatter, wiring the existing `V3b`/`V3c`/`V3d`/`V4a`/`V5a`–`V5c`/`V6a`–`V6c`/`V13b`/`V15c`/`V15f` parse-checkers over the real AST and aggregating their diagnostics — this body AST is the seam `V19c`'s executor walks and `V19e`'s composition producer parses.

**Tests.**
- `cka-49` ([implementation-notes.md](../spec_topics/implementation-notes.md) §Parser *Contract* — the un-anchored whole-file-parse MUST): `parseLoomDocument(source)` parses the *entire* `.loom`/`.warp` file into a `LoomBody = Stmt* Expr?` body statement-list AST (grammar.md §"Block expressions" `LoomBody`), returning `{ frontmatter, body, diagnostics }`; the whole file — not a single expression — is walked into the executable body the interpreter walks, per the §Parser MUST "multi-error reporting across the whole `.loom` or `.warp` file … before failing".
- `cka-49` (statement-kind coverage): `let` / `let mut` (`LetStmt`), statement-form reassignment including `+=`, `if` / `else` / `while` / `for` (`IfStmt` / `WhileStmt` / `ForStmt`), `break` / `continue`, top-level `fn` (`FnDecl`), `return`, `@`…`` query statements, code-tool calls `<name>(args)`, `invoke(...)`, bare expression statements, and the optional tail `Expr` each parse into the corresponding body-AST `Stmt` / `Expr` node.
- `cka-49` (declaration-kind coverage): `schema` (`SchemaDecl`), `enum` (`EnumDecl`), `import` / `export … from`, and `///` doc-comment runs (`DocComment`) parse into their body-AST declaration nodes at top level.
- `cka-49` (newline continuation, grammar.md §"Newline continuation"): statements close at the newline unless a continuation trigger holds (unmatched open `(` / `[` / `{`, trailing or leading binary-or-ternary operator, trailing comma); the postfix error-propagation `?` never triggers continuation, and blank lines do not break a held continuation.
- `cka-49` (whole-file multi-error diagnostic aggregation): a document with multiple independent statement errors emits every diagnostic in one pass (no fast-fail), sorted `(file, line, col)`, by delegating each statement/declaration to its existing checker over the real AST as integration witnesses — invokes `V3b`'s `checkReassignment` / `checkMutModifier` (`loom/parse/immutable-rebinding`, BNDS), `V3c`'s `checkForIterand` (CTRL), `V3d`'s `checkBareReturn` / `checkUnreachableCode` (FN), `V4a`'s `checkQuestionScope` (ERR `?`-operator), `V5a`–`V5c`'s `checkObjectSchema` / `checkDiscriminatedUnion` / `checkDocCommentPlacement` (SCHM / DESC), `V6a`–`V6c`'s `parseFrontmatter` / `parseParams` / `resolveCallableSet` (FRNT), `V13b`'s `inferQuerySchema` (QRY), `V15c`'s `checkImportedSymbols` (IMP), and `V15f`'s `checkInvokeArgTypes` / `checkInvokeReturnType` (INV) — without re-closing those checkers' owning REQ-IDs.

**Deps.** `V19a-T`, `V1a`, `V1b`, `V2a`, `V3b`, `V3c`, `V3d`, `V4a`, `V5a`, `V5b`, `V5c`, `V6a`, `V6b`, `V6c`, `V13b`, `V15c`, `V15f`, `V7a`

**Ships when.** `npm test` includes passing assertions that `parseLoomDocument(source)` returns a body statement-list AST covering every statement kind (`let`/reassignment/`if`/`while`/`for`/`break`/`continue`/`fn`/`return`/`@`-query/tool-call/`invoke`/bare-expr/tail-expr/`schema`/`enum`/import/`///`), applies the grammar.md newline-continuation triggers, and aggregates multi-error diagnostics sorted `(file, line, col)` by delegating to the existing `V3b`/`V3c`/`V3d`/`V4a`/`V5a`–`V5c`/`V6a`–`V6c`/`V13b`/`V15c`/`V15f` checkers over the real AST (`cka-49`).
