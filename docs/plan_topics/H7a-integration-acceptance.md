# `H7a` — Terminal integration-acceptance run (cross-slice end-to-end gate)

**Convention.** [`conventions.md`](./conventions.md) (phase categories — end-to-end harness).

**Adds.** A terminal integration-acceptance gate that runs a single representative multi-feature fixture `.loom` through the [`H4a`](./H4a-factory-shell-and-harness.md) end-to-end harness against the in-process Pi session double, driving the integrated pipeline — typed query → tool loop → code-tool invoke → schema lowering/validation → binder → cancellation — in a single run. It closes no new spec REQ-ID; it is a cross-slice integration-regression gate that exercises the composition the per-leaf gates verify only in isolation. Because the run executes against the session double, its fidelity is bounded by that double's contract ([`H4a`](./H4a-factory-shell-and-harness.md)); the gate targets the cross-slice composition the per-leaf gates cannot see, not host-level realism (the real-host backstop remains [`V18d`](./V18d-version-bump-acceptance.md)'s version-bump runtime-evidence gate). The committed golden transcript, golden diagnostics list, and permitted-code list are *derived*, not snapshotted from an unreviewed pipeline run: each golden turn and each `loom/...` code that maps onto a behaviour already gated by a leaf in **Deps** (`V5d`, `V8a`, `V11a`, `V11b`, `V11i`, `V11c`, `V11d`, `V11e`, `V11f`, `V13a`, `V13b`, `V13c`, `V13d`, `V13f`, `V14a`, `V14b`, `V16a`, `V17a`) is sourced from that leaf's already-established expected output — the golden diagnostics list and transcript from the subset that path actually emits, mechanically containment-checked against the permitted-code list (Tests below) — and the composed turn ordering no single per-leaf gate covers is human-reviewed against [`docs/spec.md`](../spec.md) and the cited spec topics at first commit. The permitted-code list is a **manually-maintained, best-effort** union of each Deps slice's emittable codes, kept current by a per-Deps-slice provenance obligation: with no emittable-code registry to verify against, its completeness — that it captures every code each slice *can* emit — is not mechanically verified, so a stale union can still omit a benign code a slice emits. Only the `golden ⊆ permitted` containment is mechanically gated (Tests below).

**Tests.**
- `Convention:` (phase categories — end-to-end harness) running the representative multi-feature fixture `.loom` through the `H4a` harness drives the integrated pipeline (typed query, tool loop, code-tool invoke, schema lowering/validation, binder, cancellation) and the run's appended turns match — in order, count, and per-turn content (each turn's text, tool arguments, and rendered envelope) — the committed golden transcript checked in alongside the fixture `.loom` (the enumerated, ordered turn sequence the composition produces).
- `Convention:` (phase categories — end-to-end harness) the same run emits exactly the `loom-system-note` diagnostics enumerated in the committed golden diagnostics list checked in alongside the fixture `.loom` — the set of `loom/...` codes the integrated fixture path emits, drawn only from codes the slices in **Deps** emit (no code that no slice in the composition emits) — each asserted against the diagnostics-registry *Message* strings per [`conventions.md`](./conventions.md) *Diagnostic message anchors*.
- `Convention:` (phase categories — end-to-end harness) a **committed permitted-code list** is checked in alongside the fixture `.loom`, next to the committed golden transcript and golden diagnostics list: it is a **manually-maintained, best-effort** union of the `loom/...` codes the slices in **Deps** (`H4a`, `H4b`, `V5d`, `V8a`, `V11a`, `V11b`, `V11i`, `V11c`, `V11d`, `V11e`, `V11f`, `V13a`, `V13b`, `V13c`, `V13d`, `V13f`, `V14a`, `V14b`, `V16a`, `V17a`) *can* emit, kept current by a per-Deps-slice provenance obligation rather than verified against an emittable-code registry — deliberately broader than the golden diagnostics list (which enumerates the codes the deterministic fixture path actually emits) and intended as a **superset** of it. Its completeness — that it captures every code each slice can emit — is **not** mechanically verified, so a stale union can omit a benign code a slice emits; only the `golden ⊆ permitted` containment is gated (next bullet), and that gate is what guarantees the in-process gate and the real-host smoke cannot disagree on a code the deterministic fixture path actually emits. It is the single committed, reviewed reference set the [`H4a`](./H4a-factory-shell-and-harness.md) manual real-host smoke run's pass criterion (e) and the [`H6a`](./H6a-live-corpus-activation.md) release-gate evidence record check the live run's emitted codes against, replacing a per-run by-hand reconstruction of the union.
- `Convention:` (phase categories — end-to-end harness) a containment check reads the committed golden diagnostics list and the committed permitted-code list — both checked-in artifacts read directly, neither re-derived from a pipeline run — and asserts `golden ⊆ permitted`: every code enumerated in the golden diagnostics list appears in the permitted-code list, reddening `npm test` when a golden code is not permitted, so a maintainer cannot add a golden code without permitting it and the containment the *superset* wording relies on holds by gate rather than by construction.
- `Convention:` (phase categories — end-to-end harness) the integrated run drives a binder-retry-class breach (ceiling #3, [`V11f`](./V11f-binder-retry-taxonomy.md)) co-occurring with a runtime-class ceiling (the `tool_loop.max_rounds` round boundary, ceiling #2, [`V13c`](./V13c-query-tool-loop.md)) through the live pipeline and asserts that the live sites' consultation of the [`V16a`](./V16a-ceiling-order-masked.md) arbitration seam surfaces exactly one ceiling in `CIO-5` order (#3 ahead of the runtime-class ceiling, no interleave) with `masked` enumerating the suppressed sibling — closing the live-site integration gap that `V16a`'s synthesised-candidate suite leaves open.
