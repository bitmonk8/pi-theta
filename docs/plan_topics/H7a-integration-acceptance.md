# `H7a` — Terminal integration-acceptance run (cross-slice end-to-end gate)

**Convention.** [`conventions.md`](./conventions.md) (phase categories — end-to-end harness).

**Adds.** A terminal integration-acceptance gate that runs a single representative multi-feature fixture `.loom` through the [`H4a`](./H4a-factory-shell-and-harness.md) end-to-end harness against the in-process Pi session double, driving the integrated pipeline — typed query → tool loop → code-tool invoke → schema lowering/validation → binder → cancellation — in a single run. It closes no new spec REQ-ID; it is a cross-slice integration-regression gate that exercises the composition the per-leaf gates verify only in isolation. Because the run executes against the session double, its fidelity is bounded by that double's contract ([`H4a`](./H4a-factory-shell-and-harness.md)); the gate targets the cross-slice composition the per-leaf gates cannot see, not host-level realism (the real-host backstop remains [`V18c`](./V18c-version-bump-checklist.md)'s version-bump runtime-evidence gate).

**Tests.**
- `Convention:` (phase categories — end-to-end harness) running the representative multi-feature fixture `.loom` through the `H4a` harness drives the integrated pipeline (typed query, tool loop, code-tool invoke, schema lowering/validation, binder, cancellation) and the run's appended turns match — in order and count — the committed golden transcript checked in alongside the fixture `.loom` (the enumerated, ordered turn sequence the composition produces).
- `Convention:` (phase categories — end-to-end harness) the same run emits exactly the `loom-system-note` diagnostics enumerated in the committed golden diagnostics list checked in alongside the fixture `.loom` — the set of `loom/...` codes the integrated fixture path emits, drawn only from codes the slices in **Deps** emit (no code that no slice in the composition emits) — each asserted against the diagnostics-registry *Message* strings per [`conventions.md`](./conventions.md) *Diagnostic message anchors*.
- `Convention:` (phase categories — end-to-end harness) the integrated run drives a binder-retry-class breach (ceiling #3, [`V11f`](./V11f-binder-retry-taxonomy.md)) co-occurring with a runtime-class ceiling (the `tool_loop.max_rounds` round boundary, ceiling #2, [`V13c`](./V13c-query-tool-loop.md)) through the live pipeline and asserts that the live sites' consultation of the [`V16a`](./V16a-ceiling-order-masked.md) arbitration seam surfaces exactly one ceiling in `CIO-5` order (#3 ahead of the runtime-class ceiling, no interleave) with `masked` enumerating the suppressed sibling — closing the live-site integration gap that `V16a`'s synthesised-candidate suite leaves open.

**Deps.** `H4a`, `V5d`, `V8a`, `V11f`, `V13c`, `V14a`, `V16a`, `V17a`

**Ships when.** `npm test` runs the terminal integration-acceptance fixture `.loom` end-to-end through the `H4a` harness against the in-process Pi session double, green: the integrated typed-query → tool-loop → code-tool-invoke → schema-lowering/validation → binder → cancellation pipeline produces appended turns matching the committed golden transcript (order and count) and emits exactly the `loom-system-note` codes enumerated in the committed golden diagnostics list checked in alongside the fixture `.loom`, and a co-occurring ceiling #3 / runtime-class ceiling #2 breach driven through the live pipeline surfaces exactly one ceiling in `CIO-5` order (#3 ahead of the runtime-class ceiling) with `masked` enumerating the suppressed sibling.
