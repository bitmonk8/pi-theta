# `V19c` — Loom tree-walking statement executor

**Spec.** [`../spec_topics/implementation-notes.md`](../spec_topics/implementation-notes.md), [`../spec_topics/cancellation.md`](../spec_topics/cancellation.md), [`../spec_topics/control-flow.md`](../spec_topics/control-flow.md), [`../spec_topics/functions.md`](../spec_topics/functions.md), [`../spec_topics/return.md`](../spec_topics/return.md), [`../spec_topics/errors-and-results/error-model.md`](../spec_topics/errors-and-results/error-model.md) (with their transitive normative closure).

**Adds.** The tree-walking statement executor that walks [`V19a`](./V19a-whole-program-parser.md)'s body statement AST top-to-bottom against [`V19b`](./V19b-lexical-environment.md)'s lexical environment — `let`/reassign, `if`/`while`/`for` (driving the real `ForLoopHost` / `evaluateForLoop` from [`V3c`](./V3c-control-flow.md)), `break`/`continue`, `return`, `match`, and expression-statements — segmenting each linear run of statements onto [`V17a`](./V17a-cancellation-core.md)'s `runCancellableSequence` (`CancellableStatement` / `CancellableSequenceDeps`) so the five fixed checkpoint sites gate real work, and producing the [`functions.md` FN-5](../spec_topics/functions.md#fn-5) top-level-block final value together with the [`error-model.md`](../spec_topics/errors-and-results/error-model.md#terminal-outcomes) terminal outcome; this executor is the seam [`V19d`](./V19d-effectful-statement-wiring.md) supplies real effectful hosts to and [`V19e`](./V19e-composition-producer.md)'s producer drives.

**Tests.**
- Statement-execution driver ([implementation-notes.md — *Runtime*](../spec_topics/implementation-notes.md#runtime), [`coverage-matrix.md`](./coverage-matrix.md) code-keyed-area token `cka-50`, `V19c` closure): drive the executor over a multi-statement body AST and assert it walks the loom body top-to-bottom, strictly sequentially — each statement's effect commits before the next statement is entered, no statement runs ahead of a prior one — realising the un-anchored driver/sequencing obligation the *Runtime* bullets pin ("drives it turn-by-turn"; "Within a single invocation the interpreter is strictly sequential … the next loom expression cannot run until the awaited Promise resolves"). This is the only **new** coverage-matrix row this leaf closes.
- Control-flow statements against real hosts ([`control-flow.md` CTRL-1](../spec_topics/control-flow.md#ctrl-1), integration witness — not re-closing CTRL-1, which [`V3c`](./V3c-control-flow.md) owns): `for`/`while`/`if` and `break`/`continue` execute through the executor against the real `ForLoopHost`, with the iterand evaluated exactly once at loop entry and `break`/`continue` steering the real loop; witnessed here at real hosts, not as a new closure.
- Tail-expression / empty-body final value ([`functions.md` FN-4](../spec_topics/functions.md#fn-4)/[FN-5](../spec_topics/functions.md#fn-5), [`return.md` RET-1](../spec_topics/return.md#ret-1)/[RET-2](../spec_topics/return.md#ret-2)/[RET-3](../spec_topics/return.md#ret-3), integration witnesses — not re-closing them): a body ending in a tail expression yields that expression's value, a statement-terminated or empty body yields the literal `null`, and an explicit `return expr` short-circuits to its operand; asserted through the executor's `FunctionResult` as end-to-end witnesses of rules owned on their feature leaves.
- Five-site checkpoint gating on real `runCancellableSequence` ([cancellation.md — *Granularity*](../spec_topics/cancellation.md), [`coverage-matrix.md`](./coverage-matrix.md) code-keyed-area token `cka-47`, integration witness — not re-closing `cka-47`, which [`V17a`](./V17a-cancellation-core.md)/[`V17c`](./V17c-checkpoint-granularity.md) own): the executor segments linear runs onto [`V17a`](./V17a-cancellation-core.md)'s real `runCancellableSequence`, and a signal flipped mid-body preempts at the next checkpointed sub-expression while a straight-line statement boundary is not a checkpoint ([cancellation.md — *Statement boundaries are not checkpoints*](../spec_topics/cancellation.md)); witnessed against the real sequence runner.
- Terminal-outcome production ([`error-model.md` ERR-8](../spec_topics/errors-and-results/error-model.md#err-8)/[ERR-9](../spec_topics/errors-and-results/error-model.md#err-9)/[ERR-10](../spec_topics/errors-and-results/error-model.md#err-10)/[ERR-11](../spec_topics/errors-and-results/error-model.md#err-11)/[ERR-12](../spec_topics/errors-and-results/error-model.md#err-12), integration witnesses — not re-closing them): the executor drives a body to each of the success / fail / cancel terminal outcomes and asserts the mid-stream-cancellation non-mutation / no-compensating-injection obligations hold across the driven body; witnessed here at real hosts, not as a new closure.

**Deps.** `V19c-T`, `V19a`, `V19b`, `V3c`, `V4a`, `V4b`, `V4c`, `V17a`, `V17c`, `V3d` — the paired tests task, the whole-program parser and lexical environment it walks, the control-flow / expression / match / return evaluators it drives, and the cancellation core and checkpoint granularity it segments onto.

**Ships when.** `npm test` drives a multi-statement loom body end-to-end through the executor — sequential top-to-bottom walk against the real environment and control-flow hosts — producing the correct FN-5 final value and terminal outcome with the five-site checkpoints honoured on the real `runCancellableSequence`.
