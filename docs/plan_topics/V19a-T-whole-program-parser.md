**Adds.** Failing tests for the paired `V19a` whole-program parser: they declare (or inertly stub) the `parseLoomDocument` seam so the suite compiles, then red on the body-AST-production and diagnostic-aggregation assertions the implementation must satisfy.

**Tests.**
- `cka-49` ([implementation-notes.md](../spec_topics/implementation-notes.md) §Parser *Contract* — the un-anchored whole-file-parse MUST): a failing test that `parseLoomDocument(source)` parses the *entire* `.loom`/`.warp` file into a `LoomBody = Stmt* Expr?` body statement-list AST (grammar.md §"Block expressions" `LoomBody`), returning `{ frontmatter, body, diagnostics }` — reds because no whole-file body AST is produced yet.
- `cka-49` (statement-kind coverage): failing assertions that `let` / `let mut` (`LetStmt`), statement-form reassignment including `+=`, `if` / `else` / `while` / `for` (`IfStmt` / `WhileStmt` / `ForStmt`), `break` / `continue`, top-level `fn` (`FnDecl`), `return`, `@`…`` query statements, code-tool calls `<name>(args)`, `invoke(...)`, bare expression statements, and the optional tail `Expr` each parse into the corresponding body-AST node.
- `cka-49` (declaration-kind coverage): failing assertions that `schema` (`SchemaDecl`), `enum` (`EnumDecl`), `import` / `export … from`, and `///` doc-comment runs (`DocComment`) parse into their top-level body-AST declaration nodes.
- `cka-49` (newline continuation, grammar.md §"Newline continuation"): a failing test that statements close at the newline unless a continuation trigger holds (unmatched open `(` / `[` / `{`, trailing or leading binary-or-ternary operator, trailing comma), that the postfix `?` never triggers continuation, and that blank lines do not break a held continuation.
- `cka-49` (whole-file multi-error diagnostic aggregation): a failing test that a document with multiple independent statement errors emits every diagnostic in one pass (no fast-fail), sorted `(file, line, col)`, by delegating each statement/declaration to its existing checker over the real AST as integration witnesses — `V3b`'s `checkReassignment` / `checkMutModifier` (`loom/parse/immutable-rebinding`, BNDS), `V3c`'s `checkForIterand` (CTRL), `V3d`'s `checkBareReturn` / `checkUnreachableCode` (FN), `V4a`'s `checkQuestionScope` (ERR), `V5a`–`V5c`'s `checkObjectSchema` / `checkDiscriminatedUnion` / `checkDocCommentPlacement` (SCHM / DESC), `V6a`–`V6c`'s `parseFrontmatter` / `parseParams` / `resolveCallableSet` (FRNT), `V13b`'s `inferQuerySchema` (QRY), `V15c`'s `checkImportedSymbols` (IMP), and `V15f`'s `checkInvokeArgTypes` / `checkInvokeReturnType` (INV) — without re-closing those checkers' owning REQ-IDs.

**Deps.** `V1a`, `V1b`, `V2a`, `V3b`, `V3c`, `V3d`, `V4a`, `V5a`, `V5b`, `V5c`, `V6a`, `V6b`, `V6c`, `V13b`, `V15c`, `V15f`, `V7a`

**Ships when.** The tests above exist, compile, and fail red for the intended reason.
