# `V9n` — Prompt-mode transport-error mapping

**Spec.** [`../spec_topics/pi-integration-contract/conversation-drive.md`](../spec_topics/pi-integration-contract/conversation-drive.md#prompt-mode-error-detection).

**Adds.** The prompt-mode transport-error mapping: consumes [`V9c`](./V9c-conversation-drive.md)'s `stopReason:"error"` probe-result and trailing-turn `Ok(string)` extraction seams to synthesise the `QueryError` — the `stopReason:"error"` → `kind:"transport"` mapping, the absent-`errorMessage` → `"provider transport failure"` fallback, the synchronous-`pi.sendUserMessage`-throw secondary mapping, and the `loomAbort.signal.aborted` cancellation short-circuit (`kind:"cancelled"`) that takes precedence over both the error probe and the `Ok(string)` extraction; the synthesised `transport` error's `provider` field is sourced from `V9j`'s provider-error-mapping surface, not re-derived here.

**Tests.**
- [conversation-drive.md — error detection](../spec_topics/pi-integration-contract/conversation-drive.md#pic-51) (PIC-51): after `waitForIdle()` resolves, a driven turn whose trailing `assistant` message carries `stopReason: "error"` maps to `Err(QueryError { kind: "transport", message: <errorMessage>, http_status: null, provider: <resolved-model-provider>, retryable: false })` — the `provider` value sourced from the loom's resolved `model:` `Model<Api>.api` value (the frontmatter `model:`, or the inherited `ctx.model` when `model:` is absent; resolution owned by V6a), per conversation-drive.md's resolved-`model:` rule and queryerror-variants.md's `TransportError.provider` derivation.
- [conversation-drive.md — error detection](../spec_topics/pi-integration-contract/conversation-drive.md#pic-51) (PIC-51): when that `stopReason: "error"` turn's `errorMessage` is absent, the synthesised `transport` error's `message` is the fixed string `"provider transport failure"`.
- [conversation-drive.md — error detection](../spec_topics/pi-integration-contract/conversation-drive.md#pic-50) (PIC-50): a synchronous throw from `pi.sendUserMessage` (the secondary mapping) maps to `Err(QueryError { kind: "transport", message: <error.message>, ... })`, not to `loom/runtime/internal-error`.
- [conversation-drive.md — error detection](../spec_topics/pi-integration-contract/conversation-drive.md#pic-51) (PIC-51): when `loomAbort.signal.aborted` is true the cancellation short-circuit synthesises `Err(QueryError { kind: "cancelled" })` and takes precedence over both the `stopReason: "error"` probe and the `Ok(string)` extraction — even when `waitForIdle()` resolved cleanly with no error written to session state.

**Deps.** `V9n-T`, `V9c`, `V9j`, `V17a`

**Ships when.** `npm test` drives a `stopReason: "error"` turn and observes the synthesised `kind: "transport"` `QueryError`; drives a turn with absent `errorMessage` and observes the `"provider transport failure"` fallback message; drives a synchronous `pi.sendUserMessage` throw and observes the secondary `kind: "transport"` mapping; and drives an aborted turn and observes the `kind: "cancelled"` short-circuit taking precedence over both the error probe and the `Ok(string)` extraction.
