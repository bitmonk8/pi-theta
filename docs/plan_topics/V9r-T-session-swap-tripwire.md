# `V9r-T` — Session-swap fail-fast tripwire (tests)

**Spec.** [`../spec_topics/pi-integration-contract/session-only-degraded-state.md`](../spec_topics/pi-integration-contract/session-only-degraded-state.md) (the *Session-swap fail-fast tripwire — governed-by-rebind resolution* section), [`../spec_topics/pi-integration-contract/host-prerequisites.md`](../spec_topics/pi-integration-contract/host-prerequisites.md#degraded-state-host-prerequisites) (clause (a), resolved `governed-by-rebind`), [`../spec_topics/pi-integration-contract/drain-state-contract.md`](../spec_topics/pi-integration-contract/drain-state-contract.md), [`../spec_topics/diagnostics/code-registry-host.md`](../spec_topics/diagnostics/code-registry-host.md) (the `loom/host/session-swap-instance-survived` row).

**Adds.** The failing test suite for the session-swap fail-fast tripwire that replaces the retired session-only degraded-state branch. Under the recorded `governed-by-rebind` determination the session-only reasons (`"new"`/`"resume"`/`"fork"`) run the same full teardown as `"quit"`/`"reload"` — **no** `markRuntimeDegraded` write, **no** `"degraded-needs-reload"` transition, **no** `loom/host/session-shutdown-runtime-degraded` emission, **no** degraded slash note — and the runtime additionally arms a private per-extension-instance tripwire on the closed-over `LoomRegistry`; every loom-registered slash `handler` (at entry) and the `session_start` handler assert on it and, if it is ever observed set, emit one `loom/host/session-swap-instance-survived` (E, runtime) diagnostic and fail-fast-terminate the process (the NFR-2.1 "let crash" path).

**Tests.**
- `cka-27` (`session-only-degraded-state.md` — *Session-swap fail-fast tripwire* section): on `"new"`/`"resume"`/`"fork"` the handler runs the full five-sub-step teardown and arms the private `sessionSwapTornDown` tripwire on the closed-over `LoomRegistry`; it writes no drain-state degraded tag, performs no `markRuntimeDegraded` transition, and emits no `loom/host/session-shutdown-runtime-degraded` row.
- `cka-27` (tripwire fires): a loom-registered slash `handler` (at entry) — and a `session_start` handler — that observes the tripwire armed emits exactly one `loom/host/session-swap-instance-survived` row and then terminates the process via the fail-fast path; the guard is a no-op (dormant) when the tripwire is unset (the proven governed-by-rebind steady state — the disposed instance never dispatches, a fresh instance carries an unarmed registry).
- `cka-27` (no false positive): under the proven rebind (fresh instance per swap) no slash dispatch or `session_start` ever observes an armed tripwire, so the guard never fires; arming is idempotent under multi-`session_shutdown` delivery to one instance, and the Pi-owned `/reload` command (not a loom-registered handler) is not guarded.
- `loom/host/session-swap-instance-survived` ([code-registry-host.md](../spec_topics/diagnostics/code-registry-host.md)): the diagnostic is emitted via `console.error` exactly once at the tripwire-trip site, carries `details: { event: { reason: "new" | "resume" | "fork" } }` (the armed reason), and its emission is immediately followed by process termination.
