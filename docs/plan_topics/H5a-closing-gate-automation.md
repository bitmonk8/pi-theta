# `H5a` — REQ-ID / diagnostic-code / un-anchored-obligation closing-gate automation

**Convention.** [`conventions.md`](./conventions.md) — *REQ-ID discipline*, *Diagnostic message anchors*, and the [`coverage-matrix.md`](./coverage-matrix.md) closure obligation.

**Adds.** The CI gate that reconciles the spec REQ-ID set and the diagnostics code registry against the plan's [`coverage-matrix.md`](./coverage-matrix.md) and the asserting tests — failing on an unmapped executable REQ-ID, a coverage-matrix-mapped numbered REQ-ID with no citing test in the test corpus, a registry code with no asserting test, an asserted code absent from the registry, an un-anchored normative MUST/MUST-NOT (one carrying no numbered `PREFIX-N` REQ-ID and no `loom/...` registry code; the gate does not mechanically exclude named cross-leaf seam obligations, so a seam-contract MUST surfaces as a candidate and is dispositioned via a [`coverage-matrix.md`](./coverage-matrix.md) closing leaf or the [`governance.md` GOV-15](../spec_topics/governance/source-language-stability.md#gov-15) reviewer-inspection residue) absent from [`coverage-matrix.md`](./coverage-matrix.md)'s *Code-keyed obligation areas (no numbered REQ-IDs)* table with a closing leaf, a retired/live ID clash, or a per-prefix numbering hole. The `no-broad-catch` lint allow-list reconciliation arm is owned by [`H5c`](./H5c-broad-catch-allow-list-gate.md). The standing **transitive-completeness** plan-structural arm — which reconciles each [`coverage-matrix.md`](./coverage-matrix.md) closing-leaf cell against [`H5b`](./H5b-warn-only-canary.md)'s `Deps.` — is owned by [`H5d`](./H5d-transitive-completeness-gate.md). At `H5a` the gate's pass/fail is evaluated against the seeded fixtures below; the live-corpus unmapped-executable-REQ-ID failure mode — the live spec REQ-ID set reconciled against the live [`coverage-matrix.md`](./coverage-matrix.md) — does not gate `npm test` at this leaf and first becomes binding at the loom 1.0 release gate activated by [`H6a`](./H6a-live-corpus-activation.md), per [`conventions.md`](./conventions.md) *REQ-ID discipline*. The live-corpus un-anchored-MUST failure mode — the live `spec_topics/**` normative MUST/MUST-NOT set (the recogniser scans the non-narrative `spec_topics/**` pages, those whose [REQ-ID prefix table](../spec_topics/governance/req-id-prefix-table-active-a.md) row is not the byte-exact `(no IDs — narrative)` cell per [`governance.md` GOV-3](../spec_topics/governance/req-id-prefix-table-active-a.md#gov-3), and additionally reddens on any `spec_topics/**` page absent from that table altogether — other than a [GOV-24](../spec_topics/governance/req-id-prefix-table-active-b.md#gov-24) table-of-contents hub stub — as an un-rowed-page residue defect, per [`conventions.md`](./conventions.md) *REQ-ID discipline*) reconciled against the same table — shares that posture: it is exercised against the seeded fixtures here and first becomes binding at the same [`H6a`](./H6a-live-corpus-activation.md) release-gate activation. The live-corpus numbered-REQ-ID citing-test failure mode — the live [`coverage-matrix.md`](./coverage-matrix.md) numbered-REQ-ID mappings reconciled against the citing tests in the live test corpus — shares that posture too: it is exercised against the seeded fixtures here and first becomes binding at the same [`H6a`](./H6a-live-corpus-activation.md) release-gate activation. The gate machinery also exposes a **warn-only live-corpus mode** that runs the same three live-corpus reconciliation surfaces but emits each gap as a CI finding without failing the build; the [`H5b`](./H5b-warn-only-canary.md) canary consumes this mode to pre-flight live coverage immediately before [`H6a`](./H6a-live-corpus-activation.md)'s hard-fail flip. The seeded gate fixtures — the un-enumerated-MUST fixture page, the `<new>`-placeholder-MUST fixture page (an un-anchored MUST mapped only to a `<new>` coverage-matrix row naming no real closing leaf), the fixture spec REQ-ID with no [`coverage-matrix.md`](./coverage-matrix.md) row, the fixture coverage-matrix mapping whose numbered REQ-ID has no citing test, and the un-rowed-page fixture (a non-hub-stub `spec_topics/**`-shaped page absent from the fixture prefix table) — are all permanent by design (they exist to fail every gate arm) and so reside under a dedicated test-fixtures root outside `docs/spec_topics/**` and outside the live test corpus; the gate's live-mode path selection reconciles the live spec/test corpus exclusive of that fixtures root, so no seeded fixture is ever scanned as live coverage. This exclusion covers all five seeded fixture kinds, not only the spec-page MUST fixture. The `H5c` broad-catch allow-list arm places its own seeded `// allow-broad-catch:` violation fixtures under the same fixtures root.

**Tests.**
- `Convention:` (*REQ-ID discipline*) the gate fails when a fixture spec REQ-ID has no coverage-matrix row.
- `Convention:` (*Diagnostic message anchors*) the gate fails when a test asserts a diagnostic code absent from the registry, and when a registry code has no asserting test.
- `Convention:` (*REQ-ID discipline*) the gate excludes the `loom/typecheck/*` build-time brand-string prefix from registry reconciliation.
- `Convention:` (*REQ-ID discipline* — un-anchored obligations) the gate fails when a fixture spec page carries a normative MUST/MUST-NOT with no `PREFIX-N` REQ-ID and no registry code that is absent from the [`coverage-matrix.md`](./coverage-matrix.md) *Code-keyed obligation areas (no numbered REQ-IDs)* table with a closing leaf, and passes when every such fixture MUST is enumerated with a closing leaf.
- `Convention:` (*REQ-ID discipline* — un-anchored obligations, `<new>` placeholder) the gate fails when a fixture spec page's un-anchored normative MUST/MUST-NOT maps only to a `<new>` placeholder coverage-matrix row whose closing-leaf column names no real leaf (treated identically to an un-enumerated MUST, mirroring the un-enumerated-MUST fixture above), and passes when that fixture MUST is enumerated with a real closing leaf; like the un-enumerated-MUST arm it is exercised against the seeded fixtures here and first becomes binding at the [`H6a`](./H6a-live-corpus-activation.md) release-gate activation.
- `Convention:` (*REQ-ID discipline* — citing tests) the gate fails when a fixture coverage-matrix-mapped numbered REQ-ID has no test citing that REQ-ID inline in the seeded test corpus, and passes when every mapped REQ-ID has a citing test; the recogniser is a best-effort inline-`PREFIX-N`-citation scan over the test sources and certifies citing-test existence, not the assertion's semantic fidelity.
- `Convention:` (*REQ-ID discipline* — un-rowed page) the gate reddens when a seeded non-hub-stub `spec_topics/**`-shaped fixture page is absent from the fixture [REQ-ID prefix table](../spec_topics/governance/req-id-prefix-table-active-a.md) altogether (an un-rowed-page residue defect), and stays green both when that fixture page carries a prefix-table row and when an un-rowed fixture page is a [GOV-24](../spec_topics/governance/req-id-prefix-table-active-b.md#gov-24) table-of-contents hub stub (hub stubs are excluded from the un-rowed-page defect).

**Deps.** `H4a`

**Ships when.** `npm test` runs the closing gate green against the seeded no-violation fixture — which includes a fixture page whose un-anchored normative MUST is enumerated in the *Code-keyed obligation areas* table with a closing leaf, and a seeded fixture coverage-matrix mapping whose numbered REQ-ID has a citing test in the seeded test corpus, and a seeded [GOV-24](../spec_topics/governance/req-id-prefix-table-active-b.md#gov-24) hub-stub fixture page that is itself absent from the fixture prefix table yet stays green because hub stubs are excluded from the un-rowed-page residue defect — and red against each seeded violation fixture, including a seeded un-enumerated-MUST fixture page that carries a `PREFIX-N`-less, registry-code-less normative MUST mapped to no closing leaf, a seeded `<new>`-placeholder-MUST fixture page whose un-anchored MUST maps only to a `<new>` placeholder coverage-matrix row naming no real closing leaf, a seeded fixture coverage-matrix mapping whose numbered REQ-ID has no citing test in the seeded test corpus, and a seeded un-rowed-page violation fixture (a non-hub-stub `spec_topics/**`-shaped page absent from the fixture prefix table) residing under the same dedicated test-fixtures root outside `docs/spec_topics/**` as the other seeded fixtures.
