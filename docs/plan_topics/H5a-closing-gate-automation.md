# `H5a` — REQ-ID / diagnostic-code closing-gate automation

**Convention.** [`conventions.md`](./conventions.md) — *REQ-ID discipline*, *Diagnostic message anchors*, and the [`coverage-matrix.md`](./coverage-matrix.md) closure obligation.

**Adds.** The CI gate that reconciles the spec REQ-ID set and the diagnostics code registry against the plan's [`coverage-matrix.md`](./coverage-matrix.md) and the asserting tests — failing on an unmapped executable REQ-ID, a coverage-matrix-mapped numbered REQ-ID with no citing test in the test corpus, a registry code with no asserting test, an asserted code absent from the registry, an un-anchored normative MUST/MUST-NOT (one carrying no numbered `PREFIX-N` REQ-ID and no `loom/...` registry code; the gate does not mechanically exclude named cross-leaf seam obligations, so a seam-contract MUST surfaces as a candidate and is dispositioned via a [`coverage-matrix.md`](./coverage-matrix.md) closing leaf or the [`governance.md` GOV-15](../spec_topics/governance/source-language-stability.md#gov-15) reviewer-inspection residue) absent from [`coverage-matrix.md`](./coverage-matrix.md)'s *Code-keyed obligation areas (no numbered REQ-IDs)* table with a closing leaf, a retired/live ID clash, or a per-prefix numbering hole. The `no-broad-catch` lint allow-list reconciliation arm is owned by [`H5c`](./H5c-broad-catch-allow-list-gate.md). The gate additionally runs a standing **transitive-completeness** plan-structural arm — like the per-prefix numbering-hole arm it reads only plan files (always present) and so is not deferred to the live-corpus flip: for every row in [`coverage-matrix.md`](./coverage-matrix.md)'s *Numbered REQ-IDs* and *Code-keyed obligation areas (no numbered REQ-IDs)* tables, the arm reads only that row's right-hand closing-leaf column — never the leaf-ID tokens embedded in the left-cell obligation-area prose — and requires AT LEAST ONE of the leaf IDs that closing-leaf cell lists to be a member of [`H5b`](./H5b-warn-only-canary.md)'s `Deps.` after expanding both sides' contiguous within-group `<group><letter>` ranges by letter suffix — a range whose two endpoints share the same `<group>` number, enumerated over its contiguous letter suffixes (e.g. `V2a–V2d` → `V2a, V2b, V2c, V2d`) — and `H5b`'s named singletons (`H1a`, `H5a`, `M`); a cell none of whose listed leaves is in that expanded set is a CI failure. A single-leaf cell is the degenerate case — its one leaf must be present — while a multi-leaf (primary + co-witness) cell such as the *Numbered REQ-IDs* row `CIO-5 | V16a, H7a` stays green as long as one listed leaf (here `V16a`) is reachable, so a co-witness leaf intentionally excluded from `H5b`'s `Deps.` (e.g. `H7a`, which closes no new spec REQ-ID and is pinned directly on [`H6a`](./H6a-live-corpus-activation.md)) does not redden the arm. The arm excludes the `<new>` placeholder rows (whose closing-leaf column lists no real leaf yet), so it mechanically backs the *Transitive-completeness plan-maintenance* obligation in [`conventions.md`](./conventions.md) *REQ-ID discipline*. At `H5a` the gate's pass/fail is evaluated against the seeded fixtures below; the live-corpus unmapped-executable-REQ-ID failure mode — the live spec REQ-ID set reconciled against the live [`coverage-matrix.md`](./coverage-matrix.md) — does not gate `npm test` at this leaf and first becomes binding at the loom 1.0 release gate activated by [`H6a`](./H6a-live-corpus-activation.md), per [`conventions.md`](./conventions.md) *REQ-ID discipline*. The live-corpus un-anchored-MUST failure mode — the live `spec_topics/**` normative MUST/MUST-NOT set (the recogniser scans the non-narrative `spec_topics/**` pages, those whose [REQ-ID prefix table](../spec_topics/governance/req-id-prefix-table-active-a.md) row is not the byte-exact `(no IDs — narrative)` cell per [`governance.md` GOV-3](../spec_topics/governance/req-id-prefix-table-active-a.md#gov-3), per [`conventions.md`](./conventions.md) *REQ-ID discipline*) reconciled against the same table — shares that posture: it is exercised against the seeded fixtures here and first becomes binding at the same [`H6a`](./H6a-live-corpus-activation.md) release-gate activation. The live-corpus numbered-REQ-ID citing-test failure mode — the live [`coverage-matrix.md`](./coverage-matrix.md) numbered-REQ-ID mappings reconciled against the citing tests in the live test corpus — shares that posture too: it is exercised against the seeded fixtures here and first becomes binding at the same [`H6a`](./H6a-live-corpus-activation.md) release-gate activation. The gate machinery also exposes a **warn-only live-corpus mode** that runs the same three live-corpus reconciliation surfaces but emits each gap as a CI finding without failing the build; the [`H5b`](./H5b-warn-only-canary.md) canary consumes this mode to pre-flight live coverage immediately before [`H6a`](./H6a-live-corpus-activation.md)'s hard-fail flip. The seeded gate fixtures — the un-enumerated-MUST fixture page, the fixture spec REQ-ID with no [`coverage-matrix.md`](./coverage-matrix.md) row, and the fixture coverage-matrix mapping whose numbered REQ-ID has no citing test — are all permanent by design (they exist to fail every gate arm) and so reside under a dedicated test-fixtures root outside `docs/spec_topics/**` and outside the live test corpus; the gate's live-mode path selection reconciles the live spec/test corpus exclusive of that fixtures root, so no seeded fixture is ever scanned as live coverage. This exclusion covers all three seeded fixture kinds, not only the spec-page MUST fixture. The `H5c` broad-catch allow-list arm places its own seeded `// allow-broad-catch:` violation fixtures under the same fixtures root.

**Tests.**
- `Convention:` (*REQ-ID discipline*) the gate fails when a fixture spec REQ-ID has no coverage-matrix row.
- `Convention:` (*Diagnostic message anchors*) the gate fails when a test asserts a diagnostic code absent from the registry, and when a registry code has no asserting test.
- `Convention:` (*REQ-ID discipline*) the gate excludes the `loom/typecheck/*` build-time brand-string prefix from registry reconciliation.
- `Convention:` (*REQ-ID discipline* — un-anchored obligations) the gate fails when a fixture spec page carries a normative MUST/MUST-NOT with no `PREFIX-N` REQ-ID and no registry code that is absent from the [`coverage-matrix.md`](./coverage-matrix.md) *Code-keyed obligation areas (no numbered REQ-IDs)* table with a closing leaf, and passes when every such fixture MUST is enumerated with a closing leaf.
- `Convention:` (*REQ-ID discipline* — citing tests) the gate fails when a fixture coverage-matrix-mapped numbered REQ-ID has no test citing that REQ-ID inline in the seeded test corpus, and passes when every mapped REQ-ID has a citing test; the recogniser is a best-effort inline-`PREFIX-N`-citation scan over the test sources and certifies citing-test existence, not the assertion's semantic fidelity.
- `Convention:` (*REQ-ID discipline* — transitive-completeness) the gate fails when a seeded `coverage-matrix.md` fixture has a closing-leaf cell none of whose listed leaves is present in a seeded `H5b`-`Deps.` fixture, and passes when every closing-leaf cell has at least one listed leaf present in the seeded `H5b`-`Deps.` fixture after contiguous-range and named-singleton expansion on both sides — including a seeded multi-leaf (primary + co-witness) closing-leaf cell where exactly one of the listed leaves is present.

**Deps.** `H4a`

**Ships when.** `npm test` runs the closing gate green against the seeded no-violation fixture — which includes a fixture page whose un-anchored normative MUST is enumerated in the *Code-keyed obligation areas* table with a closing leaf, and a seeded fixture coverage-matrix mapping whose numbered REQ-ID has a citing test in the seeded test corpus — and red against each seeded violation fixture, including a seeded un-enumerated-MUST fixture page that carries a `PREFIX-N`-less, registry-code-less normative MUST mapped to no closing leaf, a seeded fixture coverage-matrix mapping whose numbered REQ-ID has no citing test in the seeded test corpus, and a seeded `coverage-matrix.md` fixture with a closing-leaf cell none of whose listed leaves is present in the seeded `H5b`-`Deps.` fixture — while a seeded multi-leaf (primary + co-witness) closing-leaf cell with exactly one listed leaf present stays green.
