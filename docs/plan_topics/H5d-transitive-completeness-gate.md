# `H5d` — Transitive-completeness plan-structural closing-gate arm

**Convention.** [`conventions.md`](./conventions.md) — *REQ-ID discipline* (the *Transitive-completeness plan-maintenance* obligation) and the [`coverage-matrix.md`](./coverage-matrix.md) closure obligation.

**Adds.** The standing **transitive-completeness** plan-structural gate arm, run as part of the unified closing-gate machinery [`H5a`](./H5a-closing-gate-automation.md) establishes. Like the per-prefix numbering-hole arm it reads only plan files (always present) and so is not deferred to the live-corpus flip: for every row in [`coverage-matrix.md`](./coverage-matrix.md)'s *Numbered REQ-IDs* and *Code-keyed obligation areas (no numbered REQ-IDs)* tables, the arm reads only that row's right-hand closing-leaf column — never the leaf-ID tokens embedded in the left-cell obligation-area prose. It first tokenises that closing-leaf cell into its leaf-ID set by taking exactly the backtick-delimited code spans the cell contains — each `` `…` `` span is one leaf-ID token (or token range) — and ignoring all other cell text: the comma separators and the parenthetical annotation prose live rows interleave with their IDs. So `DISC-4`'s cell resolves to `{V10a, V9m}` with its `(collision-detection closure)` / `(superseded-entry-dispatch closure)` notes dropped; `CIO-1`'s resolves to `{V16a, V4e, V11f}`; the `PIC-17, PIC-18` row resolves to `{V9c, V15d}` with its `(prompt→prompt-invoke-path facet …)` note dropped; and the multi-leaf Code-keyed rows resolve to their backtick IDs with the surrounding annotation prose dropped. A `<new>` placeholder cell carries no backtick leaf-ID token, so its tokenised set is empty — consistent with the row exclusion stated below. It then requires AT LEAST ONE of the leaf IDs that closing-leaf cell lists to be a member of [`H5b`](./H5b-warn-only-canary.md)'s `Deps.` after expanding both sides' contiguous within-group `<group><letter>` ranges by letter suffix — a range whose two endpoints share the same `<group>` number, enumerated over its contiguous letter suffixes (e.g. `V2a–V2d` → `V2a, V2b, V2c, V2d`) — and `H5b`'s named singletons (`H1a`, `H5a`, `M`); a cell none of whose listed leaves is in that expanded set is a CI failure. A single-leaf cell is the degenerate case — its one leaf must be present — while a multi-leaf (primary + co-witness) cell such as the *Numbered REQ-IDs* row `CIO-5 | V16a, H7a` stays green as long as one listed leaf (here `V16a`) is reachable, so a co-witness leaf intentionally excluded from `H5b`'s `Deps.` (e.g. `H7a`, which closes no new spec REQ-ID and is pinned directly on [`H6a`](./H6a-live-corpus-activation.md)) does not redden the arm. The arm excludes the `<new>` placeholder rows (whose closing-leaf column lists no real leaf yet), so it mechanically backs the *Transitive-completeness plan-maintenance* obligation in [`conventions.md`](./conventions.md) *REQ-ID discipline*. Its seeded `coverage-matrix.md` and `H5b`-`Deps.` violation fixtures reside under the same dedicated test-fixtures root [`H5a`](./H5a-closing-gate-automation.md) places its seeded gate fixtures under (permanent by design, outside `docs/spec_topics/**` and outside the live test corpus); because the arm reads only ever-present plan files it has no live-corpus flip and is exercised against those seeded fixtures at every footing.

**Tests.**
- `Convention:` (*REQ-ID discipline* — transitive-completeness) the gate fails when a seeded `coverage-matrix.md` fixture has a closing-leaf cell none of whose listed leaves is present in a seeded `H5b`-`Deps.` fixture, and passes when every closing-leaf cell has at least one listed leaf present in the seeded `H5b`-`Deps.` fixture after cell tokenisation and contiguous-range and named-singleton expansion on both sides — including a seeded multi-leaf (primary + co-witness) closing-leaf cell where exactly one of the listed leaves is present, and a seeded closing-leaf cell carrying parenthetical annotation prose alongside its backtick ID(s) whose tokenisation resolves to exactly those backtick IDs (with the parenthetical prose dropped).

**Deps.** `H5a`

**Ships when.** `npm test` runs the transitive-completeness arm green against the seeded no-violation fixture — every closing-leaf cell has at least one listed leaf present in the seeded `H5b`-`Deps.` fixture, including a seeded multi-leaf (primary + co-witness) closing-leaf cell with exactly one listed leaf present and a seeded closing-leaf cell carrying parenthetical annotation prose alongside its backtick ID(s) whose tokenisation resolves to exactly those backtick IDs — and red against a seeded `coverage-matrix.md` fixture with a closing-leaf cell none of whose listed leaves is present in the seeded `H5b`-`Deps.` fixture.
