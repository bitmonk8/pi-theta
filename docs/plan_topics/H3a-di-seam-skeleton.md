# `H3a` — Dependency-injection seam skeleton

**Convention.** [`conventions.md`](./conventions.md) — *No globals, statics, singletons*.

**Adds.** The constructor-injection runtime root: a per-runtime object graph that threads the host seams (`Checkpoint`, `SchemaValidator`, `Clock`, `FileSystem`, `FileWatcher`, `TokenEstimator`, `IdSource`) as injected interfaces, with one instance per runtime and no ambient access. The ambient-access ban is mechanically checked by the architectural test below, which treats a direct reference to a banned primitive as legitimate only at an exempt ambient site — a site carrying a same-line `// allow-ambient: <primitive> — <seam>` comment, recognised by that comment alone (there is no separate allow-list artefact), per the *No globals, statics, singletons* rule in [`conventions.md`](./conventions.md). `H3a` declares each seam interface's full member signatures, sourced from the seam-contract pages the `V8*` leaves (`V8a`–`V8e`) also cite — [`host-interfaces-core.md`](../spec_topics/pi-integration-contract/host-interfaces-core.md) and [`host-interfaces-services.md`](../spec_topics/pi-integration-contract/host-interfaces-services.md) (the `Checkpoint` contract is anchored at [`host-interfaces-services.md#checkpoint-seam`](../spec_topics/pi-integration-contract/host-interfaces-services.md#checkpoint-seam), even though `V8a` lists `Checkpoint` under `host-interfaces-core.md`). These `H3a`-declared signatures are the single source of each interface's shape; the `V8*` leaves implement against them rather than redeclaring members. Seam *interfaces* only here — what the `V8*` leaves add is the seams' normative *behaviour*, not their shape.

**Tests.**
- `Convention:` (*No globals, statics, singletons*) constructing two runtime roots yields two isolated seam graphs sharing no mutable state.
- `Convention:` (*No globals, statics, singletons*) an architectural test asserts no `src/**` module *directly references* `process.env`, `process.cwd`, `crypto.randomUUID`, or the timing primitives the spec's WallClock ban enumerates — `Date.now`, `performance.now`, `Date.prototype.getTime`, `setTimeout`, `clearTimeout` (the enumerated set is sourced directly from the [`host-interfaces-services.md`](../spec_topics/pi-integration-contract/host-interfaces-services.md) `Clock`/`WallClock` ban so the scan and the spec stay in lockstep; the timing primitives' intended exempt site is the `WallClock` adapter, exempted by the same `// allow-ambient:` comment that exempts every other primitive at its owning adapter) — except at an **exempt ambient site** — a site carrying a same-line `// allow-ambient: <primitive> — <seam>` comment, recognised by that comment alone (there is no separate allow-list artefact), per the *No globals, statics, singletons* rule in [`conventions.md`](./conventions.md). No such site exists here yet; each is added in source as its seam adapter lands under the `V8*` leaves; the scan flags any direct reference to a listed primitive that is not at such a site. This identifier-keyed scan catches only direct references; indirect forms — aliased reads (`const env = process.env`), destructured reads (`const { cwd } = process`), computed access (`process["env"]`), and re-export indirection — are not mechanically detected; that residue is owned by the *Per-phase TDD ritual* self-review step in [`conventions.md`](./conventions.md).
